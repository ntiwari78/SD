Learning Probabilistic Models

The EM Algorithm and Probabilistic Inference

Section 1: The Feynman Mental Model

The Two Coin-Flippers Problem

Imagine you're in a room with two coin-flippers, but you can't see who flipped each coin — you only see the results (heads or tails). You want to figure out each person's bias: how likely they flip heads. The problem is a classic chicken-and-egg dilemma:

  • To estimate each person's bias, you need to know which flips they made
  • But to know which flips they made, you need to already know their biases

The Expectation-Maximization (EM) algorithm solves this by alternating:

  • E-step: Based on current bias estimates, guess which person likely made each flip
  • M-step: Using those guesses, update the bias estimates
  • Repeat until the estimates stabilize

Key Concepts in This Chapter

  • Statistical Learning: Learning probability distributions from data
  • Maximum Likelihood Estimation (MLE): Find parameters that make your data most likely
  • Hidden Variables: When some data is missing or unobserved
  • The EM Algorithm: Elegant solution for learning with incomplete data
  • Gaussian Mixture Models (GMMs): Soft clustering with probabilistic assignments
  • Naive Bayes: A simple but powerful probabilistic classifier
  • Bayesian Learning: Incorporating prior beliefs about parameters

Section 2: Mathematical Ascent

Maximum Likelihood Estimation

The core idea: given observations $X = \{x_1, x_2, ..., x_N\}$, find parameters $\theta$ that maximize the probability of observing this data.

Maximum Likelihood:

$$\theta^* = \arg\max_\theta \prod_{i=1}^N P(x_i | \theta)$$

In practice, we work with log-likelihood to avoid numerical underflow:

$$\ell(\theta) = \sum_{i=1}^N \log P(x_i | \theta)$$

The EM Algorithm

When you have hidden variables $Z$ (unobserved data), EM alternates between two steps:

E-Step (Expectation): Compute the expected log-likelihood of the complete data, given current parameters:

$$Q(\theta | \theta^{(t)}) = \mathbb{E}_{Z|X,\theta^{(t)}}[\log P(X,Z|\theta)]$$

M-Step (Maximization): Update parameters to maximize the expected log-likelihood:

$$\theta^{(t+1)} = \arg\max_\theta Q(\theta | \theta^{(t)})$$

EM for Gaussian Mixture Models

A GMM models data as a mixture of $K$ Gaussian distributions. Each data point "belongs" to each cluster with some probability (soft assignment).

Responsibilities (E-step): Probability that cluster $k$ generated point $i$:

$$r_{ik} = \frac{\pi_k \mathcal{N}(x_i | \mu_k, \Sigma_k)}{\sum_j \pi_j \mathcal{N}(x_i | \mu_j, \Sigma_j)}$$

Update Means (M-step):

$$\mu_k = \frac{\sum_i r_{ik} x_i}{\sum_i r_{ik}}$$

Update Mixing Weights (M-step):

$$\pi_k = \frac{1}{N}\sum_i r_{ik}$$

Naive Bayes Classifier

Naive Bayes assumes that all features are independent given the class label. Despite this "naive" assumption, it works remarkably well in practice.

$$P(C|x_1,...,x_n) \propto P(C) \prod_i P(x_i|C)$$

To classify, we compute this probability for each class and pick the one with highest score.

Bayesian Learning: Adding Priors

When you have prior beliefs about parameters (especially with little data), use Maximum A Posteriori (MAP) estimation:

$$\theta^* = \arg\max_\theta P(\theta|D) = \arg\max_\theta P(D|\theta)P(\theta)$$

The prior $P(\theta)$ acts as a regularizer, pulling the estimate toward prior beliefs when data is scarce.

Section 3: Clean-Code Implementation

Here are production-ready implementations of EM for Gaussian Mixture Models and Naive Bayes classification. These demonstrate the core algorithms clearly.

Gaussian Mixture Model with EM

import math
import random

class GaussianMixtureEM:
    """
    EM algorithm for Gaussian Mixture Models.
    The rational agent learns to cluster data into groups
    when the group memberships are hidden (unobserved).
    """
    def __init__(self, k=2, max_iter=100, tol=1e-6):
        self.k = k  # Number of clusters
        self.max_iter = max_iter
        self.tol = tol  # Convergence tolerance

    def fit(self, data):
        """Fit the GMM to data using EM."""
        n = len(data)
        # Initialize: random means, unit variance, equal weights
        self.means = [data[random.randint(0, n-1)] for _ in range(self.k)]
        self.variances = [1.0] * self.k
        self.weights = [1.0/self.k] * self.k

        prev_ll = float('-inf')
        for iteration in range(self.max_iter):
            # E-step: compute responsibilities
            responsibilities = self._e_step(data)

            # M-step: update parameters
            self._m_step(data, responsibilities)

            # Check convergence
            ll = self.log_likelihood(data)
            if abs(ll - prev_ll) < self.tol:
                print(f"Converged at iteration {iteration}")
                break
            prev_ll = ll

        return self

    def _e_step(self, data):
        """Compute soft assignment of each point to each cluster."""
        resp = []
        for x in data:
            # Compute responsibility for each cluster
            r = [self.weights[k] * self._gaussian(x, self.means[k],
                                                   self.variances[k])
                 for k in range(self.k)]
            # Normalize to sum to 1
            total = sum(r) + 1e-300
            resp.append([ri/total for ri in r])
        return resp

    def _m_step(self, data, resp):
        """Update parameters using weighted data."""
        n = len(data)
        for k in range(self.k):
            # Total responsibility for cluster k
            nk = sum(resp[i][k] for i in range(n))
            if nk < 1e-10:
                continue

            # Update mean: weighted average
            self.means[k] = sum(resp[i][k]*data[i] for i in range(n)) / nk

            # Update variance: weighted variance
            self.variances[k] = sum(resp[i][k]*(data[i]-self.means[k])**2
                                    for i in range(n)) / nk + 1e-6

            # Update mixing weight
            self.weights[k] = nk / n

    def _gaussian(self, x, mu, var):
        """Evaluate Gaussian PDF at x."""
        return (1/math.sqrt(2*math.pi*var)) * math.exp(-0.5*(x-mu)**2/var)

    def log_likelihood(self, data):
        """Compute log-likelihood of data."""
        ll = 0
        for x in data:
            prob = sum(self.weights[k]*self._gaussian(x, self.means[k],
                                                       self.variances[k])
                      for k in range(self.k)) + 1e-300
            ll += math.log(prob)
        return ll

    def predict(self, x):
        """Assign x to the most likely cluster."""
        probs = [self.weights[k]*self._gaussian(x, self.means[k],
                                                 self.variances[k])
                for k in range(self.k)]
        return probs.index(max(probs))

    def soft_predict(self, x):
        """Return soft responsibilities for x."""
        r = [self.weights[k]*self._gaussian(x, self.means[k],
                                            self.variances[k])
            for k in range(self.k)]
        total = sum(r) + 1e-300
        return [ri/total for ri in r]

Naive Bayes Classifier

class NaiveBayes:
    """
    Naive Bayes classifier: assumes features are independent given the class.
    Despite this 'naive' assumption, it works surprisingly well in practice,
    especially for text classification and spam detection.
    """
    def __init__(self):
        self.class_priors = {}
        self.feature_probs = {}

    def fit(self, X, y):
        """
        Train Naive Bayes on feature matrix X and labels y.
        X: list of lists (each inner list is feature values for one sample)
        y: list of class labels
        """
        from collections import Counter

        # Count class frequencies
        classes = Counter(y)
        n = len(y)
        self.class_priors = {c: count/n for c, count in classes.items()}

        # For each class, learn feature probabilities
        self.feature_probs = {}
        for c in classes:
            indices = [i for i, label in enumerate(y) if label == c]
            self.feature_probs[c] = {}

            # For each feature position
            for j in range(len(X[0])):
                vals = Counter(X[i][j] for i in indices)
                total = len(indices)

                # Laplace smoothing: add 1 to numerator and 2 to denominator
                self.feature_probs[c][j] = {
                    v: (count+1)/(total+2)
                    for v, count in vals.items()
                }

    def predict(self, x):
        """
        Classify a single sample x.
        Returns the most likely class.
        """
        scores = {}
        for c in self.class_priors:
            # Start with log of class prior
            score = math.log(self.class_priors[c])

            # Add log of each feature probability
            for j, val in enumerate(x):
                prob = self.feature_probs[c][j].get(val,
                    1/(len(self.feature_probs[c][j])+2))
                score += math.log(prob)

            scores[c] = score

        return max(scores, key=scores.get)

    def predict_proba(self, x):
        """
        Return probability distribution over classes.
        """
        scores = {}
        for c in self.class_priors:
            score = math.log(self.class_priors[c])
            for j, val in enumerate(x):
                prob = self.feature_probs[c][j].get(val,
                    1/(len(self.feature_probs[c][j])+2))
                score += math.log(prob)
            scores[c] = score

        # Convert log-scores to probabilities
        max_score = max(scores.values())
        exp_scores = {c: math.exp(score - max_score)
                     for c, score in scores.items()}
        total = sum(exp_scores.values())
        return {c: exp_scores[c]/total for c in exp_scores}

Section 4: Socratic Sandbox

Scenario: EM Instability

A data scientist runs EM on customer purchase data to find 3 customer segments. After 50 iterations, EM converges. But running it again with different random initialization gives completely different clusters.

Reflect and Answer:

  1. Local Optima: Why do different initializations give different results? What property of EM causes this, and what does it mean about the likelihood surface?
    Hint

    EM is a local optimization algorithm. It guarantees the likelihood never decreases, but not that you find the global maximum. The likelihood surface has many "peaks" (local optima).

  2. Mitigation Strategies: EM guarantees log-likelihood never decreases, but it doesn't guarantee finding the GLOBAL maximum. How can you mitigate this in practice?
    Hint

    Try multiple random initializations and pick the result with highest final likelihood. Use "warm start" from k-means. Try different numbers of clusters and use model selection criteria like BIC.

  3. Soft vs Hard Assignment: In the E-step, each customer gets a "soft assignment" to each segment (e.g., 0.7 to segment A, 0.2 to B, 0.1 to C). How is this different from k-means clustering? Which is more informative?
    Hint

    K-means uses hard assignments (each point assigned to one cluster). GMM/EM uses soft assignments. Soft assignments are more informative: they tell you the uncertainty. A customer with (0.5, 0.5, 0) is on the boundary between two segments.

  4. Why Naive Bayes Works: A Naive Bayes spam classifier treats "Nigerian" and "Prince" as independent features given the class. In reality, they're correlated in spam. Why does Naive Bayes still work well despite this wrong assumption?
    Hint

    Naive Bayes doesn't need to be perfect—it just needs the rank order of classes to be correct for classification. It's robust to the independence assumption violations. The independence assumption actually helps avoid overfitting!

  5. Priors and Regularization: If you have very little data, MLE overfits. MAP adds a prior. Give an intuitive example of how a prior "regularizes" the coin-flip estimation problem.
    Hint

    If you flip a coin twice and get 2 heads, MLE says P(heads)=1 (the coin is always heads). But a Beta prior (centered at 0.5) pulls the estimate closer to 0.5, giving a more reasonable P(heads)≈0.67. With more flips, the prior matters less.
