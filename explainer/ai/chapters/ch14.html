    Chapter 14: Probabilistic Reasoning over Time

    Tracking hidden state through noisy observations using Hidden Markov Models, Kalman Filters, and temporal inference.

    Feynman Mental Model: Tracking a Robot

    Imagine you're tracking a friend's location using only noisy GPS readings. Each reading is slightly off — maybe by 10 meters in any direction. But you know two things:

    • Prior belief: Your friend was somewhere before (you have prior knowledge)
    • Physics: Your friend moves at walking speed (transition model)
    • Noisy sensor: GPS reading ± 10 meters (sensor model)

    By combining these three pieces of information, you can estimate a much better position than any single GPS reading. This is what Hidden Markov Models and Kalman Filters do — they separate signal from noise by reasoning about how the world changes over time.

    The core insight: The future doesn't depend on the entire history — only on the present state. This is the Markov assumption. If you know where your friend is right now (Xt), their location 5 steps from now depends only on Xt, not on where they were 20 minutes ago.

    Key Concepts in Temporal Models

    Time slices: We represent the world at discrete time points t = 0, 1, 2, ... Each time slice has variables (position, velocity, etc.) that evolve according to rules.

    Hidden state (Xt): The true position of the robot — what we want to infer but can't observe directly.

    Observable evidence (Et): The noisy sensor readings. We observe E1, E2, ..., Et and want to infer Xt.

    Transition model P(Xt | Xt-1): Describes how the hidden state evolves. For a robot, this might say: "If at position 5, you're 80% likely to move to position 6, 20% likely to stay at 5."

    Sensor model P(Et | Xt): Describes how observations depend on true state. "If you're actually at position 5, the sensor reads 5 with 80% probability, reads 4 or 6 with 10% each."

    Four Inference Tasks

    • Filtering: What is Xt right now, given observations so far? (P(Xt | e1:t)) — Used in real-time applications like GPS tracking.
    • Prediction: What will Xt+k be? (P(Xt+k | e1:t)) — Weather forecasts, traffic prediction.
    • Smoothing: What was Xk in the past, given all observations? (P(Xk | e1:t) where k < t) — Movie stabilization, medical imaging.
    • Most likely sequence (Viterbi): What is the most probable sequence of hidden states x1:t that explains the observations? — Speech recognition, sequence alignment.

    Discrete vs. Continuous: HMM to Kalman Filter

    Hidden Markov Model (HMM): Discrete hidden state space (5 positions in a hallway) and discrete time steps. Exact inference using matrix algebra.

    Kalman Filter: Continuous hidden state space (position is a real number) and Gaussian noise. Optimal solution when everything is linear-Gaussian. The Kalman Filter maintains mean and covariance of the belief state.

    Mathematical Ascent

    The Markov Property

    $$P(X_t | X_{0:t-1}) = P(X_t | X_{t-1})$$

    The future is conditionally independent of the past given the present. This assumption allows us to avoid tracking the entire history.

    Sensor Independence

    $$P(E_t | X_{0:t}, E_{0:t-1}) = P(E_t | X_t)$$

    Observations depend only on the current hidden state, not on history or previous observations.

    Forward Algorithm (Filtering)

    To compute P(Xt | e1:t), we use a forward recursion that alternates between prediction and update:

    $$P(X_t | e_{1:t}) = \alpha \cdot P(e_t | X_t) \sum_{x_{t-1}} P(X_t | x_{t-1}) P(x_{t-1} | e_{1:t-1})$$

    Breaking this down:

    • The sum averages over all possible previous states (prediction)
    • P(Xt | xt-1) comes from the transition model
    • P(et | Xt) is the sensor model, reweighting by the new observation (update)
    • α is a normalizing constant

    In matrix form:

    $$\mathbf{f}_{1:t} = \alpha \cdot \mathbf{O}_t \mathbf{T}^T \mathbf{f}_{1:t-1}$$

    Where f is the belief state vector, T is the transition matrix, and Ot is the diagonal matrix of sensor likelihoods.

    Forward-Backward (Smoothing)

    To compute P(Xk | e1:t) where k < t, we combine forward and backward messages:

    $$P(X_k | e_{1:t}) = \alpha \cdot \mathbf{f}_{1:k} \times \mathbf{b}_{k+1:t}$$

    Forward message: belief built from observations 1 to k
    Backward message: likelihood of observations k+1 to t given each state at k

    Backward recursion:

    $$\mathbf{b}_{k+1:t} = \mathbf{T} \mathbf{O}_{k+1} \mathbf{b}_{k+2:t}$$

    Viterbi Algorithm (Most Likely Sequence)

    Find the sequence of states x1:t that maximizes P(x1:t | e1:t):

    $$m_{1:t}(x_t) = P(e_t | x_t) \max_{x_{t-1}} \left( P(x_t | x_{t-1}) \cdot m_{1:t-1}(x_{t-1}) \right)$$

    We compute the max probability of reaching each state, then backtrack to recover the path. This is dynamic programming on the trellis of states × time.

    Kalman Filter for Continuous State

    For systems with Gaussian noise and linear dynamics:

    $$\hat{x}_t = F \hat{x}_{t-1} + K_t(z_t - HF\hat{x}_{t-1})$$

    Where:

    • F: State transition matrix (how does state evolve?)
    • H: Observation matrix (how is state observed?)
    • Kt: Kalman gain — how much to trust the new observation
    • zt: Actual observation
    • HF x̂t-1: Predicted observation based on prior belief

    The Kalman gain adapts: if sensor noise is low (K large), trust the observation; if process noise is low (K small), trust the prediction.

    Clean-Code Implementation

    Hidden Markov Model in Python

    import numpy as np

class HiddenMarkovModel:
    """
    Hidden Markov Model for temporal reasoning.
    The rational agent maintains a BELIEF STATE that evolves over time,
    combining predictions from a transition model with observations
    from a sensor model.
    """
    def __init__(self, transition, sensor, prior):
        self.T = np.array(transition)  # Transition matrix P(X_t | X_{t-1})
        self.sensor = sensor            # dict: observation -> likelihoods
        self.prior = np.array(prior)    # Initial belief state

    def forward(self, observations):
        """
        Filtering: compute P(X_t | e_{1:t}) for each time step.
        Like updating your GPS estimate with each new reading.

        Returns: list of belief states over time
        """
        f = self.prior.copy()
        beliefs = [f.copy()]

        for obs in observations:
            # PREDICT: where might the robot be now?
            f = self.T.T @ f

            # UPDATE: incorporate sensor reading
            # Multiply by likelihood of observation in each state
            f = f * np.array(self.sensor[obs])

            # NORMALIZE: ensure probabilities sum to 1
            f = f / (f.sum() + 1e-10)
            beliefs.append(f.copy())

        return beliefs

    def viterbi(self, observations):
        """
        Find most likely sequence of hidden states.
        Like reconstructing the actual path your friend walked,
        given all the noisy GPS readings.

        Returns: array of state indices, one per time step
        """
        n_states = len(self.prior)
        T = len(observations)

        # Track max log-probabilities and back pointers
        delta = np.log(self.prior + 1e-10)
        psi = np.zeros((T, n_states), dtype=int)

        for t, obs in enumerate(observations):
            new_delta = np.zeros(n_states)
            likelihoods = np.array(self.sensor[obs])

            for j in range(n_states):
                # For each state j at time t, find the best
                # previous state that led to it
                candidates = delta + np.log(self.T[:, j] + 1e-10)
                psi[t, j] = np.argmax(candidates)
                new_delta[j] = candidates[psi[t, j]] + np.log(likelihoods[j] + 1e-10)

            delta = new_delta

        # Backtrack from final state to recover path
        path = np.zeros(T, dtype=int)
        path[-1] = np.argmax(delta)
        for t in range(T - 2, -1, -1):
            path[t] = psi[t + 1, path[t + 1]]

        return path

    def forward_backward(self, observations):
        """
        Smoothing: compute P(X_k | e_{1:t}) for all k <= t.
        Uses forward and backward messages.
        """
        n_states = len(self.prior)
        T = len(observations)

        # Forward pass
        forward_msgs = []
        f = self.prior.copy()
        forward_msgs.append(f.copy())

        for obs in observations:
            f = self.T.T @ f
            f = f * np.array(self.sensor[obs])
            f = f / (f.sum() + 1e-10)
            forward_msgs.append(f.copy())

        # Backward pass
        backward_msgs = [np.ones(n_states)]
        b = np.ones(n_states)

        for t in range(T - 1, -1, -1):
            obs = observations[t]
            likelihoods = np.array(self.sensor[obs])
            b = self.T @ (likelihoods * b)
            b = b / (b.sum() + 1e-10)
            backward_msgs.append(b.copy())

        backward_msgs.reverse()

        # Combine forward and backward
        smoothed = []
        for f, b in zip(forward_msgs[:-1], backward_msgs[1:]):
            s = f * b
            s = s / (s.sum() + 1e-10)
            smoothed.append(s)

        return smoothed


class KalmanFilter:
    """
    Kalman Filter for continuous state estimation.
    The rational agent models position and velocity as Gaussians,
    achieving optimal estimation for linear-Gaussian systems.
    """
    def __init__(self, F, H, Q, R, x0, P0):
        self.F = np.array(F)   # State transition matrix
        self.H = np.array(H)   # Observation matrix
        self.Q = np.array(Q)   # Process noise covariance
        self.R = np.array(R)   # Observation noise covariance
        self.x = np.array(x0)  # State estimate
        self.P = np.array(P0)  # State covariance

    def predict(self):
        """
        Prediction step: move forward in time.
        x_pred = F @ x_prev
        P_pred = F @ P_prev @ F.T + Q
        """
        self.x = self.F @ self.x
        self.P = self.F @ self.P @ self.F.T + self.Q
        return self.x.copy(), self.P.copy()

    def update(self, z):
        """
        Update step: incorporate new observation.
        Returns new state estimate and Kalman gain.
        """
        z = np.array(z).reshape(-1, 1)

        # Kalman gain: how much to trust the observation
        S = self.H @ self.P @ self.H.T + self.R
        K = self.P @ self.H.T @ np.linalg.inv(S)

        # Update state estimate
        y = z - self.H @ self.x.reshape(-1, 1)
        self.x = self.x + (K @ y).flatten()

        # Update covariance
        I = np.eye(len(self.x))
        self.P = (I - K @ self.H) @ self.P

        return self.x.copy(), K

    def filter(self, observations):
        """
        Run filter over a sequence of observations.
        Returns list of state estimates.
        """
        estimates = [self.x.copy()]
        for z in observations:
            self.predict()
            self.update(z)
            estimates.append(self.x.copy())
        return estimates

    Socratic Sandbox

    Scenario: The Hallway Robot

    A robot navigates a hallway with 5 positions arranged left-to-right. The hallway has a repeating pattern: wall, wall, open, open, wall. The robot's sensor detects "wall" or "open" at its current position, but with 20% error rate. The transition model says the robot moves forward one step with 80% probability and stays put with 20% probability. After observing the sequence ['wall', 'open', 'wall'], the forward filter converges to a nearly uniform belief — the filter can't tell where the robot is!

    1. Why did the filter converge to near-uniform belief? What property of this specific hallway layout might cause the filter to be unable to disambiguate? (Hint: the wall/open/wall pattern repeats.)
    2. If the robot's sensor had 5% error instead of 20%, how would the belief evolution differ? What does this tell us about the relationship between sensor accuracy and our ability to localize?
    3. The Markov assumption says the future depends only on the present. Give a real-world example where this assumption fails. (Hint: think about a person's motion.) How would you extend the model to handle this?
    4. How does Viterbi differ from filtering? If filtering says "probably position 3 at time t" but Viterbi says "position 2 at time t," which is correct? What different questions do they answer?
    5. A Kalman Filter assumes Gaussian noise. What happens if a sensor occasionally gives wildly wrong readings—e.g., a "heavy-tailed" distribution with occasional outliers? How does this break the optimality guarantee? What robust alternative exists?

    Discussion Points

    On uniform beliefs: When the environment is symmetric and observations are weak (noisy), the filter genuinely cannot distinguish between multiple states. This is an inherent limitation, not a flaw. With stronger sensors or more informative observations, the belief sharpens.

    On sensor accuracy: Sensor quality is exponentially valuable. A 5% error sensor provides much more information than a 20% error sensor, allowing the filter to narrow beliefs faster. In practice, sensor fusion (combining multiple sensors) is often more cost-effective than improving one sensor.

    On the Markov assumption: Real agents often have momentum, habits, or long-term goals. A person's next position depends not just on current position, but on where they were 5 steps ago. To fix this, expand the state to include velocity or abstract goals. The cost is a larger state space, but the model becomes more accurate.

    On Viterbi vs. filtering: Filtering asks "what is the most probable current state?" Viterbi asks "what is the most probable entire sequence?" For time t, they can give different answers because Viterbi commits to a consistent path, while filtering considers each time independently. Viterbi is useful when the path matters (speech recognition), filtering when only the current state matters (GPS tracking).

    On robust filtering: Outliers violate Gaussian assumptions. Robust alternatives include particle filters (samples states from a distribution, resamples based on observations) and Huber filtering (blends least-squares with robust loss). These trade optimality for resilience.
