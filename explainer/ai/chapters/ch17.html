Feynman Mental Model: The Board Game Analogy

Imagine you're playing a board game where every turn you roll dice and move. Unlike chess (where you plan one fixed sequence of moves), here you need a STRATEGY — a rule that tells you what to do in every possible situation you might find yourself in. That's a policy.

A Markov Decision Process (MDP) is the mathematical framework for finding the best strategy when outcomes are uncertain and you make decisions over time. Chapter 16 taught you how to make single decisions. This chapter teaches you how to make sequences of decisions where:

  • The world is uncertain (outcomes depend on chance)
  • Your decisions today affect your options tomorrow
  • You want to maximize total reward over time
The Key Insight: Instead of computing "what should I do now?", we compute "what should I do in EVERY situation?" This is the policy π(s) → a: given a state s, it prescribes an action a. Once you have the policy, decisions become automatic.

From Single Decisions to Sequential Decisions

In Chapter 16, we solved decision problems with a single choice: should you accept this medical test? Should you buy this stock? These are one-off decisions. But real life is sequential:

  • Robot navigation: At each cell, move closer to the goal — but take into account that the move might fail, and you'll face decisions again tomorrow.
  • Portfolio management: Allocate funds today, get returns next month, then re-allocate based on new information.
  • Game playing: Move a piece, opponent responds, you move again. Each move affects future possibilities.

In all these cases, the rational agent must look ahead and ask: "If I take action A now, I'll be in state S' next. From S', what's the best I can do?" This recursion is the heart of MDPs.

Components of an MDP

Every MDP has five components:

  • States (S): The set of all possible situations the agent can be in. In robot navigation, states are grid cells.
  • Actions (A): The moves available to the agent. In navigation: Up, Down, Left, Right.
  • Transition Model T(s, a, s'): The probability of ending up in state s' if you take action a in state s. In a stochastic world, the same action from the same state can lead to different outcomes.
  • Reward R(s): The immediate reward you get for being in state s. This could be +1 for reaching a goal, -1 for falling into a pit, or -0.04 for each step taken.
  • Discount Factor γ (gamma): A number between 0 and 1 that says "a reward now is worth more than the same reward later." For example, γ=0.9 means a $1 reward next year is worth $0.90 today.

The Policy and Utility

A policy π(s) is a function that maps each state to an action. It's the agent's complete strategy: "if you're in state s, do action π(s)." Given a policy, the agent's behavior is deterministic — no more deliberation needed.

The utility U(s) is the expected total discounted reward the agent will receive starting from state s and following the optimal policy. High-utility states are good to be in; low-utility states are bad.

Why Discount Future Rewards?

Why is γ less than 1? Several reasons:

  • Time value of money: $1 today is worth more than $1 tomorrow because you can invest it and earn returns.
  • Uncertainty: The farther in the future, the less certain we are about outcomes.
  • Impatience: Humans naturally prefer sooner rewards.
  • Mathematical convenience: Discounting ensures that the total reward doesn't become infinite in infinite-horizon problems.

With γ close to 1 (like 0.99), the agent thinks long-term and is willing to take short-term pain for long-term gain. With γ close to 0 (like 0.1), the agent is myopic — it mostly cares about immediate rewards.

The Bellman Equation: Recursion Made Concrete

The Bellman equation is the recursive definition of utility. It says:

U(s) = R(s) + γ maxa Σs' T(s, a, s') U(s')

Breaking this down:

  • R(s): The immediate reward for being in state s.
  • maxa: We choose the action that maximizes expected future reward.
  • Σs' T(s, a, s') U(s'): The expected utility of the resulting state, weighted by transition probabilities.
  • γ: Discounts future rewards relative to immediate reward.

This equation is the foundation of dynamic programming algorithms for MDPs. The key insight: the utility of a state depends on immediate reward plus the discounted utilities of possible next states. This is why it's called "dynamic programming" — we break the big problem into subproblems (states) and build solutions recursively.

From Utility to Policy

Once we know the utility of each state, finding the optimal policy is easy:

π*(s) = arg maxa Σs' T(s, a, s') U(s')

For each state, pick the action that leads to the highest expected utility. That's the optimal policy.

Mathematical Ascent: Formalizing MDPs

Now we climb into rigorous mathematics. We'll formalize MDPs and derive the algorithms that solve them.

The MDP Tuple

Formally, an MDP is a tuple:

(S, A, T, R, γ)

Where:

  • S: A finite set of states
  • A: A finite set of actions
  • T(s, a, s'): Transition function — probability of transitioning from s to s' via action a
  • R(s): Reward function — immediate reward for being in state s
  • γ ∈ [0, 1]: Discount factor

The Bellman Equation (Formal Statement)

The value (utility) function for states is defined as:

U(s) = R(s) + γ maxa ∈ A Σs' ∈ S T(s, a, s') U(s')

This is a system of |S| non-linear equations with |S| unknowns (the utilities U(s) for each state). Solving this system gives us the optimal utilities.

Value Iteration Algorithm

Value Iteration solves the Bellman equation iteratively. Start with arbitrary utilities (often all zeros), then repeatedly apply the Bellman update until convergence:

Ui+1(s) = R(s) + γ maxa Σs' T(s, a, s') Ui(s')

The algorithm converges when the maximum change in utility is small:

||Ui+1 - Ui|| < ε(1 - γ) / γ

The number of iterations grows logarithmically in 1/ε and inversely with (1 - γ). This means:

  • High discount factor γ → more iterations needed (agents planning far into the future)
  • Small ε → more iterations (higher precision required)

Optimal Policy Extraction

Once values converge, the optimal policy is:

π*(s) = arg maxa ∈ A Σs' ∈ S T(s, a, s') U(s')

For each state, choose the action that leads to highest expected utility.

Policy Iteration Algorithm

Policy Iteration is an alternative to value iteration. Instead of iterating on utilities, we iterate on policies:

  1. Policy Evaluation: Given a policy π, compute the utility of each state assuming we follow π:
    Uπ(s) = R(s) + γ Σs' T(s, π(s), s') Uπ(s')
    This is a linear system (no max), so we can solve it exactly in O(|S|³) time using linear algebra.
  2. Policy Improvement: Compute a better policy:
    πnew(s) = arg maxa Σs' T(s, a, s') Uπ(s')
  3. Repeat until the policy stabilizes (doesn't change).

Policy iteration often converges in fewer iterations than value iteration, though each iteration is more expensive (solving a linear system).

Comparison: Value Iteration vs Policy Iteration

Aspect Value Iteration Policy Iteration
Outer loop Iterate until values converge Iterate until policy converges
Per iteration cost O(|S|² |A|) — simple operations O(|S|³) — linear system solve
Typical use When |S| is small to medium When |S| is large
Convergence Usually more iterations Usually fewer iterations

Partially Observable MDPs (POMDPs)

So far, we've assumed the agent can perfectly observe which state it's in. But what if the robot can't see where it is in the grid — only whether there's a wall nearby?

A Partially Observable MDP (POMDP) adds uncertainty about the current state. Instead of knowing s exactly, the agent maintains a belief state b(s) — a probability distribution over possible states.

The belief state is updated using Bayes' rule. If the agent takes action a and observes o, the new belief is:

b'(s') = α · P(o|s') Σs T(s, a, s') b(s)

Where α is a normalization constant. Breaking this down:

  • Σs T(s, a, s') b(s): Where we could have ended up after action a
  • P(o|s'): How likely we are to observe o from state s'
  • α: Normalize so probabilities sum to 1

POMDPs are harder to solve than MDPs because the agent must simultaneously plan over both actions and observations. Optimal POMDP solutions require solving for a policy over belief states, which grows exponentially with |S|.

Practical Insight: For POMDPs, algorithms like POMCP (Partially Observable Monte Carlo Planning) use sampling and tree search instead of exact dynamic programming, trading optimality for computational efficiency.

Clean-Code Implementation: MDPs in Python

Let's implement a complete MDP solver with both Value Iteration and Policy Iteration.

class MDP:
    """
    Markov Decision Process.

    The rational agent faces sequential decisions under uncertainty.
    Unlike single-shot decisions (Ch 16), the agent must plan ahead,
    accounting for how today's actions affect tomorrow's states.

    Attributes:
        states: List of state identifiers
        actions: List of action identifiers
        T: Transition function T[s][a] = [(prob, next_state), ...]
        R: Reward function R[s] = immediate reward
        gamma: Discount factor (0 <= gamma <= 1)
    """

    def __init__(self, states, actions, transition, reward, gamma=0.9):
        """
        Initialize an MDP.

        Args:
            states: List of hashable state identifiers
            actions: List of hashable action identifiers
            transition: Dict mapping (state, action) to [(probability, next_state), ...]
            reward: Dict mapping state to immediate reward
            gamma: Discount factor
        """
        self.states = states
        self.actions = actions
        self.T = transition  # T[s][a] = [(prob, next_state), ...]
        self.R = reward      # R[s] = reward
        self.gamma = gamma

    def value_iteration(self, epsilon=0.001):
        """
        Iteratively compute the utility of each state using the Bellman equation.

        The Bellman equation says:
        U(s) = R(s) + γ * max_a Σ T(s,a,s') * U(s')

        We repeatedly apply this update until utilities converge:
        |U_{i+1} - U_i| < ε(1-γ)/γ

        Args:
            epsilon: Convergence threshold

        Returns:
            Dictionary mapping state -> utility value
        """
        U = {s: 0.0 for s in self.states}

        while True:
            U_new = {}
            delta = 0

            for s in self.states:
                # If no actions available in this state, utility = reward
                if s not in self.T or not self.T[s]:
                    U_new[s] = self.R.get(s, 0)
                    continue

                # Bellman update: max over actions
                best_action_value = float('-inf')
                for a in self.T[s]:
                    action_value = sum(
                        prob * U[s2]
                        for prob, s2 in self.T[s][a]
                    )
                    best_action_value = max(best_action_value, action_value)

                U_new[s] = self.R.get(s, 0) + self.gamma * best_action_value
                delta = max(delta, abs(U_new[s] - U[s]))

            U = U_new

            # Check convergence: ||U_{i+1} - U_i|| < ε(1-γ)/γ
            if delta < epsilon * (1 - self.gamma) / self.gamma:
                break

        return U

    def best_policy(self, U):
        """
        Extract the optimal policy from utility values.

        For each state, choose the action that maximizes expected utility:
        π*(s) = arg max_a Σ T(s,a,s') U(s')

        Args:
            U: Dictionary of state utilities

        Returns:
            Dictionary mapping state -> best action
        """
        pi = {}
        for s in self.states:
            if s not in self.T or not self.T[s]:
                pi[s] = None
                continue

            # Find action with highest expected utility
            best_action = max(
                self.T[s].keys(),
                key=lambda a: sum(
                    prob * U[s2]
                    for prob, s2 in self.T[s][a]
                )
            )
            pi[s] = best_action

        return pi

    def policy_iteration(self):
        """
        Compute the optimal policy using policy iteration.

        Alternates between:
        1. Policy Evaluation: Solve linear system for U^π
        2. Policy Improvement: π' = best_policy(U^π)

        Repeat until policy stabilizes.

        Returns:
            Tuple of (optimal_policy, utilities)
        """
        import random

        # Initialize with random policy
        pi = {}
        for s in self.states:
            if s not in self.T or not self.T[s]:
                pi[s] = None
            else:
                pi[s] = random.choice(list(self.T[s].keys()))

        while True:
            # Policy Evaluation: compute U^π
            U = self._policy_evaluation(pi)

            # Policy Improvement: greedily improve each action
            changed = False
            for s in self.states:
                if s not in self.T or not self.T[s]:
                    continue

                best_a = max(
                    self.T[s].keys(),
                    key=lambda a: sum(
                        prob * U[s2]
                        for prob, s2 in self.T[s][a]
                    )
                )

                if best_a != pi[s]:
                    pi[s] = best_a
                    changed = True

            # If no changes, policy is optimal
            if not changed:
                return pi, U

    def _policy_evaluation(self, pi, epsilon=0.001):
        """
        Evaluate the utility of a given policy.

        Solves the linear system:
        U^π(s) = R(s) + γ Σ T(s, π(s), s') U^π(s')

        We use an iterative method (Gauss-Seidel) for simplicity.
        For large |S|, use direct linear algebra (numpy.linalg.solve).

        Args:
            pi: Policy (state -> action dict)
            epsilon: Convergence threshold

        Returns:
            Dictionary of state utilities under policy pi
        """
        U = {s: 0.0 for s in self.states}

        while True:
            U_new = {}
            delta = 0

            for s in self.states:
                if s not in self.T or not self.T[s] or pi[s] is None:
                    U_new[s] = self.R.get(s, 0)
                    continue

                a = pi[s]
                U_new[s] = self.R.get(s, 0) + self.gamma * sum(
                    prob * U[s2]
                    for prob, s2 in self.T[s][a]
                )
                delta = max(delta, abs(U_new[s] - U[s]))

            U = U_new
            if delta < epsilon:
                break

        return U


# Example: The classic 4x3 grid world from AIMA
# States: (1,1) to (4,3) are cells
# Actions: Up, Down, Left, Right
# Transition: 80% intended direction, 10% each perpendicular
# Reward: +1 at (4,3), -1 at (4,2), -0.04 elsewhere
# Discount: γ = 0.99

def create_grid_mdp():
    """Create the classic AIMA 4x3 grid MDP."""
    states = [(x, y) for x in range(1, 5) for y in range(1, 4)]
    actions = ['Up', 'Down', 'Left', 'Right']

    # Rewards
    rewards = {}
    for s in states:
        if s == (4, 3):
            rewards[s] = 1.0
        elif s == (4, 2):
            rewards[s] = -1.0
        else:
            rewards[s] = -0.04

    # Transition function
    transitions = {}
    for s in states:
        transitions[s] = {}
        for a in actions:
            # Simplified: deterministic for now
            next_s = simulate_action(s, a)
            transitions[s][a] = [(1.0, next_s)]

    return MDP(states, actions, transitions, rewards, gamma=0.99)

def simulate_action(state, action):
    """Deterministic action (simplified; add stochasticity as needed)."""
    x, y = state
    if action == 'Up':
        y = min(y + 1, 3)
    elif action == 'Down':
        y = max(y - 1, 1)
    elif action == 'Left':
        x = max(x - 1, 1)
    elif action == 'Right':
        x = min(x + 1, 4)
    return (x, y)

# Usage
if __name__ == '__main__':
    mdp = create_grid_mdp()

    # Value Iteration
    print("Running Value Iteration...")
    U = mdp.value_iteration()
    pi = mdp.best_policy(U)

    print("Utilities:")
    for s in sorted(U.keys()):
        print(f"  {s}: {U[s]:.3f}, Policy: {pi[s]}")

    # Policy Iteration
    print("\nRunning Policy Iteration...")
    pi_opt, U_opt = mdp.policy_iteration()

    print("Optimal Utilities:")
    for s in sorted(U_opt.keys()):
        print(f"  {s}: {U_opt[s]:.3f}, Policy: {pi_opt[s]}")

Key Implementation Details

Convergence Criterion: We use the theoretical convergence criterion from AIMA: delta < epsilon * (1 - gamma) / gamma. This ensures we've found a ε-optimal solution.

Transition Representation: We represent T as a nested dict: T[s][a] = [(prob, next_state), ...]. This makes the code flexible and easy to understand, though sparse representations would be more efficient for large state spaces.

Edge Cases: We handle states with no actions (absorbing states) and missing entries in the reward dict (defaulting to 0).

Socratic Sandbox: Reasoning Through the Grid

Let's think deeply about MDPs with concrete scenarios and questions.

The Classic 4×3 Grid World

The canonical AIMA example: a robot in a 4×3 grid. There's a +1 reward at (4,3), a -1 at (4,2) (the pit), and -0.04 for every other step. Actions are Up, Down, Left, Right, but they're stochastic: 80% go the intended direction, 10% go left perpendicular, 10% go right perpendicular.

With γ=0.99 and R=-0.04, what does the optimal policy do?

Surprising result: The optimal policy tells the agent to move AWAY from the goal if it means avoiding the pit! From (3,2), which is adjacent to the goal (4,3) and the pit (4,2), the optimal action is to move down, even though the goal is just one step away. Why? Because there's a 10% chance the "right" action goes perpendicular and lands in the pit, losing -1. The expected value of going right is: 0.8 * U(4,3) + 0.1 * U(pit) + 0.1 * U(stay), which could be less than going down and taking a longer, safer path.

Guided Questions

Question 1: The Cost of Living
If we change R(s) from -0.04 to -2 for all non-goal, non-pit states, how does the optimal policy change? Why might the agent now be willing to risk the -1 square?

Hint: Think about the relative costs. With R=-0.04, taking 25 steps costs -1. With R=-2, taking 1 step costs -2. The long, safe path becomes less attractive.

Question 2: The Horizon Problem
What role does the discount factor γ play? Compare γ=0.1 vs γ=0.99. With γ=0.1, the agent only cares about immediate rewards—it becomes myopic. With γ=0.99, it thinks 100 steps ahead. How does this affect the policy in the grid world?

Hint: With small γ, rewards 10+ steps away contribute negligibly. With large γ, even distant rewards matter. In the grid world, does the agent prefer the short risky path or the long safe path?

Question 3: Algorithmic Tradeoffs
Value Iteration and Policy Iteration both find the optimal policy. But when would you prefer one over the other? Think about the number of states, the number of iterations, and the cost per iteration.

Hint: Value Iteration is O(|S|² |A|) per iteration, Policy Iteration is O(|S|³) per iteration. Which is better when |S| is huge? When the policy is nearly optimal already?

Question 4: Belief and Uncertainty
Now imagine the robot can't see which cell it's in—only whether there's a wall nearby (the observation). This is a POMDP. How does the problem change? What is a "belief state"? Can the robot still find an optimal policy?

Hint: The belief state is a probability distribution over cells (e.g., "50% chance I'm at (2,2), 30% at (2,1), 20% at (1,1)"). The optimal policy must decide actions based on beliefs, not true states. This makes POMDPs exponentially harder.

Question 5: The Bellman Equation Deconstructed
The Bellman equation has a max over actions. What if we remove the max and fix an action—say, "always go right"? What equation does that become?

Answer: U^π(s) = R(s) + γ Σ T(s, π(s), s') U^π(s'). This is the policy evaluation equation! It computes the utility of following a fixed policy. Without the max, it's a linear system, so we can solve it exactly instead of iteratively.

Deeper Insights

Markovian assumption: The Markov property says the future depends only on the current state, not on how we got there. This is powerful: it reduces the state space explosion. But it's not always realistic. If the robot has memory (e.g., "I've been circling for 100 steps"), the true state isn't just position—it's position + history. MDPs assume this is captured in the state space.

The infinite-horizon trap: In an infinite-horizon MDP, the agent makes decisions forever. Without discounting (γ = 1) or absorbing states, the total reward could be infinite. Discounting ensures convergence. With γ = 0.99, rewards far in the future contribute almost nothing, making the problem tractable.

Optimality and stochasticity: A crucial insight: the optimal policy is deterministic! Even though the world is stochastic (transitions are probabilistic), the policy that maximizes expected reward never needs to randomize. It always picks the single best action. (This is true for MDPs; for POMDPs, randomized policies can be optimal.)

Summary & Next Steps

What We've Learned

  • Sequential decisions: Unlike single decisions (Ch 16), agents here make a series of choices over time. A policy π(s) → a is a complete strategy for every situation.
  • MDPs formalize uncertainty: States, actions, transition probabilities, rewards, and a discount factor γ together define a well-posed decision problem.
  • The Bellman equation: U(s) = R(s) + γ max_a Σ T(s,a,s') U(s'). This recursive equation is the cornerstone of dynamic programming for MDPs.
  • Value Iteration: Repeatedly apply the Bellman update until utilities converge. Each iteration is cheap; it takes many iterations to converge.
  • Policy Iteration: Alternate between evaluating a policy (solving a linear system) and improving it greedily. Fewer outer iterations, but each is more expensive.
  • Optimal policies are deterministic: Even in stochastic worlds, the policy that maximizes expected reward never randomizes.
  • Discounting and horizons: The factor γ ∈ [0,1] controls the agent's time horizon. Small γ: myopic. Large γ: far-sighted.
  • POMDPs add belief: When the agent can't observe the true state, decisions are based on belief states—probability distributions over possible states. This makes the problem harder.

Connections to Preceding Chapters

From Chapter 16 (Decisions): Chapter 16 dealt with one-shot decisions under uncertainty. MDPs generalize this to sequential decisions. The single decision is now the innermost problem: "given that I'm in state s, which action maximizes total future reward?"

From Chapter 15 (Probabilistic Reasoning): MDPs rely heavily on probability. The transition function T(s,a,s') is a conditional probability. Belief updating in POMDPs uses Bayes' rule (just like inference in Chapter 15).

From Part III (Knowledge & Reasoning): If the agent has incomplete knowledge about the world, it must reason about its own uncertainty. This is reflected in belief states in POMDPs.

Extensions & Future Topics

Reinforcement Learning (Ch 21): Here, we assumed the MDP was fully known (we had T and R). In reinforcement learning, the agent learns T and R from experience by trial and error.

Partially Observable MDPs (POMDPs): We touched on these briefly. Solving POMDPs exactly is PSPACE-hard, but approximation algorithms like POMCP are practical.

Game Theory (Ch 5): MDPs are single-agent. When multiple agents each optimize independently, we get game-theoretic problems. Game trees in Chapter 5 are a special case of MDPs with adversarial agents.

Temporal Difference Learning: A method that combines ideas from MDPs and reinforcement learning, learning both value functions and policies from sampled trajectories.

Key Takeaway

The Bellman equation—U(s) = R(s) + γ max_a Σ T(s,a,s') U(s')—is the unifying principle. It says: the value of being in a state is the immediate reward plus the discounted expected value of the best action you can take. This one equation, applied recursively, solves sequential decision problems under uncertainty. With this framework, an agent can plan optimally in stochastic worlds.
