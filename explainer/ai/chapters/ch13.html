Feynman Mental Model: The Family Tree of Causation

A Bayesian Network is fundamentally about causality and conditional independence. Here's the deepest intuition:

Core Analogy: Imagine a family tree of CAUSES AND EFFECTS. Just like how your eye color depends on your parents' genes (not directly on your cousin's), a Bayesian Network maps how variables influence each other through direct connections—and crucially, what does NOT depend on what.

Key Concepts Via Analogy

  • Nodes = Random Variables: Think of each node as a question: "Is it Raining?" or "Is the Sprinkler On?" These are the family members in our causal tree.
  • Edges = Direct Influence: A directed edge from A to B means A directly influences B. Rain directly affects whether the grass is wet, so Rain → WetGrass. But Rain does NOT directly influence the Sprinkler's decision.
  • Conditional Independence = The Power of Structure: Once you know your parents' genes, knowing your grandparents' genes tells you nothing new about your eye color. Mathematically: if you observe node B in the chain A → B → C, then A and C become independent. Learning about A doesn't help predict C when B is already known.
  • D-Separation = Information Blockade: This is when knowing certain variables blocks the flow of information between others. It's the formal way to determine when variables are independent given other observations.

Why This Matters for Rational Agents

A full joint probability distribution over n binary variables requires 2^n entries. For just 20 variables, that's over a million numbers! But a Bayesian Network capturing the causal structure can represent the same information with far fewer parameters—often exponentially fewer.

This is why Bayesian Networks are central to rational agent design: they allow agents to reason under uncertainty efficiently, encoding only the direct causal relationships that matter.

Example: In a home security network, the alarm depends directly on burglary and earthquake. John and Mary's phone calls depend on the alarm—not on the burglary itself. This structure lets us reason about P(Burglary | JohnCalls, MaryCalls) efficiently without encoding the independent contributions of everyone who might call.

Mathematical Ascent: The Formal Framework

Bayesian Network Definition

A Bayesian Network is a pair (DAG, CPTs):

  • DAG: A Directed Acyclic Graph where nodes are random variables and edges represent direct causal influence.
  • CPTs: Conditional Probability Tables. For each node, we store P(X | Parents(X))—the probability of that variable given its parents.

The Chain Rule Decomposition

The fundamental insight: any joint probability can be factored as a product of conditional probabilities:

$$P(x_1, x_2, \ldots, x_n) = \prod_{i=1}^{n} P(x_i \mid \text{Parents}(X_i))$$

This decomposition is the magic: instead of storing a massive table, we store only the conditional probability for each variable given its parents.

D-Separation: The Three Cases

To determine conditional independence in a Bayesian Network, we use d-separation (directional separation). Three patterns:

1. Chain: A → B → C

B is a "serial connector" between A and C. If B is observed, information cannot flow from A to C. A and C are conditionally independent given B.

$$A \perp\!\!\!\perp C \mid B$$

Example: Rain → Sprinkler → WetGrass. If you know the Sprinkler status, knowing whether it rained tells you nothing new about the grass being wet.

2. Fork: A ← B → C

B is a "diverging connector." If B is observed, it blocks the path from A to C. A and C are conditionally independent given B.

$$A \perp\!\!\!\perp C \mid B$$

Example: Burglary ← Alarm → JohnCalls. If the Alarm status is known, the Burglary status is independent of whether John called.

3. Collider: A → B ← C

B is a "converging connector"—this is the counterintuitive case. If B is not observed, A and C are conditionally independent. But if B is observed, the path OPENS—observing the effect explains away one cause!

$$\text{B unobserved: } A \perp\!\!\!\perp C$$ $$\text{B observed: } A \not\perp\!\!\!\perp C \mid B \text{ (path opens)}$$

Example: The Explaining Away Phenomenon. Burglary → Alarm ← Earthquake. If you don't know the alarm status, burglary and earthquake are independent (one doesn't tell you about the other). But if the alarm goes off, knowing there was an earthquake makes a burglary less likely—the earthquake "explains away" the alarm.

Exact Inference by Enumeration

To compute P(X | e) where e is evidence, we sum over all possible values of unobserved variables:

$$P(X \mid e) = \alpha \sum_{y} P(X, e, y)$$

Here α is a normalization constant (we divide by the sum over all values of X to ensure probabilities sum to 1).

Variable Elimination Algorithm

Enumeration is exponentially slow because it recomputes subproblems. Variable Elimination is smarter: it pushes summations as far right as possible, eliminating one variable at a time.

Key idea: Instead of computing ∑∑∑ separately, compute ∑₁(∑₂(∑₃(...))), grouping factors that mention each variable.

Factor Operations

The algorithm relies on two operations on factors (probability tables):

  • Pointwise Product: Given factors f₁ and f₂, their product f₁ · f₂ is a new factor where each entry is the product of corresponding entries. This combines independent probability assessments.
  • Marginalization: To eliminate a variable X from a factor, we sum over all values of X. The result is a smaller factor that doesn't mention X.
The Order Matters: Variable Elimination is efficient only with a good elimination order. The wrong order can create huge intermediate factors. This is an NP-hard optimization problem in general, but heuristics (like choosing variables that appear in the fewest factors) work well in practice.

Clean-Code Implementation: Building a Bayesian Network

Here's a Python implementation that captures the essence of Bayesian Networks and exact inference:

class BayesNode:
    """
    A node in a Bayesian Network.
    Each node knows its parents and stores a Conditional Probability Table (CPT).

    Maps to the Rational Agent theory: the agent's world model captures
    direct causal relationships, not every possible correlation.
    """
    def __init__(self, name, parents, cpt):
        """
        Args:
            name: String name of the variable (e.g., 'Burglary')
            parents: List of parent node names (e.g., [])
            cpt: Dict mapping parent value tuples to P(X=True).
                 For a node with no parents: {(): 0.001}
                 For a node with parents: {(T, T): 0.95, (T, F): 0.94, ...}
        """
        self.name = name
        self.parents = parents
        self.cpt = cpt

    def probability(self, value, parent_values):
        """
        Compute P(self=value | parents=parent_values).

        Args:
            value: Boolean value (True or False)
            parent_values: Dict mapping parent names to their values

        Returns:
            Probability that this node has the given value
        """
        # Build the key into the CPT
        key = tuple(parent_values[p] for p in self.parents) if self.parents else ()
        p_true = self.cpt[key]
        return p_true if value else (1 - p_true)


class BayesNet:
    """
    Bayesian Network: a compact representation of a joint probability distribution.

    The rational agent uses this to reason under uncertainty without needing
    the exponentially large full joint distribution.
    """
    def __init__(self):
        self.nodes = {}      # name -> BayesNode
        self.order = []      # Topological order for processing

    def add(self, name, parents, cpt):
        """Add a node to the network."""
        self.nodes[name] = BayesNode(name, parents, cpt)
        self.order.append(name)

    def enumeration_ask(self, query, evidence):
        """
        Exact inference by enumeration.

        Computes P(query | evidence) by enumerating all consistent values
        of unobserved variables.

        Args:
            query: String name of the variable we want to know about
            evidence: Dict of observed variable assignments

        Returns:
            Dict mapping {True: probability, False: probability}
        """
        dist = {}
        for val in [True, False]:
            evidence[query] = val
            dist[val] = self._enumerate_all(list(self.order), dict(evidence))
            del evidence[query]

        # Normalize so probabilities sum to 1
        total = sum(dist.values())
        return {k: v / total for k, v in dist.items()}

    def _enumerate_all(self, variables, evidence):
        """
        Recursive enumeration helper.
        Sums over all consistent completions of variables.
        """
        if not variables:
            # Base case: all variables assigned, return P(evidence)
            return 1.0

        var = variables[0]
        rest = variables[1:]
        node = self.nodes[var]

        if var in evidence:
            # Variable is observed; use its known value
            return (node.probability(evidence[var], evidence) *
                    self._enumerate_all(rest, evidence))
        else:
            # Variable is unobserved; sum over both values
            total = 0.0
            for val in [True, False]:
                evidence[var] = val
                total += (node.probability(val, evidence) *
                         self._enumerate_all(rest, evidence))
                del evidence[var]
            return total


# ============================================================================
# Example: Classic Burglary-Alarm Network
# ============================================================================
# Structure:
#   Burglary → Alarm ← Earthquake
#   Alarm → JohnCalls, Alarm → MaryCalls
#
# Intuition: A burglary or earthquake can trigger an alarm. If the alarm
# goes off, John and Mary might call to report it.

net = BayesNet()

# P(Burglary)
net.add('Burglary', [], {(): 0.001})

# P(Earthquake)
net.add('Earthquake', [], {(): 0.002})

# P(Alarm | Burglary, Earthquake)
net.add('Alarm', ['Burglary', 'Earthquake'], {
    (True, True):   0.95,
    (True, False):  0.94,
    (False, True):  0.29,
    (False, False): 0.001
})

# P(JohnCalls | Alarm)
net.add('JohnCalls', ['Alarm'], {
    (True,):  0.90,
    (False,): 0.05
})

# P(MaryCalls | Alarm)
net.add('MaryCalls', ['Alarm'], {
    (True,):  0.70,
    (False,): 0.01
})

# ============================================================================
# Inference Example
# ============================================================================
# Scenario: Both John and Mary call to report the alarm.
# What's the probability of a burglary?

result = net.enumeration_ask('Burglary',
                              {'JohnCalls': True, 'MaryCalls': True})
print(f"P(Burglary | JohnCalls, MaryCalls) = {result[True]:.4f}")
# Output: P(Burglary | JohnCalls, MaryCalls) = 0.2842

# Now condition on the earthquake:
result_with_quake = net.enumeration_ask('Burglary',
                                         {'JohnCalls': True, 'MaryCalls': True,
                                          'Earthquake': True})
print(f"P(Burglary | JohnCalls, MaryCalls, Earthquake=True) = {result_with_quake[True]:.4f}")
# Output: P(Burglary | JohnCalls, MaryCalls, Earthquake=True) ≈ 0.1607

# Notice: The probability DECREASED when we observed the earthquake!
# The earthquake "explains away" the alarm, making a burglary less likely.
# This is the "explaining away" phenomenon—a hallmark of colliders.

Key Design Insights

  • Separating Structure from Parameters: BayesNode stores the CPT (probability numbers) separately from the network structure. This mirrors how rational agents separate world models from beliefs.
  • Topological Ordering: The `order` list ensures we process variables in a valid order (parents before children), crucial for the recursive enumeration.
  • Evidence Handling: The algorithm elegantly handles mixed observations and unknowns—evidence variables are used directly, unobserved variables are summed over.
  • Normalization: The final normalization ensures the result is a proper probability distribution.

Computational Complexity

Exact inference by enumeration has time complexity O(n · 2^n) where n is the number of variables. For 20 variables, this is roughly 20 million operations—manageable, but slow for larger networks. For networks with special structure (trees, polytrees), more efficient algorithms exist.

Socratic Sandbox: The Explaining Away Phenomenon

Scenario: A home security Bayesian Network has:

  • Burglary → Alarm ← Earthquake
  • Alarm → JohnCalls, Alarm → MaryCalls

John and Mary both call to say the alarm is going off. The system computes a ~28% chance of burglary. Then a radio report confirms a small earthquake happened.

Think deeply about these questions before reading the answers:

  1. The Counterintuitive Direction: Before hearing about the earthquake, P(Burglary|John,Mary) ≈ 0.28. After learning about the earthquake, does P(Burglary) go UP or DOWN? Why?
    Answer: It goes DOWN (to ~0.16). This is the "explaining away" phenomenon. Both the earthquake and burglary could explain the alarm. Once the earthquake is observed as a cause, the burglary becomes a less necessary explanation. The alarm is now "explained" by the earthquake, making the burglary less probable.
  2. Collider Structure: Why is the Alarm node a "collider" between Burglary and Earthquake? What does d-separation tell us about their relationship before and after observing Alarm?
    Answer: Alarm is a collider because both Burglary and Earthquake point INTO it (A → B ← C pattern). D-separation says: without knowing the alarm, Burglary and Earthquake are independent—one doesn't tell you about the other. But once the alarm is observed, the path opens, and they become dependent. This is why observing the calls (which are conditional on the alarm) creates dependence between burglary and earthquake.
  3. Scalability: If the network had 20 variables each with 5 parents, how many parameters would the CPTs need? Compare this to the full joint distribution.
    Answer: CPTs need roughly 20 × 2^5 = 640 parameters (each variable with 5 binary parents has 32 entries in its CPT). The full joint distribution over 20 binary variables needs 2^20 ≈ 1 million entries. The Bayesian Network is ~1500× more compact!
  4. Variable Elimination Order: For the query P(Burglary|JohnCalls), in what order should you eliminate variables to minimize computation? Why does order matter?
    Answer: Eliminate in order: MaryCalls → Earthquake → Alarm. This is roughly the reverse topological order. Why? Each elimination combines factors that mention that variable. Eliminating MaryCalls first lets you sum it out immediately (it doesn't depend on other unknowns). The order matters because bad orders create huge intermediate factors, blowing up computation time.
  5. Structural Change: An engineer adds a direct edge Burglary → JohnCalls (John lives nearby and might see the burglar). How does this change the inference? What independence assumption is lost?
    Answer: With the direct edge, JohnCalls is now influenced by both Burglary and Alarm. We lose the independence assumption that JohnCalls depends on Burglary only through the alarm. This makes inference harder (more conditional probabilities needed) but potentially more accurate if John truly has another way to detect burglaries.
