Chapter 23: Reinforcement Learning

Learning from Rewards and Exploration

Section 1: Feynman Mental Model

Reinforcement Learning (RL) is fundamentally different from supervised learning. There is no teacher with the correct answers. Instead, an agent takes actions in an environment, receives rewards, and must learn to maximize cumulative reward over time. The key insight: learning happens through interaction and feedback.

The Exploration/Exploitation Dilemma

Analogy: You move to a new city and want to find the best restaurant. Exploitation: go to the best restaurant you've found so far. Exploration: try a new restaurant you haven't been to. If you ALWAYS exploit, you'll never discover the amazing hidden gem around the corner. If you ALWAYS explore, you'll waste time at terrible places. The exploration/exploitation dilemma is the fundamental tension in RL.

Passive vs Active Reinforcement Learning

Passive Reinforcement Learning

In passive RL, the agent follows a fixed policy and learns the value of being in each state. It doesn't decide which actions to take—those are predetermined. The agent simply observes rewards and learns how good each state is under the current policy.

  • Direct Utility Estimation: After visiting a state, simply average all the observed rewards from that point onward. Simple but inefficient—requires many visits to converge.
  • Adaptive Dynamic Programming (ADP): Learn a model of the environment (state transitions and rewards), then solve the resulting MDP using value iteration. More sample-efficient than direct estimation.
  • Temporal-Difference (TD) Learning: Update value estimates incrementally using the bootstrapping principle: U(s) ← U(s) + α[R(s) + γU(s') − U(s)]. Combines the strengths of direct estimation and ADP.

Active Reinforcement Learning

In active RL, the agent chooses actions while learning. This introduces the exploration/exploitation tradeoff: the agent must balance trying new actions (exploration) with leveraging what it has learned (exploitation).

  • Q-Learning: Learn the value of state-action pairs directly without building a model. Off-policy: learns the optimal policy while exploring. This is the most popular RL algorithm.
  • SARSA: Similar to Q-learning but on-policy: learns the value of the actions it actually takes. More conservative and safer than Q-learning.
  • Exploration Strategies: How should the agent balance exploration and exploitation?
    • ε-Greedy: With probability ε, take a random action (explore); otherwise, take the best known action (exploit). Simple and effective.
    • UCB (Upper Confidence Bound): Explore states/actions that are uncertain (high variance). Balance is automatic based on confidence.
    • Optimistic Initialization: Start Q-values optimistically high. Reality brings them down, naturally encouraging exploration of unvisited actions.

Function Approximation

When the state space is massive or continuous, storing a table of Q-values or utilities becomes infeasible. Instead, use a function approximator (neural network, decision tree, polynomial) to estimate values. The learning process updates the parameters of the function rather than explicit table entries. This is the bridge from tabular RL to deep RL.

Policy Gradient Methods (Brief Intro)

Rather than learning values and deriving a policy, policy gradient methods directly learn a parameterized policy. The gradient of expected cumulative reward with respect to policy parameters guides learning. This is more natural for continuous action spaces and enables stochastic policies.

Section 2: Mathematical Ascent

Temporal-Difference (TD) Learning

The core update rule for learning state utilities under a fixed policy:

U(s) \leftarrow U(s) + \alpha [R(s) + \gamma U(s') - U(s)]

Here, α is the learning rate, γ is the discount factor, R(s) is the immediate reward, and U(s') is the estimated utility of the next state. The bracketed term is the TD error: the difference between what we expected and what we actually observed.

Q-Learning: Off-Policy Learning

Q-learning learns the value of state-action pairs without requiring a model of the environment. The key innovation: update using the best possible next action, regardless of what the agent actually does.

Q(s,a) \leftarrow Q(s,a) + \alpha [R(s) + \gamma \max_{a'} Q(s',a') - Q(s,a)]

This makes Q-learning off-policy: it learns the optimal policy (following max actions) while potentially exploring other actions. Over time, as Q-values converge, the greedy policy (always pick max Q) becomes optimal.

SARSA: On-Policy Learning

SARSA (State-Action-Reward-State-Action) updates using the actual next action taken, not the best possible action. This makes it on-policy: the learned policy is the one being followed.

Q(s,a) \leftarrow Q(s,a) + \alpha [R(s) + \gamma Q(s',a') - Q(s,a)]

SARSA is more conservative: if exploration leads to bad outcomes, SARSA learns to avoid exploration. Q-learning, by contrast, learns the optimal policy even if exploration is risky. For cliff-walking scenarios (see interactive), SARSA learns safe paths while Q-learning learns risky optimal paths.

ε-Greedy Policy

The ε-greedy exploration strategy: with probability ε, act randomly; otherwise, act greedily on current Q-values.

\pi(s) = \begin{cases} \arg\max_a Q(s,a) & \text{with probability } 1-\epsilon \\ \text{random action} & \text{with probability } \epsilon \end{cases}

UCB (Upper Confidence Bound) Exploration

Rather than random exploration, UCB prioritizes actions with high uncertainty. The bonus term c√(ln N(s) / N(s,a)) grows as an action is unexplored and shrinks as it is explored.

a^* = \arg\max_a \left[Q(s,a) + c\sqrt{\frac{\ln N(s)}{N(s,a)}}\right]

Bellman Optimality for Q-Values

At convergence, optimal Q-values satisfy the Bellman equation:

Q^*(s,a) = R(s) + \gamma \sum_{s'} T(s,a,s') \max_{a'} Q^*(s',a')

This says: the optimal value of an action is the immediate reward plus the discounted value of the optimal action in the next state.

Section 3: Clean-Code Implementation

Q-Learning Agent

import random
from collections import defaultdict

class QLearningAgent:
    """
    Q-Learning: model-free, off-policy reinforcement learning.
    The agent learns the value of state-action pairs directly,
    without needing a model of the environment.
    This is the key advantage: the agent learns from experience alone.
    """
    def __init__(self, actions, alpha=0.1, gamma=0.9, epsilon=0.1):
        self.actions = actions
        self.alpha = alpha      # Learning rate
        self.gamma = gamma      # Discount factor
        self.epsilon = epsilon  # Exploration rate
        self.Q = defaultdict(lambda: defaultdict(float))

    def choose_action(self, state):
        """ε-greedy: explore with probability ε, exploit otherwise."""
        if random.random() < self.epsilon:
            return random.choice(self.actions)  # Explore
        q_values = self.Q[state]
        if not q_values:
            return random.choice(self.actions)
        return max(self.actions, key=lambda a: q_values[a])  # Exploit

    def learn(self, state, action, reward, next_state, done):
        """
        Q-learning update: use the MAX over next actions (off-policy).
        Q(s,a) ← Q(s,a) + α[R + γ max_a' Q(s',a') - Q(s,a)]
        """
        if done:
            target = reward
        else:
            target = reward + self.gamma * max(
                self.Q[next_state][a] for a in self.actions
            )
        td_error = target - self.Q[state][action]
        self.Q[state][action] += self.alpha * td_error
        return td_error

SARSA Agent

class SARSAAgent:
    """
    SARSA: on-policy TD control.
    Unlike Q-learning, SARSA updates using the action actually taken
    (not the greedy max), making it more conservative and safer.
    """
    def __init__(self, actions, alpha=0.1, gamma=0.9, epsilon=0.1):
        self.actions = actions
        self.alpha = alpha
        self.gamma = gamma
        self.epsilon = epsilon
        self.Q = defaultdict(lambda: defaultdict(float))

    def choose_action(self, state):
        if random.random() < self.epsilon:
            return random.choice(self.actions)
        return max(self.actions, key=lambda a: self.Q[state][a])

    def learn(self, state, action, reward, next_state, next_action, done):
        """
        SARSA update: use the actual next action (on-policy).
        Q(s,a) ← Q(s,a) + α[R + γ Q(s',a') - Q(s,a)]
        """
        if done:
            target = reward
        else:
            target = reward + self.gamma * self.Q[next_state][next_action]
        self.Q[state][action] += self.alpha * (target - self.Q[state][action])

Section 4: Socratic Sandbox

Scenario: A Q-learning agent is navigating a cliff-walking grid (the classic example). The bottom row is a cliff with -100 reward. The goal is at the bottom-right. After 1000 episodes, the Q-learning agent's learned policy goes RIGHT ALONG the cliff edge—the shortest path. The SARSA agent's policy takes a longer path AWAY from the cliff.

Question 1: Why does Q-learning learn the risky cliff-edge path while SARSA learns the safer path? The key is in how each algorithm updates. Explain the difference.

Hint: Q-learning uses max in the update (best possible next action), while SARSA uses the actual next action. During learning with ε-greedy exploration, this difference matters. Q-learning says: "Even though I'm exploring and sometimes falling off the cliff, I know the best action along the cliff is safe, so I'll learn that policy." SARSA says: "I'm falling off the cliff while exploring, so the cliff-edge policy is dangerous—I should learn to avoid it."

Question 2: With ε=0.1, the Q-learning agent occasionally falls off the cliff during exploration. But its learned policy is still the cliff-edge path. Why doesn't the falling experience teach it to stay away?

Hint: Q-learning is off-policy. The exploration (where falls happen) is separate from the policy being learned. When Q-learning updates Q(cliff-state, go-right), it uses max Q(next-state, *), which reflects the best action, not the exploration action. The falls happen during exploration, not during the cliff-edge trajectory itself.

Question 3: If ε decreases to 0 over time, do Q-learning and SARSA converge to the same policy? Why?

Hint: As ε → 0, SARSA becomes on-policy (always acts greedily), and Q-learning's learned policy (greedy on Q-values) is already optimal. Both should converge to the same policy—the optimal one. However, their learned Q-values may differ if one learned more pessimistically (SARSA) or optimistically (Q-learning).

Question 4: A new RL agent uses ε=1.0 (always random). Can it still learn? What would its Q-values converge to?

Hint: Yes, even with ε=1.0, Q-learning can learn. The update rule uses max, so the agent learns optimal action values even though it explores randomly. Q-values would converge to the optimal policy's values. SARSA would converge to the value of the random policy, since it learns on-policy.

Question 5: Exploration/Exploitation: In the first 10 episodes, should ε be high or low? After 10,000 episodes? Design a decay schedule and justify it.

Hint: Early (episode 1-10): ε should be HIGH (e.g., 0.3-1.0) because Q-values are inaccurate. Exploration helps discover the state space. Later (episode 10,000): ε should be LOW (e.g., 0.01-0.05) because Q-values are accurate. Exploitation beats exploration. A common schedule: ε = ε₀ / (1 + episode/decay_rate) or exponential decay. This balances the two phases optimally.
