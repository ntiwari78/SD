Chapter 18: Multiagent Decision Making

When multiple independent agents interact, each choosing actions that affect the others' outcomes, game theory provides the mathematical framework for understanding their strategic behavior. From traffic intersections to business competition to climate negotiations, multiagent decision making is everywhere.

1. The Feynman Mental Model: Traffic as Game Theory

Imagine you're driving in heavy traffic. Here's what makes it a game theory problem:

"Think about driving in traffic. You're not the only decision-maker — every other driver is also choosing their speed, lane, and route. Your best action depends on what THEY do, and their best action depends on what YOU do. This circular dependency is the core challenge of game theory. Sometimes you cooperate (letting someone merge), sometimes you compete (racing for a parking spot), and sometimes you get stuck in a 'traffic jam equilibrium' where everyone would be better off if they could coordinate, but no individual has incentive to change."

Why This Matters

The traffic analogy reveals three fundamental insights:

  • Interdependence: Your payoff (time to destination) depends on others' actions
  • Strategic reasoning: Each driver must anticipate what others will do
  • Equilibrium vs. Optimality: The outcome where no one wants to change (equilibrium) might be worse than if everyone coordinated (Pareto optimal)

Key Concepts Covered

  • Game theory basics: players, strategies, and payoffs
  • Normal-form games represented as payoff matrices
  • Dominant strategies (always best, regardless of others' actions)
  • Nash Equilibrium: no player benefits from unilaterally deviating
  • Mixed strategies: playing different actions with specific probabilities
  • Mechanism design: designing the rules to incentivize good outcomes
  • Cooperation, coalition formation, and voting

2. Mathematical Ascent: Formalism of Strategic Interaction

Normal-Form Game Definition

A normal-form game is a tuple G = (N, A, u) where:

N = set of players {1, 2, ..., n}

A = set of action profiles; each agent i chooses from action set A_i, so A = A_1 \times A_2 \times \cdots \times A_n

u = utility functions; u_i: A \to \mathbb{R} gives agent i's payoff for each outcome

Nash Equilibrium

An action profile a^* = (a_1^*, a_2^*, \ldots, a_n^*) is a Nash Equilibrium if no player can improve their payoff by unilaterally changing their action:

$$\forall i, \forall a_i' \in A_i: \quad u_i(a_i^*, a_{-i}^*) \geq u_i(a_i', a_{-i}^*)$$

Here, a_{-i}^* means everyone else's equilibrium actions.

Dominant Strategy

A strategy a_i^d is dominant for player i if it's best regardless of what others do:

$$\forall a_{-i} \in A_{-i}: \quad u_i(a_i^d, a_{-i}) \geq u_i(a_i', a_{-i}) \quad \text{for all } a_i' \in A_i$$

Mixed Strategy Nash Equilibrium

A mixed strategy for player i is a probability distribution over actions: \sigma_i \in \Delta(A_i). The expected utility under a mixed strategy profile is:

$$EU_i(\sigma) = \sum_{a \in A} \left(\prod_{j=1}^{n} \sigma_j(a_j)\right) \cdot u_i(a)$$

The product term is the probability of action profile a occurring.

Minimax Theorem (Zero-Sum Games)

In a two-player zero-sum game (one's gain is the other's loss), the minimax value equals the maximin value:

$$\max_{\sigma_1 \in \Delta(A_1)} \min_{\sigma_2 \in \Delta(A_2)} EU_1(\sigma_1, \sigma_2) = \min_{\sigma_2 \in \Delta(A_2)} \max_{\sigma_1 \in \Delta(A_1)} EU_1(\sigma_1, \sigma_2)$$

This means each player can guarantee themselves the value of the game, even without knowing the opponent's strategy.

Pareto Optimality

An outcome is Pareto optimal if there is no other outcome where at least one player is better off and no player is worse off. Many Nash Equilibria are NOT Pareto optimal (the famous insight of the Prisoner's Dilemma).

3. Clean-Code Implementation: Game Theory Solver

import numpy as np
from itertools import product

class NormalFormGame:
    """
    A game where multiple rational agents choose actions simultaneously.
    Each agent's payoff depends on ALL agents' choices — the core
    challenge of multiagent systems.
    """
    def __init__(self, n_players, actions_per_player, payoff_matrices):
        """
        Initialize a normal-form game.

        Args:
            n_players: Number of players
            actions_per_player: List of action lists for each player
            payoff_matrices: Dict mapping player index to payoff matrix
                            (indexed by action profile tuple)
        """
        self.n_players = n_players
        self.actions = actions_per_player
        self.payoffs = payoff_matrices

    def is_nash_equilibrium(self, profile):
        """
        Check if an action profile is a Nash Equilibrium.

        A profile is NE if no player can unilaterally improve
        by changing their action.
        """
        for player in range(self.n_players):
            current_payoff = self.payoffs[player][profile]

            # Check all alternative actions for this player
            for alt_action in range(len(self.actions[player])):
                if alt_action == profile[player]:
                    continue

                alt_profile = list(profile)
                alt_profile[player] = alt_action
                alt_profile = tuple(alt_profile)

                # If we find a profitable deviation, not NE
                if self.payoffs[player][alt_profile] > current_payoff:
                    return False

        return True

    def find_pure_nash(self):
        """Find all pure-strategy Nash Equilibria by brute force."""
        all_profiles = list(product(*[range(len(a)) for a in self.actions]))
        nash_eq = [p for p in all_profiles if self.is_nash_equilibrium(p)]
        return nash_eq

    def dominant_strategy(self, player):
        """
        Find dominant strategy for a player, if one exists.

        A strategy is dominant if it's best against ALL possible
        action profiles of other players.
        """
        # Generate all possible action profiles for other players
        other_indices = [i for i in range(self.n_players) if i != player]
        other_actions = [self.actions[i] for i in other_indices]
        other_profiles = list(product(*[range(len(a)) for a in other_actions]))

        for action in range(len(self.actions[player])):
            is_dominant = True

            # Check against all other players' strategies
            for other_profile in other_profiles:
                # Construct full profile with our action
                full_profile = list(other_profile)
                full_profile.insert(player, action)
                my_payoff = self.payoffs[player][tuple(full_profile)]

                # Check all alternative actions
                for alt_action in range(len(self.actions[player])):
                    if alt_action == action:
                        continue

                    alt_full = list(other_profile)
                    alt_full.insert(player, alt_action)

                    if self.payoffs[player][tuple(alt_full)] > my_payoff:
                        is_dominant = False
                        break

                if not is_dominant:
                    break

            if is_dominant:
                return action

        return None

    def is_pareto_optimal(self, profile):
        """
        Check if an outcome is Pareto optimal.

        No other outcome exists where someone is better off
        and no one is worse off.
        """
        all_profiles = list(product(*[range(len(a)) for a in self.actions]))

        for other_profile in all_profiles:
            if other_profile == profile:
                continue

            # Check if other_profile Pareto dominates profile
            someone_better = False
            no_one_worse = True

            for player in range(self.n_players):
                if self.payoffs[player][other_profile] > self.payoffs[player][profile]:
                    someone_better = True
                if self.payoffs[player][other_profile] < self.payoffs[player][profile]:
                    no_one_worse = False
                    break

            if someone_better and no_one_worse:
                return False  # Dominated by other_profile

        return True


# Example: The Prisoner's Dilemma
# Actions: 0=Cooperate, 1=Defect
# Payoffs: CC=-1, CD=-3, DC=0, DD=-2 (mutual detection highest utility)

pd_game = NormalFormGame(
    n_players=2,
    actions_per_player=[['Cooperate', 'Defect'], ['Cooperate', 'Defect']],
    payoff_matrices={
        0: {(0,0): -1, (0,1): -3, (1,0): 0, (1,1): -2},
        1: {(0,0): -1, (0,1): 0, (1,0): -3, (1,1): -2}
    }
)

nash_eq = pd_game.find_pure_nash()
print(f"Nash Equilibria: {nash_eq}")  # [(1,1)] = both defect
print(f"Dominant strategy for player 0: {pd_game.dominant_strategy(0)}")  # 1 (Defect)
print(f"Is (0,0) Pareto optimal? {pd_game.is_pareto_optimal((0,0))}")  # True
print(f"Is (1,1) Pareto optimal? {pd_game.is_pareto_optimal((1,1))}")  # False

4. Socratic Sandbox: The Self-Driving Car Intersection

Scenario

Two self-driving cars approach an intersection simultaneously. Each can choose to Go or Wait.

  • If both Go: collision risk → both get -100 (catastrophic)
  • If both Wait: both stuck in traffic → both get -5 (minor delay)
  • If one Goes and one Waits: goer gets +10 (free passage), waiter gets -2 (brief wait)

Discussion Questions

Q1: Game Analysis
Write out the payoff matrix (done above). Find all Nash Equilibria. Is there a dominant strategy for either player?

Answer: The two pure Nash Equilibria are (Go, Wait) and (Wait, Go). There is NO dominant strategy — what's best depends on what the other car does. This is a "coordination game" where the cars need to somehow decide who goes.

Q2: Coordination Problem
This game has two pure-strategy Nash Equilibria. How do the cars "choose" which one? What happens in practice?

Answer: Without communication, they might both try to be polite (both Wait, stuck at -5) or both try to assert priority (crash at -100). This is why traffic lights exist—they break the symmetry by saying "Car 1's direction goes now, Car 2 waits."

Q3: Mechanism Design
A traffic light is a "mechanism" that solves this coordination problem. How does it change the game? What other mechanisms could work?

Answer: The traffic light removes one player's choice (forces them to Wait). Other mechanisms: (1) a rule "turn right has priority" (break symmetry), (2) a randomized signal (each car goes 50% of the time), (3) communication protocols (cars exchange intended actions).

Q4: Repeated Interaction
If this game is played repeatedly (same cars meet at the same intersection daily), how does strategy change? Can cooperation emerge?

Answer: In repeated play, reputation matters. A car might learn: "If I always go, the other car waits, and I get +10 repeatedly." But this depends on the other car's learning strategy. With sufficiently many rounds and proper discounting, cooperative patterns can emerge through reciprocal play.

Q5: Prisoner's Dilemma Comparison
The Prisoner's Dilemma has a different structure. Explain why the intersection game is fundamentally different.

Answer: In the Prisoner's Dilemma, both players have a DOMINANT strategy to defect. In the intersection game, there is NO dominant strategy — the best move depends on the other's choice. PD is purely about exploitation; the intersection game is about coordination.
