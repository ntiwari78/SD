Chapter 22: Deep Learning

From Hierarchical Abstraction to Transformers—Understanding Stacked Learning

Section 1: Feynman Mental Model

Layers of Abstraction. Think of a factory assembly line. Raw materials (pixels) enter one end. The first station (layer) does simple processing — detecting edges. The next station combines edges into shapes. The next combines shapes into parts (eyes, wheels). The final station assembles parts into whole objects (face, car). Each layer builds on the previous one's output, creating increasingly ABSTRACT representations. This is deep learning: stacking simple transformations to learn complex patterns.

Why Deep? The Power of Hierarchical Feature Learning

Deep learning works because of a fundamental insight: complex patterns can be decomposed into hierarchies of simpler patterns. A shallow neural network can theoretically learn any function, but it would need exponentially many neurons. By stacking layers, we leverage the structure of the data itself. Early layers discover simple features (edges, colors, textures). Middle layers combine these into mid-level features (shapes, parts). Deep layers recognize entire objects. This hierarchical composition is far more efficient than trying to learn everything at once.

Neural Network Basics: Neurons, Weights, Activation Functions

At its core, a neuron is a mathematical function. It takes inputs, multiplies each by a weight (strength of the connection), sums them, adds a bias, and passes through a nonlinear activation function. The weights are what the network learns. The activation function introduces nonlinearity—without it, stacking layers would just be matrix multiplication, which remains linear.

  • Weights: Learned parameters encoding what patterns matter.
  • Bias: A learnable offset, allowing the neuron to shift its decision boundary.
  • Activation functions: Nonlinear transformations (ReLU, sigmoid, tanh) that enable learning of complex relationships.

Backpropagation: How the Network Learns

Backpropagation is the engine of deep learning. It's the application of the chain rule of calculus flowing backward through the network. After a forward pass computes predictions, we compare to the true labels using a loss function. The loss measures error. We then ask: "How much did each weight contribute to this error?" Backprop computes these contributions via partial derivatives, flowing the gradient backward from the output layer to the input layer. Each weight is adjusted in the direction that reduces error.

The key insight: the chain rule allows us to compute gradients through many layers efficiently. Without it, training deep networks would be intractable.

Convolutional Neural Networks (CNNs): Vision from Convolution

Images have spatial structure. A fully-connected layer treats every input pixel independently, ignoring that nearby pixels are more relevant to each other. A convolutional layer uses a small filter (like a 3×3 window) and slides it across the image. This encodes two crucial properties:

  • Local connectivity: Neurons connect to a small local patch, not the entire image.
  • Parameter sharing: The same filter is reused across the image, dramatically reducing parameters and learning local patterns that appear anywhere.

The Convolution Operation: A filter slides over the image. At each position, we compute the dot product of the filter and the image patch. This becomes one output neuron. As the filter slides (with some stride), we get a 2D activation map.

Pooling: After convolution, pooling layers reduce spatial dimensions while keeping important features. Max pooling takes the maximum in a small window, preserving the strongest feature detected. This provides translation invariance: if an object shifts slightly, the pooled features remain similar.

CNN Architecture Pattern: Conv → Activation (ReLU) → Pool → Conv → Activation → Pool → ... → Fully Connected → Output. Each conv-pool block extracts increasingly abstract features. Early blocks find edges. Later blocks find shapes, parts, and whole objects.

Transformers: Attention Over Convolution

While CNNs excel at images, they have a limitation: they only look at local neighborhoods. In 2017, "Attention Is All You Need" introduced Transformers, which revolutionized NLP and then vision, audio, and more. The core mechanism is self-attention.

Self-Attention: Imagine you're reading a sentence. To understand the word "it," you need to know what it refers to. Self-attention lets every position look at every other position and learn how much to "attend" to each. Formally, for a sequence of word embeddings, we compute three matrices: Query (Q), Key (K), and Value (V). Each word's query asks "what should I look at?" Keys answer "here's what I am." The dot product QK^T measures relevance. We normalize (softmax) to get attention weights. These weights combine the values: higher weight means more of that word's information flows in.

Why Transformers are Powerful: Unlike RNNs (which process sequentially) or CNNs (which look locally), Transformers allow each position to directly relate to all others, in parallel. This enables learning long-range dependencies efficiently. Transformers are permutation-invariant by design (sequence order isn't built in), so we add Positional Encodings to give the model a sense of word order. Different attention heads can attend to different aspects simultaneously—one head might capture subject-verb relationships, another might track entities across the sequence.

Regularization: Preventing Overfitting

Deep networks have so many parameters they can memorize the training data without learning generalizable patterns. Regularization techniques prevent this:

  • Dropout: During training, randomly zero out some neuron activations. This forces the network to use redundant information and prevents co-adaptation.
  • Batch Normalization: Normalize layer inputs to have zero mean and unit variance. This stabilizes training, reduces internal covariate shift, and acts as regularization.
  • Weight decay (L2 penalty): Add a small penalty proportional to weight magnitude, encouraging smaller weights.

Transfer Learning: Standing on Giants' Shoulders

Training a large network from scratch requires massive data and compute. Transfer learning leverages pre-trained networks. For example, a CNN trained on ImageNet has learned features useful for visual tasks. We can fine-tune its weights on a new task, or freeze early layers and only train the final layers. This works because early layers (edge detectors, etc.) are task-general, while later layers (object detectors) are task-specific. By reusing the early learning, we need far less new data.

Section 2: Mathematical Ascent

The Neuron: Foundation Unit

A single neuron's output is given by:

$$y = \sigma(\mathbf{w} \cdot \mathbf{x} + b)$$
where σ is an activation function, w are weights, x are inputs, b is bias

Common Activation Functions

ReLU (Rectified Linear Unit): The most popular activation. Incredibly simple, yet powerful.

$$\text{ReLU}(x) = \max(0, x)$$

Sigmoid: Squashes inputs to (0,1), historically popular, but prone to vanishing gradients in deep networks.

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

Tanh: Squashes to (-1,1), zero-centered, slightly better than sigmoid.

$$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$$

Backpropagation: The Chain Rule in Action

Given a loss function L and a weight at layer l, backprop computes the gradient using the chain rule:

$$\frac{\partial L}{\partial w_{ij}^{(l)}} = \frac{\partial L}{\partial a_j^{(l)}} \cdot \frac{\partial a_j^{(l)}}{\partial z_j^{(l)}} \cdot \frac{\partial z_j^{(l)}}{\partial w_{ij}^{(l)}}$$
where z is pre-activation, a is post-activation

The first term (∂L/∂a) is the gradient from the next layer, backpropagated. The second term (∂a/∂z) is the derivative of the activation function. The third term (∂z/∂w) is the input. Multiplying these gives how much the weight at layer l contributed to the loss.

Why it matters: Backprop is efficient. Without the chain rule structure, computing gradients through many layers would require re-running the network many times. With it, a single backward pass computes all gradients in O(depth) time, the same as the forward pass.

Convolution: Sliding Window Dot Product

A 1D convolution is defined as:

$$(f * g)[n] = \sum_{m=-\infty}^{\infty} f[m] \cdot g[n - m]$$
For 2D images, the filter slides in both dimensions

In CNNs, f is the image, g is the filter. The output at position [n] is the dot product of the filter with the image patch starting at position n. As the filter slides by stride S, we get an output map.

Output Size of Convolution

Given input width W, filter size F, padding P, and stride S, the output width is:

$$\text{Output Width} = \frac{W - F + 2P}{S} + 1$$
Padding adds zeros around the input; stride is the step size of the filter

Self-Attention: The Heart of Transformers

Self-attention computes a weighted sum of values based on query-key relevance:

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$
Q, K, V are learned linear projections of the input; d_k is the key dimension

Breakdown: QK^T is a [seq_length × seq_length] matrix of relevance scores. We divide by √d_k to stabilize (prevents softmax from becoming too peaky). Softmax converts scores to probabilities. These weights are applied to V (the values), giving the output.

Multi-Head Attention: Multiple Perspectives

Instead of a single attention, use multiple heads, each with different Q, K, V projections. They attend to different aspects in parallel:

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O$$
$$\text{head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V)$$
Each head has its own projection matrices and attends independently

Positional Encoding: Adding Order to Attention

Self-attention is permutation-invariant—it doesn't inherently know sequence order. We add positional encodings to embeddings:

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right)$$
$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$
pos is position in sequence, i is dimension; allows model to learn relative positions

These sinusoidal encodings provide a unique "fingerprint" for each position. The wavelengths increase, so higher dimensions capture longer-range positional information.

Section 3: Clean-Code Implementation

Simple Feedforward Neural Network

import math
import random

class NeuralNetwork:
    """
    A simple feedforward neural network from scratch.
    Each layer transforms its input, building increasingly
    abstract representations — the core insight of deep learning.
    """
    def __init__(self, layer_sizes):
        """
        Initialize network with given layer sizes.

        Args:
            layer_sizes: List of integers, e.g., [784, 128, 64, 10]
                        for input, two hidden, and output layers
        """
        self.layers = []
        for i in range(len(layer_sizes) - 1):
            # Xavier initialization: scale weights by sqrt(2 / fan_in)
            scale = math.sqrt(2.0 / layer_sizes[i])
            weights = [
                [random.gauss(0, scale) for _ in range(layer_sizes[i])]
                for _ in range(layer_sizes[i + 1])
            ]
            biases = [0.0] * layer_sizes[i + 1]
            self.layers.append({'W': weights, 'b': biases})

    def relu(self, x):
        """ReLU activation: return max(0, x)"""
        return max(0, x)

    def relu_derivative(self, x):
        """Derivative of ReLU: 1 if x > 0, else 0"""
        return 1.0 if x > 0 else 0.0

    def forward(self, x):
        """
        Forward pass: input flows through layers of abstraction.

        Returns:
            activations: List of activations at each layer
            pre_activations: List of pre-activation values (z values)
        """
        activations = [x]
        pre_activations = []

        for i, layer in enumerate(self.layers):
            # z = W @ a + b
            z = [
                sum(layer['W'][j][k] * activations[-1][k]
                    for k in range(len(activations[-1])))
                + layer['b'][j]
                for j in range(len(layer['b']))
            ]
            pre_activations.append(z)

            # Apply activation (ReLU for hidden, linear for output)
            if i < len(self.layers) - 1:
                a = [self.relu(zi) for zi in z]
            else:
                a = z  # Linear output for regression
            activations.append(a)

        return activations, pre_activations

    def backward(self, activations, pre_activations, target, lr=0.01):
        """
        Backpropagation: the chain rule flows gradients backward.
        Updates weights and biases using gradient descent.

        Args:
            activations: List from forward pass
            pre_activations: List from forward pass
            target: True output vector
            lr: Learning rate
        """
        n_layers = len(self.layers)

        # Output layer: compute loss gradient (MSE)
        output = activations[-1]
        delta = [2.0 * (output[j] - target[j]) for j in range(len(output))]

        # Backpropagate through layers
        for l in reversed(range(n_layers)):
            # Gradient for weights and biases at layer l
            for j in range(len(self.layers[l]['b'])):
                # Update bias
                self.layers[l]['b'][j] -= lr * delta[j]
                # Update weights
                for k in range(len(self.layers[l]['W'][j])):
                    self.layers[l]['W'][j][k] -= lr * delta[j] * activations[l][k]

            # Propagate delta to previous layer
            if l > 0:
                new_delta = [0.0] * len(activations[l])
                for k in range(len(new_delta)):
                    for j in range(len(delta)):
                        new_delta[k] += delta[j] * self.layers[l]['W'][j][k]
                    # Apply activation function derivative
                    new_delta[k] *= self.relu_derivative(pre_activations[l - 1][k])
                delta = new_delta


def self_attention(Q, K, V, d_k):
    """
    Self-Attention mechanism — the core of Transformers.
    Every position attends to every other position,
    learning which parts of the input are relevant to each other.

    Args:
        Q: Query matrix [seq_len, d_k]
        K: Key matrix [seq_len, d_k]
        V: Value matrix [seq_len, d_v]
        d_k: Key dimension (for scaling)

    Returns:
        output: Weighted sum of values [seq_len, d_v]
        attention_weights: Attention matrix [seq_len, seq_len]
    """
    seq_len = len(Q)

    # 1. Compute attention scores: QK^T
    scores = [
        [
            sum(Q[i][d] * K[j][d] for d in range(d_k)) / math.sqrt(d_k)
            for j in range(len(K))
        ]
        for i in range(len(Q))
    ]

    # 2. Apply softmax for attention weights
    attention_weights = []
    for row in scores:
        max_val = max(row)  # For numerical stability
        exp_row = [math.exp(s - max_val) for s in row]
        total = sum(exp_row)
        attention_weights.append([e / total for e in exp_row])

    # 3. Weighted sum of values
    output = [
        [
            sum(attention_weights[i][j] * V[j][d] for j in range(len(V)))
            for d in range(len(V[0]))
        ]
        for i in range(len(Q))
    ]

    return output, attention_weights


# Example usage
if __name__ == "__main__":
    # Create a simple network: 3 inputs -> 4 hidden -> 2 outputs
    nn = NeuralNetwork([3, 4, 2])

    # Forward pass
    x = [0.5, -0.2, 0.8]
    activations, pre_activations = nn.forward(x)
    print(f"Output: {activations[-1]}")

    # Backward pass (SGD step)
    target = [0.0, 1.0]
    nn.backward(activations, pre_activations, target, lr=0.01)
    print("Weights updated via backpropagation")

Code Highlights

  • Xavier Initialization: Weights initialized with variance 2/fan_in, helping gradients flow through layers.
  • Forward Pass: Clean separation of pre-activation (z) and post-activation (a), needed for backprop.
  • ReLU Activation: Simple yet effective nonlinearity.
  • Backpropagation: Reversed loop through layers, applying chain rule at each step.
  • Self-Attention: Demonstrates the three steps: score computation, softmax, weighted sum.

Section 4: Socratic Sandbox

Scenario: The Mystery of Adversarial Examples

A CNN trained on ImageNet can classify 1000 object categories with 76% top-1 accuracy. Its first convolutional layer has learned edge detectors (horizontal lines, diagonal lines). Its middle layers detect textures and parts (fur patterns, wheel shapes). Its final layers recognize whole objects (cats, cars). But when shown an adversarial image (a panda with tiny, imperceptible perturbations added to each pixel), it confidently classifies it as a gibbon with 99% confidence. The perturbations are so small humans see the image as unchanged. Yet the network's behavior changes dramatically.

Probing Questions

  1. 1. Hierarchical Learning and Abstraction. Why do earlier layers learn edges while later layers learn objects? This relates to the "composition of functions" — explain how stacking layers creates abstraction. What would happen if you used a single layer with the same number of neurons?
  2. 2. Parameter Efficiency and Convolution. A CNN with 3 convolutional layers, each with 64 3×3 filters, has far fewer parameters than a fully-connected network processing the same image. If the image is 256×256 pixels, calculate how many parameters a fully-connected layer would have, then estimate CNN parameters (accounting for parameter sharing). What property of images makes convolution work?
  3. 3. Computational Complexity of Self-Attention. The Transformer's self-attention computes relationships between ALL pairs of positions. If a sequence has length n, the attention matrix is n×n. What is the computational cost in terms of n? Why is this both powerful (it enables learning long-range dependencies) and problematic (it limits sequence length)?
  4. 4. Numerical Stability in Attention. Why does dividing by √d_k in the attention formula matter? What goes wrong without it? (Hint: think about what happens to softmax when you feed it very large inputs. What's the effect on gradients during backprop?)
  5. 5. What Has the Network Learned? Adversarial examples fool the network with imperceptible changes. What does this reveal about what the network has actually "learned"? Is it truly understanding objects, or has it learned some spurious statistical patterns in ImageNet? If the network is fooled by pixel-level noise, should we trust it in high-stakes applications (medical diagnosis, autonomous vehicles)?

Discussion Notes

On Question 1: A single wide layer is a universal approximator, but learning requires exponentially more examples. Depth enables learning with polynomial samples because intermediate representations are reused across different tasks.

On Question 2: Convolution assumes stationarity: the same pattern appears everywhere. Parameter sharing exploits this. Fully-connected layers treat all input positions independently, wasting capacity on learning the same features repeatedly.

On Question 3: O(n²) memory and O(n²) computation. This is why Transformers struggle with very long sequences. Recent work (sparse attention, linear attention) tries to reduce this.

On Question 4: Without √d_k scaling, large d_k makes QK^T have large magnitudes, making softmax nearly one-hot (gradients vanish). The scaling keeps attention scores in a reasonable range.

On Question 5: This is an open research question. Networks may have learned brittle correlations rather than true semantic understanding. Adversarial robustness remains a major challenge and a frontier of ML research.
