1. The Feynman Mental Model: Decision Trees vs. Linear Regression

To truly understand machine learning, imagine two fundamentally different ways of thinking about data:

Decision Tree Thinking: Imagine you're sorting your email. Your mind asks a series of yes/no questions: "Does it contain URGENT?" → If yes, "Is it from the boss?" → If yes, "Action needed." This is a decision tree. It recursively splits data by asking the most informative question at each step. Every prediction follows a path of binary decisions down the tree.

Linear Regression Thinking: Now imagine predicting house prices. You plot each house as a point: (size, price). A linear regression draws a best-fit line through these points. Any new house's price is read directly from that line. There are no questions asked—just a single mathematical formula: price = 0.5×size + 50,000. Completely different cognitive model.

Supervised Learning Fundamentals

Both methods are instances of supervised learning: the agent learns a function from labeled examples (input → output pairs). The key challenge is generalizing: finding patterns that work on unseen data, not just memorizing the training set.

The Hypothesis Space and Inductive Bias

Every learning algorithm has an implicit hypothesis space—the set of functions it can represent. A decision tree can represent any discrete partition of the input space (step functions). Linear regression is limited to linear functions. This constraint is called inductive bias. It's not a weakness—it's how we make learning tractable. Without bias, every hypothesis is equally plausible.

Decision Trees: Ask the Right Questions

Decision trees learn by recursively splitting data on attributes that maximize information gain. At each node, the tree asks: "Which feature tells us most about the output?" This is elegantly rational—like a diagnostic doctor asking the most telling questions first.

Entropy and Information Gain

Entropy measures uncertainty in a set of examples. Pure data (all same label) has entropy 0. Maximum entropy is 1 (50/50 split). When we split on an attribute, we compute how much entropy decreases. This decrease is information gain. The algorithm greedily chooses the split with highest information gain.

The Overfitting Trap

Here's the catch: if a tree grows unchecked, it memorizes training data perfectly (100% accuracy) but fails on new data (maybe 55% accuracy). The tree has learned noise, not signal. This is overfitting. The tree becomes a lookup table instead of a generalizable model. The solution: pruning—remove branches that don't help on held-out validation data.

Linear Regression: The Best-Fit Line

Linear regression finds weights w that minimize prediction error on training data. Instead of asking questions, it solves an optimization problem: minimize the sum of squared errors between predicted and actual outputs.

Gradient Descent: How Learning Happens

The model doesn't "know" the optimal weights upfront. Instead, it starts with random guesses and walks downhill on the error surface, following the negative gradient. At each step: adjust each weight slightly in the direction that reduces error. Repeat until convergence. This is gradient descent, and it's beautifully simple and powerful.

The Bias-Variance Tradeoff

Here's the fundamental tension in machine learning:

  • Bias: If the model is too simple (shallow tree, linear regression on nonlinear data), it underfits. It has high bias—systematic error.
  • Variance: If the model is too complex (deep tree, high-degree polynomial), it fits noise. Small changes in training data cause big changes in predictions. High variance.

The sweet spot minimizes both. This tradeoff is not a limitation—it's the fundamental reason learning is possible.

Validation and Testing

Never test on training data. Always use cross-validation: split data into folds, train on some, test on others. This gives an honest estimate of generalization. It also helps you choose hyperparameters (tree depth, learning rate) without overfitting to the test set.

Ensemble Methods

Random Forests train many shallow trees on random subsets of data, then average their predictions. Why does this work? Each tree is biased but has low variance (it's shallow). Averaging uncorrelated biased estimators reduces variance. Boosting goes further: train trees sequentially, each one focusing on examples the previous trees got wrong. The result is a powerful combined predictor.

2. Mathematical Ascent

Decision Tree Mathematics

Entropy of a label distribution:

$$H(V) = -\sum_k P(v_k) \log_2 P(v_k)$$

Where k is each possible class and P(v_k) is the proportion of examples with class v_k.

Information Gain of an attribute:

$$IG(A) = H(V) - \sum_k \frac{|S_k|}{|S|} H(S_k)$$

Where S_k is the subset of examples where attribute A has value k. Information gain is entropy before split minus weighted average entropy after split.

Splitting criterion: At each node, choose the attribute A that maximizes IG(A). This is the most informative question to ask.

Linear Regression Mathematics

Linear model (hypothesis):

$$h_\mathbf{w}(\mathbf{x}) = w_0 + w_1 x_1 + \cdots + w_n x_n = \mathbf{w} \cdot \mathbf{x}$$

Where w is the weight vector, x is the input vector, and we include a bias term w_0.

Mean Squared Error (MSE) loss function:

$$L(\mathbf{w}) = \frac{1}{N} \sum_{i=1}^{N} (y_i - h_\mathbf{w}(\mathbf{x}_i))^2$$

The average squared error over all training examples. Lower is better. This is the objective function the algorithm minimizes.

Gradient Descent update rule:

$$w_j \leftarrow w_j - \alpha \frac{\partial L}{\partial w_j}$$

At each iteration, adjust each weight proportionally to the negative gradient. α is the learning rate (how big a step to take). The gradient tells us which direction reduces error.

The Bias-Variance Decomposition

Expected prediction error:

$$E[(y - \hat{y})^2] = \text{Bias}^2(\hat{y}) + \text{Variance}(\hat{y}) + \text{Noise}$$

Bias: Systematic error from the model being too simple. Variance: Sensitivity to changes in training data. Noise: Irreducible error in the data itself. Learning tries to minimize bias + variance.

Regularization

Regularized loss (L2/Ridge):

$$L(\mathbf{w}) = \frac{1}{N} \sum_{i=1}^{N} (y_i - h_\mathbf{w}(\mathbf{x}_i))^2 + \lambda ||\mathbf{w}||^2$$

Add a penalty for large weights. λ controls the penalty strength. Large weights are encouraged to shrink, reducing overfitting. This is like penalizing complexity.

Logistic Regression for Classification

Sigmoid activation function:

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Maps any input z to a probability in [0, 1]. Apply sigmoid to the linear model output to get a probability of the positive class. Use cross-entropy loss instead of MSE for classification.

3. Clean-Code Implementation

Here's a working implementation of decision trees and linear regression that embodies the learning principles:

import math
from collections import Counter

class DecisionTree:
    """
    Decision Tree learner: builds a tree of if-then rules from data.
    The rational agent learns by finding the most informative questions
    to ask, maximizing information gained at each split.
    """
    def __init__(self, max_depth=10, min_samples=2):
        self.max_depth = max_depth
        self.min_samples = min_samples
        self.tree = None

    def entropy(self, labels):
        """Compute Shannon entropy of a label distribution."""
        counts = Counter(labels)
        total = len(labels)
        return -sum((c/total) * math.log2(c/total) for c in counts.values() if c > 0)

    def information_gain(self, data, labels, feature_idx):
        """
        Compute information gain from splitting on feature_idx.
        IG = H(parent) - sum(weight * H(child))
        """
        parent_entropy = self.entropy(labels)
        values = set(row[feature_idx] for row in data)
        child_entropy = 0
        for val in values:
            subset_labels = [l for d, l in zip(data, labels)
                           if d[feature_idx] == val]
            weight = len(subset_labels) / len(labels)
            child_entropy += weight * self.entropy(subset_labels)
        return parent_entropy - child_entropy

    def fit(self, data, labels, depth=0):
        """
        Recursively build the decision tree.
        Stop when: all labels are same, max depth reached,
        or too few samples.
        """
        # Base case: pure node (all same label)
        if len(set(labels)) == 1:
            return labels[0]

        # Base case: stop by depth or sample count
        if depth >= self.max_depth or len(data) < self.min_samples:
            return Counter(labels).most_common(1)[0][0]

        # Find the best feature to split on
        n_features = len(data[0])
        best_feature = max(
            range(n_features),
            key=lambda f: self.information_gain(data, labels, f)
        )

        # Check if best split is worthless (no gain)
        if self.information_gain(data, labels, best_feature) == 0:
            return Counter(labels).most_common(1)[0][0]

        # Recursively build subtrees
        tree = {'feature': best_feature, 'children': {}}
        values = set(row[best_feature] for row in data)
        for val in values:
            subset = [(d, l) for d, l in zip(data, labels)
                     if d[best_feature] == val]
            sub_data, sub_labels = zip(*subset)
            tree['children'][val] = self.fit(
                list(sub_data), list(sub_labels), depth+1
            )

        self.tree = tree
        return tree

    def predict(self, x, tree=None):
        """Predict by following the tree path based on x's values."""
        if tree is None:
            tree = self.tree
        if not isinstance(tree, dict):
            return tree
        feature_val = x[tree['feature']]
        if feature_val in tree['children']:
            return self.predict(x, tree['children'][feature_val])
        return None


class LinearRegression:
    """
    Linear Regression: finds the best-fit hyperplane.
    The agent minimizes prediction error by adjusting weights via
    gradient descent. Each iteration, move in direction of steepest
    descent on the error surface.
    """
    def __init__(self, learning_rate=0.01, n_iterations=1000):
        self.lr = learning_rate
        self.n_iter = n_iterations
        self.weights = None
        self.bias = None

    def fit(self, X, y):
        """
        Train via gradient descent.
        Adjust weights to minimize MSE loss.
        """
        n_samples, n_features = len(X), len(X[0])
        self.weights = [0.0] * n_features
        self.bias = 0.0

        for iteration in range(self.n_iter):
            for i in range(n_samples):
                # Forward pass: compute prediction
                prediction = (
                    sum(self.weights[j] * X[i][j]
                        for j in range(n_features)) + self.bias
                )

                # Compute error
                error = prediction - y[i]

                # Gradient descent: update weights
                # dL/dw_j = (2/N) * error * x_j
                for j in range(n_features):
                    self.weights[j] -= (
                        self.lr * (2/n_samples) * error * X[i][j]
                    )

                # Update bias
                # dL/db = (2/N) * error
                self.bias -= self.lr * (2/n_samples) * error

    def predict(self, X):
        """Predict using the fitted model."""
        return [
            sum(self.weights[j]*x[j] for j in range(len(x))) + self.bias
            for x in X
        ]


# Example usage
if __name__ == "__main__":
    # Decision Tree example
    data = [
        [2, 0, 0],  # 2 legs, doesn't fly, doesn't swim -> mammal
        [2, 1, 0],  # 2 legs, flies, doesn't swim -> bird
        [4, 0, 0],  # 4 legs, doesn't fly, doesn't swim -> mammal
        [0, 0, 1],  # no legs, doesn't fly, swims -> fish
    ]
    labels = ['mammal', 'bird', 'mammal', 'fish']

    dt = DecisionTree(max_depth=3)
    dt.fit(data, labels)
    print("Tree prediction:", dt.predict([4, 0, 0]))

    # Linear Regression example
    X_train = [[1], [2], [3], [4], [5]]
    y_train = [2, 4, 6, 8, 10]

    lr = LinearRegression(learning_rate=0.01, n_iterations=100)
    lr.fit(X_train, y_train)
    print("Regression prediction:", lr.predict([[6]]))

4. Socratic Sandbox: Critical Thinking Questions

A decision tree is trained on medical data to predict whether a patient has the flu. The tree achieves 99% accuracy on training data but only 55% on new patients—barely better than random guessing.

Question 1: Identify the Problem

The tree has 99% training accuracy but 55% test accuracy. Name this problem and explain what happened to the tree's structure. Hint: Consider what the tree might be learning instead of generalizable patterns.

Question 2: Bias-Variance Tradeoff

Which would help more: (a) collecting 10× more training data, or (b) limiting tree depth to 3? Explain using bias and variance. Why does one address the root cause while the other might not?

Question 3: Comparing Models

A linear regression model on the same data gets 70% accuracy on both train and test. Is it better or worse than the decision tree? What does this comparison reveal about the bias-variance tradeoff?

Question 4: Information Gain Pitfall

When information gain chose "Patient ID" as the most informative feature at the root, why is this terrible? What does this reveal about a limitation of the IG splitting criterion? How would you fix it?

Question 5: Ensemble Power

If you ensemble 100 shallow decision trees (each trained on random subsets of data), averaging their predictions gives a Random Forest that achieves 92% test accuracy. Why does averaging many weak predictors outperform one strong (overfitted) tree? Think about correlation.
