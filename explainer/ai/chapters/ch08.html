The Librarian's Card Catalog

Imagine a vast library maintained by a meticulous librarian. Propositional logic is like having a simple checklist of yes/no facts: "It's raining," "The library is open." You can combine these facts, but you cannot talk about specific books, authors, or relationships between them.

First-order logic is like a sophisticated card catalog system. Instead of isolated facts, you have:

Objects

The things in your world: books, authors, shelves. In FOL, constants like John, WarAndPeace refer to specific objects.

Properties (Unary Predicates)

Attributes of objects. Novel(WarAndPeace) asserts that War and Peace is a novel—like category labels on catalog cards.

Relations (Binary+ Predicates)

Connections between objects. Wrote(Tolstoy, WarAndPeace) captures that Tolstoy wrote War and Peace—like cross-reference cards linking catalog entries.

Functions

A unique lookup mechanism. Author(WarAndPeace) = Tolstoy returns exactly one object—every book has exactly one primary author.

Variables & Quantifiers

Instead of listing every book individually, you write general rules using variables. ∀x (for all) and ∃x (there exists) let you express patterns that apply across the entire domain.

Example rule: "All novels have an author" becomes:

$$\forall x \; Novel(x) \Rightarrow \exists y \; Wrote(y, x)$$

Key insight: First-order logic lets us talk about objects, their properties, and relationships between them using variables and quantifiers. Propositional logic cannot express "It's raining in every city that has a harbor" because it has no notion of quantification over objects.

Mathematical Ascent: Syntax & Semantics

Now we formalize the grammar and interpretation of first-order logic.

BNF Grammar (Figure 8.3)

\begin{align} Sentence &\to AtomicSentence \mid ComplexSentence \\ AtomicSentence &\to Predicate(Term, \ldots) \mid Term = Term \\ Term &\to Function(Term, \ldots) \mid Constant \mid Variable \\ ComplexSentence &\to \neg Sentence \mid (Sentence \land Sentence) \\ &\mid (Sentence \lor Sentence) \mid (Sentence \Rightarrow Sentence) \\ &\mid (Sentence \Leftrightarrow Sentence) \\ &\mid \forall Variable, \ldots \; Sentence \\ &\mid \exists Variable, \ldots \; Sentence \\ Constant &\to John, Mary, 3, \ldots \\ Variable &\to x, y, z, \ldots \\ Predicate &\to Before, Loves, Brother, \ldots \\ Function &\to Mother, Father, Plus, \ldots \end{align}

Semantics: Models & Interpretations

A model $\mathcal{M}$ consists of:

  • Domain $D$: A non-empty set of objects (e.g., $D = \{John, Mary, Richard\}$)
  • Interpretation $I$: A mapping from symbols to objects/relations:
    • Constants: $I(John) = \text{John} \in D$ (a specific object)
    • Predicates: $I(Loves) \subseteq D \times D$ (a relation, set of tuples)
    • Functions: $I(Mother) : D \to D$ (a function mapping objects to objects)

Universal Quantifier

$\forall x \; P(x)$ is true in model $\mathcal{M}$ if and only if $P(x)$ is true under every extended interpretation that binds $x$ to some object in $D$.

$$\forall x \; P(x) \text{ is true in } \mathcal{M} \iff \text{for all } d \in D, \; \mathcal{M} \models P(x \gets d)$$

Existential Quantifier

$\exists x \; P(x)$ is true if there exists at least one object in $D$ that makes $P(x)$ true.

$$\exists x \; P(x) \text{ is true in } \mathcal{M} \iff \text{there exists } d \in D \text{ such that } \mathcal{M} \models P(x \gets d)$$

De Morgan's Laws for Quantifiers

\begin{align} \neg \forall x \; P &\equiv \exists x \; \neg P \\ \neg \exists x \; P &\equiv \forall x \; \neg P \\ \forall x \; P &\equiv \neg \exists x \; \neg P \\ \exists x \; P &\equiv \neg \forall x \; \neg P \end{align}

Common Patterns with Quantifiers

Universal + Implication (∀ ⇒)

∀x King(x) ⇒ Person(x) — "All kings are persons."

✓ Correct pattern: Use implication with universal quantifier. Objects that aren't kings satisfy the rule vacuously (false premise ⇒ true).

Existential + Conjunction (∃ ∧)

∃x Crown(x) ∧ OnHead(x, John) — "There is a crown on John's head."

✓ Correct pattern: Use conjunction with existential. We assert that something both is a crown AND is on John's head.

⚠ Warning: Universal + Conjunction

∀x King(x) ∧ Person(x) — WRONG!

This says "For all x, x is a king AND x is a person"—everything in the domain is both a king and a person. Too strong!

⚠ Warning: Existential + Implication

∃x Crown(x) ⇒ OnHead(x, John) — WRONG!

This is vacuously true if anything isn't a crown (the implication's premise is false, making it true). Too weak!

Nested Quantifiers

Quantifier order matters critically:

\forall x \; \exists y \; Loves(x, y)

"For every person x, there exists some person y that x loves." Each person loves at least one (possibly different) person.

\exists y \; \forall x \; Loves(x, y)

"There exists a person y such that every person x loves y." One specific person is loved by everyone.

These are NOT equivalent! The first allows $y$ to depend on $x$; the second does not.

Equality

The equality relation $=$ can appear in atomic sentences:

Father(John) = Henry

Henry is John's father.

\exists x, y \; Brother(x, Richard) \land Brother(y, Richard) \land \neg(x = y)

Richard has at least two brothers (two distinct objects x and y).

Knowledge Base: TELL/ASK Interface

\begin{align} TELL(KB, \; King(John)) \\ TELL(KB, \; \forall x \; King(x) \Rightarrow Person(x)) \\ ASK(KB, \; Person(John)) &\to \text{true} \\ ASKVARS(KB, \; Person(x)) &\to \{x/John\}, \{x/Richard\}, \ldots \end{align}

$TELL$ adds facts and rules to the KB. $ASK$ queries whether a sentence is entailed. $ASKVARS$ returns all substitutions (bindings) that satisfy a query with variables.

Example: Kinship Domain Axioms

\begin{align} \forall m, c \; Mother(c) = m &\Leftrightarrow Female(m) \land Parent(m, c) \\ \forall w, h \; Husband(h, w) &\Leftrightarrow Male(h) \land Spouse(h, w) \\ \forall p, c \; Parent(p, c) &\Leftrightarrow Child(c, p) \end{align}

Implementation: FOL with Unification

Here's a working knowledge base implementation with the unification algorithm (Robinson's algorithm) and forward chaining inference.

# ── First-Order Logic: Knowledge Base with Unification ──

from dataclasses import dataclass, field
from typing import Dict, List, Optional, Set, Tuple, Union

# ── Core Data Structures ──

@dataclass(frozen=True)
class Constant:
    """A constant symbol referring to a specific object."""
    name: str
    def __repr__(self): return self.name

@dataclass(frozen=True)
class Variable:
    """A variable symbol that can be bound to any object."""
    name: str
    def __repr__(self): return f"?{self.name}"

@dataclass(frozen=True)
class Function:
    """A function applied to terms, e.g., Father(John)."""
    name: str
    args: Tuple
    def __repr__(self): return f"{self.name}({', '.join(map(str, self.args))})"

Term = Union[Constant, Variable, Function]

@dataclass(frozen=True)
class Predicate:
    """An atomic sentence: predicate applied to terms."""
    name: str
    args: Tuple[Term, ...]
    def __repr__(self):
        return f"{self.name}({', '.join(map(str, self.args))})"

@dataclass(frozen=True)
class Rule:
    """A universally quantified Horn clause: premises => conclusion.
    Represents: ∀ vars . (premise₁ ∧ premise₂ ∧ ...) ⇒ conclusion
    """
    premises: Tuple[Predicate, ...]
    conclusion: Predicate
    def __repr__(self):
        prems = " ∧ ".join(map(str, self.premises))
        return f"{prems} ⇒ {self.conclusion}"


# ── Unification (Robinson's Algorithm) ──

Substitution = Dict[Variable, Term]

def unify(x: Term, y: Term, theta: Optional[Substitution] = None) -> Optional[Substitution]:
    """Unify two terms, returning a substitution or None on failure.

    Core idea: find variable bindings that make x and y identical.
    If unification succeeds, returns θ such that SUBST(θ, x) = SUBST(θ, y).
    If unification fails, returns None.
    """
    if theta is None:
        theta = {}
    if theta is None:  # failure propagated from nested unification
        return None
    if x == y:
        return theta
    if isinstance(x, Variable):
        return unify_var(x, y, theta)
    if isinstance(y, Variable):
        return unify_var(y, x, theta)
    if isinstance(x, Predicate) and isinstance(y, Predicate):
        if x.name != y.name or len(x.args) != len(y.args):
            return None
        for a, b in zip(x.args, y.args):
            theta = unify(a, b, theta)
            if theta is None:
                return None
        return theta
    if isinstance(x, Function) and isinstance(y, Function):
        if x.name != y.name or len(x.args) != len(y.args):
            return None
        for a, b in zip(x.args, y.args):
            theta = unify(a, b, theta)
            if theta is None:
                return None
        return theta
    return None

def unify_var(var: Variable, x: Term, theta: Substitution) -> Optional[Substitution]:
    """Unify a variable with a term, handling already-bound variables."""
    if var in theta:
        return unify(theta[var], x, theta)
    if isinstance(x, Variable) and x in theta:
        return unify(var, theta[x], theta)
    # Occur check: prevent infinite structures like ?x = f(?x)
    if occurs_check(var, x, theta):
        return None
    return {**theta, var: x}

def occurs_check(var: Variable, x: Term, theta: Substitution) -> bool:
    """Check if var occurs inside x (prevents infinite terms)."""
    if var == x:
        return True
    if isinstance(x, Variable) and x in theta:
        return occurs_check(var, theta[x], theta)
    if isinstance(x, (Function, Predicate)):
        return any(occurs_check(var, arg, theta) for arg in x.args)
    return False

def substitute(term: Term, theta: Substitution) -> Term:
    """Apply a substitution to a term."""
    if isinstance(term, Constant):
        return term
    if isinstance(term, Variable):
        if term in theta:
            return substitute(theta[term], theta)
        return term
    if isinstance(term, Function):
        return Function(term.name, tuple(substitute(a, theta) for a in term.args))
    if isinstance(term, Predicate):
        return Predicate(term.name, tuple(substitute(a, theta) for a in term.args))
    return term


# ── FOL Knowledge Base with Forward Chaining ──

class FOLKnowledgeBase:
    """A first-order logic knowledge base using forward chaining.

    Supports TELL (assertions + rules) and ASK (queries).
    Uses the TELL/ASK interface from Section 8.3.1.
    """
    def __init__(self):
        self.facts: Set[Predicate] = set()
        self.rules: List[Rule] = []

    def tell(self, sentence):
        """Add a fact or rule to the knowledge base."""
        if isinstance(sentence, Predicate):
            self.facts.add(sentence)
        elif isinstance(sentence, Rule):
            self.rules.append(sentence)

    def ask(self, query: Predicate) -> List[Substitution]:
        """Query the KB using forward chaining.
        Returns list of substitutions that make query true."""
        all_facts = self.forward_chain()
        results = []
        for fact in all_facts:
            theta = unify(query, fact)
            if theta is not None:
                results.append(theta)
        return results

    def forward_chain(self) -> Set[Predicate]:
        """Forward chaining: repeatedly apply rules until fixed point.

        Implements FOL-FC-ASK from Section 9.3.
        Returns all facts derivable from initial facts and rules.
        """
        known = set(self.facts)
        changed = True
        while changed:
            changed = False
            for rule in self.rules:
                for theta in self._match_premises(rule.premises, known, {}):
                    new_fact = substitute(rule.conclusion, theta)
                    if new_fact not in known:
                        known.add(new_fact)
                        changed = True
        return known

    def _match_premises(self, premises, facts, theta):
        """Recursively match all premises against known facts."""
        if not premises:
            yield theta
            return
        first, rest = premises[0], premises[1:]
        first_sub = substitute(first, theta)
        for fact in facts:
            new_theta = unify(first_sub, fact, dict(theta))
            if new_theta is not None:
                yield from self._match_premises(rest, facts, new_theta)


# ── Example: Royal Family Domain ──

if __name__ == "__main__":
    kb = FOLKnowledgeBase()

    # Constants
    John = Constant("John")
    Richard = Constant("Richard")

    # TELL facts
    kb.tell(Predicate("King", (John,)))
    kb.tell(Predicate("Person", (Richard,)))

    # TELL rule: ∀x King(x) ⇒ Person(x)
    x = Variable("x")
    kb.tell(Rule(
        premises=(Predicate("King", (x,)),),
        conclusion=Predicate("Person", (x,))
    ))

    # ASK: Person(John)?
    results = kb.ask(Predicate("Person", (John,)))
    print(f"Person(John)? {len(results) > 0}")  # True

    # ASKVARS: Person(?q)?
    results = kb.ask(Predicate("Person", (Variable("q"),)))
    for r in results:
        print(f"  Person found: {r}")  # John, Richard

Key algorithms explained:

  • Unification (Robinson's algorithm): Finds variable bindings that make two terms identical. The occur check prevents infinite structures like $?x = f(?x)$.
  • Forward chaining: Repeatedly applies rules to derive new facts until no new facts can be inferred (fixed point). Implements exhaustive inference from given facts and rules.
  • Pattern matching: Recursively matches rule premises against known facts, collecting all valid variable substitutions.

Socratic Sandbox: Common Pitfalls

▼ Q1: I wrote ∀x King(x) ∧ Person(x) to say "all kings are persons," but my KB thinks everything is both a king AND a person. What went wrong?

Answer: The quantifier scopes over everything to its right. Your sentence is parsed as:

$$\forall x \; [King(x) \land Person(x)]$$

This means "for every x, x is a king AND x is a person"—everything in the domain must be both a king and a person. Way too strong!

The correct form uses implication:

$$\forall x \; [King(x) \Rightarrow Person(x)]$$

With implication, objects that aren't kings satisfy the rule vacuously: $\neg King(x)$ makes the antecedent false, which makes the entire implication true. Only actual kings are asserted to be persons.

Pattern: ∀ with ⇒ is the natural choice for "all A are B" statements.

▼ Q2: My existential query ∃x Crown(x) ⇒ OnHead(x, John) returns true even though John has no crown. How?

Answer: The sentence $\exists x \; [Crown(x) \Rightarrow OnHead(x, John)]$ is satisfied if there exists any object for which the implication holds.

Pick any non-crown object (say, a book). The implication becomes: $Crown(\text{book}) \Rightarrow OnHead(\text{book}, John)$. Since $Crown(\text{book})$ is false, the implication is trivially true (false premise makes it true).

So the existential is satisfied with no actual crown on John's head!

The correct form uses conjunction:

$$\exists x \; [Crown(x) \land OnHead(x, John)]$$

Now we assert: there exists something that is both a crown AND on John's head.

Pattern: ∃ with ∧ is the natural choice for "there is an A that is B" statements.

▼ Q3: I'm confused about ∀x ∃y Loves(x,y) vs ∃y ∀x Loves(x,y). Aren't they the same?

Answer: No! Quantifier order is critical. They have opposite meanings.

$$\forall x \; \exists y \; Loves(x, y)$$

"For each person, there exists someone they love."

Each person has a beloved, but different people can love different people. The choice of $y$ depends on $x$.

$$\exists y \; \forall x \; Loves(x, y)$$

"There exists a specific person loved by everyone."

One fixed person is universally adored. No dependence: $y$ is chosen first, then all people love that same $y$.

The second is strictly stronger: if everyone loves one specific person, then certainly everyone loves someone. But the converse isn't true.

Key insight: In $\forall \ldots \exists$, the existential's witness can depend on the universal variable. In $\exists \ldots \forall$, the witness is fixed globally.

▼ Q4: My unification algorithm goes into an infinite loop when I try to unify ?x with f(?x). What's happening?

Answer: Without an occurs check, unifying $?x$ with $f(?x)$ would naively produce the binding $?x = f(?x)$. When you try to substitute $?x$ in expressions, you get:

  • $?x = f(?x)$
  • $f(?x) = f(f(?x))$
  • $f(f(?x)) = f(f(f(?x)))$
  • ... ad infinitum

This creates an infinitely nested term structure, which is invalid in classical FOL (terms must be finite).

Solution: Occurs Check

Before binding a variable $v$ to a term $t$, check if $v$ occurs anywhere inside $t$. If it does, unification fails.

\text{OccursCheck}(x, f(x)) = \text{true} \implies \text{unification fails}

While the occurs check is O(n) per unification step, omitting it leads to unsound inference and potential infinite loops. Standard implementations include it.

▼ Q5: My forward chaining derives facts forever and never reaches a fixed point. When does this happen?

Answer: Forward chaining with function symbols can generate infinitely many distinct terms. Consider:

\forall x \; P(x) \Rightarrow P(f(x))

Starting with $P(A)$:

  • Derive: $P(f(A))$
  • Derive: $P(f(f(A)))$
  • Derive: $P(f(f(f(A))))$
  • ... forever

This is a fundamental limitation of first-order logic:

First-order logic entailment is only semidecidable. If a sentence is entailed, an algorithm will eventually find a proof. But if a sentence is not entailed, the algorithm may run forever, never concluding it's false.

Practical solution: Restrict to Datalog

Disallow function symbols entirely, restricting to ground terms (constants only). This makes inference decidable and terminating, though less expressive than full FOL.
