Chapter 7: Kolmogorov Complexity

The shortest program that prints the universe

The Essence

Feynman's Insight: What is the shortest computer program that prints the complete works of Shakespeare? Whatever that length is — that's the Kolmogorov complexity of Shakespeare.

If no program shorter than the text itself can produce it, then the text is random — incompressible. This is the ultimate, individual-sequence notion of information content. Not "how much information on average," but "how much information in this specific string."

Kolmogorov complexity captures something profound: the inherent algorithmic content of any object. A string that has deep structure — patterns, repetitions, mathematical relationships — can be generated by a short program. A truly random string cannot be compressed at all.

Models of Computation (7.1)

Kolmogorov complexity is defined relative to a choice of universal computer. The formal definition:

$$K_U(x) = \min \{ |p| : U(p) = x \}$$

Here, $U$ is a universal Turing machine, $p$ is a program (a bit string), and $|p|$ is its length in bits. We search over all programs that halt and output exactly $x$, and pick the shortest.

The Invariance Theorem

Does the choice of universal computer matter? Yes, but only by a constant:

$$K_U(x) = K_V(x) + O(1)$$

For any two universal computers $U$ and $V$, their Kolmogorov complexities differ by at most a constant number of bits. This constant depends on translating between the two machines, but grows no further with $|x|$.

Consequence: We can speak of "the" Kolmogorov complexity $K(x)$ without worrying about which computer we're using. The choice is irrelevant asymptotically.

Kolmogorov Complexity and Shannon Entropy (7.3)

Here's where the two notions of information meet:

$$\mathbb{E}[K(X_1, \ldots, X_n)] \approx nH(X)$$

For i.i.d. sequences drawn from a probability distribution with entropy $H(X)$, the expected Kolmogorov complexity over $n$ symbols is approximately $nH(X)$ bits.

Key Insight: Kolmogorov complexity is the individual-sequence analog of Shannon entropy. Shannon entropy asks: "On average, how many bits per symbol do we need to encode outcomes from this distribution?" Kolmogorov complexity asks: "For this specific sequence, what's the shortest program that produces it?"

For "most" sequences, $K(x) \approx H(X) \cdot n$ — they are typical.

Algorithmic Randomness (7.5)

A sequence is algorithmically random if and only if its Kolmogorov complexity is close to its length:

$$K(x) \geq |x| - c$$

where $c$ is a small constant. In other words, the string is incompressible. No short program can reproduce it.

Most Strings are Random

Here's a counting argument: There are $2^n$ possible bit strings of length $n$. But there are only $2^{n-1} - 1$ programs shorter than $n$ bits. So most strings of length $n$ cannot be the output of any program shorter than $n$ bits.

Therefore: Most sequences are algorithmically random. They have no structure; they cannot be compressed.

The Halting Problem & Chaitin's Omega (7.7, 7.8)

Kolmogorov complexity connects to one of computer science's deepest problems: the Halting Problem. Given a program, does it eventually halt or run forever? In general, this is undecidable.

Chaitin's Omega: The Halting Probability

Define $\Omega$ as the probability that a randomly chosen program halts:

$$\Omega = \sum_{\text{p halts}} 2^{-|p|}$$

This is a sum over all programs that halt, weighted by $2^{-|p|}$ (the "probability" of a program of length $|p|$).

  • Well-defined: The sum converges (always less than 1).
  • Algorithmically random: $K(\Omega) \approx |\Omega|$. The bits of Omega are incompressible.
  • Universal: The digits of $\Omega$ encode the answers to all halting problems.

The existence of $\Omega$ shows that algorithmic randomness is not just theoretical—it has concrete manifestations in fundamental computability questions.

Occam's Razor (7.10)

Kolmogorov complexity provides a formal justification for one of science's guiding principles:

Principle of Parsimony: Among competing hypotheses that explain the observed data equally well, prefer the simplest one.

In the language of Kolmogorov complexity: among all models (programs) that generate the data, choose the one with the smallest description length. This is not arbitrary preference for simplicity—it's a mathematical principle:

  • A simple model (short program) generalizes better to new data.
  • Complex models risk overfitting, encoding noise as if it were signal.
  • Kolmogorov complexity formalizes what "simplicity" means: algorithmic brevity.

Code Lab: Compression Ratio Estimation

Below is a Python script that estimates Kolmogorov complexity by computing compression ratios. The script tries several heuristics: run-length encoding, pattern matching, and entropy calculation.

import math from collections import Counter def estimate_kolmogorov_complexity(s): """ Estimate Kolmogorov complexity by: 1. Trying run-length encoding 2. Computing Shannon entropy 3. Detecting repeating patterns """ n = len(s) # Run-length encoding estimate rle_length = run_length_encode(s) # Shannon entropy freq = Counter(s) entropy = -sum((count/n) * math.log2(count/n) for count in freq.values()) # Estimate: overhead + entropy-based bits overhead = 10 # for describing the encoding estimated_bits = overhead + n * entropy return { 'string': s, 'length': n, 'rle_length': rle_length, 'entropy': entropy, 'estimated_complexity': estimated_bits, 'compression_ratio': estimated_bits / n } def run_length_encode(s): """Simple RLE: count consecutive chars.""" if not s: return 0 encoded = [] i = 0 while i < len(s): char = s[i] count = 1 while i+count < len(s) and s[i+count] == char: count += 1 # Encode as: count + char encoded.append(f"{count}{char}") i += count return sum(len(e) for e in encoded) # Test cases tests = [ "01010101", "01101001", "000000000000", "abcabcabc" ] for test in tests: result = estimate_kolmogorov_complexity(test) print(f"String: {result['string']}") print(f" Length: {result['length']}") print(f" Entropy: {result['entropy']:.3f}") print(f" Est. Complexity: {result['estimated_complexity']:.1f} bits") print(f" Compression: {result['compression_ratio']:.2%}") print()

Key Functions

  • Run-Length Encoding: Counts consecutive identical characters. Patterns like "00000000" compress well.
  • Shannon Entropy: Measures the information content based on character frequency.
  • Estimated Complexity: Combines overhead with entropy-weighted length.

Caveat: This is an estimate. True Kolmogorov complexity is uncomputable. We can only bound it.

Socratic Sandbox: Explore Your Understanding

Reflect on these questions to deepen your grasp of Kolmogorov complexity.

Q1: Why is Kolmogorov complexity uncomputable?

To compute $K(x)$, you'd need to search through all possible programs, run each one, and check if it produces $x$. But some programs never halt. The Halting Problem is undecidable—there's no algorithm that can determine whether a program halts or runs forever. Therefore, no algorithm can definitively compute $K(x)$.

You can compute upper bounds on $K(x)$ (just take the length of $x$ itself as a trivial program that prints $x$). You can compute lower bounds using compression techniques. But you cannot compute the exact value in the general case.

Q2: If most strings are incompressible, why do we live in a compressible world?

Excellent question! While most random strings are incompressible, the data we encounter—language, images, DNA, physics—has structure. A sentence in English is far less random than a random string of characters. DNA sequences have evolutionary correlations. Physics equations emerge because the universe has simple, compressible laws.

We are biased toward observing low-Kolmogorov-complexity data because high-complexity (random) data is less likely to contain meaning or patterns. Evolution, language evolution, and scientific discovery are all processes that seek out and exploit structure—i.e., low complexity.

Q3: Can an object have high Shannon entropy but low Kolmogorov complexity?

Yes! Here's an example: Consider the string "01010101...01" repeated $2^{30}$ times. When measured in aggregate (computing the overall distribution), this has entropy approaching $H = 1$ bit per symbol (roughly balanced 0s and 1s). But Kolmogorov complexity is very low: just ~50 bits (a program that says "repeat '01' $2^{30}$ times").

The difference: Shannon entropy is a statistical property; Kolmogorov complexity is about compressibility of the specific sequence. A large i.i.d. sample from a high-entropy distribution will have high average Kolmogorov complexity. But a highly structured, repetitive sequence can have high local entropy statistics yet be highly compressible.

Q4: Does Chaitin's Omega have any practical applications?

Omega is profoundly theoretical rather than practical. Its bits are incompressible and encode the answers to all halting problems—making it uncomputable and unknowable in any finite time. However, it has deep conceptual value:

  • Metamathematics: Shows that mathematics has true randomness and incompleteness.
  • Provability limits: Not all digits of $\Omega$ are provable within any formal system.
  • Perspective: Demonstrates that randomness and incomputability are fundamental, not mere artifacts of our algorithms.

Its main application is philosophical and theoretical—reshaping how we think about information, proof, and computation.

Summary

Kolmogorov complexity $K(x)$ is the length of the shortest computer program that produces string $x$. It formalizes the intuition that information content is about compressibility.

Key insights:

  • $K(x)$ is defined relative to a universal computer, but the choice matters only up to a constant (Invariance Theorem).
  • For typical sequences, $\mathbb{E}[K(X^n)] \approx nH(X)$—Kolmogorov complexity and Shannon entropy agree on average.
  • A string is algorithmically random if $K(x) \geq |x| - c$—it is incompressible.
  • Most strings are random by a simple counting argument.
  • The Halting Problem (undecidable) makes Kolmogorov complexity itself uncomputable.
  • Chaitin's Omega, the halting probability, is algorithmically random and encodes all halting problem answers.
  • Kolmogorov complexity formalizes Occam's Razor: prefer shorter (simpler) models.
