Chapter 13: Rate Distortion Theory

Richard Feynman

"Every MP3 file is a proof that perfect fidelity isn't always worth the cost. Rate distortion theory answers: for a given quality tolerance D, what's the minimum bit rate R? This is the fundamental limit of lossy compression — and it traces a beautiful curve from zero distortion (infinite rate) to maximum distortion (zero rate)."

Part I: Quantization and Distortion Measures

In lossless compression, we preserve the source exactly. But in the real world, we often tolerate some loss of fidelity. An MP3 discards frequencies your ear can't hear. A JPEG blurs fine details. A video codec drops frames.

The fundamental question: Given that we tolerate average distortion D, what is the minimum bit rate R required to describe the source?

Quantization: From Continuous to Discrete

Quantization maps continuous values to a discrete alphabet. Instead of transmitting exact pixel values (256 possibilities per channel in 8-bit), we might use only 4 levels, cutting the rate by half at the cost of precision.

Discrete alphabet: X̂ ∈ {1, 2, ..., M}

Quantizer: Q: X → X̂

Distortion Measures

A distortion measure d(x, x̂) quantifies how much error we incur when representing symbol x by x̂.

Common Distortion Measures

Hamming Distortion (Binary Symbols)

$$d_H(x, \hat{x}) = \begin{cases} 0 & \text{if } x = \hat{x} \\ 1 & \text{if } x \neq \hat{x} \end{cases}$$

Counts errors: one bit wrong = distortion 1.

Squared Error (Continuous Signals)

$$d_{SE}(x, \hat{x}) = (x - \hat{x})^2$$

Large errors are heavily penalized. Used for audio, images, and most signal processing.

Expected Distortion

$$D = \mathbb{E}[d(X, \hat{X})]$$

The constraint: we require average distortion ≤ threshold D.

Part II: The Rate Distortion Function R(D)

Definition

The rate distortion function is the minimum rate (in bits per symbol) needed to achieve average distortion at most D.

$$R(D) = \min_{p(\hat{x}|x): \mathbb{E}[d(X,\hat{X})] \leq D} I(X; \hat{X})$$

We search over all conditional distributions p(x̂|x) that satisfy the distortion constraint, and pick the one minimizing mutual information.

Key Properties

  • R(0) = H(X): Zero distortion means lossless compression — we need the full entropy.
  • R(D) is decreasing: More distortion allowed → less rate needed.
  • R(D) is convex: The curve bends downward. Marginal gains shrink as D increases.
  • R(D_max) = 0: If we tolerate maximum distortion (output constant), zero rate suffices.

Example 1: Binary Symmetric Source, Hamming Distortion

Source: X ∈ {0, 1} with P(X=1) = p. Measure: Hamming distance.

$$R(D) = H(p) - H(D) \quad \text{for } D \leq \min(p, 1-p)$$

where H(D) = -D \log D - (1-D) \log(1-D) is the binary entropy of error rate D.

Intuition

If p = 0.5 (unbiased coin), then H(p) = 1 bit. If we allow D = 0.1 (10% error), then H(D) ≈ 0.47, so R(0.1) ≈ 0.53 bits. We save about half the rate by tolerating 10% errors!

Example 2: Gaussian Source, Squared Error Distortion

Source: X ~ N(0, σ²). Measure: squared error.

$$R(D) = \frac{1}{2} \log \frac{\sigma^2}{D} \quad \text{for } 0 < D \leq \sigma^2$$

  • At D = σ², R(D) = 0: we're allowed average error equal to the source variance, so output the constant mean.
  • At D → 0, R(D) → ∞: need infinite rate for perfect fidelity.
  • Halving distortion costs 0.5 extra bit per symbol.

Part III: Converse and Achievability (Shannon's Theorem)

The Converse: Lower Bound

If you want average distortion ≤ D, you must use at least R(D) bits per symbol on average. No code can beat this limit.

Why?

Suppose your code achieves D with rate less than R(D). Then the mutual information I(X; X̂) is less than R(D), contradicting the definition of R(D) as the minimum.

Achievability: Upper Bound

For any ε > 0, there exists a code with rate R(D) + ε that achieves average distortion D. With large enough blocklength n, we can get arbitrarily close to R(D).

Blocklength Effects

Like channel coding, the gap between achievable rate and R(D) shrinks as blocklength n → ∞. Real systems (MP3, JPEG, H.264) operate above the theoretical limit because they use finite blocklengths.

Part IV: Computing R(D) — Blahut-Arimoto Algorithm

To find the optimal conditional distribution p*(x̂|x) that minimizes I(X; X̂) subject to the distortion constraint, we use an iterative algorithm.

Algorithm Overview

  1. Initialize: Set p(x̂|x) uniformly or randomly.
  2. E-step: Compute p(x̂) = Σ_x p(x) p(x̂|x).
  3. M-step: Update p(x̂|x) using Lagrange multiplier β:

    $$p(x\hat{x}|x) \propto p(\hat{x}) \exp(-\beta d(x, \hat{x}))$$

  4. Check: If distortion ≤ D, done. Else adjust β and repeat.

The parameter β controls the rate-distortion tradeoff. Small β → more distortion, low rate. Large β → little distortion, high rate.

Why It Works

The update rule comes from Lagrangian duality: at optimality, the gradient of mutual information equals the gradient of the constraint, scaled by β. Iterating converges to the optimal p(x̂|x).

Part V: Code Lab — Computing R(D) in Python

Here's a minimal implementation for binary and Gaussian sources:

import numpy as np from scipy.special import entropy # Binary source, Hamming distortion def rd_binary(p, D): """Rate distortion for binary source.""" if D > min(p, 1-p): return 0 # Maximum distortion h_p = -p * np.log2(p + 1e-10) - (1-p) * np.log2(1-p + 1e-10) h_d = -D * np.log2(D + 1e-10) - (1-D) * np.log2(1-D + 1e-10) return max(0, h_p - h_d) # Gaussian source, squared error def rd_gaussian(sigma2, D): """Rate distortion for Gaussian source.""" if D >= sigma2: return 0 if D <= 0: return float('inf') return 0.5 * np.log2(sigma2 / D) # Example: MP3-like scenario p = 0.3 # Source bias D_values = np.linspace(0.001, min(p, 1-p)-0.001, 100) rates = [rd_binary(p, d) for d in D_values] print(f"Binary source, p={p}") print(f" D=0.05: R(D) = {rd_binary(p, 0.05):.2f} bits") print(f" D=0.15: R(D) = {rd_binary(p, 0.15):.2f} bits") sigma2 = 1.0 print(f"\nGaussian source, σ²={sigma2}") print(f" D=0.1: R(D) = {rd_gaussian(sigma2, 0.1):.2f} bits") print(f" D=0.5: R(D) = {rd_gaussian(sigma2, 0.5):.2f} bits")
Output Interpretation

For the binary source with p=0.3, H(p) ≈ 0.88 bits (lossless). Allowing D=0.15 cuts this to ~0.23 bits — saving 75% of bits for just 15% error. This is the power of lossy compression.

Socratic Sandbox: Questions & Insights

Q1: "At what distortion does MP3 operate?"
MP3 targets psychoacoustic distortion — it removes frequencies you can't hear, quieter sounds masked by louder ones, and temporal components below 20 Hz or above 20 kHz. The effective D is tuned to human perception, not mathematical MSE. A 128 kbps MP3 achieves remarkable quality at ~0.125 bits per sample because much of the source variance is imperceptible to human ears.
Q2: "R(D) = 0 at D = σ². What does this mean?"
If you're allowed average squared error equal to the source variance, the optimal encoder is the constant zero (or the mean, equivalently). The decoder needs zero information — it always outputs the same value. This makes sense: if the distortion tolerance matches the typical spread of the signal, describing individual values adds no value. You're better off guessing the mean.
Q3: "Why is R(D) convex and decreasing?"
More distortion tolerance → less discrimination needed → lower rate. Convexity (the curve bends downward) follows from the minimization structure: the first unit of distortion tolerance saves more rate than the last unit, because you can coarse-quantize aggressively without triggering the distortion constraint early on. This is a consequence of the mutual information being jointly convex in p(x̂|x) and the distortion constraint being linear.
Q4: "Can we achieve R(D) with finite blocklength?"
Like channel coding, there's a gap that shrinks as n → ∞. Practical codes (vector quantization, transform coding like DCT in JPEG, temporal prediction in H.264) operate above R(D) because they use modest blocklengths and must remain computationally feasible. The good news: modern codes get within fractions of a bit of R(D), especially for medium-to-high rates.

Applications in Practice

Audio: MP3, AAC, Opus

These codecs use perceptual models to define "imperceptible distortion," then apply Huffman or arithmetic coding to the residual. At 128 kbps (128 bits per 16,000 samples), they achieve transparency for most listeners — not because R(D) theory mandates 128 kbps, but because human audition has high D for imperceptible frequencies.

Images: JPEG, WebP, HEIF

JPEG uses DCT (discrete cosine transform) to decorrelate pixels, then quantizes high-frequency components more coarsely. The quantization table defines the distortion D implicitly; R(D) theory predicts the rate-distortion tradeoff. A quality slider (90 vs. 80 vs. 50) moves along the R(D) curve.

Video: H.264, VP9, AV1

Video codecs apply motion compensation (predicting current frame from previous), then quantize prediction residuals. The quantization parameter (QP) controls the distortion D; R(D) theory explains why halving the quantization step requires ~0.5 extra bits per sample.

Key Takeaways

  • Rate-distortion function R(D) is the minimum rate for average distortion ≤ D.
  • R(0) = H(X) (lossless); R(D_max) = 0 (lossy limit).
  • R(D) is convex, decreasing, and continuous.
  • Binary and Gaussian sources have closed-form expressions.
  • Blahut-Arimoto algorithm computes R(D) iteratively via Lagrangian duality.
  • Real codecs (MP3, JPEG, H.264) exploit perceptual or structural redundancy to approach R(D) with practical blocklengths.
  • The fundamental tradeoff: Lower quality (higher D) → exponentially fewer bits needed. This curve is universal across all distortion measures and sources.
