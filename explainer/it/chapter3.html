Chapter 3: The Asymptotic Equipartition Property

Where randomness reveals order, and typical sequences rule the roost

Mental Model: The Monkey Theorem

Imagine a million monkeys typing randomly. Almost none will produce Shakespeare, but here's the surprise: almost all will produce sequences that look statistically alike.

The Asymptotic Equipartition Property (AEP) says precisely this: among all possible outcomes of a long random sequence, only a tiny fraction—the typical set—carry essentially all the probability. And this tiny fraction has roughly \(2^{nH}\) members.

Think of it like this: in a lottery of \(2^n\) possible outcomes, almost all the winning combinations are clustered in a region of size \(2^{nH}\), where \(H\) is the entropy. The AEP is the mathematical law that makes this order emerge from chaos.

Theorem 3.1.1: The Asymptotic Equipartition Property

Let \(X_1, X_2, \ldots, X_n\) be i.i.d. (independent and identically distributed) random variables with distribution \(p(x)\). Then:

$$-\frac{1}{n} \log p(X_1, \ldots, X_n) \xrightarrow{P} H(X)$$

The sample entropy (negative log-probability divided by sequence length) converges in probability to the true entropy \(H(X)\).

Proof Sketch

The joint probability factors as:

$$p(X_1, \ldots, X_n) = \prod_{i=1}^n p(X_i)$$

Taking logs:

$$-\frac{1}{n} \log p(X_1, \ldots, X_n) = -\frac{1}{n} \sum_{i=1}^n \log p(X_i)$$

By the Weak Law of Large Numbers, as \(n \to \infty\):

$$-\frac{1}{n} \sum_{i=1}^n \log p(X_i) \to E[-\log p(X)] = H(X)$$

Thus, the sample entropy converges to the true entropy with high probability. ∎

The Typical Set

The typical set \(A_\epsilon^{(n)}\) is defined as all sequences whose sample entropy is close to the true entropy:

$$A_\epsilon^{(n)} = \left\{ (x_1, \ldots, x_n) : \left| -\frac{1}{n} \log p(x_1, \ldots, x_n) - H(X) \right| \le \epsilon \right\}$$

Three Key Properties

  1. High Probability: For sufficiently large \(n\), $$\Pr(A_\epsilon^{(n)}) > 1 - \epsilon$$ Almost all sequences are typical.
  2. Upper Bound on Size: $$|A_\epsilon^{(n)}| \le 2^{n(H(X) + \epsilon)}$$ The typical set is much smaller than the total set of \(2^n\) sequences.
  3. Lower Bound on Size: $$|A_\epsilon^{(n)}| \ge (1 - \epsilon) 2^{n(H(X) - \epsilon)}$$ The typical set contains roughly \(2^{nH}\) sequences.

These properties are profound: almost all probability is concentrated on a set of size \(2^{nH}\), which is exponentially smaller than the total number of possible sequences when \(H < \log |\mathcal{X}|\).

Consequence: Data Compression

The AEP has a direct and profound implication for data compression. Instead of encoding all \(2^n\) possible sequences, we can encode only the typical set, which has roughly \(2^{nH}\) elements.

Here's the trick:

  1. Check if the incoming sequence is typical (by testing if its sample entropy is close to \(H\)).
  2. If it's typical, assign it an index from 1 to \(|A_\epsilon^{(n)}| \approx 2^{n(H+\epsilon)}\), requiring about \(n(H + \epsilon)\) bits.
  3. If it's atypical (vanishingly rare), use a fallback code.

Since almost all sequences are typical, this scheme achieves a compression rate of approximately \(H(X)\) bits per symbol—the fundamental limit of lossless compression!

High Probability Sets vs. Typical Sets

One might ask: is the typical set the smallest set that contains, say, 90% of the probability?

Not necessarily! The smallest high-probability set containing probability \(1 - \delta\) (a set that contains the most probable sequences) may differ structurally from the typical set.

However, both sets have roughly the same size asymptotically: \(\sim 2^{nH}\) elements. This is because the probability distribution is so spread out over many sequences that both constructions yield sets of comparable cardinality. The typical set is distinguished not by being the smallest, but by being the set of sequences with "typical" entropy values.

Code Lab: Python Implementation

Compute the AEP and typical set directly in Python. 

import numpy as np

def is_typical(seq, p, epsilon, H):
    """Check if binary sequence is in the typical set."""
    n = len(seq)
    # For binary source: log_prob = sum of log(p) for 1s, log(1-p) for 0s
    log_prob = np.sum(seq * np.log2(p) + (1 - seq) * np.log2(1 - p))
    sample_entropy = -log_prob / n
    return abs(sample_entropy - H) <= epsilon

def simulate_aep(p, n, num_trials=10000):
    """Demonstrate the AEP: sample entropy converges to H."""
    # True entropy of binary source
    H = -p * np.log2(p) - (1 - p) * np.log2(1 - p)

    # Generate random i.i.d. sequences
    sequences = np.random.binomial(1, p, size=(num_trials, n))

    # Compute log-probability for each sequence
    log_probs = sequences * np.log2(p) + (1 - sequences) * np.log2(1 - p)

    # Sample entropy = negative average log-probability
    sample_entropies = -np.sum(log_probs, axis=1) / n

    return sample_entropies, H

# Example usage
p = 0.7
n = 100
sample_ents, true_H = simulate_aep(p, n)

print(f"True Entropy H(X): {true_H:.4f}")
print(f"Mean Sample Entropy: {np.mean(sample_ents):.4f}")
print(f"Std Dev Sample Entropy: {np.std(sample_ents):.4f}")

# Count typical sequences (epsilon = 0.1)
epsilon = 0.1
typical_mask = np.abs(sample_ents - true_H) <= epsilon
print(f"Proportion Typical: {np.mean(typical_mask):.2%}")
print(f"Typical Set Size (approx): {np.sum(typical_mask)}")
print(f"Predicted Size 2^{{nH}}: {2**(n * true_H):.0f}")

Socratic Sandbox: The Compression Paradox

Q1: If we have a binary source with p=0.9, there are \(2^{100}\) possible sequences of length 100. How many are typical?
A: The true entropy is \(H = -0.9 \log_2 0.9 - 0.1 \log_2 0.1 \approx 0.469\) bits per symbol. So the typical set has approximately \(2^{100 \cdot 0.469} = 2^{46.9} \approx 10^{14}\) sequences. This is a vanishingly small fraction of \(2^{100} \approx 10^{30}\)!
Q2: Why can't we just store the single most probable sequence and ignore the rest?
A: The most probable sequence (one with 90 ones and 10 zeros) has probability \(p^{90}(1-p)^{10} \approx 2^{-46.9}\), which equals the probability of a "typical" sequence. In fact, no single sequence dominates—all typical sequences have essentially the same probability. This is the paradox of the typical set: it's not the set of high-probability sequences individually, but rather the set where probability mass is concentrated collectively.
Q3: The AEP requires i.i.d. random variables. Can it be extended to processes with memory (like language)?
A: Yes! The Shannon-McMillan-Breiman theorem extends the AEP to stationary ergodic processes. Instead of the single-letter entropy \(H(X)\), we use the entropy rate \(H_\infty = \lim_{n \to \infty} \frac{1}{n} H(X_1, \ldots, X_n)\). The typical set still has roughly \(2^{nH_\infty}\) elements. This is why we can compress English text beyond the single-letter entropy!
Q4: If \(|A_\epsilon^{(n)}| \approx 2^{nH}\), does this mean we need exactly \(nH\) bits to describe any source sequence?
A: Approximately, yes. The Source Coding Theorem (which we'll prove in a later chapter) says: we can describe a typical sequence using \(n(H + \epsilon)\) bits, but we cannot do better than \(n(H - \epsilon)\) bits for all typical sequences, no matter what code we design. The entropy is the fundamental limit of lossless compression.

Summary

  • The AEP: For i.i.d. random variables, the sample entropy converges to the true entropy as sequence length increases.
  • The Typical Set: Almost all probability mass is concentrated on a set of roughly \(2^{nH}\) sequences—exponentially smaller than all \(2^n\) possible sequences.
  • Compression: We can encode only the typical set, achieving \(nH(X)\) bits per symbol, the fundamental limit of lossless compression.
  • Extension: The Shannon-McMillan-Breiman theorem extends the AEP to dependent sequences via the entropy rate.
  • Philosophical Insight: Randomness creates order—typical sequences are rare in count but dominant in probability, and they all "look alike" statistically.
