Chapter 10: The Gaussian Channel

From Cover & Thomas, Elements of Information Theory, pp. 239–264

Part I: Feynman Mental Model

"You have a fixed power budget and several parallel channels with different noise levels. Where do you allocate power? Pour it like water into cups of different heights — fill the noisiest channels last. That's the water-filling solution, and it achieves the maximum total data rate."

This intuition captures the essence of optimal power allocation across parallel channels. Just as water naturally fills to a uniform level, we allocate power to maintain equal "marginal benefit" across all active channels—but never waste resources on channels so noisy that improvement is trivial.

Part II: The Gaussian Channel (10.1)

The Model

A discrete-time Gaussian channel is described by:

$$Y = X + Z$$

where:

  • X is the transmitted signal
  • Z ~ N(0, N) is zero-mean Gaussian noise with variance N
  • Y is the received signal
  • Power constraint: E[X²] ≤ P

Capacity: The Shannon-Hartley Theorem

The capacity of the Gaussian channel is:

$$C = \frac{1}{2}\log_2\left(1 + \frac{P}{N}\right) \text{ bits per channel use}$$

This is one of the most important results in information theory. The Gaussian input distribution (with variance P) is optimal and achieves this capacity.

Interpretation

  • SNR = P/N: The signal-to-noise ratio. The capacity grows logarithmically with SNR.
  • Doubling SNR: Adds only about 1 bit/channel use (since log₂(2) = 1). Diminishing returns with increasing SNR.
  • High SNR: C ≈ ½ log₂(P/N), linear in log(P).

Part III: The Converse (10.2)

The Shannon-Hartley result is not just an upper bound—it's a tight characterization. The converse theorem states:

Converse: Any code with transmission rate R > C has error probability bounded away from zero, no matter how long the codeword.

Combined with the achievability proof (showing that rates R < C are achievable with vanishing error), this proves that C is the exact boundary between reliable and unreliable communication.

Part IV: Band-Limited Channels (10.3)

For a continuous-time channel with bandwidth W Hz and noise power spectral density N₀:

$$C = W \log_2\left(1 + \frac{P}{N_0 W}\right) \text{ bits/sec}$$

where P is the total power and N₀ is the one-sided noise PSD.

Key Insights

  • Bandwidth-Power Tradeoff: Increasing W can either increase or decrease capacity depending on the operating point.
  • Low SNR Regime (P ≪ N₀W): C ≈ W · P/(N₀W ln 2) = P/(N₀ ln 2) bits/sec. Doubling power doubles capacity. Bandwidth barely helps.
  • High SNR Regime (P ≫ N₀W): C ≈ W log₂(P/(N₀W)) bits/sec. Capacity scales as W log(P). Doubling W doubles capacity; doubling P adds one bit per Hz.
Conclusion: At low SNR, bandwidth is "cheap" and power is precious. At high SNR, the opposite is true. Modern systems must choose their regime carefully.

Part V: Parallel Gaussian Channels (10.4)

Consider k parallel channels with independent Gaussian noise:

  • Channel i: Y_i = X_i + Z_i, where Z_i ~ N(0, N_i)
  • Total power constraint: Σ E[X_i²] ≤ P

The capacity is:

$$C = \sum_{i=1}^{k} \frac{1}{2}\log_2\left(1 + \frac{P_i}{N_i}\right)$$

where power allocation {P_i} is chosen optimally via water-filling.

Water-Filling Algorithm

Allocate power so that the marginal capacity gain per Watt is equal across all channels:

$$P_i = \max\left(0, \lambda - N_i\right)$$

where λ (the "water level") is chosen such that Σ P_i = P.

Part VI: Colored Gaussian Noise (10.5)

When noise is not white but has correlation (colored noise), we can use Fourier analysis to decompose the channel into parallel sub-channels, one for each frequency.

The key insight: Diagonalize via the Karhunen-Loève transform. This converts a continuous colored-noise channel into a set of independent parallel Gaussian channels, each with its own noise power spectral density N(f).

Then water-fill over the spectrum:

$$C = \int_0^W \frac{1}{2}\log_2\left(\frac{\lambda}{N(f)}\right) df$$

where λ is set such that ∫ [λ - N(f)]₊ df = P.

Practical Impact: In wireless channels with frequency-selective fading, transmit more power on "good" frequencies (low noise) and less on "bad" frequencies. This is why adaptive modulation and OFDM are so effective.

Part VII: Gaussian Channel with Feedback (10.6)

A surprising result: Feedback does not increase the capacity of a Gaussian channel.

Why? Because the optimal input distribution for the Gaussian channel is already Gaussian and white (all frequencies equally powered). Feedback can help reduce error probability for a fixed rate, but it cannot exceed the fundamental limit C = ½ log₂(1 + P/N).

This is different from discrete memoryless channels, where feedback can sometimes help (though it does not increase capacity there either). For Gaussian channels, the result is even more absolute: feedback is useless for capacity.

Historical Note: Feedback was once thought to be a game-changer. The Gaussian no-feedback result was counterintuitive and remains a key lesson: measure information content, not just intuition.

Part VIII: Code Lab – Water-Filling Algorithm

Implement the water-filling solution in Python. Given noise levels N and total power P, find the optimal power allocation {P_i}. 

import numpy as np

def water_filling(noise_levels, total_power, tolerance=1e-6):
    """
    Compute optimal power allocation via water-filling.

    Args:
        noise_levels: array of noise powers [N_1, ..., N_k]
        total_power: total power budget P
        tolerance: convergence tolerance for lambda

    Returns:
        power_allocation: array of optimal powers [P_1, ..., P_k]
        lambda_: the water level
    """
    N = len(noise_levels)
    noise_sorted = np.sort(noise_levels)

    # Binary search for lambda
    lambda_min, lambda_max = noise_sorted[0], noise_sorted[-1] + total_power / N

    while lambda_max - lambda_min > tolerance:
        lambda_ = (lambda_min + lambda_max) / 2
        power_used = np.sum(np.maximum(0, lambda_ - noise_levels))

        if power_used < total_power:
            lambda_min = lambda_
        else:
            lambda_max = lambda_

    lambda_ = (lambda_min + lambda_max) / 2
    power_allocation = np.maximum(0, lambda_ - noise_levels)

    return power_allocation, lambda_

# Example: 4 channels with noise [0.5, 1.0, 2.0, 0.8], budget P=3
noise_levels = np.array([0.5, 1.0, 2.0, 0.8])
total_power = 3.0

P_opt, lambda_ = water_filling(noise_levels, total_power)

print(f"Noise levels: {noise_levels}")
print(f"Optimal powers: {P_opt}")
print(f"Water level λ: {lambda_:.4f}")
print(f"Total power used: {np.sum(P_opt):.4f}")

# Capacity
capacity = 0.5 * np.sum(np.log2(1 + P_opt / noise_levels))
print(f"Total capacity: {capacity:.4f} bits/use")

Part IX: Socratic Sandbox

Q1: WiFi SNR = 30 dB. What's the max data rate per Hz?
30 dB = 10^(30/10) = 1000 (linear). C/W = ½ log₂(1 + 1000) = ½ log₂(1001) ≈ ½ · 9.97 ≈ 10 bits/s/Hz. Modern WiFi 6 achieves ~7–8 bits/s/Hz—impressively close to the limit! The gap accounts for overhead, imperfect channel estimation, and non-ideal coding.
Q2: Why does doubling bandwidth help more than doubling power?
From C = W log₂(1 + P/(N₀W)):
  • Double W: C → 2W log₂(1 + P/(2N₀W)) ≈ 2W log₂(P/(N₀W)) = 2C (approximately, for high SNR).
  • Double P: C → W log₂(1 + 2P/(N₀W)) = C + W log₂(2) = C + W (approximately, for high SNR).
Doubling W multiplies capacity by 2; doubling P adds only W bits. Bandwidth is more valuable, especially at high SNR.
Q3: In water-filling, why allocate ZERO power to some channels?
If a channel is too noisy (N_i > λ), adding power yields marginal gain 1/(2(λ − N_i)) < 1/(2·0) = ∞. But more precisely, the capacity per Watt is ½ log₂(1 + 1/ε) for tiny allocation ε > 0. It's better to reallocate that power to quieter channels, where marginal gain is higher. This is the optimality condition: equal marginal gain across all active channels.
Q4: Does C = ½ log₂(1 + SNR) apply to complex channels (e.g., wireless)?
For real channels: C = ½ log₂(1 + SNR) (as derived).
For complex channels: C = log₂(1 + SNR) (no ½). Why? A complex Gaussian variable carries twice the information (real + imaginary components). In MIMO (multiple antennas), each spatial stream is effectively a complex channel, so we water-fill across multiple parallel complex channels. This is why MIMO systems scale capacity linearly with min(M, N) antennas.

Summary

  • Gaussian Channel Capacity: C = ½ log₂(1 + P/N) bits/channel use. Optimal input: Gaussian, variance P.
  • Shannon-Hartley (Continuous-Time): C = W log₂(1 + P/(N₀W)) bits/sec. Bandwidth and power both matter, with diminishing returns at high SNR.
  • Parallel Channels: Water-fill power to maximize total capacity. Quieter channels get priority.
  • Colored Noise: Decompose via Fourier/KLT, then water-fill over the spectrum.
  • Feedback: Does not increase Gaussian channel capacity, but can help reduce coding complexity.
  • Complex Channels (Wireless): Capacity is log₂(1 + SNR) (no ½), enabling MIMO scaling.
