Chapter 8

Channel Capacity

The Speed Limit of Communication

Cover & Thomas, pp. 183–222

The Intuition: Shouting Across a Noisy Room

"Imagine shouting across a noisy room. There's a maximum rate at which you can communicate reliably — no matter how cleverly you encode. Shannon proved this limit exists AND is achievable. That's channel capacity: the speed limit of communication."

This simple image captures the essence of Shannon's most profound discovery. Every communication channel — whether a WiFi link, an optical fiber, or two people in a crowd — has a fundamental speed limit. This limit is:

  • Universal: No code, no matter how sophisticated, can exceed it.
  • Achievable: We can get arbitrarily close with the right approach.
  • Computable: From the channel's physical properties, we can calculate it exactly.

This is the bridge between information theory and real communication systems. Channel capacity is where mathematics meets physics.

Channel Capacity: Formal Definition

The channel capacity of a discrete memoryless channel (DMC) is defined as:

$$C = \max_{p(x)} I(X;Y)$$

where the maximum is over all input probability distributions $p(x)$, and $I(X;Y)$ is the mutual information between the channel input $X$ and output $Y$.

In other words: find the input distribution that maximizes how much information can flow through the channel.

Key Intuitions

  • Units: Capacity is measured in bits per channel use (or nats per use).
  • Optimization: Not all input distributions are equally good. The capacity-achieving distribution is often non-uniform.
  • Noise Trade-off: More noise means more uncertainty at the receiver, which lowers capacity.

Quick Examples

Noiseless Binary Channel:
$$C = 1 \text{ bit}$$

Each symbol carries full information. Uniform input distribution is optimal.

Noisy 4-Symbol Channel (Special Case):
$$C = 1 \text{ bit}$$

Even though 4 symbols could carry 2 bits, noise forces us to use only 2 of them reliably. Capacity remains 1 bit.

Binary Symmetric Channel (BSC):
$$C = 1 - H(p)$$

where $p$ is the crossover probability and $H(p) = -p \log_2 p - (1-p) \log_2(1-p)$ is the binary entropy function. As $p \to 0$ (noiseless), $C \to 1$. As $p \to 0.5$ (complete noise), $C \to 0$.

Binary Erasure Channel (BEC):
$$C = 1 - \alpha$$

where $\alpha$ is the erasure probability. Each symbol is either received correctly or erased. Capacity is simply the probability of not being erased.

Shannon's Channel Coding Theorem (Theorem 8.7)

Shannon's crowning achievement can be stated in two parts:

Direct Part (Achievability)

For any rate $R < C$, there exists a code with:

  • $M = 2^{nR}$ codewords
  • Block length $n$
  • Maximum probability of error $P_e^{(n)} \to 0$ as $n \to \infty$

In plain language: If you want to send information at a rate below capacity, you can make the error probability arbitrarily small by making your code longer.

Converse Part (Impossibility)

For any rate $R > C$, the error probability is bounded away from zero for all sufficiently large $n$:

$$P_e^{(n)} \geq 1 - O(1/n)$$

In plain language: If you try to send above capacity, you'll always have a non-negligible error rate, no matter how long your code is.

The Proof Sketch

Why does random coding work? This is Shannon's profound insight: pick codewords independently and uniformly at random from the input alphabet. Analyze the error probability by counting how many codewords could be confused with the received sequence.

The key observation: typical sequences have a special property — they cluster in a small region of the high-dimensional space. With random coding, different codewords spread out uniformly, making their typical sets nearly disjoint.

For the converse, use Fano's inequality: if the error probability is small, the mutual information $I(X;Y)$ must be close to the entropy of $X$. But this can't happen if $R > C$.

Jointly Typical Sequences (Section 8.6)

The mathematical machinery behind the coding theorem rests on the notion of jointly typical sequences.

Definition

A pair of sequences $(x^n, y^n)$ is jointly $\delta$-typical if:

$$\left| -\frac{1}{n} \log p(x^n) - H(X) \right| < \delta$$ $$\left| -\frac{1}{n} \log p(y^n) - H(Y) \right| < \delta$$ $$\left| -\frac{1}{n} \log p(x^n, y^n) - H(X,Y) \right| < \delta$$

In other words: the empirical frequencies match the true distributions closely.

Key Properties

  • High Probability: For large $n$, most sequences are jointly typical.
  • Small Size: The number of jointly typical pairs is approximately $2^{n(H(X) + H(Y))}$.
  • Decoding Rule: Find the unique codeword $(x^n, y^n)$ that is jointly typical with the received sequence. If exactly one exists, decode it; otherwise, declare an error.

This elegant principle connects probability (typical sets) with coding (finding the right codeword).

Hamming Codes & Error Correction (Section 8.11)

While Shannon's theorem proves that near-capacity performance is possible, practical codes must be explicitly constructed. A classic example is the Hamming [7,4] code.

[7,4] Hamming Code

  • Block length: $n = 7$ bits
  • Information bits: $k = 4$ bits
  • Parity bits: $7 - 4 = 3$ bits
  • Code rate: $R = 4/7 \approx 0.571$ bits/use
  • Error correction: Corrects all single-bit errors

How it works: Three parity-check equations relate the seven bits. When a single bit flips, the pattern of parity violations (the syndrome) uniquely identifies which bit was corrupted.

Trade-off: For a Binary Symmetric Channel with very low crossover probability $p$, the Hamming code is efficient: you get $4/7$ of the information with single-error correction. As $p$ increases, random coding schemes achieve higher rates closer to capacity.

$$\text{For } p = 0.01: \quad C = 1 - H(0.01) \approx 0.919 \text{ bits}$$ $$\text{Hamming [7,4] rate: } 4/7 \approx 0.571 \text{ bits}$$

There's a gap — random codes can achieve rates up to 0.919 bits with vanishing error; Hamming gets 0.571 with guaranteed single-error correction. This is the trade-off between practical structure and optimal performance.

Feedback Capacity (Section 8.12): A Surprise

Intuitively, feedback should help. If the receiver can tell the sender what was received, the sender can adapt. Surely that's useful?

Shannon proved otherwise for discrete memoryless channels:

Feedback Does NOT Increase DMC Capacity

$$C_{\text{feedback}} = C_{\text{no feedback}}$$

Why? For a DMC, the channel is memoryless: future outputs don't depend on past inputs. Feedback is information, but it's information about things that don't affect the future. From an information-theoretic standpoint, it doesn't help reach the fundamental limit.

What Feedback DOES Help With

Although feedback doesn't increase capacity, it provides practical benefits:

  • Simpler Codes: Variable-length codes become easier to implement with feedback (e.g., ARQ — Automatic Repeat reQuest).
  • Lower Delay: Shorter average block lengths for some schemes.
  • Error Exponents: The rate at which error probability decreases can be improved.
  • Channels with Memory: For channels with memory, feedback CAN increase capacity. Shannon's result applies only to memoryless channels.

Code Lab: Random Coding Capacity Estimation

Here's a Python implementation that estimates channel capacity for a Binary Symmetric Channel using Monte Carlo simulation of mutual information:

import numpy as np

def binary_entropy(p):
    """Compute binary entropy H(p)."""
    if p == 0 or p == 1:
        return 0
    return -p * np.log2(p) - (1 - p) * np.log2(1 - p)

def bsc_capacity(p):
    """Analytic BSC capacity: C = 1 - H(p)."""
    return 1 - binary_entropy(p)

def estimate_mutual_information(p_crossover, n_samples=100000):
    """
    Estimate I(X; Y) for BSC with uniform input distribution.
    X ~ Bernoulli(0.5)
    """
    # Generate random input
    X = np.random.randint(0, 2, n_samples)

    # Generate output: flip bit with probability p
    Y = X.copy()
    flip_mask = np.random.rand(n_samples) < p_crossover
    Y[flip_mask] = 1 - Y[flip_mask]

    # Compute mutual information via empirical distributions
    # I(X;Y) = H(Y) - H(Y|X)

    # H(Y): marginal entropy
    p_y0 = np.mean(Y == 0)
    p_y1 = np.mean(Y == 1)
    h_y = -p_y0 * np.log2(p_y0 + 1e-10) - p_y1 * np.log2(p_y1 + 1e-10)

    # H(Y|X): conditional entropy
    h_y_given_x = 0
    for x_val in [0, 1]:
        mask = X == x_val
        if np.any(mask):
            y_given_x = Y[mask]
            p_y0_given_x = np.mean(y_given_x == 0)
            p_y1_given_x = np.mean(y_given_x == 1)
            h_y_x = (-p_y0_given_x * np.log2(p_y0_given_x + 1e-10)
                     - p_y1_given_x * np.log2(p_y1_given_x + 1e-10))
            h_y_given_x += np.mean(mask) * h_y_x

    mi = h_y - h_y_given_x
    return mi

# Example: estimate capacity at p = 0.1
p = 0.1
analytic_c = bsc_capacity(p)
estimated_c = estimate_mutual_information(p)

print(f"BSC with p = {p}")
print(f"Analytic capacity: {analytic_c:.4f} bits")
print(f"Estimated capacity: {estimated_c:.4f} bits")
print(f"Error: {abs(analytic_c - estimated_c):.6f} bits")

Key Insights:

  • The analytic formula $C = 1 - H(p)$ is exact.
  • The Monte Carlo estimate converges to the true value as $n_{\text{samples}} \to \infty$.
  • For practical coding, you'd simulate the error probability of a specific code at various rates and compare against capacity.

Q1: BSC with $p = 0.01$. Can we send at 0.9 bits/use reliably?

A: Let's compute capacity:

$$C = 1 - H(0.01) = 1 - (-0.01 \log_2 0.01 - 0.99 \log_2 0.99)$$ $$\approx 1 - 0.0808 = 0.919 \text{ bits}$$

Since $0.9 < 0.919$, yes! According to Shannon's theorem, we can construct a code with rate 0.9 and vanishing error probability as we increase block length. This requires explicit code construction, but it's theoretically possible.

Q2: Why does random coding work? Isn't structure needed?

A: This is Shannon's genius! Random codes work because they spread codewords uniformly in high-dimensional space. When block length is large:

  • Each codeword has a "typical set" — the set of sequences it might be confused with after noise.
  • Random coding makes these typical sets nearly disjoint.
  • The receiver can uniquely identify which codeword was sent.

Structure (like in Hamming codes) helps make decoding practical and fast, but structure is not required for approaching capacity. Nature didn't design structured codes; Shannon proved random ones work too.

Q3: Feedback doesn't increase DMC capacity. What DOES it help with?

A: For discrete memoryless channels, feedback is "information about the past," which doesn't affect the future. So capacity stays the same. But feedback helps with:

  • Simplicity: ARQ (Automatic Repeat reQuest) is far simpler than designing an optimal fixed-rate code.
  • Latency: Variable-length codes can use fewer symbols on average.
  • Error exponents: The exponent $E(R)$ governing how fast $P_e \to 0$ can improve.
  • Channels with memory: Feedback CAN increase capacity if the channel is not memoryless.
Q4: The theorem needs $n \to \infty$. What about real finite-block-length codes?

A: Excellent question. Shannon's theorem is asymptotic. In practice:

  • Rate Penalty: With finite $n$, achievable rate is $\approx C - O(1/\sqrt{n})$, where the $O(1/\sqrt{n})$ term depends on the channel and target error probability.
  • Trade-off: Longer codes approach capacity but increase delay and complexity.
  • Finite Blocklength Regime: There's a growing field of research on exact capacity-error-blocklength relationships (beyond Shannon's asymptotic limit).
  • Example: A code with $n = 1000$ on a BSC with $p = 0.01$ might achieve 0.910 bits instead of 0.919, a small gap.

Modern wireless and data transmission systems operate in the finite-blocklength regime, where both capacity and latency matter.
