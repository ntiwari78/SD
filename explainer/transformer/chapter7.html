Decoder Models & Seq2Seq Architecture

From GPT-1 to GPT-3: Autoregressive generation, in-context learning, and the T5 text-to-text framework

⚡ Level 1: The Feynman Bridge (Zero Jargon)

Let's start with a simple analogy to understand decoder models and sequence-to-sequence architectures.

Analogy: The Storyteller vs The Editor vs The Translator

Encoder (BERT) = Editor: Reads the entire document all at once, understands context from all directions (both left and right). Like an editor reviewing a complete manuscript.

Decoder (GPT) = Storyteller: Writes one word at a time, can only see what came before (like writing a story left to right, never peeking ahead). Each new word builds on the previous ones.

Encoder-Decoder (T5) = Translator: First reads the entire source text (encoder), then writes the translation word by word (decoder). Best of both worlds: full understanding, sequential generation.

GPT Family: Evolution from GPT-1 to GPT-3

GPT-1: Generative Pre-Training (2018)

Architecture: Stack of 12 transformer decoder layers, 12 masked self-attention heads, d_model = 768, d_ff = 3072, learned positional embeddings.

Pre-training objective (Causal Language Modeling):

$$L_1(U) = -\sum_{t} \log P(w_t | w_{t-1}, \ldots, w_{t-k}; \Theta)$$

Supervised fine-tuning objective:

$$L_2(C) = -\sum_{(x,y)} \log P(y | x; W)$$

Combined objective:

$$L_{total} = L_2(C) + \lambda L_1(C)$$

Training data: BooksCorpus + 1B Word Benchmark (~7,000 unique books)

GPT-2: Zero-Shot Transfer (2019)

Key advancement: Scaled up to 1.542B parameters, 48 layers, and proved zero-shot learning without task-specific fine-tuning.

Architecture modifications from GPT-1:

  • Pre-LayerNorm: Layer norm applied BEFORE attention and FFN (not after)
  • Final LayerNorm: Extra layer norm after the final decoder block
  • Residual scaling: Residual layer weights initialized with 1/√N scaling
  • Tokenization: BPE (Byte-Pair Encoding) with 50,257 vocabulary

Pre-LayerNorm vs Post-LayerNorm:

$$\text{Post-LN (Standard):} \quad x + \text{Sublayer}(\text{LayerNorm}(x))$$
$$\text{Pre-LN (GPT-2):} \quad \text{LayerNorm}(x + \text{Sublayer}(x)) \text{ — norm applied before sublayer}$$

Result: Achieved state-of-the-art on 7/8 benchmarks without any fine-tuning.

GPT-3: In-Context Learning at Scale (2020)

Breakthrough: 175B parameters enable powerful in-context learning with no gradient updates needed.

Model Specifications:

  • 96 decoder layers
  • d_model = 12,288
  • 96 attention heads, d_k = d_v = 128
  • d_ff = 4 × d_model = 49,152
  • Training: 300B tokens, context window = 2048
  • Attention pattern: Alternating dense and locally banded sparse attention (Sparse Transformer)

In-Context Learning Paradigm:

$$\text{Zero-shot:} \quad P(y \mid \text{task description})$$
$$\text{One-shot:} \quad P(y \mid \text{task description}, x_1 \rightarrow y_1)$$
$$\text{Few-shot:} \quad P(y \mid \text{task description}, x_1 \rightarrow y_1, \ldots, x_K \rightarrow y_K)$$

Crucially: No gradient updates during evaluation — pure in-context learning from the prompt.

T5: Text-to-Text Transfer Transformer

The T5 Philosophy: All Tasks Are Text-to-Text

T5's key insight: reframe every NLP task as transforming an input text string to an output text string.

  • Translation: "translate English to German: The house is wonderful." → "Das Haus ist wunderbar."
  • Summarization: "summarize: Long article text..." → "Summary text..."
  • Sentiment: "sst2 sentence: This movie was fantastic!" → "positive"
  • Question Answering: "question: What is the capital of France? context: Paris is..." → "Paris"
  • Grammar Correction: "cola sentence: The boy run to store." → "The boy runs to the store."

Architecture: Standard encoder-decoder transformer

  • Encoder: Fully visible self-attention (every position sees all positions)
  • Decoder: Causal self-attention (position i only sees positions ≤ i)
  • Cross-attention: Decoder attends to encoder outputs

Pre-training Objective: Span Corruption

$$\text{Input: "The quick } \langle\text{X}\rangle \text{ fox jumps } \langle\text{Y}\rangle\text{"}$$
$$\text{Target: "}\langle\text{X}\rangle \text{ brown } \langle\text{Y}\rangle \text{ over the lazy dog"}$$

Benefit: Same model, same loss function, same hyperparameters for ALL tasks.

ByT5: Operating at the Byte Level

Innovation: Tokenization-free model that operates directly on raw UTF-8 bytes (256 embeddings + 3 special tokens).

  • No SentencePiece tokenization — eliminates vocabulary bottleneck
  • Asymmetric encoder-decoder: Encoder has 3× more layers (byte sequences are longer)
  • Noise robustness: Tested on 6 types: deletions, mutations, duplications, capitalization changes, uppercasing, random casing
  • Multilingual: Works directly with any UTF-8 text

Trade-off: Byte sequences are ~4× longer than token sequences, so inference is slower but robustness increases.

ELECTRA: Efficient Pretraining via Discriminator (§5.3)

Problem with MLM: BERT masks only 15% of tokens, so the model learns from only 15% of the data per iteration. Inefficient.

ELECTRA Solution: Replaced Token Detection (RTD)

  1. Generator (small MLM): Corrupt input by replacing masked positions with random tokens: $x^{\text{masked}} = \text{REPLACE}(x, m, [\text{MASK}])$, learns $P_G(x_t | x^{\text{masked}})$
  2. Create corruption: Sample replacements from generator: $x^c = \text{REPLACE}(x, m, \hat{x}_i)$ where $\hat{x}_i \sim P_G(x_i | x^{\text{masked}})$
  3. Discriminator (ELECTRA): Binary classification — is each token original or replaced? Learns from ALL tokens, not just 15%

Result: ~4× more sample efficient than BERT. The discriminator learns richer representations because it processes every token.

💾 Code Block 1: Causal Self-Attention from Scratch
import torch import torch.nn as nn import torch.nn.functional as F class CausalSelfAttention(nn.Module): """GPT-style masked self-attention — tokens only attend to the past""" def __init__(self, d_model, n_heads, max_seq_len=2048): super().__init__() self.d_model = d_model self.n_heads = n_heads self.d_k = d_model // n_heads # Combined projection for Q, K, V self.W_qkv = nn.Linear(d_model, 3 * d_model) self.W_o = nn.Linear(d_model, d_model) # Causal mask: lower triangular mask = torch.tril(torch.ones(max_seq_len, max_seq_len)) self.register_buffer('mask', mask.view(1, 1, max_seq_len, max_seq_len)) def forward(self, x): B, T, D = x.shape qkv = self.W_qkv(x) Q, K, V = qkv.chunk(3, dim=-1) # Reshape for multi-head: (B,T,D) → (B,heads,T,d_k) Q = Q.view(B, T, self.n_heads, self.d_k).transpose(1, 2) K = K.view(B, T, self.n_heads, self.d_k).transpose(1, 2) V = V.view(B, T, self.n_heads, self.d_k).transpose(1, 2) # Scaled dot-product with causal mask scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5) scores = scores.masked_fill( self.mask[:, :, :T, :T] == 0, float('-inf')) attn = F.softmax(scores, dim=-1) out = torch.matmul(attn, V) out = out.transpose(1, 2).contiguous().view(B, T, D) return self.W_o(out)

⚡ Key: The causal mask ensures position i can only attend to positions ≤ i, preventing information leakage from future tokens during autoregressive generation.

💾 Code Block 2: GPT-2 Text Generation with HuggingFace
from transformers import GPT2LMHeadModel, GPT2Tokenizer import torch tokenizer = GPT2Tokenizer.from_pretrained('gpt2') model = GPT2LMHeadModel.from_pretrained('gpt2') model.eval() def generate_text(prompt, max_new_tokens=50, temperature=0.8, top_k=40): """Autoregressive generation with sampling controls""" input_ids = tokenizer.encode(prompt, return_tensors='pt') with torch.no_grad(): output = model.generate( input_ids, max_new_tokens=max_new_tokens, temperature=temperature, # Higher = more random top_k=top_k, # Sample from k most likely do_sample=True, pad_token_id=tokenizer.eos_token_id ) return tokenizer.decode(output[0], skip_special_tokens=True) # Example: zero-shot task print(generate_text("Translate to French: Hello, how are you?"))

Decoding Strategies:

  • Greedy: Always pick highest-probability token (deterministic but repetitive)
  • Top-k: Sample from k most likely tokens (k=40 common)
  • Top-p (nucleus): Sample from smallest set with cumulative prob ≥ p
  • Temperature: Scale logits before softmax (T < 1 = focused, T > 1 = diverse)
  • Beam search: Maintain k best partial sequences
💾 Code Block 3: T5 Text-to-Text — One Model, Many Tasks
from transformers import T5Tokenizer, T5ForConditionalGeneration tokenizer = T5Tokenizer.from_pretrained('t5-base') model = T5ForConditionalGeneration.from_pretrained('t5-base') def t5_generate(input_text, max_length=128): """T5 text-to-text: prepend task prefix, get text output""" inputs = tokenizer(input_text, return_tensors="pt", max_length=512, truncation=True) outputs = model.generate(**inputs, max_length=max_length, num_beams=4) return tokenizer.decode(outputs[0], skip_special_tokens=True) # ALL tasks use the SAME model — just change the prefix! tasks = { "Translation": "translate English to German: The house is wonderful.", "Summarization": "summarize: The transformer architecture...", "Sentiment": "sst2 sentence: This movie was fantastic!", } for task_name, input_text in tasks.items(): result = t5_generate(input_text) print(f"{task_name}: {result}")

💡 Insight: By framing everything as text-to-text, T5 uses the same architecture, loss function, and hyperparameters for ALL tasks. Transfer learning between tasks becomes natural.

💾 Code Block 4: Building a Mini GPT Decoder Block
class GPTBlock(nn.Module): """Single GPT-2 style decoder block with Pre-LayerNorm""" def __init__(self, d_model, n_heads, d_ff, dropout=0.1): super().__init__() # Pre-LayerNorm (GPT-2 style) self.ln1 = nn.LayerNorm(d_model) self.ln2 = nn.LayerNorm(d_model) self.attn = CausalSelfAttention(d_model, n_heads) self.ffn = nn.Sequential( nn.Linear(d_model, d_ff), nn.GELU(), # GPT uses GELU, not ReLU nn.Linear(d_ff, d_model), nn.Dropout(dropout) ) self.dropout = nn.Dropout(dropout) def forward(self, x): # Pre-LN + Residual for attention x = x + self.dropout(self.attn(self.ln1(x))) # Pre-LN + Residual for FFN x = x + self.ffn(self.ln2(x)) return x class MiniGPT(nn.Module): def __init__(self, vocab_size, d_model=768, n_heads=12, n_layers=12, d_ff=3072, dropout=0.1): super().__init__() self.token_embed = nn.Embedding(vocab_size, d_model) self.pos_embed = nn.Embedding(1024, d_model) self.blocks = nn.ModuleList([ GPTBlock(d_model, n_heads, d_ff, dropout) for _ in range(n_layers) ]) self.ln_f = nn.LayerNorm(d_model) self.head = nn.Linear(d_model, vocab_size, bias=False) # Weight tying self.head.weight = self.token_embed.weight def forward(self, idx): B, T = idx.shape tok_emb = self.token_embed(idx) pos_emb = self.pos_embed(torch.arange(T, device=idx.device)) x = tok_emb + pos_emb for block in self.blocks: x = block(x) x = self.ln_f(x) return self.head(x) # (B, T, vocab_size)

  • GPT-1: d=768, h=12, L=12, d_ff=3072 → ~117M params
  • GPT-2: d=1600, h=25, L=48, d_ff=6400 → ~1.5B params
  • GPT-3: d=12288, h=96, L=96, d_ff=49152 → ~175B params

🤔 Level 4: Socratic Sandbox (Model Failure Scenarios)

These diagnostic questions reveal common pitfalls and deep understanding of how decoder models work.

▼
Q1: Your GPT-2 model generates fluent text but keeps repeating phrases in loops ("The cat sat on the mat. The cat sat on the mat..."). You're using greedy decoding. What's causing this and how do you fix it?

Answer:

Greedy decoding always picks the highest-probability token, creating repetition loops when the model enters a high-probability cycle. This is because certain token sequences have very high co-occurrence in the training data, so once the model generates them, it becomes trapped in a local probability maximum.

Solutions:

  • Top-k sampling: Restrict to k most likely tokens (k=40 is common). Reduces extreme probabilities.
  • Nucleus (Top-p) sampling: Sample from smallest set with cumulative probability ≥ p (p=0.9 common). More adaptive than top-k.
  • Temperature scaling: Divide logits by T before softmax (T > 1 increases diversity, T < 1 increases focus). T=1 is no change.
  • Repetition penalty: Downweight tokens that have already been generated. Directly penalizes repetition.
  • Beam search: Maintain k best partial sequences and rescore with coverage penalties.

Code example:

# Problem: greedy decoding output = model.generate(input_ids, max_new_tokens=50) # Solution: top-k + temperature output = model.generate(input_ids, max_new_tokens=50, top_k=40, temperature=0.9, do_sample=True) # Solution: repetition penalty output = model.generate(input_ids, max_new_tokens=50, repetition_penalty=1.5)

Q2: You fine-tune T5-base on summarization and achieve great ROUGE scores. But when you try to also use it for translation (with "translate" prefix), translation quality is terrible. What happened?

Answer: Catastrophic Forgetting

Fine-tuning on a single task causes the model to overfit to that task's loss landscape, forgetting pre-trained capabilities. The model's weights shifted to optimize for summarization, destroying translation ability. T5 was pre-trained on many tasks, but single-task fine-tuning erases that multi-task knowledge.

Why it happens:

  • Gradient updates only optimize for the current task
  • Cross-entropy loss on summarization pulls weights away from translation optima
  • The model has no incentive to preserve translation capability

Solutions:

  • Multi-task fine-tuning: Mix summarization + translation data in the training set. Model learns both simultaneously.
  • Adapter layers: Freeze pre-trained weights, only train small adapter modules for each task. Preserves original knowledge.
  • LoRA (Low-Rank Adaptation): Freeze most weights, train small low-rank updates. Minimal memory overhead.
  • Prompt tuning: Only learn soft prompts, freeze all model weights. Extreme parameter efficiency.
  • Continual learning: Fine-tune sequentially but use experience replay or elastic weight consolidation to protect previous tasks.

Best practice: Always validate on the original pre-trained task distribution during fine-tuning to catch catastrophic forgetting early.

▼
Q3: You're using GPT-3 with few-shot prompting for sentiment analysis. With 5 examples in one order, accuracy is 85%. Reordering those examples drops accuracy to 65%. Why is the model so sensitive to example ordering?

Answer: Position-Dependent In-Context Learning

GPT-3's in-context learning has strong recency bias — the model weights later examples more heavily because they appear closer to the input being classified. Additionally, if the last few examples happen to be all positive, the model becomes biased toward predicting positive regardless of the actual input.

Why this happens:

  • Causal attention: Position i can only attend to positions ≤ i, so later examples have more influence on the final prediction
  • No explicit example symmetry: The model treats examples as part of a sequence, not as a balanced set
  • Shallow in-context learning: The model doesn't learn "meta-patterns" from examples; it infers mostly from surface co-occurrences

Solutions:

  • Multiple orderings + ensemble: Try 5-10 random orderings, average predictions. Reduces order sensitivity.
  • Balanced example placement: Place diverse examples at the end (closer to the test input). Put extreme/outlier examples first.
  • Balanced class ordering: Alternate positive/negative examples. Avoid clustering all examples of one class.
  • Explicit task description: Prepend a clear task description so the model doesn't over-rely on example patterns.
  • More examples (scaling): Increase K from 5 to 50. The benefit of additional examples often overcomes ordering noise.

Empirical finding (from GPT-3 paper): Performance variance due to example ordering ranges from 1-5% on average, but can spike to 20%+ on adversarial orderings for difficult tasks.

Q4: You've built a ByT5 model for multilingual text processing. It handles noisy user input well but is 4× slower than mT5 at inference time on the same hardware. Why?

Answer: Quadratic Attention Complexity

ByT5 operates on raw UTF-8 bytes, making sequences ~4× longer than token sequences (tokens compress multiple bytes). Since self-attention is O(T²) in sequence length, processing time increases quadratically.

Why byte sequences are longer:

  • Most UTF-8 characters = 2-4 bytes each
  • mT5 uses SentencePiece tokenization which compresses: "hello" = 1 token, but 5 bytes
  • ByT5 "hello" = 5 bytes to process individually

Attention complexity:

$$\text{Time} = O(T^2 \cdot d)$$
$$\text{If } T_{\text{bytes}} = 4 \times T_{\text{tokens}}, \text{ then Time increases by } 16\times$$

Why ByT5 still works: The 3:1 encoder-decoder ratio helps. Encoder has 3× more layers to process longer byte sequences, but decoder is smaller for shorter output. Still doesn't fully compensate.

Solutions:

  • Use heavier encoder, lighter decoder: Asymmetry exploits that encoder input is long but output is shorter
  • Batch efficiently: Batch size of 1 is expensive; increase batch size to amortize overhead
  • Sparse attention: Use local attention or sparse patterns to reduce T² complexity
  • Fall back to mT5 for latency-critical: Use ByT5 for robustness (training), mT5 for production (inference)
  • Hierarchical byte compression: Pre-compress bytes into sub-word units before ByT5 processing

Trade-off: ByT5 trades latency for robustness to typos, noise, and unknown scripts. Accept 4× slower inference for better multilingual handling.

Q5: You train a GPT-style model and notice it generates grammatically correct text but frequently states false facts with high confidence ("The capital of Australia is Sydney"). The training data contains correct information. What's happening?

Answer: Hallucination via Statistical Co-occurrence

The causal language model optimizes for P(next_token | context), not factual accuracy. It learns statistical co-occurrences, not truth. "Sydney" and "Australia" have very high co-occurrence in training data (Sydney is mentioned in Australian contexts frequently), even though the capital is Canberra.

Why this happens:

  • No explicit fact-checking mechanism: The model has no knowledge graph, no access to truth values
  • Optimization mismatch: Training objective (perplexity) ≠ truth objective (factual accuracy)
  • Frequency bias: Sydney appears more frequently in text about Australia than Canberra does
  • Confidence from familiarity: Familiar token sequences get high probability, even if factually wrong

This is the "hallucination problem": The model outputs false information with high confidence, making it dangerous for knowledge-intensive tasks.

Solutions:

  • Retrieval-Augmented Generation (RAG): Retrieve relevant documents, condition generation on retrieved context. Model can only say what's in the retrieved docs.
  • Knowledge grounding: Augment model with external knowledge bases (DBpedia, Wikidata). Verify generations against knowledge graphs.
  • Fact-checking post-processing: After generation, run a fact-checking model to filter false claims.
  • RLHF (Reinforcement Learning from Human Feedback): Train a reward model that scores factual accuracy, use it to align generations. Reduces hallucinations significantly.
  • Constrained decoding: Restrict generation to facts in a curated knowledge base. Most restrictive but safest.

Key insight: Language modeling ≠ knowledge. Even 175B parameter models like GPT-3 don't "know" facts; they reproduce statistical patterns. Closing this gap requires either external knowledge or explicit training for factuality.

Summary: From GPT to T5

GPT Family (Decoder-only): Autoregressive models that generate text one token at a time. GPT-1 introduced generative pre-training + fine-tuning. GPT-2 proved zero-shot transfer at scale. GPT-3 enabled few-shot in-context learning at 175B parameters.

T5 (Encoder-Decoder): Unified text-to-text framework that reframes all NLP as string transformation. Same model, same loss, different task prefixes. Remarkably simple and generalizable.

Key Technical Insights:

  • Causal masking prevents future information leakage, enabling online generation
  • Pre-LayerNorm (GPT-2 style) stabilizes training better than post-LayerNorm
  • Scale (parameters + training data) dramatically improves few-shot learning
  • In-context learning requires no gradient updates — pure statistical inference
  • ELECTRA's discriminator learns from all tokens (4× efficiency vs. BERT's 15% masking)
  • Text-to-text framing enables multi-task learning with zero architecture changes
