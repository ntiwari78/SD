🧠 The Speed Reading Analogy

Imagine reading a 1,000-word document where every word needs to reference every other word to understand meaning. That's 500,000 comparisons for full attention. It's exhausting and slow.

Efficient transformers are like speed reading techniques: instead of comparing every word to every other word, you might:

  • Read each paragraph + its neighbors (sliding window)
  • Skim random passages for context (random attention)
  • Always reference the introduction and conclusion (global attention)
  • Use a hash table to group similar ideas together (LSH)
  • Compress redundant comparisons into a kernel function (FAVOR+)

The result: same understanding, dramatically faster. That's the core idea behind Longformer, Reformer, Performer, and BigBird.

🏗️ Modified Architectures: Simple Analogies

Funnel-Transformer

Analogy: "Reading an article, then summarizing paragraph by paragraph, then page by page." As you move through the network, sequences shrink (pooling) so later layers work with compressed information. This reduces computation while preserving important patterns.

DeLighT

Analogy: "Expanding your thinking before compressing to an answer." Each layer first expands to a higher dimension (think deeply), then compresses back (focus on key insights). This lets small models achieve surprising expressiveness.

RealFormer

Analogy: "Remembering which parts of a book you highlighted in previous read-throughs." Current layer's attention logits are mixed with previous layer's attention patterns. This creates residual connections that help the model re-focus on important information across layers.

Universal Transformers

Analogy: "Re-reading difficult paragraphs until you understand." The model applies the same layer multiple times (recurrence in depth) but can dynamically decide when to stop re-processing. Hard problems get more compute; easy tokens skip early.

Transformer-XL

Analogy: "Remembering what you read in previous chapters while reading the current one." When processing segment N, the model has access to cached hidden states from segment N-1. No forgetting at segment boundaries. This enables learning very long-range dependencies (up to millions of tokens).

💡 Key Insight: Modern efficient transformers achieve 95%+ of full attention's performance with 10–100× better scaling. The trade-off: slightly reduced expressiveness for dramatically improved speed and memory.

§5.1 Architectural Modifications

1. Funnel-Transformer: Progressive Pooling

Funnel-Transformer applies progressive pooling between transformer blocks to reduce sequence length. The query comes from pooled representations while key and value come from un-pooled sequences.

Pooling Operation:

$$h'_i = \text{Pool}(h_{2i-1:2i+1})$$
Every two consecutive hidden states are pooled into one (e.g., via convolution or average), reducing T → T/2.

Attention Complexity:

$$\text{Complexity} = O(d \cdot T'^2 + T \cdot d^2)$$
Where T' = T/2^k for k pooling stages. Time complexity becomes O((T/4)² · d) instead of O(T² · d).

2. DeLighT: Grouped Linear Transformations

DeLighT uses grouped linear transformations (GLT) to expand dimension then contract, achieving higher expressiveness with fewer parameters than standard dense layers.

Transformation:

$$\text{GLT}(x) = W_{\text{down}}(\sigma(W_{\text{up}}(x)))$$
Input x ∈ ℝ^d first expands to ℝ^{d·r} (r = expansion ratio, e.g. 4), applies activation σ, then contracts back to ℝ^d. Total parameters: 2ddr ≈ 2d²r, compared to d² for standard dense layer (when r < 1).

3. RealFormer: Residual Attention

RealFormer adds previous layer's attention logits to current layer's pre-softmax scores, allowing the model to learn cumulative attention patterns across depth.

Accumulated Attention:

$$A^{(n)} = A^{(n-1)} + Q^{(n)}(K^{(n)})^T$$
Attention matrix at layer n incorporates attention matrix from layer n-1 before softmax normalization. This creates explicit residual connections in the attention mechanism.

Output:

$$\text{Attn}^{(n)} = \text{softmax}(A^{(n)}/\sqrt{d_k}) V^{(n)}$$

4. Universal Transformers: Recurrent Depth with Halting

Universal Transformers apply the same layer recurrently while using Adaptive Computation Time (ACT) to dynamically halt processing per token.

Halting Probability:

$$p_t^{(n)} = \sigma(W_h h_t^{(n)} + b_h)$$
At each recurrence step n and token position t, compute probability that this token is "done" processing and should not process further.

Halting Mechanism:

$$\text{halt}_t^{(n)} = \begin{cases} 1 & \text{if } p_t^{(n)} > \tau \text{ or } \sum p_t < 1 \\ 0 & \text{otherwise} \end{cases}$$
Token halts when cumulative probability exceeds 1.0 or probability crosses threshold τ. This enables adaptive depth per token, reducing wasted compute on "easy" tokens.

5. Transformer-XL: Segment-Level Recurrence & Relative Positional Encoding

Transformer-XL introduces segment-level recurrence (caching hidden states across segments) and relative positional encoding to handle very long sequences.

Segment Recurrence:

$$\tilde{h}_{t+1}^{(l)} = [\text{StopGradient}(h_t^{(l)}); h_{t+1}^{(l)}]$$
Hidden states from segment t are concatenated to segment t+1 (with gradients stopped to avoid backpropagation through all segments). Enables attention across segment boundaries without quadratic memory.

Relative Positional Encoding (Decomposition):

$$q_i k_j^T = (e_i + p_i) W_q W_k^T (e_j + p_j)^T$$

Expand into four terms:

$$= e_i W_q W_{k,E}^T e_j^T + e_i W_q W_{k,R}^T R_{i-j}^T + u W_{k,E}^T e_j^T + v W_{k,R}^T R_{i-j}^T$$
u and v are learnable biases. W_{k,E} and W_{k,R} are separate projections for content and relative position. R_{i-j} is relative distance encoding (sinusoidal). This allows the model to learn different attention patterns based on absolute content vs. relative position.

Final Attention (Eq 5.20):

$$\text{Attn}_t^{(n)} = \text{softmax}(A_t^{(n)} / \sqrt{d_k}) V_t^{(n)}$$
Standard softmax normalization of accumulated scores across current and cached segments.

§5.2 Efficient Attention Mechanisms

Complexity Review & Memory Table

Standard scaled dot-product attention has quadratic complexity:

$$\text{Attention Complexity} = O(L^2 \cdot d_k + L^2 \cdot d_v) = O(L^2 \cdot d)$$

Memory usage for attention matrix alone (L² floats):

Sequence Length (L) Attention Matrix Size Memory (float32)
600 600 × 600 1.4 MB
1,000 1000 × 1000 4 MB
4,096 4096 × 4096 64 MB
20,000 20000 × 20000 1.6 GB
65,536 65536 × 65536 16.4 GB

6. Longformer: Sliding Window + Global Attention

Longformer combines sliding window attention (local context) with global attention (specific important tokens attend to everything).

Sliding Window Attention:

$$\text{Complexity} = O(L \cdot w)$$
Each position attends to w/2 positions before and after (window size w). For w=256 and L=4096, this is 1M comparisons vs 16M for full attention.

Dilated Sliding Window:

$$\text{Positions attended: } [i - w, i - 2w, i - 3w, ..., i, ..., i + 3w, i + 2w, i + w]$$
Skip d positions between attention points (dilation d). Extends effective context from w to w·(1 + d) with same number of comparisons.

Global Attention:

$$\text{Attn}(\text{global}) = \text{softmax}(Q_g (K)^T / \sqrt{d_k}) V$$
Separate Q_g, K_g, V_g for global tokens (e.g., [CLS] in classification tasks). These tokens can attend to all positions, and all positions can attend to them.

7. Reformer: Locality-Sensitive Hashing (LSH) Attention

Reformer uses LSH to group similar vectors into buckets, computing attention only within buckets. Q = K for efficiency.

LSH Hash Function:

$$h(x) = \arg\max([x R_1; -x R_2]) \in [1, ..., n_{\text{buckets}}]$$
Project vector x by random matrices R₁, R₂, then take argmax of concatenation. Similar vectors hash to same bucket with high probability.

Masked Attention within Buckets:

$$a_i = \sum_j \exp(q_i k_j^T / \sqrt{d_k} - m(i, S_i) - \log Z(i, S_i)) v_j$$
m(i, S_i) masks out-of-bucket positions; Z(i, S_i) normalizes only in-bucket scores. Complexity O(L log L) for multi-round hashing.

Reversible Residual Layers (Memory Efficiency):

$$y_1 = x_1 + F(x_2), \quad y_2 = x_2 + G(y_1)$$
Instead of storing activations for backprop, recompute from outputs using inverses. Reduces memory from O(L·d) to O(d) for activations.

8. Performer: FAVOR+ (Fast Attention with Positive Features)

Performer approximates attention as kernel product, enabling near-linear complexity through feature maps.

Kernel Approximation:

$$A_{ij} = \exp(q_i k_j^T) \approx \langle \phi(q_i^T) \phi(k_j^T) \rangle$$
Approximate exponential attention using inner product of random feature maps φ. Complexity: O(L·r·d_k) where r is feature dimension, typically much smaller than L.

FAVOR+ Feature Map:

$$\exp(q \cdot k^T) = \mathbb{E}_\omega[\cosh(\omega^T(q + k))]$$
Expectation over random ω sampled from standard normal. φ uses random projections and cosine/sine basis to approximate this expectation numerically.

Complexity:

$$\text{Time: } O(L \cdot r \cdot d_k), \quad \text{Memory: } O(L \cdot r)$$
Near-linear in L when r << L. Trades accuracy for speed compared to full attention.

9. BigBird: Block-Sparse Attention

BigBird combines three attention types (random, sliding window, global) into a block-sparse pattern that is computationally efficient on GPUs.

Attention as Graph:

$$\text{Adjacency matrix } A \in \{0,1\}^{L \times L}$$
Represent attention as directed graph where edge (i,j) means token i attends to token j.

Three Components:

Random (Erdős-Rényi): Each token randomly attends to r other tokens.
Sliding Window (Watts-Strogatz): Each token attends to w neighbors (local structure).
Global: G internal tokens or g external tokens attend to all positions and vice versa.

Combined Complexity:

$$\text{Total Attention Edges} = O(r \cdot L + w \cdot L + g \cdot L) = O(L)$$
Linear in L. BigBird is Turing-complete and a universal approximator despite sparse connectivity.

10. Linformer: Linear Complexity via Low-Rank Projection

Linformer projects key and value to lower rank, reducing quadratic attention to linear.

Low-Rank Projection:

$$K' = K R, \quad V' = V R$$
Project K, V ∈ ℝ^{L×d} to ℝ^{k×d} using projection matrix R ∈ ℝ^{L×k}, where k << L.

Attention:

$$\bar{A}_{ij} = \text{softmax}(Q_i (K')^T / \sqrt{d_k/h}) \in \mathbb{R}^{L \times k}$$
Instead of L × L attention matrix, compute L × k, dramatically reducing memory and computation.

Complexity:

$$\text{Time: } O(L \cdot k \cdot d_k + L \cdot d_k \cdot d_v), \quad \text{Memory: } O(L \cdot k)$$
Linear in L when k is small constant. Different parameter sharing strategies (headwise, layerwise, etc.) balance expressiveness and efficiency.

11. Talking-Heads Attention

Talking-Heads projects attention logits and weights across attention heads, enabling communication between heads.

Three Head Types:

h_k, h_v: project queries and keys/values across heads.
h: project attention logits (pre-softmax) and weights (post-softmax) across heads.

Logits Projection:

$$A'_{head} = W_\alpha A_{head}$$
W_α ∈ ℝ^{n_heads × n_heads} learns cross-head interactions. Applied before softmax.

§5.3 Other Modifications

12. Clustered Attention: LSH + k-means

Clustering:

$$\text{Clusters } C_1, ..., C_M = \text{k-means}(Q)$$
Cluster queries into M clusters using k-means. Compute centroid query matrix Q^c.

Attention:

$$\text{Attn} = \text{softmax}(Q Q^{c,T}) \cdot \text{softmax}(Q^c K^T) V$$
First attend to cluster centroids, then within clusters. Complexity O(C·L·d_k + L·C·d_v).

13. Luna: Linear Unified Nested Attention

Two-Step Attention:

$$Y_P = \text{Attn}(P, C) \quad \text{(Packing, } O(l \cdot m) \text{)}$$ $$Y_X = \text{Attn}(X, Y_P) \quad \text{(Unpacking, } O(l \cdot n) \text{)}$$
P: packed representation; C: context; X: input (length n); m: packed length. Total complexity O(lm + ln), linear in input length.

14. ELECTRA: Replaced Token Detection

ELECTRA uses a generator-discriminator approach: the generator produces MLM predictions, the discriminator detects replaced tokens.

Generator (MLM-style):

$$P_G(x_t | x_{-t}) = \text{softmax}(W_g h_t^G)$$
Small model predicts masked tokens. Sample replacements: x'_t ~ P_G(· | x_{-t}).

Discriminator (Token-level Classification):

$$P_D(x_t \text{ replaced} | x') = \sigma(w_d^T h_t^D)$$
Binary classification for each token: original or replaced. Larger model (becomes final model) learns from 100% of tokens, not just 15% masked tokens.

15. Switch Transformer: Mixture of Experts with Hard Routing

Expert Routing:

$$\text{Expert}(x) = \arg\max(\text{softmax}(W_r \cdot x)) \in [1, ..., n_e]$$
Each token routed to exactly one expert via hard max. Routing matrix W_r learned end-to-end.

Scaling:

$$\text{Tokens per expert: } L / n_e \quad \text{(capacity factor c)}$$
Distribute tokens evenly across experts. Auxiliary loss encourages balanced load to avoid expert starvation.

Code Block 1: Sliding Window Attention from Scratch

Implement Longformer-style sliding window attention to understand the efficiency trade-off.

import torch import torch.nn as nn import torch.nn.functional as F class SlidingWindowAttention(nn.Module): """Longformer-style sliding window attention""" def __init__(self, d_model, n_heads, window_size): super().__init__() self.d_model = d_model self.n_heads = n_heads self.window_size = window_size # one-sided self.d_k = d_model // n_heads self.W_q = nn.Linear(d_model, d_model) self.W_k = nn.Linear(d_model, d_model) self.W_v = nn.Linear(d_model, d_model) self.W_o = nn.Linear(d_model, d_model) def forward(self, x): B, L, D = x.shape # Project Q, K, V Q = self.W_q(x).view(B, L, self.n_heads, self.d_k).transpose(1, 2) K = self.W_k(x).view(B, L, self.n_heads, self.d_k).transpose(1, 2) V = self.W_v(x).view(B, L, self.n_heads, self.d_k).transpose(1, 2) # Build sliding window mask: O(L*w) instead of O(L²) w = self.window_size mask = torch.zeros(L, L, device=x.device, dtype=torch.bool) for i in range(L): start = max(0, i - w) end = min(L, i + w + 1) mask[i, start:end] = True # Compute attention with mask scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5) scores = scores.masked_fill(~mask.unsqueeze(0).unsqueeze(0), float('-inf')) attn = F.softmax(scores, dim=-1) out = torch.matmul(attn, V) out = out.transpose(1, 2).contiguous().view(B, L, D) return self.W_o(out) # ⚡ OPTIMIZATION NOTE: # Real Longformer uses custom CUDA kernels for true O(L*w) memory. # Dense mask above is O(L²) in memory; use for L < 8192 tokens.
Key Insight: This implementation is educational but not production-optimized. Dense mask creation is O(L²) in memory. Real Longformer uses block-sparse CUDA ops for true O(L·w) memory usage.

Code Block 2: LSH Attention (Reformer-style)

Simplified implementation showing how Locality-Sensitive Hashing groups similar vectors for efficient attention.

class LSHAttention(nn.Module): """Simplified Reformer-style LSH attention""" def __init__(self, d_model, n_hashes=4, n_buckets=32): super().__init__() self.d_model = d_model self.n_hashes = n_hashes self.n_buckets = n_buckets # Random projection matrix for hashing self.register_buffer('R', torch.randn(d_model, n_buckets // 2)) def hash_vectors(self, x): """LSH: h(x) = argmax([xR; -xR])""" projected = x @ self.R hashes = torch.cat([projected, -projected], dim=-1) return hashes.argmax(dim=-1) def forward(self, x): B, L, D = x.shape # In Reformer, Q = K for efficiency qk = x v = x all_attn = torch.zeros_like(x) for r in range(self.n_hashes): R = torch.randn_like(self.R) projected = qk @ R buckets = torch.cat([projected, -projected], dim=-1).argmax(dim=-1) # Sort by bucket, attend within buckets sorted_idx = buckets.argsort(dim=-1) sorted_qk = torch.gather(qk, 1, sorted_idx.unsqueeze(-1).expand(-1,-1,D)) sorted_v = torch.gather(v, 1, sorted_idx.unsqueeze(-1).expand(-1,-1,D)) # Chunk into blocks and compute local attention chunk_size = max(1, L // (self.n_buckets // 2)) for start in range(0, L, chunk_size): end = min(start + chunk_size, L) q_chunk = sorted_qk[:, start:end] kv_start = max(0, start - chunk_size) k_chunk = sorted_qk[:, kv_start:end] v_chunk = sorted_v[:, kv_start:end] scores = torch.matmul(q_chunk, k_chunk.transpose(-2,-1)) scores = scores / (D ** 0.5) weights = F.softmax(scores, dim=-1) chunk_out = torch.matmul(weights, v_chunk) orig_idx = sorted_idx[:, start:end] all_attn.scatter_add_(1, orig_idx.unsqueeze(-1).expand(-1,-1,D), chunk_out) return all_attn / self.n_hashes # 🔑 Complexity: O(L log L) because LSH groups similar vectors. # Multi-round hashing reduces approximation error.

" * 500 inputs = tokenizer(long_text, return_tensors="pt", max_length=4096, truncation=True, padding="max_length") # Set global attention on [CLS] token for classification global_attention_mask = torch.zeros_like(inputs['input_ids']) global_attention_mask[:, 0] = 1 # CLS token gets global attention outputs = model(**inputs, global_attention_mask=global_attention_mask) logits = outputs.logits prediction = torch.argmax(logits, dim=-1) # 💡 TIP: Global attention tokens are task-specific: # - Classification: [CLS] token # - QA: all question tokens # - Summarization: beginning-of-doc tokens

Model Failure Scenarios & Diagnostics

🧩 Diagnostic Questions

Expand each question to see detailed explanations and solutions for common failure modes.

Q1: Longformer fails on cross-paragraph reasoning ▼

Scenario: Your Longformer model performs well on QA tasks asking about single paragraphs, but fails when questions require cross-paragraph reasoning across 20+ pages. Window size is 256 tokens.

Root Cause: The sliding window is too small to propagate information between distant paragraphs. Even with global attention on question tokens, the window can't bridge gaps between passages that are 500+ tokens apart.

Solutions:

  • Increase window size: Change w from 256 to 512 or 1024. Trade: higher memory/compute but better long-range coverage.
  • Add global attention tokens: Mark paragraph boundary tokens or section headers as global. They attend to everything, acting as "bridges" between passages.
  • Use dilated sliding window: In upper layers, use dilation d=2 or d=4 to extend effective context from w to w·(1+d).
  • Hybrid approach: Sliding window for lower layers (computational efficiency), larger windows or global attention in upper layers (information flow).
Q2: Reformer predictions are inconsistent at inference ▼

Scenario: Your Reformer model trains well, but at inference, running the same input twice produces different outputs. Prediction probabilities vary by ±10%.

Root Cause: LSH is stochastic. The random projection matrices R used for hashing change each forward pass. With few hash rounds (n_hashes=1 or 2), different tokens may hash to different buckets across runs, changing which vectors attend to each other.

Solutions:

  • Increase n_hashes: Use n_hashes=8 or 16 instead of 1–2. Multiple hash rounds ensure consistent token pairings even if individual hashes differ. Complexity: O(L log L · n_hashes).
  • Use deterministic hashing at inference: Seed random number generator with fixed value before forward pass. Trades off stochasticity for reproducibility.
  • Average multiple runs: For critical decisions, run the model 5–10 times and average predictions. Reduces variance.
  • Fine-tune with higher n_hashes: If trained with n_hashes=2, fine-tune with n_hashes=8 at inference to match distribution shift.
Q3: Performer accuracy degrades when extending to longer sequences ▼

Scenario: Performer trained on 512 tokens works well. When you extend to 4,096 tokens at inference (using the same model), performance drops 5–15% despite "linear complexity" claims.

Root Cause: FAVOR+ kernel approximation error grows with sequence length. The random features ω were calibrated for shorter sequences. The approximation exp(q·k^T) ≈ ⟨φ(q)^T φ(k)⟩ becomes less accurate as the distribution of dot products shifts—longer sequences have more outliers.

Solutions:

  • Increase feature dimension r: Use more random features (e.g., r=256 instead of 64). Better approximation of kernel, higher O(L·r) cost but still subquadratic.
  • Periodically resample ω: During inference, resample random projection matrices every 512 tokens. Keeps approximation error bounded.
  • Fine-tune on longer sequences: Train on L=4,096 instead of L=512. Model adapts to distribution shift in dot products.
  • Hybrid FAVOR+/full attention: Use FAVOR+ in lower layers, full attention in upper layers. Lower layers capture local patterns (FAVOR+ is good), upper layers benefit from precise long-range attention.
Q4: Switch Transformer quality drops despite same training FLOPs ▼

Scenario: You switch from 12-layer standard Transformer to 12-layer Switch Transformer with 128 experts per layer. Training FLOPS are identical (each token still routes to 1 expert ≈ equivalent FFN compute), but translation quality drops 2–3 BLEU.

Root Cause: Load imbalance. Not all experts are used equally. Some experts become "superstars" receiving 10–20% of tokens while others get <1%. This means:

  • Popular experts have limited capacity → bottleneck → dropped tokens or reduced expressiveness
  • Unused experts waste parameters
  • No incentive for load balancing if all tokens route to same expert

Solutions:

  • Add auxiliary load-balancing loss: Penalize imbalance: L_aux = α · (variance of expert counts). Forces uniform load.
  • Use expert dropout: Randomly drop some expert outputs during training. Prevents over-reliance on popular experts.
  • Tune capacity factor c: If c=1.0, each expert handles L/n_e tokens. Too small → dropped tokens; too large → empty experts. Use c=1.25–1.5.
  • Reduce number of experts: Try 32 or 64 experts per layer instead of 128. Easier to load-balance; still better than dense FFN.
  • Soft routing (weighted average): Instead of hard routing to 1 expert, route to top-2 or top-4 with learned weights. More flexible but higher compute.
Q5: BigBird fails on code understanding despite working on NLI ▼

Scenario: BigBird with random attention, sliding window (w=3), and global tokens works well on English NLI (MNLI, etc.). But when applied to code understanding (bracket matching, scoping), accuracy drops 10–20%.

Root Cause: BigBird's sparsity is probabilistic/statistical, not deterministic. Code requires precise long-range pairing (e.g., matching opening and closing brackets separated by hundreds of tokens). Random attention provides coverage in expectation, not guarantee. Sliding window covers only local context. Global tokens aggregate but don't precisely pair.

  • Random attention: bracket at position 100 might attend to position 350 with probability 1/L, but not guaranteed.
  • Sliding window w=3: covers [i-3...i+3] only; can't bridge jumps of 200+ tokens between matched pairs.
  • Global tokens: collect information but don't encode *which* brackets match *which*.

Solutions:

  • Use standard full attention for code: Code is small (< 2K tokens). Full attention is fine. O(4M) << O(16G) for longer text, but much more accurate.
  • Longformer with structured global attention: Mark bracket/delimiter tokens as global. They attend to everything, enabling precise matching.
  • Hybrid sparse+dense: Lower layers use BigBird sparsity (efficiency). Upper layers switch to full attention for precise reasoning.
  • Synthetic pre-training: Pre-train on synthetic code (generated bracket matching sequences) to prime the model to track long-range dependencies before fine-tuning on real code.
  • Fine-tune with larger window: Increase w from 3 to 64 or 128 for code tasks specifically. Accept higher compute for better accuracy.
🎯 Lesson: Efficiency trade-offs are real. Sparse attention saves compute but loses expressiveness. Always profile your specific task: if accuracy drop > 2%, dense attention or hybrid approaches may be worth the extra compute.
