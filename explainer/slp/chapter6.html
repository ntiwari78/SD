Neural Networks

"The beauty of a neural network is that it doesn't need to understand the world—it just needs to learn from examples."

Neural networks are the computational foundation of modern NLP and deep learning. From sentiment classification to large language models, understanding how networks learn through backpropagation is essential. This chapter builds your intuition from individual neurons up to multi-layer architectures, showing why deep networks with non-linearities can solve problems that simple linear models cannot.

The Factory Analogy

Feynman

Imagine a factory assembly line. Each unit (neuron) is a worker standing at a station. Each worker's job is simple: they receive items (inputs) from the previous station, multiply each item by a weight (their skill level at that task), add everything up, and pass the result through one of three devices (activation functions). The transformed item moves to the next station.

A layer is a row of workers at adjacent stations. Multiple layers form a deep pipeline. Data enters the first station, gets transformed by the first layer of workers, passes to the second layer, and so on, until the final product (prediction) emerges.

The magic happens during backpropagation. At the end of the line, a quality inspector (loss function) checks the final product against the target. If the product is wrong, the inspector becomes a "foreman" who walks backward through the line, telling each worker: "Your item was wrong by this much. You need to adjust your weights like this." Each worker slightly tweaks their weights based on this feedback. Over thousands of products, workers collectively learn to produce the right outputs.

Without non-linear activation functions, every layer would just be a scaled-up version of the previous transformation—like having multiple multiplications in a row, which collapse into a single multiplication. The non-linearity at each station (sigmoid, tanh, ReLU) bends and folds the data into new shapes, allowing the network to learn complex, non-linear patterns.

Technical Foundations

Technical

6.1: Units and Activation Functions

A single neuron (unit) computes a weighted sum of its inputs plus a bias term, then passes it through an activation function.

$$z = \mathbf{w} \cdot \mathbf{x} + b$$ $$a = f(z)$$

where w are weights, x are inputs, b is bias, and f is an activation function.

Common Activation Functions:

$$\text{Sigmoid: } \sigma(z) = \frac{1}{1 + e^{-z}}$$ $$\text{Tanh: } \tanh(z) = \frac{e^{z} - e^{-z}}{e^{z} + e^{-z}}$$ $$\text{ReLU: } \text{ReLU}(z) = \max(0, z)$$

The vanishing gradient problem occurs with sigmoid and tanh: as you backpropagate through many layers, gradients multiply by small values (sigmoid/tanh derivatives are between 0 and 0.25), so deep layers receive vanishingly small gradient signals. ReLU has derivative 1 for positive inputs, mitigating this issue.

6.2: The XOR Problem

A single perceptron (one-layer network with no hidden layer) can only solve linearly separable problems. The XOR problem is not linearly separable:

$$\text{XOR: } (0,0) \to 0, \quad (0,1) \to 1, \quad (1,0) \to 1, \quad (1,1) \to 0$$

No single line can separate the 1s from the 0s. This was proven by Minsky & Papert in 1969, triggering an AI winter. However, a 2-layer network with non-linear activations (e.g., ReLU) can solve XOR by:

  • Layer 1 (2 ReLU units): transforms the 2D input space so that the four points are rearranged to become linearly separable.
  • Layer 2 (1 output unit): applies a linear decision boundary to the transformed space.

6.3: Feedforward Networks

A feedforward network (also called MLP or multilayer perceptron) has layers stacked sequentially. Data flows from input → hidden layers → output layer.

$$\mathbf{h}^{[1]} = f(\mathbf{W}^{[1]} \mathbf{x} + \mathbf{b}^{[1]})$$ $$\mathbf{h}^{[2]} = f(\mathbf{W}^{[2]} \mathbf{h}^{[1]} + \mathbf{b}^{[2]})$$ $$\hat{\mathbf{y}} = \text{softmax}(\mathbf{W}^{[L]} \mathbf{h}^{[L-1]} + \mathbf{b}^{[L]})$$

where superscript [i] denotes the i-th layer, and softmax is used for multi-class classification:

$$\text{softmax}(\mathbf{z})_j = \frac{e^{z_j}}{\sum_{k} e^{z_k}}$$

Why Non-linearity Matters: If all layers use only linear transformations (no activation function), then composing them yields another linear transformation. A 5-layer purely linear network is equivalent to a 1-layer network. Non-linearities break this collapse, enabling the network to approximate complex functions.

6.4: NLP Classification with Feedforward Networks

For text classification (e.g., sentiment), a typical pipeline is:

  • Feature Extraction: Convert text to vectors (bag-of-words, TF-IDF, or learned embeddings).
  • Feedforward Network: Pass vectors through layers of ReLU units to learn non-linear decision boundaries.
  • Output: Softmax over classes (positive/negative/neutral).

Early NLP relied on hand-crafted features (n-grams, syntax trees). Modern approaches let the network learn features in hidden layers, often outperforming hand-built ones.

6.5: Embeddings as Network Input

Instead of one-hot encoding (which is sparse and high-dimensional), we use learned embeddings. An embedding matrix E ∈ ℝ^{|V| × d} maps each word in vocabulary V to a d-dimensional dense vector:

$$\mathbf{x}_i = \mathbf{E} \mathbf{e}_i$$

where ei is the one-hot vector for word i.

For a sentence, we can use mean-pooling (average all word embeddings) or concatenation (append embeddings) to create a fixed-size input. For a neural language model, embeddings from previous words are concatenated:

$$\mathbf{x} = [\mathbf{E} w_{t-n+1}, \ldots, \mathbf{E} w_t]$$

6.6: Training

Cross-Entropy Loss for binary classification:

$$L_{CE} = -[y \log \hat{y} + (1-y) \log(1-\hat{y})]$$

For multi-class:

$$L_{CE} = -\sum_{k=1}^{K} y_k \log \hat{y}_k$$

Backpropagation computes gradients via the chain rule. Construct a computation graph where each operation (matrix multiply, add bias, apply ReLU) is a node. During backward pass, gradients flow from output to input:

$$\frac{\partial f}{\partial x} = \frac{\partial f}{\partial u} \cdot \frac{\partial u}{\partial x}$$

Hyperparameters:

  • Learning Rate (η): Controls step size. Too high → overshooting, too low → slow learning. Often decayed during training.
  • Batch Size: Number of examples before updating weights. Larger batches = more stable gradients but slower overall training.
  • Network Architecture: Number of layers, layer widths, activation functions.
  • Regularization: L2 weight decay, dropout (randomly zero out units with probability p during training to prevent co-adaptation).

Building a Neural Network from Scratch

Code

Below is a complete implementation of a 2-layer feedforward neural network using NumPy. This teaches the core mechanics of forward propagation, loss computation, backpropagation, and gradient descent.

# Neural Network from Scratch
import numpy as np

class NeuralNetwork:
    def __init__(self, input_size, hidden_size, output_size):
        # Initialize weights and biases
        self.W1 = np.random.randn(input_size, hidden_size) * 0.01
        self.b1 = np.zeros((1, hidden_size))
        self.W2 = np.random.randn(hidden_size, output_size) * 0.01
        self.b2 = np.zeros((1, output_size))

    def relu(self, z):
        # ReLU activation: max(0, z)
        return np.maximum(0, z)

    def relu_derivative(self, z):
        # Derivative: 1 if z > 0, else 0
        return (z > 0).astype(float)

    def softmax(self, z):
        # Softmax for multi-class: e^z_i / sum(e^z_j)
        exp_z = np.exp(z - np.max(z, axis=1, keepdims=True))
        return exp_z / np.sum(exp_z, axis=1, keepdims=True)

    def forward(self, X):
        # Forward pass through 2 layers
        self.z1 = np.dot(X, self.W1) + self.b1
        self.h = self.relu(self.z1)
        self.z2 = np.dot(self.h, self.W2) + self.b2
        self.output = self.softmax(self.z2)
        return self.output

    def compute_loss(self, predictions, targets):
        # Cross-entropy loss
        m = targets.shape[0]
        log_probs = -np.log(predictions[np.arange(m), targets] + 1e-8)
        return np.mean(log_probs)

    def backward(self, X, targets, learning_rate):
        # Backpropagation: compute gradients and update
        m = X.shape[0]

        # Output layer gradient
        dz2 = self.output.copy()
        dz2[np.arange(m), targets] -= 1
        dz2 /= m

        dW2 = np.dot(self.h.T, dz2)
        db2 = np.sum(dz2, axis=0, keepdims=True)

        # Hidden layer gradient
        dh = np.dot(dz2, self.W2.T)
        dz1 = dh * self.relu_derivative(self.z1)

        dW1 = np.dot(X.T, dz1)
        db1 = np.sum(dz1, axis=0, keepdims=True)

        # Update weights (SGD)
        self.W1 -= learning_rate * dW1
        self.b1 -= learning_rate * db1
        self.W2 -= learning_rate * dW2
        self.b2 -= learning_rate * db2

    def train(self, X, y, epochs=100, batch_size=32, learning_rate=0.01):
        # Mini-batch SGD training loop
        n_samples = X.shape[0]
        for epoch in range(epochs):
            # Shuffle data
            indices = np.random.permutation(n_samples)
            X_shuffled = X[indices]
            y_shuffled = y[indices]

            # Mini-batch updates
            for i in range(0, n_samples, batch_size):
                X_batch = X_shuffled[i:i+batch_size]
                y_batch = y_shuffled[i:i+batch_size]

                self.forward(X_batch)
                self.backward(X_batch, y_batch, learning_rate)

            # Log progress
            if epoch % 10 == 0:
                preds = self.forward(X)
                loss = self.compute_loss(preds, y)
                acc = np.mean(np.argmax(preds, axis=1) == y)
                print(f"Epoch {epoch}: Loss {loss:.4f}, Acc {acc:.2%}")

# Example usage
if __name__ == "__main__":
    # Generate synthetic classification data
    np.random.seed(42)
    X = np.random.randn(200, 4)
    y = np.random.randint(0, 3, 200)

    # Create and train network
    nn = NeuralNetwork(input_size=4, hidden_size=8, output_size=3)
    nn.train(X, y, epochs=50, batch_size=16, learning_rate=0.1)

Key Points:

  • Forward pass: Compute h = ReLU(X @ W1 + b1), then output = softmax(h @ W2 + b2).
  • Loss: Cross-entropy measures prediction error against true labels.
  • Backward pass: Use chain rule to compute dW1, db1, dW2, db2 from dL/doutput backward.
  • SGD: Update weights in the direction opposite to gradients by step size (learning rate).
  • Mini-batches: Process 16-32 samples at once for stability and computational efficiency.

From MLPs to Transformers

Modern

The feedforward networks you've learned about are not relics of the past—they are building blocks of state-of-the-art models in 2026. Inside every transformer (the architecture powering GPT, Claude, Llama, and other large language models), there are MLP sub-layers.

Transformer Feed-Forward Layer

Each transformer block contains a "feed-forward network" after the attention layer:

$$\text{FFN}(x) = \text{ReLU}(x W_1 + b_1) W_2 + b_2$$

In modern architectures, this is often written with GELU or SwiGLU activation instead of ReLU, but the principle remains: a 2-layer MLP adds non-linear expressiveness within the transformer. The parameters in these MLP sub-layers constitute a large portion of the model's total parameters.

Scaling Principles

When training billion-parameter models, the same backpropagation algorithm runs—just at scale. Modern LLMs use:

  • Distributed training: Gradient computation spread across thousands of GPUs.
  • Optimizers: Adam or AdamW automatically adjust learning rates per parameter, converging faster than vanilla SGD.
  • Layer normalization: Normalizes outputs of each layer to mean 0, variance 1, stabilizing training.
  • Residual connections: Skip connections that add the input directly to the output (y = f(x) + x) let gradients flow directly backward, solving vanishing gradient problems in very deep networks.
  • Dropout & regularization: Still used to prevent overfitting even in 100B-parameter models.

Why This Matters

Understanding how individual neurons, layers, and backpropagation work is essential to understanding modern LLMs. When you read a paper describing a model with "24 transformer layers, each with a 3072-dimensional feed-forward network," you now know:

  • Each layer has an MLP with hidden size 3072.
  • During training, gradients flow backward through all 24 layers via the chain rule.
  • The model is solving a scaled-up version of the same optimization problem you see in the toy examples above.

The scaling is dramatic (from millions to trillions of parameters), but the fundamental mechanics—forward pass, loss computation, backpropagation, gradient descent—are identical.

Socratic Knowledge Check

Socratic

Scenario: The Training Plateau

You've trained a 5-layer feedforward network for binary sentiment classification on a dataset of 10,000 movie reviews. After 100 epochs, training loss is stuck at 0.70 (barely better than random), validation accuracy hovers at 52% (50% is random for binary classification), and there's no improvement. What could be wrong?

Use the questions below to diagnose the problem. Click each to reveal diagnostic hints.

Q1: What activation function are you using in the hidden layers?
If using sigmoid or tanh: This is likely your problem. In a 5-layer network with sigmoid, gradients during backpropagation multiply by values between 0 and 0.25 at each layer. By the time you reach the first layer, gradients are effectively zero (e.g., 0.25^4 = 0.004). The early layers can't learn. Solution: Switch to ReLU for hidden layers. Use sigmoid only on the final output for binary classification.
$$\text{Better:} h^{[1]} = \text{ReLU}(W^{[1]} x + b^{[1]}), \ldots, \text{output} = \sigma(W^{[5]} h^{[4]} + b^{[5]})$$
Q2: What is your learning rate? Have you tried different values?
Too high (e.g., 0.1): Loss oscillates wildly and never decreases—the model overshoots the minimum. Too low (e.g., 0.00001): Training is glacially slow; you need 10,000 epochs instead of 100. Solution: Try 0.001 to 0.01 first. Use learning rate scheduling: start at 0.01, decay to 0.001 after 50 epochs. Or use Adam optimizer, which adapts the learning rate per parameter automatically.
Q3: Have you plotted gradient magnitudes during backpropagation? Could you have dead ReLU neurons?
Dead ReLU problem: If many neurons receive mostly negative inputs during forward pass, their ReLU output is always 0. Their gradient is also 0, so they never update their weights. Once dead, they're stuck. Diagnosis: Log activations; if a neuron is 0% active (never fires), it's dead. Solutions: (1) Use Leaky ReLU: f(z) = max(0.01z, z), so gradient is 0.01 for negative inputs. (2) Reduce learning rate or batch size. (3) Initialize weights more carefully (He initialization: variance = 2/input_size).
Q4: What are your input features? Are you using pre-trained embeddings or random?
If using random embeddings: On a 10K-sample dataset, the embedding matrix (e.g., vocab_size=5000, embedding_dim=100) has 500K parameters. These must all be learned from 10K examples—easy to overfit and hard to generalize. Solution: Use pre-trained embeddings (Word2Vec, GloVe, or from a language model) which encode semantic relationships learned from billions of words. Alternatively, reduce embedding dimension or add dropout. If you must train embeddings, freeze them initially and fine-tune later.
Q5: Is your cross-entropy loss correctly implemented? Common bugs?
Common bugs:
  • Forgetting the log: L = y * ŷ (wrong) vs L = -y * log(ŷ) (correct).
  • Softmax axis error: If you apply softmax to wrong axis, outputs don't sum to 1 per sample.
  • Numerical instability: log(0) is undefined. Use log(ŷ + 1e-8) or compute cross-entropy in log-space (logsumexp trick).
  • Scaling: Are you dividing by batch size? L = sum(losses) / batch_size.
Quick fix: Use a library (PyTorch's nn.CrossEntropyLoss, TensorFlow's SparseCategoricalCrossentropy) which handles these correctly.

Integrated Diagnosis

Most likely culprit in a 5-layer network: vanishing gradients with sigmoid (Q1) or dead ReLUs (Q3). The fix is almost always to use ReLU. If ReLU doesn't help, the next step is checking your learning rate and loss implementation. Debugging neural networks is an empirical skill—you'll develop intuition by training many models and observing what works.
