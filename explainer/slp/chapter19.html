Dependency Parsing

From phrase structure trees to syntactic networks: understanding how words depend on each other to form sentences.

From Phrase Trees to Word Connections

The Core Insight: Chapter 18 taught us to build phrase structure trees, where words are grouped into nested constituents (noun phrases, verb phrases, etc.). Now we shift perspective entirely. Instead of asking "what groups with what?", we ask "who depends on whom?"

The Org Chart Analogy

Imagine a company:

  • Constituency parsing is like an org chart: departments contain teams, teams contain people. The structure is hierarchical and nested.
  • Dependency parsing is like a reporting network: every person reports to exactly one boss (or no one if they're the CEO). The structure is a directed graph where each word has one head.

Consider the sentence: "The cat sat on the mat"

  • In constituency parsing, we might build: [S [NP [Det The] [N cat]] [VP [V sat] [PP [P on] [NP [Det the] [N mat]]]]]
  • In dependency parsing, we draw arrows: cat ← sat (subject), on ← sat (modifier), mat ← on (object of preposition)

Key Concepts

Head-Dependent Relationships: In a dependency relation, the head (or governor) is the central word, and the dependent (or modifier) is the subordinate word. A directed arc from head to dependent represents the relationship.

The Universal Dependency Framework: Modern dependency parsing uses a standardized set of 37 relation types (like NSUBJ, OBJ, NMOD) that work consistently across 100+ languages. This enables cross-lingual research and multilingual models.

Intuition: Every word except the sentence root depends on exactly one other word. The root word (usually the main verb) has no head. This creates a tree structure where arrows point from heads to dependents.

Technical Ascent: Formalism & Algorithms

19.1: Dependency Relations and Trees

A dependency structure is a directed graph $G = (V, A)$ where:

  • $V$ = set of vertices (words in the sentence, plus a special ROOT token)
  • $A$ = set of arcs (head → dependent relations)

Each arc is labeled with a dependency relation type. Universal Dependencies defines 37 universal relation types:

Clausal Relations:
  • nsubj — nominal subject
  • obj — direct object
  • iobj — indirect object
  • ccomp — clausal complement
Modifier Relations:
  • nmod — nominal modifier (e.g., adjective)
  • amod — adjectival modifier
  • det — determiner
  • case — case marking (prepositions)
Other Relations:
  • conj — conjunction (coordinate structure)
  • cc — coordinating conjunction
  • compound — compound word

Dependency Tree Constraints

A valid dependency tree must satisfy four properties:

  1. Single Root: Exactly one word (usually the main verb) has no head.
  2. Connectedness: Every word has a path to the root.
  3. Acyclicity: No cycles in the graph (no circular dependencies).
  4. Single Head: Each word (except the root) has exactly one head.

These constraints mean a valid dependency structure is always a tree rooted at the ROOT token.

Projectivity

A dependency tree is projective if all arcs can be drawn above the sentence without crossing. Formally, an arc $(h, d)$ is projective if for all words between $h$ and $d$, there is a path from $h$ to that word.

Empirical Fact: English is highly projective (~98% of sentences). However, languages with freer word order (Czech, German, Dutch, Latin) have frequent non-projective structures (~20-30%). This distinction is crucial: some parsing algorithms only handle projective trees.

19.2: Transition-Based Dependency Parsing

Transition-based parsers build the dependency tree incrementally through a series of local decisions. The parser maintains:

  • A stack $\sigma$: partially processed words
  • A buffer $\beta$: unprocessed words
  • A set of arcs $A$: dependency relations created so far

Arc-Standard System

The arc-standard transition system defines three operations:

SHIFT: Move the first word from the buffer to the top of the stack.
Precondition: buffer is non-empty
LEFT-ARC(label): Create a dependency arc from the top of the stack (head) to the second-top element (dependent), then remove the second-top element.
Precondition: stack has at least 2 elements; the second-top element is not the ROOT.
RIGHT-ARC(label): Create a dependency arc from the second-top element of the stack (head) to the top (dependent), then remove the top element.
Precondition: stack has at least 2 elements.

Oracle and Classifier

To train a transition-based parser, we need an oracle: a function that, given the current configuration and the gold-standard parse, tells us which transition to take. The oracle ensures that following its decisions will produce the gold parse.

A classifier (typically a neural network) learns to predict the oracle's decision given features of the current configuration:

  • Top of stack: word, POS tag, dependency label of its head
  • First buffer word: word, POS tag
  • Second buffer word: word, POS tag (lookahead)
  • Embeddings of these words and their relations

Modern parsers use BiLSTM encoders or Transformers to compute contextualized representations of stack and buffer items, then pass these through a softmax classifier to predict the transition.

Complexity

A sentence with $n$ words requires $2n$ transitions (n SHIFT, n LEFT/RIGHT-ARC operations). With a trained classifier, parsing is linear in time: $O(n)$. However, transition-based parsers cannot produce non-projective parses (by design of the system).

19.3: Graph-Based Dependency Parsing

Instead of building the tree step-by-step, graph-based parsers view dependency parsing as an optimization problem: find the highest-scoring spanning tree.

Score Function: For each potential arc $(h \to d)$ with label $l$, compute a score $s(h, d, l)$. This is typically a learned function.

Objective: Find the directed spanning tree (rooted at ROOT) that maximizes the sum of arc scores:

$$\text{Best Parse} = \arg\max_T \sum_{(h,d,l) \in T} s(h, d, l)$$

Maximum Spanning Tree Algorithms

Chu-Liu-Edmonds Algorithm: Finds the maximum spanning tree in a directed graph in $O(n^2)$ time. Key insight: recursively remove cycles and contract them.

Eisner's Algorithm: For projective graphs, this dynamic-programming algorithm runs in $O(n^3)$ time and is more efficient for projective languages like English.

Neural Arc Scoring

Modern systems use biaffine classifiers to score arcs. A Transformer or BiLSTM encodes the sentence, producing representations $\mathbf{h}_i$ for each word $i$. Then we compute head and dependent projections:

$$\mathbf{h}_i^{\text{head}} = \text{Linear}_h(\mathbf{h}_i)$$ $$\mathbf{h}_j^{\text{dep}} = \text{Linear}_d(\mathbf{h}_j)$$

The biaffine score for arc $(i \to j)$ is:

$$s(i, j) = \mathbf{h}_i^{\text{head}\top} \mathbf{U} \mathbf{h}_j^{\text{dep}} + \mathbf{w}^\top [\mathbf{h}_i^{\text{head}}; \mathbf{h}_j^{\text{dep}}] + b$$ where $\mathbf{U}$ is a learned matrix, $\mathbf{w}$ is a learned weight vector, and $b$ is a bias term. This bilinear form allows the score to capture multiplicative interactions between head and dependent representations.

Advantages and Disadvantages

Transition-Based:
  • ✓ Linear time complexity
  • ✓ Easy to implement
  • ✗ Cannot produce non-projective parses
  • ✗ Greedy (commits to decisions early)
Graph-Based:
  • ✓ Can handle non-projective trees
  • ✓ Global optimization (no greedy mistakes)
  • ✗ Slower ($O(n^2)$ to $O(n^3)$)
  • ✗ More complex inference

19.4: Evaluation Metrics

Two standard metrics evaluate dependency parsers:

Unlabeled Attachment Score (UAS): Percentage of words whose head is correctly predicted (ignoring dependency labels). $$\text{UAS} = \frac{\# \text{ correct heads}}{n \text{ words}} \times 100\%$$
Labeled Attachment Score (LAS): Percentage of words whose head AND label are correct. $$\text{LAS} = \frac{\# \text{ correct (head, label) pairs}}{n \text{ words}} \times 100\%$$

State-of-the-Art: Modern parsers trained on the Penn Treebank achieve ~95% LAS for English. Cross-lingual models trained on Universal Dependencies reach ~85% LAS averaged across all languages.

Arc-Standard Transition Parser

class TransitionParser: """Arc-standard transition-based dependency parser.""" def __init__(self): self.stack = [] self.buffer = [] self.arcs = [] def init_config(self, words): """Initialize configuration: stack=[ROOT], buffer=all words.""" self.stack = [('ROOT', 0)] self.buffer = [(w, i+1) for i, w in enumerate(words)] self.arcs = [] def shift(self): """Move first buffer word to top of stack.""" self.stack.append(self.buffer.pop(0)) def left_arc(self, label): """Create arc: stack[-1] → stack[-2], remove stack[-2].""" dependent = self.stack[-2] head = self.stack[-1] self.arcs.append((head[1], dependent[1], label)) self.stack.pop(-2) def right_arc(self, label): """Create arc: stack[-2] → stack[-1], remove stack[-1].""" dependent = self.stack[-1] head = self.stack[-2] self.arcs.append((head[1], dependent[1], label)) self.stack.pop() def is_terminal(self): """Check if parsing is complete.""" return len(self.buffer) == 0 and len(self.stack) == 1 def parse_with_oracle(self, words, oracle_actions): """Parse by following oracle transitions.""" self.init_config(words) for action, label in oracle_actions: if action == 'SHIFT': self.shift() elif action == 'LEFT-ARC': self.left_arc(label) elif action == 'RIGHT-ARC': self.right_arc(label) return self.arcs

This parser builds a tree step-by-step. At each configuration, the oracle classifier predicts whether to SHIFT (move buffer word to stack), LEFT-ARC (create arc from top-of-stack to second element), or RIGHT-ARC (opposite direction).

Eisner's Algorithm for Projective MST Parsing

import numpy as np def eisner_parse(scores): """Eisner's algorithm for projective MST dependency parsing. Args: scores: ndarray of shape (n+1, n+1) where scores[h][d] is the score of the arc from word h to word d. (Word 0 is the ROOT token.) Returns: heads: list where heads[i] is the head of word i. """ n = scores.shape[0] # C[s][t][dir][complete]: score of best tree over span [s,t] # dir: 0 = left-pointing, 1 = right-pointing # complete: 0 = incomplete, 1 = complete C = np.full((n, n, 2, 2), -np.inf) backpointers = [[[[-1] for _ in range(2)] for _ in range(2)] for _ in range(n)] for _ in range(n)] # Base case: single words for s in range(n): C[s][s][0][1] = 0.0 C[s][s][1][1] = 0.0 # Fill DP table by span width for width in range(1, n): for s in range(n - width): t = s + width # Incomplete spans (arc not yet complete) for q in range(s, t): # Right arc: s is head, t is dependent score = C[s][q][1][1] + C[q+1][t][0][1] + scores[s][t] if score > C[s][t][1][0]: C[s][t][1][0] = score backpointers[s][t][1][0] = q # Left arc: t is head, s is dependent score = C[s][q][1][1] + C[q+1][t][0][1] + scores[t][s] if score > C[s][t][0][0]: C[s][t][0][0] = score backpointers[s][t][0][0] = q # Complete spans for q in range(s, t): C[s][t][0][1] = max(C[s][t][0][1], C[s][q][0][1] + C[q][t][0][0]) for q in range(s+1, t+1): C[s][t][1][1] = max(C[s][t][1][1], C[s][q][1][0] + C[q][t][1][1]) return C[0][n-1][1][1] # Score of best tree

Eisner's algorithm uses dynamic programming over spans. For each span [s, t], we track the best incomplete and complete trees. This finds the globally optimal parse in $O(n^3)$ time.

Evaluation: UAS and LAS

def evaluate_dependencies(gold_arcs, pred_arcs, n_words): """Compute UAS and LAS for dependency parsing. Args: gold_arcs: list of tuples (head, dependent, label) pred_arcs: list of tuples (head, dependent, label) n_words: number of words (excluding ROOT) Returns: dict with 'UAS' and 'LAS' scores as percentages. """ uas_correct = 0 las_correct = 0 # Build dictionaries: dependent → (head, label) gold_dict = {dep: (head, label) for head, dep, label in gold_arcs} pred_dict = {dep: (head, label) for head, dep, label in pred_arcs} # Check each dependent word for word_id in range(1, n_words + 1): if word_id in gold_dict and word_id in pred_dict: g_head, g_label = gold_dict[word_id] p_head, p_label = pred_dict[word_id] # UAS: head is correct if g_head == p_head: uas_correct += 1 # LAS: both head and label are correct if g_label == p_label: las_correct += 1 uas = uas_correct / n_words * 100 las = las_correct / n_words * 100 return {'UAS': f'{uas:.1f}%', 'LAS': f'{las:.1f}%'} # Example usage gold = [(0, 1, 'nsubj'), (0, 2, 'root'), (4, 3, 'det'), (2, 4, 'obj')] pred = [(0, 1, 'nsubj'), (0, 2, 'root'), (4, 3, 'det'), (2, 4, 'nmod')] print(evaluate_dependencies(gold, pred, 4)) # Output: {'UAS': '100.0%', 'LAS': '75.0%'}

UAS counts words whose head attachment is correct (ignoring the relation label). LAS requires both the head and the label to match. LAS is stricter and more commonly reported.

Socratic Challenge: Projectivity Trade-offs

The Challenge: Transition-based parsers are fast ($O(n)$) but cannot handle non-projective structures. Graph-based parsers are slower ($O(n^2)$ to $O(n^3)$) but can handle any tree. In languages with frequent non-projective constructions (Czech ~23%, German ~20%), should we sacrifice speed for coverage?

The Empirical Landscape

English (Penn Treebank):
  • Non-projective sentences: ~2%
  • Parsing speed: Fast is fine
  • Best approach: Transition-based (MaltParser, neural variants)
Czech (Prague Dependency Treebank):
  • Non-projective sentences: ~23%
  • Parsing speed: Trade-off needed
  • Best approach: Graph-based or pseudo-projective transition-based
German/Dutch/Latin:
  • Non-projective sentences: ~10-20%
  • Free word order allows crossing arcs
  • Best approach: Hybrid systems (fast projective + non-projective recovery)

Three Solutions

1. Pseudo-Projective Transformation

Convert a non-projective tree into a projective approximation by lifting dependents. During decoding, reverse the transformation.

  • ✓ Allows transition-based parsers to handle crossing arcs
  • ✗ Information loss; less accurate on truly non-projective structures

2. Hybrid Approach

Use a fast transition-based parser for the common case, and rerank with a graph-based parser for non-projective recovery.

  • ✓ Speed on projective sentences; accuracy on non-projective
  • ✗ Complex engineering

3. Pure Graph-Based

Accept the $O(n^2)$ or $O(n^3)$ cost and use an MST parser. Modern hardware and batching make this acceptable.

  • ✓ Simple, elegant, handles all cases
  • ✗ Slower for large batches
What the field chose: Modern state-of-the-art systems (Dozat & Manning 2017, BERT-based parsers) predominantly use graph-based approaches. Speed is less critical than accuracy in modern NLP. However, transition-based systems remain popular for real-time or resource-constrained settings.

Modern Approaches & Extensions

Biaffine Neural Parsers (Dozat & Manning 2017)

The current state-of-the-art architecture combines:

  1. BiLSTM Encoder: Contextual word representations
  2. Head/Dependent Projections: Separate embeddings for head and dependent roles
  3. Biaffine Classifier: Score all $(h, d)$ pairs using the biaffine equation
  4. MST Decoding: Chu-Liu-Edmonds or Eisner for global optimization

This architecture achieves ~95% LAS on Penn Treebank English.

Transformer-Based Parsers

Replacing the BiLSTM with a pre-trained Transformer (BERT, RoBERTa, etc.):

  • ✓ Stronger contextualized representations from deep self-attention
  • ✓ Transfer learning from large-scale pre-training
  • ✓ Multilingual models enable zero-shot cross-lingual parsing
  • ✗ Slower inference; larger memory footprint

Universal Dependencies

The Universal Dependencies (UD) project provides:

  • 37 universal dependency relation types (designed to be language-independent)
  • Treebanks in 100+ languages, all annotated with the same scheme
  • Shared metrics for cross-lingual evaluation
  • Enables training multilingual parsers that generalize across languages

State-of-the-art on UD: ~85% LAS (average across all languages).

Joint POS Tagging + Parsing

Modern systems often jointly predict POS tags and dependencies in a single model:

  • Multi-task learning improves both tasks
  • Captures the mutual dependence of POS and syntactic structure
  • More efficient than pipelined approaches

Implicit Dependency Knowledge in LLMs

Large language models (GPT, LLaMA, Claude) implicitly learn dependency structures through pre-training. Probing studies show:

  • Internal representations encode syntactic dependencies
  • Attention heads correlate with dependency relations
  • LLMs can be used as zero-shot dependency parsers (via prompting)
  • Fine-tuned LLMs achieve state-of-the-art results on parsing benchmarks

Historical Context: The Evolution of Dependency Parsing

Dependency parsing has a rich history spanning linguistic theory and computational methods.

1959: Tesnière's "Éléments de syntaxe structurale" ▼

Lucien Tesnière published the foundational linguistic work on dependency grammar. He introduced the key insight: syntax is about dependency (which words govern which) rather than constituency (how words group). This was revolutionary, though largely overshadowed by Chomsky's constituency-based generative grammar in the 1960s.

1960s–1990s: The Constituency Dominance ▼

Chomsky's phrase structure grammar dominated linguistic and computational theory. Dependency parsing remained largely a niche approach used in a few languages (notably, it was standard in Soviet/Russian linguistics). The Penn Treebank (1990s) cemented constituency parsing's dominance in English NLP.

1965: Chu–Liu–Edmonds Algorithm ▼

Yoeng-jin Chu and Tseng-hong Liu (and independently, Jack Edmonds) developed the algorithm for finding maximum spanning trees in directed graphs. This became the theoretical foundation for graph-based dependency parsing decades later.

1996: Eisner's Algorithm ▼

Jason Eisner developed a cubic-time dynamic programming algorithm specifically for projective dependency parsing. This made graph-based parsing practical for English and other projective languages.

2003: Arc-Eager Transition-Based Parsing ▼

Joakim Nivre introduced the arc-eager transition system (a variant of arc-standard). This spawned a family of transition-based algorithms that became dominant due to their simplicity and speed.

2004–2006: MaltParser ▼

Joakim Nivre and Johan Hall released MaltParser, an open-source implementation of transition-based dependency parsing using SVMs. It became widely adopted and remained the standard for years. MaltParser's ease of use and speed made dependency parsing accessible to many languages.

2005: McDonald's Graph-Based Parser ▼

Ryan McDonald developed the first practical graph-based dependency parser, combining edge-factored scoring with MST inference. This showed that global optimization could compete with greedy transition-based methods.

2014: Neural Transition-Based Parsing ▼

Danqi Chen and Christopher Manning applied neural networks (dense embeddings + neural classifier) to transition-based parsing, achieving significant improvements over SVM-based systems. This opened the era of neural dependency parsing.

2017: Biaffine Parsers (Dozat & Manning) ▼

Timothy Dozat and Christopher Manning introduced the biaffine classifier architecture for graph-based parsing. Combined with BiLSTM encoders, this became the new state-of-the-art, achieving 95%+ LAS on English and remained the dominant approach for years.

2018: Universal Dependencies ▼

The Universal Dependencies project, initiated by a consortium of linguists, released treebanks in 100+ languages using a unified annotation scheme. This enabled large-scale multilingual parsing research and enabled training parsers that generalize across languages.

2018+: Transformer-Based Parsers ▼

Following the success of BERT and other pre-trained Transformers, researchers applied fine-tuned Transformers to dependency parsing. Models like mBERT and XLM-RoBERTa enabled zero-shot cross-lingual parsing and achieved state-of-the-art results on nearly all benchmarks.

Key Insight: Dependency parsing evolved from a niche linguistic theory to a core NLP task. The shift from constituency to dependency parsing in modern NLP reflects both linguistic intuitions (dependencies are simpler) and practical considerations (faster parsing, easier cross-lingual transfer).

Summary & Key Takeaways

Core Concepts

  • Dependency structures model how words grammatically depend on each other, forming a tree with one root.
  • 37 universal dependency relations (NSUBJ, OBJ, NMOD, etc.) enable consistent annotation across 100+ languages.
  • Projectivity (no crossing arcs) is a common assumption but not universal; some languages require non-projective parses.

Two Main Paradigms

  • Transition-Based: Build the tree incrementally via local decisions. Fast ($O(n)$) but greedy. Cannot handle non-projective trees.
  • Graph-Based: Score all possible arcs and find the global maximum spanning tree. Slower ($O(n^2)$ to $O(n^3)$) but optimal. Handles non-projectivity naturally.

State-of-the-Art (2026)

  • Architecture: Pre-trained Transformers (BERT, mBERT, XLM-RoBERTa) + Biaffine classifier + MST decoding.
  • Performance: ~95% LAS on English (Penn Treebank); ~85% LAS averaged across all UD languages.
  • Emerging trend: LLMs (GPT, LLaMA) implicitly learn syntactic dependencies; zero-shot or few-shot parsing is now possible.
