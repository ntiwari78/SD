CHAPTER 4

Logistic Regression and Text Classification

"One could note down in a book all the things that are known about classification and still have a small, manageable volume. But the universe of possible classifications is infinite. We classify to understand, to organize chaos into order."

— inspired by Borges' The Library of Babel
This chapter introduces logistic regression, the foundational classification algorithm for NLP. We'll explore the sigmoid function, softmax for multiclass problems, cross-entropy loss, gradient descent optimization, and the evaluation metrics (precision, recall, F₁) that guide machine learning practice. From sentiment analysis to toxicity detection, logistic regression powers text classification pipelines across industry and academia.

The Sorting Hat for Text

Feynman Bridge
Analogy 1: Movie Review Thumbs Up/Down

Imagine you're a movie critic assistant. You read reviews and must decide: positive or negative? You have a mental checklist of clues — words like "amazing," "excellent," and "wonderful" get +points; words like "terrible," "awful," and "boring" get −points. You scan the review, tally up all the evidence, and if the total score is positive, you give a thumbs up. That's logistic regression: multiply features by learned weights, sum them, and decide. The sigmoid function just squashes your sum into a 0–1 probability, letting you say "I'm 75% confident this is positive."

Analogy 2: The Thermostat Dial (Sigmoid)

The sigmoid function is like a thermostat dial that smoothly transitions between OFF (0) and ON (1). No matter how extreme your input, the output stays between 0 and 1. A huge positive input (like z = 10) pushes you close to 1 (very confident positive). A huge negative input (like z = −10) pushes toward 0 (very confident negative). An input of exactly 0 gives exactly 0.5 (total uncertainty). The curve is steep near 0.5 and flattens out at the extremes — the model becomes more confident the further you move from the boundary.

Analogy 3: Hiking Downhill (Gradient Descent)

Imagine you're blindfolded on a hilly landscape and must find the lowest valley. At each step, you feel the slope under your feet and take a step in the steepest downhill direction. That's gradient descent — you compute which direction reduces your error the most, and take a step that way. The learning rate is how big each step is: take steps that are too big and you overshoot the valley and climb the other side; take steps that are too small and it takes forever to reach the bottom. The art is finding the sweet spot.

Analogy 4: The Report Card (Precision & Recall)

Precision is like asking "Of all the students I gave an A to, how many actually deserved it?" Recall is "Of all the students who deserved an A, how many did I actually give one to?" A teacher who gives A's to every student has perfect recall (catches all deserving students) but terrible precision (most A's are wrong). A teacher who gives one A only to the absolute best student has perfect precision (every A is right) but terrible recall (missed all the other deserving students). Finding the balance is the real challenge of classification.

Technical Ascent

Technical Depth

5.1 Classification Framework

A supervised classification task involves:

  • Input: Observation x (e.g., a text document)
  • Feature vector: x = [x₁, x₂, ..., xₙ] (e.g., word counts, TF-IDF weights)
  • Output classes: Y = {y₁, y₂, ..., yₘ} (e.g., {positive, negative} or {spam, not-spam})
  • Training set: {(x⁽¹⁾, y⁽¹⁾), ..., (x⁽ᵐ⁾, y⁽ᵐ⁾)}

Any classifier has four key components:

  1. Feature representation: How do we convert raw text into numbers?
  2. Classification function: How do we compute class probabilities from features?
  3. Objective/loss function: How do we measure prediction error?
  4. Optimization algorithm: How do we find the best weights?

5.2 The Sigmoid Function

For binary classification, we use a linear combination (called the logit or log-odds):

$$z = \mathbf{w} \cdot \mathbf{x} + b$$
Logit: dot product of weights and features plus bias

We then apply the sigmoid function to squash z into [0, 1]:

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$
Sigmoid function

Key properties:

  • Domain: ℝ (all real numbers)
  • Range: (0, 1) (strictly between 0 and 1)
  • σ(0) = 0.5 (decision boundary)
  • Nearly linear around 0, saturates at extremes
  • Monotonically increasing (larger z → larger σ(z))

Probabilistic interpretation:

$$P(y = 1 | \mathbf{x}) = \sigma(\mathbf{w} \cdot \mathbf{x} + b)$$ $$P(y = 0 | \mathbf{x}) = 1 - \sigma(\mathbf{w} \cdot \mathbf{x} + b)$$
Class probabilities from sigmoid

The logit (inverse sigmoid):

$$\text{logit}(p) = \ln\left(\frac{p}{1-p}\right) = \sigma^{-1}(p)$$
Log-odds: transforms probability back to log-scale

5.3 Sentiment Classification Example

Let's classify movie reviews as positive or negative. We extract six features:

  • x₁ = count of positive words (e.g., "good," "great," "amazing")
  • x₂ = count of negative words (e.g., "bad," "terrible," "awful")
  • x₃ = presence of negation "not" or "no" (binary: 1 or 0)
  • x₄ = count of pronouns (features speaker involvement)
  • x₅ = presence of exclamation marks (binary: 1 or 0)
  • x₆ = log(document length) (longer reviews may have different tone)

Learned weights: w = [2.5, −5.0, −1.2, 0.5, 2.0, 0.7], b = 0.1

Example review: "This movie is great! I love the characters."

  • x₁ = 2 (positive words: "great," "love")
  • x₂ = 0 (no negative words)
  • x₃ = 0 (no negation)
  • x₄ = 1 (one pronoun: "I")
  • x₅ = 1 (one exclamation mark)
  • x₆ = ln(8) ≈ 2.08
$$z = (2.5)(2) + (-5.0)(0) + (-1.2)(0) + (0.5)(1) + (2.0)(1) + (0.7)(2.08) + 0.1$$ $$= 5.0 + 0 + 0 + 0.5 + 2.0 + 1.456 + 0.1 = 9.056$$ $$\sigma(9.056) \approx 0.9999$$
Prediction: 99.99% confidence in positive class

5.4 Cross-Entropy Loss

We model the binary classification as a Bernoulli random variable:

$$p(y | \mathbf{x}) = \hat{y}^y (1 - \hat{y})^{1-y}$$
Bernoulli likelihood for a single example

Taking the negative log-likelihood (cross-entropy loss):

$$\mathcal{L}_{\text{CE}}(\hat{y}, y) = -[y \log \hat{y} + (1-y) \log(1 - \hat{y})]$$
Cross-entropy loss for binary classification

Interpretation:

  • When y = 1: L = −log(ŷ). If ŷ is close to 1, loss ≈ 0 (good!). If ŷ is close to 0, loss → ∞ (penalizes confidently wrong predictions)
  • When y = 0: L = −log(1 − ŷ). If ŷ is close to 0, loss ≈ 0 (good!). If ŷ is close to 1, loss → ∞

Average loss over training set:

$$J(\mathbf{w}, b) = \frac{1}{m} \sum_{i=1}^{m} \mathcal{L}_{\text{CE}}(\hat{y}^{(i)}, y^{(i)})$$
Objective function to minimize

5.5 Gradient Descent

Goal: Find weights that minimize loss:

$$\hat{\boldsymbol{\theta}} = \arg\min_{\boldsymbol{\theta}} \frac{1}{m} \sum_{i=1}^{m} \mathcal{L}_{\text{CE}}(\hat{y}^{(i)}, y^{(i)})$$
Maximum likelihood estimation

Gradient (partial derivatives):

$$\nabla \mathcal{L} = \left[ \frac{\partial \mathcal{L}}{\partial w_1}, \frac{\partial \mathcal{L}}{\partial w_2}, \ldots, \frac{\partial \mathcal{L}}{\partial b} \right]$$
Vector of partial derivatives

Update rule (gradient descent step):

$$\boldsymbol{\theta}^{t+1} = \boldsymbol{\theta}^t - \eta \nabla \mathcal{L}$$
Move in direction of steepest descent by learning rate η

For logistic regression specifically:

$$\frac{\partial \mathcal{L}_{\text{CE}}}{\partial w_j} = (\sigma(\mathbf{w} \cdot \mathbf{x} + b) - y) x_j = (\hat{y} - y) x_j$$ $$\frac{\partial \mathcal{L}_{\text{CE}}}{\partial b} = \hat{y} - y$$
Gradients for binary logistic regression

This elegant form shows the gradient is simply the prediction error scaled by the input feature. Larger errors → larger updates.

Learning rate η (step size): Controls how much we move each iteration. Common values: 0.01, 0.001, adaptive (Adam).

Variants:

  • Batch GD: Use entire training set each step (expensive but stable)
  • Stochastic GD (SGD): Use one example per step (fast but noisy)
  • Mini-batch GD: Use a small batch (32, 128, 256 examples) each step (best of both)

Convexity guarantee: For logistic regression, the loss function is convex, so any local minimum is the global minimum. Gradient descent will find the optimal solution.

5.6 Multinomial Logistic Regression & Softmax

For multiclass problems (e.g., sentiment with {very-negative, negative, neutral, positive, very-positive}), we generalize to K classes. The softmax function is the multiclass generalization of sigmoid:

$$\text{softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}$$
Softmax: normalized exponentials

We have a weight matrix W (shape K × f, where f is the number of features):

$$\mathbf{z} = \mathbf{W} \mathbf{x} + \mathbf{b} \quad \text{(K×f) × (f×1) = K×1}$$ $$\hat{\mathbf{y}} = \text{softmax}(\mathbf{z})$$
Multiclass prediction

Multinomial cross-entropy loss:

$$\mathcal{L}_{\text{CE}} = -\sum_{k=1}^{K} y_k \log \hat{y}_k = -\log \hat{y}_c$$
Only the correct class c matters (one-hot encoding)

The loss is simply −log of the predicted probability for the correct class. This penalizes confident mistakes.

5.7 Evaluation Metrics

Confusion matrix (binary classification):

Predicted → Pos Neg
Actual Pos TP FN
Actual Neg FP TN

From these four numbers, we compute:

$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$ $$\text{Precision} = \frac{TP}{TP + FP} \quad \text{(of predicted positives, how many correct?)}$$ $$\text{Recall} = \frac{TP}{TP + FN} \quad \text{(of actual positives, how many found?)}$$ $$F_1 = \frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}} \quad \text{(harmonic mean)}$$
Standard evaluation metrics

When to use which:

  • Accuracy: Balanced classes, generic performance
  • Precision: When false positives are costly (spam detection — we don't want to flag legitimate emails)
  • Recall: When false negatives are costly (disease detection — we don't want to miss cases)
  • F₁: When both matter equally, summarizes precision-recall tradeoff

Multiclass averaging:

  • Macro-average: Compute metric for each class, then average (treats all classes equally)
  • Micro-average: Aggregate TP/FP/FN/TN across classes, compute metric once (weights by class frequency)

Cross-validation: For reliable estimates, use k-fold cross-validation (typically k=10). Split data into 10 folds, train on 9 and test on 1, repeat 10 times, average results.

5.8 Regularization

Without regularization, the model can overfit by learning large weights that perfectly memorize the training data. We add a penalty term:

$$J(\mathbf{w}, b) = \frac{1}{m} \sum_{i=1}^{m} \mathcal{L}_{\text{CE}}(\hat{y}^{(i)}, y^{(i)}) + \alpha \cdot R(\mathbf{w})$$
Regularized objective with penalty α

L2 regularization (Ridge):

$$R(\mathbf{w}) = \sum_{j} w_j^2$$
Penalizes large weights; prefers small, distributed weights

L1 regularization (Lasso):

$$R(\mathbf{w}) = \sum_{j} |w_j|$$
Promotes sparsity; forces some weights to exactly zero (feature selection)

Elastic Net combines both: R(w) = λ₁ Σ |wⱼ| + λ₂ Σ wⱼ²

The regularization parameter α (or λ) controls the strength: larger α → more regularization → simpler model with smaller test loss but possibly higher training loss.

Code Implementation

Hands-On Code

7.1 Logistic Regression from Scratch

import numpy as np

class LogisticRegression:
    """Binary logistic regression with SGD training."""

    def __init__(self, n_features, lr=0.01):
        self.w = np.zeros(n_features)  # Weight vector, one per feature
        self.b = 0.0                    # Bias term (intercept)
        self.lr = lr                    # Learning rate η

    def sigmoid(self, z):
        """σ(z) = 1/(1+e^(-z)): squash any real number to (0,1)."""
        return 1 / (1 + np.exp(-np.clip(z, -500, 500)))  # clip for numerical stability

    def predict_proba(self, X):
        """Compute P(y=1|x) = σ(w·x + b) for each example."""
        z = X @ self.w + self.b   # Matrix multiply: [m×f] @ [f] = [m]
        return self.sigmoid(z)     # Apply sigmoid element-wise

    def predict(self, X):
        """Hard classification: 1 if P(y=1) > 0.5, else 0."""
        return (self.predict_proba(X) >= 0.5).astype(int)

    def cross_entropy_loss(self, y_true, y_pred):
        """L_CE = -[y log(ŷ) + (1-y) log(1-ŷ)], averaged over batch."""
        eps = 1e-15  # Small constant to avoid log(0)
        y_pred = np.clip(y_pred, eps, 1 - eps)
        return -np.mean(y_true * np.log(y_pred) + (1-y_true) * np.log(1-y_pred))

    def fit(self, X, y, epochs=100, batch_size=32):
        """Train using mini-batch stochastic gradient descent."""
        m = X.shape[0]  # Number of training examples
        losses = []
        for epoch in range(epochs):
            # Shuffle data each epoch for stochasticity
            indices = np.random.permutation(m)
            X_shuffled, y_shuffled = X[indices], y[indices]

            for i in range(0, m, batch_size):
                X_batch = X_shuffled[i:i+batch_size]
                y_batch = y_shuffled[i:i+batch_size]

                # Forward pass: compute predictions
                y_hat = self.predict_proba(X_batch)

                # Compute gradients: ∂L/∂w = (1/m)(ŷ-y)ᵀX, ∂L/∂b = mean(ŷ-y)
                error = y_hat - y_batch
                grad_w = X_batch.T @ error / len(y_batch)
                grad_b = np.mean(error)

                # Update: θ ← θ - η∇L
                self.w -= self.lr * grad_w
                self.b -= self.lr * grad_b

            # Track loss each epoch
            loss = self.cross_entropy_loss(y, self.predict_proba(X))
            losses.append(loss)
        return losses

    def evaluate(self, X, y):
        """Compute precision, recall, F1."""
        preds = self.predict(X)
        tp = np.sum((preds == 1) & (y == 1))
        fp = np.sum((preds == 1) & (y == 0))
        fn = np.sum((preds == 0) & (y == 1))
        precision = tp / (tp + fp) if (tp + fp) > 0 else 0
        recall = tp / (tp + fn) if (tp + fn) > 0 else 0
        f1 = 2*precision*recall/(precision+recall) if (precision+recall) > 0 else 0
        return {'precision': precision, 'recall': recall, 'f1': f1}

7.2 Using Scikit-learn

from sklearn.linear_model import LogisticRegression
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics import precision_recall_fscore_support
from sklearn.model_selection import cross_val_score

# Load movie reviews (IMDB dataset)
X_train, y_train = load_imdb_data(split='train')
X_test, y_test = load_imdb_data(split='test')

# Convert text to TF-IDF vectors (extract features)
vectorizer = TfidfVectorizer(max_features=5000, ngram_range=(1, 2))
X_train_vec = vectorizer.fit_transform(X_train)
X_test_vec = vectorizer.transform(X_test)

# Train logistic regression
model = LogisticRegression(C=1.0, solver='lbfgs', max_iter=1000)
model.fit(X_train_vec, y_train)

# Evaluate
y_pred = model.predict(X_test_vec)
precision, recall, f1, support = precision_recall_fscore_support(y_test, y_pred, average='binary')
print(f"Precision: {precision:.3f}, Recall: {recall:.3f}, F1: {f1:.3f}")

# 10-fold cross-validation
cv_scores = cross_val_score(model, X_train_vec, y_train, cv=10, scoring='f1')
print(f"Cross-validation F1: {cv_scores.mean():.3f} +/- {cv_scores.std():.3f}")

From Linear to Deep: The Modern Bridge

Modern LLMs

How Logistic Regression Connects to Modern NLP

Logistic regression is literally a single-layer neural network. In fact, it's a neuron with:

  • Input layer: feature vector x
  • Weight matrix: w (shape 1 × f)
  • Bias: b
  • Activation function: sigmoid
  • Output layer: probability

Stack two logistic regressions (hidden layer → output layer), add ReLU activations, and you've got a neural network. This is Chapter 6.

Softmax in Transformers: When you use GPT, BERT, or Claude, the final layer is a softmax over vocabulary. Those 50,257 output logits for GPT are exactly a multinomial logistic regression on top of the transformer's hidden representation. The "next token prediction" loss is cross-entropy loss applied to softmax outputs. Same mathematics, scaled to billions of parameters.

Gradient descent → Adam optimizer: In practice, modern deep learning doesn't use vanilla gradient descent; it uses adaptive optimizers like Adam (Adaptive Moment Estimation) that maintain momentum and per-parameter learning rates. But the core principle is the same: compute gradients, update parameters in the direction of steepest descent.

Feature engineering → Representation learning: In Chapter 4, we manually design features (word counts, TF-IDF, etc.). Modern LLMs learn representations automatically through unsupervised pretraining on huge text corpora. The hidden states of these models become rich embeddings that capture semantic and syntactic information. Then we can train a logistic regression on top of these frozen embeddings — this is called a "linear probe" and it's a standard technique to evaluate what information the model has learned.

Regularization: L2/L1 regularization prevent overfitting in logistic regression. In deep learning, we use dropout (randomly zero out activations during training), weight decay (equivalent to L2), and early stopping (stop training when validation loss increases).

Evaluation beyond metrics: For LLMs, precision/recall/F₁ are still used, but they're augmented with human evaluation (does the model output make sense?), benchmark scores (performance on standardized tasks), and qualitative analysis (cherry-picked examples). The reason: human language is subtle, and automatic metrics miss nuance.

Historical Context

Timeline
Pierre-François Verhulst (1838): The Belgian mathematician introduced the logistic function to model population growth. It naturally captures the S-curve behavior: slow growth at first, rapid growth in the middle, slowdown as resources limit.
Joseph Berkson (1944): The American statistician coined the term "logit" (from "logistic unit") and championed logistic regression as the statistical method for binary outcomes, especially in biomedical research. The probit function was the old standard; logit became dominant because it's simpler and matches biological probability.
David Cox (1958): Formalized logistic regression as a formal statistical model and derived its properties (maximum likelihood estimation, asymptotic distributions). This made it the standard tool in statistics and epidemiology.
Frank Rosenblatt (1958): Invented the Perceptron, a linear classifier that outputs 0 or 1 (no probabilities, no sigmoid). It's the spiritual cousin of logistic regression; in fact, the perceptron convergence theorem guarantees that if data is linearly separable, the perceptron will find a separating hyperplane. But logistic regression is more principled: it outputs calibrated probabilities, not hard decisions.
Rumelhart, Hinton & Williams (1986): Pioneered backpropagation, the algorithm for computing gradients in multi-layer neural networks. They showed you could apply gradient descent to networks with hidden layers, opening the door to deep learning. Logistic regression's gradient descent was the prototype.
Pang, Lee & Vaithyanathan (2002): Published a landmark paper on sentiment analysis using machine learning classifiers (Naïve Bayes, SVMs, logistic regression) on movie reviews. This sparked the modern era of NLP text classification, and logistic regression became the go-to baseline for 20+ years.
Collobert & Weston (2008) onward: Neural networks re-emerged for NLP with embedding-based methods. But even with deep neural networks, a final logistic regression layer (softmax) is standard. The shift was in the representations, not the classifier.
Vaswani et al. (2017): Introduced the Transformer architecture, which now dominates NLP. Yet the final layer is still softmax (multinomial logistic regression) over the vocabulary. Logistic regression was not replaced; it's embedded in every LLM.

Failure Scenario: The Cooking Forum Toxicity Detector

A toxicity detection classifier trained on social media data is deployed to moderate comments on a cooking forum. The classifier flags the comment "This recipe is killer! Absolutely to die for!" as toxic with 87% confidence because the words "killer" and "die" have large positive weights toward the toxic class. Meanwhile, genuinely harmful comments using coded language (like "go touch grass" as dismissive harassment) pass through undetected.

Question 1: Why did the classifier misinterpret "killer" and "to die for" in this context? What does this reveal about the bag-of-words feature representation?

The model learned that "killer" and "die" are strong toxicity signals in social media (where these words often appear in genuine insults like "I want to kill you" or "go die"). Bag-of-words treats each word independently and ignores context — it counts word frequency but throws away word order and semantic relationships. So "to die for" (idiomatic expression meaning "excellent") is parsed as separate words: "to" (neutral), "die" (toxic signal), "for" (neutral). The compositional meaning is lost. This highlights a fundamental weakness of bag-of-words for language with metaphor, idiom, and pragmatic meaning.

Question 2: If we add a feature for bigrams (pairs of consecutive words), would that fix the "to die for" problem? What are the tradeoffs?

Adding bigrams helps. Now "to_die_for" is a single feature that the model can learn is not toxic. But tradeoffs emerge: (1) Feature explosion: Bigrams increase vocabulary size roughly quadratically. If unigrams give 10,000 features, bigrams might give 100,000+. (2) Data sparsity: Each bigram appears less frequently, so the model needs more training data to learn reliable weights. (3) No true context: Bigrams help for short phrases but still fail on longer-range dependencies. "This recipe is absolutely killer!" might have "is_absolutely" and "absolutely_killer" but the classifier can't connect "killer" back through multiple words to the positive context. True contextualization requires understanding word order and semantic relationships across variable-length spans — which is what neural networks with attention (transformers) excel at.

Question 3: The classifier has high recall for toxic content but low precision. What does this mean practically for forum users? Which metric should we prioritize?

High recall, low precision = many false positives. The classifier catches most actual toxic comments (good!), but it also flags many innocent comments like "killer recipe" as toxic (bad!). Practically: users see their reviews removed or hidden even though they're harmless. This creates frustration and trust loss. Which metric to prioritize? It depends on costs. In moderation, false positives (flagging good content) often feel worse to users than false negatives (missing some toxicity), so we might prioritize precision — only flag comments we're very confident about. Conversely, for spam detection or malware filtering where missing one bad item could cause major damage, we prioritize recall. For this cooking forum, precision seems more important to avoid frustrating users.

Question 4: Suppose we have weights: w_killer = 3.2, w_die = 2.8, and all other features sum to −1.5 with b = −2.0. If the comment "This recipe is killer!" has features [0, 0, 1, 0, 0, 3.0] for those six feature dims, what is σ(z)? Does the model's confidence match reality?

We have z = w·x + b = (3.2)(0) + (2.8)(0) + (1)(1)? No wait, let me re-read: the "other features sum to −1.5" probably means those first 4 features don't apply. Let's say features are [pos_words, neg_words, has_killer, other_signal_1, ..., log_length]. The comment "This recipe is killer!" might have has_killer=1. Then z = 3.2(0) + 2.8(0) + ... + w_killer(1) + [other features] = 3.2 + (−1.5) + (−2.0) = −0.3. So σ(−0.3) ≈ 0.43, meaning 43% confident the comment is toxic. That's actually not super confident, closer to uncertain. But wait, if w_killer=3.2 (positive weight), then having "killer" should push toward positive class. Let me reconsider: if the model learned killer=toxic, then the weight should be positive. z = 3.2(1) + (−1.5) + (−2.0) = −0.3, giving 43% toxic. Hmm. Actually, σ(−0.3) = 1/(1+e^{0.3}) ≈ 0.426. So the model outputs 43% confidence, which seems low. But the problem statement said the classifier flags it with 87% confidence. Let me recompute: if σ(z)=0.87, then z ≈ 1.95. So the weights and features must be different. The key insight: the model's confidence doesn't match reality because it's based on token-level signals that ignore pragmatic context.

Question 5: How would a modern LLM-based classifier handle this differently? What fundamental advantage does contextual understanding provide?

An LLM-based classifier (e.g., fine-tuned BERT or a prompt-based GPT system) operates on learned representations that encode context, syntax, and semantics. The model doesn't just count "killer" and "die"; it understands that "to die for" is an idiom, that "killer recipe" is praise, and that "this" and "is" modify and contextualize the adjective. Through multi-head attention, the model can attend to relevant context words when deciding on the overall sentiment. The learned embeddings capture nuanced relationships — "killer" in a positive context has a different representation than "killer" in "I want to kill you." For the coded language ("go touch grass"), an LLM has seen enough diverse examples that it can learn this emerging slang means harassment, even if the individual words seem innocuous. Fundamental advantage: Logistic regression on bag-of-words is fundamentally limited to linear decisions in a fixed feature space. LLMs use deep representations and non-linear transformations (attention, multiple layers) to capture the complexity of human language. This doesn't make them immune to errors — they have other failure modes — but they handle context and compositionality far better.
