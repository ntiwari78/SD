Chapter 8: Transformers

"The true art of memory is the art of attention."
— Samuel Johnson, Idler #74, September 1759
The Transformer architecture stands as the foundation of modern natural language processing. Introduced by Vaswani et al. in 2017, it replaces recurrence with pure attention mechanisms, enabling unprecedented parallelization, interpretability, and scaling. This chapter explores the mathematics and mechanics of self-attention, how transformers process and generate sequences, and the design principles that have made them the dominant architecture in deep learning.

Chapter Progress

Transformers Without Jargon

Feynman

Imagine a round table where each person represents a token in your sequence. Everyone is trying to understand what every other person is saying, and to respond appropriately, each person needs to figure out: "Who should I listen to right now?"

The Three Name Tags

Each person wears three special name tags that they can adjust based on what they're thinking:

  • Query (Q): "Here's what I'm looking for" — what information does this person need?
  • Key (K): "Here's what I can offer" — what's special about this person's information?
  • Value (V): "Here's the actual data I'm sharing" — the concrete information they provide

Self-Attention: The Conversation

When person i looks around the table, they glance at everyone's Key tag and compare it with their own Query: "How well do your offering match what I'm looking for?" They compute a match score with each person—some will match very well (high score), others poorly (low score).

These scores are turned into percentages (softmax) so they sum to 100%. Person i then gathers information from everyone using these percentages as weights. If person j has a 40% match, they get 40% of person j's Value. This weighted sum is the output for person i.

Multiple Conversations

But here's where it gets clever: Multi-head attention means multiple independent conversations are happening simultaneously at the same table. One conversation might focus on syntax ("Who's the subject here?"), another on semantics ("What's the main topic?"), another on position ("How far away are you?"). Each conversation has its own Q, K, V and contributes its own perspective.

The Residual Stream

Imagine there's a shared notebook being passed around the table. After the attention conversation, each person writes their findings into this notebook, adding it to what was already there. This residual stream ensures that information doesn't get lost—it flows through and accumulates.

The Stacking

One round of attention isn't enough. So you do this multiple times (multiple layers): each layer's output becomes the input to the next round of conversations. In the first round, people might notice syntax. In the second round, they refine their understanding based on syntax patterns. By the final round, each position has seen the whole sequence multiple times and has a rich, contextual representation.

Intuition: A transformer is a stack of collaborative filtering layers. Each layer asks: "Given what I know about the whole sequence, how should I refine my understanding of this position?" Repeat many times with different "conversation lenses," and you get a deep, nuanced representation of the input.

The Mathematics of Self-Attention

Technical

8.1 Self-Attention Foundation

Simplified Version

Let's start with the simplest idea. Each token x_i should be updated by looking at all other tokens. A weighted average:

$$a_i = \sum_{j} \alpha_{ij} x_j$$

The weights α_{ij} are computed using a score function. The most natural choice is the dot product of the tokens:

$$\text{score}(x_i, x_j) = x_i \cdot x_j$$

Convert scores to probabilities (weights that sum to 1) using softmax:

$$\alpha_{ij} = \text{softmax}(\text{score}(x_i, x_j)) = \frac{\exp(\text{score}(x_i, x_j))}{\sum_k \exp(\text{score}(x_i, x_k))}$$

Query, Key, Value

Raw token embeddings do double duty (both look for matches and provide content), which is limiting. Instead, we project each token into three spaces using learned weight matrices (Equations 8.9–8.10):

$$q_i = x_i W^Q, \quad k_j = x_j W^K, \quad v_j = x_j W^V$$

Now we compute match scores using Q and K:

$$\text{score}(q_i, k_j) = q_i \cdot k_j$$

Scaled Dot-Product Attention

When embeddings get large (high dimension d_k), dot products get very large, and softmax gradients become tiny. Scaling by 1/\sqrt{d_k} helps (Equation 8.11):

$$\text{score}(q_i, k_j) = \frac{q_i \cdot k_j}{\sqrt{d_k}}$$

Attention Weights

Apply softmax to get attention distribution (Equation 8.12):

$$\alpha_{ij} = \text{softmax}\left(\frac{q_i \cdot k_j}{\sqrt{d_k}}\right) = \frac{\exp(q_i \cdot k_j / \sqrt{d_k})}{\sum_{j'} \exp(q_i \cdot k_j' / \sqrt{d_k})}$$

Attention Head Output

Gather values using attention weights (Equation 8.13):

$$\text{head}_i = \sum_j \alpha_{ij} v_j$$

Project the concatenated head back to the original dimension (Equation 8.14):

$$a_i = \text{head}_i W^O$$

Multi-Head Attention

One head is limited. Instead, use multiple heads in parallel (Equations 8.15–8.20). Each head c has independent Q, K, V projections:

$$q_i^c = x_i W_c^Q, \quad k_j^c = x_j W_c^K, \quad v_j^c = x_j W_c^V$$

Compute each head's output:

$$\text{head}_i^c = \text{Attention}(q_i^c, k_j^c, v_j^c)$$

Concatenate and project:

$$a_i = \text{Concat}(\text{head}_i^1, \ldots, \text{head}_i^h) W^O$$

8.2 Transformer Blocks

Feed-Forward Network

After attention, apply a simple two-layer feed-forward MLP (Equation 8.21):

$$\text{FFN}(x) = \text{ReLU}(x W_1 + b_1) W_2 + b_2$$

This applies token-by-token and allows nonlinearity.

Layer Normalization

Normalize activations to zero mean and unit variance (Equations 8.22–8.25):

$$\mu = \frac{1}{d}\sum_{i=1}^{d} x_i, \quad \sigma = \sqrt{\frac{1}{d}\sum_{i=1}^{d}(x_i - \mu)^2 + \epsilon}$$
$$\text{LayerNorm}(x) = \gamma \odot \frac{x - \mu}{\sigma} + \beta$$

Complete Transformer Block (Pre-Norm)

Modern transformers use pre-normalization (applying layer norm before the sublayer, Equations 8.26–8.31):

$$t^1 = \text{LayerNorm}(x_i)$$ $$t^2 = \text{MultiHeadAttention}(t^1)$$ $$t^3 = t^2 + x_i \quad \text{(residual connection)}$$ $$t^4 = \text{LayerNorm}(t^3)$$ $$t^5 = \text{FFN}(t^4)$$ $$h_i = t^5 + t^3 \quad \text{(residual connection)}$$

The residual connections preserve information flow through deep networks. Layer norm stabilizes training.

8.3 Matrix Computation

To handle sequences efficiently, we stack all token embeddings into a matrix X \in \mathbb{R}^{N \times d} and compute attention for the entire sequence (Equation 8.32):

$$Q = XW^Q, \quad K = XW^K, \quad V = XW^V$$

Scaled dot-product for all pairs (Equation 8.33):

$$\text{Attention} = \text{softmax}(\text{mask}(QK^T / \sqrt{d_k})) V$$

For autoregressive generation, apply a causal mask that prevents attending to future tokens (upper triangle set to -\infty, Equation 8.34):

$$\text{mask}(S)_{ij} = \begin{cases} S_{ij} & \text{if } j \leq i \\ -\infty & \text{if } j > i \end{cases}$$

8.4 Input Embeddings

Tokens are converted to embeddings using a learned embedding matrix E \in \mathbb{R}^{|V| \times d}:

$$x_i = E[\text{id}(i)]$$

Add positional information (absolute or relative). Classic sinusoidal positional encoding:

$$PE(pos, 2j) = \sin(pos / 10000^{2j/d}), \quad PE(pos, 2j+1) = \cos(pos / 10000^{2j/d})$$

Modern variants use rotary embeddings (RoPE) or relative biases in attention scores.

8.5 Language Modeling Head

After L layers, the final representation h_N^L for the last token is projected back to vocabulary space (Equations 8.46–8.47):

$$u = h_N^L E^T$$ $$y = \text{softmax}(u)$$

Weight tying (sharing the embedding matrix) reduces parameters and improves performance.

8.6 Sampling Strategies

During generation, we don't always take argmax. Common strategies:

Top-k Sampling: Only sample from the k most likely tokens (Equation 8.48):

$$y_t \sim \text{TopK}(p, k) = \text{Renormalize}(\{p_i : i \in \text{top-k}\})$$

Top-p (Nucleus) Sampling: Sample from the smallest set of tokens whose cumulative probability is at least p (usually 0.9).

8.7 Training

Transformers are trained with teacher forcing: feed ground-truth tokens as input while computing loss against ground-truth outputs. Cross-entropy loss:

$$\mathcal{L} = -\sum_i \log p(y_i | y_{

Sequences can be processed in parallel during training (all positions computed simultaneously), unlike RNNs. This enables scaling to massive datasets.

8.8 Scaling and Efficiency

Scaling Laws: Language model loss scales predictably with parameters (Equation 8.49):

$$L(N) = \left(\frac{N_c}{N}\right)^{\alpha_N}$$

Parameter count approximation (Equation 8.52):

$$N \approx 12 \cdot n_{\text{layer}} \cdot d^2$$

KV Cache: During generation, K and V don't change—reuse them! Store cache in memory to avoid recomputation.

LoRA (Low-Rank Adaptation): Fine-tune efficiently by updating low-rank adapters instead of full weights (Equation 8.54):

$$h = xW + xAB$$

where A \in \mathbb{R}^{d \times r}, B \in \mathbb{R}^{r \times d} with r \ll d.

8.9 Interpretability

In-Context Learning: Transformers learn to use context within a single forward pass—no gradient updates needed to adapt to new tasks.

Induction Heads: Some attention heads specialize in pattern completion: if they see A...B...A, they predict B. This is a learned algorithm for in-context learning.

Logit Lens: Decode hidden representations at each layer to see what predictions the model makes at different depths—earlier layers often have noisier, generic predictions; later layers refine them.

Implementation from Scratch

Code

Scaled Dot-Product Attention

import numpy as np

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Compute scaled dot-product attention.

    Args:
        Q: Query matrix of shape (seq_len, d_k)
        K: Key matrix of shape (seq_len, d_k)
        V: Value matrix of shape (seq_len, d_v)
        mask: Optional causal mask (upper triangle = -inf)

    Returns:
        output: Attention-weighted values (seq_len, d_v)
        weights: Attention weights (seq_len, seq_len)
    """
    d_k = Q.shape[-1]

    # Compute similarity scores: Q @ K^T
    scores = np.matmul(Q, K.T) / np.sqrt(d_k)

    # Apply causal mask if provided (for autoregressive models)
    if mask is not None:
        scores = np.where(mask, scores, -1e9)

    # Apply softmax to get attention weights
    weights = softmax(scores, axis=-1)

    # Gather values using attention weights
    output = np.matmul(weights, V)

    return output, weights

def softmax(x, axis=-1):
    """Numerically stable softmax."""
    exp_x = np.exp(x - np.max(x, axis=axis, keepdims=True))
    return exp_x / np.sum(exp_x, axis=axis, keepdims=True)

Multi-Head Attention

class MultiHeadAttention:
    def __init__(self, d_model, num_heads):
        assert d_model % num_heads == 0, "d_model must be divisible by num_heads"

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        # Weight matrices for each head
        self.W_q = np.random.randn(d_model, d_model) * np.sqrt(2.0 / d_model)
        self.W_k = np.random.randn(d_model, d_model) * np.sqrt(2.0 / d_model)
        self.W_v = np.random.randn(d_model, d_model) * np.sqrt(2.0 / d_model)
        self.W_o = np.random.randn(d_model, d_model) * np.sqrt(2.0 / d_model)

    def forward(self, Q, K, V, mask=None):
        # Linear projections
        Q_proj = np.matmul(Q, self.W_q)
        K_proj = np.matmul(K, self.W_k)
        V_proj = np.matmul(V, self.W_v)

        # Split into multiple heads
        Q_heads = self.split_heads(Q_proj)  # (num_heads, seq_len, d_k)
        K_heads = self.split_heads(K_proj)
        V_heads = self.split_heads(V_proj)

        # Apply attention for each head
        outputs = []
        for i in range(self.num_heads):
            out, _ = scaled_dot_product_attention(
                Q_heads[i], K_heads[i], V_heads[i], mask
            )
            outputs.append(out)

        # Concatenate heads and apply output projection
        concat = np.concatenate(outputs, axis=-1)  # (seq_len, d_model)
        output = np.matmul(concat, self.W_o)

        return output

    def split_heads(self, x):
        # Reshape (seq_len, d_model) -> (seq_len, num_heads, d_k) -> (num_heads, seq_len, d_k)
        seq_len = x.shape[0]
        x = x.reshape(seq_len, self.num_heads, self.d_k)
        return np.transpose(x, (1, 0, 2))

Layer Normalization

class LayerNorm:
    def __init__(self, d_model, eps=1e-6):
        self.gamma = np.ones(d_model)  # learnable scale
        self.beta = np.zeros(d_model)   # learnable shift
        self.eps = eps

    def forward(self, x):
        # x shape: (seq_len, d_model)

        # Compute mean and variance
        mean = np.mean(x, axis=-1, keepdims=True)
        var = np.var(x, axis=-1, keepdims=True)

        # Normalize
        x_norm = (x - mean) / np.sqrt(var + self.eps)

        # Scale and shift
        output = self.gamma * x_norm + self.beta

        return output

Transformer Block (Pre-Norm)

class TransformerBlock:
    def __init__(self, d_model, num_heads, d_ffn, dropout=0.1):
        self.attention = MultiHeadAttention(d_model, num_heads)
        self.norm1 = LayerNorm(d_model)
        self.norm2 = LayerNorm(d_model)

        # Feed-forward network: d_model -> d_ffn -> d_model
        self.W1 = np.random.randn(d_model, d_ffn) * np.sqrt(2.0 / d_model)
        self.W2 = np.random.randn(d_ffn, d_model) * np.sqrt(2.0 / d_ffn)
        self.b1 = np.zeros(d_ffn)
        self.b2 = np.zeros(d_model)

    def forward(self, x, mask=None):
        # Pre-norm attention block
        x_norm = self.norm1.forward(x)
        attn_out = self.attention.forward(x_norm, x_norm, x_norm, mask)
        x = x + attn_out  # residual connection

        # Pre-norm feed-forward block
        x_norm = self.norm2.forward(x)
        ffn_out = np.maximum(0, np.matmul(x_norm, self.W1) + self.b1)  # ReLU
        ffn_out = np.matmul(ffn_out, self.W2) + self.b2
        x = x + ffn_out  # residual connection

        return x

Sinusoidal Positional Encoding

def positional_encoding(seq_len, d_model):
    """
    Create sinusoidal positional encodings.
    pos: position in sequence (0 to seq_len-1)
    i: dimension index (0 to d_model-1)
    PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
    PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
    """
    pe = np.zeros((seq_len, d_model))
    position = np.arange(seq_len).reshape(-1, 1)

    # Compute the denominator: 10000^(2i/d_model)
    div_term = np.exp(np.arange(0, d_model, 2) * -(np.log(10000.0) / d_model))

    # Apply sine to even indices (0, 2, 4, ...)
    pe[:, 0::2] = np.sin(position * div_term)

    # Apply cosine to odd indices (1, 3, 5, ...)
    pe[:, 1::2] = np.cos(position * div_term)

    return pe

Causal Mask for Autoregressive Models

def create_causal_mask(seq_len):
    """
    Create a causal mask where position i can only attend to positions j <= i.
    Returns a boolean mask: True where attention is allowed, False where it's masked.
    """
    mask = np.tril(np.ones((seq_len, seq_len)), k=0)).astype(bool)
    return mask

Transformers in 2026: Modern Developments

Modern

Architecture Variants

The transformer blueprint has spawned diverse lineages:

  • GPT Family (OpenAI): Decoder-only, autoregressive. GPT-4 uses 1.7T parameters with mixture-of-experts. Excels at generation and few-shot learning.
  • Llama (Meta): Efficient decoder-only models (7B, 13B, 70B). Uses RoPE positional embeddings, SwiGLU activations, and group-query attention. Open-source standard for research.
  • Mistral (Mistral AI): Focuses on efficient inference. 7B model achieves GPT-3.5 performance. Sliding window attention reduces compute.
  • Gemma (Google): Efficient models trained on high-quality data. Emphasizes training stability and safety through feedback tuning.
  • BERT Family (Google): Encoder-only, bidirectional. Pre-training objective is masked language modeling (predict randomly masked tokens).
  • T5 (Google): Encoder-decoder, treats all tasks as sequence-to-sequence. Unified framework for translation, summarization, QA, etc.

Efficiency Innovations

Standard transformers scale as O(N²) in sequence length. Modern variants optimize this:

  • FlashAttention: IO-aware attention kernel written in CUDA. Reduces memory accesses by treating attention as a blocked matrix multiply. 2-4x speedup in practice.
  • Group-Query Attention (GQA): Share K and V across multiple query head groups instead of separate K, V per head. Reduces KV cache size by ~8x for 8 heads sharing K,V.
  • Rotary Positional Embeddings (RoPE): Rotate query and key vectors by position-dependent angles. Naturally encodes relative distance, works well for extrapolation to longer contexts.
  • Multi-Query Attention (MQA): Extreme case: single K, V shared by all query heads. Fast generation but slight quality loss.

Long Context

Original transformers (2017) worked with 512 tokens. Today:

  • Context Windows: Llama 2: 4K. Llama 3: 8K. GPT-4: 128K. Claude 3.5: 200K. Gemini: 1M (effective).
  • Ring Attention: Distribute attention computation across multiple GPUs by splitting sequence. Enables 100K+ tokens on TPU/GPU clusters.
  • Sparse Attention Patterns: Local + strided patterns (attend to nearby tokens + every kth token). Reduces O(N²) to O(N log N).
  • Approximations: Linformer, Performer use kernel tricks to approximate softmax attention in linear time.

Mixture of Experts (MoE)

Instead of applying all parameters to every token, use a router that selects a sparse subset of "expert" feed-forward networks:

  • Structure: Token-specific router learns which experts to activate. Usually 2-8 experts active per token, hundreds total.
  • Scaling: Can increase parameter count dramatically (from 70B to 141B) while keeping computation similar.
  • Examples: GShard (Google), Switch Transformers, GPT-4 (likely uses MoE internally).
  • Challenge: Load balancing (ensuring tokens are routed roughly equally) and training instability.

Multimodal Transformers

Process images, audio, video, and text in a unified architecture:

  • Vision Transformers (ViT): Divide image into patches, embed patches like tokens, apply standard transformer. Matches/exceeds CNNs for image classification.
  • Cross-Modal Attention: Text transformer can attend to image patches. Vision transformer can attend to text embeddings. Bridges modalities.
  • Unified Embedding Space: Train image and text embeddings to align (CLIP). Can then use text-image similarity for zero-shot classification.
  • Examples: DALL-E (text→image), GPT-4V (image+text→text), Gemini (any modal combination).

The Residual Stream as Information Highway

Modern interpretability research views the residual stream (the vector flowing through and accumulated by all layers) as the "information highway":

  • Superposition: Multiple pieces of information are encoded simultaneously in a single vector.
  • Steering: Intervening on residual stream activations can control model behavior (e.g., make it more/less sarcastic).
  • Composition: Each attention/FFN layer adds information to the stream. The final output is a sum of all these contributions.
  • Path Tracing: Analyzing how information flows from input tokens to output logits through the network—which connections matter most?

Historical Development

History

The transformer didn't appear from nowhere. It emerged from years of work on attention mechanisms and sequence modeling.

2013
Graves introduces attention for handwriting generation. First learned mechanism to focus on relevant parts of input.
2015 (Early)
Bahdanau, Cho, Bengio propose attention for neural machine translation. Encoder-decoder with attention enables models to align words across languages.
2015 (Mid)
Sukhbaatar et al., Weston et al. develop Memory Networks. Learnable attention over external memory—precursor to self-attention.
2015-2016
Ling, Cheng, Liu, and others explore self-attention ideas. Ling & Dyer use attention within sequence. Cheng et al. (2016) use self-attention for relation extraction.
2017
Vaswani et al. publish "Attention Is All You Need" at NeurIPS. Introduces the Transformer: first architecture without recurrence or convolution, purely attention-based. Parallel training enabled. Scaling new heights.
2018
OpenAI (Radford et al.): GPT-1 (117M parameters). Shows transformer can learn language understanding via next-token prediction. BERT (Devlin et al.): Bidirectional transformer encoder. Masked LM objective outperforms. Two trees from one seed.
2019
OpenAI: GPT-2 (1.5B). Demonstrates zero-shot task adaptation. Concerns about misuse. T5 (Raffel et al.): Unified text-to-text transformer. All NLP as seq2seq.
2020
OpenAI: GPT-3 (175B). Few-shot in-context learning breakthrough. Kaplan et al.: Scaling laws characterize loss as function of model/data size. Neural scaling laws are smooth and predictable.
2021-2022
Mechanistic Interpretability: Elhage et al. (Anthropic) and others begin reverse-engineering transformer circuits. Olsson et al. (2022): Identify "induction heads"—attention heads that implement the algorithm: find repeated tokens, predict what follows. Interpretability becomes empirical.
2023
Efficiency Revolution: LoRA (Hu et al.): Fine-tune large models by updating low-rank adapters. QLoRA: Quantize then adapt. FlashAttention2. Boom in open-source models: Llama, Llama 2, Mistral. Practical fine-tuning becomes accessible.
2024-2026
Multimodal & Frontier: Vision transformers (ViT) now dominant in vision. Multimodal LLMs (GPT-4V, Gemini, Claude 3.5). Long contexts (100K+). MoE scaling (Mixture of Experts). New architectures explored (Mamba, state-space models) but transformers still reign. Interpretability tools mature. Constitutional AI and RLHF refine behavior.

Socratic Knowledge Check

⚡ The Scaling Challenge

You're implementing a transformer-based language model for a mobile app. Your model has 12 layers, d=512, 8 attention heads, and a vocabulary of 32K tokens. During inference, generation slows dramatically when producing sequences longer than 200 tokens, and the app runs out of memory at around 500 tokens. Your team needs the model to handle 2000-token contexts while maintaining <50ms latency per token on a mid-range phone (4GB RAM).

Below are 5 diagnostic questions. Click "Show Answer" to reveal the reasoning.

1. Why does generation get slower as the sequence gets longer?
Without a KV cache, at each new token you must recompute attention over ALL previous tokens. This is O(N²) in sequence length—at N=200, it's 40k attention operations; at N=500, it's 250k. With a KV cache, you avoid recomputation but the cache itself grows in size (linear in N). Additionally, the attention operation is inherently quadratic: computing Q @ K^T for a sequence of length N produces an N×N matrix, even with caching.
2. What is the memory footprint of the KV cache for your model?
Each layer stores K and V matrices. Size = 2 × n_layers × N × d_k × n_heads = 2 × 12 × N × (512/8) × 8 = 2 × 12 × N × 512 = 12,288N floats. Using float32 (4 bytes/float), that's ~49KB per token. At N=500 tokens, the cache is ~24.5MB. At N=2000, it's ~98MB—not fitting in 4GB available RAM when the model itself (12×12×d²) takes ~300MB+.
3. What are three concrete strategies to reduce memory requirements?
Strategy A - Grouped-Query Attention (GQA): Share K and V across head groups. With 8 heads split into 2 groups, cache shrinks by 4x. Now the model fits comfortably.

Strategy B - Quantization: Store KV cache in INT8 or FP16 instead of FP32. FP16 cuts cache size in half with minimal quality loss. INT8 is even smaller but requires calibration.

Strategy C - Sliding Window Attention: Only cache K and V for the last W tokens (e.g., W=512). Distant tokens don't contribute much to attention anyway (attention is sparse). Keeps cache bounded.
4. LoRA is mentioned as an efficiency trick. How does it help with your deployment?
LoRA reduces fine-tuning parameters but doesn't directly reduce the inference model size. However, here's where it helps: (1) You can ship a quantized base model (smaller), (2) Ship task-specific LoRA adapters separately (tiny, ~1-2MB each), (3) Combine them at runtime. This lets you have a small base model + lightweight task-specific tuning. For the memory bottleneck, LoRA is indirect relief—the main gains come from quantization + GQA.
5. The team suggests using a state-space model (like Mamba) instead. What would the fundamental advantage be?
State-space models (SSMs) like Mamba replace the quadratic attention with a recurrent state update. Memory doesn't grow with sequence length—instead, you maintain a fixed-size hidden state and update it as new tokens arrive. This gives O(1) memory and O(1) computation per token (compared to transformers' O(N) memory and O(N) computation during generation). For 2000-token sequences, this is a game-changer. The tradeoff: SSMs are less flexible for parallel training (more recurrent), and the research is newer. But for mobile deployment with long sequences, SSMs are promising.
