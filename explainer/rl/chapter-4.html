🎯 The Feynman Bridge

Dynamic Programming Is Like Planning a Road Trip

Imagine you're planning a road trip from New York to Los Angeles. You have a perfect map. You know every route, every toll booth cost, every scenic view along the way. You can sit at home and work backwards from LA to figure out the optimal route from anywhere.

This is Dynamic Programming (DP). You don't need to actually drive the routes to learn them — you know the model (the map). You can plan ahead using this model, computing the value of being in each location and the best action to take from each location.

The Core Insight

You have a map = you have the full model of the environment (transition probabilities \(p(s'|s,a)\) and reward function \(r(s,a)\)). You compute backwards = you use the Bellman equation to work backwards from the goal. You plan without driving = you find optimal policies before actually taking actions in the environment.

The catch? You need the complete map. In Chapters 5–6, we'll drop this assumption and learn from experience alone.

Dynamic Programming breaks the optimal control problem into two interacting processes:

1. Policy Evaluation (Prediction): Given a policy \(\pi\), compute its value function \(V^\pi(s)\) — how good is it to be in state \(s\) and then follow policy \(\pi\) forever?

2. Policy Improvement (Control): Given the value function, find a better policy by greedily selecting actions that lead to higher-valued states.

Repeat these two steps, and you converge to the optimal policy. This is the dance of Generalized Policy Iteration (GPI) — evaluation and improvement spiraling around each other toward optimality.

📐 The Technical Ascent

Policy Evaluation, Improvement, and Iteration

At the heart of DP are two equations working in tandem: the Bellman expectation equation (for evaluation) and the Bellman optimality equation (for control).

Policy Evaluation

Given a policy \(\pi\), compute its value function by iteratively applying the Bellman expectation equation:

$$v_{k+1}(s) = \sum_a \pi(a|s) \sum_{s',r} p(s',r|s,a) \left[ r + \gamma v_k(s') \right]$$

This says: the value of state \(s\) under policy \(\pi\) is the expected immediate reward plus the discounted value of the next state. Repeat this update for all states until the values converge.

🧮 Why Policy Evaluation Converges ▼

The iterative policy evaluation update is a contraction mapping. In operator form:

$$v_{k+1} = T^\pi v_k$$

where \(T^\pi\) is the Bellman operator. The key property: for any two value functions \(v\) and \(v'\):

$$\|T^\pi v - T^\pi v'\|_\infty \leq \gamma \|v - v'\|_\infty$$

Since \(\gamma < 1\), repeated application of \(T^\pi\) shrinks differences exponentially. By the Banach fixed point theorem, \(v_k \to v^\pi\), the true value function under policy \(\pi\).

Policy Improvement

Given a value function \(v\), compute a better policy by acting greedily with respect to the current value estimates:

$$\pi'(s) = \arg\max_a \sum_{s',r} p(s',r|s,a) \left[ r + \gamma v(s') \right]$$

This greedy policy is guaranteed to be at least as good as the old policy. Why? Because of the Policy Improvement Theorem:

The Policy Improvement Theorem

If the action-value of the greedy action in state \(s\) is at least as good as the action-value under the current policy, then the new policy is at least as good overall:

\(\pi'(s) = \arg\max_a q_\pi(s,a) \geq v_\pi(s)\) for all \(s\)

\(\Rightarrow v_{\pi'}(s) \geq v_\pi(s)\) for all \(s\)

🧮 Proof Sketch of Policy Improvement Theorem ▼

Denote the old policy as \(\pi\) and the new (greedy) policy as \(\pi'\). For any state \(s\):

$$v_{\pi}(s) \leq q_\pi(s, \pi'(s)) = \mathbb{E}[R_{t+1} + \gamma V_\pi(S_{t+1}) | S_t = s, A_t = \pi'(s)]$$

By the Bellman equation for \(\pi'\):

$$q_\pi(s, \pi'(s)) = \mathbb{E}[R_{t+1}] + \gamma \mathbb{E}[V_\pi(S_{t+1})]$$

Now apply the Bellman equation again, viewing it from the perspective of \(\pi'\) starting from state \(S_{t+1}\):

$$= \mathbb{E}[R_{t+1}] + \gamma \mathbb{E}[q_\pi(S_{t+1}, \pi'(S_{t+1}))] \leq \mathbb{E}[R_{t+1}] + \gamma \mathbb{E}[V_{\pi'}(S_{t+1})] = V_{\pi'}(s)$$

Therefore \(V_{\pi'}(s) \geq V_\pi(s)\) for all \(s\).

Policy Iteration

Combine evaluation and improvement: starting with an arbitrary policy, evaluate it, improve it, evaluate the new policy, improve again, and repeat until convergence:

$$\pi_0 \xrightarrow{\text{E}} V_{\pi_0} \xrightarrow{\text{I}} \pi_1 \xrightarrow{\text{E}} V_{\pi_1} \xrightarrow{\text{I}} \pi_2 \xrightarrow{\text{E}} \cdots \rightarrow \pi^*$$

This converges to the optimal policy in a finite number of iterations (for finite state/action spaces).

Value Iteration

An alternative: skip full policy evaluation and just do one sweep of the Bellman optimality equation at each step:

$$v_{k+1}(s) = \max_a \sum_{s',r} p(s',r|s,a) \left[ r + \gamma v_k(s') \right]$$

This also converges to the optimal value function, but typically requires more total updates (though fewer "policy evaluation loops").

🧮 Value Iteration Convergence ▼

Value iteration uses the Bellman optimality operator:

$$v_{k+1} = T^* v_k, \quad (T^* v)(s) = \max_a \mathbb{E}[R + \gamma V(S')]$$

This is also a contraction with factor \(\gamma\):

$$\|T^* v - T^* v'\|_\infty \leq \gamma \|v - v'\|_\infty$$

So \(v_k \to v^*\) exponentially fast. The optimal policy is then \(\pi^*(s) = \arg\max_a q^*(s,a)\).

Asynchronous DP

In practice, you don't need to update all states in lockstep. Asynchronous DP updates states in any order, even repeatedly sampling the same states, and still converges. This is more flexible and can focus computation on relevant states.

Generalized Policy Iteration (GPI)

Both policy iteration and value iteration are instances of Generalized Policy Iteration: the general pattern of alternating between evaluation and improvement.

The GPI Convergence Guarantee

As long as both processes make progress toward their respective goals, GPI converges to the optimal policy and value function. You don't need to fully evaluate the policy or fully optimize the values — even approximate updates eventually converge.

Generalized Policy Iteration (GPI)

GPI is the unified framework. All DP algorithms — policy iteration, value iteration, and beyond — are instances of evaluation and improvement interacting. This diagram shows the conceptual flow:

Policy Evaluation
Compute \(V_\pi\)
Evaluate →
Policy Improvement
Act greedily
↓ ↑ (Arrows point both directions — convergence spiral)

Key Property: As evaluation and improvement iterate, the policy and value function spiral toward the optimal policy \(\pi^*\) and value function \(V^*\). Neither process needs to be perfect — only that both make progress.

Which Algorithms Are These?

  • Policy Iteration (PI): Full evaluation \(\rightarrow\) single improvement sweep \(\rightarrow\) repeat
  • Value Iteration (VI): One-step lookahead (implicit improvement) in each sweep
  • Asynchronous DP: Update states in any order, not synchronously
💻 Algorithm Implementation

Policy Iteration & Value Iteration in Python

Below is a complete, annotated implementation of both algorithms on a gridworld, mapping each line to the theory above:

Python / NumPy 
import numpy as np
from collections import defaultdict

class GridworldDP:
    """
    Dynamic Programming solver for gridworld environment.
    Implements both Policy Iteration and Value Iteration.

    Environment: 4x4 grid with goal at (0,3) top-right.
    """

    def __init__(self, size=4, gamma=0.9, theta=1e-6):
        # size: grid dimensions
        # gamma (γ): discount factor
        # theta (θ): convergence threshold for policy evaluation
        self.size = size
        self.gamma = gamma
        self.theta = theta

        # State space: (row, col) for each cell
        self.states = [(i, j) for i in range(size) for j in range(size)]
        self.goal = (0, size - 1)

        # Action space: up, down, left, right
        self.actions = [(-1, 0), (1, 0), (0, -1), (0, 1)]
        self.action_names = ['↑', '↓', '←', '→']

        # Value function V(s)
        self.V = {s: 0.0 for s in self.states}
        self.V[self.goal] = 1.0  # Terminal state has value 1

        # Policy π(a|s): uniform random initially
        self.policy = {s: [0.25] * len(self.actions) for s in self.states}
        self.policy[self.goal] = [0] * len(self.actions)  # No actions from goal

    def get_transitions(self, state, action):
        """
        Deterministic transitions (for now).
        state: (row, col)
        action: index into self.actions

        Returns: next_state, reward
        """
        if state == self.goal:
            return state, 0  # Absorbing state

        dr, dc = self.actions[action]
        next_row = state[0] + dr
        next_col = state[1] + dc

        # Boundary handling: bounce back to same state
        if not (0 <= next_row < self.size) or not (0 <= next_col < self.size):
            next_row, next_col = state

        next_state = (next_row, next_col)

        # Reward: -1 for each step (path length penalty), +1 at goal
        reward = -1.0
        if next_state == self.goal:
            reward = 1.0

        return next_state, reward

    # ───────────────────────────────────────────────────────
    # POLICY EVALUATION (Prediction)
    # ───────────────────────────────────────────────────────

    def policy_evaluation(self, eval_sweeps=3):
        """
        ALGORITHM: Iterative Policy Evaluation

        Equation: v_{k+1}(s) = Σ_a π(a|s) Σ_{s',r} p(s',r|s,a)[r + γv_k(s')]

        Args:
            eval_sweeps: number of sweeps (default 3 per PI iteration)
        """
        for sweep in range(eval_sweeps):
            delta = 0  # Track maximum change

            for state in self.states:
                if state == self.goal:
                    continue  # Skip goal state

                # Old value estimate
                v_old = self.V[state]

                # Compute new value using Bellman expectation equation
                v_new = 0.0

                for action_idx, pi_prob in enumerate(self.policy[state]):
                    # Probability of this action under the policy
                    if pi_prob == 0:
                        continue

                    # Get next state and reward for this action
                    next_state, reward = self.get_transitions(state, action_idx)

                    # Bellman expectation: r + γV(s')
                    # Line below maps directly to the equation above:
                    v_new += pi_prob * (reward + self.gamma * self.V[next_state])

                # Update value
                self.V[state] = v_new
                delta = max(delta, abs(v_old - v_new))

            # Convergence check
            if delta < self.theta:
                break

    # ───────────────────────────────────────────────────────
    # POLICY IMPROVEMENT (Control)
    # ───────────────────────────────────────────────────────

    def policy_improvement(self):
        """
        ALGORITHM: Policy Improvement

        π'(s) = arg_max_a Σ_{s',r} p(s',r|s,a)[r + γV(s')]

        Returns True if policy has stabilized (converged).
        """
        policy_stable = True

        for state in self.states:
            if state == self.goal:
                continue

            # Old greedy action
            old_action = np.argmax(self.policy[state])

            # Compute action values q(s, a) for all actions
            action_values = []

            for action_idx in range(len(self.actions)):
                next_state, reward = self.get_transitions(state, action_idx)

                # Action-value function:
                # q(s,a) = Σ_{s',r} p(s',r|s,a)[r + γV(s')]
                q_value = reward + self.gamma * self.V[next_state]
                action_values.append(q_value)

            # Greedy action with respect to current value function
            best_action = np.argmax(action_values)

            # Create new deterministic policy
            new_policy = [0.0] * len(self.actions)
            new_policy[best_action] = 1.0

            # Check if policy changed
            if old_action != best_action:
                policy_stable = False

            self.policy[state] = new_policy

        return policy_stable

    # ───────────────────────────────────────────────────────
    # POLICY ITERATION
    # ───────────────────────────────────────────────────────

    def policy_iteration(self):
        """
        ALGORITHM: Policy Iteration

        π_0 --E--> V_{π_0} --I--> π_1 --E--> V_{π_1} --I--> ... ---> π*

        Alternates between policy evaluation and policy improvement
        until the policy stabilizes.
        """
        iteration = 0

        while True:
            # E: Evaluate current policy
            self.policy_evaluation(eval_sweeps=3)

            # I: Improve the policy
            policy_stable = self.policy_improvement()

            iteration += 1

            if policy_stable:
                break

        return iteration

    # ───────────────────────────────────────────────────────
    # VALUE ITERATION
    # ───────────────────────────────────────────────────────

    def value_iteration(self):
        """
        ALGORITHM: Value Iteration

        v_{k+1}(s) = max_a Σ_{s',r} p(s',r|s,a)[r + γv_k(s')]

        Directly applies the Bellman optimality equation without
        explicit policy evaluation steps.
        """
        iteration = 0

        while True:
            delta = 0

            for state in self.states:
                if state == self.goal:
                    continue

                v_old = self.V[state]

                # Compute max_a q(s,a)
                max_q = -np.inf

                for action_idx in range(len(self.actions)):
                    next_state, reward = self.get_transitions(state, action_idx)

                    # One-step lookahead with max operator
                    q_value = reward + self.gamma * self.V[next_state]
                    max_q = max(max_q, q_value)

                self.V[state] = max_q
                delta = max(delta, abs(v_old - self.V[state]))

            iteration += 1

            # Stop when values have converged
            if delta < self.theta:
                break

        # Reconstruct optimal policy from converged values
        for state in self.states:
            if state == self.goal:
                continue

            action_values = []
            for action_idx in range(len(self.actions)):
                next_state, reward = self.get_transitions(state, action_idx)
                q_value = reward + self.gamma * self.V[next_state]
                action_values.append(q_value)

            best_action = np.argmax(action_values)
            new_policy = [0.0] * len(self.actions)
            new_policy[best_action] = 1.0
            self.policy[state] = new_policy

        return iteration

    def get_greedy_action(self, state):
        """Return the greedy action in state."""
        return np.argmax(self.policy[state])


# ═══════════════════════════════════════════════════════
# USAGE EXAMPLE
# ═══════════════════════════════════════════════════════

if __name__ == '__main__':
    print("=== Policy Iteration ===")
    solver_pi = GridworldDP(gamma=0.9)
    iters_pi = solver_pi.policy_iteration()
    print(f"Converged in {iters_pi} iterations")
    print(f"Final policy: {solver_pi.policy}")

    print("\n=== Value Iteration ===")
    solver_vi = GridworldDP(gamma=0.9)
    iters_vi = solver_vi.value_iteration()
    print(f"Converged in {iters_vi} iterations")
    print(f"Final policy: {solver_vi.policy}")

Note: The code above uses deterministic dynamics for clarity. In the full Sutton & Barto text, the transition function \(p(s',r|s,a)\) may be stochastic. The algorithms work identically — just sum over all possible outcomes.

📊 Comparative Analysis

Policy Iteration vs. Value Iteration

Both algorithms converge to the optimal policy, but they differ in computational cost, convergence rate, and practical considerations:

Aspect Policy Iteration Value Iteration
Inner Loop Full policy evaluation (multiple sweeps until convergence) Single Bellman update sweep
Sweeps per Iteration K sweeps (K ≥ 1), often 3–10 Exactly 1 sweep
Total Updates Fewer policy iterations, more state updates More value iterations, fewer state updates per iteration
Convergence Finite iterations (guaranteed for tabular, finite S×A) Asymptotic, exponential convergence
Wall-Clock Time Often slower (deep evaluation) Often faster (shallower passes)
Memory Requires current and previous value tables during eval Single value table
Asynchronous? Harder (policy eval needs convergence) Easier (one-step lookahead)
Use When... Small state space, want fewer policy checks Large state space, or anytime policy (asynchronous)
Practical Rule of Thumb

Value Iteration is typically preferred in practice because each iteration is faster (one sweep), making wall-clock time shorter even though convergence in iterations may be slower. Policy Iteration is elegant and has a clean finite-iteration guarantee, but the deep evaluation per iteration can be expensive.

🧠 Socratic Sandbox

Deep Dives & Debugging

Test your understanding with these questions. They cover edge cases, convergence, and the limits of the DP assumption.

1 Why does policy iteration converge to the optimal policy in a finite number of iterations (for finite state/action spaces), but value iteration only converges asymptotically? ▼

Policy iteration converges finitely because the policy space is finite — there are at most \(|A|^{|S|}\) possible deterministic policies. Each policy improvement step either keeps the policy unchanged (convergence) or strictly improves it by the Policy Improvement Theorem. Since the value function is monotonically increasing under each improvement and there are finitely many policies, we must eventually reach a stable policy.

Value iteration, by contrast, solves the Bellman optimality equation \(v = T^* v\) where \(T^*\) is a contraction mapping. Banach's fixed-point theorem guarantees convergence, but only in the limit. Each iteration brings us closer (exponentially) but never exactly to the solution in finite time.

2 What happens to the optimal policy when \(\gamma = 0\)? And when \(\gamma = 1\)? ▼

γ = 0: The agent is completely myopic — it only cares about immediate reward. The optimal policy is to greedily maximize the one-step reward from the current state, ignoring all future consequences. In many settings, this leads to poor long-term behavior.

γ = 1: The agent weights all future rewards equally with the present. This can lead to non-stationary value functions (the return might not converge). In continuing tasks (infinite horizon), the return can be infinite, making the problem ill-posed. The Bellman equation may not have a unique solution. In practice, γ should be strictly less than 1 for continuing tasks, typically 0.9–0.99.

3 When is asynchronous DP (updating states in any order, possibly repeatedly) better than synchronous DP (updating all states each sweep)? ▼

Asynchronous DP is better when:

  • States have different relevance: Focus computation on reachable or important states (e.g., near the goal), avoiding wasted updates on irrelevant states.
  • Real-time planning: Don't wait for a full sweep; update based on recent experience and provide a useful (suboptimal) policy anytime.
  • Large state spaces: Synchronous DP requires storing and iterating all \(|S|\) states. Asynchronous can visit a subset.
  • Changing environment: If the model changes slightly, you can focus async updates on affected states rather than redoing everything.

The key insight: as long as every state is updated infinitely often (which the algorithm guarantees), convergence is maintained.

4 What breaks without a model? Why can't we use policy iteration or value iteration to learn directly from experience? ▼

Both PI and VI require the full model: knowing \(p(s',r|s,a)\) — the transition probabilities and reward function for every state-action pair. With a model, we can simulate outcomes and compute expected values without ever stepping into the environment.

Without a model, we have three problems:

  • Can't compute expectations: We don't know what next states or rewards are possible, so we can't evaluate \(\sum p(s',r|s,a)[r + \gamma V(s')]\).
  • Can't explore intelligently: We can't plan which actions to try. We're stuck with trial-and-error.
  • Exploration-exploitation tradeoff: Every action we take costs (in time or reward). We have to learn from the consequences.

This is where Temporal Difference (TD) Learning (Chapters 5–6) comes in. It learns the model implicitly from experience, updating estimates based on actual outcomes rather than imagined ones.

5 How does GPI generalize beyond tabular RL? What happens when we use function approximation (e.g., a neural network as \(V_\theta(s)\))? ▼

GPI is beautifully general. Even with function approximation:

  • Evaluation: Instead of updating a table entry, update the parameters \(\theta\) of a neural network to minimize TD error or fit the Bellman equation.
  • Improvement: Greedily select actions based on \(V_\theta(s)\), even though \(V_\theta\) is an approximation and may not perfectly represent the true value.

The guarantee weakens: convergence is no longer guaranteed. The approximator might oscillate or diverge. But in practice, with careful hyperparameter tuning and regularization, GPI with function approximation (e.g., DQN, policy gradient methods) achieves remarkable results on complex domains.

See Part II of the book for the theory and practice of function approximation in RL.
