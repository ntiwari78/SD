🎯 The Feynman Bridge

Learning House Price Rules

Imagine you're a real estate appraiser tasked with valuing every house in a city with 10 million properties. You could memorize the exact value of each house — but that's table lookup, and it doesn't scale. Houses you've never seen before would have zero value.

Instead, you learn rules: "Each extra bedroom adds $50,000. Each mile from downtown subtracts $10,000. Every year built in the last decade adds $5,000." These rules are your weights (the \(w\) vector). The house's attributes — number of bedrooms, distance from downtown, age — are the features (the \(x(s)\) vector).

The Core Analogy

Houses with unknown values = infinite state spaces. Your price rules = function approximation. Weights (w) = the parameters you learn (e.g., "$50K per bedroom"). Features (x) = the house attributes you observe. Your estimate = \(\hat{v}(s,w) = w^T x(s)\) — weights times features.

In RL, instead of house prices, we're learning state values. Instead of real estate rules, we're learning which state features matter. The mechanism is identical: we observe features, compute an estimate using our learned weights, and update those weights when our estimates are wrong.

Two critical advantages of this approach:

1. Generalization. If you've seen a 3-bedroom house, you can estimate the value of a 3.5-bedroom house without ever having seen it. The rules transfer.

2. Tractability. Instead of learning millions of parameters (one per state), you learn maybe dozens or hundreds (the weights). Learning is faster, storage is tiny, and the approach works even when the state space is infinite or continuous.

The Trade-off: Representation Power vs. Simplicity

Linear rules (bedroom count × weight) are interpretable and efficient, but they can't capture complex patterns. A house's value isn't just linear in its features — perhaps 3 bedrooms are worth more than twice the value of 1.5 bedrooms. Non-linear approximators like neural networks can learn arbitrary patterns, but at the cost of interpretability and computational complexity. Choosing the right approximator is an art.

📐 The Technical Ascent

Value-Function Approximation & Gradient Descent

We want to approximate the true value function \(v_\pi(s)\) with a parameterized function \(\hat{v}(s, \mathbf{w})\), where \(\mathbf{w}\) is a weight vector. The goal is to find weights such that \(\hat{v}(s, \mathbf{w}) \approx v_\pi(s)\) for all states \(s\).

🧮 The Prediction Objective: Mean Squared Value Error ▼

We measure approximation quality using the Mean Squared Value Error (VE):

$$\text{VE}(\mathbf{w}) = \sum_s \mu(s) \left[v_\pi(s) - \hat{v}(s, \mathbf{w})\right]^2$$

Here, \(\mu(s)\) is a weighting over states — usually the on-policy state occupancy (how often we visit state \(s\) under the current policy). We want to minimize VE by adjusting \(\mathbf{w}\).

In supervised learning, we'd use exact targets \(y_i\) like labels. In RL, we don't have exact values — only noisy estimates. That's where temporal difference data comes in.

Since we don't know the true value \(v_\pi(s)\), we use samples — rewards and bootstrapped estimates — as noisy targets. The key idea is stochastic gradient descent (SGD): take small steps in the direction that reduces error.

🧮 Stochastic Gradient Descent Update ▼

For a single sample \((S_t, v_\text{target})\), the SGD update is:

$$\mathbf{w}_{t+1} = \mathbf{w}_t + \alpha \left[v_\text{target} - \hat{v}(S_t, \mathbf{w}_t)\right] \nabla_{\mathbf{w}} \hat{v}(S_t, \mathbf{w}_t)$$

Here, \(\alpha\) is the step-size (learning rate), and \(\nabla_{\mathbf{w}} \hat{v}\) is the gradient of the approximator with respect to its weights. The update pushes the estimate toward the target by moving weights in the direction of steepest descent.

The term \(\left[v_\text{target} - \hat{v}(S_t, \mathbf{w}_t)\right]\) is the temporal difference (TD) error — how far off our estimate is.

In on-policy prediction with bootstrapping, we face a subtlety: we use the current weight vector to estimate the next state's value. This means the target itself depends on \(\mathbf{w}\), violating the i.i.d. assumption of SGD. The fix is the semi-gradient method: treat the bootstrapped estimate as a fixed target (ignore its dependence on \(\mathbf{w}\)), then take the gradient.

🧮 Semi-gradient TD(0) Update ▼

The core on-policy prediction update with function approximation:

$$\mathbf{w}_{t+1} = \mathbf{w}_t + \alpha \left[R_{t+1} + \gamma \hat{v}(S_{t+1}, \mathbf{w}_t) - \hat{v}(S_t, \mathbf{w}_t)\right] \nabla_{\mathbf{w}} \hat{v}(S_t, \mathbf{w}_t)$$

The TD target is \(R_{t+1} + \gamma \hat{v}(S_{t+1}, \mathbf{w}_t)\). We treat this as fixed (the "semi" part), then take only the gradient with respect to \(\hat{v}(S_t, \mathbf{w}_t)\), ignoring the gradient of the bootstrapped \(\hat{v}(S_{t+1}, \mathbf{w}_t)\) term.

Why? Taking the full gradient would make convergence less reliable. The semi-gradient approach sacrifices theoretical optimality for practical stability. It's the foundation of deep RL methods like DQN.

📑 Implementation

Linear Methods & Feature Engineering

The simplest and most interpretable function approximator is linear:

$$\hat{v}(s, \mathbf{w}) = \mathbf{w}^T \mathbf{x}(s) = \sum_i w_i x_i(s)$$

Here, \(\mathbf{x}(s)\) is a feature vector extracted from state \(s\). Each feature \(x_i(s)\) encodes some aspect of the state (e.g., distance to goal, number of enemies). The weight \(w_i\) tells us how much that feature contributes to the value.

Gradient for linear approximators: \(\nabla_{\mathbf{w}} \hat{v}(s, \mathbf{w}) = \mathbf{x}(s)\). Simple! The gradient is just the feature vector itself.

Feature Construction Techniques

Polynomials: For 1D state \(s\), use \([1, s, s^2, s^3, \ldots]\). Higher powers capture nonlinear relationships. Risk: high-order polynomials oscillate wildly (Runge's phenomenon).

Fourier Basis: Use sine/cosine functions at different frequencies: \([1, \sin(\pi s), \cos(\pi s), \sin(2\pi s), \ldots]\). Better numerical stability and frequency-based interpretation.

Coarse Coding: Divide the state space into overlapping bins. If state \(s\) falls in bin \(i\), set \(x_i(s) = 1\); otherwise 0. Simple and works well for continuous spaces.

Tile Coding: Multiple coarse-coded tilings, offset from each other. A state activates one tile in each tiling layer. More tiles = finer resolution. Very efficient in practice.

RBFs (Radial Basis Functions): Gaussians centered at different points in the state space. \(x_i(s) = \exp\left(-\frac{\|s - c_i\|^2}{2\sigma^2}\right)\). Smooth and interpretable, but sensitive to center placement.

Step-Size Parameters

The learning rate \(\alpha\) controls how fast we update. Too high: divergence or instability. Too low: slow learning. In linear function approximation, it's safe to use a constant step-size and expect convergence (under certain conditions). With nonlinear approximators like neural networks, careful scheduling (reducing \(\alpha\) over time) is often necessary.

For nonlinear approximators like neural networks, the gradient-descent principle remains the same, but the gradient is now computed via backpropagation. The challenge: loss landscape is non-convex, and semi-gradient methods don't guarantee convergence. Yet they work remarkably well in practice (e.g., DQN, A3C, PPO).

💻 Algorithm

Semi-gradient TD(0) with Linear Approximation

Below is a complete, working Python implementation of on-policy value prediction using semi-gradient TD with linear function approximation. This forms the foundation for more complex RL algorithms.

python 
import numpy as np
from collections import defaultdict

class LinearFunctionApproximator:
    """Value function approximator: v_hat(s, w) = w^T x(s)"""

    def __init__(self, num_features, initial_weights=None):
        self.num_features = num_features
        if initial_weights is None:
            self.w = np.zeros(num_features)
        else:
            self.w = initial_weights.copy()

    def predict(self, features):
        """Return estimated value: w^T x"""
        return np.dot(self.w, features)

    def update(self, features, td_error, alpha):
        """SGD update: w <- w + alpha * td_error * x"""
        self.w += alpha * td_error * features

    def gradient(self, features):
        """Gradient w.r.t. w is simply the feature vector"""
        return features


class SemiGradientTDPredictor:
    """On-policy prediction using semi-gradient TD(0)"""

    def __init__(self, gamma=0.99, alpha=0.01, num_features=10):
        self.gamma = gamma           # discount factor
        self.alpha = alpha           # step-size
        self.approximator = LinearFunctionApproximator(num_features)
        self.value_errors = []       # track TD errors

    def get_features(self, state):
        """
        Extract features from state.
        Example: polynomial features for 1D state in [0, 1]
        """
        # Simple polynomial features: [1, s, s^2, ..., s^(n-1)]
        s = float(state) if not np.isscalar(state) else state
        features = np.array([s ** i for i in range(self.approximator.num_features)])
        return features

    def predict_value(self, state):
        """Estimate the value of a state"""
        features = self.get_features(state)
        return self.approximator.predict(features)

    def update_weights(self, state_t, reward_t1, state_t1, gamma=None):
        """
        Semi-gradient TD(0) update:
        w <- w + alpha * [R + gamma * v_hat(S', w) - v_hat(S, w)] * x(S)

        Key: we treat v_hat(S', w) as a fixed target (don't take gradient through it).
        """
        if gamma is None:
            gamma = self.gamma

        # Get features
        x_t = self.get_features(state_t)

        # Current estimate
        v_t = self.approximator.predict(x_t)

        # Bootstrap estimate for next state (using current weights as fixed target)
        v_t1 = self.approximator.predict(self.get_features(state_t1))

        # TD target
        td_target = reward_t1 + gamma * v_t1

        # TD error
        td_error = td_target - v_t
        self.value_errors.append(td_error)

        # Update weights
        self.approximator.update(x_t, td_error, self.alpha)

        return td_error

    def get_weights(self):
        return self.approximator.w.copy()

    def set_weights(self, w):
        self.approximator.w = w.copy()

    def mean_value_error(self, states, true_values, state_dist=None):
        """
        Compute Mean Squared Value Error over a set of states.
        VE(w) = sum_s mu(s) * [v_pi(s) - v_hat(s,w)]^2
        """
        if state_dist is None:
            state_dist = np.ones(len(states)) / len(states)

        mse = 0.0
        for s, v_true, mu in zip(states, true_values, state_dist):
            v_hat = self.predict_value(s)
            mse += mu * (v_true - v_hat) ** 2
        return mse


def simulate_episode(predictor, env_dynamics_fn, start_state, episode_length=20):
    """
    Run one episode and update value function.

    Args:
        predictor: SemiGradientTDPredictor
        env_dynamics_fn: function(state) -> (next_state, reward)
        start_state: starting state
        episode_length: number of steps
    """
    state = start_state
    total_return = 0.0

    for t in range(episode_length):
        # Environment step
        next_state, reward = env_dynamics_fn(state)

        # Semi-gradient TD update
        td_error = predictor.update_weights(state, reward, next_state)

        total_return += (predictor.gamma ** t) * reward
        state = next_state

    return total_return


# ───── Example: Simple 1D Value Learning ─────
if __name__ == "__main__":
    # Define a simple environment: random walk with reward structure
    def simple_env(state):
        """Random walk: bias toward high values"""
        next_state = state + np.random.randn() * 0.1
        next_state = np.clip(next_state, 0, 1)  # stay in [0, 1]
        # Reward increases toward right
        reward = next_state + 0.1 * np.random.randn()
        return next_state, reward

    # Create predictor with polynomial features
    predictor = SemiGradientTDPredictor(gamma=0.99, alpha=0.01, num_features=5)

    # Training loop
    num_episodes = 100
    for episode in range(num_episodes):
        start_state = np.random.uniform(0, 1)
        return_val = simulate_episode(predictor, simple_env, start_state)

        if (episode + 1) % 20 == 0:
            print(f"Episode {episode+1}: weights = {predictor.get_weights()}, "
                  f"return = {return_val:.3f}")

    # Final values at some test points
    test_states = [0.0, 0.25, 0.5, 0.75, 1.0]
    print("\nFinal learned values:")
    for s in test_states:
        v = predictor.predict_value(s)
        print(f"  v({s:.2f}) = {v:.4f}")

Key points:

  • The update_weights method implements the semi-gradient TD(0) update. Notice how we compute the TD target using the current weights (not updated during the gradient).
  • Feature extraction via polynomial basis is simple but limited. Real systems use richer features: tile coding, RBFs, hand-engineered domain knowledge, or learned representations (deep learning).
  • The algorithm converges because linear approximators in on-policy settings have favorable convergence properties under SGD.
  • This architecture scales to continuous state spaces without discretization, a huge advantage over tabular methods.
Deep RL Bridge

From Linear Weights to Neural Networks

The semi-gradient TD update you just learned is universal. Swap out the linear approximator for a neural network, and you get the foundation of Deep Q-Networks (DQN) — arguably the most important algorithm in modern RL.

DQN (2015, Mnih et al.): Uses a convolutional neural network as the function approximator, playing Atari from raw pixels. The same semi-gradient update runs, but now the network learns to extract features automatically. The network's hidden layers discover that edges matter, sprites matter, and patterns in the game state.

The leap: Instead of hand-engineering tile coding or RBF centers, let the neural network learn them. This is the power of representation learning. DQN is just semi-gradient TD with a deep approximator — conceptually simple, practically revolutionary.

Debug & Explore: Five Questions

Use these questions to stress-test your understanding. Click to reveal answers.

1
Why is it called "semi-gradient"? What are we NOT taking the gradient of?
▼
In the TD update \(\mathbf{w}_{t+1} = \mathbf{w}_t + \alpha [R_{t+1} + \gamma \hat{v}(S_{t+1}, \mathbf{w}_t) - \hat{v}(S_t, \mathbf{w}_t)] \nabla \hat{v}(S_t, \mathbf{w}_t)\), the TD target \(R_{t+1} + \gamma \hat{v}(S_{t+1}, \mathbf{w}_t)\) depends on \(\mathbf{w}_t\) through the second term. In a true gradient-descent approach, we'd take the gradient through both terms. Instead, we treat \(\hat{v}(S_{t+1}, \mathbf{w}_t)\) as a constant (not differentiating it), so we only take the gradient w.r.t. \(\hat{v}(S_t, \mathbf{w}_t)\). This gives faster, more stable learning at the cost of theoretical convergence guarantees. It's "semi" because we omit part of the true gradient.
2
Why doesn't full gradient descent (taking the gradient of the entire TD target) work well in practice?
▼
Full gradient would give us a true SGD update to minimize the squared TD error. However, the target itself is estimated using the same weights, so the landscape is moving as we learn. This creates a non-stationary learning problem: the "ground truth" keeps shifting. The bootstrap target (using current weights) is biased when those weights are inaccurate. Semi-gradient sidesteps this by treating the next-state value as fixed, breaking the feedback loop and enabling stable learning, even though the final fixed point is biased.
3
Tile coding vs. polynomial features: when would you choose tile coding?
▼
Tile coding shines when: (1) you need local approximation (nearby states should have similar values), (2) you want control over resolution (add more tilings for finer detail), (3) you're working with multi-dimensional continuous states (polynomial features blow up in dimension), and (4) you want sparse, interpretable features. Polynomial features are good for low-dimensional problems with clear global trends but fail in high dimensions and can exhibit Runge's phenomenon (oscillations at high orders). Tile coding is more robust and is still widely used in practice.
4
When does linear approximation fundamentally fail? Give an example.
▼
Linear approximation fails when the true value function is inherently nonlinear. A simple example: in a 2D gridworld, the optimal value is the minimum distance to a goal. If states are represented by their (x, y) coordinates, the true value is something like \(v(x,y) = \sqrt{x^2 + y^2}\) — highly nonlinear! Linear approximation \(\hat{v}(x,y,w) = w_0 + w_1 x + w_2 y\) can't capture this Euclidean distance structure, no matter what weights we choose. You'd need polynomial features (like \(x^2, y^2, xy\)) or a nonlinear approximator (neural network) to fit it well.
5
Neural networks are nonlinear approximators, but why are they tricky to train with semi-gradient TD?
▼
Neural networks have non-convex loss landscapes: lots of local minima, saddle points, and plateaus. With linear approximators, the loss is convex, so any local minimum is global. With neural networks, convergence isn't guaranteed — you can get stuck. Also, the semi-gradient TD target uses bootstrapping (current network weights), which introduces a moving target: the network chases a goal that's shifting as the network learns. Experience replay (storing and replaying past transitions) and target networks (using an old copy of the weights for bootstrap targets) help stabilize training, but these are tricks on top of the core semi-gradient idea. Deep RL is fundamentally harder than supervised learning because of this moving-target issue.

Chapter Summary & Next Steps

In Chapter 9, you learned how to escape the curse of tabular methods by using function approximation. Key concepts:

  • Value-function approximation scales value estimation to infinite or very large state spaces using learned weights and hand-engineered or discovered features.
  • The prediction objective (VE) quantifies approximation error: mean squared error across states, weighted by occupancy.
  • Stochastic gradient descent is the core learning mechanism: update weights in the direction that reduces TD error.
  • Semi-gradient methods treat bootstrapped estimates as fixed targets, sacrificing convergence guarantees for stability and speed.
  • Linear approximators are interpretable, stable, and still practical (tile coding, RBFs). Nonlinear approximators (neural networks) unlock arbitrary representation power, leading to deep RL.
  • Feature engineering is an art: polynomials, Fourier basis, coarse coding, tile coding, RBFs, or learned representations each have trade-offs.
