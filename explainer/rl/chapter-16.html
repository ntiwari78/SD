🏆 The Feynman Bridge

Victory Lap: From Theory to Triumph

This chapter is the victory lap — where abstract algorithms become world-changing applications. From a backgammon program that taught itself to beat world champions, to AlphaGo defeating the greatest Go player in history, to DQN learning to play Atari from raw pixels.

Each system in this chapter tells a story:

  • Samuel's Checkers Player (1959): Proved that machines could learn strategy through self-play.
  • TD-Gammon (1992): Combined temporal-difference learning with neural networks to beat the world's best backgammon players.
  • DQN (2013): Showed that deep learning could master Atari games from raw pixels — a watershed moment for modern RL.
  • AlphaGo (2016): Integrated tree search, policy networks, and value networks to defeat the world Go champion.
  • AlphaGo Zero (2017): Learned Go from pure self-play, with no human knowledge — a glimpse of AGI-level learning.
Why These Applications Matter

These aren't toy problems. Backgammon has 10^20 possible positions. Go has 10^170 — more than atoms in the universe. Chess, Atari, and Go represent domains where human intuition fails and only learning algorithms can scale. Each breakthrough opened new frontiers.

⚙️ Technical Ascent

Key Architecture Patterns

Each system innovated in different ways. Here are the core architectural patterns:

TD-Gammon (1992)

Combined temporal-difference learning with a neural network. The network learned a value function V(s) directly from self-play. No minimax tree search — just bootstrapping from future value estimates. Trained on 1.5 million games of self-play.

DQN (2013)

Introduced two key innovations:

  • Experience Replay: Store transitions in a buffer, sample randomly during training. Breaks correlation in sequential experience.
  • Target Networks: Maintain a separate network Q_target that's updated slowly. Stabilizes learning by reducing oscillations.

Processed raw Atari frames (84×84 pixels) through a CNN to output Q-values for each action. One network, multiple games, no domain knowledge.

AlphaGo (2016)

Integrated multiple components:

  • Supervised Learning: Policy network π_SL trained on 30 million human Go moves.
  • Policy Gradient RL: Policy network π_RL trained via self-play to win more games.
  • Value Network: Predicts probability of winning from any board state.
  • Monte Carlo Tree Search: Uses policy and value networks to guide search. Evaluates leaf nodes with value network instead of rollouts.

AlphaGo Zero (2017)

Tabula rasa learning — starts with zero knowledge of Go. No supervised learning on human games. Only self-play and MCTS. Single neural network outputs both policy and value. Achieves superhuman play in 40 days. Demonstrates that human knowledge is optional.

System: Reinforcement learning agent for optimal bet sizing during Jeopardy dailies.

Key Challenge: Given current score, opponent scores, and category difficulty — decide how much to wager.

Approach: Learned value function over game states; computed expected value of each bet.

Impact: Demonstrated RL for real-world decision-making under uncertainty in high-stakes scenarios.
2013
DQN Learns Atari from Pixels [Click for details]
System: Deep Q-Network with CNN, experience replay, target networks.

Input: 84×84 grayscale Atari frames. No hand-crafted features.

Architecture: Conv layers → Dense layers → Q-values for each action.

Innovations: Experience replay buffer, target network τ, ε-greedy exploration.

Achievement: Superhuman performance on 29 of 49 Atari games. Single agent, no task-specific tuning.

Impact: Watershed moment. Proved deep RL was practical and scalable.
2016
AlphaGo Defeats Lee Sedol [Click for details]
System: Hybrid architecture: supervised learning + policy gradient RL + MCTS + value networks.

Training Pipeline: (1) SL on 30M human games. (2) RL via self-play. (3) Value network trained on game outcomes.

Search: MCTS guided by policy and value networks. Evaluates leaf nodes with neural network instead of Monte Carlo rollouts.

Achievement: Defeated world champion Lee Sedol 4–1 (1 draw). 10^170 possible board states.

Impact: Go is intuitive, strategic, not amenable to brute force. Proved deep RL could match human intuition.
2017
AlphaGo Zero: Learning from Scratch [Click for details]
System: Single neural network (policy + value). Pure self-play. No human data.

Training: 40 days, 5 million self-play games. Alternates MCTS + training.

Key Insight: Human knowledge was not the bottleneck. Learning algorithm was.

Achievement: Surpassed AlphaGo within days. Won 100–0 against original AlphaGo.

Impact: Demonstrated tabula rasa learning. Showed AGI-level learning is possible.

Key Training Loop

1. Sample: Draw batch from experience replay buffer.

2. Compute Target: y = r + γ max Q_target(s', a)

3. Loss: (y - Q(s, a))²

4. Optimize: Gradient descent on DQN parameters.

5. Update Target: Every C steps, copy Q → Q_target (soft or hard).

Why This Works: Convolutional layers extract spatial features from pixels (edges, objects, patterns). Fully connected layers map features to action values. Experience replay decorrelates data. Target networks stabilize learning. Together, they learn end-to-end without hand-crafted features.

🎯 AlphaGo Pipeline

Hybrid Learning: Supervised + RL + Tree Search

AlphaGo combines three learning modalities. Click each stage to see details:

AlphaGo Training Pipeline

Stage 1: Supervised Learning (SL)

Train policy network π_SL on 30 million expert human Go games. Predict the next move given any board state.

Why: Initializes network with human intuition. Provides baseline 57% accuracy on professional games.

↓

Stage 2: Policy Gradient RL

Self-play with policy gradient (REINFORCE): Play game, win/lose reward = ±1, backprop to improve policy.

Why: Improves on supervised learning. Learns to win, not just mimic humans.

↓

Stage 3: Value Network

Train separate network V(s) to predict winning probability from position. Trained on outcomes from RL self-play games.

Why: Faster than Monte Carlo rollouts. Evaluates leaf nodes in tree search.

↓

Stage 4: Monte Carlo Tree Search (MCTS)

Build tree incrementally. At each node, use policy network π to choose promising moves. Use value network V(s) to evaluate terminal nodes.

Why: Tree search + neural networks = planning + learning. Balances exploration and exploitation.

The AlphaGo Zero Simplification

AlphaGo Zero removed Stage 1 (supervised learning). A single network outputs both policy and value. Self-play alone recovers and exceeds AlphaGo performance, proving that human knowledge accelerates but isn't necessary.

🤔 Socratic Sandbox

Five Critical Questions

Use these questions to deepen your understanding. Click to reveal thoughtful answers:

1
What made TD-Gammon succeed where earlier backgammon engines failed?
▼
Key Insight: TD-Gammon succeeded because it learned value functions from self-play, not from endgame tables or hand-engineered evaluation heuristics. Earlier engines relied on brute-force search and human-crafted evaluation functions — both limited. TD-Gammon bootstrapped: each position was evaluated by predicting the value of the next position, creating a learning signal from millions of games. This generalized better than any hand-coded heuristic. The combination of TD learning + neural networks showed that machines could learn their own evaluation functions at world-champion level.
2
Why is experience replay essential for DQN, but not necessary for TD-Gammon?
▼
Core Reason: DQN uses deep neural networks (CNNs with millions of parameters), while TD-Gammon uses a small 3-layer network. Large networks are prone to overfitting to correlated sequences of transitions (temporal autocorrelation destabilizes learning). Experience replay breaks these correlations by sampling uniformly from past experience. Also, deep networks are more sensitive to target network oscillations — target networks in DQN stabilize learning. TD-Gammon's smaller network was more stable, so replay wasn't as critical. This illustrates a scaling law: as function approximators grow larger, the need for decorrelation and stability mechanisms increases.
3
Why did AlphaGo need both policy and value networks, while AlphaGo Zero uses a single network?
▼
Architectural Trade-off: AlphaGo trained on human data first (supervised learning), then used policy and value networks as separate experts. The policy network was good at mimicking humans, the value network was good at predicting outcomes. MCTS used both. AlphaGo Zero discovered that a single network can learn both — this is more parameter-efficient and avoids the need for human data initialization. The single network outputs both policy logits and a scalar value. It's not that two networks are wrong; it's that one network is sufficient if you train purely from self-play with enough compute. This is a lesson in architectural simplicity: when in doubt, start with one network and add more only if needed.
4
Can the techniques from AlphaGo be applied to real-world problems like robotics or control?
▼
Challenges and Solutions: AlphaGo operates in a perfect-information, turn-based, deterministic environment with clear win/loss signals. Real-world robotics is continuous, partially observable, noisy, and samples are expensive. However, the core techniques transfer: (1) Self-play works in simulation — physics engines allow endless interaction. (2) Policy networks learn control primitives. (3) Value networks estimate long-term reward. (4) Tree search can be replaced by learned forward models. Recent work (AlphaStar, MuZero) applies AlphaGo principles to more complex domains. The bottleneck isn't the algorithm — it's simulation realism and sample efficiency. Robotics still struggles because every real-world interaction is costly; games provide free rollouts.
5
What are the fundamental limitations of these systems? Can they become truly general intelligence?
▼
Honest Assessment: These systems are domain-specific. TD-Gammon learned only backgammon. DQN learned Atari, but a single DQN doesn't play chess. AlphaGo Zero masters Go, but can't play Shogi or chess without retraining. Each system is optimized for a specific reward signal and action space. Generality requires: (1) Transfer learning (skills from one domain → another). (2) Curriculum learning (master simple tasks first). (3) World models (learning how the environment works, not just win/loss). (4) Composite objectives (multi-task learning). Modern RL (e.g., GPT-powered agents, multi-task RL) tackles these, but we haven't solved the hard problem: learning general-purpose reasoning from limited experience, the way human intelligence does. These systems are narrow superintelligences in narrow domains.
