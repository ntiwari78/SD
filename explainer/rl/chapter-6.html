🎯 The Feynman Bridge

TD Learning: Update Your GPS While Driving

Imagine you're driving to a destination and your GPS estimates you'll arrive in 2 hours. You drive for 15 minutes and hit traffic.

Monte Carlo approach: You keep driving. At the end, when you arrive, you learn the total trip took 3 hours instead of 2. You update your estimate retroactively.

Temporal-Difference approach: You don't wait. Every few minutes, you glance at your current location and the updated GPS estimate for remaining travel time. You revise your arrival estimate right now, based on what you observe happening in real-time. You're bootstrapping: using your own predictions to update your predictions.

The Core Insight

TD = Monte Carlo + Bootstrapping. Like Monte Carlo, TD learns from actual samples (you don't need a model of traffic patterns). But like dynamic programming, TD uses its own value estimates to bootstrap faster learning. You don't wait for the episode to end — you update incrementally, every step.

This is the genius of Chapter 6: you can learn model-free (from experience, like MC) but faster (using bootstrapping, like DP). This hybrid approach gives you the best of both worlds.

TD learning is the foundation of modern deep reinforcement learning. DQN, PPO, AlphaGo — they all use the core idea: update value estimates online, using your current predictions as targets.

📐 The Technical Ascent

TD(0) Prediction and the TD Error

TD learning comes in many forms, but the simplest — TD(0) — is the workhorse of modern RL.

TD(0) Value Update

After observing a single transition \((S_t, A_t, R_{t+1}, S_{t+1})\), TD(0) updates the value of the current state:

$$V(S_t) \leftarrow V(S_t) + \alpha \left[ R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \right]$$

The key term in brackets is the TD error (often written as \(\delta_t\)):

$$\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)$$
What is the TD Error?

The TD error measures the surprise in your prediction. It's the difference between what you actually observed (\(R_{t+1} + \gamma V(S_{t+1})\), the "bootstrapped return") and what you predicted (\(V(S_t)\)). If \(\delta_t > 0\), the state was better than expected — increase its value. If \(\delta_t < 0\), it was worse — decrease its value. The magnitude tells you how wrong you were.

🧮 Why This Matters: MC vs TD vs DP ▼

The three learning paradigms differ in what they use as the "target" for the value estimate:

Monte Carlo (MC): Target is the full return \(G_t\) (the actual sum of all rewards to episode end)

$$V(S_t) \leftarrow V(S_t) + \alpha[G_t - V(S_t)]$$

Temporal-Difference (TD): Target is the bootstrapped return \(R_{t+1} + \gamma V(S_{t+1})\)

$$V(S_t) \leftarrow V(S_t) + \alpha[R_{t+1} + \gamma V(S_{t+1}) - V(S_t)]$$

Dynamic Programming (DP): Target is the expected bootstrapped return (requires knowing the environment model)

$$V(S_t) \leftarrow \mathbb{E}[R_{t+1} + \gamma V(S_{t+1}) | S_t]$$

TD is the sweet spot: it uses the bootstrapped return (like DP) but learns it from samples (like MC), so you don't need the model.

Sarsa: On-Policy TD Control

To control the environment (not just predict), you extend TD(0) to action values. SARSA (State-Action-Reward-State-Action) learns the value of state-action pairs:

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t) \right]$$

SARSA is on-policy: it learns the value of the policy it's currently following. When you move from \(S_{t+1}\) using action \(A_{t+1}\), you use \(Q(S_{t+1}, A_{t+1})\) — the value of the actual next action you'll take. This makes it conservative: it learns a safe policy by exploring with \(\epsilon\)-greedy and incorporating the risk of exploratory actions.

Q-learning: Off-Policy TD Control

Q-learning is a small but powerful modification to SARSA:

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma \max_a Q(S_{t+1}, a) - Q(S_t, A_t) \right]$$

The difference: instead of using \(Q(S_{t+1}, A_{t+1})\) (the next action you actually take), Q-learning uses \(\max_a Q(S_{t+1}, a)\) (the best possible next action). This makes Q-learning off-policy: it learns an optimal policy while exploring with a different (e.g., \(\epsilon\)-greedy) policy.

The Power of Q-learning

Q-learning decouples behavior (how you explore) from learning (what policy you're learning about). You can explore freely and still learn the optimal policy. This flexibility makes Q-learning one of the most important algorithms in RL.

Expected Sarsa

A hybrid approach uses the expected value over the next action distribution:

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma \sum_a \pi(a|S_{t+1}) Q(S_{t+1}, a) - Q(S_t, A_t) \right]$$

Expected Sarsa reduces variance compared to SARSA (by averaging over actions) while remaining on-policy. It's smoother than SARSA but slower than Q-learning when the policy is \(\epsilon\)-greedy.

The Maximization Bias and Double Q-learning

Q-learning has a subtle flaw: \(\max_a Q(S_{t+1}, a)\) overestimates values because we take the maximum of noisy estimates. When you estimate many Q-values with noise, the max tends to be high. This maximization bias (also called overestimation) accumulates over time and can lead to suboptimal policies.

Double Q-learning fixes this by using two separate Q-tables:

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma Q'(S_{t+1}, \arg\max_a Q(S_{t+1}, a)) - Q(S_t, A_t) \right]$$

One table (\(Q\)) selects the best action, the other (\(Q'\)) evaluates it. This decorrelation reduces overestimation.

💻 Algorithm Implementation

SARSA and Q-learning on Cliff Walking

Below is a complete implementation of both algorithms on the Cliff Walking environment. Study how they differ only in one line: the backup target.

Python / NumPy 
import numpy as np
from collections import defaultdict

class CliffWalking:
    """
    Gridworld: 4 rows x 12 columns.
    Start: (3, 0) — bottom left
    Goal: (3, 11) — bottom right
    Cliff: (3, 1) to (3, 10) — reward -100, then reset to start
    Normal step: reward -1

    Actions: 0=up, 1=down, 2=right, 3=left
    """

    def __init__(self):
        self.start = (3, 0)
        self.goal = (3, 11)
        self.reset()

    def reset(self):
        self.state = self.start
        return self.state

    def step(self, action):
        """Execute action, return (next_state, reward, done)"""
        r, c = self.state

        # Movement delta for each action
        deltas = {0: (-1, 0), 1: (1, 0), 2: (0, 1), 3: (0, -1)}
        dr, dc = deltas.get(action, (0, 0))

        next_r, next_c = r + dr, c + dc

        # Boundary clipping
        next_r = max(0, min(3, next_r))
        next_c = max(0, min(11, next_c))

        self.state = (next_r, next_c)

        # Reward and terminal check
        if self.state in [(3, i) for i in range(1, 11)]:
            # Fell off cliff
            reward = -100
            done = True
            self.state = self.start  # Reset to start
        elif self.state == self.goal:
            reward = 0
            done = True
        else:
            reward = -1
            done = False

        return self.state, reward, done


class TDControl:
    """Base class for TD control algorithms"""

    def __init__(self, env, alpha=0.1, gamma=0.99, epsilon=0.1):
        self.env = env
        self.alpha = alpha
        self.gamma = gamma
        self.epsilon = epsilon
        self.Q = defaultdict(lambda: np.zeros(4))
        self.episode_rewards = []

    def get_action(self, state, training=True):
        """Epsilon-greedy action selection"""
        if training and np.random.random() < self.epsilon:
            return np.random.randint(4)
        return np.argmax(self.Q[state])

    def train_episode(self):
        """Run one episode and return total reward"""
        state = self.env.reset()
        total_reward = 0
        done = False

        while not done:
            action = self.get_action(state, training=True)
            next_state, reward, done = self.env.step(action)

            # This line differs between SARSA and Q-learning!
            self.update(state, action, reward, next_state, done)

            state = next_state
            total_reward += reward

        self.episode_rewards.append(total_reward)
        return total_reward

    def update(self, state, action, reward, next_state, done):
        """Override in subclasses"""
        raise NotImplementedError


class SARSA(TDControl):
    """On-policy TD control"""

    def update(self, state, action, reward, next_state, done):
        """
        SARSA: use the next action actually taken
        Q(s,a) <- Q(s,a) + α[r + γQ(s',a') - Q(s,a)]
        """
        if done:
            target = reward
        else:
            # Next action is selected by the same policy (ε-greedy)
            next_action = self.get_action(next_state, training=True)
            target = reward + self.gamma * self.Q[next_state][next_action]

        # TD update
        td_error = target - self.Q[state][action]
        self.Q[state][action] += self.alpha * td_error


class QLearning(TDControl):
    """Off-policy TD control"""

    def update(self, state, action, reward, next_state, done):
        """
        Q-learning: use the best next action
        Q(s,a) <- Q(s,a) + α[r + γ max_a' Q(s',a') - Q(s,a)]
        """
        if done:
            target = reward
        else:
            # Next action is the greedy best action (not necessarily taken!)
            best_next_value = np.max(self.Q[next_state])
            target = reward + self.gamma * best_next_value

        # TD update (same form as SARSA, but different target)
        td_error = target - self.Q[state][action]
        self.Q[state][action] += self.alpha * td_error


def compare_algorithms(episodes=1000, runs=10):
    """Run both algorithms multiple times and average"""

    sarsa_rewards = np.zeros((runs, episodes))
    qlearn_rewards = np.zeros((runs, episodes))

    for run in range(runs):
        env = CliffWalking()

        # Train SARSA
        sarsa = SARSA(env, alpha=0.1, gamma=0.99, epsilon=0.1)
        for ep in range(episodes):
            sarsa.train_episode()
        sarsa_rewards[run] = sarsa.episode_rewards

        # Train Q-learning
        env = CliffWalking()
        qlearn = QLearning(env, alpha=0.1, gamma=0.99, epsilon=0.1)
        for ep in range(episodes):
            qlearn.train_episode()
        qlearn_rewards[run] = qlearn.episode_rewards

    # Plot results
    sarsa_mean = np.mean(sarsa_rewards, axis=0)
    qlearn_mean = np.mean(qlearn_rewards, axis=0)

    print("SARSA average reward (last 100 episodes):", np.mean(sarsa_mean[-100:]))
    print("Q-learning average reward (last 100 episodes):", np.mean(qlearn_mean[-100:]))

    # Q-learning achieves -25 (optimal safe path along cliff edge)
    # SARSA achieves ~-30 to -40 (plays it safer due to on-policy learning)


if __name__ == "__main__":
    compare_algorithms()
🔆 Comparative Analysis

DP vs MC vs TD: The Three Paradigms

Chapter 6 unifies three approaches to learning. Here's how they relate:

Aspect Dynamic Programming (DP) Monte Carlo (MC) Temporal-Difference (TD)
Bootstraps? Yes — uses \(V(s')\) to update \(V(s)\) No — uses actual returns \(G_t\) Yes — uses \(V(s')\) but from samples
Samples? No — requires full model Yes — learns from episodes Yes — learns from single steps
Model Required? Yes — \(p(s',r|s,a)\) No — model-free No — model-free
Online Learning? Yes — continuous improvement No — wait for episode end Yes — update every step
Bias Biased (uses estimates) Unbiased (uses actual returns) Biased (uses estimates)
Variance Low (many bootstraps) High (full trajectory) Low-Medium (one-step bootstrap)
Convergence Speed Fastest (if model accurate) Slowest (waits for episodes) Fast (bootstraps early)
Sample Efficiency Perfect (no samples) Poor (needs many episodes) Good (per-step updates)
Key Equation \(V(s) = \mathbb{E}[R + \gamma V(s')]\) \(V(s) = \mathbb{E}[G_t]\) \(V(s) \gets V(s) + \alpha[R + \gamma V(s') - V(s)]\)
Used In Policy Iteration, Value Iteration (Ch 4) Ch 5 (learning from complete episodes) Ch 6 (most modern deep RL)
The Spectrum

DP ↔ TD ↔ MC. As you reduce bootstrapping (more steps before using a target value), you go from DP to TD(n) to MC. As \(n \to \infty\) in TD(n), you approach MC. As \(n=0\), you're pure bootstrapping (but learning from samples, not using the model).

💭 Socratic Sandbox

Deep Questions About TD Learning

Click each question to reveal the answer. These probe the intuition behind why TD works and when it's appropriate.

1
Why does TD(0) converge faster than Monte Carlo, even though both are unbiased?
▼

TD is not unbiased. MC is unbiased — the return \(G_t\) is an unbiased estimate of the true expected return. But TD(0) is biased because it uses \(V(S_{t+1})\) as part of the target. However, TD learns faster because it updates every step, while MC waits for episode termination. More frequent updates + lower variance (one-step bootstrap vs. full trajectory) = faster convergence in practice, even with the bias. The bias typically doesn't cause problems because it's small and TD is averaging over many steps.

2
In the Cliff Walking example, why does SARSA learn a safe path while Q-learning learns the risky optimal path?
▼

On-policy vs. off-policy. SARSA is on-policy: it learns the value of the policy it's following (ε-greedy exploration included). When it updates, it incorporates the risk of taking exploratory random actions. It learns "given that I might occasionally slip and fall, here's the safe strategy." Q-learning is off-policy: it learns the optimal policy regardless of how it explores. It assumes "if I ever reach this state, I'll take the best action," ignoring the fact that exploration might cause falls. In practice, Q-learning learns the "optimal" cliff-hugging policy (reward ~-25), but SARSA learns a safer path (reward ~-30 to -40) because it accounts for exploration risk.

3
What is maximization bias, and why does it matter?
▼

When you estimate many values with noise, the max is biased high. Q-learning uses \(\max_a Q(S_{t+1}, a)\) as the target. Each Q-estimate \(Q(s,a)\) has noise (estimation error). Taking the max of noisy estimates tends to select overestimated values. Over time, this bias compounds: overestimated values lead to suboptimal policy decisions. Double Q-learning fixes this by using two decoupled Q-tables — one to select the best action, another to evaluate it. This decorrelation reduces the tendency to pick overestimated values. The impact is most visible in tasks where overestimation is severe (e.g., Atari games), where Double DQN significantly outperforms DQN.

4
When would you use Expected Sarsa over Q-learning?
▼

Expected Sarsa is a compromise. It's on-policy (like SARSA) so it accounts for exploration when learning. But it averages over the action distribution instead of picking just one action, reducing variance. The result: lower variance than SARSA (good for stability) but higher bias than Q-learning (because you're averaging, not taking the best action). Use Expected Sarsa when variance is a problem (e.g., very noisy environments) and you want the stability of on-policy learning without SARSA's sensitivity to a single chosen action. In practice, with good ε, SARSA and Expected Sarsa are often similar, and Q-learning dominates if stability is not a critical issue.

5
What is the "deadly triad" and why is it a warning for deep RL?
▼

The deadly triad: (1) function approximation, (2) bootstrapping, (3) off-policy learning. When all three are combined (as in DQN), convergence guarantees break down. Function approximation generalizes values across states. Bootstrapping uses estimated targets. Off-policy learning uses a behavior policy different from the learning policy. Together, they can cause divergence or oscillation. DQN addresses this with experience replay (breaks correlation) and target networks (stabilize targets), but the fundamental challenge remains. This is why modern deep RL requires careful engineering and why convergence is not guaranteed without additional tricks. The chapter mentions this as a preview of deeper issues in later chapters.

✓ Recap

Key Takeaways: TD Learning

1. TD Bootstrapping

Core insight: Don't wait for the episode to end. Update values online using your current estimates as targets. This is bootstrapping — you're using your predictions to improve your predictions. It's biased but converges faster and with lower variance than MC.

2. TD(0) and the TD Error

The TD error \(\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)\) measures surprise. It's the difference between your bootstrapped estimate and your current prediction. Positive error → state was good → increase value. Negative error → state was bad → decrease value.

3. SARSA: On-Policy Learning

Learn the value of the policy you're following. Use the next action \(A_{t+1}\) (the one you'll actually take) in the backup. Conservative and safer because it accounts for exploration risk.

4. Q-learning: Off-Policy Learning

Learn the optimal policy independently of how you explore. Use the best next action \(\max_a Q(S_{t+1}, a)\) in the backup. More aggressive and efficient, but vulnerable to overestimation bias.

5. The Unification

DP, MC, and TD are not separate paradigms — they're points on a spectrum. DP uses a model and bootstraps fully. MC learns from samples but waits for episodes. TD learns from samples AND bootstraps, giving you the best of both. Modern deep RL is almost entirely based on TD with function approximation (DQN, PPO, etc.).
