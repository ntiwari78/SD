🎯 The Feynman Bridge

An MDP Is Like a Board Game

Imagine you're playing a strategy board game. You're sitting on a square (your state). You roll dice, consider your options, and choose a move (your action). The game rules—determined by probability—send you to a new square (a transition) and maybe you collect or lose coins (your reward). You want to maximize your total coins by the end of the game.

The Core Analogy

Your square = state \(s\). Your move = action \(a\). Game rules = transition probability \(p(s'|s,a)\). Coins gained/lost = reward \(r(s,a)\). Total coins at the end = return \(G_t\).

An MDP is a formal way to describe this game. The Markov property says: the next square only depends on your current square and your action — not on how you got there. This makes the problem mathematically tractable.

Every RL agent lives inside an MDP, though it may not know all the game rules. The agent learns by playing episodes and receiving rewards. Eventually, it figures out a good strategy (policy) — roughly: "If you're on square X, do action Y."

Three key reasons MDPs matter:

1. Formalization: We can write down equations for what we mean by "good strategy" and "future rewards."

2. Markov property: The future is conditionally independent of the past given the present state. No need to remember your entire game history — just where you are now.

3. Finite: We focus on finite state and action spaces, which lets us solve some MDPs exactly (unlike infinite spaces, covered in Part II).

📐 The Technical Ascent

The Agent-Environment Interface

At each discrete time step \(t = 0, 1, 2, \ldots\), the agent and environment interact as follows:

  1. The agent receives state \(S_t \in \mathcal{S}\) from the environment.
  2. The agent selects action \(A_t \in \mathcal{A}(S_t)\) according to its policy \(\pi\).
  3. The environment responds with:
    • Reward \(R_{t+1} \in \mathcal{R} \subset \mathbb{R}\) (one number, scalar feedback).
    • Next state \(S_{t+1} \in \mathcal{S}\).
  4. The loop repeats until a terminal state (if episodic) or forever (if continuing).

This is the fundamental Agent-Environment Loop that defines RL. Everything flows from this interaction.

Markov Property

An environment satisfies the Markov property if the probability of transitioning to state \(S_{t+1} = s'\) and receiving reward \(R_{t+1} = r\) depends only on the current state \(S_t = s\) and action \(A_t = a\), not on the entire history \(S_0, A_0, R_1, \ldots, S_{t-1}, A_{t-1}\).

Formally: \(P(S_{t+1}=s', R_{t+1}=r | S_t=s, A_t=a) = P(S_{t+1}=s', R_{t+1}=r | S_0, A_0, R_1, \ldots, S_t, A_t)\).

🧱 MDP Dynamics

Goals, Rewards, and the MDP Dynamics Function

Goals and Rewards. The agent's objective is to maximize the expected cumulative reward over time. Rewards are immediate signals (one number per step) that guide the agent toward the goal. Designing the reward function is often the hardest part of RL in practice.

🧮 Mathematical Curiosity: The Dynamics Function \(p\) ▼

The four-argument dynamics function (Eq. 3.2):

$$p(s', r \mid s, a) := P(S_{t+1} = s', R_{t+1} = r \mid S_t = s, A_t = a)$$

This is the joint probability distribution over next states and rewards, given current state and action. It completely specifies the environment's behavior. All other quantities (transition probabilities, expected rewards) can be derived from \(p\).

State-transition probabilities (Eq. 3.4):

$$p(s' \mid s, a) := P(S_{t+1} = s' \mid S_t = s, A_t = a) = \sum_{r \in \mathcal{R}} p(s', r \mid s, a)$$

The probability of transitioning to \(s'\) from \(s\) via action \(a\), marginalizing over all possible rewards.

Expected rewards (Eq. 3.5):

$$r(s, a) := \mathbb{E}[R_{t+1} \mid S_t = s, A_t = a] = \sum_{r \in \mathcal{R}} r \sum_{s' \in \mathcal{S}} p(s', r \mid s, a)$$

The expected immediate reward for taking action \(a\) in state \(s\).

These functions define everything about how the environment works. With \(p(s', r | s, a)\) in hand, we can compute:

  • How likely an action is to succeed or fail.
  • What rewards to expect immediately.
  • What states we'll end up in.

Most real environments are unknown — the agent must learn them (model-based RL) or ignore them entirely (model-free RL).

📊 Returns & Episodes

The Return, Episodes, and Unified Notation

The agent's goal is to maximize the expected return — the cumulative future reward. But how do we define "future"?

🧮 Mathematical Curiosity: Computing Returns ▼

Episodic tasks terminate at a natural endpoint (e.g., a chess game ends when someone wins). For a task that ends at time \(T\):

$$G_t := R_{t+1} + R_{t+2} + \cdots + R_T$$

Continuing tasks never terminate (e.g., a thermostat agent that runs forever). For these, we discount future rewards to avoid infinite returns (Eq. 3.8):

$$G_t := R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$

where \(\gamma \in [0, 1]\) is the discount factor:

  • \(\gamma = 0\): Agent is myopic — only cares about immediate reward.
  • \(\gamma = 1\): Agent is far-sighted — all future rewards weighted equally.
  • \(\gamma = 0.9\) or \(0.99\): Typical choice — values nearer rewards more.

Key recursion (Eq. 3.9):

$$G_t = R_{t+1} + \gamma G_{t+1}$$

The return at time \(t\) is the immediate reward plus the discounted return from the next step. This recursive structure is the foundation of dynamic programming and temporal-difference learning.

Unified notation: We can write episodic and continuing tasks with a single formula by defining a "terminal reward" state with \(V(s_{\text{terminal}}) = 0\). Both cases then use \(G_t = R_{t+1} + \gamma G_{t+1}\).

The discount factor \(\gamma\) is a critical hyperparameter. Small \(\gamma\) makes the agent greedy (short-term focus). Large \(\gamma\) makes it sacrifice now for distant rewards.

📝 Policies & Values

Policies and Value Functions

A policy \(\pi\) is the agent's strategy — a mapping from states to actions (or probabilities over actions). A value function estimates how good a state is under that policy.

🧮 Mathematical Curiosity: Value Functions ▼

Deterministic policy: \(\pi(s) = a\) — always take a specific action in state \(s\).

Stochastic policy: \(\pi(a|s) = P(A_t = a | S_t = s)\) — probability of taking action \(a\) in state \(s\). Useful for exploration or when the optimal policy is randomized.

State-value function (Eq. 3.12):

$$v_\pi(s) := \mathbb{E}_\pi [G_t \mid S_t = s] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \bigg| S_t = s \right]$$

The expected return starting from state \(s\) and following policy \(\pi\). "How good is it to be in this state, given my policy?"

Action-value function (Q-function) (Eq. 3.13):

$$q_\pi(s, a) := \mathbb{E}_\pi [G_t \mid S_t = s, A_t = a] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \bigg| S_t = s, A_t = a \right]$$

The expected return if you take action \(a\) in state \(s\), then follow policy \(\pi\). "How good is this state-action pair?"

Relationship: \(v_\pi(s) = \sum_a \pi(a|s) q_\pi(s, a)\) — the state value is the policy-weighted average of all action values.

Computing value functions is the core of RL. Once we know V(s) or Q(s,a), we can act greedily: always pick the action with the highest value. This is called policy improvement (Chapter 4).

⚡ Bellman Equations

Optimal Policies and Bellman Optimality

The Bellman equation is the cornerstone of dynamic programming and RL. It says that the value of a state can be recursively expressed in terms of the values of successor states.

🧮 Mathematical Curiosity: Bellman Equations ▼

Bellman equation for \(v_\pi\) (Eq. 3.14):

$$v_\pi(s) = \sum_a \pi(a|s) \sum_{s',r} p(s',r|s,a) [r + \gamma v_\pi(s')]$$

The value of state \(s\) under policy \(\pi\) is: the policy's probability of each action, times the sum over all transitions of (reward + discounted value of next state). It's recursive: V(s) depends on V(s').

Bellman optimality for V* (Eq. 3.19):

$$v_*(s) = \max_a \sum_{s',r} p(s',r|s,a) [r + \gamma v_*(s')]$$

The optimal state value is the maximum over all actions. Take the greedy action and recurse. This defines optimal value.

Bellman optimality for Q* (Eq. 3.20):

$$q_*(s, a) = \sum_{s',r} p(s',r|s,a) [r + \gamma \max_{a'} q_*(s',a')]$$

The optimal action value: take action \(a\), collect the reward and transition, then pick the best next action. The key insight for Q-learning!

Optimal policy: \(\pi_*(s) = \arg\max_a q_*(s,a)\) — always take the action with the highest optimal Q-value.

These equations are not algorithms yet—they're circular definitions. But they give us a target to approximate, and they're the foundation for solving MDPs exactly (e.g., value iteration, policy iteration in Chapter 4).

💨 Comparative Analysis

Episodic vs. Continuing Tasks

MDPs can be classified by whether they terminate or run forever. This affects how we compute returns and structure our algorithms.

Aspect Episodic Tasks Continuing Tasks
Definition Task ends at a terminal state (e.g., game ends when you win/lose) Task runs indefinitely (e.g., robotic control, trading agent)
Return formula \(G_t = R_{t+1} + R_{t+2} + \cdots + R_T\) (sum to terminal) \(G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots\) (infinite with discount)
Discount factor γ Can be 1 (all future rewards equal weight until terminal) Must be < 1 (else \(\sum \gamma^k R_k = \infty\) may not converge)
Value interpretation Expected score until the task ends Expected infinite stream of rewards (weighted by discount)
Examples Chess, Go, Tic-Tac-Toe, reaching a goal position Inventory management, power grid control, autonomous vehicles
Algorithm adaptation Update values after episode ends. Exploration is decoupled. Update values continuously. Exploration must be managed (e.g., ε-greedy)
Unified notation trick Add a dummy absorbing terminal state with V = 0. Then both use \(G_t = R_{t+1} + \gamma G_{t+1}\) Both cases fit into the same recursive formula
💻 Algorithm Implementation

Iterative Policy Evaluation on a Gridworld

Below is a Python implementation of iterative policy evaluation (Algorithm 4.1 preview), which computes the value function for a given policy by repeatedly applying the Bellman equation.

Python / NumPy 
import numpy as np

class GridworldMDP:
    """
    A finite MDP: 4x4 gridworld.
    State: (row, col) position. Goal: (3, 3).
    Actions: up, down, left, right (deterministic transitions).
    Reward: +1 for reaching goal, -0.01 for each step.
    """

    def __init__(self, gamma=0.9, goal_reward=1.0, step_cost=0.01):
        self.gamma = gamma                    # Discount factor
        self.goal_reward = goal_reward
        self.step_cost = step_cost
        self.nrows, self.ncols = 4, 4
        self.goal = (3, 3)
        self.start = (0, 0)

        # All 16 states
        self.states = [(r, c) for r in range(self.nrows)
                              for c in range(self.ncols)]
        self.state_to_idx = {s: i for i, s in enumerate(self.states)}
        self.n_states = len(self.states)

        # 4 cardinal actions
        self.actions = [(-1, 0), (1, 0), (0, -1), (0, 1)]  # up, down, left, right
        self.n_actions = len(self.actions)

    def get_next_state(self, state, action_idx):
        """Deterministic transition: move in the chosen direction."""
        r, c = state
        dr, dc = self.actions[action_idx]
        nr, nc = r + dr, c + dc

        # Boundary check: stay in place if you hit a wall
        if not (0 <= nr < self.nrows and 0 <= nc < self.ncols):
            return state

        return (nr, nc)

    def get_reward(self, state, action_idx, next_state):
        """
        Reward = goal_reward if we just reached the goal,
                 else -step_cost for taking a step.
        """
        if next_state == self.goal:
            return self.goal_reward
        else:
            return -self.step_cost

    def uniform_random_policy(self):
        """
        Return shape (n_states, n_actions):
        pi(a|s) = 1/n_actions for all a, s.
        """
        return np.ones((self.n_states, self.n_actions)) / self.n_actions

    def iterative_policy_evaluation(self, policy, theta=1e-4, max_iters=1000):
        """
        Policy Evaluation (Algorithm 4.1).

        Input:
          policy: shape (n_states, n_actions), policy(s, a) = pi(a|s)
          theta: convergence threshold for max change in V

        Output:
          V: value function, shape (n_states,), V[s] = v_pi(s)
        """
        V = np.zeros(self.n_states)

        for iteration in range(max_iters):
            delta = 0  # Track largest change in V

            for s_idx, state in enumerate(self.states):
                # ────────────────────────────────────────────────────────
                # Bellman equation: v_pi(s) = Sum_a pi(a|s) * Sum_{s',r}
                #   p(s',r | s,a) * [r + gamma * v_pi(s')]
                # ────────────────────────────────────────────────────────

                old_v = V[s_idx]
                new_v = 0.0

                # Iterate over actions weighted by policy
                for action_idx in range(self.n_actions):
                    prob_action = policy[s_idx, action_idx]  # pi(a|s)

                    # Deterministic environment: one next state per action
                    next_state = self.get_next_state(state, action_idx)
                    next_s_idx = self.state_to_idx[next_state]

                    # Reward for this transition
                    reward = self.get_reward(state, action_idx, next_state)

                    # Bellman backup:
                    # v_pi(s) += pi(a|s) * [r + gamma * v_pi(s')]
                    new_v += prob_action * (reward + self.gamma * V[next_s_idx])

                V[s_idx] = new_v
                delta = max(delta, abs(old_v - new_v))

            # Check convergence
            if delta < theta:
                print(f"Converged in {iteration + 1} iterations (delta={delta:.6f})")
                break

        return V

    def print_value_grid(self, V):
        """Pretty-print the 4x4 value grid."""
        print("\nValue Function (Gridworld):")
        for r in range(self.nrows):
            for c in range(self.ncols):
                s_idx = self.state_to_idx[(r, c)]
                print(f"{V[s_idx]:7.3f}", end="  ")
            print()


# ════════════════════════════════════════════════════════════════
# USAGE: Create an MDP, set a policy, evaluate it
# ════════════════════════════════════════════════════════════════

if __name__ == '__main__':
    # Create a gridworld with gamma=0.9, goal reward=1, step cost=0.01
    mdp = GridworldMDP(gamma=0.9, goal_reward=1.0, step_cost=0.01)

    # Uniform random policy: 25% each direction in every state
    policy = mdp.uniform_random_policy()

    # Evaluate this policy
    V = mdp.iterative_policy_evaluation(policy, theta=1e-4)

    # Print the result
    mdp.print_value_grid(V)

    print(f"\nValue at start (0,0): {V[mdp.state_to_idx[(0, 0)]]:.4f}")
    print(f"Value at goal (3,3): {V[mdp.state_to_idx[(3, 3)]]:.4f}")
    print(f"Value at (1,1): {V[mdp.state_to_idx[(1, 1)]]:.4f}")
🧪 Socratic Sandbox

Debug the MDP Agent

A team is trying to deploy a robot in a warehouse. The robot must navigate a gridworld to collect items and return. You observe several issues with the learned value function. Let's diagnose them together.

Scenario: The Confused Robot

The robot's value function says that state (0,0) has value 0.05, and state (3,3) (the goal) also has value 0.05. But (3,3) should be much more valuable! The goal rewards +1, each step costs -0.01. What's wrong?

1 Why might the goal state (3,3) have such a low value (0.05), despite yielding +1 reward? ▼

The issue is likely the value of the goal state itself. Once the robot reaches (3,3), what happens next? If it's not a terminal state, the robot must take another action—maybe it bounces back into the gridworld and incurs more step costs. If the episode doesn't end at the goal, the value of (3,3) itself is only as good as what comes after. If the policy repeatedly moves it away from the goal (e.g., bouncing it back), then V(goal) can indeed be low. The solution: either make the goal a terminal state (episode ends), or design the reward to incentivize staying near the goal.

2 The team set \(\gamma = 0.5\). How does this affect the value landscape? Should they increase or decrease it? ▼

A low \(\gamma = 0.5\) makes the agent myopic — future rewards beyond 2-3 steps are nearly ignored. For a gridworld where the goal might be 6 steps away, \(\gamma = 0.5\) means the robot doesn't "care" about reaching it (rewards get discounted away). They should increase \(\gamma\) to ~0.9 or 0.99 so the robot values states many steps in the future. The tradeoff: higher \(\gamma\) also makes learning unstable if the reward function has noise, and it values long-term rewards more (which may not be desired if the task is short).

3 The robot uses a uniform random policy (equal probability for all actions). Is this the right policy for learning values? What would happen with a better policy? ▼

A uniform random policy is fine for policy evaluation — you can evaluate any policy. But it's a terrible policy for the robot to act with! A random policy will wobble around inefficiently; it will rarely reach the goal. This is why Chapter 4 introduces policy improvement: once you've evaluated the current policy, compute a better one by acting greedily with respect to the values. With iterative policy evaluation + improvement (policy iteration), the agent converges to an optimal policy that goes straight to the goal.

4 The reward for each step is -0.01. What happens if you increase this cost to -1? Will the value function change? Why? ▼

The value function will change dramatically. Rewards directly affect values via the Bellman equation: \(v_\pi(s) = \sum_a \pi(a|s) \sum_{s',r} p(s',r|s,a) [r + \gamma v_\pi(s')]\). Higher step costs mean lower values everywhere (since you pay to take steps). Specifically, states far from the goal become increasingly negative, and the agent is incentivized to reach the goal quickly. In the extreme (cost = -∞), the agent would rather take no steps at all. Reward shaping is a crucial design choice in RL.

5 A colleague suggests: "Let's make the goal state absorbing (no further transitions possible)." How does this change the Bellman equation for (3,3)? ▼

If (3,3) is absorbing, its value becomes \(v_\pi(3,3) = \mathbb{E}[R | S=\text{goal}] + \gamma \cdot v_\pi(\text{goal})\). Since it doesn't transition anywhere (stays in itself), we have \(v_\pi(\text{goal}) = 1 + \gamma \cdot v_\pi(\text{goal})\), which solves to \(v_\pi(\text{goal}) = \frac{1}{1-\gamma}\). For \(\gamma=0.9\), that's \(\frac{1}{0.1} = 10\)! A much higher value, appropriately reflecting that you're in a terminal state where you can't be punished further. This is the "standard trick" for episodic tasks mentioned in the textbook.
