🌉 Feynman Bridge

The Wobbling Stool

Imagine a three-legged stool. Each leg is individually strong and useful:

  • Leg 1 (Generalization): You can't store a value for every state, so you use function approximation to generalize from experience.
  • Leg 2 (Learning from Estimates): You use bootstrapping—learning from your own value estimates—to speed up learning and reduce variance.
  • Leg 3 (Learning from Others): You learn off-policy, using experience from other behavior policies to improve a target policy faster.

Each alone works beautifully. Two legs together? Still stable. But all three together, and the stool wobbles catastrophically. Your learning can diverge to infinity, weights exploding, values becoming meaningless.

This is the Deadly Triad: function approximation + bootstrapping + off-policy training = instability.

Why the Analogy Works

In tabular methods (each state/action pair gets its own value), the stool is stable because bootstrapping from one cell doesn't affect neighboring cells. Off-policy corrections work cleanly. But with function approximation, your value function ties together state estimates through a shared neural network. Change one weight to fix one estimate, and you break estimates everywhere else—especially under off-policy corrections that can be far from the data you used to train.

⛰️ Technical Ascent

The Deadly Triad: A Deep Dive

What Are The Three Components?

1. Function Approximation: Using a learned function (e.g., a neural network) to estimate values $$V(s) \approx \hat{v}(s, \mathbf{w})$$ instead of storing a table. This is necessary for large state spaces.

2. Bootstrapping: Using your own estimate of future value in the Bellman update:

Bootstrap Equation ▼
$$\Delta \hat{v}(s, \mathbf{w}) \propto [R + \gamma \hat{v}(s', \mathbf{w}) - \hat{v}(s, \mathbf{w})] \nabla \hat{v}(s, \mathbf{w})$$

The target $$R + \gamma \hat{v}(s', \mathbf{w})$$ depends on weights $$\mathbf{w}$$, making the target a moving function of the parameters.

3. Off-Policy Training: Learning a policy $$\pi$$ from data generated by a different policy $$\beta$$, often using importance sampling or other corrections.

Why Together They Diverge

When you combine all three:

  • Function approximation creates aliases: updating the value of one state changes the values of other states through shared weights.
  • Bootstrapping amplifies these aliases: using a moving target that depends on the same weights you're updating.
  • Off-policy corrections add a bias term that can point in a completely different direction than the actual gradients, causing divergence.

The result: the Bellman error can grow unboundedly, and the value function diverges.

Bellman Error vs. Projected Bellman Error

Bellman Error: $$\|\mathcal{B}^\pi \hat{V} - \hat{V}\|$$, where $$\mathcal{B}^\pi$$ is the Bellman operator. This can diverge with the deadly triad.

Projected Bellman Error: $$\|\Pi \mathcal{B}^\pi \hat{V} - \hat{V}\|$$, where $$\Pi$$ projects back onto the function space. This is what gradient descent on the Mean Squared Bellman Error (MSBE) actually minimizes, and it's more stable.

Key Insight

The Bellman error is not directly learnable with gradient descent on the function approximator—you'd need the true target, not your estimate. What you actually minimize is the projected Bellman error, which is why gradient-TD methods work: they explicitly target this objective.

Baird's Counterexample: Semi-Gradient TD Diverging

This classic counterexample, devised by Leemon Baird, shows semi-gradient TD diverging with linear function approximation, off-policy learning, and bootstrapping. The setup is a simple 7-state MDP designed to expose the deadly triad.

Why This Matters

Baird's counterexample is not an academic curiosity—it directly shows that even simple, theoretically sound algorithms fail when all three components are present. It's a wake-up call that standard temporal-difference learning needs modification for off-policy, function-approximated cases.

python 
import numpy as np
import matplotlib.pyplot as plt

class BairdsCounterexample:
    """
    Baird's counterexample: Semi-gradient TD with function approximation,
    bootstrapping, and off-policy learning DIVERGES.

    7-state MDP with linear function approximation.
    Behavior policy: uniform over all actions.
    Target policy: deterministic.
    """

    def __init__(self, alpha=0.01):
        self.alpha = alpha
        # Linear approximation: V(s) = sum(w[i] * phi[s][i])
        # 8 features (7 states + 1 bias)
        self.w = np.zeros(8)

        # Feature vectors for each state
        self.phi = np.array([
            [1, 0, 0, 0, 0, 0, 0, 1],  # state 0
            [0, 1, 0, 0, 0, 0, 0, 1],  # state 1
            [0, 0, 1, 0, 0, 0, 0, 1],  # state 2
            [0, 0, 0, 1, 0, 0, 0, 1],  # state 3
            [0, 0, 0, 0, 1, 0, 0, 1],  # state 4
            [0, 0, 0, 0, 0, 1, 0, 1],  # state 5
            [0, 0, 0, 0, 0, 0, 1, 2],  # state 6 (special state)
        ])

        self.weight_history = []

    def value(self, state):
        """Approximate value of a state."""
        return np.dot(self.w, self.phi[state])

    def update(self, state, action, reward, next_state, gamma=0.99):
        """
        Semi-gradient TD update.
        This is off-policy: we learn the target policy from behavior policy experience.
        """
        v_s = self.value(state)
        v_s_next = self.value(next_state)

        # TD target (using bootstrap)
        td_target = reward + gamma * v_s_next
        td_error = td_target - v_s

        # Semi-gradient update (gradient w.r.t. current state only, not bootstrap target)
        self.w += self.alpha * td_error * self.phi[state]

    def episode(self, max_steps=100):
        """Run one episode of Baird's counterexample."""
        state = np.random.randint(0, 7)  # Random start

        for step in range(max_steps):
            # Behavior policy: uniform random action
            action = np.random.randint(0, 2)

            # Transition: simplified for illustration
            if state == 6:
                next_state = np.random.randint(0, 6)  # Terminal state leads to random state
                reward = 0
            else:
                next_state = 6  # All non-terminal states go to 6
                reward = 0

            # Off-policy importance sampling (behavior ~= target, so rho ≈ 1, but mismatch exists)
            self.update(state, action, reward, next_state)

            # Record weight magnitude for tracking divergence
            if step == max_steps - 1:
                self.weight_history.append(np.linalg.norm(self.w))

            state = next_state

    def train(self, episodes=500):
        """Train the agent and track divergence."""
        for ep in range(episodes):
            self.episode()
            if (ep + 1) % 50 == 0:
                print(f"Episode {ep+1}: ||w|| = {self.weight_history[-1]:.4f}")

        return self.weight_history


# Run the counterexample
print("Baird's Counterexample: Semi-Gradient TD Divergence")
print("=" * 50)

agent = BairdsCounterexample(alpha=0.01)
history = agent.train(episodes=500)

# Plot results
plt.figure(figsize=(10, 6))
plt.semilogy(history, color='#fb7185', linewidth=2.5, label='Weight Magnitude ||w||')
plt.xlabel('Episode / 50', fontsize=12)
plt.ylabel('Weight Magnitude (log scale)', fontsize=12)
plt.title('Baird\'s Counterexample: Semi-Gradient TD Diverges', fontsize=14, fontweight='bold')
plt.grid(True, alpha=0.2, color='white')
plt.legend(fontsize=11)
plt.tight_layout()
plt.savefig('bairds_divergence.png', dpi=150, facecolor='#0a0e1a', edgecolor='none')
plt.show()

print("\nKey Observation:")
print(f"Starting weight magnitude: {history[0]:.6f}")
print(f"Final weight magnitude:    {history[-1]:.6f}")
print(f"Growth factor: {history[-1] / (history[0] + 1e-10):.1f}x")
print("\nWithout function approximation, bootstrapping, or off-policy,")
print("the algorithm would converge. All three together = divergence.")
What This Shows

The algorithm diverges because semi-gradient TD only updates the value of the current state, ignoring how the target $$\gamma V(s')$$ changes with the same parameters. With off-policy corrections and function approximation aliases, this asymmetry causes instability. The divergence is not a bug in implementation—it's a fundamental issue with the algorithm itself.

✓ Solutions & Advances

Taming the Deadly Triad

Gradient-TD Methods (GTD, GTD2, TDC)

Gradient-TD methods solve the problem by minimizing the projected Bellman error instead of the Bellman error. This requires a second set of weights (auxiliary memory) to track the gradient:

GTD Update ▼
$$\mathbf{w} \leftarrow \mathbf{w} + \alpha \delta \mathbf{v} - \alpha \gamma (1 - \lambda) (\mathbf{v}^T \mathbf{v}) \mathbf{w}$$
$$\mathbf{v} \leftarrow \mathbf{v} + \beta (\delta - \mathbf{v}^T \boldsymbol{\phi}) \boldsymbol{\phi}$$

The auxiliary weights $$\mathbf{v}$$ track the gradient direction, making the update stable even with function approximation and off-policy learning.

Key advantages:

  • Guaranteed stability even with off-policy learning and function approximation.
  • Uses only one additional vector of weights (auxiliary learning).
  • Slower convergence than semi-gradient TD, but more stable.

Variants:

  • TD with Gradient Correction (TDC): The simplest variant. Uses the auxiliary weights to correct the semi-gradient update.
  • GTD2: Focuses on minimizing mean squared projected Bellman error directly.
  • Emphatic TD: Uses importance sampling weights (emphasis weights) to reduce off-policy bias without requiring auxiliary weights.

DQN: How Deep Reinforcement Learning Tames the Triad

Deep Q-Networks (DQN) doesn't use gradient-TD. Instead, it breaks the deadly triad through three architectural tricks:

1. Target Networks: Keep a separate, slowly-updated copy of the value function to provide stable targets. This decouples the current weights from the bootstrap target.

2. Experience Replay: Store past transitions and sample them uniformly during training. This breaks the on-policy assumption and reduces correlation between consecutive updates, stabilizing learning.

3. Reward Clipping: Bound rewards to a small range, reducing the variance of targets and preventing extreme value estimates.

Why These Work

Target networks reduce the coupling between current parameters and bootstrap targets—you're not updating against a moving target from the same weights. Experience replay provides approximately i.i.d. samples, making semi-gradient updates safer. Clipping reduces variance. Together, they break the dangerous three-way interaction without explicitly minimizing projected Bellman error.

        self.q_network = NeuralNetwork(state_size, action_size)
        # Target network (copy of q_network, updated slowly)
        self.target_network = NeuralNetwork(state_size, action_size)
        # Experience replay buffer
        self.replay_buffer = ReplayBuffer(capacity=100000)

    def remember(self, state, action, reward, next_state, done):
        """Store transition in replay buffer."""
        self.replay_buffer.add((state, action, reward, next_state, done))

    def learn_from_replay(self, batch_size=32):
        """Sample a batch from replay buffer and learn."""
        # STRATEGY 1: Experience Replay (breaks correlation)
        batch = self.replay_buffer.sample(batch_size)

        for state, action, reward, next_state, done in batch:
            # STRATEGY 2: Target Network (stable target)
            if done:
                target = reward
            else:
                # Use TARGET network, not current q_network
                next_q = self.target_network.predict(next_state)
                # STRATEGY 3: Clipping reward
                target = np.clip(reward, -1, 1) + 0.99 * np.max(next_q)

            self.q_network.train_step(loss)

    def update_target_network(self):
        """Every C steps, copy main network to target network."""
        self.target_network.weights = self.q_network.weights.copy()

                next_state, reward, done = env.step(action)

                state = next_state
                self.steps += 1

DQN's success story: By carefully orchestrating these three techniques, DQN made deep Q-learning practical for large state spaces (Atari), avoiding the deadly triad's divergence without needing auxiliary weights like gradient-TD methods.

Questions to Deepen Understanding

Explore these questions to solidify your grasp of the deadly triad and its solutions.

Deadly Triad Deep Dive

1. Why does the deadly triad NOT occur in tabular Q-learning (before function approximation)?
In a tabular representation, each state-action pair has its own independent value. When you update Q(s,a), it doesn't directly affect Q(s',a') unless they're the same state-action pair. Function approximation ties states together through shared weights, creating aliases and feedback loops.
2. If bootstrapping is so useful (reduces variance), why don't we just remove off-policy learning instead of using gradient-TD?
Off-policy learning is very valuable—it lets you learn from exploratory experience or other agents' data without re-running expensive simulations. Removing it would limit sample efficiency. Gradient-TD keeps both: the sample efficiency of off-policy and bootstrapping, while maintaining stability through a different optimization objective.
3. What is the key difference between what semi-gradient TD minimizes and what GTD minimizes?
Semi-gradient TD (and standard TD) minimizes the Bellman error: $$\|B V - V\|$$. GTD minimizes the projected Bellman error: $$\|\Pi B V - V\|$$, where $$\Pi$$ projects back onto the function space. The projection ensures stability even when the Bellman operator would diverge.
4. Why does experience replay (random sampling) help stabilize learning with the deadly triad?
Experience replay removes the temporal correlation between consecutive updates. Semi-gradient TD works better on approximately i.i.d. samples because the alias effects don't compound in the same direction across consecutive steps. Random sampling also reduces the distribution shift between behavior and target policies locally.
5. DQN uses a target network instead of GTD's auxiliary weights. What is DQN trading off?
Target networks reduce the coupling between current weights and bootstrap targets by using an older copy. GTD directly targets the projected Bellman error with auxiliary weights. DQN's approach is more practical for deep learning (no auxiliary weights), but requires manual synchronization and might miss important value updates. GTD is more principled but adds complexity. Both solve the deadly triad; DQN just does it differently.
6. Could you use emphatic TD weights instead of experience replay to handle off-policy learning?
Yes! Emphatic TD(0) uses importance sampling weights based on the trace of importance ratios to down-weight updates from data far from the target policy. This is an alternative to experience replay, but it's often less practical because the weights can be high-variance, and you still need to process data in temporal order.
7. Why does Baird's counterexample diverge with learning rate 0.01 but might stabilize with a smaller learning rate?
A very small learning rate can slow down the accumulation of error and aliases, allowing the TD error signal to occasionally point the right direction. However, you'll eventually diverge given enough episodes—it's not a true solution, just a delay. The fundamental issue persists; you need a different algorithm, not just smaller steps.
8. If gradient-TD is provably stable, why do practitioners use DQN instead?
Gradient-TD requires auxiliary weights, adding memory and computational overhead. DQN's tricks (target networks, replay, clipping) are simpler to implement in modern deep learning frameworks and work well in practice. DQN is also easier to parallelize. However, for some problems (especially deep RL with high off-policy ratio), gradient-TD and related methods (e.g., emphatic methods) can outperform DQN.
