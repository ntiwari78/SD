Chapter 10: Tuples — Immutable Sequences

Discover the power of immutability. Tuples are ordered sequences that cannot be modified after creation, making them ideal for protecting data and using as dictionary keys.

🧠 Mental Model: Tuples as Locked Boxes

Think of a tuple like a locked gift box:

  • You can open it and look inside (read elements)
  • You can see what's inside (access by index)
  • But you cannot rearrange, add, or remove items once it's sealed
  • You can compare boxes and put boxes inside other boxes (nesting)
  • The lock protects the contents from accidental modification

Why this matters: Immutability means you can pass tuples between functions with confidence that they won't be accidentally modified. It also makes tuples hashable, so they can be dictionary keys.

What are Tuples?

A tuple is an immutable, ordered sequence of elements in Python. Once created, you cannot add, remove, or modify elements. Tuples are created using parentheses () and are perfect for protecting data.

Creating Tuples

# Basic tuple creation
fruits = ("apple", "banana", "cherry")
print(fruits)  # ('apple', 'banana', 'cherry')

# Empty tuple
empty = ()
print(len(empty))  # 0

# Single-element tuple (note the trailing comma!)
single = ("only",)
print(single)  # ('only',)

# Without comma, it's just a string in parentheses
not_tuple = ("string")
print(type(not_tuple))  # <class 'str'>

# Creating from other sequences
my_list = [1, 2, 3]
as_tuple = tuple(my_list)
print(as_tuple)  # (1, 2, 3)

# Tuples can contain mixed types
mixed = (42, "hello", 3.14, True)
print(mixed)  # (42, 'hello', 3.14, True)
Key Point

The trailing comma is crucial for single-element tuples: ("item",) creates a tuple, while ("item") creates a string. Always remember the comma!

Accessing Tuple Elements

Tuples are indexed just like lists, starting from 0. You can access individual elements, slice tuples, and use negative indexing.

Indexing and Slicing

colors = ("red", "green", "blue", "yellow", "purple")

# Positive indexing (left to right, starting from 0)
print(colors[0])    # red
print(colors[2])    # blue
print(colors[-1])   # purple
print(colors[-2])   # yellow

# Slicing
print(colors[1:3])      # ('green', 'blue')
print(colors[:2])       # ('red', 'green')
print(colors[2:])       # ('blue', 'yellow', 'purple')
print(colors[::2])      # ('red', 'blue', 'purple')
print(colors[::-1])     # ('purple', 'yellow', 'blue', 'green', 'red')

Example: Extracting Coordinates

# Working with coordinate tuples
point = (10, 20, 30)
x, y, z = point  # Tuple unpacking

print(f"X: {x}, Y: {y}, Z: {z}")  # X: 10, Y: 20, Z: 30

# Slicing ranges
coordinates = ((1, 2), (3, 4), (5, 6), (7, 8))
first_two = coordinates[:2]
print(first_two)  # ((1, 2), (3, 4))

Deep Dive: Tuple Unpacking

▼

Tuple unpacking is a powerful feature that assigns multiple tuple elements to variables in one operation:

# Basic unpacking
a, b, c = (1, 2, 3)

# Extended unpacking with *
first, *middle, last = (1, 2, 3, 4, 5)
print(first, middle, last)  # 1 [2, 3, 4] 5

# Ignoring values with _
x, _, z = (10, 20, 30)
print(x, z)  # 10 30

# Nested unpacking
(a, b), (c, d) = ((1, 2), (3, 4))
print(a, b, c, d)  # 1 2 3 4

Looping in Tuples

Iterate through tuple elements using for loops, enumerate(), or zip().

colors = ("red", "green", "blue")

# Simple iteration
for color in colors:
    print(color)
# Output: red, green, blue

# With enumerate() - get index and value
for i, color in enumerate(colors):
    print(f"{i}: {color}")
# Output: 0: red, 1: green, 2: blue

# Multiple tuples with zip()
names = ("Alice", "Bob", "Charlie")
ages = (25, 30, 35)

for name, age in zip(names, ages):
    print(f"{name} is {age} years old")
# Alice is 25 years old
# Bob is 30 years old
# Charlie is 35 years old

Example: Processing Tuples with enumerate()

scores = (85, 92, 78, 95, 88)

for index, score in enumerate(scores):
    if score >= 90:
        print(f"Student {index} scored {score} - Excellent!")
    else:
        print(f"Student {index} scored {score}")

Basic Tuple Operations

Concatenation and Repetition

# Concatenation with +
t1 = (1, 2, 3)
t2 = (4, 5, 6)
combined = t1 + t2
print(combined)  # (1, 2, 3, 4, 5, 6)

# Repetition with *
repeated = ("x", "y") * 3
print(repeated)  # ('x', 'y', 'x', 'y', 'x', 'y')

# Membership testing with 'in'
t = ("apple", "banana", "cherry")
print("banana" in t)      # True
print("grape" in t)       # False
print("grape" not in t)   # True

# Length
print(len((1, 2, 3)))  # 3

# Comparison
print((1, 2) < (1, 3))       # True
print((1, 2) == (1, 2))      # True
print((1, 2, 3) > (1, 2))    # True
Operation Example Result
Concatenation (1, 2) + (3, 4) (1, 2, 3, 4)
Repetition (1, 2) * 3 (1, 2, 1, 2, 1, 2)
Length len((1, 2, 3)) 3
Membership 2 in (1, 2, 3) True

Using Built-in Functions

numbers = (5, 2, 8, 1, 9, 3)

# Length
print(len(numbers))        # 6

# Min and max
print(min(numbers))        # 1
print(max(numbers))        # 9

# Sum (numbers only)
print(sum(numbers))        # 28

# Sorted (returns a list!)
print(sorted(numbers))     # [1, 2, 3, 5, 8, 9]
print(sorted(numbers, reverse=True))  # [9, 8, 5, 3, 2, 1]

# Convert to tuple
my_list = [1, 2, 3]
as_tuple = tuple(my_list)
print(as_tuple)            # (1, 2, 3)
Note

sorted() returns a list, not a tuple. If you need a tuple, wrap it: tuple(sorted(my_tuple))

Tuple Methods

Tuples have only two methods since they're immutable:

count() and index()

numbers = (1, 2, 3, 2, 4, 2, 5)

# count() - returns number of occurrences
print(numbers.count(2))     # 3
print(numbers.count(99))    # 0

# index() - returns first index of value
print(numbers.index(2))     # 1
print(numbers.index(4))     # 4

# index() raises ValueError if not found
try:
    print(numbers.index(99))
except ValueError:
    print("99 not found in tuple")
Method Purpose Example
count(x) Count occurrences of x (1, 2, 1, 1).count(1) → 3
index(x) Find first index of x (1, 2, 3).index(2) → 1

Tuple Varieties

Nested Tuples

# Tuples containing tuples
matrix = ((1, 2, 3),
          (4, 5, 6),
          (7, 8, 9))

print(matrix[0])         # (1, 2, 3)
print(matrix[0][1])      # 2
print(matrix[2][2])      # 9

# Tuples of lists
data = ([1, 2], [3, 4], [5, 6])
data[0][1] = 99  # Modify list inside tuple
print(data)  # ([1, 99], [3, 4], [5, 6])

Named Tuples

from collections import namedtuple

# Define a named tuple type
Point = namedtuple('Point', ['x', 'y'])
Person = namedtuple('Person', ['name', 'age', 'email'])

# Create instances
p = Point(10, 20)
print(p.x, p.y)         # 10 20
print(p[0], p[1])       # 10 20 (still indexable)

# With Person
alice = Person('Alice', 30, 'alice@example.com')
print(alice.name)       # Alice
print(alice.age)        # 30

# Named tuples are still immutable
try:
    alice.age = 31
except AttributeError:
    print("Cannot modify named tuple")

Deep Dive: Named Tuples vs Dataclasses

▼

Named tuples are lightweight and immutable:

from collections import namedtuple
from dataclasses import dataclass

# Named tuple - immutable, lightweight
Point_NT = namedtuple('Point', ['x', 'y'])
p = Point_NT(1, 2)
# p.x = 5  # TypeError!

# Dataclass - mutable, more features
@dataclass
class Point_DC:
    x: int
    y: int

p = Point_DC(1, 2)
p.x = 5  # OK

# Performance: named tuples are faster
import timeit
nt = Point_NT(1, 2)
dc = Point_DC(1, 2)

print(timeit.timeit(lambda: (nt.x, nt.y), number=100000))
print(timeit.timeit(lambda: (dc.x, dc.y), number=100000))

Use named tuples when: You need immutability and lightweight objects. Use dataclasses when: You need mutability and more features.

Programs from the Book

Example 1: Coordinate System

def distance(p1, p2):
    """Calculate distance between two 2D points"""
    x1, y1 = p1
    x2, y2 = p2
    return ((x2 - x1)**2 + (y2 - y1)**2)**0.5

point1 = (0, 0)
point2 = (3, 4)
print(distance(point1, point2))  # 5.0

Example 2: RGB Color Tuple

def blend_colors(color1, color2, ratio=0.5):
    """Blend two RGB colors"""
    r1, g1, b1 = color1
    r2, g2, b2 = color2

    r = int(r1 * (1 - ratio) + r2 * ratio)
    g = int(g1 * (1 - ratio) + g2 * ratio)
    b = int(b1 * (1 - ratio) + b2 * ratio)

    return (r, g, b)

red = (255, 0, 0)
blue = (0, 0, 255)
purple = blend_colors(red, blue, 0.5)
print(purple)  # (127, 0, 127)

Example 3: Multiple Return Values

def analyze_numbers(numbers):
    """Return min, max, and average"""
    return (min(numbers), max(numbers), sum(numbers) / len(numbers))

data = (5, 2, 8, 1, 9)
min_val, max_val, avg = analyze_numbers(data)
print(f"Min: {min_val}, Max: {max_val}, Avg: {avg:.2f}")

Expert: Under the Hood

Why Tuples Are Faster Than Lists

▼

Immutability enables optimizations:

  • No memory reallocation: Lists must reserve extra space for growth; tuples are fixed size
  • Optimized storage: CPython can allocate tuples more efficiently
  • Interning: Small tuples are often cached by Python for reuse
  • Hash calculation: Can be precomputed once at creation
import timeit

# Creating and accessing
t = tuple(range(1000))
l = list(range(1000))

# Iteration is typically 10-20% faster with tuples
print(timeit.timeit(lambda: [x for x in t], number=10000))
print(timeit.timeit(lambda: [x for x in l], number=10000))

Tuples as Dictionary Keys

▼

Only hashable objects can be dictionary keys. Tuples (when they contain hashable elements) are hashable, but lists are not.

# Tuples as keys (immutable, hashable)
locations = {
    (0, 0): "origin",
    (1, 2): "point_a",
    (5, 5): "point_b"
}

print(locations[(0, 0)])  # origin

# Lists cannot be keys (mutable, unhashable)
# cache = {[1, 2]: "value"}  # TypeError!

# Tuples containing lists cannot be keys
# bad_key = ([1, 2], 3)
# cache = {bad_key: "value"}  # TypeError!

# But tuples of immutable objects work fine
good_key = ((1, 2), (3, 4))  # Nested tuple
cache = {good_key: "nested"}
print(cache[good_key])  # nested

Tuple Free-List Recycling

▼

CPython maintains a "free list" of small tuples to avoid repeated allocation:

# CPython caches tuples of size 0-20
# Creating many small tuples is very efficient

# Efficient: reuses cached tuples
t1 = ()
t2 = ()
print(t1 is t2)  # True! Same object

# Small tuples are interned
t3 = (1, 2)
t4 = (1, 2)
print(t3 is t4)  # Often True (not guaranteed)

# Larger tuples are not interned
big1 = tuple(range(100))
big2 = tuple(range(100))
print(big1 is big2)  # False, different objects

struct.pack Equivalence

▼

For low-level binary operations, struct.pack produces tuples:

import struct

# Pack data into binary format, returns bytes
data = struct.pack('HH', 1000, 2000)
print(data)  # b'\xe8\x03\xd0\x07'

# Unpack returns a tuple!
result = struct.unpack('HH', data)
print(result)  # (1000, 2000)
print(type(result))  # <class 'tuple'>

# Useful for binary protocols
format_string = '4s I f'  # 4-char string, unsigned int, float
packed = struct.pack(format_string, b'TEST', 42, 3.14)
unpacked = struct.unpack(format_string, packed)
print(unpacked)  # (b'TEST', 42, 3.140000104904175)

Socratic Q&A: Test Your Understanding

1. Why does Python require a trailing comma for single-element tuples? ▼

Without the comma, ("item") is just a string in parentheses, which are used for grouping expressions. The comma distinguishes it as a tuple constructor. This is why ("item",) creates a 1-element tuple while ("item") creates just a string.

2. If tuples are immutable, how can we modify a list inside a tuple? ▼

The tuple itself is immutable—you cannot replace the list or remove it from the tuple. But the list object that the tuple references is still mutable. Think of the tuple as holding a reference (pointer) to the list; the reference is fixed, but what it points to can change.

3. What's the difference between sorted(tuple) and manually sorting? ▼

sorted() returns a list, not a tuple. If you need a sorted tuple, use tuple(sorted(my_tuple)). Lists can be sorted in-place with .sort(), but tuples cannot be modified at all.

4. Why are tuples useful as dictionary keys? ▼

Dictionary keys must be hashable (immutable and have a consistent hash value). Lists are mutable and cannot be keys, but tuples are immutable. This is perfect for coordinates, multi-part identifiers, or any composite key that shouldn't change.

5. Can you use a tuple containing a list as a dictionary key? ▼

No. While the tuple itself is immutable, lists are not hashable. Any tuple containing an unhashable element (like a list or dict) becomes unhashable too. Only tuples of hashable elements (like tuples of tuples, strings, or numbers) can be dictionary keys.

6. What does tuple unpacking do, and why is it useful? ▼

Tuple unpacking assigns multiple elements to separate variables in one statement. x, y, z = (1, 2, 3) is cleaner than x = (1, 2, 3)[0], etc. It's especially useful for returning multiple values from functions or destructuring data structures.

7. How does zip() relate to tuples in loops? ▼

zip() combines multiple iterables into tuples. zip([1, 2], ['a', 'b']) produces [(1, 'a'), (2, 'b')]. When looping, you can unpack these tuples: for num, letter in zip(nums, letters):

8. What's the difference between count() and index()? ▼

count(x) returns how many times x appears (0, 1, 2, etc.). index(x) returns the position of the first occurrence. index() raises ValueError if not found; count() returns 0.

9. When should you use named tuples instead of regular tuples? ▼

Use named tuples when you want readability: point.x is clearer than point[0]. Named tuples are still immutable and hashable, making them ideal for lightweight immutable data structures with named fields.

10. Why are tuples sometimes faster than lists? ▼

Tuples are immutable, so CPython can optimize them: no space reservation for growth, possible interning of small tuples, and precomputed hash values. Lists need flexibility to grow, which adds overhead. For read-heavy workloads, tuples are 10-20% faster.

11. What happens when you concatenate a large tuple many times? ▼

Each concatenation creates a new tuple, which is O(n) in time and space. Repeatedly concatenating in a loop is O(n²). Use tuple(itertools.chain(...)) or collect in a list then convert to tuple for better performance.

12. How does Python's tuple interning work and when should you care? ▼

CPython caches small tuples (size 0-20) to save memory. (1, 2) and (1, 2) might be the same object in memory. This is an implementation detail you shouldn't rely on, but it means small tuples are very efficient to create.
