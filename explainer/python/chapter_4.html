Chapter 4 — Phase I: Foundations

Strings & Text Processing

Master immutable sequences of Unicode characters, string manipulation methods, and unlock the power of text in Python.

💡 Mental Model: The Immutable Pearl

Think of a string like a pearl necklace made of beads (characters) strung together. Once you've created the pearl, you can't modify the beads inside it—they're permanently ordered. But you CAN create a new necklace by taking pieces of the original and rearranging them. Each method like upper() or replace() doesn't change the original pearl; it crafts a brand new one. This immutability is Python's promise: strings never change, only create new versions of themselves.

What Are Strings?

A string is an immutable, ordered sequence of Unicode characters. It's one of Python's fundamental data types and represents textual data. Strings are immutable, meaning once created, they cannot be modified in-place—only replaced with new strings.

String Literals: Three Quoting Styles

Python offers three ways to create string literals, each with specific use cases:

Style Example Best For Special Feature
Single quotes 'Hello' Basic strings Can contain double quotes

# Single quotes
name = 'Alice'

poem = """Roses are red,

# Raw strings (escape sequences literal)

String Properties

Every string in Python has three crucial properties:

  • Immutable: Once created, characters cannot be changed in-place. s[0] = 'X' raises a TypeError.
  • Ordered: Characters have a fixed sequence and position (index). The 1st character is always at index 0.
  • Iterable: You can loop over each character using for char in string:

Accessing String Elements

Strings support both indexing (accessing single characters) and slicing (extracting substrings). Python uses zero-based indexing and supports negative indices for reverse access.

Indexing (Single Character Access)

Access individual characters by position using square brackets. Negative indices count backward from the end:


s = "Python"

# Forward indexing (0-based)
print(s[0])    # 'P'
print(s[1])    # 'y'
print(s[5])    # 'n'

# Negative indexing (from the end)
print(s[-1])   # 'n' (last character)
print(s[-2])   # 'o' (second-to-last)
print(s[-6])   # 'P' (first character)

# Out of bounds raises IndexError
print(s[10])   # IndexError: string index out of range

Slicing (Substring Extraction)

Extract substrings using the slice syntax: string[start:stop:step]

Parameter Meaning Default
start Beginning index (inclusive) 0
stop Ending index (exclusive) len(string)
step Increment between indices 1 

word = "Mathematics"

# Basic slicing
print(word[0:4])     # "Math" (indices 0,1,2,3)
print(word[4:])      # "ematics" (from index 4 to end)
print(word[:4])      # "Math" (from start to index 4)

# Slicing with step
print(word[::2])     # "Mhmtc" (every 2nd character)
print(word[1::2])    # "aaai" (every 2nd, starting at index 1)

# Negative indices in slicing
print(word[-4:])     # "tics" (last 4 characters)
print(word[:-4])     # "Mathemat" (all but last 4)

# Reverse a string
print(word[::-1])    # "scitamhtaM"

Built-in Functions for Strings

Python provides several built-in functions that work with strings by treating them as sequences of characters.

len() — String Length

Returns the number of characters in a string:


s = "Hello"
print(len(s))        # 5

empty = ""
print(len(empty))    # 0

phrase = "Hello, World!"
print(len(phrase))   # 13 (includes space and comma)

min() & max() — Character Ordering

Find the lexicographically smallest and largest characters:


s = "Python"

# Based on ASCII/Unicode values
print(min(s))    # 'P' (uppercase comes before lowercase)
print(max(s))    # 'y' (highest ASCII value)

word = "algorithm"
print(min(word)) # 'a'
print(max(word)) # 't'

sorted() — Sort Characters

Return a list of characters in sorted order:


s = "banana"
chars = sorted(s)
print(chars)           # ['a', 'a', 'a', 'b', 'n', 'n']
print(''.join(chars))  # "aaabnn"

# Reverse sort
reverse_sorted = sorted(s, reverse=True)
print(''.join(reverse_sorted))  # "nnbaaa"

enumerate() — Position & Character

Iterate with both index and character:


word = "code"

for index, char in enumerate(word):
    print(f"{index}: {char}")

# Output:
# 0: c
# 1: o
# 2: d
# 3: e

# Starting from a custom index
for index, char in enumerate(word, start=1):
    print(f"Position {index}: {char}")

String Methods

Python strings have dozens of built-in methods for manipulation and analysis. These methods never modify the original string (immutability); they return new strings.

Case Transformation


s = "Hello World"

print(s.upper())       # "HELLO WORLD"
print(s.lower())       # "hello world"
print(s.capitalize())  # "Hello world" (first char upper, rest lower)
print(s.title())       # "Hello World" (each word's first char upper)
print(s.swapcase())    # "hELLO wORLD"

Whitespace & Trimming


s = "  Python  "

print(s.strip())       # "Python" (remove leading/trailing)
print(s.lstrip())      # "Python  " (remove from left)
print(s.rstrip())      # "  Python" (remove from right)

# Remove specific characters
path = "///home/user///"
print(path.strip('/'))  # "home/user"

Search & Replace


text = "The quick brown fox jumps"

# find() — returns index or -1
idx = text.find("quick")
print(idx)             # 4

# replace() — substitute all occurrences
new_text = text.replace("quick", "lazy")
print(new_text)        # "The lazy brown fox jumps"

# Multiple replacements
code = "a = 1; b = 2; c = 3"
print(code.replace(";", "\n"))

# count() — occurrences
print(text.count("o"))  # 4

String Testing (Predicates)


word = "Python123"

print(word.isalpha())      # False (contains digits)
print(word.isdigit())      # False (contains letters)
print(word.isalnum())      # True (letters and digits only)
print(word.isspace())      # False (has non-whitespace)

# More predicates
print("123".isdigit())     # True
print("hello".isalpha())   # True
print("   ".isspace())     # True

# Checking prefixes/suffixes
filename = "document.pdf"
print(filename.startswith("doc"))    # True
print(filename.endswith(".pdf"))     # True

Joining & Splitting

Two complementary operations for converting between strings and lists:


# split() — string to list
sentence = "The quick brown fox"
words = sentence.split()
print(words)  # ['The', 'quick', 'brown', 'fox']

# Split on specific delimiter
csv = "apple,banana,cherry"
fruits = csv.split(",")
print(fruits)  # ['apple', 'banana', 'cherry']

# join() — list to string
items = ["red", "green", "blue"]
colors = ", ".join(items)
print(colors)  # "red, green, blue"

# Practical: build a path
path_parts = ["home", "user", "documents", "file.txt"]
filepath = "/".join(path_parts)
print(filepath)  # "home/user/documents/file.txt"

Formatting

The format() method substitutes placeholders with values:


# Positional arguments
msg = "Hello {}, you are {} years old".format("Alice", 25)
print(msg)  # "Hello Alice, you are 25 years old"

# Named arguments
template = "{name} scored {score} points"
result = template.format(name="Bob", score=98)
print(result)  # "Bob scored 98 points"

# Number formatting
price = 19.995
print(f"Price: ${price:.2f}")  # "Price: $20.00"

String Conversions

Converting between strings and other data types is fundamental to text processing. Python provides built-in functions for these transformations.

str() — Convert to String


# Convert various types to string
print(str(42))         # "42"
print(str(3.14))       # "3.14"
print(str(True))       # "True"
print(str(None))       # "None"
print(str([1,2,3]))    # "[1, 2, 3]"

# String is idempotent
s = "hello"
print(str(s))  # "hello"

int() & float() — Parse Numbers


# String to integer
num_str = "42"
num = int(num_str)
print(num + 8)  # 50 (arithmetic works)

# Parse with different bases
binary = int("1010", 2)      # 10 (binary)
hexadecimal = int("FF", 16)  # 255 (hexadecimal)

# String to float
price = float("19.99")
tax = price * 1.08

# Invalid conversions raise ValueError
try:
    int("abc")  # ValueError: invalid literal for int()
except ValueError as e:
    print(e)

ord() & chr() — Character Codes

Work with Unicode code points:


# ord() — character to Unicode value
print(ord('A'))    # 65
print(ord('a'))    # 97
print(ord('😀'))   # 128512

# chr() — Unicode value to character
print(chr(65))     # 'A'
print(chr(97))     # 'a'
print(chr(128512)) # '😀'

# Practical: Caesar cipher
def shift_char(char, shift=1):
    code = ord(char)
    return chr(code + shift)

print(shift_char('A', 1))  # 'B'

String Comparisons

Strings support equality and ordering comparisons based on lexicographic (dictionary) order.

Equality Operators


# Exact equality (case-sensitive)
print("hello" == "hello")    # True
print("Hello" == "hello")    # False
print("hello" != "world")    # True

# Comparing empty strings
empty = ""
print(empty == "")     # True
print(len(empty) == 0) # True

Ordering Comparisons (Lexicographic)

Strings are compared character-by-character using Unicode values:


# Alphabetical ordering
print("apple" < "banana")     # True (a < b)
print("apple" > "apple pie")  # False (apple is prefix)
print("apple" < "apple pie")  # True (shorter is less)

# Case matters (uppercase < lowercase in ASCII)
print("Zebra" < "apple")      # True (Z=90 < a=97)
print("APPLE" < "apple")      # True (A=65 < a=97)

# Numeric strings (lexicographic, not numeric)
print("10" < "2")   # True! ("1" < "2" in first position)
print(10 < 2)       # False (numeric comparison)

is vs == for Strings

String interning means some strings are cached, but don't rely on identity:


# For short/interned strings, 'is' might work
a = "hello"
b = "hello"
print(a is b)   # True (usually, due to interning)
print(a == b)   # True

# But never use 'is' for string comparison!
x = "hello" + " "
y = "hello "
print(x == y)           # True (content equality)
print(x is y)           # False (different objects)

# Always use == for content equality
# Use 'is' only for None, True, False checks

def is_palindrome(s):
    """Check if string is a palindrome (ignoring spaces/case)"""
    clean = s.replace(" ", "").lower()
    return clean == clean[::-1]

test_words = [
    "racecar",
    "hello",
    "A man a plan a canal Panama",
    "Madam"
]

for word in test_words:
    result = "is" if is_palindrome(word) else "is not"

PEP 393: Compact Unicode Storage

Python 3.3+ uses flexible string representation to minimize memory. Strings are stored in the most efficient encoding possible:

  • 1-byte (Latin-1): Characters 0-255 (ASCII-compatible)
  • 2-byte (UCS-2): Characters up to 65,535
  • 4-byte (UCS-4): Full Unicode range (0x10FFFF)

import sys

# Check internal representation
s1 = "hello"       # ASCII-only → 1 byte per char
s2 = "café"        # Extended Latin → 1 byte per char
s3 = "你好"         # Chinese → 2+ bytes per char

for s in [s1, s2, s3]:
    print(f"'{s}' → {sys.getsizeof(s)} bytes")

String Interning & Object Identity

Python caches certain strings (identifiers, short literals) for performance. This is called interning:


# Interned (same object)
a = "hello"
b = "hello"
print(a is b)  # Usually True!

# Not guaranteed with dynamic strings
x = "hello" + ""
y = "hello"
print(x is y)  # False (created at runtime)

# Force interning
import sys
z1 = sys.intern("hello world")
z2 = sys.intern("hello world")
print(z1 is z2)  # True (now same object)

Encode/Decode: Working with Bytes

Strings must be encoded to bytes for storage/transmission, and bytes decoded back to strings:


# String → bytes
text = "Hello, 世界"
utf8_bytes = text.encode('utf-8')
print(utf8_bytes)  # b'Hello, \xe4\xb8\x96\xe7\x95\x8c'
print(len(utf8_bytes))  # 12 bytes

# Bytes → string
decoded = utf8_bytes.decode('utf-8')
print(decoded)  # "Hello, 世界"

# Other encodings
ascii_bytes = "Hello".encode('ascii')
latin1_bytes = "café".encode('latin-1')

# Errors parameter
try:
    "café".encode('ascii')  # UnicodeEncodeError
except UnicodeEncodeError:
    # Use error handling
    safe = "café".encode('ascii', errors='ignore')  # "caf"
    replace = "café".encode('ascii', errors='replace')  # "caf?"

Think Like a Compiler: Socratic Q&A

1
Why can't you modify a string with s[0] = 'X'?
+

Strings are immutable in Python. This design choice provides several benefits: it allows Python to optimize memory (reuse identical strings), enables strings to be dictionary keys (keys must be hashable and unchanging), and makes concurrent code safer (no risk of modification during iteration).

When you want to "modify" a string, you must create a new one:

s = "hello" # Wrong: s[0] = 'H' → TypeError # Right: create new string s = 'H' + s[1:] # "Hello"
2
In the slice s[2:5], why is the stop index exclusive?
+

This convention makes slicing more intuitive. With exclusive stops, the length of a slice is always stop - start. So s[0:3] gives exactly 3 characters. Additionally, it simplifies adjacency: s[:i] and s[i:] perfectly partition the string with no overlap or gap.

s = "Python" # s[1:4] = "yth" (indices 1, 2, 3) # Length = 4 - 1 = 3 ✓ # Partitioning works perfectly: left = s[:3] # "Pyt" right = s[3:] # "hon" # left + right == s (no overlap, no gap)
3
What's the difference between s.replace("a", "b") and directly modifying characters?
+

replace() returns a new string with substitutions applied—it never modifies the original. This respects immutability and is thread-safe.

original = "banana" modified = original.replace("a", "o") print(original) # "banana" (unchanged!) print(modified) # "bonona" (new string) print(original is modified) # False (different objects)
4
Why does "10" < "2" return True but 10 < 2 return False?
+

String comparison is lexicographic (character-by-character), not numeric. The comparison looks at the first character: "1" (ASCII 49) is less than "2" (ASCII 50), so "10" < "2" is true. This is why parsing strings to numbers matters:

# String comparison (lexicographic) print("10" < "2") # True ("1" comes before "2") print("10" < "20") # True ("1" comes before "2") # Numeric comparison print(10 < 2) # False (10 is greater) print(int("10") < int("2")) # False (numeric comparison)
5
What does enumerate() offer that a simple for loop doesn't?
+

enumerate() provides both the index and value in one iteration, eliminating the need for manual index tracking. This is cleaner and more Pythonic:

word = "code" # Without enumerate (tedious) for i in range(len(word)): print(f"{i}: {word[i]}") # With enumerate (clean and Pythonic) for i, char in enumerate(word): print(f"{i}: {char}") # Starting from a custom index for pos, char in enumerate(word, start=1): print(f"Position {pos}: {char}")
6
When you do s.split() vs s.split(" "), what's the difference?
+

split() with no argument splits on any whitespace (spaces, tabs, newlines) and removes empty strings. split(" ") splits only on single spaces, potentially leaving empty strings if there are multiple spaces:

text = " Hello world \n Python " # split() — splits on any whitespace, removes empty strings print(text.split()) # ['Hello', 'world', 'Python'] # split(" ") — splits only on single spaces print(text.split(" ")) # ['', '', 'Hello', '', '', 'world', '', '\n', '', 'Python', '', ''] # For user input, use split() without arguments! user_input = " apple orange banana " items = user_input.split() # Clean, robust
7
What's the purpose of raw strings (r"...") and when would you use them?
+

Raw strings treat backslashes as literal characters, not escape sequences. They're essential for file paths (Windows), regex patterns, and LaTeX:

# Regular string (backslash is escape char) path = "C:\Users\name\file.txt" # \U, \n interpreted! # Raw string (backslash is literal) path = r"C:\Users\name\file.txt" # Correct! # Regex patterns pattern = r"\d{3}-\d{4}" # Phone number pattern # Without 'r', you'd need: "\\d{3}-\\d{4}"
8
Why does ord() return different values for uppercase vs lowercase?
+

ord() returns the Unicode code point of a character. Uppercase and lowercase letters are completely different characters in Unicode:

print(ord('A')) # 65 print(ord('a')) # 97 # Difference: 32 # This is why lowercase comes after uppercase in ordering print('A' < 'a') # True (65 < 97) # To compare case-insensitively, convert to same case first print('Apple'.lower() == 'apple'.lower()) # True
9
Why does .format() and f-strings exist when you can concatenate with +?
+

Concatenation with + requires converting all values to strings first, is hard to read with many values, and is inefficient. format() and f-strings are more readable, efficient, and support advanced features like number formatting:

name, age, score = "Alice", 25, 98.5 # Concatenation (verbose, error-prone) msg = name + " is " + str(age) + " and scored " + str(score) # format() (cleaner) msg = "{} is {} and scored {}".format(name, age, score) # f-string (most Pythonic, fastest) msg = f"{name} is {age} and scored {score}" # Formatting support price = 19.99 print(f"Price: ${price:.2f}") # Can't do this with +!
10
What's the performance difference between s.replace() and manually looping?
+

Built-in methods like replace() are implemented in C and are orders of magnitude faster than Python loops. Always prefer built-in string methods:

import timeit s = "hello world hello world" * 100 target, replacement = "hello", "goodbye" # Manual loop (slow) def manual_replace(s, old, new): result = "" for char in s: if s[i:i+len(old)] == old: result += new i += len(old) else: result += char return result # Built-in method (fast) result = s.replace(target, replacement) # Timing shows 100x+ difference! print("Use s.replace(), not manual loops!")
11
In what order does max() find the "maximum" character?
+

max() uses Unicode code point values. ASCII uppercase (65-90) comes before lowercase (97-122), and digits (48-57) come before letters:

print(max("aAbBcC")) # 'c' (ASCII 99) print(max("123abc")) # 'c' (ASCII 99) # Custom sorting with key parameter s = "Zap" print(max(s)) # 'p' (112) print(max(s, key=str.lower)) # 'z' (ignore case)
12
What happens if you try to convert non-numeric strings to int()?
+

A ValueError is raised. Always handle this when parsing user input:

# This raises ValueError try: num = int("abc") except ValueError: print("Invalid number!") # Better: use try-except or input validation user_input = input("Enter a number: ") try: num = int(user_input) print(f"You entered {num}") except ValueError: print("That's not a valid integer!")
