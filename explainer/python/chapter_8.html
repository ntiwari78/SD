Console I/O & Formatting

Master the art of communication with users. Learn to receive input, display output, and format data beautifully in the console.

Console Input: Getting Data From Users

🧠 Mental Model: The Conversation

Imagine a conversation between you and a friend. You ask a question, your friend thinks, then answers. In Python, input() is like asking a question - the program pauses, waits for the user to type, then continues with what they typed.

The input() Function

The input() function displays a prompt and waits for the user to type something, then returns what they typed as a string.

name = input("What is your name? ")
print(f"Hello, {name}!")

# When run:
# What is your name? Alice
# Hello, Alice!

🟢 Simple Input

age_text = input("How old are you? ")
print(age_text)

# Output when user enters 25:
# 25 (as a string, not a number!)

Key: input() always returns a string!

🟡 Converting Input to Numbers

age_text = input("How old are you? ")
age = int(age_text)
print(f"Next year you'll be {age + 1}")

# Or in one line:
age = int(input("How old are you? "))

🔴 Input with Error Handling

while True:
    try:
        age = int(input("Enter age: "))
        if age < 0:
            print("Age can't be negative")
            continue
        break
    except ValueError:
        print("Please enter a valid number")

print(f"You are {age} years old")

Type Conversion Functions

Function Example Result
int() int("42") 42 (integer)
float() float("3.14") 3.14 (decimal)
str() str(42) "42" (string)
bool() bool("True") True (boolean)

Program: Simple Calculator

num1 = float(input("Enter first number: "))
num2 = float(input("Enter second number: "))
operation = input("Enter operation (+, -, *, /): ")

if operation == "+":
    result = num1 + num2
elif operation == "-":
    result = num1 - num2
elif operation == "*":
    result = num1 * num2
elif operation == "/":
    result = num1 / num2
else:
    result = "Invalid operation"

⚠️ The eval() Trap: eval() can execute arbitrary code. Never use it on untrusted input! 
# DANGEROUS - don't do this

# But user could type: "__import__('os').system('rm -rf /')" !

Program: Number Guessing Game

import random

secret = random.randint(1, 100)
guess = 0
attempts = 0

while guess != secret:
    guess = int(input("Guess a number (1-100): "))
    attempts += 1

    if guess < secret:
        print("Too low, try again")
    elif guess > secret:
        print("Too high, try again")
    else:
        print(f"Correct! You got it in {attempts} attempts")

Formatted Printing: Making Output Beautiful

🧠 Mental Model: Templating

Imagine filling in a form. You have a template with blanks, and you fill in the specific values. String formatting works the same way: you have a template with placeholders, and you fill in the actual values.

Three Formatting Methods

Method 1: F-Strings (Python 3.6+) - RECOMMENDED

The modern, readable way to format strings:

name = "Alice"
age = 25

print(f"Name: {name}, Age: {age}")
# Output: Name: Alice, Age: 25

🟢 Simple Variables

x = 10
y = 20
print(f"x = {x}, y = {y}, sum = {x + y}")

# Output: x = 10, y = 20, sum = 30

🟡 Format Specifiers

pi = 3.14159
print(f"Pi to 2 decimals: {pi:.2f}")
print(f"Percentage: {0.856:.1%}")

# Output:
# Pi to 2 decimals: 3.14
# Percentage: 85.6%

🟡 Alignment and Padding

name = "Bob"
print(f"|{name:<10}|")  # Left align
print(f"|{name:>10}|")  # Right align
print(f"|{name:^10}|")  # Center

# Output:
# |Bob       |
# |       Bob|
# |   Bob    |

🔴 Complex Formatting

values = [1234.5678, 0.9876, 99.1]
for val in values:
    print(f"Value: {val:>10.2f}")

# Output:
# Value:    1234.57
# Value:       0.99
# Value:      99.10

Method 2: .format() Method (Python 3+)

name = "Charlie"
age = 30
print("Name: {}, Age: {}".format(name, age))
print("Name: {n}, Age: {a}".format(n=name, a=age))

# Output: Name: Charlie, Age: 30

Method 3: % Operator (Old Style)

# Still used in legacy code
name = "Diana"
age = 28
print("Name: %s, Age: %d" % (name, age))

# Output: Name: Diana, Age: 28
Format Code Meaning Example
.2f 2 decimal places {3.14159:.2f} → "3.14"
.0f No decimals (rounded) {3.7:.0f} → "4"
:10 Width of 10 characters {"hello":10} → "hello "
:>10 Right-align in 10 chars {"hello":>10} → " hello"
:^10 Center in 10 chars {"hello":^10} → " hello "
:.1% As percentage {0.56:.1%} → "56.0%"
:, Thousands separator {1234567:,} → "1,234,567"

Program: Formatted Multiplication Table

print(f"{'X':<5} | {'Y':<5} | {'Product':<10}")
print("-" * 25)

for x in range(1, 4):
    for y in range(1, 4):
        product = x * y
        print(f"{x:<5} | {y:<5} | {product:<10}")

# Output:
# X     | Y     | Product
# -------------------------
# 1     | 1     | 1
# 1     | 2     | 2
# ...

Program: Financial Report

transactions = [
    ("Coffee", 4.50),
    ("Lunch", 12.75),
    ("Gas", 45.00),
    ("Movie", 15.50)
]

total = 0
print(f"{'Item':<15} {'Amount':>10}")
print("-" * 27)

for item, amount in transactions:
    print(f"{item:<15} ${amount:>9.2f}")
    total += amount

print("-" * 27)
print(f"{'TOTAL':<15} ${total:>9.2f}")

Expert Deep Dives

🔴 sys.stdin, sys.stdout, and sys.stderr

▼

Python treats input and output as streams - continuous flows of data:

  • sys.stdin - Standard input (keyboard)
  • sys.stdout - Standard output (console, where print() goes)
  • sys.stderr - Standard error (console, for error messages)
import sys

# Print to stdout (normal)
print("Normal output", file=sys.stdout)

# Print to stderr (errors)
print("Error message", file=sys.stderr)

# Read from stdin directly
line = sys.stdin.readline()  # Read one line
print(f"You typed: {line}")

Why this matters: You can redirect streams in the shell:

# Redirect stdout to a file
python script.py > output.txt

# Redirect stderr to a file
python script.py 2> errors.txt

# Redirect both
python script.py > output.txt 2> errors.txt

🔴 Understanding Buffering: TextIOWrapper → BufferedWriter → FileIO

▼

When you print to the console, data goes through multiple layers:

  1. TextIOWrapper: Handles text encoding/decoding
  2. BufferedWriter: Accumulates data before sending
  3. FileIO: Low-level file operations
  4. Kernel: Operating system actually sends to terminal
import sys
import time

# Buffering issue
print("Loading", end="", flush=False)  # Not flushed yet
time.sleep(1)
print(".", end="", flush=False)  # Still waiting
time.sleep(1)
print(".", end="", flush=False)
time.sleep(1)
print(" Done!", flush=True)  # Now it all appears at once

# Better: use flush=True
print("A", end="", flush=True)
time.sleep(0.5)
print("B", end="", flush=True)
time.sleep(0.5)
print("C", end="", flush=True)

flush=True forces the buffer to send data immediately instead of waiting.

🔴 The __format__ Protocol: Custom Formatting

▼

When you use f"{value:.2f}", Python calls value.__format__(".2f"). You can implement this for custom classes:

class Temperature:
    def __init__(self, celsius):
        self.celsius = celsius

    def __format__(self, format_spec):
        if format_spec == 'f':
            fahrenheit = (self.celsius * 9/5) + 32
            return f"{self.celsius}°C ({fahrenheit:.1f}°F)"
        elif format_spec == 'c':
            return f"{self.celsius}°C"
        else:
            return f"{self.celsius}"

temp = Temperature(25)
print(f"Temperature: {temp:f}")  # Temperature: 25°C (77.0°F)
print(f"Temperature: {temp:c}")  # Temperature: 25°C

🔴 FORMAT_VALUE Bytecode: How F-Strings Are Compiled

▼

F-strings are compiled into efficient bytecode. Let's see how:

import dis

code = compile('f"x={x:.2f}"', '', 'eval')
dis.dis(code)

# Output shows FORMAT_VALUE opcode:
# LOAD_NAME
# FORMAT_VALUE
# BUILD_STRING

This is why f-strings are faster than .format() or % operator - less overhead at runtime. The formatting happens during compilation!

🔴 Robust Input Validation Pattern

▼

Always validate user input. Here's a pattern:

def get_positive_integer(prompt):
    """Get a positive integer from user, with validation"""
    while True:
        try:
            value = int(input(prompt))
            if value <= 0:
                print("Please enter a positive number")
                continue
            return value
        except ValueError:
            print("That's not a valid integer. Try again.")
        except KeyboardInterrupt:
            print("\nOperation cancelled")
            return None
        except EOFError:
            print("\nNo input provided")
            return None

age = get_positive_integer("Enter your age: ")
if age:
    print(f"You are {age} years old")

🔴 Advanced Number Formatting

▼ 
# Scientific notation
print(f"{1234567:.2e}")  # 1.23e+06

# Binary, octal, hex
print(f"{255:b}")  # 11111111 (binary)
print(f"{255:o}")  # 377 (octal)
print(f"{255:x}")  # ff (hex lowercase)
print(f"{255:X}")  # FF (hex uppercase)

# Padding with zeros
print(f"{42:05d}")  # 00042

# Thousands separator
print(f"{1234567:,}")  # 1,234,567
print(f"{1234567:_}")  # 1_234_567 (Python 3.6+)

Socratic Q&A: Test Your Understanding

Click each question to reveal the answer. Think about it first!

1. What type does input() return?

▼

input() always returns a string, even if the user types numbers.

age = input("Enter age: ")  # User types "25"
type(age)  # 

# To use it as a number:
age = int(input("Enter age: "))

2. How do you convert a string to an integer?

▼

Use the int() function:

num_string = "42"
num = int(num_string)
print(num + 8)  # 50

# What if it fails?
int("42.5")  # ValueError! Can't convert decimal string to int
int("hello")  # ValueError! Not a number

3. What's the difference between end="\n" and end=""?

▼

end="\n" (default) adds a newline after printing. end="" doesn't add anything.

# With newline (default)
print("A")
print("B")
# Output:
# A
# B

# Without newline
print("A", end="")
print("B", end="")
# Output: AB

4. How do f-strings differ from .format()?

▼

F-strings are faster and more readable:

# f-string (modern, recommended)
name = "Alice"
age = 25
print(f"Name: {name}, Age: {age}")

# .format() (older, still works)
print("Name: {}, Age: {}".format(name, age))

# % operator (very old, avoid)
print("Name: %s, Age: %d" % (name, age))

5. What does the sep parameter in print() do?

▼

sep sets the separator between multiple values:

print("A", "B", "C")  # A B C (default sep=" ")
print("A", "B", "C", sep="-")  # A-B-C
print("A", "B", "C", sep="")  # ABC

6. How do you format a float to 2 decimal places?

▼ 
pi = 3.14159
print(f"{pi:.2f}")  # 3.14

# Other precisions
print(f"{pi:.0f}")  # 3 (no decimals)
print(f"{pi:.4f}")  # 3.1416 (four decimals)

7. What's wrong with using eval() on user input?

▼

eval() executes any Python code, which is a massive security risk:

# NEVER do this:
result = eval(input("2 + 2 = "))

# User could type malicious code:
# "__import__('os').system('rm -rf /')"
# Or: "open('/etc/passwd').read()"

Use int(), float(), or other specific converters instead.

8. How do you right-align text in formatting?

▼ 
name = "Alice"
print(f"{name:>10}")  # "     Alice" (right-aligned)
print(f"{name:<10}")  # "Alice     " (left-aligned)
print(f"{name:^10}")  # "  Alice   " (centered)

9. What's the purpose of flush=True in print()?

▼

flush=True forces the output to appear immediately instead of being buffered:

import time

# Without flush - all appears at once after 3 seconds
print(".", end="")
time.sleep(1)
print(".", end="")
time.sleep(1)
print(".")

# With flush - dots appear as they print
print(".", end="", flush=True)
time.sleep(1)
print(".", end="", flush=True)
time.sleep(1)
print(".")

10. How do you convert a number to a string for printing?

▼

Use str() or f-strings (which do it automatically):

num = 42
text = str(num)  # "42"

# F-strings handle conversion automatically
print(f"The answer is {num}")  # Works seamlessly

11. What does sys.stderr do?

▼

sys.stderr is for error messages. Both stdout and stderr go to the console, but they can be redirected separately:

import sys

print("Normal message", file=sys.stdout)
print("Error message", file=sys.stderr)

# In the shell:
# python script.py > output.txt 2> errors.txt

12. How do you format numbers with thousands separators?

▼ 
big_num = 1234567
print(f"{big_num:,}")  # 1,234,567 (comma separator)
print(f"{big_num:_}")  # 1_234_567 (underscore separator)

Key Takeaways

  • input() always returns a string - convert with int(), float(), etc.
  • Always validate user input to prevent errors
  • print() has useful parameters: sep, end, file, flush
  • F-strings are the modern, fast way to format output
  • Use format codes like :.2f, :>10, :, for professional output
  • Never use eval() on untrusted input
  • sys.stdout, sys.stderr, and sys.stdin are the underlying streams
  • Use flush=True when you need output to appear immediately
  • Understand the three formatting methods (f-string, .format(), % operator)
