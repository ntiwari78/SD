Setting Up Your Python Workshop

Feynman Mental Model

The Workshop Analogy

Think of programming like woodworking. Python itself is the engine—the fundamental tool that does work. The IDE (Integrated Development Environment) is your workbench—where you design, build, and test. The third-party libraries (like NumPy, Django, etc.) are your specialized tools—a circular saw for specific cuts, a power drill for specific holes. And pip is your hardware store—you walk in, request a tool by name, and it shows up at your door. A good workshop has a well-organized workbench, quality tools, and easy access to supplies. In this chapter, we're setting up your Python workshop properly so you can work efficiently from day one.

Understanding Python: Specification vs Implementation

Before installing Python, understand this crucial distinction:

Python the Specification

Python is a language specification—a detailed document describing how the language should behave. It specifies syntax rules, built-in types, operators, and the behavior of standard libraries. Think of it as the blueprint.

Python Implementations

An implementation is actual software that follows the specification. Multiple implementations exist:

Implementation Written In Best For When to Use
CPython C General purpose, web, data science Default choice for almost everything
PyPy RPython CPU-intensive, long-running programs When CPython is too slow
Jython Java Java ecosystem integration Java shops needing Python
IronPython C# .NET integration Microsoft environments

When you download Python from python.org, you're getting CPython. This is the reference implementation and what 95% of the world uses.

How Python Executes: The Full Pipeline

Every Python implementation, whether CPython or PyPy, follows the same basic architecture:

Source Code (.py) → Compiler → Bytecode → Interpreter → Execution

Step 1: Compiler Takes your Python source code and translates it to low-level bytecode (CPU-agnostic instructions).

Step 2: Bytecode Stored in .pyc files (cached in __pycache__ directory) for faster subsequent runs.

Step 3: Interpreter (Virtual Machine) Executes bytecode instruction by instruction, managing memory, object creation, and operations.

Step 4: Execution Your program runs.

This is why Python is called "interpreted"—from the user's perspective, there's no separate compilation step. But technically, compilation to bytecode happens transparently.

Installation

Windows Installation

1. Visit python.org and download the latest Python installer (3.11+ recommended)

2. Run the installer. IMPORTANT: Check the box "Add Python to PATH" at the bottom. This lets you run Python from the command line.

3. Choose "Install for all users" (requires admin) or "Install just for me"

4. Wait for installation to complete

5. Verify installation in Command Prompt:

python --version
# Output: Python 3.11.0 (or your version)

Linux Installation

Most Linux distributions come with Python, but you may need to install or update:

# Check if Python is installed
python3 --version

# If not installed, use your package manager
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install python3 python3-pip

# Fedora
sudo dnf install python3 python3-pip

# macOS (via Homebrew)
brew install python3

Verify installation:

which python3
# /usr/bin/python3

Working with Python: Two Modes

Interactive Mode

Start the Python interpreter to execute code line-by-line. Perfect for exploring, experimenting, and testing ideas.

$ python3
Python 3.11.0 (main, Oct  3 2022, 09:26:40)
[GCC 12.0.0] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> print("Hello, Interactive Mode!")
Hello, Interactive Mode!
>>> 2 + 3
5
>>> exit()
$

Use cases:

  • Learning—immediately see results of code
  • Prototyping—test algorithms before writing full programs
  • Debugging—inspect variables and data structures
  • Quick calculations

Advantages: Immediate feedback, no file management, explore the language interactively

Disadvantages: Not suitable for large programs, difficult to save work, can't organize code

Script Mode

Write complete programs in .py files and execute them. Suitable for production code, larger projects, and distributable applications.

Example: hello.py

# This is a complete Python program saved in a file
print("Hello from a script!")
name = input("What's your name? ")
print(f"Nice to meet you, {name}!")

Execute the script:

$ python3 hello.py
Hello from a script!
What's your name? Alice
Nice to meet you, Alice!

Use cases:

  • Production applications
  • Larger programs with multiple files
  • Automation scripts
  • Distributable programs
  • Code that runs on a schedule

Advantages: Organized code, reusable, can be run repeatedly, version control friendly

Disadvantages: Less immediate feedback than interactive mode

Third-Party Libraries and pip

What is PyPI?

The Python Package Index (pypi.org) is a repository of over 400,000 free Python packages. Anyone can upload packages; anyone can download them. It's Python's ecosystem of code.

What is pip?

pip is the package installer for Python. It's your hardware store—you ask for what you need by name, and it downloads and installs it from PyPI.

Installing packages:

pip install packagename

Example: Installing NumPy (numerical computing)

pip install numpy

# Now use it in your code
import numpy as np
arr = np.array([1, 2, 3])
print(arr)

Virtual Environments (venv)

Create isolated Python environments for different projects. Each project has its own packages and Python version.

# Create a virtual environment
python3 -m venv myproject_env

# Activate it (Windows)
myproject_env\Scripts\activate

# Activate it (Linux/macOS)
source myproject_env/bin/activate

# Now pip installs packages only in this environment
pip install requests pandas

# Deactivate when done
deactivate

Why use venv? Prevents "dependency hell" where different projects need conflicting versions of the same library.

Determining Your Python Version

Different versions of Python have different features. Check which version you're using:

Version considerations:

  • Python 3.6-3.8: Older but still widely supported. Most libraries work.
  • Python 3.9-3.11: Modern stable versions. Recommended.
  • Python 3.12+: Latest features, fastest performance.

Use Python 3.9 or later unless you have specific compatibility requirements.

DEEP DIVE Modern Python Toolchain: pyproject.toml, virtualenv, uv, ruff, and mypy

Professional Python projects use a modern toolchain that goes beyond basic pip:

pyproject.toml (Project Configuration)

Replaces setup.py for defining project metadata, dependencies, and build configuration. Standard format introduced in PEP 517/518.

[project]
name = "my-app"
version = "1.0.0"
description = "A Python application"
requires-python = ">=3.9"
dependencies = [
    "requests>=2.28.0",
    "pandas>=1.5.0"
]

[tool.pytest.ini_options]
testpaths = ["tests"]

virtualenv / venv (Isolated Environments)

Create project-specific Python environments so dependencies don't conflict. Each project has its own isolated set of installed packages.

python -m venv myenv
source myenv/bin/activate
pip install -r requirements.txt

uv (Fast Package Manager)

Modern replacement for pip written in Rust. 10-100x faster than pip. Handles dependencies better and faster.

uv pip install numpy pandas
uv venv

ruff (Linter)

Fast Python linter that checks for code quality issues: unused imports, undefined names, style violations. Catches bugs before runtime.

ruff check . --fix  # Check and auto-fix issues

mypy (Type Checker)

Static type checker that catches type errors before runtime. Add type hints to your code, and mypy verifies correctness.

# With type hints
def greet(name: str) -> str:
    return f"Hello, {name}"

mypy myscript.py  # Catches type errors

Professional Workflow:

uv venv
source .venv/bin/activate
uv pip install -e ".[dev]"
ruff check .
mypy src/
pytest

Think Like a Compiler

Understanding the toolchain helps you appreciate why things work the way they do. When you run pip install numpy, here's what happens step by step:

1. pip receives your request
2. pip contacts PyPI (pypi.org) and searches for "numpy"
3. PyPI returns matching packages and their versions
4. pip downloads the latest compatible version
5. pip extracts the package files
6. If the package includes C code, it compiles it for your system
7. pip installs files into your Python's site-packages directory
8. pip updates your environment so "import numpy" works
9. Your code can now use numpy

This is why you can't install packages meant for Windows on Linux without recompiling—the compiled C code is platform-specific. But pure Python packages work everywhere.

When you run Python, the interpreter loads the standard library from its installation directory, then searches site-packages for third-party packages. This search path (sys.path) determines what you can import.

Socratic Q&A: Test Your Understanding

Work through these questions to deepen your understanding of Python tooling.

1. What's the difference between the Python specification and a Python implementation?

Answer: The specification is a document describing how Python should behave—its syntax, semantics, and standard library. An implementation is actual executable software that follows the specification. Multiple implementations exist: CPython (default, in C), PyPy (fast JIT compiler), Jython (runs on Java VM), and IronPython (runs on .NET). All follow the same specification so code written for one works on all.

2. Describe the complete process Python goes through from your source code (.py file) to execution.

Answer:

  1. Source Code (hello.py) is read from disk
  2. Compiler tokenizes and parses it, creating an Abstract Syntax Tree
  3. Compilation translates the AST to bytecode (CPU-agnostic instructions)
  4. Caching stores bytecode in .pyc files (__pycache__ directory)
  5. Bytecode is fed to the Python Virtual Machine (interpreter)
  6. Interpreter executes each bytecode instruction, managing memory and objects
  7. Execution produces the output/result

This happens transparently—you just run "python hello.py" and it works. But understanding this pipeline helps you reason about performance (step 6 is where slowness comes from) and debugging (error messages reference bytecode concepts).

3. What do C:\>, $, and >>> signify? When do you see each?

Answer:

  • C:\> Windows Command Prompt (OS shell). For OS commands like dir, cd, copy.
  • $ Linux/macOS Terminal (OS shell, usually Bash). For OS commands like ls, cd, mkdir.
  • >> Python Interactive Prompt (Python interpreter). For Python code: print(), x = 5, etc.

Key distinction: The first two are your operating system shell. The third is inside Python. You type Python code at the >>> prompt and Python executes it immediately.

4. Compare interactive mode and script mode. What are the use cases for each?

Answer:

Interactive Mode:

  • Start Python interpreter and type code line-by-line
  • Results appear immediately
  • No file needed
  • Use cases: Learning, experimentation, testing code snippets, debugging, calculations

Script Mode:

  • Write code in a .py file and execute it
  • Entire program runs from start to finish
  • Suitable for larger programs
  • Use cases: Production applications, automation, distribution, reusable code

Start learning in interactive mode. Move to script mode as programs get larger.

5. What is PyPI and what is pip? How do they relate?

Answer: PyPI (Python Package Index, at pypi.org) is an online repository containing over 400,000 Python packages. Anyone can upload code; anyone can download it. pip is a command-line tool that interacts with PyPI. When you run pip install numpy, pip contacts PyPI, finds the numpy package, downloads it, and installs it on your computer. pip is the client; PyPI is the server of packages.

6. List 5 third-party libraries with their purpose and explain why you'd use each.

Answer: Any five of:

  • NumPy: Numerical computing with arrays. For mathematical operations, faster than lists.
  • Pandas: Data analysis with DataFrames. For tabular data manipulation and analysis.
  • Matplotlib: Data visualization. For creating charts, graphs, plots.
  • Django/Flask: Web frameworks. For building web applications and APIs.
  • TensorFlow/PyTorch: Deep learning. For neural networks and machine learning.
  • Beautiful Soup: Web scraping. For parsing HTML and extracting data from web pages.
  • Requests: HTTP library. For making API calls and downloading from the web.
7. What are virtual environments and why are they important?

Answer: Virtual environments (venv) are isolated Python environments. Each project can have its own site-packages directory with its own set of installed packages.

Why important: Without venv, all projects share the same Python and packages. If Project A needs Django 3.0 and Project B needs Django 4.0, they conflict. venv isolates them—each project has exactly what it needs without affecting others.

Workflow:

python -m venv myproject
source myproject/bin/activate  # Now in isolated environment
pip install requests            # Installs only in this venv
# ... work on project ...
deactivate                       # Exit venv
8. What's the difference between IDLE, PyCharm, and VS Code? When would you use each?

Answer:

  • IDLE: Minimal IDE bundled with Python. Syntax highlighting, debugging, shell. Use: Beginners learning Python.
  • PyCharm: Full-featured professional IDE. Project management, advanced debugging, testing. Use: Professional projects, teams, complex applications.
  • VS Code: Lightweight code editor with Python extension. Fast, customizable, popular. Use: Professional work, web development, systems programming. Sweet spot between simplicity and power.

As a beginner, start with IDLE or VS Code. PyCharm is overkill initially but great as you grow.

9. Explain what happens when you run "pip install numpy".

Answer:

  1. pip sends a query to PyPI asking for "numpy"
  2. PyPI responds with available versions and metadata
  3. pip selects the latest compatible version
  4. pip downloads the package (usually a .whl file)
  5. pip extracts the downloaded file
  6. If the package has C code, pip compiles it for your OS
  7. pip copies files to site-packages directory
  8. pip updates the search path so Python can find it
  9. You can now "import numpy" in your code
10. True or False: Python 2 is still actively supported and recommended for new projects.

Answer: False. Python 2 reached end-of-life on January 1, 2020. It receives no updates, bugfixes, or security patches. All new projects should use Python 3 (preferably 3.9+). Only legacy systems still use Python 2, and they're being actively migrated to Python 3.

11. Explain the difference between built-in modules and third-party packages. Give 3 examples of each.

Answer:

Built-in modules: Come with Python automatically. No pip install needed. Examples: os, sys, math, random, datetime, json, re, collections, itertools, unittest.

Third-party packages: Written by community. Install via pip. Examples: numpy, pandas, matplotlib, requests, django, flask, tensorflow, pytorch, beautifulsoup4, scrapy.

Built-ins are stable and standard. Third-party packages are specialized tools for specific domains. Use built-ins for basics (file I/O, math, date/time). Use third-party for advanced features (data science, web frameworks, machine learning).
