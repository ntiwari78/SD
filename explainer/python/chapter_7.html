Loops & Iteration

Master the art of repetition — doing things again and again without writing the same code multiple times. Learn to automate patterns and process data efficiently.

Repetition Control: Why Loops Matter

🧠 Mental Model: The Assembly Line

Imagine you're on an assembly line at a factory. Instead of giving separate instructions to each worker for each item, you teach them a process once: "Check if the item is defective. If not, box it. Move to the next item. Repeat."

Loops work the same way. You write the code once, and the computer repeats it automatically until a condition changes.

The Problem Loops Solve

Without loops, to add numbers 1 through 100, you'd need to write:

sum = 0
sum = sum + 1
sum = sum + 2
sum = sum + 3
# ... 97 more lines!
sum = sum + 100

With loops, you write:

sum = 0
for i in range(1, 101):
    sum = sum + i

Two patterns of repetition exist: while loops (repeat while condition is true) and for loops (repeat over a sequence).

While Loops: Condition-Controlled Repetition

Syntax and How It Works

A while loop repeats a block of code as long as a condition is true.

while condition:
    # Code here executes repeatedly
    # as long as condition is True

Execution Flow

🟢 Beginner: Simple Counter

count = 0
while count < 5:
    print(f"Count is {count}")
    count = count + 1

# Output:
# Count is 0
# Count is 1
# Count is 2
# Count is 3
# Count is 4

Key point: The variable count must change inside the loop, or it repeats forever!

🟡 Intermediate: User Validation

password = ""
while password != "secret":
    password = input("Enter password: ")
    if password == "secret":
        print("Access granted!")
    else:
        print("Try again.")

# Keeps asking until correct

Practical use: validate input until user provides correct data.

🔴 Expert: Sentinel-Controlled Loop

# Read numbers until user enters -1
total = 0
count = 0
num = int(input("Enter number (-1 to stop): "))
while num != -1:
    total += num
    count += 1
    num = int(input("Enter number (-1 to stop): "))

if count > 0:
    average = total / count
    print(f"Average: {average}")

# -1 is "sentinel" (end marker)

The Infinite Loop Pitfall

⚠️ Warning: If your condition never becomes false, your loop runs forever! 
# DANGEROUS - Don't run this!
while True:
    print("This repeats forever...")

To avoid infinite loops, always ensure your condition will eventually become false. Three strategies:

Strategy Example Best For
Counter-Controlled count < 10; count += 1 Repeating fixed number of times
Flag-Controlled found == False Searching for something
Sentinel-Controlled user_input != "quit" User interaction, file reading

Program: Factorial with While Loop

def factorial_while(n):
    """Calculate n! using while loop"""
    result = 1
    current = 1
    while current <= n:
        result *= current
        current += 1
    return result

print(factorial_while(5))  # Output: 120 (5! = 5×4×3×2×1)

For Loops: Sequence-Controlled Iteration

A for loop iterates over a sequence (list, string, range, etc.), assigning each element to a variable in turn.

for item in sequence:
    # Code executes once per item in sequence

Basic For Loop Examples

🟢 Iterating Over a List

fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

# Output:
# apple
# banana
# cherry

🟢 Iterating Over a String

word = "Python"
for letter in word:
    print(letter)

# Output:
# P, y, t, h, o, n (one per line)

🟢 Nested Loops

for i in range(1, 4):
    for j in range(1, 4):
        print(f"({i},{j})", end=" ")
    print()

# Output:
# (1,1) (1,2) (1,3)
# (2,1) (2,2) (2,3)
# (3,1) (3,2) (3,3)

The Range Function

range() generates a sequence of numbers. It's the most common way to repeat a loop a specific number of times.

Function Call What It Produces Use Case
range(5) 0, 1, 2, 3, 4 Repeat 5 times from 0
range(1, 6) 1, 2, 3, 4, 5 Repeat 5 times from 1 to 5
range(0, 10, 2) 0, 2, 4, 6, 8 Every other number (step=2)
range(10, 0, -1) 10, 9, 8, ..., 1 Countdown (negative step)

Program: Fibonacci Sequence with For Loop

def fibonacci(n):
    """Generate first n Fibonacci numbers"""
    if n <= 0:
        return []
    elif n == 1:
        return [1]

    fib_list = [1, 1]
    for i in range(2, n):
        next_fib = fib_list[i-1] + fib_list[i-2]
        fib_list.append(next_fib)

    return fib_list

print(fibonacci(7))  # [1, 1, 2, 3, 5, 8, 13]

For vs While: When to Use Each

Feature For Loop While Loop
When you know How many times to repeat A condition to check
Example Process each item in a list Repeat until user enters "quit"
Sequence? Yes, required No, uses boolean condition
Cleaner code? Usually more readable Better for open-ended loops

Loop Control: Break and Continue

Sometimes you need to exit a loop early or skip to the next iteration. Two keywords help:

🟢 Break: Exit the Loop

for i in range(10):
    if i == 5:
        break  # Exit loop immediately
    print(i)

# Output: 0 1 2 3 4

break stops the loop immediately, skipping remaining iterations.

🟢 Continue: Skip This Iteration

for i in range(5):
    if i == 2:
        continue  # Skip to next iteration
    print(i)

# Output: 0 1 3 4

continue jumps to the next iteration, skipping the rest of current iteration.

Program: Finding a Number in a List

def find_number(numbers, target):
    """Find target in list, return index or -1"""
    for i in range(len(numbers)):
        if numbers[i] == target:
            return i  # Found it!
    return -1  # Not found

# Example:
result = find_number([10, 20, 30, 40], 30)
print(f"Index: {result}")  # Output: 2

Program: Sum of Integers (Skip Negatives)

numbers = [5, -3, 8, -1, 10, 2]
total = 0

for num in numbers:
    if num < 0:
        continue  # Skip negative numbers
    total += num

print(f"Sum: {total}")  # Output: 25 (5+8+10+2)

The Else Block: Completion Without Break

Python allows an else block after a loop. It executes only if the loop completes normally (without break).

💡 The "NoBreak" Clause

Think of else as: "Run this only if we got through the loop without hitting break." It's like a success condition.

🟢 Simple For-Else

for i in range(3):
    print(i)
else:
    print("Loop completed!")

# Output:
# 0
# 1
# 2
# Loop completed!

🟡 Search with For-Else

target = 50
numbers = [10, 20, 30, 40]

for num in numbers:
    if num == target:
        print("Found!")
        break
else:
    print("Not found in list")

# Output: Not found in list

Program: Prime Number Checker

def is_prime(n):
    """Check if n is prime using for-else"""
    if n < 2:
        return False

    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False  # Found a divisor, not prime

    return True  # No divisors found, is prime

# This demonstrates: if we break, n is composite
# if loop completes, n is prime

Program: Palindrome Checker

def is_palindrome(text):
    """Check if text reads same forwards and backwards"""
    text = text.lower().replace(" ", "")

    for i in range(len(text) // 2):
        if text[i] != text[-(i+1)]:
            return False  # Mismatch found

    return True

print(is_palindrome("A man a plan a canal Panama"))  # True

Expert Deep Dives

🔴 The Iteration Protocol: __iter__ and __next__

▼

When you write for item in sequence:, Python doesn't directly access the sequence. Instead, it uses the iteration protocol:

  1. __iter__() method returns an iterator object
  2. Iterator has __next__() method that returns one item at a time
  3. __next__() raises StopIteration when done
# Manual iteration (what for loops do internally)
numbers = [1, 2, 3]
iterator = iter(numbers)  # Calls __iter__()

print(next(iterator))  # 1, calls __next__()
print(next(iterator))  # 2
print(next(iterator))  # 3
# next(iterator)  # Would raise StopIteration

# Equivalent to:
for num in numbers:
    print(num)

Custom iterator example:

class CountUp:
    def __init__(self, max):
        self.current = 0
        self.max = max

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < self.max:
            self.current += 1
            return self.current
        else:
            raise StopIteration

for num in CountUp(3):
    print(num)  # 1, 2, 3

🔴 How For Loops Actually Work Internally

▼

This for loop:

for x in [1, 2, 3]:
    print(x)

Is equivalent to this while loop:

iterator = iter([1, 2, 3])
while True:
    try:
        x = next(iterator)
        print(x)
    except StopIteration:
        break

Understanding this helps you:

  • Make custom iterable classes
  • Understand lazy evaluation (generators)
  • Debug iteration behavior

🔴 enumerate(): Index + Item Together

▼

Often you need both the index and the item. Instead of manual indexing:

# The hard way
fruits = ["apple", "banana", "cherry"]
for i in range(len(fruits)):
    print(f"{i}: {fruits[i]}")

Use enumerate():

fruits = ["apple", "banana", "cherry"]
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

# Output:
# 0: apple
# 1: banana
# 2: cherry

You can also specify the starting index:

for i, fruit in enumerate(fruits, start=1):
    print(f"{i}: {fruit}")

# Output:
# 1: apple
# 2: banana
# 3: cherry

🔴 zip(): Iterate Over Multiple Sequences

▼

Pair up elements from multiple sequences:

names = ["Alice", "Bob", "Charlie"]
ages = [25, 30, 35]

for name, age in zip(names, ages):
    print(f"{name} is {age} years old")

# Output:
# Alice is 25 years old
# Bob is 30 years old
# Charlie is 35 years old

zip() stops at the shortest sequence:

a = [1, 2, 3, 4, 5]
b = ['a', 'b', 'c']

for num, letter in zip(a, b):
    print(f"{num}{letter}", end=" ")

# Output: 1a 2b 3c
# (4 and 5 are ignored)

Combine with enumerate():

names = ["Alice", "Bob"]
ages = [25, 30]

for i, (name, age) in enumerate(zip(names, ages)):
    print(f"{i}: {name} ({age})")

🔴 The Walrus Operator (:=) in Loops

▼

The walrus operator := (assignment expression) lets you assign and use a value in one expression:

# Without walrus
data = [1, 2, 3, 4, 5]
index = 0
while index < len(data):
    print(data[index])
    index += 1

# With walrus (Python 3.8+)
data = [1, 2, 3, 4, 5]
index = 0
while (value := data[index] if index < len(data) else None) is not None:
    print(value)
    index += 1

More useful example - reading until you get input:

# Assign and check in one go
while (user_input := input("Enter something: ")) != "quit":
    print(f"You said: {user_input}")

# Better than:
user_input = input("Enter something: ")
while user_input != "quit":
    print(f"You said: {user_input}")
    user_input = input("Enter something: ")

🔴 Complex Iteration: Sorting Within Loops

▼

Bubble Sort: Compare adjacent elements, swap if needed

def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                # Swap
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

print(bubble_sort([64, 34, 25, 12, 22]))  # [12, 22, 25, 34, 64]

Selection Sort: Find minimum, place at front

def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

print(selection_sort([64, 34, 25, 12, 22]))  # [12, 22, 25, 34, 64]

Socratic Q&A: Test Your Understanding

Click each question to reveal the answer. Think about it first!

1. What's the difference between a for loop and a while loop?

▼

For loops iterate over a sequence (list, range, string) - you know in advance what you're iterating over.

While loops repeat based on a boolean condition - you keep going until the condition becomes false.

for is cleaner when you know the sequence; while is better for open-ended scenarios (like waiting for user input).

2. What happens if you forget to update your counter in a while loop?

▼

You create an infinite loop - the condition never becomes false, so the loop repeats forever.

count = 0
while count < 5:
    print("Oops!")  # Forgot count += 1
    # This runs forever!

Always ensure something inside the loop will eventually make your condition false.

3. When should you use range(start, stop, step)?

▼

When you need to repeat a specific number of times with custom start/end/increment values.

  • range(5) - 5 times (0-4)
  • range(1, 5) - 4 times (1-4)
  • range(0, 10, 2) - every other number (0, 2, 4, 6, 8)
  • range(10, 0, -1) - countdown (10 down to 1)

4. What's the difference between break and continue?

▼

break exits the loop entirely - remaining iterations are skipped.

continue skips to the next iteration - the current iteration's remaining code is skipped, but the loop continues.

# break example
for i in range(5):
    if i == 3:
        break  # Loop stops, nothing after this
    print(i)  # Output: 0, 1, 2

# continue example
for i in range(5):
    if i == 2:
        continue  # Skip rest of iteration
    print(i)  # Output: 0, 1, 3, 4

5. When does the else block of a loop execute?

▼

The else block executes only if the loop completes normally without hitting break.

for i in range(3):
    print(i)
else:
    print("Done!")  # Executes (no break)

for i in range(3):
    if i == 1:
        break
    print(i)
else:
    print("Never prints")  # break was hit!

6. Write a loop to print all even numbers from 2 to 20

▼ 
# Option 1: Using step
for i in range(2, 21, 2):
    print(i)

# Option 2: Using condition
for i in range(2, 21):
    if i % 2 == 0:
        print(i)

7. What does enumerate() do? When would you use it?

▼

enumerate() gives you both the index and the value as you iterate.

fruits = ["apple", "banana", "cherry"]
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

# Output:
# 0: apple
# 1: banana
# 2: cherry

Use it: When you need to know the position of each item in a list.

8. How does zip() work with multiple lists?

▼

zip() pairs up elements from multiple sequences, iterating until the shortest one ends.

names = ["Alice", "Bob", "Charlie"]
scores = [95, 87]

for name, score in zip(names, scores):
    print(f"{name}: {score}")

# Output:
# Alice: 95
# Bob: 87
# (Charlie is skipped - no score)

9. Write a factorial function using a for loop

▼ 
def factorial(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result

print(factorial(5))  # 120 (5! = 5×4×3×2×1)

Key points: Start result at 1 (not 0), use range(1, n+1) to include n itself.

10. How would you find if a number is prime using a for loop?

▼

Check if any number from 2 to √n divides it evenly. If yes, not prime; if loop completes, it's prime.

def is_prime(n):
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False  # Found a divisor
    return True  # No divisors, is prime

print(is_prime(17))  # True
print(is_prime(12))  # False

11. What is the iteration protocol (__iter__ and __next__)?

▼

When you use a for loop, Python internally:

  1. Calls __iter__() on the sequence to get an iterator
  2. Repeatedly calls __next__() to get each item
  3. Catches StopIteration exception to know when done
# This for loop:
for x in [1, 2, 3]:
    print(x)

# Is equivalent to:
iterator = iter([1, 2, 3])  # Calls __iter__()
while True:
    try:
        x = next(iterator)  # Calls __next__()
        print(x)
    except StopIteration:
        break

12. Why would you use the walrus operator (:=) in a while loop?

▼

The walrus operator allows you to assign and check a value in one expression, making code cleaner.

# Without walrus
user_input = input("Enter something: ")
while user_input != "quit":
    print(f"You said: {user_input}")
    user_input = input("Enter something: ")  # Repeated!

# With walrus (Python 3.8+)
while (user_input := input("Enter something: ")) != "quit":
    print(f"You said: {user_input}")
    # Input is in parentheses, condition checks it

Key Takeaways

  • For loops are best when iterating over a sequence you know
  • While loops are best for open-ended repetition with a condition
  • Always ensure your loop will terminate (counter updates, condition changes)
  • break exits immediately; continue skips to next iteration
  • else after a loop runs only if no break occurred
  • range() is the standard way to repeat a fixed number of times
  • Use enumerate() when you need both index and value
  • Use zip() to pair up elements from multiple sequences
  • Understand the iteration protocol for deeper insight into how loops work
