Chapter 6 — Phase II: Flow Control

Decision Control — Conditionals

Master the if, elif, and else statements to direct program flow based on conditions and make intelligent decisions in code.

💡 Mental Model: Branching Paths

Imagine walking through a forest with multiple paths. At each fork in the road, you check a sign (the condition) that tells you which path to take. Python's conditionals are exactly like this: they evaluate conditions and choose execution paths accordingly. The program doesn't execute all paths; it evaluates the condition and follows only the matching branch.

Decision Control Instructions

Decision control instructions (or conditionals) allow programs to make choices based on conditions. Instead of executing code sequentially, conditionals enable branching: different code runs based on whether a condition is true or false.

The Three Forms of Conditionals

Python provides three fundamental conditional structures:

  • if: Execute block if condition is true
  • if-else: Execute one block if true, another if false
  • if-elif-else: Multiple conditions, execute first matching block

Basic if Statement

Execute a code block only if a condition is true:


age = 18

if age >= 18:
    print("You are an adult")

# Output: "You are an adult"

# Without the condition being true:
age = 15
if age >= 18:
    print("You are an adult")

# Output: (nothing printed)

if-else Statement

Execute one block if true, another if false:


temperature = 20

if temperature > 25:
    print("It's warm")
else:
    print("It's cool")

# Output: "It's cool"

# The else block is a fallback
score = 95
if score >= 90:
    print("Excellent!")
else:
    print("Good work")

# Output: "Excellent!"

if-elif-else Chain

Multiple conditions, testing each in order until one matches:


grade_percentage = 85

if grade_percentage >= 90:
    grade = "A"
elif grade_percentage >= 80:
    grade = "B"
elif grade_percentage >= 70:
    grade = "C"
elif grade_percentage >= 60:
    grade = "D"
else:
    grade = "F"

print(f"Your grade: {grade}")
# Output: "Your grade: B"

# Important: only the FIRST matching block executes
# If two conditions match, the earlier one wins!

Nuances of Conditions: Truthiness & Falsiness

In Python, any value can be evaluated in a boolean context. Most values are "truthy" (treated as True), but certain values are "falsy" (treated as False).

Falsy Values

These values are treated as False in conditionals:

Value Description Example
None The absence of value if None: → False
False Boolean false if False: → False
0, 0.0, 0j Zero (any numeric type) if 0: → False
'' (empty string) Empty sequence if '': → False
[] (empty list) Empty sequence if []: → False
{} (empty dict) Empty sequence if {}: → False
() (empty tuple) Empty sequence if (): → False

Using Truthiness Directly

You can use values directly without explicit comparison:


# Checking if string is empty
username = ""

if username:
    print(f"Welcome {username}")
else:
    print("Please enter a username")

# Output: "Please enter a username"

# More Pythonic than: if username != ""

# Checking if list has items
items = []

if items:
    for item in items:
        print(item)
else:
    print("No items in list")

# Output: "No items in list"

# Checking if value is None
result = None

if result is None:
    print("No result yet")
else:
    print(f"Result: {result}")

Truthy Values

Everything else is truthy, including:

  • Non-zero numbers: 1, -5, 3.14
  • Non-empty strings: "hello", " " (even single space)
  • Non-empty collections: [1], {'key': 'value'}
  • Objects and custom instances

# Truthy examples
if 1:
    print("1 is truthy")  # Prints

if "0":
    print('"0" is truthy')  # Prints (non-empty string!)

if [False]:
    print("[False] is truthy")  # Prints (non-empty list!)

Logical Operators: and, or, not

Logical operators combine or negate conditions. Python's logical operators have special behavior: they return operands, not just True/False.

and — Both Conditions True

Returns the first falsy value, or the last value if all are truthy:


# Using 'and' with booleans
age = 25
has_license = True

if age >= 18 and has_license:
    print("You can drive")
else:
    print("You cannot drive")

# Output: "You can drive"

# Important: 'and' returns operands, not just True/False
result = 5 and 10
print(result)  # 10 (both truthy, so returns last one)

result = 0 and 10
print(result)  # 0 (first falsy value)

result = "" and "hello"
print(result)  # "" (first falsy value)

# Short-circuit evaluation
def always_true():
    print("This runs")
    return True

def never_runs():
    print("This doesn't run")
    return True

# Short-circuit: never_runs() is never called
if always_true() and never_runs():
    print("Both true")

or — At Least One Condition True

Returns the first truthy value, or the last value if all are falsy:


# Using 'or' with booleans
is_admin = False
is_owner = True

if is_admin or is_owner:
    print("Access granted")
else:
    print("Access denied")

# Output: "Access granted"

# 'or' returns operands
result = 0 or 10
print(result)  # 10 (first falsy, returns second)

result = 5 or 10
print(result)  # 5 (first truthy, stops here)

result = "" or "default"
print(result)  # "default" (used for default values!)

# Practical: default values
username = "" or "Guest"
print(username)  # "Guest"

user_choice = None or "auto"
print(user_choice)  # "auto"

not — Negate a Condition

Returns the opposite boolean value:


# Using 'not'
is_connected = False

if not is_connected:
    print("Please connect to internet")
else:
    print("Already connected")

# Output: "Please connect to internet"

# 'not' always returns True or False
result = not True
print(result)  # False

result = not 0
print(result)  # True (0 is falsy)

result = not "hello"
print(result)  # False ("hello" is truthy)

# Double negation (rarely useful)
value = True
print(not not value)  # True

Operator Precedence

The order of evaluation matters. Use parentheses for clarity:


# Precedence: not > and > or

# Confusing without parentheses
result = True or False and False
print(result)  # True (and binds tighter than or)

# Better: use parentheses
result = (True or False) and False
print(result)  # False

result = True or (False and False)
print(result)  # True

# Always clarify with parentheses for readability!
is_admin = True
is_owner = False
is_moderator = True

if (is_admin or is_owner) and is_moderator:
    print("Moderator action allowed")

Conditional Expressions: Ternary Operator

Python's ternary operator provides a compact way to assign a value based on a condition.

Syntax: value_if_true if condition else value_if_false


age = 18
status = "adult" if age >= 18 else "minor"
print(status)  # "adult"

# More readable than:
# if age >= 18:
#     status = "adult"
# else:
#     status = "minor"

# Works with any expressions
score = 85
message = "Pass" if score >= 60 else "Fail"
print(message)  # "Pass"

# Can be chained
grade = 95
letter = "A" if grade >= 90 else "B" if grade >= 80 else "C"
print(letter)  # "A"

# But chaining gets hard to read—use if-elif-else instead!

Using in Function Calls


count = 1
print(f"You have {count} item" if count == 1 else f"You have {count} items")
# Output: "You have 1 item"

count = 5
print(f"You have {count} item" if count == 1 else f"You have {count} items")
# Output: "You have 5 items"

all() and any() Functions

These functions check multiple boolean conditions efficiently.

all() — All Values True

Returns True only if all items are truthy (or list is empty):


# All conditions met?
scores = [85, 90, 92, 88]

if all(score >= 80 for score in scores):
    print("All scores are above 80")
else:
    print("Some scores are below 80")

# Output: "All scores are above 80"

# With lists
print(all([True, True, True]))   # True
print(all([True, False, True]))  # False
print(all([]))                   # True (empty is all-true!)

# Practical: password validation
password = "SecurePass123!"
has_uppercase = any(c.isupper() for c in password)
has_lowercase = any(c.islower() for c in password)
has_digit = any(c.isdigit() for c in password)

if all([has_uppercase, has_lowercase, has_digit]):
    print("Password is strong")
else:
    print("Password needs more complexity")

any() — At Least One Value True

Returns True if at least one item is truthy:


# At least one condition met?
permissions = [False, False, True]

if any(permissions):
    print("At least one permission granted")
else:
    print("No permissions")

# Output: "At least one permission granted"

# Empty list returns False
print(any([False, False, False]))  # False
print(any([]))                     # False

# Practical: checking for errors
errors = []

if not any(errors):
    print("No errors found")
else:
    print(f"Found {len(errors)} errors")

Receiving Input: input() Function

The input() function reads user input from the keyboard. Crucially, it always returns a string, even if the user types a number.

Basic input()


# Get user input
name = input("What is your name? ")
print(f"Hello, {name}")

# Input is always a string
age_str = input("How old are you? ")
print(type(age_str))  # 
print(age_str)        # "25" (as string, not number)

# Wrong: you can't do arithmetic on string
# total = age_str + 5  # TypeError!

Converting Input to Numbers

Always convert input to the correct type:


# Convert to integer
age = int(input("Enter your age: "))
print(f"Next year you'll be {age + 1}")

# Convert to float
height = float(input("Enter your height in meters: "))
bmi = 70 / (height ** 2)  # Can now do math
print(f"BMI: {bmi:.1f}")

# Converting with defaults
try:
    score = int(input("Enter a score: "))
except ValueError:
    print("That's not a valid number")
    score = 0

Input with Conditionals


# Get age and check eligibility
age_str = input("Enter your age: ")
age = int(age_str)

if age >= 18:
    print("You are eligible to vote")
else:
    print(f"You can vote in {18 - age} years")

# Get choice and branch
choice = input("Enter 'A' or 'B': ").upper()

if choice == "A":
    print("You chose A")
elif choice == "B":
    print("You chose B")
else:
    print("Invalid choice")

The pass Statement

The pass statement is a null operation—when executed, nothing happens. It's useful as a placeholder.

When to Use pass

  • Empty code blocks (you can't leave them blank)
  • Planning stub code
  • Handling exceptions without action

# Placeholder: not yet implemented
if user_is_admin:
    pass  # TODO: implement admin actions

# Better style: add a comment or docstring
if user_is_admin:
    # Admin actions coming soon
    pass

# Handling exceptions
try:
    risky_operation()
except ValueError:
    pass  # Ignore this error type

# Function stub
def calculate_total(items):
    pass  # Will implement later

# Note: you'll often see 'pass' in class definitions
class Animal:
    pass  # Placeholder class

# If you remove 'pass', you get IndentationError
# if condition:
# next_line()  # ERROR: needs indented block!

Short-Circuit Evaluation & Optimization

Python uses short-circuit evaluation: in a and b, if a is falsy, b is never evaluated. This is more than optimization—it's essential for safety:


# Safe: avoiding division by zero
divisor = 0
if divisor != 0 and 100 / divisor > 50:
    print("Result is large")

# The second condition is never evaluated!
# If divisor is 0, the 'and' short-circuits

# Unsafe (would error):
# if 100 / divisor > 50 and divisor != 0:
#     print("Result is large")

# For 'or', short-circuits when first value is truthy
user_prefs = "" or "default_theme"
# The 'or' stops at the second value (the first truthy one)

Python 3.10+: match/case Structural Pattern Matching

Python 3.10 introduced match/case for pattern matching, offering a more powerful alternative to if-elif-else:


# Traditional if-elif-else
day = input("What day is it? ").lower()
if day == "monday":
    print("Start of work week")
elif day == "friday":
    print("Almost weekend!")
elif day in ["saturday", "sunday"]:
    print("Weekend!")
else:
    print("Midweek")

# Modern match/case (Python 3.10+)
match day:
    case "monday":
        print("Start of work week")
    case "friday":
        print("Almost weekend!")
    case "saturday" | "sunday":  # Multiple values with |
        print("Weekend!")
    case _:  # Default case (underscore is wildcard)
        print("Midweek")

# match/case with complex patterns
point = (3, 0)
match point:
    case (0, 0):
        print("Origin")
    case (0, y):
        print(f"Y-axis at {y}")
    case (x, 0):
        print(f"X-axis at {x}")
    case (x, y):
        print(f"Point at ({x}, {y})")

The Walrus Operator := in Conditions

The walrus operator (:=, introduced in Python 3.8) assigns a value within an expression:


# Traditional: need separate assignment
data = get_data()
if data:
    process(data)

# With walrus operator: assign and check in one line
if data := get_data():
    process(data)

# Practical: reading until empty
while (line := input("Enter text (or blank to quit): ")) != "":
    print(f"You entered: {line}")

# In list comprehensions
numbers = [1, 2, 3, 4, 5]
filtered = [y for x in numbers if (y := x * 2) > 5]
print(filtered)  # [6, 8, 10]

# Avoids calling the same function twice
if (length := len(very_expensive_function())) > 100:
    print(f"Length is {length}")

Identity vs Equality: is vs ==

The is operator checks object identity, while == checks value equality. They behave differently:


# Equality (==): compares values
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)  # True (same content)
print(a is b)  # False (different objects)

# Identity (is): compares object ID
x = None
if x is None:
    print("x is None")  # Use 'is' for None!

# Boolean: use 'is' with True/False
flag = True
if flag is True:
    print("flag is True")  # Safe

# Numbers: small numbers are interned (same object)
a = 257
b = 257
print(a is b)  # Usually False (not guaranteed)

a = 5
b = 5
print(a is b)  # True (small ints are cached)

# Rule: use 'is' for None, True, False
# Use '==' for everything else

Think Like a Compiler: Socratic Q&A

1
Why can you use truthy values directly in conditions without comparing to True/False?
+

Python's if statement evaluates conditions using truthiness rules, not strict boolean equality. Any value can be tested: the condition passes if the value is "truthy" (which includes most non-zero, non-empty values).

# These are equivalent: if name == True: print("Redundant!") # Better: let Python evaluate truthiness if name: print("More Pythonic!") # Works because non-empty strings are truthy name = "Alice" if name: print(f"Hello {name}") # Prints
2
Why does and return operands instead of just True/False?
+

This design enables elegant default-value patterns. By returning operands, or can serve as a fallback mechanism:

# Classic Python pattern: provide defaults theme = user_choice or "light" # If user_choice is falsy, use "light" port = config.get('port') or 8080 # If config returns None/empty, use 8080 # This wouldn't work if 'or' just returned True/False: theme = True or "light" # Would be True, not "light"!
3
When you write elif, can multiple branches execute?
+

No. In an if-elif-else chain, only the FIRST matching block executes. Once a condition is true, Python skips all remaining elif/else blocks:

x = 85 if x >= 90: print("A") elif x >= 80: print("B") # This executes elif x >= 70: print("C") # This does NOT execute else: print("F") # This does NOT execute # Output: "B" (only one) # To execute multiple blocks, use separate if statements: if x >= 80: print("Good score") # Executes if x >= 70: print("Passing") # Also executes
4
Why must input() always be converted to a number before arithmetic?
+

input() returns a string regardless of what the user types. Python doesn't assume "the user probably meant a number"—it treats everything as text until you explicitly convert:

# User types: 5 x = input("Number: ") # "5" (string, not 5) print(type(x)) # print(x + 3) # ERROR: can't add string and int # Must convert x = int(input("Number: ")) # Now it's 5 (integer) print(x + 3) # 8 (works!) # This strictness prevents silent bugs # Where unclear data types could cause chaos
5
What's the difference between if x == 0 and if x in a condition?
+

if x uses truthiness (0 is falsy), while if x == 0 explicitly checks for the value 0. They're equivalent for numbers but differ for other types:

# For numbers: equivalent x = 0 if x: # False (0 is falsy) if x == 0: # True (explicitly 0) # For other types: they differ x = "" # Empty string if x: # False (empty string is falsy) if x == 0: # False (string is not equal to 0) # Best practice: if x == 0: # For explicit comparison if not x: # For truthiness/falsiness if x is None: # For None checks
6
Why use all() and any() instead of chaining with and/or?
+

all() and any() are clearer when dealing with sequences or generator expressions, especially with many conditions:

# Tedious chaining: if x > 0 and y > 0 and z > 0 and w > 0: print("All positive") # Cleaner with all(): if all(val > 0 for val in [x, y, z, w]): print("All positive") # With lists: scores = [85, 90, 78, 92, 88] if all(score >= 70 for score in scores): print("All passing") # any() similarly: if any(score < 70 for score in scores): print("At least one failing")
7
When should you use conditional expressions (ternary) vs if-else statements?
+

Use ternary operators for simple, single-expression choices. For complex logic or multiple statements, use if-else:

# Good ternary use (assignment): status = "adult" if age >= 18 else "minor" # Good ternary use (function argument): print("Pass" if score >= 60 else "Fail") # Bad ternary use (complex logic): # result = action1() if check1 else action2() if check2 else action3() # Hard to read! # Better: if check1: result = action1() elif check2: result = action2() else: result = action3() # Rule: if it fits on one line clearly, use ternary # Otherwise, use if-else for readability
8
Why does not x exist when we could use x == False?
+

not checks truthiness, not strict equality. This is more useful for checking empty values:

# Truthiness check (what we usually want) if not items: print("List is empty") # Would fail with == if items == False: # Always False! print("This never prints") # 'not' works for any falsy value: if not "": print("Empty string is falsy") if not None: print("None is falsy") if not 0: print("Zero is falsy") # All work because they're truthy checks
9
What happens if you use and/or assuming they return True/False?
+

You'll get unexpected values when trying to use them in arithmetic or string operations:

# Expecting True/False: result = 5 and 10 print(result) # 10 (not True!) # If you try arithmetic: value = (0 or 20) * 2 print(value) # 40 (works because 20 * 2) # But with unpredictable values: result = ("hello" or "world") + " there" print(result) # "hello there" (works) result = (None or "world") + " there" print(result) # "world there" (works) # The key: don't rely on True/False when you mean something else # Use explicit boolean conversion if needed: result = bool(5 and 10) print(result) # True
10
How does short-circuit evaluation prevent errors in conditionals?
+

By not evaluating unnecessary expressions, short-circuit prevents operations that would error:

# Safe: checks divisor before dividing divisor = 0 if divisor != 0 and 100 / divisor > 50: print("Result is large") # Division never happens because divisor != 0 is False # Unsafe: division happens even if divisor is 0 # if 100 / divisor > 50 and divisor != 0: # ERROR! # print("Result is large") # Practical: checking list length items = [1, 2, 3] if items and items[0] > 0: print("First item is positive") # items check happens first; if items is empty, items[0] is skipped # Without short-circuit: # if items[0] > 0 and items: # ERROR if empty! # print("First item is positive")
11
When should you use is None instead of == None?
+

Always use is None. There's only one None object in Python, so identity comparison is correct and slightly faster:

result = None # Correct (and Pythonic) if result is None: print("No result") # Technically works but wrong style # if result == None: # print("No result") # Same for True/False if flag is True: # Correct if flag is False: # Correct # But usually just: if flag: # Is truthy if not flag: # Is falsy # PEP 8 (Python style guide) says: # Comparisons to singletons like None should always be done with is or is not # This is a style/convention, not a functional difference # But follow PEP 8 for consistency with professional code
12
What's the purpose of the pass statement if you can just leave a block empty?
+

Python requires at least one statement in code blocks (if, for, def, class, etc.). You can't leave them empty—pass is a do-nothing placeholder:

# This is a SYNTAX ERROR: if condition: # IndentationError: expected an indented block # This works: if condition: pass # Useful for stub implementations: def calculate_total(items): pass # TODO: implement later # For empty exception handling: try: risky_operation() except FileNotFoundError: pass # Ignore missing files # Without pass, Python interpreter cannot parse the code # It's not just convention—it's required by Python's syntax
