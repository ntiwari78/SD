Python Basics: Types & Operators

Variables, Data Types, and Operations

Feynman Mental Model

Variables as Labeled Boxes in a Warehouse

Imagine a massive warehouse with millions of storage bins. Each bin holds something—a number, text, a list of items. In your program, variables are like labels you put on these bins. The label says "this bin is called name" and "this bin is called age." The type tells you what kind of thing fits in the bin: numbers go in number bins, text goes in text bins, lists go in list bins. When you assign a value (x = 5), you're putting something in the bin and labeling it. When you use that variable (print(x)), you're reading what's in the labeled bin. Python's dynamic typing means you can relabel a bin—put a number in, then swap it for text. The bin doesn't care what shape the object is; the label just tracks what's inside.

Identifiers and Keywords

Identifiers: Naming Your Variables

An identifier is a name you give to a variable, function, class, or module. Python has rules for valid identifiers:

  • Must start with a letter (a-z, A-Z) or underscore (_) — Can't start with a number
  • Can contain letters, numbers, and underscores
  • Case-sensitive — age, Age, and AGE are three different variables
  • Can't use Python keywords — Reserved words like if, while, def
# Valid identifiers
name = "Alice"
_private_var = 42
user_age = 25
x1 = 10

# Invalid identifiers (will cause errors)
# 1st_place = "gold"    # Can't start with number
# user-name = "bob"     # Can't use hyphens
# class = 5             # 'class' is a keyword
# for = 10              # 'for' is a keyword

Python Keywords: Reserved Words

These 33 keywords are reserved—you cannot use them as variable names:

False  await  else      import   pass
None   break  except    in       raise
True   class  finally   is       return
and    continue for     lambda   try
as     def    from      nonlocal while
assert del    global    not      with
async  elif   if        or       yield

If you try to use a keyword as a variable name, Python raises a SyntaxError.

Naming Conventions

While Python doesn't enforce these, follow them for readable code:

  • Variables and functions: Use snake_case (lowercase with underscores) — user_name, calculate_total
  • Classes: Use PascalCase (capitalize each word) — UserProfile, BankAccount
  • Constants: Use UPPER_SNAKE_CASE — MAX_RETRIES, PI
  • Private variables: Start with underscore — _internal_var

Python's Type System: Three Categories

Python has three main categories of types:

1. Basic Types (Scalar)

Single values representing fundamental data:

  • int — Whole numbers (integers): 42, -5, 0, 1000000
  • float — Decimal numbers: 3.14, -0.5, 2.0
  • complex — Complex numbers: 3+2j (3 real + 2i imaginary)
  • bool — Boolean values: True or False
  • str — Text strings: "hello", 'world', """multi-line"""
  • bytes — Binary data: b'\x48\x65\x6c\x6c\x6f'

2. Container Types (Collections)

Hold multiple values together:

  • list — Ordered, mutable: [1, 2, 3] or ["a", "b", "c"]
  • tuple — Ordered, immutable: (1, 2, 3)
  • set — Unordered, unique: {1, 2, 3}
  • dict — Key-value mappings: {"name": "Alice", "age": 25}

3. User-Defined Types

Classes you create yourself (covered later). Custom objects with their own data and behavior.

Basic Types: Understanding Each

Integers (int)

Whole numbers with no fractional part. Python supports arbitrary precision—integers can be as large as memory allows.

# Decimal (base 10) - most common
decimal = 42
negative = -17

# Binary (base 2) - prefix with 0b
binary = 0b1010  # Same as 10 in decimal

# Octal (base 8) - prefix with 0o
octal = 0o755    # File permissions

# Hexadecimal (base 16) - prefix with 0x
hexadecimal = 0xFF  # Same as 255 in decimal

# Arbitrary precision - no overflow!
huge_number = 123456789012345678901234567890
result = huge_number ** 2
print(len(str(result)))  # Works fine, no overflow error
Integer Representation Note

All these bases represent the same value internally. Python only cares about the numeric value, not how you wrote it. 10, 0b1010, 0o12, and 0xA are all equal.

Floats (float)

Decimal numbers. Internally stored as IEEE 754 double precision (64-bit) with ~15-17 significant digits.

# Regular decimal notation
pi = 3.14159
gravity = 9.81
zero = 0.0

# Exponential notation
scientific = 1.23e-4  # 0.000123
large = 6.02e23       # Avogadro's number

# Special float values
infinity = float('inf')
not_a_number = float('nan')
print(infinity > 1000000)  # True
print(not_a_number == not_a_number)  # False (NaN is weird)
Floating Point Precision

Floats can't represent all decimals exactly. 0.1 + 0.2 doesn't equal 0.3—it equals 0.30000000000000004 due to binary representation. For financial calculations, use the decimal module.

Complex Numbers (complex)

Numbers with real and imaginary parts, useful in engineering and physics.

# Imaginary unit is 'j' in Python (not 'i')
z1 = 3 + 2j
z2 = complex(1, -1)

# Operations work as expected
print(z1 + z2)  # (4+1j)
print(z1 * z2)  # (5+1j)
print(abs(z1))  # 3.605551... (magnitude)

Booleans (bool)

Only two values: True and False. Used for conditions and comparisons. Technically a subclass of int where True==1 and False==0.

is_adult = True
is_expired = False

# Booleans result from comparisons
x = 10
print(x > 5)   # True
print(x == 5)  # False

# Booleans in arithmetic (True=1, False=0)
print(True + True)   # 2
print(True * 5)      # 5
print(False + 10)    # 10

Strings (str)

Text enclosed in quotes. Immutable (can't change after creation) and can use single, double, or triple quotes.

# Different quote styles
single = 'hello'
double = "world"
triple = '''This is a
multi-line
string'''

# Concatenation with +
greeting = "Hello" + " " + "World"

# Repetition with *
stars = "*" * 10  # "**********"

# Escape characters
newline = "Line 1\nLine 2"
tab = "Col1\tCol2"
quote = 'He said "Hi"'

# String formatting (modern f-strings)
name = "Alice"
age = 25
message = f"{name} is {age} years old"

Bytes (bytes)

Binary data, used for file I/O and network communication. Immutable sequence of integers 0-255.

# Create from string
encoded = "hello".encode('utf-8')
print(encoded)  # b'hello'

# Create directly
binary = b'\x48\x65\x6c\x6c\x6f'  # "Hello" in hex

# Access individual bytes (returns int)
print(binary[0])  # 72 (ASCII value of 'H')

Checking Types with type()

The built-in type() function tells you an object's type:

x = 42
print(type(x))  # 

y = 3.14
print(type(y))  # 

name = "Alice"
print(type(name))  # 

flag = True
print(type(flag))  # 

items = [1, 2, 3]
print(type(items))  #

Integer and Float Ranges

Integer Range: Arbitrary Precision

Python integers have unlimited precision. Unlike Java or C, there's no overflow.

# Start small
x = 2 ** 10  # 1024

# Go huge
x = 2 ** 100  # 1267650600228229401496703205376
# Python handles it fine

# Even bigger
x = 2 ** 1000  # A number with 302 digits
# Still works! Limited only by available memory

Float Range: Limited by IEEE 754

Python floats are 64-bit IEEE 754 doubles with limits:

  • Maximum: approximately 1.8 × 10^308
  • Minimum (positive): approximately 2.2 × 10^-308
  • Precision: ~15-17 significant digits
import sys

print(sys.float_info.max)   # 1.7976931348623157e+308
print(sys.float_info.min)   # 2.2250738585072014e-308

# Exceeding range gives infinity
huge = 10.0 ** 400
print(huge)  # inf

# Underflow gives 0
tiny = 10.0 ** -500
print(tiny)  # 0.0
Why Not Everything is a Float

Floats have limited precision and range. Integers are exact. For large numbers, integer arithmetic is better than float arithmetic.

Dynamic Typing and Variable Assignment

Type Inference

Python automatically determines a variable's type based on what you assign.

x = 42          # x is now an int
x = 3.14        # x is now a float (type changed!)
x = "hello"     # x is now a str (type changed again!)
x = True        # x is now a bool

# This is valid and normal in Python
# The variable is just a label; the type is on the object

Multiple Assignment

Assign to multiple variables in one statement:

# Parallel assignment
x, y, z = 1, 2, 3
print(x)  # 1
print(y)  # 2
print(z)  # 3

# Useful for swapping without a temp variable
a, b = 10, 20
a, b = b, a  # Swap!
print(a, b)  # 20, 10

# Unpacking from sequences
colors = ["red", "green", "blue"]
r, g, b = colors
print(r)  # "red"

Chained Assignment

Assign the same value to multiple variables:

# All variables get the same value
x = y = z = 0
print(x, y, z)  # 0, 0, 0

# Modify one - others unchanged (for immutable types)
x = 10
print(x, y, z)  # 10, 0, 0

Arithmetic Operators

Python supports standard mathematical operations:

Operator Operation Example Result
+ Addition 10 + 3 13
- Subtraction 10 - 3 7
* Multiplication 10 * 3 30
/ Division (always float result) 10 / 3 3.333...
// Floor Division (rounds toward -infinity) 10 // 3 3
% Modulo (remainder) 10 % 3 1
** Exponentiation (power) 2 ** 3 8 
# Basic arithmetic
print(10 + 3)      # 13
print(10 - 3)      # 7
print(10 * 3)      # 30

# Division is always float
print(10 / 3)      # 3.3333...
print(10 / 2)      # 5.0 (not 5!)

# Floor division rounds toward -infinity
print(10 // 3)     # 3
print(-10 // 3)    # -4 (not -3!)

# Modulo: returns remainder
print(10 % 3)      # 1 (10 = 3*3 + 1)
print(-10 % 3)     # 2 (different from C!)

# Exponentiation (right-associative)
print(2 ** 3)      # 8
print(2 ** 3 ** 2) # 512 (not 64) - evaluated right to left

Operation Nuances

Division Always Returns Float

In Python 3, / always returns a float, even if the result is a whole number. Use // for integer division.

Floor Division Rounds Toward Negative Infinity

Not toward zero like in C. Important for negative numbers.

print(10 // 3)   # 3 (10 / 3 = 3.33, floor is 3)
print(-10 // 3)  # -4 (-10 / 3 = -3.33, floor is -4)
print(10 // -3)  # -4 (10 / -3 = -3.33, floor is -4)

Modulo with Negative Numbers

Result takes the sign of the divisor (not the dividend like in C).

print(10 % 3)    # 1
print(-10 % 3)   # 2 (not -1!)
print(10 % -3)   # -2 (not 1!)

Operator Precedence and Associativity

Python follows PEMDAS order, with some specifics:

Precedence (High to Low) Operator(s) Associativity
1 (Highest) ** (exponentiation) Right-to-left (important!)
2 +x, -x, ~x (unary) Right-to-left
3 *, /, //, % Left-to-right
4 +, - (binary) Left-to-right
5 Comparisons: <, <=, >, >=, ==, != Left-to-right
6 not (logical NOT) Right-to-left
7 and (logical AND) Left-to-right
8 (Lowest) or (logical OR) Left-to-right 
# Exponentiation is right-associative
print(2 ** 3 ** 2)  # 512 (= 2 ** (3 ** 2) = 2 ** 9)
print((2 ** 3) ** 2)  # 64 (different!)

# Standard arithmetic order
print(2 + 3 * 4)    # 14 (not 20) - multiply first
print((2 + 3) * 4)  # 20 - parentheses override

# Chains of comparisons work
x = 5
print(0 < x < 10)   # True (evaluated as (0 < x) and (x < 10))

Type Conversions

Convert between types using built-in functions:

# int() - convert to integer
print(int(3.14))        # 3 (truncates, doesn't round)
print(int("42"))        # 42
print(int("0xFF", 16))  # 255 (parse as hex)

# float() - convert to float
print(float(42))        # 42.0
print(float("3.14"))    # 3.14

# str() - convert to string
print(str(42))          # "42"
print(str(3.14))        # "3.14"

# bool() - convert to boolean
print(bool(1))          # True
print(bool(0))          # False
print(bool("text"))     # True
print(bool(""))         # False (empty string is False)
print(bool([]))         # False (empty list is False)

# complex() - convert to complex
print(complex(2, 3))    # (2+3j)
Truthiness in Python

Falsy values: 0, 0.0, "", [], {}, None. Everything else is truthy. This matters in conditions: if items: checks if the list is non-empty.

Python provides essential functions for common operations:

Container Types Overview

Python provides four built-in container types:

Lists: [1, 2, 3]

Ordered, mutable (changeable). Most common for collections of items.

fruits = ["apple", "banana", "cherry"]
fruits[0] = "orange"  # Mutable - can change
fruits.append("grape")  # Can add/remove

Tuples: (1, 2, 3)

Ordered, immutable (unchangeable). Use when you need fixed data that shouldn't change.

coordinates = (10, 20, 30)
# coordinates[0] = 5  # Error! Tuples are immutable

Sets: {1, 2, 3}

Unordered, unique values. No duplicates allowed.

unique = {1, 2, 3, 1, 2}
print(unique)  # {1, 2, 3} - duplicates removed

Dicts: {"key": value}

Key-value mappings. Like a real-world dictionary.

person = {"name": "Alice", "age": 25, "city": "NYC"}
print(person["name"])  # "Alice"
person["age"] = 26  # Modify

Comments and Indentation

Comments

Text ignored by Python. Use # for single-line comments:

# This is a comment
x = 5  # Comment at end of line

# Comments should explain WHY, not WHAT
# GOOD: Add 10% tax to total
total = subtotal * 1.1

# BAD: This line adds
total = subtotal * 1.1

Indentation: Significant Whitespace

Python uses indentation to define code blocks. This is enforced syntax, not optional style.

if x > 0:
    print("Positive")  # Indented - part of if
    print("It's greater than zero")  # Still indented

print("Done")  # Not indented - outside if

# Wrong indentation = SyntaxError
# if x > 0:
# print("Positive")  # Error! Not indented

Indentation rules:

  • Use 4 spaces per indentation level (PEP 8 standard)
  • Be consistent—all spaces or all tabs (mixing = error)
  • Increase indentation for code blocks (if, for, function definitions)

Multi-Line Statements

Long statements can span multiple lines:

# Method 1: Backslash continuation
total = 10 + 20 + 30 + \
        40 + 50 + 60
print(total)  # 210

# Method 2: Implicit continuation in parentheses
total = (10 + 20 + 30 +
         40 + 50 + 60)
print(total)  # 210

# Method 3: Inside brackets
items = [1, 2, 3,
         4, 5, 6]
print(items)  # [1, 2, 3, 4, 5, 6]

Classes and Objects: Introduction

Everything in Python is an object—even numbers and strings are instances of classes.

# Even primitives are objects
x = 42
print(type(x))  # 
print(dir(x))   # Shows all methods and attributes

# Strings have methods
text = "hello"
print(text.upper())       # "HELLO"
print(text.capitalize())  # "Hello"

# Lists have methods
items = [1, 2, 3]
items.append(4)  # Modifies the list
print(items)  # [1, 2, 3, 4]

We'll explore creating custom classes in later chapters. For now, understand that even basic types like int and str are objects with methods and attributes.

DEEP DIVE PyObject Internals: How Python Stores Everything

At the C level, CPython represents every value with a PyObject struct containing:

typedef struct {
    int ob_refcnt;        // Reference count for garbage collection
    PyTypeObject *ob_type; // Points to the object's type (class)
} PyObject;

Reference Counting

Every object tracks how many references point to it. When count reaches zero, Python frees the memory. This is why manual memory management isn't needed.

import sys
x = []
print(sys.getrefcount(x))  # How many references?
y = x  # Another reference
print(sys.getrefcount(x))  # Count increased by 1

The Small Integer Cache

Python caches integers -5 to 256 for performance. The same object is reused:

a = 5
b = 5
print(a is b)  # True (same object in memory)

c = 257
d = 257
print(c is d)  # False (different objects)

IEEE 754 Binary Representation

Floats are stored as 64-bit IEEE 754 doubles with 3 parts:

  • Sign bit (1 bit): positive or negative
  • Exponent (11 bits): magnitude
  • Mantissa (52 bits): precision

This is why 0.1 + 0.2 ≠ 0.3 and why floats have limited precision.

Think Like a Compiler

When you write x = 5 + 3, what happens internally?

  1. Tokenization: Split into tokens: IDENTIFIER(x), ASSIGN, NUMBER(5), PLUS, NUMBER(3)
  2. Parsing: Build a tree: ASSIGN(IDENTIFIER(x), BINARY_OP(PLUS, NUMBER(5), NUMBER(3)))
  3. Compilation: Generate bytecode to load 5, load 3, add them, store result
  4. Execution:
    • Python VM loads the integer object 5
    • Loads integer object 3
    • Calls int.__add__(5, 3) (the + operator is a method call!)
    • Creates a new integer object containing 8
    • Stores a reference to this object in the variable x

This is why Python is slower than C—every operation involves method lookups, type checking, and memory management. But it's more flexible and safer.

Socratic Q&A: Test Your Understanding

Work through these questions to solidify your grasp of Python types and operations.

1. What are the three categories of Python types? Give two examples of each.

Answer:

  • Basic/Scalar Types: Single values — int, float, complex, bool, str, bytes
  • Container Types: Multiple values — list, tuple, set, dict
  • User-Defined Types: Custom classes you create
2. Explain the difference between / and // operators. Why does Python distinguish them?

Answer:

  • / (division) always returns a float: 10 / 3 = 3.333...
  • // (floor division) returns an integer (floor rounded toward -infinity): 10 // 3 = 3

Python 3 distinguished them because division and integer division serve different purposes. In Python 2, 10/3 returned 3 (confusing). Python 3 clarified: use / for true division (float result), // for integer division.

3. Integers in Python have unlimited precision. Floats don't. Why is this important?

Answer: Integers can represent any magnitude exactly (limited only by RAM). Floats are 64-bit IEEE 754 with max ~1.8×10^308. For huge calculations (cryptography, big integers), use int. For scientific data with decimals, float is fine but remember:

  • Limited precision: ~15 significant digits
  • Can't represent all decimals: 0.1 + 0.2 = 0.30000000000000004
  • For financial calculations, use the decimal module
4. What is the difference between mutable and immutable types? List three mutable and three immutable types.

Answer:

Mutable: Can change after creation. Examples: list, dict, set. You can modify contents without creating a new object.

items = [1, 2, 3]
items[0] = 99  # Modify in place

Immutable: Cannot change after creation. Examples: int, float, str, tuple, bool, bytes. You can't modify; you create new objects.

x = 5
x = 10  # New object, not modification
text = "hello"
text = "world"  # New object

This matters for performance (immutable can be cached/optimized) and for data structures (immutable can be dict keys or set members).

5. Explain operator precedence and associativity. Why is exponentiation right-associative?

Answer: Precedence determines evaluation order. 2 + 3 * 4 = 14 (multiply before add). Associativity breaks ties: 10 - 5 - 2 = 3 (left-to-right: (10-5)-2). Exponentiation is right-to-left: 2^3^2 = 2^(3^2) = 2^9 = 512, not (2^3)^2 = 64. This matches mathematical convention: tower of exponents means "build from top down."

6. Write code that demonstrates dynamic typing. Show a variable changing types.

Answer:

x = 42
print(type(x))  # 

x = 3.14
print(type(x))  # 

x = "hello"
print(type(x))  # 

x = [1, 2, 3]
print(type(x))  # 

# The variable x is just a label
# The type is on the object
# You can reassign to any type at any time
7. Explain the difference between print() and input(). What does input() return?

Answer:

  • print() outputs to console. Takes any values, displays them.
  • input() reads from user. Always returns a STRING, even if user types a number.
age = input("Age: ")  # User types: 25
print(type(age))      #  - it's "25", not 25
print(age + 1)        # Error! Can't add str and int

# Must convert:
age = int(input("Age: "))
print(age + 1)  # Works!
8. What is the purpose of the id() function? What does it return?

Answer: id() returns an object's identity—its memory address in CPython. Used to check if two variables refer to the same object (use the is operator instead, usually).

x = [1, 2, 3]
y = [1, 2, 3]
print(id(x) == id(y))  # False - different objects
print(x == y)          # True - same contents
print(x is y)          # False - not the same object

z = x
print(id(x) == id(z))  # True - same object
print(x is z)          # True
9. True or False: The type() function can be used at runtime to check a variable's type. Is this good practice?

Answer: True—you can check types at runtime with type(x) or isinstance(x, int). But it's often bad practice. Python follows "duck typing"—if it walks like a duck and quacks like a duck, it's a duck. Check what an object CAN DO, not what its type is. Use isinstance() if you must check, not type() (isinstance respects inheritance).

# Bad: checking type directly
if type(x) == int:
    print("It's an integer")

# Better: checking capability (duck typing)
if hasattr(x, '__len__'):
    print("It's a sequence")

# Good: isinstance if you need to check
if isinstance(x, (int, float)):
    print("It's a number")
10. Explain what happens when you do 10 % 3 vs -10 % 3. Why does -10 % 3 = 2, not -1?

Answer: Modulo in Python always returns a result with the sign of the divisor. This is different from C and ensures a useful mathematical property: (a // b) * b + (a % b) always equals a.

# Positive divisor
print(10 % 3)   # 1 because 10 = 3*3 + 1
print(-10 % 3)  # 2 because -10 = 3*(-4) + 2 (result has sign of divisor)

# Verify the property
a, b = -10, 3
print((a // b) * b + (a % b))  # -10 (equals a)

This design is more mathematically useful than C-style (sign of dividend). It ensures consistent behavior in modular arithmetic.

11. What are the falsy values in Python? Why does bool([]) evaluate to False?

Answer: Falsy values (evaluate to False in boolean context):

  • None
  • False
  • 0 (int), 0.0 (float), 0j (complex)
  • "" (empty string)
  • [] (empty list)
  • {} (empty dict)
  • () (empty tuple)
  • set() (empty set)

Empty containers are falsy because emptiness semantically means "nothing" or "absent". Everything else is truthy. This enables clean code:

if items:         # True if list is non-empty
    print("Has items")

if not errors:    # True if list is empty
    print("No errors")
12. Explain the difference between "42", '42', """42""". Are they the same?

Answer: All three are identical strings. Python allows single, double, and triple quotes for strings:

  • "text" — double quotes
  • 'text' — single quotes (identical to double)
  • """text""" — triple quotes (can span multiple lines)
a = "42"
b = '42'
c = """42"""
print(a == b == c)  # True - all identical

# Triple quotes useful for multi-line strings
poem = """Roses are red
Violets are blue"""

# And useful for strings containing quotes
message = """He said "Hello" """
