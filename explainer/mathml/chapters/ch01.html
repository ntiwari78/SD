Vectors and Vector Spaces

From Iris datasets to abstract algebraic structures

~25 min read
6 code blocks
5 Socratic questions
The Feynman Bridge — Mental Model

Think of vectors as recipe cards

Imagine you're in a kitchen, and every recipe is written on a card. A chocolate cake recipe might say: 2 cups flour, 1 cup sugar, 3 eggs, half a cup of cocoa. That list of four numbers — (2, 1, 3, 0.5) — is a vector. Each number sits in a specific slot that means something. You can't swap the flour with the eggs and get the same cake.

Now imagine you want to bake for twice as many people. You'd double every number: (4, 2, 6, 1). That's scalar multiplication — multiplying every ingredient by the same amount. What if you want to combine two recipes into a party platter? Just add the numbers slot-by-slot: cake (2, 1, 3, 0.5) + cookies (1, 2, 0, 0.3) = combined shopping list (3, 3, 3, 0.8). That's vector addition.

The entire cookbook — every possible recipe you could ever write — is a vector space. It follows certain rules: you can always scale a recipe, combine two recipes, and there's a "do nothing" recipe (0, 0, 0, 0) that adds nothing. That's really all there is to it. The rest of this chapter makes these kitchen-table ideas mathematically precise.

Section 1.1

What is a Vector Space?

The mathematics of machine learning rests on three pillars: linear algebra, calculus, and probability theory. Linear algebra describes how to represent and manipulate data; calculus helps us fit models; probability helps us interpret them. We begin with representing data.

To ground our study, consider the famous Iris dataset — 150 measurements of three flower species, each described by four features: sepal length, sepal width, petal length, and petal width. In code:

Python
from sklearn.datasets import load_iris  # Load the classic ML benchmark dataset

data = load_iris()                      # Returns a Bunch object with data + labels
X, y = data["data"], data["target"]     # X = measurements (matrix), y = species (labels)
X[:10]                                  # Preview first 10 samples — each is a 4D vector

Each row is a data point — a list of four measurements. In mathematical terms, the entire dataset forms a matrix, and each data point forms a vector. Representing multiple measurements as a tuple \((x_1, x_2, \ldots, x_n)\) is a natural idea. Any two tuples \(\mathbf{x} = (x_1, \ldots, x_n)\) and \(\mathbf{y} = (y_1, \ldots, y_n)\) can be added together:

$$\mathbf{x} + \mathbf{y} = (x_1 + y_1, \ldots, x_n + y_n)$$

and multiplied with scalars: for \(c \in \mathbb{R}\),

$$c\mathbf{x} = (cx_1, \ldots, cx_n)$$

Geometrically, addition is translation and scalar multiplication is stretching (or squeezing, if \(|c| < 1\)). These two operations are the heartbeat of everything that follows.

Now let's formalize this. The definition below looks heavy, but every axiom just says "vectors behave like numbers."

Definition 1.1.1
Vector Spaces

A vector space is a mathematical structure \((V, F, +, \cdot)\), where:

  • \(V\) is the set of vectors,
  • \(F\) is a field of scalars (most commonly \(\mathbb{R}\) or \(\mathbb{C}\)),
  • \(+ : V \times V \to V\) is addition, satisfying:
    • \(\mathbf{x} + \mathbf{y} = \mathbf{y} + \mathbf{x}\) (commutativity)
    • \(\mathbf{x} + (\mathbf{y} + \mathbf{z}) = (\mathbf{x} + \mathbf{y}) + \mathbf{z}\) (associativity)
    • There exists \(\mathbf{0} \in V\) such that \(\mathbf{x} + \mathbf{0} = \mathbf{x}\) (null vector)
    • For each \(\mathbf{x}\), there exists \(-\mathbf{x}\) such that \(\mathbf{x} + (-\mathbf{x}) = \mathbf{0}\) (additive inverse)
  • \(\cdot : F \times V \to V\) is scalar multiplication, satisfying:
    • \(a(b\mathbf{x}) = (ab)\mathbf{x}\) (associativity)
    • \(a(\mathbf{x} + \mathbf{y}) = a\mathbf{x} + a\mathbf{y}\) (distributivity)
    • \(1\mathbf{x} = \mathbf{x}\)

Examples of Vector Spaces

Example 1. The most ubiquitous instance is \((\mathbb{R}^n, \mathbb{R}, +, \cdot)\). When \(n = 2\), we're talking about the familiar Euclidean plane. This is the canonical model guiding our studies.

Example 2. The space of polynomials with real coefficients, \(\mathbb{R}[x] = \bigl\{\sum_{i=0}^n p_i x^i : p_i \in \mathbb{R}\bigr\}\), is a vector space. Two polynomials are added coefficient-by-coefficient.

Example 3. The set of all continuous real functions \(C([0,1])\) forms a vector space with pointwise addition: \((f+g)(x) := f(x) + g(x)\). Functions can be vectors too!

Section 1.2

The Basis

Although vector spaces contain infinitely many vectors, we can reduce the complexity by finding special subsets that can express any other vector. In \(\mathbb{R}^n\), the standard unit vectors \(\mathbf{e}_1 = (1,0,\ldots,0)\), \(\mathbf{e}_2 = (0,1,\ldots,0)\), ..., \(\mathbf{e}_n = (0,0,\ldots,1)\) let us write every vector as:

$$\mathbf{x} = \sum_{i=1}^{n} x_i \mathbf{e}_i, \quad x_i \in \mathbb{R}, \quad \mathbf{e}_i \in \mathbb{R}^n$$

A vector is not the same as its coordinates — a single vector can have different coordinates in different systems. This is why bases are fundamental.

Linear Combinations & Independence

Sums of the form \(\sum_{i=1}^n x_i \mathbf{v}_i\) — where the \(\mathbf{v}_i\) are vectors and the \(x_i\) are scalars — are called linear combinations. A set of vectors can potentially represent the same vector in multiple ways. For example, with \(\mathbf{v}_1 = (1,0)\), \(\mathbf{v}_2 = (0,1)\), \(\mathbf{v}_3 = (1,1)\):

$$(2,1) = 2\mathbf{v}_1 + \mathbf{v}_2 = \mathbf{v}_1 + \mathbf{v}_3$$

This redundancy is formalized by linear dependence:

Definition 1.2.1
Linear Dependence and Independence

Let \(V\) be a vector space and \(S = \{\mathbf{v}_1, \ldots, \mathbf{v}_n\}\) a subset. \(S\) is linearly dependent if it only contains the zero vector, or if there is a nonzero \(\mathbf{v}_k\) that can be expressed as a linear combination of the others. \(S\) is linearly independent if it is not linearly dependent.

Theorem 1.2.1

A set \(S = \{\mathbf{v}_1, \ldots, \mathbf{v}_n\}\) is linearly dependent if and only if the null vector \(\mathbf{0}\) can be written as a nontrivial linear combination of the elements. That is, \(\mathbf{0} = \sum_{i=1}^n x_i \mathbf{v}_i\) where not all \(x_i\) are zero.

Spans

The set of all possible linear combinations of a set \(S\) is called its span:

$$\text{span}(S) = \left\{ \sum_{i=1}^{n} x_i \mathbf{v}_i \;:\; n \in \mathbb{N},\; \mathbf{v}_i \in S,\; x_i \text{ is a scalar} \right\}$$

In three dimensions, the span of two linearly independent vectors is a plane through the origin. Visually, you can reach any point on that plane by combining the two vectors with the right weights.

Proposition 1.2.1

(a) If \(S_1 \subseteq S_2\), then \(\text{span}(S_1) \subseteq \text{span}(S_2)\).

(b) \(\text{span}(\text{span}(S)) = \text{span}(S)\).

Proof. Property (a) follows directly from the definition. For (b), we show \(\text{span}(\text{span}(S)) \subseteq \text{span}(S)\) and \(\text{span}(S) \subseteq \text{span}(\text{span}(S))\).

The latter is immediate. For the former: if \(\mathbf{x} \in \text{span}(\text{span}(S))\), then \(\mathbf{x} = \sum_i x_i \mathbf{v}_i\) where each \(\mathbf{v}_i \in \text{span}(S)\). So \(\mathbf{v}_i = \sum_j v_{i,j} \mathbf{u}_j\) for some \(\mathbf{u}_j \in S\). Substituting:

$$\mathbf{x} = \sum_i x_i \sum_j v_{i,j}\mathbf{u}_j = \sum_j \left(\sum_i x_i v_{i,j}\right)\mathbf{u}_j$$

which is a linear combination of elements of \(S\), so \(\mathbf{x} \in \text{span}(S)\). ∎

Bases — The Minimal Generating Sets

A set \(S\) with \(\text{span}(S) = V\) is called a generating set. We want the smallest possible one:

Definition 1.2.2
Basis

A subset \(S\) of a vector space \(V\) is a basis if: (a) \(S\) is linearly independent, and (b) \(\text{span}(S) = V\). The elements of \(S\) are called basis vectors.

Every vector can be uniquely written as a linear combination of basis vectors. In \(\mathbb{R}^3\), the set \(\{(1,0,0), (0,1,0), (0,0,1)\}\) is the standard basis, but \(\{(1,1,1), (1,1,0), (0,1,1)\}\) is also a valid basis. Multiple bases exist for the same space.

Theorem 1.2.2

Let \(S = \{\mathbf{v}_1, \ldots, \mathbf{v}_n\}\) be a subset of vectors. The following are equivalent:

(a) \(S\) is a basis.

(b) \(S\) is linearly independent and for any \(\mathbf{x} \in V \setminus S\), the set \(S \cup \{\mathbf{x}\}\) is linearly dependent. (That is, \(S\) is a maximal linearly independent set.)

Proof. (a) ⟹ (b): If \(S\) is a basis, any \(\mathbf{x} \in V\) can be written as \(\mathbf{x} = \sum x_i \mathbf{v}_i\), so \(S \cup \{\mathbf{x}\}\) is linearly dependent by definition.

(b) ⟹ (a): By assumption, \(S \cup \{\mathbf{x}\}\) is linearly dependent for any \(\mathbf{x}\). So \(\mathbf{0} = \alpha\mathbf{x} + \sum x_i \mathbf{v}_i\) with not all coefficients zero. Since \(S\) is linearly independent, \(\alpha \neq 0\). Thus \(\mathbf{x} = -\sum \frac{x_i}{\alpha}\mathbf{v}_i\), showing \(\text{span}(S) = V\). ∎

Finite Dimensional Spaces

All bases of the same vector space have the same size. This fundamental fact gives us the dimension:

Theorem 1.2.4

Let \(S_1\) and \(S_2\) be two bases of a vector space \(V\). Then \(|S_1| = |S_2|\).

The dimension of \(V\), written \(\dim(V)\), is the number of elements in any basis. For example, \(\mathbb{R}^n\) is \(n\)-dimensional. Vector spaces with finite bases are called finite-dimensional. A crucial fact: all finite-dimensional real vector spaces are essentially \(\mathbb{R}^n\) (they are isomorphic).

Why Bases Matter

If every finite-dimensional real vector space is just \(\mathbb{R}^n\), why bother with abstraction? Because it reveals why things are defined the way they are. Consider matrix multiplication — seemingly arbitrary rules about summing products. But through the lens of bases and linear transformations (Chapter 4), the formula becomes inevitable: the \((i,j)\)-th element of the product \(AB\) is

$$\sum_{k=1}^{n} a_{i,k}\, b_{k,j}$$

because matrix multiplication describes the composition of linear transformations. Understanding bases is the key that unlocks this insight.

Theorem 1.2.5

Every vector space has a basis.

Subspaces

Definition 1.2.3
Subspaces

Let \(V\) be a vector space. A set \(U \subseteq V\) is a subspace if it is closed under addition and scalar multiplication. \(U\) is a proper subspace if \(U \subset V\).

Every vector space has at least two trivial subspaces: itself and \(\{\mathbf{0}\}\). The span of any set of vectors is always a subspace. Two subspaces can be combined via the direct sum:

Definition 1.2.4
Direct Sum of Subspaces

For subspaces \(U_1, U_2\) of \(V\): \(U_1 + U_2 = \{\mathbf{u}_1 + \mathbf{u}_2 : \mathbf{u}_1 \in U_1, \mathbf{u}_2 \in U_2\}\).

Section 1.3

Vectors in Practice

Theory gives us a language for reasoning about structure. But computations require concrete data structures. Python offers three candidates for representing vectors: tuples, lists, and NumPy arrays. Let's see why only one of them is fit for machine learning.

Tuples

Tuples are immutable ordered sequences. Great for safety, bad for computation:

Python
# Tuples: immutable ordered sequences — like a sealed envelope
v_tuple = (1, 3.5, -2.71, "a string", 42)
v_tuple[0]       # Access first element → 1
len(v_tuple)     # Number of elements → 5
v_tuple[1:4]     # Slicing → (3.5, -2.71, 'a string')

# Tuples are IMMUTABLE — trying to change them fails:
# v_tuple[0] = 2  → TypeError: 'tuple' does not support item assignment

Lists

Lists are mutable, but they don't behave like vectors at all:

Python
# Lists: mutable but NOT suitable for vector math
[1, 2, 3] + [4, 5, 6]   # Concatenation, NOT addition → [1, 2, 3, 4, 5, 6]
3 * [1, 2, 3]            # Repetition, NOT scaling   → [1, 2, 3, 1, 2, 3, 1, 2, 3]

# We wanted [5, 7, 9] and [3, 6, 9] — but Python lists have other ideas!
# Lists are designed for general-purpose programming, not linear algebra.

NumPy Arrays

NumPy gives us C arrays wrapped in a Pythonic interface — contiguous memory, homogeneous types, and element-wise operations by default:

Python
import numpy as np

# Creating arrays: the workhorse data structure for ML
X = np.array([87.7, 4.5, -4.1, 42.1414, -3.14, 2.001])  # From a Python list
np.ones(shape=7)        # Seven ones  → array([1., 1., 1., 1., 1., 1., 1.])
np.zeros(shape=5)       # Five zeros  → array([0., 0., 0., 0., 0.])
np.random.rand(10)      # 10 random floats in [0, 1)

# Key constraint: single data type per array (unlike lists)
X.dtype                  # → dtype('float64') — all elements are 64-bit floats

NumPy Arrays as Vectors

NumPy arrays behave exactly like mathematical vectors — operations are element-wise by default:

Python
v_1 = np.array([-4.0, 1.0, 2.3])    # First vector in R^3
v_2 = np.array([-8.3, -9.6, -7.7])  # Second vector in R^3

v_1 + v_2      # Element-wise addition  → array([-12.3, -8.6, -5.4])
10.0 * v_1     # Scalar multiplication  → array([-40.,  10.,  23. ])
v_1 * v_2      # Element-wise product   → array([ 33.2, -9.6, -17.71])

# Broadcasting: scalar + array works naturally
np.zeros(shape=3) + 1   # → array([1., 1., 1.])

# Functions work element-wise too!
def f(x):
    return 3*x**2 - x**4    # Polynomial applied to every element
f(v_1)                       # → array([-208., 2., -12.1141])
Performance Deep-Dive

The Vectorization Mindset: NumPy vs. Pure Python

When training neural networks, the same operations repeat millions of times. Even a small speedup compounds. NumPy achieves this by running operations in compiled C on contiguous memory blocks, instead of looping through Python objects one by one.

Python
from timeit import timeit

n_runs = 100000
size = 1000

# Pure Python: zip, loop, add each pair in the interpreter
t_builtin = timeit(
    "[x + y for x, y in zip(v_1, v_2)]",
    setup=f"size={size}; v_1=[0]*size; v_2=[0]*size",
    number=n_runs
)

# NumPy: single C-level call on contiguous float64 memory
t_numpy = timeit(
    "v_1 + v_2",
    setup=f"import numpy as np; size={size}; "
          f"v_1=np.zeros(size); v_2=np.zeros(size)",
    number=n_runs
)

print(f"Built-in:    {t_builtin:.2f}s")
print(f"NumPy:       {t_numpy:.2f}s")
print(f"Speedup:     {t_builtin/t_numpy:.1f}x")
# → NumPy is ~35x faster for size=1000!

Why NumPy wins: (1) contiguous memory — no pointer chasing; (2) homogeneous types — no per-element type checking; (3) compiled C loops — no interpreter overhead. The catch: for tiny arrays (size < ~20), Python's built-in operations are faster because NumPy has startup overhead. As arrays grow, NumPy dominates.

Section 1.4

Summary

Vectors are not just numbers bundled together — they are elements of a vector space, a mathematical structure with addition and scalar multiplication that follow precise rules. This structure allows us to reason about data both in theory and practice.

Vector spaces are best described by bases: minimal, linearly independent generating sets. The number of basis vectors defines the dimension. All finite-dimensional real vector spaces are isomorphic to \(\mathbb{R}^n\), so our canonical model is well-chosen.

On the practical side, vectorization with NumPy compresses complex logic into one-liners like:

Python
# The vectorization one-liner: standard scaling in a single expression
X_scaled = (X - X.mean(axis=0)) / X.std(axis=0)  # Zero-mean, unit-variance
Why This Matters for AI

Connecting vectors to machine learning

  • Data as vectors: Images are pixel vectors (\(28 \times 28 = 784\) dimensions for MNIST), text becomes embedding vectors (GPT uses 12,288 dimensions), audio is waveform vectors.
  • Vector spaces formalize structure: ML models exploit the algebraic structure of vector spaces — adding features, scaling weights, projecting data.
  • Bases = coordinate systems: Changing basis is essentially feature engineering. PCA (Chapter 7) finds the "best" basis for your data.
  • Vectorization = speed: NumPy's vectorized operations are the foundation of every tensor library — PyTorch, TensorFlow, and JAX all build on the same principle.

Socratic Knowledge Check

Instead of a quiz, reason through this broken scenario. Each question peels back another layer.

The Scenario: Your colleague claims to have built a custom "vector space" for representing RGB colors. They define vectors as \((r, g, b)\) tuples where each component is an integer from 0 to 255. Addition is defined component-wise, and scalar multiplication by an integer \(c\) gives \((cr, cg, cb)\).
1
Is the set \(\{0, 1, \ldots, 255\}^3\) closed under the proposed addition? What happens when you add \((200, 100, 50) + (100, 200, 100)\)?
\((200+100, 100+200, 50+100) = (300, 300, 150)\). But \(300 > 255\), so the result is outside the set. The set is not closed under addition — the very first axiom of a vector space is violated!
2
Does a null vector \((0, 0, 0)\) exist in this space? Does it satisfy \(\mathbf{x} + \mathbf{0} = \mathbf{x}\)?
Yes! \((0, 0, 0)\) is in the set and \((r, g, b) + (0, 0, 0) = (r, g, b)\) for all valid RGB values. This axiom passes.
3
For the vector \((100, 50, 25)\), does an additive inverse exist within \(\{0, \ldots, 255\}^3\)?
The inverse would be \((-100, -50, -25)\), but negative values aren't in \(\{0, \ldots, 255\}\). No additive inverses exist! Another axiom violated.
4
What minimal change would make this a valid vector space?
Use all of \(\mathbb{R}^3\) (or \(\mathbb{Z}^3\)) instead of restricting to \([0, 255]\). The restriction to non-negative bounded integers breaks closure under addition and the existence of additive inverses.
5
BONUS: In practice, ML frameworks do clamp pixel values to \([0, 255]\). What mathematical concept describes this restricted set? (Hint: it's NOT a vector space.)
It's a convex set — specifically, a hypercube \([0, 255]^3\). Convexity means any weighted average of two points stays inside (if \(\lambda \in [0,1]\), then \(\lambda\mathbf{x} + (1-\lambda)\mathbf{y}\) is valid), but arbitrary sums or negations can escape. Convex sets appear throughout optimization (Chapter 13).
