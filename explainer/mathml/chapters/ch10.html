Numbers, Sequences, and Series

From natural numbers to reals, convergence of sequences, Big-O notation, and infinite series

Part II — Calculus ~40 min read
Mental Model

Training Is a Sequence That Must Converge

Every time you run gradient descent, you produce a sequence of loss values: \(L_1, L_2, L_3, \ldots\). You hope this sequence converges — gets closer and closer to some minimum. But does it? How fast? What guarantees do you have? These are questions about sequences and series. The learning rate schedule is a sequence. The Taylor expansion that approximates your activation function is a series. The Big-O notation you use to reason about algorithmic complexity is the language of asymptotic analysis. This chapter builds the machinery that makes "converges" and "how fast" precise.

Section 10.1

The Number Systems

The Hierarchy of Numbers

Mathematics builds numbers in layers, each extending the previous to solve equations that were previously unsolvable.

Definition 10.1 — Number Systems
From Counting to Continuity
  • Natural numbers \(\mathbb{N} = \{0, 1, 2, 3, \ldots\}\) — for counting. Closed under addition and multiplication.
  • Integers \(\mathbb{Z} = \{\ldots, -2, -1, 0, 1, 2, \ldots\}\) — adds negatives so subtraction always works.
  • Rationals \(\mathbb{Q} = \{p/q : p \in \mathbb{Z},\, q \in \mathbb{Z} \setminus \{0\}\}\) — adds fractions so division (except by 0) always works.
  • Reals \(\mathbb{R}\) — "fills the gaps" in \(\mathbb{Q}\). Contains all limits of convergent rational sequences. The completeness property: every bounded, increasing sequence of reals has a limit in \(\mathbb{R}\).

The containment is strict: \(\mathbb{N} \subset \mathbb{Z} \subset \mathbb{Q} \subset \mathbb{R}\).

The step from \(\mathbb{Q}\) to \(\mathbb{R}\) is the most important for analysis. The rationals have "holes" — for instance, there is no rational number whose square is 2. The reals fill every such hole, giving us the completeness axiom: every Cauchy sequence of reals converges to a real number. Without completeness, calculus cannot work.

Floating-Point: The Numbers Computers Actually Use

Computers don't use \(\mathbb{R}\). They use a finite subset called floating-point numbers, typically IEEE 754 double-precision (64-bit) or single-precision (32-bit). This has profound consequences for ML. 

Python
import numpy as np
import sys

# Floating-point is NOT the reals
print("=== Floating-Point Surprises ===")

eps_64 = np.finfo(np.float64).eps
print(f"Machine epsilon (float32): {eps_32:.2e}")
print(f"Machine epsilon (float64): {eps_64:.2e}")
print()

    info = np.finfo(dtype)
    print(f"{dtype.__name__}: [{info.min:.2e}, {info.max:.2e}], "
          f"smallest positive: {info.tiny:.2e}")

print()

# Catastrophic cancellation
a = 1e16
b = 1e16 + 1
print(f"a = {a:.1f}, b = {b:.1f}")
print(f"b - a = {b - a:.1f}  (should be 1.0)")
# Lost precision because a and b are close relative to their magnitude

# The right way: use np.isclose instead of ==
print(f"\nnp.isclose(0.1 + 0.2, 0.3)? {np.isclose(0.1 + 0.2, 0.3)}")
ML Implication

Floating-point limitations explain why ML uses float32 by default (not float64): it's 2× faster and uses half the memory, and the noise from stochastic gradient descent already exceeds float32 precision. Modern training often uses float16 or bfloat16 for even greater speed, with careful "mixed-precision" tricks to avoid numerical problems.

Section 10.2

Sequences and Convergence

The \(\varepsilon\)–\(N\) Definition of a Limit

Definition 10.2 — Sequence
An Ordered List of Numbers

A sequence is a function \(a : \mathbb{N} \to \mathbb{R}\). We write \((a_n)_{n=1}^{\infty}\) or simply \((a_n)\). Each \(a_n\) is a term of the sequence.

Definition 10.3 — Convergence
The \(\varepsilon\)–\(N\) Criterion

A sequence \((a_n)\) converges to a limit \(L \in \mathbb{R}\), written \(\lim_{n \to \infty} a_n = L\) or \(a_n \to L\), if for every \(\varepsilon > 0\) there exists an \(N \in \mathbb{N}\) such that

$$n > N \implies |a_n - L| < \varepsilon.$$

In words: no matter how small a tolerance \(\varepsilon\) you demand, the sequence eventually stays within \(\varepsilon\) of \(L\) and never leaves.

This definition is the bedrock of all of analysis. It replaces the vague "gets closer and closer" with a precise, checkable criterion. 

Python
import numpy as np

def demonstrate_convergence(a_n, L, epsilons, max_n=10000):
    """Show the ε-N definition in action."""
    print(f"Limit L = {L}")
    for eps in epsilons:
        # Find smallest N such that |a_n - L| < eps for all n > N
        for N in range(1, max_n):
            # Check that ALL subsequent terms (up to a reasonable bound) are within eps
            tail = np.array([a_n(n) for n in range(N, N + 200)])
            if np.all(np.abs(tail - L) < eps):
                print(f"  ε = {eps:.0e} → N = {N:5d}  "
                      f"(a_N = {a_n(N):.8f}, |a_N - L| = {abs(a_n(N) - L):.2e})")
                break
    print()

# Sequence 1: a_n = 1/n → 0
print("=== a_n = 1/n → 0 ===")
demonstrate_convergence(lambda n: 1/n, 0, [1e-1, 1e-3, 1e-6])

# Sequence 2: a_n = (1 + 1/n)^n → e
print("=== a_n = (1 + 1/n)^n → e ===")
demonstrate_convergence(lambda n: (1 + 1/n)**n, np.e, [1e-1, 1e-3, 1e-5])

# Sequence 3: a_n = n/(n+1) → 1
print("=== a_n = n/(n+1) → 1 ===")
demonstrate_convergence(lambda n: n/(n+1), 1, [1e-1, 1e-3, 1e-6])

Properties of Limits

Theorem 10.1 — Algebra of Limits

If \(a_n \to L\) and \(b_n \to M\), then:

(a) \(a_n + b_n \to L + M\)   (b) \(a_n \cdot b_n \to L \cdot M\)   (c) \(a_n / b_n \to L / M\) (if \(M \neq 0\))

(d) \(\alpha a_n \to \alpha L\) for any constant \(\alpha\)   (e) If \(a_n \leq b_n\) eventually, then \(L \leq M\).

Theorem 10.2 — Squeeze Theorem

If \(a_n \leq c_n \leq b_n\) for all \(n\) sufficiently large, and \(a_n \to L\) and \(b_n \to L\), then \(c_n \to L\).

The Squeeze Theorem is how we prove many limits in practice — we bound the sequence from above and below by simpler sequences that we already know converge to the same limit.

Definition 10.4 — Cauchy Sequence
Convergence Without Knowing the Limit

A sequence \((a_n)\) is Cauchy if for every \(\varepsilon > 0\) there exists \(N\) such that

$$m, n > N \implies |a_m - a_n| < \varepsilon.$$

In \(\mathbb{R}\), a sequence converges if and only if it is Cauchy. This is the completeness of the reals — the terms get arbitrarily close to each other, and there's a real number they must be approaching.

Section 10.3

Big-O Notation and Asymptotic Analysis

Big-O notation captures how fast sequences and functions grow, ignoring constant factors. It's the standard language for algorithmic complexity and convergence rates.

Definition 10.5 — Big-O, Big-Omega, Big-Theta
Asymptotic Growth Rates

For functions \(f, g : \mathbb{N} \to \mathbb{R}^+\):

  • \(f(n) = O(g(n))\) if there exist \(C > 0\) and \(N_0\) such that \(f(n) \leq C \cdot g(n)\) for all \(n \geq N_0\). ("f grows at most as fast as g")
  • \(f(n) = \Omega(g(n))\) if \(g(n) = O(f(n))\). ("f grows at least as fast as g")
  • \(f(n) = \Theta(g(n))\) if \(f(n) = O(g(n))\) and \(f(n) = \Omega(g(n))\). ("f and g grow at the same rate")
Python
import numpy as np
import time

# Empirically verify Big-O complexity

def matrix_multiply(n):
    """O(n³) operation."""
    A = np.random.randn(n, n)
    B = np.random.randn(n, n)
    return A @ B

def sort_array(n):
    """O(n log n) operation."""
    arr = np.random.randn(n)
    return np.sort(arr)

def linear_scan(n):
    """O(n) operation."""
    arr = np.random.randn(n)
    return arr.sum()

operations = [
    ("Matrix multiply", matrix_multiply, "O(n³)"),
    ("Sort", sort_array, "O(n log n)"),
    ("Linear scan", linear_scan, "O(n)"),
]

print(f"{'Operation':<20s} {'n':>8s} {'Time (ms)':>10s} {'Ratio':>8s} {'Expected':>10s}")
print("-" * 60)

for name, func, complexity in operations:
    times = []
    ns = [500, 1000, 2000] if "Matrix" in name else [100000, 200000, 400000]

    for n in ns:
        t0 = time.perf_counter()
        func(n)
        elapsed = (time.perf_counter() - t0) * 1000
        times.append(elapsed)

    for i, (n, t) in enumerate(zip(ns, times)):
        ratio = f"{times[i]/times[0]:.1f}×" if i > 0 else "  —"
        print(f"{name:<20s} {n:>8d} {t:>9.1f}ms {ratio:>8s} {complexity:>10s}")
Proposition 10.1 — Convergence Rates in ML

Common convergence rates for optimization algorithms:

  • Gradient descent on convex functions: \(\|x_k - x^*\| = O(1/k)\) — sublinear convergence.
  • Gradient descent on strongly convex functions: \(\|x_k - x^*\| = O(\rho^k)\) with \(\rho < 1\) — linear (geometric) convergence.
  • Newton's method: \(\|x_k - x^*\| = O(\|x_{k-1} - x^*\|^2)\) — quadratic convergence (near the solution).
  • SGD with step size \(\eta_k = O(1/k)\): \(\mathbb{E}[L(x_k)] - L^* = O(1/\sqrt{k})\).
The "Little-o" Notation

We write \(f(n) = o(g(n))\) if \(f(n)/g(n) \to 0\) as \(n \to \infty\). This means \(f\) grows strictly slower than \(g\). In Taylor series, the remainder term is often expressed as \(o(h^k)\), meaning it vanishes faster than \(h^k\). This distinction between \(O\) (at most as fast) and \(o\) (strictly slower) matters for convergence proofs.

Section 10.4

Infinite Series

An infinite series is the sum of the terms of a sequence. Whether this sum makes sense — whether it converges to a finite value — is the central question.

Definition 10.6 — Infinite Series
Definition 10.6 — Infinite Series
Summing Infinitely Many Terms

Given a sequence \((a_n)\), the infinite series \(\sum_{n=1}^{\infty} a_n\) is defined as the limit of the partial sums:

$$S = \sum_{n=1}^{\infty} a_n = \lim_{N \to \infty} S_N, \quad \text{where } S_N = \sum_{n=1}^{N} a_n.$$

The series converges if the sequence of partial sums \((S_N)\) converges. Otherwise it diverges.

Proposition 10.2 — The Geometric Series

The geometric series \(\sum_{n=0}^{\infty} r^n\) converges if and only if \(|r| < 1\), and its sum is:

$$\sum_{n=0}^{\infty} r^n = \frac{1}{1 - r}, \qquad |r| < 1.$$

This is the prototypical convergent series and appears everywhere: discounted rewards in reinforcement learning, exponential moving averages, and the convergence analysis of gradient descent with momentum. 

Python
import numpy as np

def partial_sums(a_n, N_max):
    """Compute partial sums S_1, S_2, ..., S_N."""
    terms = np.array([a_n(n) for n in range(1, N_max + 1)])
    return np.cumsum(terms)

# Geometric series: Σ r^n
r = 0.5
exact = 1 / (1 - r)  # = 2.0 (for sum starting at n=0)
sums = partial_sums(lambda n: r**(n-1), 30)
print(f"Geometric series Σ (0.5)^n, exact = {exact:.4f}")
for N in [5, 10, 20, 30]:
    print(f"  S_{N:2d} = {sums[N-1]:.10f}  error = {abs(sums[N-1] - exact):.2e}")
print()

# Harmonic series: Σ 1/n — diverges!
sums_harm = partial_sums(lambda n: 1/n, 10000)
print("Harmonic series Σ 1/n (divergent):")
for N in [10, 100, 1000, 10000]:
    print(f"  S_{N:5d} = {sums_harm[N-1]:.4f}")
print()

# p-series: Σ 1/n^p — converges for p > 1
for p in [1.5, 2, 3]:
    sums_p = partial_sums(lambda n, p=p: 1/n**p, 10000)
    print(f"p-series (p={p}): S_10000 = {sums_p[-1]:.6f}", end="")
    if p == 2:
        print(f"  (exact = π²/6 ≈ {np.pi**2/6:.6f})")
    else:
        print()

Convergence Tests

Given a series, how do you determine whether it converges? Here are the main tools.

Theorem 10.3 — Convergence Tests

(a) Divergence test: If \(a_n \not\to 0\), then \(\sum a_n\) diverges. (The converse is false: \(a_n \to 0\) does not guarantee convergence — see the harmonic series.)

(b) Comparison test: If \(0 \leq a_n \leq b_n\) and \(\sum b_n\) converges, then \(\sum a_n\) converges.

(c) Ratio test: If \(\lim_{n\to\infty} |a_{n+1}/a_n| = L\), then the series converges absolutely if \(L < 1\) and diverges if \(L > 1\). (Inconclusive if \(L = 1\).)

(d) Absolute convergence: If \(\sum |a_n|\) converges, then \(\sum a_n\) converges. The converse is false: \(\sum (-1)^n/n\) converges but \(\sum 1/n\) diverges.

Python
import numpy as np

def ratio_test(a_n, start=1, count=20):
    """Apply the ratio test empirically."""
    ratios = []
    for n in range(start, start + count):
        r = abs(a_n(n + 1) / a_n(n)) if a_n(n) != 0 else float('inf')
        ratios.append(r)
    L = ratios[-1]
    verdict = "CONVERGES" if L < 1 else ("DIVERGES" if L > 1 else "INCONCLUSIVE")
    return L, verdict

# Test various series
series = [
    ("1/n!",     lambda n: 1 / np.math.factorial(n)),
    ("1/2^n",    lambda n: 1 / 2**n),
    ("n/2^n",    lambda n: n / 2**n),
    ("1/n",      lambda n: 1 / n),
    ("1/n^2",    lambda n: 1 / n**2),
]

print(f"{'Series':<12s} {'Ratio L':>10s} {'Verdict':<15s}")
print("-" * 40)
for name, a_n in series:
    L, verdict = ratio_test(a_n, start=10, count=30)
    print(f"{name:<12s} {L:10.6f} {verdict:<15s}")

The Cauchy Product

When we multiply two infinite series, the result is not simply term-by-term multiplication. The correct formula is the Cauchy product.

Definition 10.7 — Cauchy Product
Multiplying Two Series

Given two series \(A = \sum_{n=0}^{\infty} a_n\) and \(B = \sum_{n=0}^{\infty} b_n\), their Cauchy product is:

$$A \cdot B = \sum_{n=0}^{\infty} c_n, \quad \text{where } c_n = \sum_{k=0}^{n} a_k b_{n-k}.$$

Each \(c_n\) is a discrete convolution of the coefficient sequences. If both series converge absolutely, the Cauchy product converges absolutely to \(A \cdot B\).

Theorem 10.4 — Mertens' Theorem

If \(\sum a_n\) converges absolutely with sum \(A\) and \(\sum b_n\) converges with sum \(B\), then the Cauchy product converges to \(A \cdot B\).

The Cauchy product is the convolution of two sequences — the same operation that appears in polynomial multiplication, generating function manipulation, and (discrete) convolution layers in CNNs. 

Python
import numpy as np

def cauchy_product(a_coeffs, b_coeffs, N):
    """Compute the Cauchy product of two series up to N terms."""
    c = np.zeros(N)
    for n in range(N):
        for k in range(n + 1):
            if k < len(a_coeffs) and (n - k) < len(b_coeffs):
                c[n] += a_coeffs[k] * b_coeffs[n - k]
    return c

# Example: e^x · e^x = e^(2x)
# e^x = Σ x^n/n!, so multiply the series at x=1
N = 15
a = np.array([1 / np.math.factorial(n) for n in range(N)])  # coeffs of e^x
b = a.copy()  # same series

c = cauchy_product(a, b, N)

print("Cauchy product of e^x · e^x at x = 1:")
print(f"  Partial sum of product: {np.sum(c):.10f}")
print(f"  e^2 (exact):            {np.e**2:.10f}")
print(f"  Error:                  {abs(np.sum(c) - np.e**2):.2e}")
print()

# Connection to np.convolve (which IS the Cauchy product)
c_numpy = np.convolve(a, b)[:N]
print(f"Via np.convolve: {np.sum(c_numpy):.10f}")
print(f"Match? {np.allclose(c, c_numpy)}")
Performance Deep-Dive

From Cauchy Product to FFT Convolution

The naive Cauchy product is \(O(N^2)\). But by the Convolution Theorem, we can compute it in \(O(N \log N)\) using the Fast Fourier Transform: transform both sequences, multiply pointwise, and transform back. This is exactly how modern deep learning frameworks implement large convolution layers. 

Python
import numpy as np
import time

def cauchy_naive(a, b):
    """O(N²) Cauchy product."""
    N = len(a) + len(b) - 1
    c = np.zeros(N)
    for i in range(len(a)):
        for j in range(len(b)):
            c[i + j] += a[i] * b[j]
    return c

def cauchy_fft(a, b):
    """O(N log N) Cauchy product via FFT."""
    N = len(a) + len(b) - 1
    fa = np.fft.fft(a, N)
    fb = np.fft.fft(b, N)
    return np.real(np.fft.ifft(fa * fb))

for N in [100, 1000, 5000]:
    a = np.random.randn(N)
    b = np.random.randn(N)

    t0 = time.perf_counter()
    c1 = cauchy_naive(a, b) if N <= 1000 else None
    t_naive = time.perf_counter() - t0 if N <= 1000 else float('inf')

    t0 = time.perf_counter()
    c2 = cauchy_fft(a, b)
    t_fft = time.perf_counter() - t0

    if c1 is not None:
        print(f"N={N:5d}  naive: {t_naive:.4f}s  FFT: {t_fft:.4f}s  "
              f"speedup: {t_naive/t_fft:.0f}×  match: {np.allclose(c1, c2)}")
    else:
        print(f"N={N:5d}  naive: too slow   FFT: {t_fft:.4f}s")
Section 10.5

Summary

This chapter built the analytical foundations needed for calculus. We traced the number systems from naturals to reals, highlighting the completeness property that makes limits possible and the floating-point approximation that computers actually use. We formalised convergence via the \(\varepsilon\)–\(N\) definition, introduced Big-O notation for growth rates and convergence speeds, and developed the theory of infinite series with practical convergence tests. The Cauchy product connected series multiplication to convolution — a bridge from pure analysis to signal processing and deep learning.

Why This Matters for AI

Convergence Is the Core Question of Training

  • Training loops — Every optimizer produces a sequence of loss values. The \(\varepsilon\)–\(N\) definition formalises what "convergence" means. Learning rate schedules are designed to ensure this sequence converges.
  • Taylor series — Activation functions, log-likelihoods, and KL divergences are often approximated by truncated Taylor series. Knowing when the series converges (and how fast) determines the quality of the approximation.
  • Geometric series and discounting — In reinforcement learning, the discounted return \(G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k}\) is a geometric series. It converges because \(|\gamma| < 1\).
  • Floating-point awareness — Understanding machine epsilon, catastrophic cancellation, and numerical stability prevents silent bugs. Use log_softmax instead of log(softmax); use logsumexp instead of log(sum(exp)).
  • Complexity analysis — Big-O notation tells you whether your model can scale: \(O(n^2)\) attention is the bottleneck in Transformers; \(O(n \log n)\) alternatives (linear attention, FFT-based methods) are an active research area.

Socratic Knowledge Check

You are debugging a neural network that refuses to converge. The loss curve oscillates wildly. You suspect numerical issues. Think through these questions to diagnose the problem.
1
Your loss values are: \(2.3, 1.8, 1.5, 1.5, 1.5, 1.5, \ldots\). Has the sequence converged? What value of \(N\) would you need for \(\varepsilon = 0.01\)?
The sequence appears to converge to \(L = 1.5\). For \(\varepsilon = 0.01\), we need \(|a_n - 1.5| < 0.01\) for all \(n > N\). Since the terms are exactly 1.5 from \(n = 3\) onward, \(N = 3\) suffices. In practice, you'd check that the loss stays within \(\varepsilon\) for many subsequent epochs (not just a few), since flat regions can be followed by sudden drops (e.g., learning rate warm-up effects or saddle points).
2
You compute log(softmax(x)) and get NaN for large logits (e.g., \(x_i = 1000\)). Explain the numerical failure using what you know about floating-point, and describe the fix.
softmax(x) computes \(e^{x_i} / \sum_j e^{x_j}\). For \(x_i = 1000\), \(e^{1000} \approx 10^{434}\), which overflows float64 (max \(\approx 10^{308}\)), producing inf. Then inf/inf = NaN. The fix is the log-sum-exp trick: subtract \(\max(x)\) before exponentiating. Since \(\text{softmax}(x) = \text{softmax}(x - c)\) for any constant \(c\), this doesn't change the result but keeps all exponents small. Even better, use log_softmax directly: \(\log\text{softmax}(x_i) = x_i - \log\sum_j e^{x_j} = x_i - \text{logsumexp}(x)\), which never materializes the huge exponentials.
3
Your gradient descent uses a fixed learning rate \(\eta = 0.1\) and the loss decrease per epoch is roughly proportional to \(1/k\) where \(k\) is the epoch number. Is this sublinear, linear, or superlinear convergence? What would you change to get faster convergence?
A decrease proportional to \(1/k\) means the error after \(k\) steps is \(O(1/k)\) — this is sublinear convergence. Each step gives diminishing returns. For strongly convex problems, gradient descent with the right learning rate gives linear convergence: error \(= O(\rho^k)\) with \(\rho < 1\), which is exponentially faster. To speed up: (1) use momentum (accelerated gradient descent achieves \(O(1/k^2)\) for convex problems), (2) use adaptive methods like Adam, (3) add strong convexity via L2 regularization, or (4) use Newton's method for quadratic convergence near the optimum.
4
In reinforcement learning, the discounted return is \(G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k}\) with \(\gamma = 0.99\) and rewards bounded by \(|r| \leq 1\). What is the maximum possible value of \(G_t\), and why does the series converge?
Since \(|r_{t+k}| \leq 1\), we have \(|G_t| \leq \sum_{k=0}^{\infty} \gamma^k \cdot 1 = \frac{1}{1-\gamma} = \frac{1}{1-0.99} = 100\). The series converges because it's bounded by a convergent geometric series (the comparison test) with ratio \(\gamma = 0.99 < 1\). This maximum value of 100 is important for scaling: value function outputs should be on the order of \(1/(1-\gamma)\), and gradient magnitudes scale accordingly. A discount factor close to 1 (like 0.99) gives a large maximum return and makes credit assignment harder — the agent must consider rewards up to \(\sim 1/(1-\gamma) = 100\) steps into the future.
5
Your Transformer model has \(O(n^2)\) attention complexity. You're told a new architecture achieves \(O(n \log n)\). For a sequence length of \(n = 10{,}000\), what's the theoretical speedup? Why might the practical speedup differ?
The theoretical speedup is \(O(n^2) / O(n \log n) = O(n / \log n)\). For \(n = 10{,}000\): \(n / \log_2 n \approx 10{,}000 / 13.3 \approx 750\times\). But Big-O hides constant factors! The \(O(n \log n)\) method might have a constant factor 10× larger than standard attention (more complex operations per step, less GPU-friendly memory access patterns, inability to use optimized CUDA kernels). So the practical speedup might be only 10–50× rather than 750×. Additionally, for moderate \(n\) (say 512), the \(O(n^2)\) method with tiny constants might actually be faster. Big-O tells you about asymptotic behaviour — it's the right tool for reasoning about scalability, but wall-clock time requires benchmarking.
