Matrices as verbs, not nouns

In Chapters 1–3, we treated matrices as rectangular grids of numbers — static objects you multiply and transpose. Now we shift perspective entirely: a matrix is an action. It's a function that takes in a vector and spits out a new one. When you multiply a vector by a rotation matrix, you're literally spinning that vector around the origin. A projection matrix flattens 3D space onto a 2D plane — like a shadow cast by the sun.

This "matrices as verbs" viewpoint is the single most important conceptual leap in linear algebra. Every neural network layer is a linear transformation followed by a nonlinearity. Every PCA computation is a change of basis. Every image convolution is a linear map. Once you see matrices as transformations, the entire machinery of machine learning clicks into place.

Section 4.1

What Is a Linear Map?

A linear map (or linear transformation) is a function between vector spaces that respects the two operations that define those spaces: vector addition and scalar multiplication. Formally:

Definition 4.1.1
Linear Map

Let \(V\) and \(W\) be vector spaces over the same field. A function \(T: V \to W\) is a linear map if for all \(\mathbf{u}, \mathbf{v} \in V\) and all scalars \(c\):

  • Additivity: \(T(\mathbf{u} + \mathbf{v}) = T(\mathbf{u}) + T(\mathbf{v})\)
  • Homogeneity: \(T(c\mathbf{v}) = c\, T(\mathbf{v})\)

Equivalently, \(T(\alpha\mathbf{u} + \beta\mathbf{v}) = \alpha\, T(\mathbf{u}) + \beta\, T(\mathbf{v})\) for all scalars \(\alpha, \beta\).

The definition says: "transforming a sum equals the sum of the transforms" and "transforming a scaled vector equals scaling the transform." These two conditions are deceptively powerful — they force the map to preserve the entire geometric structure of the vector space. Lines stay lines, the origin stays fixed, and parallel lines remain parallel.

What's not a linear map? Translation, for instance. Shifting every point by a fixed vector \(\mathbf{b}\) gives \(T(\mathbf{v}) = \mathbf{v} + \mathbf{b}\). Check: \(T(\mathbf{0}) = \mathbf{b} \neq \mathbf{0}\). A linear map must send the zero vector to zero, so translation fails.

Python
import numpy as np

def is_linear(T, num_tests=1000, dim=3, tol=1e-10):
    """Test whether a function T: R^dim -> R^? is linear.

    Checks additivity and homogeneity on random vectors.
    """
    for _ in range(num_tests):
        u = np.random.randn(dim)
        v = np.random.randn(dim)
        c = np.random.randn()

        # Test additivity: T(u + v) == T(u) + T(v)
        if not np.allclose(T(u + v), T(u) + T(v), atol=tol):
            return False

        # Test homogeneity: T(c * v) == c * T(v)
        if not np.allclose(T(c * v), c * T(v), atol=tol):
            return False

    return True

# Example 1: Matrix multiplication IS linear
A = np.array([[2, -1, 0], [0, 3, 1]])              # 2x3 matrix (maps R^3 -> R^2)
T_linear = lambda v: A @ v

print(f"Matrix multiplication is linear: {is_linear(T_linear)}")  # True

# Example 2: Translation is NOT linear
b = np.array([1.0, 2.0])                            # Translation vector
T_translate = lambda v: A @ v + b                   # Affine, not linear

print(f"Translation is linear: {is_linear(T_translate)}")          # False

# Example 3: Squaring components is NOT linear
T_square = lambda v: v**2

print(f"Squaring is linear: {is_linear(T_square)}")               # False

# Example 4: The zero map IS linear (trivially)
T_zero = lambda v: np.zeros(2)

print(f"Zero map is linear: {is_linear(T_zero)}")                 # True

Every Linear Map Is a Matrix (and Vice Versa)

Here is the central theorem of linear algebra in finite dimensions: every linear map between finite-dimensional vector spaces can be represented as matrix multiplication, and every matrix defines a linear map.

Theorem 4.1.1

Linear maps and matrices are the same thing. Let \(V\) and \(W\) be finite-dimensional vector spaces with bases \(\mathcal{B}_V = \{\mathbf{b}_1, \ldots, \mathbf{b}_n\}\) and \(\mathcal{B}_W = \{\mathbf{c}_1, \ldots, \mathbf{c}_m\}\). Then for every linear map \(T: V \to W\), there exists a unique matrix \(A \in \mathbb{R}^{m \times n}\) such that $$T(\mathbf{v}) = A\mathbf{v}$$ for all \(\mathbf{v} \in V\) (expressed in coordinates relative to these bases). Conversely, every matrix \(A \in \mathbb{R}^{m \times n}\) defines a linear map from \(\mathbb{R}^n\) to \(\mathbb{R}^m\).

Forward direction: Since \(T\) is linear, it's fully determined by its action on the basis vectors. Write \(T(\mathbf{b}_j) = \sum_{i=1}^{m} a_{ij} \mathbf{c}_i\) for some scalars \(a_{ij}\). Then for any \(\mathbf{v} = \sum_j x_j \mathbf{b}_j\):

\(T(\mathbf{v}) = T\!\left(\sum_j x_j \mathbf{b}_j\right) = \sum_j x_j T(\mathbf{b}_j) = \sum_j x_j \sum_i a_{ij} \mathbf{c}_i = \sum_i \left(\sum_j a_{ij} x_j\right) \mathbf{c}_i\)

The coordinate vector of \(T(\mathbf{v})\) with respect to \(\mathcal{B}_W\) is \(A\mathbf{x}\), where \(A = (a_{ij})\).

Converse: For any matrix \(A\), define \(T(\mathbf{v}) = A\mathbf{v}\). Additivity: \(A(\mathbf{u}+\mathbf{v}) = A\mathbf{u} + A\mathbf{v}\). Homogeneity: \(A(c\mathbf{v}) = cA\mathbf{v}\). Both follow from the distributive property of matrix multiplication. ∎

The key insight: to build the matrix of a linear map, you just need to know what \(T\) does to each basis vector. The columns of \(A\) are the images of the basis vectors.

Python
import numpy as np

def build_transformation_matrix(T, dim_in, dim_out):
    """Build the matrix of a linear map T: R^dim_in -> R^dim_out.

    Strategy: apply T to each standard basis vector e_j.
    The j-th column of the matrix is T(e_j).
    """
    A = np.zeros((dim_out, dim_in))

    for j in range(dim_in):
        # Create the j-th standard basis vector
        e_j = np.zeros(dim_in)
        e_j[j] = 1.0

        # The j-th column of A is T(e_j)
        A[:, j] = T(e_j)

    return A

# Example: a known linear map T(x, y, z) = (2x - y, 3y + z)
def T(v):
    return np.array([2*v[0] - v[1], 3*v[1] + v[2]])

A = build_transformation_matrix(T, dim_in=3, dim_out=2)
print("Transformation matrix:")
print(A)
# [[2, -1, 0],
#  [0,  3, 1]]

# Verify: T(v) should equal A @ v for any vector v
for _ in range(100):
    v = np.random.randn(3)
    assert np.allclose(T(v), A @ v), "Mismatch!"

print("All 100 tests passed: T(v) == A @ v")

Kernel and Image

Two fundamental subspaces are attached to every linear map. They tell you what information the map destroys and what it can produce.

Definition 4.1.2
Kernel and Image

For a linear map \(T: V \to W\):

  • Kernel (null space): \(\ker(T) = \{\mathbf{v} \in V : T(\mathbf{v}) = \mathbf{0}\}\) — all vectors that map to zero.
  • Image (range): \(\text{im}(T) = \{T(\mathbf{v}) : \mathbf{v} \in V\}\) — all vectors that \(T\) can produce.

The kernel measures what the map "forgets." If \(\ker(T) = \{\mathbf{0}\}\), the map is injective (one-to-one): no information is lost. If the kernel is large, many different inputs map to the same output.

Theorem 4.1.2 — Rank–Nullity Theorem

For a linear map \(T: V \to W\) with \(V\) finite-dimensional: $$\dim(\ker(T)) + \dim(\text{im}(T)) = \dim(V)$$ Equivalently: nullity + rank = number of columns.

This is a conservation law: the dimension of the domain splits between what the map preserves (the image) and what it destroys (the kernel). If you have a 10-dimensional input and a rank-7 matrix, you're losing exactly 3 dimensions of information.

Python
import numpy as np

# A 2x3 matrix: maps R^3 -> R^2, so it must destroy at least 1 dimension
A = np.array([[1, 2, 3],
              [4, 5, 6]])

# Rank = dimension of image
rank = np.linalg.matrix_rank(A)
print(f"Rank (dim of image): {rank}")                  # 2

# Nullity = dim(V) - rank = 3 - 2 = 1
nullity = A.shape[1] - rank
print(f"Nullity (dim of kernel): {nullity}")           # 1

# Find the kernel (null space) using SVD
# The null space is spanned by the columns of V corresponding to zero singular values
U, S, Vt = np.linalg.svd(A)
null_mask = np.abs(S) < 1e-10                          # Which singular values are zero?
# For a 2x3 matrix, we need to check all 3 columns of Vt
null_space = Vt[len(S):, :]                            # Rows beyond rank
if null_space.shape[0] == 0:
    # Pad with remaining rows of Vt
    null_space = Vt[rank:, :]

print(f"\nNull space basis vector(s):")
for v in null_space:
    print(f"  {v}")
    print(f"  A @ v = {A @ v}")                        # Should be ~[0, 0]

# Verify rank-nullity: nullity + rank = number of columns
print(f"\nRank-nullity check: {nullity} + {rank} = {nullity + rank} = {A.shape[1]}")

# A more practical approach using scipy
from scipy.linalg import null_space as scipy_null_space
ns = scipy_null_space(A)
print(f"\nNull space via scipy (shape {ns.shape}):")
print(ns)
print(f"A @ null_vector = {A @ ns[:, 0]}")             # Should be ~[0, 0]
Key Intuition

In machine learning, the kernel of a weight matrix tells you which input variations the network ignores. If your model's first layer has a large null space, there are directions in input space that produce no signal. This is related to why dimensionality reduction works — you can safely throw away the kernel directions without losing predictive information.

Section 4.2

Change of Basis

A linear map doesn't change when you look at it from a different angle — but its matrix representation does. This is because the matrix depends on which bases you choose for the domain and codomain. Changing the basis is like changing the coordinate system on a map: the terrain is the same, but the numbers describing each location change.

Definition 4.2.1
Change of Basis Matrix

Let \(\mathcal{B} = \{\mathbf{b}_1, \ldots, \mathbf{b}_n\}\) and \(\mathcal{B}' = \{\mathbf{b}'_1, \ldots, \mathbf{b}'_n\}\) be two bases of \(V\). The change of basis matrix from \(\mathcal{B}'\) to \(\mathcal{B}\) is the matrix \(P\) whose columns are the coordinate vectors of \(\mathbf{b}'_1, \ldots, \mathbf{b}'_n\) with respect to \(\mathcal{B}\): $$\mathbf{b}'_j = \sum_{i=1}^{n} p_{ij} \mathbf{b}_i$$ If \(\mathbf{x}\) is the coordinate vector of \(\mathbf{v}\) in basis \(\mathcal{B}'\), then \(P\mathbf{x}\) is the coordinate vector of \(\mathbf{v}\) in basis \(\mathcal{B}\).

How a Transformation Matrix Changes with Basis

If \(A\) is the matrix of a linear map \(T\) with respect to the standard basis, and \(P\) is the change of basis matrix from a new basis \(\mathcal{B}\) to the standard basis, then the matrix of \(T\) in the new basis is: $$A' = P^{-1} A P$$ This formula is called a similarity transformation. The matrices \(A\) and \(A'\) are similar — they represent the same linear map, just described in different coordinates.

Python
import numpy as np

# Original transformation matrix (in standard basis)
A = np.array([[2, 1],
              [1, 3]])

# New basis: columns of P are the new basis vectors
# Let's use the eigenvectors as our new basis (foreshadowing Chapter 6!)
eigenvalues, P = np.linalg.eig(A)
print("Eigenvalues:", eigenvalues)
print("Change of basis matrix P (eigenvectors as columns):")
print(P)

# Transform A into the new basis: A' = P^{-1} A P
P_inv = np.linalg.inv(P)
A_prime = P_inv @ A @ P
print("\nA in new basis (A' = P^{-1} A P):")
print(np.round(A_prime, 10))
# Should be diagonal (eigenvalues on diagonal) — because we chose eigenvector basis!

# Verify: the transformation does the same thing regardless of basis
v_standard = np.array([3.0, 1.0])                    # Vector in standard basis

# Method 1: Apply A directly in standard basis
result_1 = A @ v_standard

# Method 2: Convert to new basis, apply A', convert back
v_new_basis = P_inv @ v_standard                     # Convert to new basis coords
result_new = A_prime @ v_new_basis                    # Apply transformation in new basis
result_2 = P @ result_new                             # Convert back to standard basis

print(f"\nDirect:   A @ v = {result_1}")
print(f"Via basis: P @ A' @ P^-1 @ v = {result_2}")
print(f"Same result? {np.allclose(result_1, result_2)}")  # True!

# A practical example: simplifying a transformation
# Rotation + scaling is hard to reason about in arbitrary basis
# But in the "right" basis (eigenbasis), it becomes just scaling!
print(f"\nIn eigenbasis, T just scales by {eigenvalues[0]:.2f} and {eigenvalues[1]:.2f}")
Remark 4.2.1

This is why PCA works. Principal Component Analysis finds the eigenvectors of the covariance matrix and uses them as a new basis. In this new basis, the data's covariance matrix is diagonal — the features are uncorrelated. The "principal components" are just the standard coordinate axes in the eigenbasis.

Section 4.3

Geometric Transformations in \(\mathbb{R}^2\)

Linear maps in 2D have beautiful geometric interpretations. Every 2D linear transformation can be decomposed into a combination of rotations, reflections, scalings, and shears. Let's build each one from scratch.

Rotations

A rotation by angle \(\theta\) counterclockwise around the origin sends the point \((x, y)\) to: $$R_\theta \begin{pmatrix} x \\ y \end{pmatrix} = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix} \begin{pmatrix} x \\ y \end{pmatrix}$$

Definition 4.3.1
Rotation Matrix

The rotation matrix by angle \(\theta\) counterclockwise is: $$R_\theta = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}$$

Properties: \(\det(R_\theta) = 1\), \(R_\theta^{-1} = R_{-\theta} = R_\theta^T\), and \(R_\alpha R_\beta = R_{\alpha+\beta}\).

Where does this formula come from? Apply \(R_\theta\) to the standard basis: \(R_\theta \mathbf{e}_1 = (\cos\theta, \sin\theta)^T\) and \(R_\theta \mathbf{e}_2 = (-\sin\theta, \cos\theta)^T\). These are exactly where the basis vectors land after rotating by \(\theta\).

Python
import numpy as np

def rotation_matrix(theta):
    """Build the 2D rotation matrix for angle theta (in radians).

    Rotates vectors counterclockwise by theta.
    """
    c, s = np.cos(theta), np.sin(theta)
    return np.array([[c, -s],
                     [s,  c]])

# Rotate the vector (1, 0) by 90 degrees
R90 = rotation_matrix(np.pi / 2)
v = np.array([1.0, 0.0])
print(f"Rotate (1,0) by 90°: {np.round(R90 @ v, 10)}")       # → [0, 1]

# Rotate by 45 degrees
R45 = rotation_matrix(np.pi / 4)
print(f"Rotate (1,0) by 45°: {R45 @ v}")                      # → [0.707, 0.707]

# Key properties
theta = np.pi / 6                                              # 30 degrees

R = rotation_matrix(theta)
print(f"\ndet(R) = {np.linalg.det(R):.6f}")                   # 1.0 (preserves area)
print(f"R @ R.T = \n{np.round(R @ R.T, 10)}")                 # Identity (orthogonal)
print(f"R^(-1) = R(-theta)? {np.allclose(np.linalg.inv(R), rotation_matrix(-theta))}")

# Composition: R(a) @ R(b) = R(a+b)
alpha, beta = np.pi/4, np.pi/6
R_comp = rotation_matrix(alpha) @ rotation_matrix(beta)
R_sum = rotation_matrix(alpha + beta)
print(f"\nR(a)R(b) == R(a+b)? {np.allclose(R_comp, R_sum)}")  # True!

# Rotate a batch of points (columns of a matrix)
angles = np.linspace(0, 2*np.pi, 8, endpoint=False)
points = np.array([[np.cos(a), np.sin(a)] for a in angles]).T  # Unit circle points
R30 = rotation_matrix(np.pi / 6)
rotated = R30 @ points                                        # Rotate all points at once

print(f"\nOriginal points shape: {points.shape}")              # (2, 8)
print(f"Rotated points shape:  {rotated.shape}")               # (2, 8)

Reflections

A reflection across a line through the origin (at angle \(\alpha\) from the x-axis) flips the plane over that line. The reflection matrix is: $$S_\alpha = \begin{pmatrix} \cos 2\alpha & \sin 2\alpha \\ \sin 2\alpha & -\cos 2\alpha \end{pmatrix}$$

Definition 4.3.2
Reflection Matrix

The reflection matrix across the line at angle \(\alpha\) from the x-axis is: $$S_\alpha = \begin{pmatrix} \cos 2\alpha & \sin 2\alpha \\ \sin 2\alpha & -\cos 2\alpha \end{pmatrix}$$

Properties: \(\det(S_\alpha) = -1\), \(S_\alpha^2 = I\) (reflecting twice gives identity), and \(S_\alpha = S_\alpha^T = S_\alpha^{-1}\).

Python
import numpy as np

def reflection_matrix(alpha):
    """Build the 2D reflection matrix across line at angle alpha.

    Reflects vectors across the line through origin at angle alpha.
    """
    c, s = np.cos(2 * alpha), np.sin(2 * alpha)
    return np.array([[c,  s],
                     [s, -c]])

# Reflect across the x-axis (alpha = 0)
Sx = reflection_matrix(0)
v = np.array([1.0, 2.0])
print(f"Reflect (1,2) across x-axis: {Sx @ v}")              # → [1, -2]

# Reflect across the y-axis (alpha = pi/2)
Sy = reflection_matrix(np.pi / 2)
print(f"Reflect (1,2) across y-axis: {Sy @ v}")              # → [-1, 2]

# Reflect across y = x (alpha = pi/4)
Sxy = reflection_matrix(np.pi / 4)
print(f"Reflect (1,2) across y=x: {np.round(Sxy @ v, 10)}") # → [2, 1] (swap coords!)

# Key properties
S = reflection_matrix(np.pi / 6)
print(f"\ndet(S) = {np.linalg.det(S):.6f}")                  # -1 (reverses orientation)
print(f"S^2 = I? {np.allclose(S @ S, np.eye(2))}")           # True (involutory)
print(f"S = S^T? {np.allclose(S, S.T)}")                     # True (symmetric)

# Key insight: Rotation = composition of two reflections
alpha1, alpha2 = 0, np.pi / 6
S1 = reflection_matrix(alpha1)
S2 = reflection_matrix(alpha2)
composed = S2 @ S1                                            # Two reflections

# The angle of rotation is 2 * (alpha2 - alpha1)
R_expected = rotation_matrix(2 * (alpha2 - alpha1))
print(f"\nTwo reflections = rotation? {np.allclose(composed, R_expected)}")  # True!

Orthogonal Projections

A projection "shadows" a vector onto a subspace. Given a unit vector \(\hat{\mathbf{u}}\), the orthogonal projection onto the line spanned by \(\hat{\mathbf{u}}\) is: $$P_{\hat{\mathbf{u}}} = \hat{\mathbf{u}}\hat{\mathbf{u}}^T$$ This is an outer product, producing a matrix that extracts the component of any vector in the direction of \(\hat{\mathbf{u}}\).

Definition 4.3.3
Orthogonal Projection Matrix

The projection matrix onto the line spanned by unit vector \(\hat{\mathbf{u}}\) is: $$P = \hat{\mathbf{u}}\hat{\mathbf{u}}^T$$

For projection onto a subspace spanned by columns of \(U\) (with orthonormal columns): $$P = UU^T$$

Properties: \(P^2 = P\) (idempotent), \(P = P^T\) (symmetric), and \(\text{rank}(P)\) equals the dimension of the subspace.

Python
import numpy as np

def projection_matrix(u):
    """Build the projection matrix onto the line spanned by vector u.

    The matrix P = uu^T / (u^T u) projects any vector onto span(u).
    """
    u = u.reshape(-1, 1)                               # Make column vector
    return (u @ u.T) / (u.T @ u)                       # Outer product, normalized

# Project onto the x-axis direction
u_x = np.array([1.0, 0.0])
P_x = projection_matrix(u_x)
v = np.array([3.0, 4.0])
print(f"Project (3,4) onto x-axis: {P_x @ v}")        # → [3, 0]

# Project onto the line y = x (direction (1, 1))
u_diag = np.array([1.0, 1.0])
P_diag = projection_matrix(u_diag)
print(f"Project (3,4) onto y=x: {P_diag @ v}")        # → [3.5, 3.5]

# Project onto a direction at 30 degrees
theta = np.pi / 6
u_30 = np.array([np.cos(theta), np.sin(theta)])
P_30 = projection_matrix(u_30)
print(f"Project (3,4) onto 30° line: {np.round(P_30 @ v, 4)}")

# Key properties
P = P_diag
print(f"\nP^2 = P? {np.allclose(P @ P, P)}")          # True (idempotent)
print(f"P = P^T? {np.allclose(P, P.T)}")              # True (symmetric)
print(f"rank(P) = {np.linalg.matrix_rank(P)}")        # 1 (projecting onto 1D line)

# Projection onto 2D subspace in R^3
# Columns of U form an orthonormal basis for the subspace
U = np.array([[1, 0],
              [0, 1],
              [0, 0]], dtype=float)                     # xy-plane in R^3
P_plane = U @ U.T                                      # 3x3 projection matrix
v3 = np.array([2.0, 3.0, 5.0])
print(f"\nProject (2,3,5) onto xy-plane: {P_plane @ v3}")  # → [2, 3, 0]

# The residual (what's left) is orthogonal to the subspace
residual = v3 - P_plane @ v3
print(f"Residual: {residual}")                         # → [0, 0, 5]
print(f"Residual dot projection: {np.dot(residual, P_plane @ v3):.10f}")  # → 0
Remark 4.3.1

Projections are everywhere in machine learning. Least-squares regression projects the target vector onto the column space of the feature matrix. PCA projects high-dimensional data onto the subspace of maximum variance. Attention mechanisms in Transformers compute projections of queries onto key-value subspaces.

Section 4.4

Determinants

The determinant of a square matrix is a single number that captures how the linear map scales volume. If \(T\) maps a unit cube to a parallelepiped, the determinant is the (signed) volume of that parallelepiped.

Definition 4.4.1
Determinant

For a \(2 \times 2\) matrix: $$\det\begin{pmatrix} a & b \\ c & d \end{pmatrix} = ad - bc$$

For an \(n \times n\) matrix, the determinant is defined recursively by cofactor expansion along the first row: $$\det(A) = \sum_{j=1}^{n} (-1)^{1+j}\, a_{1j}\, \det(A_{1j})$$ where \(A_{1j}\) is the \((n{-}1) \times (n{-}1)\) submatrix obtained by deleting row 1 and column \(j\).

Theorem 4.4.1

Properties of the determinant:

  • (a) \(\det(AB) = \det(A)\det(B)\) — determinants multiply.
  • (b) \(\det(A^T) = \det(A)\) — transposing doesn't change the determinant.
  • (c) \(A\) is invertible if and only if \(\det(A) \neq 0\).
  • (d) \(\det(cA) = c^n \det(A)\) for \(A \in \mathbb{R}^{n \times n}\).
  • (e) \(|\det(A)|\) is the volume of the parallelepiped formed by the columns of \(A\).
  • (f) \(\det(A^{-1}) = 1/\det(A)\).
Python
import numpy as np

def det_2x2(A):
    """Compute determinant of a 2x2 matrix: ad - bc."""
    return A[0, 0] * A[1, 1] - A[0, 1] * A[1, 0]

def det_recursive(A):
    """Compute determinant by cofactor expansion along first row.

    This is O(n!) — educational only! Use np.linalg.det for real work.
    """
    n = A.shape[0]

    # Base case: 1x1 matrix
    if n == 1:
        return A[0, 0]

    # Base case: 2x2 matrix
    if n == 2:
        return det_2x2(A)

    # Recursive case: expand along first row
    det = 0
    for j in range(n):
        # Submatrix: delete row 0 and column j
        submatrix = np.delete(np.delete(A, 0, axis=0), j, axis=1)
        # Cofactor: (-1)^(1+j) * a_{0,j} * det(submatrix)
        cofactor = ((-1) ** j) * A[0, j] * det_recursive(submatrix)
        det += cofactor

    return det

# Test on 2x2
A2 = np.array([[3, 7],
               [1, -4]], dtype=float)
print(f"2x2 det (custom):  {det_2x2(A2)}")            # 3*(-4) - 7*1 = -19
print(f"2x2 det (numpy):   {np.linalg.det(A2):.1f}")  # -19.0

# Test on 3x3
A3 = np.array([[1, 2, 3],
               [4, 5, 6],
               [7, 8, 0]], dtype=float)
print(f"\n3x3 det (custom):  {det_recursive(A3)}")
print(f"3x3 det (numpy):   {np.linalg.det(A3):.1f}")

# Geometric interpretation: area scaling
print("\n--- Geometric Interpretation ---")

# The unit square has area 1
# After transformation by A, the area becomes |det(A)|
A_scale = np.array([[2, 0],
                     [0, 3]], dtype=float)              # Scale x by 2, y by 3
print(f"Scaling matrix det: {np.linalg.det(A_scale)}")  # 6 (area scales by 6)

# Rotation preserves area
R = np.array([[np.cos(0.5), -np.sin(0.5)],
              [np.sin(0.5),  np.cos(0.5)]])
print(f"Rotation det: {np.linalg.det(R):.6f}")         # 1.0 (area preserved)

# Reflection flips orientation (negative determinant)
S = np.array([[1,  0],
              [0, -1]], dtype=float)                    # Reflect across x-axis
print(f"Reflection det: {np.linalg.det(S)}")            # -1 (flipped orientation)

# Singular matrix: determinant is 0 (crushes dimension)
A_singular = np.array([[1, 2],
                        [2, 4]], dtype=float)           # Row 2 = 2 * Row 1
print(f"Singular matrix det: {np.linalg.det(A_singular):.1f}")  # 0

# Property: det(AB) = det(A) * det(B)
A = np.random.randn(3, 3)
B = np.random.randn(3, 3)
print(f"\ndet(AB) = {np.linalg.det(A @ B):.6f}")
print(f"det(A)*det(B) = {np.linalg.det(A) * np.linalg.det(B):.6f}")
print(f"Equal? {np.allclose(np.linalg.det(A @ B), np.linalg.det(A) * np.linalg.det(B))}")
Performance Deep-Dive

Cofactor expansion is catastrophically slow

Our recursive det_recursive has \(O(n!)\) complexity — it computes \(n!\) products. For a 20×20 matrix, that's ~2.4 quintillion operations. NumPy uses LU decomposition instead: factor \(A = LU\), then \(\det(A) = \det(L)\det(U) = \prod u_{ii}\). This is \(O(n^3)\) — fast enough for matrices with thousands of rows.

Python
import numpy as np
from timeit import timeit

# Compare: custom O(n!) vs NumPy O(n^3)
sizes = [5, 8, 10]
for n in sizes:
    A = np.random.randn(n, n)

    if n <= 10:
        t_custom = timeit(lambda: det_recursive(A), number=10)
    t_numpy = timeit(lambda: np.linalg.det(A), number=10000)

    if n <= 10:
        print(f"n={n}: custom={t_custom:.4f}s (10 runs), numpy={t_numpy:.4f}s (10k runs)")
    else:
        print(f"n={n}: numpy={t_numpy:.4f}s (10k runs)")

# For n=1000, numpy handles it instantly; custom would take longer than the age of the universe
A_big = np.random.randn(1000, 1000)
t = timeit(lambda: np.linalg.det(A_big), number=10)
print(f"\nn=1000: numpy={t:.4f}s (10 runs)")          # ~0.1s total
Section 4.5

Summary

This chapter shifted our perspective from matrices as static tables of numbers to matrices as actions — linear transformations that stretch, rotate, reflect, project, and shear vector spaces. This viewpoint is the backbone of understanding how machine learning models work.

Key takeaways:

  • Linear maps preserve addition and scaling. A function is linear if and only if it can be represented as matrix multiplication. The columns of the matrix are the images of the basis vectors.
  • Kernel and image: the kernel tells you what information the map destroys; the image tells you what it can produce. The rank-nullity theorem guarantees nullity + rank = input dimension.
  • Change of basis transforms a matrix via \(A' = P^{-1}AP\). The same linear map has different matrix representations in different bases. Choosing the right basis (like the eigenbasis) can diagonalize the matrix.
  • Rotations preserve lengths and angles (\(\det = 1\)). Reflections flip orientation (\(\det = -1\)). Projections are idempotent (\(P^2 = P\)) and map onto subspaces.
  • The determinant measures how a transformation scales volume. Zero determinant means the map is singular (crushes a dimension). The sign indicates whether orientation is preserved or flipped.
Why This Matters for AI

Transformations power every neural network

  • Dense layers = linear maps: A neural network layer computes \(\mathbf{y} = \sigma(W\mathbf{x} + \mathbf{b})\). The weight matrix \(W\) is a linear transformation; the bias \(\mathbf{b}\) makes it affine; the activation \(\sigma\) adds the nonlinearity that makes deep learning work.
  • Kernel = blind spots: If a weight matrix has a large null space, there are input directions the network completely ignores. This is related to feature selection and model capacity.
  • PCA = change of basis: Principal Component Analysis finds the eigenvector basis of the covariance matrix, where the data's variance structure is diagonal. Dimensionality reduction is just projecting onto the top eigenvectors.
  • Determinants in normalizing flows: Generative models like normalizing flows transform a simple distribution (Gaussian) through invertible maps. Computing the likelihood requires the determinant of the Jacobian at each step — the volume-scaling factor of the transformation.
  • Attention = projection: In Transformers, queries, keys, and values are produced by linear maps \(Q = XW_Q, K = XW_K, V = XW_V\). Attention computes a weighted projection of values based on query-key similarity.

Socratic Knowledge Check

You're designing neural network layers. Reason through how linear transformations govern the flow of information.

The Scenario: You're building a neural network with an input layer of 784 neurons (flattened 28×28 images), a hidden layer of 128 neurons, and an output of 10 classes. Each layer applies a linear transformation (weight matrix) followed by a nonlinearity.
1
The first weight matrix \(W_1\) is 128×784. What are the kernel and image dimensions guaranteed by rank-nullity? If \(W_1\) has rank 128, what does that mean for the network?
By rank-nullity: nullity + rank = 784. If rank = 128, then nullity = 656. The network discards 656 dimensions of the input — that's 83% of the input information! The kernel contains all input variations that produce zero output. This massive compression is exactly what we want: the network learns to keep only the 128 most useful directions.
2
You accidentally initialize \(W_1\) as a projection matrix (so \(W_1^2 = W_1\)). Why is this a bad initialization? What happens to the gradients?
A projection matrix has eigenvalues of only 0 and 1. This means the transformation either keeps a direction unchanged or kills it completely — there's no "gentle" scaling. During backpropagation, gradients flowing through a projection are either passed unchanged or zeroed out. This leads to poor gradient flow and makes training difficult. Good initializations (like Xavier/He) ensure eigenvalues are distributed around 1 with some variance.
3
Someone proposes using a rotation matrix as the weight matrix for a layer. What are the advantages and disadvantages compared to a general learned matrix?
Advantages: rotation matrices have determinant 1 and preserve norms, so they prevent gradient explosion/vanishing — the gradient magnitude stays constant across layers. This is why orthogonal initialization works well for RNNs. Disadvantage: rotation can only rotate, not scale or compress. A general learned matrix can learn to amplify important directions and suppress irrelevant ones. Rotation matrices also require the input and output to have the same dimension (they're square and invertible).
4
You compute \(\det(W_1)\) and get a very small number (like \(10^{-30}\)). Is this a problem? What does it tell you about the transformation?
First, \(W_1\) is 128×784, which is non-square — it doesn't have a determinant! The question is a trap. Determinants are only defined for square matrices. For non-square matrices, the analogous quantity is the product of singular values. If the smallest singular values are near zero, it means the transformation nearly collapses some dimensions, which could cause numerical instability and poor conditioning.
5
You want to add a "skip connection" so the output of a layer is \(\mathbf{y} = W\mathbf{x} + \mathbf{x}\). Rewrite this as a single linear transformation. What does this tell you about ResNets?
\(\mathbf{y} = W\mathbf{x} + I\mathbf{x} = (W + I)\mathbf{x}\). The skip connection means the effective transformation is \(W + I\). Since \(I\) has all eigenvalues equal to 1, the eigenvalues of \(W + I\) are shifted by 1 compared to \(W\). This means even if \(W\) has small eigenvalues (near-zero gradients), the effective transformation has eigenvalues near 1, preserving gradient flow. This is the core insight behind ResNets — skip connections solve the vanishing gradient problem by ensuring the effective transformation is close to the identity.
