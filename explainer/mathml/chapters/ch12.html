Differentiation

Derivative theory, differentiation rules, chain rule, higher-order derivatives, and numerical differentiation in code.

45 min read Part II · Calculus
Mental Model

The Slope of a Microscope

Imagine zooming into a smooth curve with an infinitely powerful microscope. No matter how curved it looks from far away, as you zoom in the curve eventually looks like a straight line. The derivative is the slope of that line — the best local linear approximation. Every time a neural network computes a gradient, it is performing exactly this operation: asking "if I nudge each weight by a tiny amount, how much does the loss change?" That ratio — change in output over change in input, in the infinitesimal limit — is the derivative.

Section 12.1

The Derivative

Definition — Derivative

Let \(f : (a, b) \to \mathbb{R}\). The derivative of \(f\) at \(x_0 \in (a, b)\) is

\[f'(x_0) = \lim_{h \to 0} \frac{f(x_0 + h) - f(x_0)}{h},\]

provided this limit exists. If \(f'(x_0)\) exists, we say \(f\) is differentiable at \(x_0\).

The fraction \(\frac{f(x_0 + h) - f(x_0)}{h}\) is the slope of the secant line through the points \((x_0, f(x_0))\) and \((x_0 + h, f(x_0 + h))\). As \(h \to 0\), this secant line pivots into the tangent line, and its slope becomes the derivative.

Theorem — Differentiable Implies Continuous

If \(f\) is differentiable at \(x_0\), then \(f\) is continuous at \(x_0\).

We need to show \(\lim_{x \to x_0} f(x) = f(x_0)\). Write \(f(x) - f(x_0) = \frac{f(x) - f(x_0)}{x - x_0} \cdot (x - x_0)\) for \(x \neq x_0\). As \(x \to x_0\), the first factor tends to \(f'(x_0)\) (a finite number) and the second tends to 0. By the product rule for limits, \(f(x) - f(x_0) \to 0\), so \(f(x) \to f(x_0)\). \(\blacksquare\)

The converse is false: \(f(x) = |x|\) is continuous everywhere but not differentiable at \(x = 0\). The left-hand limit of the difference quotient is \(-1\) and the right-hand limit is \(+1\), so the two-sided limit does not exist. This is exactly why ReLU, defined by \(\max(0, x)\), is not differentiable at the origin — yet frameworks like PyTorch handle it by defining the derivative as 0 at that single point.

Leibniz Notation

The derivative is written in several equivalent ways: \(f'(x)\), \(\frac{df}{dx}\), \(\frac{d}{dx}f(x)\), or \(Df(x)\). Leibniz notation \(\frac{df}{dx}\) is especially useful because it reminds us that the derivative is a ratio of infinitesimal changes. The "operator" notation \(\frac{d}{dx}\) treats differentiation as a linear map acting on the space of functions.

Key Derivatives to Know

\(\frac{d}{dx} x^n = nx^{n-1}\) (power rule), \(\frac{d}{dx} e^x = e^x\), \(\frac{d}{dx} \ln x = \frac{1}{x}\), \(\frac{d}{dx} \sin x = \cos x\), \(\frac{d}{dx} \cos x = -\sin x\). These building blocks, combined with the rules of the next section, let you differentiate almost any function you will encounter.

Section 12.2

Differentiation Rules

Rather than computing limits from scratch every time, we use a small collection of rules that let us differentiate any algebraic or transcendental expression built from known pieces.

Theorem — Algebraic Rules

If \(f\) and \(g\) are differentiable at \(x\) and \(c \in \mathbb{R}\):

Constant multiple: \((cf)' = cf'\)

Sum: \((f + g)' = f' + g'\)

Product: \((fg)' = f'g + fg'\)

Quotient: \(\left(\frac{f}{g}\right)' = \frac{f'g - fg'}{g^2}\) when \(g(x) \neq 0\)

Product Rule. We compute:

\[\frac{(fg)(x+h) - (fg)(x)}{h} = \frac{f(x+h)g(x+h) - f(x)g(x)}{h}.\]

Add and subtract \(f(x+h)g(x)\) in the numerator:

\[= f(x+h)\frac{g(x+h) - g(x)}{h} + g(x)\frac{f(x+h) - f(x)}{h}.\]

As \(h \to 0\): \(f(x+h) \to f(x)\) (since \(f\) is differentiable, hence continuous), the first difference quotient \(\to g'(x)\), and the second \(\to f'(x)\). So the limit is \(f(x)g'(x) + g(x)f'(x)\). \(\blacksquare\)

The linearity of differentiation (constant multiple + sum rules) is the mathematical reason that gradients distribute across additions. Every node in a computation graph that adds two values simply passes the gradient through to both inputs unchanged.

Common ML Activation Derivatives

Applying these rules to the activation functions used in neural networks:

Sigmoid: \(\sigma(x) = \frac{1}{1 + e^{-x}}\). Using the quotient rule (or a clever rewrite), \(\sigma'(x) = \sigma(x)(1 - \sigma(x))\).

Tanh: \(\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}\). Its derivative is \(\tanh'(x) = 1 - \tanh^2(x)\).

Softplus: \(\text{softplus}(x) = \ln(1 + e^x)\). Its derivative is \(\frac{e^x}{1 + e^x} = \sigma(x)\) — the sigmoid function itself.

Python
import numpy as np

# Activation functions and their analytical derivatives
def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def sigmoid_deriv(x):
    s = sigmoid(x)
    return s * (1 - s)

def tanh_deriv(x):
    return 1 - np.tanh(x)**2

def relu_deriv(x):
    return (x > 0).astype(float)

# Verify with numerical derivative
def numerical_deriv(f, x, h=1e-7):
    return (f(x + h) - f(x - h)) / (2 * h)

x = np.array([-2.0, -0.5, 0.0, 0.5, 2.0])
print("Sigmoid analytical:", sigmoid_deriv(x))
print("Sigmoid numerical: ", numerical_deriv(sigmoid, x))
# They match to ~10 decimal places
Section 12.3

The Chain Rule

The chain rule is the most important differentiation rule for machine learning. It tells us how to differentiate compositions of functions — and a neural network is nothing but a deep composition.

Theorem — Chain Rule

If \(g\) is differentiable at \(x\) and \(f\) is differentiable at \(g(x)\), then the composition \(f \circ g\) is differentiable at \(x\) and

\[(f \circ g)'(x) = f'(g(x)) \cdot g'(x).\]

In Leibniz notation: if \(y = f(u)\) and \(u = g(x)\), then \(\frac{dy}{dx} = \frac{dy}{du} \cdot \frac{du}{dx}\).

Define \(\phi(t) = \frac{f(t) - f(g(x))}{t - g(x)}\) for \(t \neq g(x)\) and \(\phi(g(x)) = f'(g(x))\). Then \(\phi\) is continuous at \(g(x)\) (by differentiability of \(f\)). We can write:

\[f(g(x+h)) - f(g(x)) = \phi(g(x+h))\bigl(g(x+h) - g(x)\bigr).\]

Dividing by \(h\): \(\frac{f(g(x+h)) - f(g(x))}{h} = \phi(g(x+h)) \cdot \frac{g(x+h) - g(x)}{h}\).

As \(h \to 0\): \(g(x+h) \to g(x)\) (continuity of \(g\)), so \(\phi(g(x+h)) \to \phi(g(x)) = f'(g(x))\), and \(\frac{g(x+h)-g(x)}{h} \to g'(x)\). Thus \((f \circ g)'(x) = f'(g(x)) \cdot g'(x)\). \(\blacksquare\)

The Chain Rule as Backpropagation

Consider a two-layer network computing \(L = \ell(\sigma(Wx + b))\), where \(\ell\) is a loss function, \(\sigma\) is an activation, and \(W, b\) are parameters. To compute \(\frac{\partial L}{\partial W}\), we apply the chain rule through the entire composition:

\[\frac{\partial L}{\partial W} = \underbrace{\frac{\partial \ell}{\partial \sigma}}_{\text{loss derivative}} \cdot \underbrace{\frac{\partial \sigma}{\partial z}}_{\text{activation derivative}} \cdot \underbrace{\frac{\partial z}{\partial W}}_{\text{linear layer}}\]

where \(z = Wx + b\). This is exactly what backpropagation does: it traverses the computation graph in reverse, multiplying local derivatives at each node. The chain rule guarantees this product gives the correct overall gradient.

Chain Rule for Multiple Compositions

For a depth-\(d\) network \(f = f_d \circ f_{d-1} \circ \cdots \circ f_1\), the chain rule gives \(f'(x) = f_d'(z_{d-1}) \cdot f_{d-1}'(z_{d-2}) \cdots f_1'(x)\), where \(z_k = f_k(z_{k-1})\). When these factors are consistently less than 1 in magnitude, we get vanishing gradients; when greater than 1, exploding gradients.

Python
import numpy as np

def chain_rule_demo():
    """Demonstrate the chain rule through a 3-layer network."""
    np.random.seed(42)

    # Forward pass: x → z1 → z2 → loss
    x = np.array([1.0, 2.0])
    W1 = np.random.randn(3, 2) * 0.5
    W2 = np.random.randn(2, 3) * 0.5

    z1 = W1 @ x                          # linear layer 1
    a1 = np.maximum(z1, 0)               # ReLU activation
    z2 = W2 @ a1                         # linear layer 2
    loss = 0.5 * np.sum(z2**2)           # MSE-style loss

    # Backward pass: chain rule in reverse
    dL_dz2 = z2                           # d(0.5 ||z2||²)/dz2 = z2
    dL_da1 = W2.T @ dL_dz2               # chain through linear layer 2
    dL_dz1 = dL_da1 * (z1 > 0)           # chain through ReLU
    dL_dW1 = np.outer(dL_dz1, x)         # chain through linear layer 1

    print(f"Loss: {loss:.4f}")
    print(f"dL/dW1 shape: {dL_dW1.shape}")
    print(f"dL/dW1:\n{dL_dW1}")

    # Verify with numerical gradient
    eps = 1e-5
    dL_dW1_num = np.zeros_like(W1)
    for i in range(W1.shape[0]):
        for j in range(W1.shape[1]):
            W1[i,j] += eps
            loss_p = 0.5 * np.sum((W2 @ np.maximum(W1 @ x, 0))**2)
            W1[i,j] -= 2*eps
            loss_m = 0.5 * np.sum((W2 @ np.maximum(W1 @ x, 0))**2)
            W1[i,j] += eps
            dL_dW1_num[i,j] = (loss_p - loss_m) / (2*eps)

    print(f"\nMax error vs numerical: {np.max(np.abs(dL_dW1 - dL_dW1_num)):.2e}")

chain_rule_demo()
Section 12.4

Higher-Order Derivatives

If \(f'\) is itself differentiable, its derivative \(f'' = (f')'\) is the second derivative. The \(n\)-th derivative is denoted \(f^{(n)}\) or \(\frac{d^n f}{dx^n}\).

Definition — Smoothness Classes

A function \(f\) belongs to the class \(C^k\) if its first \(k\) derivatives exist and are all continuous. The class \(C^\infty\) (smooth functions) consists of functions with derivatives of all orders. Most activation functions used in deep learning are \(C^\infty\) (sigmoid, tanh, softplus, GELU), except ReLU which is only \(C^0\).

The Second Derivative and Curvature

The first derivative gives rate of change; the second derivative gives rate of change of the rate of change — curvature. For optimisation:

If \(f''(x_0) > 0\), the function curves upward at \(x_0\) (local minimum shape). If \(f''(x_0) < 0\), it curves downward (local maximum shape). If \(f''(x_0) = 0\), the test is inconclusive (could be an inflection point).

Theorem — Taylor's Theorem (with Remainder)

If \(f \in C^{n+1}\) on an interval containing \(x_0\), then for any \(x\) in that interval:

\[f(x) = \sum_{k=0}^{n} \frac{f^{(k)}(x_0)}{k!}(x - x_0)^k + R_{n+1}(x),\]

where the remainder \(R_{n+1}(x) = \frac{f^{(n+1)}(c)}{(n+1)!}(x - x_0)^{n+1}\) for some \(c\) between \(x_0\) and \(x\).

The first-order Taylor expansion \(f(x) \approx f(x_0) + f'(x_0)(x - x_0)\) is exactly the tangent line — and is the linearisation that gradient descent uses. The second-order expansion \(f(x) \approx f(x_0) + f'(x_0)(x - x_0) + \frac{1}{2}f''(x_0)(x - x_0)^2\) gives a quadratic approximation and is the basis of Newton's method.

Newton's Method in 1D

To find the minimum of \(f\), set the derivative of the quadratic approximation to zero: \(f'(x_0) + f''(x_0)(x - x_0) = 0\), yielding the update \(x_{k+1} = x_k - \frac{f'(x_k)}{f''(x_k)}\). This converges quadratically near a minimum where \(f'' > 0\).

Section 12.5

The Mean Value Theorem

Theorem — Mean Value Theorem

If \(f\) is continuous on \([a, b]\) and differentiable on \((a, b)\), then there exists \(c \in (a, b)\) such that

\[f'(c) = \frac{f(b) - f(a)}{b - a}.\]

Define \(g(x) = f(x) - f(a) - \frac{f(b)-f(a)}{b-a}(x - a)\). Then \(g(a) = g(b) = 0\), \(g\) is continuous on \([a,b]\) and differentiable on \((a,b)\). By Rolle's theorem, there exists \(c \in (a,b)\) with \(g'(c) = 0\). Computing: \(g'(c) = f'(c) - \frac{f(b)-f(a)}{b-a} = 0\), which gives the result. \(\blacksquare\)

The MVT is the workhorse of analysis proofs. It converts information about derivatives into information about function values. A direct consequence: if \(f'(x) = 0\) for all \(x\) in an interval, then \(f\) is constant — the derivative completely determines the function up to a constant.

Proposition — Lipschitz from Bounded Derivative

If \(|f'(x)| \leq L\) for all \(x \in (a, b)\), then \(|f(x) - f(y)| \leq L|x - y|\) for all \(x, y \in [a, b]\). That is, a bounded derivative implies Lipschitz continuity with the same constant.

This connects directly to Chapter 11: the derivative provides a computable way to verify Lipschitz continuity. In ML, bounding the gradient norm bounds the Lipschitz constant of the loss, which in turn controls the stability of gradient descent.

Section 12.6

Numerical Differentiation

When analytic derivatives are unavailable or you need to verify them, numerical approximations step in. Three standard finite-difference formulas are:

Forward difference: \(f'(x) \approx \frac{f(x+h) - f(x)}{h}\) — error \(O(h)\)

Backward difference: \(f'(x) \approx \frac{f(x) - f(x-h)}{h}\) — error \(O(h)\)

Central difference: \(f'(x) \approx \frac{f(x+h) - f(x-h)}{2h}\) — error \(O(h^2)\)

The central difference is more accurate because the first-order error terms cancel. By Taylor expansion: \(f(x+h) = f(x) + f'(x)h + \frac{1}{2}f''(x)h^2 + O(h^3)\) and \(f(x-h) = f(x) - f'(x)h + \frac{1}{2}f''(x)h^2 - O(h^3)\). Subtracting and dividing by \(2h\) kills the \(h^2\) terms.

Performance Deep-Dive

Choosing the Optimal Step Size

There is a fundamental tension: smaller \(h\) reduces truncation error but increases floating-point round-off error. For central differences, the optimal step size is approximately \(h^* \approx \varepsilon_{\text{mach}}^{1/3} \approx 6 \times 10^{-6}\) (for float64), giving total error \(\approx \varepsilon_{\text{mach}}^{2/3} \approx 4 \times 10^{-11}\).

import numpy as np

def numerical_derivatives(f, x, hs=None):
    """Compare forward, backward, and central differences."""
    if hs is None:
        hs = np.logspace(-1, -15, 15)

    results = {"h": [], "forward": [], "central": []}
    for h in hs:
        fwd = (f(x + h) - f(x)) / h
        ctr = (f(x + h) - f(x - h)) / (2 * h)
        results["h"].append(h)
        results["forward"].append(fwd)
        results["central"].append(ctr)
    return results

# Test on f(x) = sin(x) at x = 1.0, true derivative = cos(1)
f = np.sin
x0, true_deriv = 1.0, np.cos(1.0)

res = numerical_derivatives(f, x0)
print(f"True f'({x0}) = {true_deriv:.15f}\n")
print(f"{'h':>12s}  {'Forward err':>14s}  {'Central err':>14s}")
for h, fwd, ctr in zip(res["h"], res["forward"], res["central"]):
    print(f"{h:12.1e}  {abs(fwd-true_deriv):14.2e}  "
          f"{abs(ctr-true_deriv):14.2e}")
# Central difference achieves ~1e-11 error at h ≈ 1e-5

Second Derivative via Finite Differences

The standard second-derivative approximation is:

\[f''(x) \approx \frac{f(x+h) - 2f(x) + f(x-h)}{h^2},\]

which has error \(O(h^2)\). This "three-point stencil" appears throughout scientific computing and is used in finite-difference PDE solvers.

Section 12.7

Automatic Differentiation

Numerical differentiation is slow (requires extra function evaluations) and imprecise (round-off). Symbolic differentiation is exact but produces expression swell for complex functions. Automatic differentiation (autodiff) combines the best of both: exact derivatives at machine precision with computational cost proportional to the forward pass.

Forward Mode

Forward-mode autodiff propagates dual numbers \(a + b\varepsilon\) (where \(\varepsilon^2 = 0\)) through the computation. If \(f(a + \varepsilon) = f(a) + f'(a)\varepsilon\), then the coefficient of \(\varepsilon\) is the exact derivative. Each arithmetic operation obeys: \((a + b\varepsilon)(c + d\varepsilon) = ac + (ad + bc)\varepsilon\).

Python
import math

class Dual:
    """Dual number for forward-mode automatic differentiation."""
    def __init__(self, val, deriv=0.0):
        self.val = val
        self.deriv = deriv

    def __add__(self, other):
        o = other if isinstance(other, Dual) else Dual(other)
        return Dual(self.val + o.val, self.deriv + o.deriv)

    def __radd__(self, other):
        return self.__add__(other)

    def __mul__(self, other):
        o = other if isinstance(other, Dual) else Dual(other)
        return Dual(self.val * o.val,
                    self.val * o.deriv + self.deriv * o.val)

    def __rmul__(self, other):
        return self.__mul__(other)

    def __sub__(self, other):
        o = other if isinstance(other, Dual) else Dual(other)
        return Dual(self.val - o.val, self.deriv - o.deriv)

    def __truediv__(self, other):
        o = other if isinstance(other, Dual) else Dual(other)
        return Dual(self.val / o.val,
                    (self.deriv * o.val - self.val * o.deriv) / o.val**2)

    def __neg__(self):
        return Dual(-self.val, -self.deriv)

    def __pow__(self, n):
        return Dual(self.val**n, n * self.val**(n-1) * self.deriv)

def sin(x):
    if isinstance(x, Dual):
        return Dual(math.sin(x.val), math.cos(x.val) * x.deriv)
    return math.sin(x)

def exp(x):
    if isinstance(x, Dual):
        e = math.exp(x.val)
        return Dual(e, e * x.deriv)
    return math.exp(x)

# Compute derivative of f(x) = sin(x²) · exp(x) at x = 1.5
x = Dual(1.5, 1.0)   # seed derivative = 1.0
result = sin(x**2) * exp(x)
print(f"f(1.5)  = {result.val:.10f}")
print(f"f'(1.5) = {result.deriv:.10f}")  # exact to machine precision

Reverse Mode (Backpropagation)

For a function \(f : \mathbb{R}^n \to \mathbb{R}\), forward mode computes one directional derivative per pass — computing the full gradient requires \(n\) passes. Reverse mode computes the entire gradient in a single backward pass, which is why it is the method of choice for neural networks (where \(n\) can be billions of parameters but the output is a scalar loss).

Reverse mode first performs a forward pass, recording all intermediate values. Then it traverses the computation graph backward, accumulating gradients via the chain rule. The total cost is roughly 2–3× the forward pass — independent of the number of parameters.

Why This Matters for AI/ML

Differentiation is the Engine of Learning

  • Backpropagation = chain rule: Every training step in deep learning applies the chain rule through a computation graph. The chain rule is not just a mathematical convenience — it is the algorithm.
  • Gradient descent: The update \(\theta \leftarrow \theta - \eta \nabla L(\theta)\) uses the derivative to point "downhill." Without differentiation, there is no systematic way to improve parameters.
  • Activation design: The derivative of an activation function controls gradient flow. Sigmoid saturates (\(\sigma' \to 0\) for large \(|x|\)), causing vanishing gradients. ReLU maintains unit gradient for positive inputs but has zero gradient for negative — the "dying ReLU" problem.
  • Second-order methods: Newton's method and natural gradient use second derivatives (Hessian) for better curvature-aware updates. Higher-order Taylor approximations underpin trust-region methods and L-BFGS.
  • Autodiff frameworks: PyTorch, JAX, and TensorFlow all implement reverse-mode autodiff. Understanding the mathematics lets you debug gradient issues, write custom backward passes, and know when to use torch.no_grad().

Socratic Knowledge Check

You are training a deep neural network and observing that gradients vanish after a few layers. Your colleague suggests switching from sigmoid to ReLU activations. Using the theory from this chapter, analyse the situation.
1
Compute the derivative of \(f(x) = x \cdot \sigma(x)\) (the SiLU/Swish activation). Is it always positive?
Using the product rule: \(f'(x) = \sigma(x) + x \cdot \sigma(x)(1 - \sigma(x)) = \sigma(x)(1 + x(1 - \sigma(x)))\). This is not always positive — for sufficiently negative \(x\), \(f'(x)\) becomes slightly negative (the function dips below zero near \(x \approx -1.28\)). Unlike ReLU which has zero derivative for \(x < 0\), SiLU maintains small non-zero gradients, helping avoid the dying ReLU problem.
2
Why does the chain rule predict vanishing gradients with sigmoid activations in a deep network?
For a depth-\(d\) network, the chain rule gives \(\frac{\partial L}{\partial W_1} = \prod_{k=1}^{d} \sigma'(z_k) \cdot (\text{other terms})\). Since \(\sigma'(x) = \sigma(x)(1-\sigma(x)) \leq 1/4\) for all \(x\), the product \(\prod \sigma'(z_k) \leq (1/4)^d\), which shrinks exponentially with depth. For \(d = 20\) layers, gradients are attenuated by at least \(4^{-20} \approx 10^{-12}\). ReLU has gradient 1 for positive inputs, so the product can remain \(O(1)\).
3
You compute a numerical gradient using central differences with \(h = 10^{-15}\) and get a wildly wrong answer. Why?
At \(h = 10^{-15}\), the perturbation is near the machine epsilon (\(\approx 2.2 \times 10^{-16}\) for float64). The values \(f(x + h)\) and \(f(x - h)\) differ by less than one ULP (unit in the last place) of \(f(x)\), so the subtraction suffers catastrophic cancellation. The "difference" is dominated by round-off noise. The optimal step size for central differences is \(h^* \approx \varepsilon_{\text{mach}}^{1/3} \approx 6 \times 10^{-6}\).
4
Forward-mode autodiff computes one directional derivative per pass. How many passes would you need for a model with 100M parameters? Why is reverse mode better here?
Forward mode requires one pass per input dimension, so 100 million passes — completely impractical. Reverse mode computes the gradient with respect to all 100M parameters in a single backward pass, at roughly 2–3× the cost of a forward pass. This is because the loss is a scalar (\(\mathbb{R}^{100M} \to \mathbb{R}\)), and reverse mode is efficient when the output dimension is much smaller than the input dimension. If you needed Jacobians of \(\mathbb{R}^n \to \mathbb{R}^m\) with \(m \gg n\), forward mode would be more efficient.
5
Using Taylor's theorem, explain why the central difference formula has \(O(h^2)\) error while the forward difference has only \(O(h)\) error.
Taylor expand: \(f(x+h) = f(x) + f'h + \frac{1}{2}f''h^2 + \frac{1}{6}f'''h^3 + \cdots\) and \(f(x-h) = f(x) - f'h + \frac{1}{2}f''h^2 - \frac{1}{6}f'''h^3 + \cdots\). The forward difference gives \(\frac{f(x+h)-f(x)}{h} = f' + \frac{1}{2}f''h + O(h^2)\), so the error is \(O(h)\). The central difference gives \(\frac{f(x+h)-f(x-h)}{2h} = \frac{2f'h + \frac{2}{6}f'''h^3 + \cdots}{2h} = f' + \frac{1}{6}f'''h^2 + O(h^4)\). The even-order terms (\(f''\)) cancel, leaving error \(O(h^2)\) — a full order of magnitude better.
