Multivariable Functions

What is a multivariable function, linear functions in multiple variables, and the curse of dimensionality.

30 min read Part III · Multivariable Calculus
Mental Model

From a Single Dial to a Control Panel

In single-variable calculus, a function is like a machine with one dial: you turn the dial (change \(x\)) and read one output (\(f(x)\)). A multivariable function is a machine with an entire control panel — dozens, thousands, or even billions of dials. Turn any combination and a new output appears. Every neural network is exactly this: the input might be a 224×224×3 image (150,528 dials), the parameters are millions more dials, and the output is a class probability vector. To do calculus on these systems, we first need to understand what a "function of many variables" actually means.

Section 15.1

What Is a Multivariable Function?

Definition — Multivariable Function

A multivariable function (or multivariate function) is a map

\[f : \mathbb{R}^n \to \mathbb{R}^m\]

that assigns to each input vector \(\mathbf{x} = (x_1, x_2, \ldots, x_n) \in \mathbb{R}^n\) an output vector \(\mathbf{y} = (y_1, y_2, \ldots, y_m) \in \mathbb{R}^m\). The number \(n\) is the input dimension and \(m\) is the output dimension.

Special cases by dimension:

\(f : \mathbb{R}^n \to \mathbb{R}\) — a scalar-valued function of \(n\) variables. Examples: loss functions, probability densities, the determinant of a matrix. These are the functions we optimise in ML.

\(f : \mathbb{R}^n \to \mathbb{R}^m\) with \(m > 1\) — a vector-valued function. Examples: neural network forward passes (mapping inputs to logits), feature extractors, residual connections.

\(f : \mathbb{R} \to \mathbb{R}^m\) — a curve in \(\mathbb{R}^m\). Example: the trajectory of gradient descent through parameter space is a curve \(\theta(t)\) parameterised by "time" (iteration number).

Notation Convention

We write vectors in bold (\(\mathbf{x}\)), scalars in regular type (\(x_i\)), and matrices in bold uppercase (\(\mathbf{W}\)). A function \(f : \mathbb{R}^n \to \mathbb{R}^m\) can always be decomposed into \(m\) scalar-valued component functions: \(f(\mathbf{x}) = (f_1(\mathbf{x}), \ldots, f_m(\mathbf{x}))\), where each \(f_i : \mathbb{R}^n \to \mathbb{R}\).

Section 15.2

Vectors as Inputs and Outputs

In practice, the "multiple variables" of a multivariable function are packed into vectors and tensors. This is not just notation — it reflects how hardware processes data.

Python
import numpy as np

# A scalar-valued function of 3 variables: f(x₁, x₂, x₃) = x₁² + x₂x₃
def f_scalar(x):
    return x[0]**2 + x[1] * x[2]

x = np.array([1.0, 2.0, 3.0])
print(f"f({x}) = {f_scalar(x)}")  # 1 + 6 = 7

# A vector-valued function: f: R³ → R²
def f_vector(x):
    return np.array([
        x[0] + x[1],       # f₁
        x[1] * x[2] - x[0] # f₂
    ])

print(f"f({x}) = {f_vector(x)}")  # [3, 5]

# Neural network layer: f(x) = σ(Wx + b), mapping R³ → R²
W = np.random.randn(2, 3) * 0.5
b = np.zeros(2)
sigmoid = lambda z: 1 / (1 + np.exp(-z))

def layer(x):
    return sigmoid(W @ x + b)

print(f"layer({x}) = {layer(x)}")  # two probabilities in (0, 1)

Level Sets and Contour Plots

For scalar-valued functions \(f : \mathbb{R}^2 \to \mathbb{R}\), a level set (or contour) at value \(c\) is the set \(\{\mathbf{x} \in \mathbb{R}^2 : f(\mathbf{x}) = c\}\). Contour plots visualise these level sets — each contour line connects points with the same function value. For loss surfaces, the minimum sits at the centre of nested contour ellipses (for quadratics) or more complex shapes (for non-convex losses).

Python
import numpy as np

def make_contour_data(f, xlim=(-3, 3), ylim=(-3, 3), n=100):
    """Generate grid data for contour plotting."""
    x = np.linspace(*xlim, n)
    y = np.linspace(*ylim, n)
    X, Y = np.meshgrid(x, y)
    Z = np.array([[f(np.array([xi, yi])) for xi in x] for yi in y])
    return X, Y, Z

# Quadratic loss: f(x) = x₁² + 4x₂² (elliptical contours)
quadratic = lambda x: x[0]**2 + 4*x[1]**2
X, Y, Z = make_contour_data(quadratic)
print(f"Min value: {Z.min():.4f} at grid centre")

# Rosenbrock: f(x) = (1-x₁)² + 100(x₂-x₁²)²
rosenbrock = lambda x: (1 - x[0])**2 + 100*(x[1] - x[0]**2)**2
X2, Y2, Z2 = make_contour_data(rosenbrock, xlim=(-2, 2), ylim=(-1, 3))
print(f"Rosenbrock min: {Z2.min():.6f} (true min = 0 at (1,1))")
Section 15.3

Linear Functions in Multiple Variables

Linear functions are the simplest and most important class of multivariable functions. They are exactly the matrix-vector products from Part I, now viewed through the lens of calculus.

Definition — Linear Function

A function \(f : \mathbb{R}^n \to \mathbb{R}^m\) is linear if:

1. \(f(\mathbf{x} + \mathbf{y}) = f(\mathbf{x}) + f(\mathbf{y})\) for all \(\mathbf{x}, \mathbf{y} \in \mathbb{R}^n\), and

2. \(f(\alpha \mathbf{x}) = \alpha f(\mathbf{x})\) for all \(\alpha \in \mathbb{R}\) and \(\mathbf{x} \in \mathbb{R}^n\).

Every linear function from \(\mathbb{R}^n\) to \(\mathbb{R}^m\) can be represented as \(f(\mathbf{x}) = A\mathbf{x}\) for a unique \(m \times n\) matrix \(A\).

An affine function adds a constant: \(f(\mathbf{x}) = A\mathbf{x} + \mathbf{b}\). Affine functions are the building blocks of neural networks — every "linear layer" (or "dense layer" or "fully connected layer") computes exactly \(\mathbf{z} = W\mathbf{x} + \mathbf{b}\).

Proposition — Linear Functions Are Smooth

Every linear function \(f(\mathbf{x}) = A\mathbf{x}\) is infinitely differentiable (\(C^\infty\)), Lipschitz continuous with constant \(\|A\|\) (operator norm), and its derivative at every point is the matrix \(A\) itself. This is why linear layers are computationally easy to differentiate — backpropagation through a linear layer is just a matrix transpose multiplication.

The Role of Nonlinearity

A composition of linear functions is still linear: \(A_2(A_1 \mathbf{x}) = (A_2 A_1)\mathbf{x}\). Without nonlinear activations, a 100-layer network would collapse to a single matrix multiplication — no more expressive than a one-layer network. This is precisely why activation functions (ReLU, sigmoid, tanh) are essential: they break linearity and allow deep networks to represent complex, nonlinear functions.

Section 15.4

Nonlinear Multivariable Functions

Most functions in machine learning are nonlinear. The key insight from calculus is that we can still understand nonlinear functions by locally approximating them with linear ones — this is the entire programme of differential calculus extended to multiple variables.

Common Nonlinear Functions in ML

Softmax: \(\text{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_j e^{z_j}}\), mapping \(\mathbb{R}^n \to \Delta^{n-1}\) (the probability simplex). This is the output layer of nearly every classification network.

Norms: \(\|\mathbf{x}\|_2 = \sqrt{\sum_i x_i^2}\), mapping \(\mathbb{R}^n \to \mathbb{R}\). The squared norm \(\|\mathbf{x}\|^2\) appears in L2 regularisation and least-squares losses.

Element-wise activations: \(\sigma(\mathbf{z}) = (\sigma(z_1), \ldots, \sigma(z_n))\) applies a scalar function to each component independently. This is a map \(\mathbb{R}^n \to \mathbb{R}^n\).

Python
import numpy as np

def softmax(z):
    """Numerically stable softmax: R^n → probability simplex."""
    z_shifted = z - np.max(z)  # subtract max for stability
    exp_z = np.exp(z_shifted)
    return exp_z / exp_z.sum()

# Softmax maps logits to probabilities
logits = np.array([2.0, 1.0, 0.1])
probs = softmax(logits)
print(f"Logits:  {logits}")
print(f"Softmax: {probs}")        # sums to 1
print(f"Sum:     {probs.sum():.6f}")

# A full neural network: composition of affine + nonlinear
def two_layer_net(x, W1, b1, W2, b2):
    """f: R^n → R^k, a 2-layer network."""
    z1 = W1 @ x + b1                   # affine: R^n → R^h
    a1 = np.maximum(z1, 0)             # ReLU:   R^h → R^h (nonlinear!)
    z2 = W2 @ a1 + b2                  # affine: R^h → R^k
    return softmax(z2)                  # softmax: R^k → Δ^{k-1}

np.random.seed(0)
n, h, k = 5, 10, 3
W1, b1 = np.random.randn(h, n)*0.3, np.zeros(h)
W2, b2 = np.random.randn(k, h)*0.3, np.zeros(k)
x = np.random.randn(n)
print(f"\nNetwork output: {two_layer_net(x, W1, b1, W2, b2)}")
Section 15.5

The Curse of Dimensionality

When we move from one or two variables to hundreds or millions, our geometric intuition breaks down in surprising ways. This collection of phenomena is called the curse of dimensionality.

Volume Concentrates at the Boundary

Consider the unit ball \(B_d = \{\mathbf{x} \in \mathbb{R}^d : \|\mathbf{x}\| \leq 1\}\). The fraction of volume in the "shell" between radius \(1 - \varepsilon\) and \(1\) is \(1 - (1-\varepsilon)^d\). For \(d = 100\) and \(\varepsilon = 0.05\):

\[1 - (0.95)^{100} \approx 1 - 0.006 = 0.994.\]

Over 99% of the volume is in the thin outer shell! In high dimensions, "almost all" of a ball's volume is concentrated near its surface.

Distances Become Uniform

For \(n\) random points uniformly distributed in the unit cube \([0,1]^d\), as \(d \to \infty\) the ratio of the maximum pairwise distance to the minimum pairwise distance approaches 1. All points are "roughly equally far apart" — the concept of nearest neighbour loses meaning.

Python
import numpy as np
from scipy.spatial.distance import pdist

def curse_demo(dims, n_points=200, n_trials=10):
    """Demonstrate distance concentration in high dimensions."""
    print(f"{'Dim':>6}  {'Mean dist':>10}  {'Std dist':>10}  {'Max/Min':>10}")
    for d in dims:
        ratios = []
        for _ in range(n_trials):
            X = np.random.rand(n_points, d)
            dists = pdist(X)
            ratios.append(dists.max() / dists.min())
        mean_d = np.sqrt(d / 6)  # expected dist for uniform in [0,1]^d
        std_d = np.std(pdist(np.random.rand(n_points, d)))
        print(f"{d:6d}  {mean_d:10.2f}  {std_d:10.4f}  "
              f"{np.mean(ratios):10.4f}")

curse_demo([2, 10, 50, 100, 500, 1000])

Sampling Becomes Impossible

To cover a \(d\)-dimensional unit cube with a grid of spacing \(h\), you need \((1/h)^d\) points. With \(h = 0.1\) (10 points per axis): \(d = 2\) needs 100 points, \(d = 10\) needs \(10^{10}\), and \(d = 100\) needs \(10^{100}\). This is why grid-based methods (including Simpson's rule from Chapter 14) are useless in high dimensions, and why Monte Carlo and gradient-based methods dominate ML.

Performance Deep-Dive

Volume of the Unit Ball Vanishes

The volume of the \(d\)-dimensional unit ball approaches zero as \(d \to \infty\), even though it always fits inside the unit cube.

import numpy as np
from scipy.special import gamma

def unit_ball_volume(d):
    """Volume of the d-dimensional unit ball."""
    return np.pi**(d/2) / gamma(d/2 + 1)

# The volume shrinks dramatically with dimension
for d in [1, 2, 3, 5, 10, 20, 50, 100]:
    vol = unit_ball_volume(d)
    cube_vol = 2**d  # volume of [-1,1]^d
    ratio = vol / cube_vol  # fraction of cube occupied by ball
    print(f"d={d:3d}: V(ball) = {vol:12.4e}, "
          f"ball/cube = {ratio:12.4e}")
Section 15.6

Multivariable Functions in ML

Why This Matters for AI/ML

Everything in ML Is Multivariable

  • Inputs are high-dimensional: Images (150K+ pixels), text (token embeddings of dimension 768–12288), tabular data (tens to hundreds of features). Single-variable calculus cannot handle these.
  • Parameters are higher-dimensional: GPT-scale models have billions of parameters. The loss is a function \(L : \mathbb{R}^d \to \mathbb{R}\) where \(d\) can exceed \(10^{11}\). Understanding this function's geometry is the core challenge of training.
  • The curse of dimensionality shapes algorithm design: k-NN fails in high dimensions (all neighbours are equidistant). Grid search is impossible. This is why gradient-based optimisation, random projections, and learned representations are the dominant paradigms.
  • Linear layers are matrix multiplications: The connection between linear algebra (Part I) and multivariable calculus (Part III) is direct. A linear layer is both a matrix multiplication and a linear function — its derivative is the weight matrix itself.
  • The forward pass is function composition: A deep network computes \(f = f_L \circ \cdots \circ f_1\), where each \(f_i\) is a multivariable function (affine + activation). Understanding how to differentiate these compositions (Chapter 16) is the mathematical foundation of backpropagation.

Socratic Knowledge Check

You are designing a feature engineering pipeline for a tabular dataset with 500 features. A colleague suggests adding all pairwise interaction terms (products of every pair of features). Think about what this means from the perspective of dimensionality.
1
What are the input and output dimensions of a convolutional layer that takes a 32×32×3 image and produces 16 feature maps of size 30×30?
The input dimension is \(32 \times 32 \times 3 = 3{,}072\) and the output dimension is \(30 \times 30 \times 16 = 14{,}400\). As a multivariable function, this layer maps \(\mathbb{R}^{3072} \to \mathbb{R}^{14400}\). Note that despite having fewer input than output dimensions, the number of parameters is much smaller (e.g., 16 filters of size 3×3×3 plus 16 biases = 448 parameters) due to weight sharing — a key design choice that avoids the curse of dimensionality.
2
Why is a composition of two linear functions \(g(\mathbf{x}) = B(A\mathbf{x})\) still linear? What does this imply about a "deep linear network" (no activations)?
\(g(\mathbf{x}) = B(A\mathbf{x}) = (BA)\mathbf{x}\), which is multiplication by the single matrix \(C = BA\). The composition is linear. A deep linear network with layers \(W_L, \ldots, W_1\) computes \(W_L \cdots W_1 \mathbf{x}\), which is equivalent to a single linear transformation \(C\mathbf{x}\) where \(C = W_L \cdots W_1\). Adding more layers adds no representational power — the network can only learn linear functions. This is why nonlinear activations are essential for depth to matter.
3
In 100 dimensions, what fraction of the unit cube \([0,1]^{100}\) lies within distance 0.01 of a face? (Hint: points are "far from a face" only if all coordinates are in \([0.01, 0.99]\).)
A point is far from every face only if every coordinate \(x_i \in [0.01, 0.99]\). The volume of this inner cube is \(0.98^{100} \approx 0.133\). So the fraction near a face is \(1 - 0.98^{100} \approx 0.867\). Over 86% of points lie within 0.01 of a boundary! In high dimensions, almost all data lives near the edges of any bounding box — another manifestation of the curse of dimensionality.
4
Your colleague wants to add all pairwise interaction features to the 500-feature dataset. How many new features does this create? Why might this be problematic?
Pairwise interactions create \(\binom{500}{2} = 124{,}750\) new features. Combined with the original 500, the total is 125,250 features. This is problematic because: (1) the curse of dimensionality means you need exponentially more data to fill this higher-dimensional space, (2) most pairwise interactions are likely irrelevant, adding noise, and (3) the model may overfit with \(\sim 125K\) features unless you have a correspondingly large dataset. A neural network learns relevant nonlinear interactions implicitly, which is more parameter-efficient than explicitly enumerating all pairs.
5
The softmax function maps \(\mathbb{R}^n \to \Delta^{n-1}\) (the probability simplex). Is softmax linear? Is it injective? What happens as one logit \(z_i \to \infty\)?
Softmax is not linear: \(\text{softmax}(\mathbf{z}_1 + \mathbf{z}_2) \neq \text{softmax}(\mathbf{z}_1) + \text{softmax}(\mathbf{z}_2)\). It is not injective: \(\text{softmax}(\mathbf{z}) = \text{softmax}(\mathbf{z} + c\mathbf{1})\) for any constant \(c\) (shift invariance). As \(z_i \to \infty\) with other logits fixed, \(\text{softmax}(\mathbf{z})_i \to 1\) and all other components \(\to 0\). The output approaches a one-hot vector, which is why "temperature scaling" (dividing logits by \(T\)) controls the sharpness of the distribution: low \(T\) makes outputs nearly one-hot, high \(T\) makes them nearly uniform.
