It's Just Logic

The foundation of mathematical reasoning — propositions, connectives, predicates, and quantifiers that make all formal mathematics possible.

25 min read
Foundation
The Big Picture

Mathematics is built on logic

Before we can do any mathematics — before we can prove theorems, solve equations, or train neural networks — we need to understand the rules of reasoning. This appendix covers the logical foundations: how to build and reason about true/false statements (propositions), how to combine them (connectives), and how to make general claims about objects (quantifiers). These aren't abstract philosophy. They're the underpinning of every algorithm, every proof, and every decision boundary you'll encounter in ML. Understanding logic lets you read mathematical definitions with confidence, spot logical flaws in arguments, and design more rigorous learning systems.

Section A.1

Propositions and Truth Values

A proposition is a declarative sentence that is either true or false, but not both.

Definition — Proposition

A proposition is a statement that has exactly one truth value: true (T) or false (F).

Examples:

  • "2 + 2 = 4" is a proposition (true)
  • "The Earth is square" is a proposition (false)
  • "Is it raining?" is NOT a proposition (it's a question)
  • "This statement is false" is NOT a proposition (paradox)

We denote propositions with letters like \(P\), \(Q\), \(R\), etc. We can assign each a truth value: \(P = \text{true}\) or \(P = \text{false}\).

Notation and conventions

Different texts use different notations:

Notation — Truth Values
  • \(\text{True} = T = 1 = \top\)
  • \(\text{False} = F = 0 = \bot\)

In machine learning, we often use 1 for true and 0 for false (like boolean flags in code).

Section A.2

Logical Connectives

We combine propositions using logical connectives (also called logical operators). Five are fundamental:

Conjunction (AND): \(\land\)

The conjunction \(P \land Q\) is true only when both \(P\) and \(Q\) are true.

Truth Table — AND

\(P\) | \(Q\) | \(P \land Q\)
T | T | T
T | F | F
F | T | F
F | F | F

Disjunction (OR): \(\lor\)

The disjunction \(P \lor Q\) is true when at least one of \(P\) or \(Q\) is true (inclusive OR).

Truth Table — OR

\(P\) | \(Q\) | \(P \lor Q\)
T | T | T
T | F | T
F | T | T
F | F | F

Negation (NOT): \(\neg\)

The negation \(\neg P\) flips the truth value.

Truth Table — NOT

\(P\) | \(\neg P\)
T | F
F | T

Implication: \(\rightarrow\)

The implication \(P \rightarrow Q\) ("if P then Q") is false only when \(P\) is true and \(Q\) is false.

Truth Table — Implication

\(P\) | \(Q\) | \(P \rightarrow Q\)
T | T | T
T | F | F
F | T | T
F | F | T

Key insight: An implication is true when the hypothesis is false (vacuous truth). "If unicorns exist, then 2+2=5" is technically true because unicorns don't exist.

Biconditional: \(\leftrightarrow\)

The biconditional \(P \leftrightarrow Q\) ("P if and only if Q") is true when both propositions have the same truth value.

Truth Table — Biconditional

\(P\) | \(Q\) | \(P \leftrightarrow Q\)
T | T | T
T | F | F
F | T | F
F | F | T

Computing with truth tables

We can evaluate any complex logical expression by building a truth table:

Implementation

Truth table generator in Python

import itertools

def truth_table(propositions, expression_func):
    """
    Generate truth table for a logical expression.
    propositions: list of variable names, e.g., ['P', 'Q', 'R']
    expression_func: function that takes dict of assignments and returns T/F
    """
    results = []
    for assignment in itertools.product([True, False], repeat=len(propositions)):
        values = dict(zip(propositions, assignment))
        result = expression_func(values)
        results.append((*assignment, result))
    return results

# Example: (P AND Q) OR (NOT Q)
def example(vals):
    P, Q = vals['P'], vals['Q']
    return (P and Q) or (not Q)

table = truth_table(['P', 'Q'], example)
for row in table:
    P, Q, result = row
    print(f"P={P}, Q={Q} -> {result}")
Section A.3

Propositional Calculus

Propositional calculus is the study of how propositions and connectives interact. Three key concepts:

Tautologies

A tautology is a proposition that is always true, regardless of the truth values of its components.

Definition — Tautology

Example: \(P \lor \neg P\) (the law of excluded middle) is always true.

P | T | F
¬P | F | T
P ∨ ¬P | T | T

Contradictions

A contradiction is always false.

Definition — Contradiction

Example: \(P \land \neg P\) is always false.

Logical equivalence

Two propositions are logically equivalent (written \(\equiv\)) if they have the same truth value in all cases.

Theorem — De Morgan's Laws

\(\neg (P \land Q) \equiv (\neg P) \lor (\neg Q)\)

\(\neg (P \lor Q) \equiv (\neg P) \land (\neg Q)\)

In words: the negation of a conjunction is the disjunction of the negations, and vice versa.

For the first law, \(\neg(P \land Q) \equiv (\neg P) \lor (\neg Q)\):

P | Q | P∧Q | ¬(P∧Q) | ¬P | ¬Q | (¬P)∨(¬Q)
T | T | T | F | F | F | F
T | F | F | T | F | T | T
F | T | F | T | T | F | T
F | F | F | T | T | T | T

The columns for ¬(P∧Q) and (¬P)∨(¬Q) are identical, so they're logically equivalent.

More useful equivalences:

Property — Common Logical Equivalences
  • Commutative: \(P \land Q \equiv Q \land P\), \(P \lor Q \equiv Q \lor P\)
  • Associative: \((P \land Q) \land R \equiv P \land (Q \land R)\)
  • Distributive: \(P \land (Q \lor R) \equiv (P \land Q) \lor (P \land R)\)
  • Implication as OR: \(P \rightarrow Q \equiv \neg P \lor Q\)
  • Contrapositive: \(P \rightarrow Q \equiv \neg Q \rightarrow \neg P\)
Section A.4

Predicates and Quantifiers

A predicate is a function that returns true or false depending on an input.

Definition — Predicate

A predicate \(P(x)\) is a statement involving a variable \(x\) whose truth depends on the value of \(x\).

Examples:

  • \(P(x) = \text{"} x > 0 \text{"}\) is true for positive \(x\), false otherwise
  • \(Q(x, y) = \text{"} x + y = 10 \text{"}\) is a two-variable predicate

Predicates become propositions through quantifiers:

Universal quantifier: \(\forall\)

"\(\forall x, P(x)\)" means "for all \(x\), \(P(x)\) is true."

Definition — Universal Quantifier

\(\forall x \in S, P(x)\) is true if \(P(x)\) is true for every element \(x\) in the set \(S\).

Example: \(\forall x \in \mathbb{R}, x^2 \geq 0\) is true.

Existential quantifier: \(\exists\)

"\(\exists x, P(x)\)" means "there exists an \(x\) such that \(P(x)\) is true."

Definition — Existential Quantifier

\(\exists x \in S, P(x)\) is true if there is at least one element \(x\) in \(S\) for which \(P(x)\) is true.

Example: \(\exists x \in \mathbb{R}, x^2 = 2\) is true (because \(\sqrt{2}\) exists).

Negating quantified statements

Negation swaps quantifiers:

Theorem — Negation of Quantifiers

\(\neg(\forall x, P(x)) \equiv \exists x, \neg P(x)\)

\(\neg(\exists x, P(x)) \equiv \forall x, \neg P(x)\)

Nested quantifiers

Order matters when quantifiers nest:

Example — Nested Quantifiers

\(\forall x, \exists y, P(x, y)\): "For every \(x\), there exists a \(y\) such that..." (y can depend on x)

\(\exists y, \forall x, P(x, y)\): "There exists a \(y\) such that for every \(x\), ..." (y is fixed for all x)

These are NOT equivalent. The first is usually easier to satisfy.

Implementation

Predicates and quantifiers in NumPy

import numpy as np

# Universal quantifier: forall x, x > 0
data = np.array([1, 2, 3, 4])
forall_positive = np.all(data > 0)  # True

# Existential quantifier: exists x such that x < 0
exists_negative = np.any(data < 0)  # False

# Predicate on arrays: P(x) = x % 2 == 0 (x is even)
is_even = (data % 2) == 0  # [False, True, False, True]

# Combining: forall x, (x > 0) AND (x % 2 == 0)
forall_positive_even = np.all((data > 0) & ((data % 2) == 0))  # False (odd numbers present)
Section A.5

Logic in Machine Learning

Why This Matters for ML

Logic is everywhere in AI

  • Boolean masks: Selecting data with conditions like "forall rows where age > 18"
  • Decision boundaries: A classifier learns predicates like "if x > threshold, predict class 1"
  • Logic gates in neural networks: AND, OR, NOT can be learned by small networks
  • Formal verification: Proving properties of ML systems using logical reasoning
  • Causal reasoning: Understanding "if we change X, then Y will happen" (logical implication)
  • Constraints: Many optimization problems are "maximize loss subject to constraint P(x)"

Example: Learning an AND gate

Can a neural network learn the AND function? Let's see:

Implementation

Training a network to learn logical AND

import numpy as np

# Training data: AND function
X = np.array([[0, 0], [0, 1], [1, 0], [1, 1]])
y = np.array([0, 0, 0, 1])  # AND truth table

# Simple single-neuron network: y = sigmoid(w1*x1 + w2*x2 + b)
# We want: w1 = w2 = 2, b = -3 (approximately)
w1, w2, b = 2.0, 2.0, -3.0

def sigmoid(z):
    return 1 / (1 + np.exp(-np.clip(z, -500, 500)))

# Forward pass
z = w1 * X[:, 0] + w2 * X[:, 1] + b
y_pred = sigmoid(z)

print("Input | Predicted | Target")
for i in range(len(X)):
    print(f"{X[i]} | {y_pred[i]:.3f} | {y[i]}")

Boolean masks and selection

In data analysis, logical predicates become boolean masks:

Implementation

Boolean indexing with logical predicates

import pandas as pd
import numpy as np

# Dataset
data = pd.DataFrame({
    'age': [25, 30, 45, 22, 58],
    'income': [50000, 80000, 120000, 35000, 150000],
    'employed': [True, True, True, False, True]
})

# Predicate P(x) = age > 25
mask_age = data['age'] > 25

# Predicate Q(x) = income > 60000
mask_income = data['income'] > 60000

# Conjunction: forall rows where P(x) AND Q(x)
result = data[mask_age & mask_income]
print(result)
# Rows 1, 2, 4 match (age > 25 AND income > 60000)

# Disjunction: rows where P(x) OR Q(x)
result_or = data[mask_age | mask_income]

# Negation: rows where NOT employed
result_not = data[~data['employed']]

Thinking in Logic

You're designing a recommendation system that filters items based on user preferences. Your system must decide: "Show this item if (category matches interest) AND (price is under budget) OR (is highly rated)." A user complains that they're seeing items they don't want. You suspect a logical error in how your OR is interpreted.
1
Write out the condition above in logical notation. What does "(A AND B) OR C" mean? Is it the same as "A AND (B OR C)"?
Using A = category match, B = under budget, C = highly rated:

(A ∧ B) ∨ C means: either (category AND price) or rating
A ∧ (B ∨ C) means: category AND (price OR rating)

These are different! The first shows items if they're highly rated (even if wrong category or expensive). The second requires category match AND at least one price/rating condition.
2
A user says: "I want items that are NOT (cheap AND in my category)." Using De Morgan's laws, rewrite this as an OR statement.
¬(A ∧ B) ≡ (¬A) ∨ (¬B)

So "NOT (cheap AND category)" becomes:
"(NOT cheap) OR (NOT category)"
= "expensive OR not-in-my-category"

This means: exclude items that are both cheap and in my category. Items that are expensive are OK, items outside my category are OK.
3
Your system learns a decision boundary: "If user_rating > 4.5, show the item." Is this the same as "If the item is recommended AND rating > 4.5, show it"? Why or why not?
No, they're different:

P → Q: "If rating > 4.5, then show"
This is equivalent to: "NOT(rating > 4.5) OR show" = "rating ≤ 4.5 OR show"
It doesn't require being recommended first.

R ∧ P → Q: "If recommended AND rating > 4.5, then show"
This is more restrictive—both conditions must hold.

Confusing these is a common bug in conditional logic!
4
You want to ensure: "For every user, there exists a recommendation they like." Write this in logical notation. Now write the negation: what would it mean for this guarantee to fail?
∀ user, ∃ recommendation, Likes(user, rec)

Negation: ¬(∀ user, ∃ rec, Likes(user, rec))
= ∃ user, ∀ rec, ¬Likes(user, rec)

Translation: "There exists a user such that for all recommendations, they don't like any of them."

This is your failure case—a user who is unhappy with every recommendation you show them.
