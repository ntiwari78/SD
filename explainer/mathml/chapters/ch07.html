Matrix Factorizations

SVD, spectral decomposition, orthogonal projections, power iteration, and the QR algorithm

Part I — Linear Algebra ~50 min read
Mental Model

Every Matrix Tells a Three-Act Story

Think of any matrix as a three-step transformation: first, rotate the input into a special coordinate system; second, stretch along each axis by a different amount; third, rotate the result into the output space. This is the Singular Value Decomposition (SVD). The stretching amounts — the singular values — reveal how much "energy" the matrix puts into each direction. By keeping only the largest singular values, you get the best low-rank approximation of your data — the mathematical heart of dimensionality reduction, recommender systems, and image compression.

Section 7.1

Spectral Decomposition

Orthogonal Diagonalisation

In Chapter 6 we saw that the Spectral Theorem guarantees every real symmetric matrix \(A \in \mathbb{R}^{n \times n}\) can be diagonalised by an orthogonal matrix. Let's make this concrete.

Theorem 7.1 — Spectral Decomposition

If \(A \in \mathbb{R}^{n \times n}\) is symmetric, there exists an orthogonal matrix \(Q\) (i.e., \(Q^\top Q = I\)) and a diagonal matrix \(\Lambda\) such that

$$A = Q \Lambda Q^\top = \sum_{i=1}^{n} \lambda_i \mathbf{q}_i \mathbf{q}_i^\top$$

where the columns \(\mathbf{q}_i\) of \(Q\) are orthonormal eigenvectors and the diagonal entries of \(\Lambda\) are the corresponding eigenvalues. The outer-product form shows that \(A\) is a sum of rank-1 matrices, each weighted by an eigenvalue.

The outer-product expansion is powerful because it decomposes the matrix into layers of decreasing importance (ordered by \(|\lambda_i|\)). This is the foundation of Principal Component Analysis. 

Python
import numpy as np

# Symmetric matrix (covariance-like)
np.random.seed(42)
M = np.random.randn(4, 4)
A = M.T @ M  # symmetric positive semi-definite

eigenvalues, Q = np.linalg.eigh(A)

# Sort in descending order (eigh returns ascending)
idx = np.argsort(eigenvalues)[::-1]
eigenvalues = eigenvalues[idx]
Q = Q[:, idx]

print("Eigenvalues (descending):", np.round(eigenvalues, 4))
print()

# Verify spectral decomposition via outer products
A_reconstructed = np.zeros_like(A)
for i in range(len(eigenvalues)):
    q = Q[:, i:i+1]               # column vector (4×1)
    A_reconstructed += eigenvalues[i] * (q @ q.T)

print("Max reconstruction error:", np.max(np.abs(A - A_reconstructed)))
print()

# Show how much each component contributes
total = np.sum(np.abs(eigenvalues))
for i, lam in enumerate(eigenvalues):
    pct = abs(lam) / total * 100
    print(f"Component {i+1}: λ = {lam:8.4f}  ({pct:5.1f}% of total)")
Key Insight

The spectral decomposition only works for symmetric (or Hermitian) matrices. For arbitrary rectangular or non-symmetric matrices, we need the more general Singular Value Decomposition.

Section 7.2

Singular Value Decomposition (SVD)

The SVD is perhaps the single most useful matrix factorization in applied mathematics and ML. It generalises the spectral decomposition to any matrix — not just square, not just symmetric.

Theorem 7.2 — Singular Value Decomposition

Every matrix \(A \in \mathbb{R}^{m \times n}\) admits a factorization

$$A = U \Sigma V^\top$$

where \(U \in \mathbb{R}^{m \times m}\) is orthogonal (left singular vectors), \(V \in \mathbb{R}^{n \times n}\) is orthogonal (right singular vectors), and \(\Sigma \in \mathbb{R}^{m \times n}\) is a "diagonal" matrix with non-negative entries \(\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_{\min(m,n)} \geq 0\) on the main diagonal and zeros elsewhere. The values \(\sigma_i\) are the singular values.

Proof sketch. The matrix \(A^\top A \in \mathbb{R}^{n \times n}\) is symmetric and positive semi-definite. By the Spectral Theorem, there exists an orthogonal \(V\) such that \(A^\top A = V D V^\top\) with \(D = \text{diag}(d_1, \ldots, d_n)\), \(d_i \geq 0\).

Define \(\sigma_i = \sqrt{d_i}\) and \(\Sigma\) as the \(m \times n\) matrix with \(\sigma_i\) on the diagonal. For each \(\sigma_i > 0\), define \(\mathbf{u}_i = \frac{1}{\sigma_i} A \mathbf{v}_i\). One can verify that the \(\mathbf{u}_i\) are orthonormal: \(\mathbf{u}_i^\top \mathbf{u}_j = \frac{1}{\sigma_i \sigma_j} \mathbf{v}_i^\top A^\top A \mathbf{v}_j = \frac{d_j}{\sigma_i \sigma_j} \delta_{ij} = \delta_{ij}\).

Extend \(\{\mathbf{u}_i\}\) to a full orthonormal basis of \(\mathbb{R}^m\) to get \(U\). Then \(A V = U \Sigma\), so \(A = U \Sigma V^\top\). \(\blacksquare\)

Geometric Interpretation

The SVD tells us that every linear map is a composition of three operations:

Step 1 (\(V^\top\)): Rotate the input space so the axes align with the "natural directions" of the transformation.

Step 2 (\(\Sigma\)): Stretch along each axis by the corresponding singular value.

Step 3 (\(U\)): Rotate the result into the output space.

The image of the unit sphere \(\|\mathbf{x}\| = 1\) under \(A\) is an ellipsoid whose semi-axes have lengths \(\sigma_1, \sigma_2, \ldots\) and directions \(\mathbf{u}_1, \mathbf{u}_2, \ldots\).

Definition 7.1 — Compact and Truncated SVD
Reduced Forms of the SVD

If \(A\) has rank \(r\), then only \(r\) singular values are non-zero. The compact SVD keeps only these:

$$A = U_r \Sigma_r V_r^\top$$

where \(U_r \in \mathbb{R}^{m \times r}\), \(\Sigma_r \in \mathbb{R}^{r \times r}\), \(V_r \in \mathbb{R}^{n \times r}\). The truncated SVD (rank-\(k\) approximation) keeps only the top \(k \leq r\) singular values.

Computing the SVD in Practice

Python
import numpy as np

A = np.array([[3, 2, 2],
              [2, 3, -2]])

# Full SVD
U, sigma, Vt = np.linalg.svd(A, full_matrices=True)

print("U (left singular vectors):")
print(np.round(U, 4))
print("\nSingular values:", np.round(sigma, 4))
print("\nV^T (right singular vectors):")
print(np.round(Vt, 4))

# Reconstruct A from SVD
Sigma_full = np.zeros_like(A, dtype=float)
np.fill_diagonal(Sigma_full, sigma)
A_reconstructed = U @ Sigma_full @ Vt
print("\nReconstruction error:", np.max(np.abs(A - A_reconstructed)))

# Compact SVD (full_matrices=False)
U_c, sigma_c, Vt_c = np.linalg.svd(A, full_matrices=False)
print(f"\nFull U shape: {U.shape}, Compact U shape: {U_c.shape}")
print(f"Full Vt shape: {Vt.shape}, Compact Vt shape: {Vt_c.shape}")

The relationship between the SVD and the eigendecomposition is fundamental: the singular values of \(A\) are the square roots of the eigenvalues of \(A^\top A\) (or equivalently \(A A^\top\)), and the right singular vectors are the eigenvectors of \(A^\top A\).

Proposition 7.1 — SVD–Eigenvalue Connection

If \(A = U\Sigma V^\top\), then:

  • \(A^\top A = V \Sigma^\top \Sigma V^\top\) — eigenvalues are \(\sigma_i^2\), eigenvectors are columns of \(V\).
  • \(A A^\top = U \Sigma \Sigma^\top U^\top\) — eigenvalues are \(\sigma_i^2\), eigenvectors are columns of \(U\).

The Truncated SVD and Low-Rank Approximation

One of the most remarkable facts in all of matrix theory is that the best rank-\(k\) approximation to a matrix (in both Frobenius and spectral norm) is obtained simply by truncating its SVD.

Theorem 7.3 — Eckart–Young–Mirsky

Let \(A = U \Sigma V^\top\) with singular values \(\sigma_1 \geq \cdots \geq \sigma_r > 0\). For any \(k \leq r\), define the rank-\(k\) truncated SVD:

$$A_k = \sum_{i=1}^{k} \sigma_i \mathbf{u}_i \mathbf{v}_i^\top.$$

Then \(A_k\) is the closest rank-\(k\) matrix to \(A\) in both the Frobenius norm and the spectral (operator) norm:

$$\|A - A_k\|_F = \sqrt{\sigma_{k+1}^2 + \cdots + \sigma_r^2}, \qquad \|A - A_k\|_2 = \sigma_{k+1}.$$

This theorem is the mathematical backbone of recommender systems (Netflix Prize), latent semantic analysis, and image compression. 

Python
import numpy as np

np.random.seed(0)

# Create a matrix with known rank structure + noise
true_rank = 3
m, n = 50, 30
U_true = np.linalg.qr(np.random.randn(m, true_rank))[0]
V_true = np.linalg.qr(np.random.randn(n, true_rank))[0]
sigma_true = np.array([100, 30, 10])
A_clean = U_true @ np.diag(sigma_true) @ V_true.T
A = A_clean + 0.5 * np.random.randn(m, n)  # add noise

U, sigma, Vt = np.linalg.svd(A, full_matrices=False)

print("Top 8 singular values:", np.round(sigma[:8], 2))
print()

# Truncated approximations
for k in [1, 2, 3, 5, 10]:
    A_k = U[:, :k] @ np.diag(sigma[:k]) @ Vt[:k, :]
    error = np.linalg.norm(A - A_k, 'fro')
    total = np.linalg.norm(A, 'fro')
    compression = (m * k + k + n * k) / (m * n) * 100
    print(f"Rank-{k:2d} | Error: {error:8.2f} ({error/total*100:5.1f}%) | "
          f"Storage: {compression:5.1f}% of original")
Performance Deep-Dive

Full SVD vs Randomized SVD

The full SVD costs \(O(\min(m,n)^2 \cdot \max(m,n))\) — prohibitive for the million-dimensional matrices in recommender systems. Randomized SVD (Halko, Martinsson, Tropp, 2011) computes only the top \(k\) singular triplets in \(O(mn \cdot k)\) time using random projections. 

Python
import numpy as np
from scipy.linalg import svd as scipy_svd
import time

def randomized_svd(A, k, n_oversamples=10, n_iter=2):
    """Randomized SVD via Halko-Martinsson-Tropp algorithm."""
    m, n = A.shape
    # Step 1: Random projection to capture column space
    Omega = np.random.randn(n, k + n_oversamples)
    Y = A @ Omega
    # Power iteration for better approximation
    for _ in range(n_iter):
        Y = A @ (A.T @ Y)
    Q, _ = np.linalg.qr(Y)
    # Step 2: Project A into low-dimensional space
    B = Q.T @ A              # (k+p) × n — small!
    Uhat, sigma, Vt = np.linalg.svd(B, full_matrices=False)
    U = Q @ Uhat
    return U[:, :k], sigma[:k], Vt[:k, :]

# Benchmark
for shape in [(1000, 500), (5000, 2000)]:
    m, n = shape
    A = np.random.randn(m, n)
    k = 20

    t0 = time.perf_counter()
    np.linalg.svd(A, full_matrices=False)
    t_full = time.perf_counter() - t0

    t0 = time.perf_counter()
    U_r, s_r, Vt_r = randomized_svd(A, k)
    t_rand = time.perf_counter() - t0

    print(f"{m}×{n}  full SVD: {t_full:.3f}s  "
          f"randomized (k={k}): {t_rand:.3f}s  "
          f"speedup: {t_full/t_rand:.1f}×")
Section 7.3

Orthogonal Projections

Projections are linear maps that "collapse" a space onto a subspace. They appear everywhere: least-squares regression projects the observation vector onto the column space of the design matrix; PCA projects data onto the principal subspace.

Definition 7.2 — Orthogonal Projection
Projection onto a Subspace

Let \(W\) be a subspace of \(\mathbb{R}^n\) with orthonormal basis \(\{\mathbf{q}_1, \ldots, \mathbf{q}_k\}\), collected into \(Q \in \mathbb{R}^{n \times k}\). The orthogonal projection onto \(W\) is the matrix

$$P_W = QQ^\top.$$

For any \(\mathbf{x} \in \mathbb{R}^n\), the projection \(P_W \mathbf{x}\) is the closest point in \(W\) to \(\mathbf{x}\):

$$P_W \mathbf{x} = \arg\min_{\mathbf{w} \in W} \|\mathbf{x} - \mathbf{w}\|.$$

Proposition 7.2 — Properties of Projection Matrices

A matrix \(P\) is an orthogonal projection if and only if:

  • Idempotent: \(P^2 = P\) (projecting twice is the same as projecting once).
  • Symmetric: \(P = P^\top\).

Its eigenvalues are all 0 or 1, and \(\text{tr}(P) = \text{rank}(P) = \dim(W)\).

When the basis vectors are not orthonormal, the projection formula involves the pseudoinverse:

$$P_W = X(X^\top X)^{-1} X^\top$$

where the columns of \(X\) span \(W\). This is exactly the "hat matrix" from linear regression: the predicted values are \(\hat{\mathbf{y}} = P_W \mathbf{y}\). 

Python
import numpy as np

# Orthogonal projection via QR
np.random.seed(7)
X = np.random.randn(5, 2)          # 5D space, 2D subspace

# Method 1: QR-based (numerically stable)
Q, R = np.linalg.qr(X)
P_qr = Q @ Q.T

# Method 2: Normal equations (less stable)
P_ne = X @ np.linalg.inv(X.T @ X) @ X.T

print("P via QR == P via normal equations?",
      np.allclose(P_qr, P_ne))

# Verify projection properties
print(f"Idempotent (P² = P)?  {np.allclose(P_qr @ P_qr, P_qr)}")
print(f"Symmetric  (P = Pᵀ)? {np.allclose(P_qr, P_qr.T)}")
print(f"trace(P) = {np.trace(P_qr):.1f}  (should be {X.shape[1]})")
print(f"rank(P)  = {np.linalg.matrix_rank(P_qr)}")
print()

# Project a point
x = np.random.randn(5)
x_proj = P_qr @ x
residual = x - x_proj
print(f"||residual|| = {np.linalg.norm(residual):.4f}")
print(f"residual ⊥ subspace? {np.allclose(Q.T @ residual, 0)}")
SVD Connection

The SVD provides the optimal projection in the Frobenius-norm sense. If you want the best rank-\(k\) subspace to project your data onto (minimising reconstruction error), the answer is the subspace spanned by the top \(k\) left singular vectors — which are the same as the top \(k\) eigenvectors of the covariance matrix. This is PCA.

Section 7.4

Algorithms: Power Iteration and the QR Algorithm

How does NumPy actually compute eigenvalues and singular values? Two foundational algorithms underpin modern numerical linear algebra.

Power Iteration

Power iteration is the simplest algorithm for finding the dominant eigenvalue (the one with the largest absolute value) and its eigenvector. The idea is beautifully simple: repeatedly multiply by the matrix and normalise.

Algorithm 7.1 — Power Iteration
Finding the Dominant Eigenpair

Given \(A \in \mathbb{R}^{n \times n}\) and a random starting vector \(\mathbf{b}_0\):

$$\mathbf{b}_{k+1} = \frac{A \mathbf{b}_k}{\|A \mathbf{b}_k\|}$$

Under mild conditions (dominant eigenvalue is unique in magnitude, \(\mathbf{b}_0\) is not orthogonal to the dominant eigenvector), the sequence converges to the dominant eigenvector. The corresponding eigenvalue is approximated by the Rayleigh quotient: \(\lambda \approx \mathbf{b}_k^\top A \mathbf{b}_k\).

The convergence rate is \(O(|\lambda_2/\lambda_1|^k)\) — it's fast when the dominant eigenvalue is well-separated, but slow when two eigenvalues are close in magnitude. 

Python
import numpy as np

def power_iteration(A, max_iter=100, tol=1e-10):
    """Find the dominant eigenvalue and eigenvector."""
    n = A.shape[0]
    b = np.random.randn(n)
    b /= np.linalg.norm(b)

    history = []
    for k in range(max_iter):
        Ab = A @ b
        b_new = Ab / np.linalg.norm(Ab)

        # Rayleigh quotient for eigenvalue estimate
        lam = b_new @ A @ b_new
        history.append(lam)

        # Check convergence
        if np.linalg.norm(b_new - b) < tol:
            print(f"Converged in {k+1} iterations")
            return lam, b_new, history

        b = b_new

    print(f"Did not converge in {max_iter} iterations")
    return lam, b_new, history

# Example
np.random.seed(42)
M = np.random.randn(5, 5)
A = M.T @ M  # symmetric PD — dominant eigenvalue is well-separated

lam_power, v_power, hist = power_iteration(A)
lam_true = np.max(np.linalg.eigvalsh(A))

print(f"Power iteration: λ₁ = {lam_power:.10f}")
print(f"numpy.linalg:    λ₁ = {lam_true:.10f}")
print(f"Relative error:      {abs(lam_power - lam_true)/lam_true:.2e}")

Power iteration also forms the basis for computing the top singular values: applying power iteration to \(A^\top A\) gives the top right singular vector, and the top singular value is the square root of the dominant eigenvalue of \(A^\top A\). Google's original PageRank algorithm was essentially power iteration on the web's link matrix.

The QR Algorithm

The QR algorithm is the workhorse behind np.linalg.eig and the gold standard for computing all eigenvalues of a dense matrix. It works by repeatedly computing QR factorizations.

Algorithm 7.2 — Basic QR Algorithm
Computing All Eigenvalues

Starting from \(A_0 = A\), repeat:

$$A_k = Q_k R_k \qquad \text{(QR factorization)}$$

$$A_{k+1} = R_k Q_k \qquad \text{(reverse multiply)}$$

Under suitable conditions, \(A_k\) converges to an upper triangular matrix (Schur form) whose diagonal entries are the eigenvalues. For symmetric matrices, \(A_k\) converges to a diagonal matrix.

The key insight is that \(A_{k+1} = Q_k^\top A_k Q_k\), so all iterates are similar to \(A\) and share the same eigenvalues. Each step "pushes" off-diagonal entries toward zero. 

Python
import numpy as np

def basic_qr_algorithm(A, max_iter=200, tol=1e-12):
    """Basic QR algorithm for eigenvalues of a symmetric matrix."""
    Ak = A.copy().astype(float)
    n = A.shape[0]

    for k in range(max_iter):
        Q, R = np.linalg.qr(Ak)
        Ak = R @ Q

        # Check convergence: off-diagonal entries should vanish
        off_diag = np.sum(np.abs(Ak)) - np.sum(np.abs(np.diag(Ak)))
        if off_diag < tol:
            print(f"Converged in {k+1} iterations")
            return np.sort(np.diag(Ak))[::-1]

    print(f"Warning: did not fully converge (off-diag = {off_diag:.2e})")
    return np.sort(np.diag(Ak))[::-1]

# Symmetric matrix
np.random.seed(42)
M = np.random.randn(5, 5)
A = M.T @ M

eigs_qr = basic_qr_algorithm(A)
eigs_np = np.sort(np.linalg.eigvalsh(A))[::-1]

print("\nQR algorithm eigenvalues:", np.round(eigs_qr, 6))
print("NumPy eigenvalues:       ", np.round(eigs_np, 6))
print("Max difference:          ", np.max(np.abs(eigs_qr - eigs_np)))
Performance Deep-Dive

Why Production QR Is Much Faster

Our basic QR algorithm is \(O(n^3)\) per iteration. The production version used by LAPACK (and thus NumPy) applies three critical optimizations: (1) Hessenberg reduction — a one-time \(O(n^3)\) preprocessing step that makes each subsequent QR factorization only \(O(n^2)\); (2) Wilkinson shifts — the eigenvalue of the trailing \(2 \times 2\) block is used as a shift, giving cubic convergence; (3) Deflation — once an eigenvalue converges, the matrix size is reduced. The result is an algorithm that typically converges in \(O(n)\) iterations, giving \(O(n^3)\) total cost.

The QR Factorization Itself

Don't confuse the QR algorithm (an iterative eigenvalue algorithm) with the QR factorization (decomposing \(A = QR\) into an orthogonal matrix \(Q\) and an upper triangular matrix \(R\)). The QR factorization is a subroutine used inside the QR algorithm, but it also stands alone as a tool for solving least-squares problems: the normal equations \(X^\top X \boldsymbol{\beta} = X^\top \mathbf{y}\) are equivalent to \(R \boldsymbol{\beta} = Q^\top \mathbf{y}\), which is just back-substitution.

Section 7.5

Summary

This chapter covered the major matrix factorizations that power modern machine learning. The spectral decomposition breaks symmetric matrices into orthogonal eigenvector directions scaled by eigenvalues. The SVD generalises this to any matrix and provides optimal low-rank approximations via the Eckart–Young–Mirsky theorem. Orthogonal projections connect these factorizations to least-squares regression and PCA. Under the hood, power iteration and the QR algorithm are the numerical workhorses that make these decompositions computable at scale.

Why This Matters for AI

Factorizations Are the Engines of ML

  • PCA — Computes the eigendecomposition (or SVD) of the covariance matrix to find the principal directions of variation. The truncated SVD is the default implementation in scikit-learn for large datasets.
  • Recommender systems — Matrix factorization (a close cousin of SVD) was the winning approach in the Netflix Prize. The user-item matrix is approximately decomposed into low-rank factors representing latent user preferences and item attributes.
  • Latent Semantic Analysis — Truncated SVD of the term-document matrix reveals latent "topics" that connect related words and documents — an early predecessor to modern embeddings.
  • Pseudoinverse and least-squares — The Moore–Penrose pseudoinverse \(A^+ = V \Sigma^+ U^\top\) (computed from the SVD) provides the minimum-norm least-squares solution, which underpins every linear model.
  • Data whitening — The spectral decomposition of the covariance matrix is used to decorrelate and normalise features, a critical preprocessing step for many algorithms.
  • Graph neural networks — Spectral GNNs operate in the eigenspace of the graph Laplacian, using its eigenvectors as a Fourier basis for signals on graphs.

Socratic Knowledge Check

You are building a movie recommendation engine. You have a 10,000-user × 5,000-movie rating matrix (mostly sparse). You plan to use matrix factorization to predict missing ratings. Think through these questions.
1
You compute the SVD of the rating matrix and find that the top 20 singular values capture 85% of the Frobenius norm, while the remaining 4,980 capture only 15%. What does this tell you about the rating data, and what rank would you choose for your factorization?
This tells us the rating matrix is approximately low-rank — most of the "signal" lives in a 20-dimensional subspace. Users' preferences and movies' characteristics can be well-described by roughly 20 latent factors (e.g., genre preference, tone, era). By the Eckart–Young theorem, a rank-20 approximation is optimal and captures 85% of the data's energy. In practice you'd try ranks 10–50 and cross-validate on held-out ratings to find the best bias–variance trade-off.
2
Your rating matrix is 10,000 × 5,000. Computing the full SVD takes too long. What algorithmic approach would you use instead, and why?
Use a randomized SVD (Halko–Martinsson–Tropp). It computes only the top \(k\) singular triplets in \(O(mnk)\) time rather than \(O(mn \cdot \min(m,n))\) for the full SVD. For \(k = 20\), this is roughly 250× faster. Scikit-learn's TruncatedSVD uses this approach internally. Alternatively, for very sparse matrices, you could use iterative Krylov-based methods (like ARPACK via scipy.sparse.linalg.svds) that exploit sparsity.
3
You project your 5,000-dimensional movie feature vectors onto the top 50 principal components. A colleague asks: "How do we know the projection is optimal?" What theorem would you cite, and what does "optimal" mean here?
The Eckart–Young–Mirsky theorem guarantees that the rank-\(k\) truncated SVD minimises the Frobenius-norm reconstruction error over all rank-\(k\) matrices. "Optimal" means: among all possible 50-dimensional linear projections, PCA (equivalently, the truncated SVD of the centered data) minimises \(\sum_i \|\mathbf{x}_i - \hat{\mathbf{x}}_i\|^2\), the total squared reconstruction error. No other 50-dimensional subspace can do better.
4
Your projection matrix \(P\) for a 50-dimensional subspace in a 5,000-dimensional space has shape 5000 × 5000. How many independent parameters does it actually have, and why should you never store it as a dense matrix?
Although \(P\) is 5000 × 5000 (25 million entries), it has rank 50, so it can be represented as \(P = QQ^\top\) where \(Q\) is 5000 × 50 (only 250,000 entries). The number of independent parameters in an orthogonal projection onto a \(k\)-dimensional subspace of \(\mathbb{R}^n\) is \(k(n - k)\) — the dimension of the Grassmannian. Storing and multiplying with the dense 5000 × 5000 matrix would use 100× more memory and compute than working with the factored form \(Q\).
5
You use power iteration to find the top singular vector of the rating matrix for an online recommendation system. After deployment, you notice convergence is very slow. What could cause this, and how would you fix it?
Slow convergence in power iteration means \(|\sigma_2/\sigma_1| \approx 1\) — the top two singular values are very close in magnitude. This makes the ratio \(|\sigma_2/\sigma_1|^k\) decay slowly. Fixes include: (1) use inverse iteration with shift if you have a good eigenvalue estimate, giving convergence rate \(|(\sigma_1 - \mu)/(\sigma_2 - \mu)|\); (2) use a block (subspace) iteration that tracks multiple vectors simultaneously; (3) switch to an implicitly restarted Lanczos/Arnoldi method (what ARPACK uses), which has much better convergence for clustered singular values.
