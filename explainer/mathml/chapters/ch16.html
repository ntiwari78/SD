Derivatives and Gradients

Partial and total derivatives, the gradient, directional derivatives, Jacobians, and Hessian matrices.

50 min read Part III · Multivariable Calculus
Mental Model

Slopes in Every Direction

On a mountain, the steepness depends on which direction you walk. Head north and you might climb steeply; head east and the terrain might be flat. A partial derivative measures the slope when you walk along one coordinate axis only. The gradient assembles all these single-axis slopes into a vector that points in the direction of steepest ascent. When a neural network computes gradients during backpropagation, it is answering: "for each of the billions of weight dials, how does the loss change if I turn just that one dial?" The gradient packs all those answers into one giant vector — and then gradient descent takes a step in the opposite direction.

Section 16.1

Partial Derivatives

Definition — Partial Derivative

Let \(f : \mathbb{R}^n \to \mathbb{R}\). The partial derivative of \(f\) with respect to \(x_i\) at \(\mathbf{a}\) is

\[\frac{\partial f}{\partial x_i}(\mathbf{a}) = \lim_{h \to 0} \frac{f(\mathbf{a} + h\mathbf{e}_i) - f(\mathbf{a})}{h},\]

where \(\mathbf{e}_i\) is the \(i\)-th standard basis vector. We hold all variables except \(x_i\) fixed and differentiate with respect to \(x_i\) alone.

Computing partial derivatives is straightforward: treat every variable except \(x_i\) as a constant, then differentiate as in single-variable calculus. The notation \(\frac{\partial f}{\partial x_i}\), \(f_{x_i}\), \(\partial_i f\), and \(D_i f\) are all equivalent.

Example

Let \(f(x_1, x_2, x_3) = x_1^2 x_2 + \sin(x_3)\). Then: \(\frac{\partial f}{\partial x_1} = 2x_1 x_2\), \(\frac{\partial f}{\partial x_2} = x_1^2\), \(\frac{\partial f}{\partial x_3} = \cos(x_3)\). Each partial derivative is itself a function of \(\mathbf{x}\).

A critical subtlety: the existence of all partial derivatives does not guarantee continuity or differentiability (in the total sense). Consider \(f(x, y) = \frac{xy}{x^2 + y^2}\) at the origin (with \(f(0,0) = 0\)). Both \(\frac{\partial f}{\partial x}(0,0)\) and \(\frac{\partial f}{\partial y}(0,0)\) exist and equal 0, but \(f\) is not even continuous at \((0,0)\) (as shown in Chapter 11). Total differentiability requires something stronger.

Python
import numpy as np

def partial_derivative(f, x, i, h=1e-7):
    """Compute ∂f/∂xᵢ at x using central differences."""
    x_plus = x.copy()
    x_minus = x.copy()
    x_plus[i] += h
    x_minus[i] -= h
    return (f(x_plus) - f(x_minus)) / (2 * h)

def numerical_gradient(f, x, h=1e-7):
    """Compute full gradient ∇f(x) via central differences."""
    grad = np.zeros_like(x)
    for i in range(len(x)):
        grad[i] = partial_derivative(f, x, i, h)
    return grad

# f(x₁, x₂, x₃) = x₁²x₂ + sin(x₃)
f = lambda x: x[0]**2 * x[1] + np.sin(x[2])
x = np.array([2.0, 3.0, np.pi/4])

grad = numerical_gradient(f, x)
# Analytical: [2*x₁*x₂, x₁², cos(x₃)] = [12, 4, √2/2]
analytical = np.array([2*x[0]*x[1], x[0]**2, np.cos(x[2])])

print(f"Numerical:  {grad}")
print(f"Analytical: {analytical}")
print(f"Max error:  {np.max(np.abs(grad - analytical)):.2e}")
Section 16.2

The Gradient

Definition — Gradient

The gradient of a scalar-valued function \(f : \mathbb{R}^n \to \mathbb{R}\) at \(\mathbf{a}\) is the vector of all partial derivatives:

\[\nabla f(\mathbf{a}) = \begin{pmatrix} \frac{\partial f}{\partial x_1}(\mathbf{a}) \\ \vdots \\ \frac{\partial f}{\partial x_n}(\mathbf{a}) \end{pmatrix} \in \mathbb{R}^n.\]

The gradient lives in the same space as the input \(\mathbf{x}\). For a loss function \(L : \mathbb{R}^d \to \mathbb{R}\), the gradient \(\nabla L(\boldsymbol\theta)\) is a vector in \(\mathbb{R}^d\) — one entry per parameter — and gradient descent updates \(\boldsymbol\theta_{k+1} = \boldsymbol\theta_k - \eta \nabla L(\boldsymbol\theta_k)\).

Theorem — Gradient Points in the Steepest Ascent Direction

If \(\nabla f(\mathbf{a}) \neq \mathbf{0}\), then among all unit vectors \(\mathbf{u}\), the directional derivative \(D_{\mathbf{u}} f(\mathbf{a})\) is maximised when \(\mathbf{u} = \frac{\nabla f(\mathbf{a})}{\|\nabla f(\mathbf{a})\|}\). The maximum rate of increase is \(\|\nabla f(\mathbf{a})\|\).

The directional derivative is \(D_{\mathbf{u}}f(\mathbf{a}) = \nabla f(\mathbf{a}) \cdot \mathbf{u} = \|\nabla f\|\|\mathbf{u}\|\cos\theta = \|\nabla f\|\cos\theta\), where \(\theta\) is the angle between \(\nabla f\) and \(\mathbf{u}\). This is maximised when \(\cos\theta = 1\), i.e., \(\theta = 0\), meaning \(\mathbf{u}\) points in the same direction as \(\nabla f\). The maximum value is \(\|\nabla f\|\). \(\blacksquare\)

This is the mathematical reason gradient descent works: \(-\nabla f\) is the direction of steepest descent, so stepping that way decreases the function most rapidly (locally).

Proposition — Gradient Is Perpendicular to Level Sets

At any point \(\mathbf{a}\) where \(\nabla f(\mathbf{a}) \neq \mathbf{0}\), the gradient is orthogonal to the level set \(\{f(\mathbf{x}) = f(\mathbf{a})\}\). Geometrically: the gradient points "away from" the contour line, straight towards higher values.

Section 16.3

Directional Derivatives

Definition — Directional Derivative

The directional derivative of \(f\) at \(\mathbf{a}\) in the direction of a unit vector \(\mathbf{u}\) is

\[D_{\mathbf{u}} f(\mathbf{a}) = \lim_{t \to 0} \frac{f(\mathbf{a} + t\mathbf{u}) - f(\mathbf{a})}{t}.\]

If \(f\) is differentiable at \(\mathbf{a}\), then \(D_{\mathbf{u}} f(\mathbf{a}) = \nabla f(\mathbf{a}) \cdot \mathbf{u}\).

Partial derivatives are special cases: \(\frac{\partial f}{\partial x_i} = D_{\mathbf{e}_i} f\). The directional derivative generalises this to any direction. It tells you the rate of change of \(f\) as you move in direction \(\mathbf{u}\).

Python
import numpy as np

def directional_derivative(f, x, u, h=1e-7):
    """Compute D_u f(x) using the central difference along u."""
    u = u / np.linalg.norm(u)  # ensure unit vector
    return (f(x + h*u) - f(x - h*u)) / (2*h)

# f(x,y) = x² + 3y²   (elliptical paraboloid)
f = lambda x: x[0]**2 + 3*x[1]**2
point = np.array([1.0, 1.0])
grad = np.array([2*point[0], 6*point[1]])  # [2, 6]

# Directional derivative along various directions
directions = {
    "gradient":    grad / np.linalg.norm(grad),
    "neg gradient": -grad / np.linalg.norm(grad),
    "tangent":     np.array([-grad[1], grad[0]]) / np.linalg.norm(grad),
}

for name, u in directions.items():
    dd_num = directional_derivative(f, point, u)
    dd_ana = grad @ u
    print(f"D_{name:12s} f = {dd_num:+8.4f} (analytical: {dd_ana:+8.4f})")
# gradient direction: maximum rate of increase
# neg gradient: minimum (most negative) rate
# tangent to level set: rate ≈ 0
Section 16.4

The Jacobian Matrix

The gradient is defined for scalar-valued functions. For a vector-valued function \(\mathbf{f} : \mathbb{R}^n \to \mathbb{R}^m\), we need the Jacobian — the matrix of all partial derivatives.

Definition — Jacobian Matrix

The Jacobian of \(\mathbf{f} = (f_1, \ldots, f_m) : \mathbb{R}^n \to \mathbb{R}^m\) at \(\mathbf{a}\) is the \(m \times n\) matrix

\[J_{\mathbf{f}}(\mathbf{a}) = \begin{pmatrix} \frac{\partial f_1}{\partial x_1} & \cdots & \frac{\partial f_1}{\partial x_n} \\ \vdots & \ddots & \vdots \\ \frac{\partial f_m}{\partial x_1} & \cdots & \frac{\partial f_m}{\partial x_n} \end{pmatrix}.\]

Row \(i\) is the gradient of \(f_i\), transposed. For \(m = 1\), the Jacobian is \(\nabla f^\top\) (a row vector).

The Jacobian is the best linear approximation to \(\mathbf{f}\) near \(\mathbf{a}\):

\[\mathbf{f}(\mathbf{a} + \mathbf{h}) \approx \mathbf{f}(\mathbf{a}) + J_{\mathbf{f}}(\mathbf{a})\,\mathbf{h}.\]

This is the multivariable generalisation of the tangent-line approximation \(f(a + h) \approx f(a) + f'(a)h\). The Jacobian replaces the scalar derivative.

Python
import numpy as np

def numerical_jacobian(f, x, h=1e-7):
    """Compute the Jacobian matrix of f: Rⁿ → Rᵐ at x."""
    f0 = np.atleast_1d(f(x))
    m, n = len(f0), len(x)
    J = np.zeros((m, n))
    for j in range(n):
        x_plus = x.copy();  x_plus[j] += h
        x_minus = x.copy(); x_minus[j] -= h
        J[:, j] = (f(x_plus) - f(x_minus)) / (2 * h)
    return J

# Softmax: R³ → R³
def softmax(z):
    e = np.exp(z - np.max(z))
    return e / e.sum()

z = np.array([2.0, 1.0, 0.5])
J = numerical_jacobian(softmax, z)
print("Jacobian of softmax:")
print(np.array2string(J, precision=4, suppress_small=True))

# Verify: Jacobian of softmax has formula J_ij = s_i(δ_ij - s_j)
s = softmax(z)
J_analytical = np.diag(s) - np.outer(s, s)
print(f"\nMax error vs analytical: {np.max(np.abs(J - J_analytical)):.2e}")
Jacobian Dimensions in Neural Networks

A layer \(\mathbf{f} : \mathbb{R}^n \to \mathbb{R}^m\) has an \(m \times n\) Jacobian. For a linear layer \(\mathbf{f}(\mathbf{x}) = W\mathbf{x} + \mathbf{b}\), the Jacobian is simply \(W\) — the weight matrix. For an element-wise activation \(\sigma(\mathbf{z})\), the Jacobian is diagonal: \(J = \text{diag}(\sigma'(z_1), \ldots, \sigma'(z_n))\). Backpropagation computes vector-Jacobian products \(\mathbf{v}^\top J\) without ever forming the full Jacobian.

Section 16.5

The Multivariable Chain Rule

Theorem — Chain Rule for Vector Functions

If \(\mathbf{g} : \mathbb{R}^n \to \mathbb{R}^k\) is differentiable at \(\mathbf{a}\) and \(\mathbf{f} : \mathbb{R}^k \to \mathbb{R}^m\) is differentiable at \(\mathbf{g}(\mathbf{a})\), then \(\mathbf{f} \circ \mathbf{g}\) is differentiable at \(\mathbf{a}\) with Jacobian

\[J_{\mathbf{f} \circ \mathbf{g}}(\mathbf{a}) = J_{\mathbf{f}}(\mathbf{g}(\mathbf{a})) \cdot J_{\mathbf{g}}(\mathbf{a}).\]

The Jacobian of a composition is the matrix product of the individual Jacobians.

For a scalar loss \(L = \ell(\mathbf{f}(\mathbf{g}(\mathbf{x})))\), the gradient with respect to \(\mathbf{x}\) is:

\[\nabla_{\mathbf{x}} L = J_{\mathbf{g}}^\top \cdot J_{\mathbf{f}}^\top \cdot \nabla \ell.\]

Backpropagation computes this right-to-left: start with \(\nabla \ell\) (a vector), multiply by \(J_{\mathbf{f}}^\top\), then by \(J_{\mathbf{g}}^\top\). Each step is a vector-Jacobian product (VJP), costing \(O(\text{layer size})\) — never forming full Jacobians.

Performance Deep-Dive

Backpropagation as Jacobian Products

Trace the chain rule through a concrete two-layer network.

import numpy as np

def relu(z):
    return np.maximum(z, 0)

def relu_deriv(z):
    return (z > 0).astype(float)

# Forward pass
np.random.seed(42)
x = np.random.randn(4)
W1 = np.random.randn(3, 4) * 0.5
W2 = np.random.randn(2, 3) * 0.5

z1 = W1 @ x            # R⁴ → R³
a1 = relu(z1)           # R³ → R³  (element-wise)
z2 = W2 @ a1            # R³ → R²
loss = 0.5 * np.sum(z2**2)  # R² → R

# Backward pass: chain rule with VJPs
dL_dz2 = z2                             # ∇ₗ: R² → R²
dL_da1 = W2.T @ dL_dz2                  # J_W2ᵀ · dL_dz2
dL_dz1 = dL_da1 * relu_deriv(z1)        # diag(σ') · dL_da1
dL_dW1 = np.outer(dL_dz1, x)            # ∂L/∂W₁ = dL_dz1 · xᵀ
dL_dW2 = np.outer(dL_dz2, a1)           # ∂L/∂W₂ = dL_dz2 · a1ᵀ

print(f"dL/dW1 shape: {dL_dW1.shape} (3×4)")
print(f"dL/dW2 shape: {dL_dW2.shape} (2×3)")

# Verify dL/dW1 numerically
eps = 1e-5
dL_dW1_num = np.zeros_like(W1)
for i in range(W1.shape[0]):
    for j in range(W1.shape[1]):
        W1[i,j] += eps
        loss_p = 0.5*np.sum((W2 @ relu(W1 @ x))**2)
        W1[i,j] -= 2*eps
        loss_m = 0.5*np.sum((W2 @ relu(W1 @ x))**2)
        W1[i,j] += eps
        dL_dW1_num[i,j] = (loss_p - loss_m) / (2*eps)

print(f"Max error: {np.max(np.abs(dL_dW1 - dL_dW1_num)):.2e}")
Section 16.6

The Hessian Matrix

Definition — Hessian

The Hessian of a twice-differentiable function \(f : \mathbb{R}^n \to \mathbb{R}\) at \(\mathbf{a}\) is the \(n \times n\) matrix of second partial derivatives:

\[H_f(\mathbf{a}) = \begin{pmatrix} \frac{\partial^2 f}{\partial x_1^2} & \frac{\partial^2 f}{\partial x_1 \partial x_2} & \cdots & \frac{\partial^2 f}{\partial x_1 \partial x_n} \\ \frac{\partial^2 f}{\partial x_2 \partial x_1} & \frac{\partial^2 f}{\partial x_2^2} & \cdots & \frac{\partial^2 f}{\partial x_2 \partial x_n} \\ \vdots & \vdots & \ddots & \vdots \\ \frac{\partial^2 f}{\partial x_n \partial x_1} & \cdots & \cdots & \frac{\partial^2 f}{\partial x_n^2} \end{pmatrix}.\]

Theorem — Symmetry of the Hessian (Schwarz / Clairaut)

If the second partial derivatives of \(f\) are continuous, then \(\frac{\partial^2 f}{\partial x_i \partial x_j} = \frac{\partial^2 f}{\partial x_j \partial x_i}\) for all \(i, j\). That is, \(H_f\) is symmetric.

The Hessian is the Jacobian of the gradient: \(H_f = J_{\nabla f}\). It captures the curvature of \(f\). The second-order Taylor expansion at \(\mathbf{a}\) is:

\[f(\mathbf{a} + \mathbf{h}) \approx f(\mathbf{a}) + \nabla f(\mathbf{a})^\top \mathbf{h} + \frac{1}{2}\mathbf{h}^\top H_f(\mathbf{a})\,\mathbf{h}.\]

Eigenvalues of the Hessian

Since \(H_f\) is symmetric, it has real eigenvalues (Spectral Theorem, Chapter 6). These eigenvalues determine the curvature in each eigendirection. At a critical point \(\nabla f = \mathbf{0}\):

All eigenvalues \(> 0\) (positive definite) → local minimum.

All eigenvalues \(< 0\) (negative definite) → local maximum.

Mixed signs (indefinite) → saddle point.

In high-dimensional loss landscapes, saddle points vastly outnumber local minima. If a function has \(n\) parameters and each Hessian eigenvalue is equally likely to be positive or negative at a critical point, the probability of a local minimum (all \(n\) eigenvalues positive) is \(2^{-n}\) — exponentially small.

Python
import numpy as np

def numerical_hessian(f, x, h=1e-5):
    """Compute Hessian of f: Rⁿ → R using finite differences."""
    n = len(x)
    H = np.zeros((n, n))
    f0 = f(x)
    for i in range(n):
        for j in range(i, n):
            x_pp = x.copy(); x_pp[i] += h; x_pp[j] += h
            x_pm = x.copy(); x_pm[i] += h; x_pm[j] -= h
            x_mp = x.copy(); x_mp[i] -= h; x_mp[j] += h
            x_mm = x.copy(); x_mm[i] -= h; x_mm[j] -= h
            H[i,j] = (f(x_pp) - f(x_pm) - f(x_mp) + f(x_mm)) / (4*h*h)
            H[j,i] = H[i,j]  # symmetry
    return H

# Saddle point: f(x,y) = x² - y² at origin
f_saddle = lambda x: x[0]**2 - x[1]**2
H = numerical_hessian(f_saddle, np.array([0.0, 0.0]))
eigvals = np.linalg.eigvalsh(H)
print(f"Hessian:\n{H}")
print(f"Eigenvalues: {eigvals}")  # [−2, +2] → saddle point

# Minimum: f(x,y) = x² + 4y² at origin
f_min = lambda x: x[0]**2 + 4*x[1]**2
H2 = numerical_hessian(f_min, np.array([0.0, 0.0]))
eigvals2 = np.linalg.eigvalsh(H2)
print(f"\nHessian:\n{H2}")
print(f"Eigenvalues: {eigvals2}")  # [2, 8] → minimum
print(f"Condition number: {max(eigvals2)/min(eigvals2):.1f}")  # κ = 4
Section 16.7

Gradients in ML

Why This Matters for AI/ML

The Gradient Toolkit for Deep Learning

  • Gradient = direction of descent: The gradient \(\nabla L(\boldsymbol\theta)\) tells us exactly how to update every parameter to reduce the loss. This is the entire basis of training neural networks.
  • Jacobian = layer derivative: Each layer \(\mathbf{f}_\ell\) has a Jacobian \(J_\ell\). Backpropagation computes the chain \(\nabla L = J_1^\top J_2^\top \cdots J_L^\top \nabla \ell\) via successive vector-Jacobian products — never materialising the full Jacobian.
  • Hessian = curvature: The Hessian's eigenvalues determine whether a critical point is a minimum, maximum, or saddle. The condition number \(\kappa = \lambda_{\max}/\lambda_{\min}\) of the Hessian controls the convergence rate of gradient descent (Chapter 13).
  • Hessian-free methods: For \(n\) parameters, the full Hessian has \(n^2\) entries — prohibitive for large models. Hessian-vector products \(H\mathbf{v}\) can be computed in \(O(n)\) via autodiff, enabling second-order methods (conjugate gradient, L-BFGS) without storing the Hessian.
  • The Jacobian of softmax: \(J_{ij} = s_i(\delta_{ij} - s_j)\) where \(s = \text{softmax}(z)\). This dense Jacobian is why cross-entropy loss is preferred over MSE for classification — cross-entropy simplifies the gradient to \(s - y\), avoiding the softmax Jacobian entirely.

Socratic Knowledge Check

You are debugging a neural network that is training slowly. Profiling shows that the loss surface near the current parameters has a Hessian condition number of 10,000. Using the concepts from this chapter, diagnose and propose solutions.
1
Compute the gradient of \(f(x, y) = x^2 y + e^{xy}\) at the point \((1, 0)\).
\(\frac{\partial f}{\partial x} = 2xy + ye^{xy}\), \(\frac{\partial f}{\partial y} = x^2 + xe^{xy}\). At \((1,0)\): \(\frac{\partial f}{\partial x} = 0 + 0 = 0\), \(\frac{\partial f}{\partial y} = 1 + 1 = 2\). So \(\nabla f(1,0) = (0, 2)^\top\). The function is increasing in the \(y\)-direction and flat in the \(x\)-direction at this point.
2
What is the Jacobian of a linear layer \(\mathbf{f}(\mathbf{x}) = W\mathbf{x} + \mathbf{b}\)? What is the Jacobian of ReLU applied element-wise?
For the linear layer \(\mathbf{f}(\mathbf{x}) = W\mathbf{x} + \mathbf{b}\), \(\frac{\partial f_i}{\partial x_j} = W_{ij}\), so the Jacobian is \(J = W\). For element-wise ReLU \(g_i(\mathbf{z}) = \max(0, z_i)\), the Jacobian is diagonal: \(J = \text{diag}(\mathbf{1}_{z_i > 0})\), where entry \((i,i)\) is 1 if \(z_i > 0\) and 0 otherwise. This diagonal structure is why ReLU backprop is just a mask multiplication.
3
A function \(f : \mathbb{R}^{100} \to \mathbb{R}\) has a Hessian with 50 positive and 50 negative eigenvalues at a critical point. Is this a local minimum, maximum, or saddle point?
This is a saddle point. With mixed-sign eigenvalues, the Hessian is indefinite: the function curves upward in 50 directions and downward in 50 others. This is typical in high-dimensional optimisation — at random critical points in \(\mathbb{R}^{100}\), the probability that all 100 eigenvalues are positive is extremely small (\(\sim 2^{-100}\)), so almost all critical points are saddle points rather than local minima.
4
Back to the debugging scenario: the Hessian condition number \(\kappa = 10{,}000\). What does this tell you about the loss surface, and what are two practical remedies?
A condition number of 10,000 means \(\lambda_{\max}/\lambda_{\min} = 10^4\): the loss surface is a very narrow, elongated valley. Gradient descent zigzags across the narrow dimension while making slow progress along the long dimension. From Chapter 13, vanilla GD needs \(O(\kappa) = O(10^4)\) iterations. Remedies: (1) Momentum/Adam: damps oscillations, effectively reducing the condition number. Nesterov acceleration needs \(O(\sqrt{\kappa}) = O(100)\) iterations. (2) Preconditioning: multiply the gradient by an approximation to \(H^{-1}\) (as in L-BFGS or natural gradient), which transforms the narrow valley into a rounder bowl.
5
Why does backpropagation compute vector-Jacobian products \(\mathbf{v}^\top J\) rather than full Jacobians? What would happen if you materialised all Jacobians in a large network?
A layer with input dimension \(n\) and output dimension \(m\) has an \(m \times n\) Jacobian. For a large hidden layer with \(n = m = 4096\), the full Jacobian has \(4096^2 \approx 16.7\) million entries — per layer, per sample. A 100-layer network would need billions of entries. The VJP \(\mathbf{v}^\top J\) takes an \(m\)-vector and produces an \(n\)-vector in \(O(mn)\) time without ever forming the \(m \times n\) matrix. This is the key insight of reverse-mode autodiff: we only need the product, not the matrix. Memory stays \(O(n)\) per layer rather than \(O(mn)\).
