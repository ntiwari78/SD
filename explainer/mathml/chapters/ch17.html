Optimization in Multiple Variables

From single-variable hill-climbing to the full multi-dimensional gradient descent that powers modern machine learning.

35 min read
Advanced
The Big Picture

From hills to high-dimensional landscapes

In Chapter 13 we learned to slide downhill along a single-variable curve. In Chapter 16 we built the gradient — a vector telling us the steepest uphill direction in \(\mathbb{R}^n\). Now we marry the two ideas: gradient descent navigates a loss surface with millions of parameters, guided entirely by that gradient vector. This single algorithm — and its variants — is the engine behind virtually every neural network ever trained.

Section 17.1

Multivariable Functions in Code

Before diving into theory, let's establish our running example — the Rosenbrock function, a classic test surface for optimization algorithms. It has a narrow curved valley that makes it notoriously difficult to minimize:

\[f(x, y) = (a - x)^2 + b(y - x^2)^2\]

With \(a = 1, b = 100\), the global minimum sits at \((1, 1)\) where \(f = 0\). The challenge is that the valley floor curves gently downward while the walls rise steeply, trapping naive optimizers.

Pythonimport numpy as np

def rosenbrock(xy):
    """Rosenbrock function: minimum at (1, 1) with value 0."""
    x, y = xy
    return (1 - x)**2 + 100 * (y - x**2)**2

def rosenbrock_grad(xy):
    """Analytical gradient of the Rosenbrock function."""
    x, y = xy
    dfdx = -2 * (1 - x) + 200 * (y - x**2) * (-2 * x)
    dfdy = 200 * (y - x**2)
    return np.array([dfdx, dfdy])

# Evaluate at a test point
p = np.array([-1.0, 1.0])
print(f"f({p}) = {rosenbrock(p)}")         # 400.0
print(f"∇f({p}) = {rosenbrock_grad(p)}")   # [  -4. -400.]

The gradient at \((-1, 1)\) is \((-4, -400)\): the \(y\)-component is a hundred times larger than the \(x\)-component, reflecting the steep valley walls versus the gentle slope along the valley floor. This anisotropy is the central challenge of high-dimensional optimization.

Section 17.2

Critical Points Revisited

Chapter 13 defined critical points where \(f'(x) = 0\). In multiple dimensions, the condition becomes:

Definition — Critical Point

A point \(\mathbf{x}^* \in \mathbb{R}^n\) is a critical point of \(f: \mathbb{R}^n \to \mathbb{R}\) if the gradient vanishes:

\[\nabla f(\mathbf{x}^*) = \mathbf{0}\]

Critical points are classified as local minima, local maxima, or saddle points.

In one dimension there are only three possibilities for a critical point. In higher dimensions, saddle points become overwhelmingly more common. At a saddle point the function curves upward along some directions and downward along others — think of a mountain pass between two peaks.

The necessity condition

Theorem — First-Order Necessary Condition

If \(\mathbf{x}^*\) is a local minimum of a differentiable \(f: \mathbb{R}^n \to \mathbb{R}\), then \(\nabla f(\mathbf{x}^*) = \mathbf{0}\).

Suppose \(\mathbf{x}^*\) is a local minimum and, for contradiction, that \(\nabla f(\mathbf{x}^*) \neq \mathbf{0}\). Set \(\mathbf{d} = -\nabla f(\mathbf{x}^*)\). Then the directional derivative is:

\[D_{\mathbf{d}} f(\mathbf{x}^*) = \nabla f(\mathbf{x}^*) \cdot \mathbf{d} = -\|\nabla f(\mathbf{x}^*)\|^2 < 0\]

So \(f\) decreases in direction \(\mathbf{d}\). By the definition of the directional derivative, for sufficiently small \(t > 0\):

\[f(\mathbf{x}^* + t\mathbf{d}) < f(\mathbf{x}^*)\]

This contradicts \(\mathbf{x}^*\) being a local minimum. Hence \(\nabla f(\mathbf{x}^*) = \mathbf{0}\). \(\blacksquare\)

The converse is false: \(\nabla f = \mathbf{0}\) does not guarantee a minimum. We need second-order information to distinguish minima from saddle points.

Section 17.3

Second-Order Conditions

The Hessian matrix \(\mathbf{H}_f(\mathbf{x}^*)\), introduced in Chapter 16, provides the multivariable analogue of the second derivative test.

Theorem — Second-Order Sufficient Conditions

Let \(\mathbf{x}^*\) be a critical point of \(f \in C^2\) (i.e., \(\nabla f(\mathbf{x}^*) = \mathbf{0}\)). Then:

1. If \(\mathbf{H}_f(\mathbf{x}^*)\) is positive definite (all eigenvalues \(> 0\)), then \(\mathbf{x}^*\) is a strict local minimum.

2. If \(\mathbf{H}_f(\mathbf{x}^*)\) is negative definite (all eigenvalues \(< 0\)), then \(\mathbf{x}^*\) is a strict local maximum.

3. If \(\mathbf{H}_f(\mathbf{x}^*)\) has both positive and negative eigenvalues, then \(\mathbf{x}^*\) is a saddle point.

By Taylor's theorem in \(\mathbb{R}^n\), for \(\mathbf{d} = \mathbf{x} - \mathbf{x}^*\) near zero:

\[f(\mathbf{x}) = f(\mathbf{x}^*) + \underbrace{\nabla f(\mathbf{x}^*)^\top \mathbf{d}}_{= 0} + \frac{1}{2}\mathbf{d}^\top \mathbf{H}_f(\mathbf{x}^*)\mathbf{d} + o(\|\mathbf{d}\|^2)\]

Since \(\mathbf{H}_f(\mathbf{x}^*)\) is positive definite, there exists \(\lambda_{\min} > 0\) (its smallest eigenvalue) such that \(\mathbf{d}^\top \mathbf{H}_f(\mathbf{x}^*)\mathbf{d} \geq \lambda_{\min}\|\mathbf{d}\|^2\). For \(\mathbf{d}\) small enough, the quadratic term dominates the \(o(\|\mathbf{d}\|^2)\) remainder, giving \(f(\mathbf{x}) > f(\mathbf{x}^*)\) for all \(\mathbf{x} \neq \mathbf{x}^*\) in a neighbourhood. \(\blacksquare\)

Pythondef classify_critical_point(hessian):
    """Classify a critical point using eigenvalues of the Hessian."""
    eigenvalues = np.linalg.eigvalsh(hessian)
    print(f"Eigenvalues: {eigenvalues}")
    if np.all(eigenvalues > 0):
        return "strict local minimum"
    elif np.all(eigenvalues < 0):
        return "strict local maximum"
    elif np.any(eigenvalues > 0) and np.any(eigenvalues < 0):
        return "saddle point"
    else:
        return "inconclusive (semi-definite)"

# Rosenbrock Hessian at the minimum (1, 1)
H_min = np.array([[802, -400],
                   [-400,  200]])
print(classify_critical_point(H_min))
# Eigenvalues: [  0.399  1001.6]  →  strict local minimum

# Saddle point example: f(x,y) = x² - y² at origin
H_saddle = np.array([[2, 0],
                      [0, -2]])
print(classify_critical_point(H_saddle))
# Eigenvalues: [-2.  2.]  →  saddle point
Key Insight

The condition number of the Hessian — the ratio \(\lambda_{\max}/\lambda_{\min}\) — controls how hard a problem is to optimize. For the Rosenbrock Hessian above, \(\kappa \approx 2509\), meaning the surface is 2509 times steeper in the worst direction than the best. This is why the Rosenbrock function is so difficult for gradient descent.

Section 17.4

Constrained Optimization

Many ML problems have constraints: probability distributions must sum to one, regularised weights must stay inside a ball, SVMs enforce margin constraints. The method of Lagrange multipliers turns constrained problems into unconstrained ones.

Definition — Lagrangian

Given \(f: \mathbb{R}^n \to \mathbb{R}\) subject to equality constraints \(g_i(\mathbf{x}) = 0\) for \(i = 1, \ldots, m\), the Lagrangian is:

\[\mathcal{L}(\mathbf{x}, \boldsymbol{\lambda}) = f(\mathbf{x}) + \sum_{i=1}^m \lambda_i \, g_i(\mathbf{x})\]

where \(\lambda_1, \ldots, \lambda_m\) are Lagrange multipliers.

Theorem — Lagrange Multiplier Condition

At a constrained extremum \(\mathbf{x}^*\), the gradient of \(f\) is a linear combination of the constraint gradients:

\[\nabla f(\mathbf{x}^*) = -\sum_{i=1}^m \lambda_i \nabla g_i(\mathbf{x}^*)\]

Equivalently, \(\nabla_{\mathbf{x}} \mathcal{L}(\mathbf{x}^*, \boldsymbol{\lambda}^*) = \mathbf{0}\) and \(g_i(\mathbf{x}^*) = 0\) for all \(i\).

The geometric intuition: at a constrained optimum, the gradient of \(f\) must be perpendicular to the constraint surface. If it weren't, you could move along the constraint and improve \(f\).

Pythonfrom scipy.optimize import minimize

# Minimize f(x,y) = x² + y² subject to x + y = 1
# Using scipy's constrained optimizer
result = minimize(
    fun=lambda xy: xy[0]**2 + xy[1]**2,
    x0=[0.0, 0.0],
    method='SLSQP',
    jac=lambda xy: np.array([2*xy[0], 2*xy[1]]),
    constraints={'type': 'eq', 'fun': lambda xy: xy[0] + xy[1] - 1}
)
print(f"Constrained minimum: {result.x}")   # [0.5, 0.5]
print(f"f(x*) = {result.fun:.4f}")           # 0.5000

# Verify via Lagrange: ∇f = λ∇g  →  [2x, 2y] = λ[1,1]
# Plus constraint x + y = 1  →  x = y = 0.5, λ = 1
print(f"Lagrange multiplier λ = {2 * result.x[0]:.4f}")  # 1.0

KKT conditions for inequalities

When inequality constraints \(g_i(\mathbf{x}) \leq 0\) are present, the Karush–Kuhn–Tucker (KKT) conditions extend Lagrange multipliers with complementary slackness: \(\lambda_i \geq 0\) and \(\lambda_i g_i(\mathbf{x}^*) = 0\). Either the constraint is active (\(g_i = 0\)) or its multiplier is zero. KKT conditions are the theoretical foundation of support vector machines.

Section 17.5

Full Gradient Descent

In Chapter 13 we wrote the scalar update rule \(x \leftarrow x - \eta f'(x)\). Now we upgrade to the full multivariable form.

Definition — Gradient Descent in \(\mathbb{R}^n\)
The update rule

Starting from \(\mathbf{x}_0 \in \mathbb{R}^n\), iterate:

\[\mathbf{x}_{k+1} = \mathbf{x}_k - \eta \,\nabla f(\mathbf{x}_k)\]

where \(\eta > 0\) is the learning rate (step size). Each step moves in the direction of steepest descent.

This is exactly how neural networks train: the parameter vector \(\mathbf{x}\) contains all the weights (often millions), \(f\) is the loss function, and \(\nabla f\) is computed by backpropagation. The entire training loop is repeated application of this single equation.

Gradient Descent on the Rosenbrock Function

Watch how GD struggles with the ill-conditioned Rosenbrock valley. Track the path, loss, and gradient norm.

def gradient_descent(grad_fn, x0, lr, n_steps, loss_fn=None):
    """Full gradient descent with trajectory logging."""
    x = np.array(x0, dtype=float)
    trajectory = [x.copy()]
    losses = [loss_fn(x)] if loss_fn else []

    for k in range(n_steps):
        g = grad_fn(x)
        x = x - lr * g
        trajectory.append(x.copy())
        if loss_fn:
            losses.append(loss_fn(x))

    return np.array(trajectory), losses

# Run GD on Rosenbrock from (-1, 1)
x0 = [-1.0, 1.0]
traj, losses = gradient_descent(
    rosenbrock_grad, x0, lr=0.001, n_steps=10000,
    loss_fn=rosenbrock
)

print(f"Start:  f = {losses[0]:.2f}")
print(f"Step 1000:  x = {traj[1000]}, f = {losses[1000]:.4f}")
print(f"Step 5000:  x = {traj[5000]}, f = {losses[5000]:.6f}")
print(f"Final: x = {traj[-1]}, f = {losses[-1]:.8f}")
print(f"Distance to optimum: {np.linalg.norm(traj[-1] - [1,1]):.6f}")

Learning rate sensitivity

The learning rate \(\eta\) is the most critical hyperparameter. Too small and convergence is glacially slow; too large and the iterates diverge. The stability condition from Chapter 13 generalises: for a quadratic with Hessian \(\mathbf{H}\), convergence requires:

\[\eta < \frac{2}{\lambda_{\max}(\mathbf{H})}\]

For the Rosenbrock Hessian at \((1,1)\) with \(\lambda_{\max} \approx 1001.6\), this gives \(\eta < 0.002\). In practice, the optimal learning rate is \(\eta^* = 2/(\lambda_{\max} + \lambda_{\min})\), which balances progress across all eigendirections.

Section 17.6

Convergence Theory

The convergence rate depends on two properties of the loss surface: smoothness and convexity.

Definition — L-Smoothness

A differentiable \(f: \mathbb{R}^n \to \mathbb{R}\) is \(L\)-smooth if its gradient is Lipschitz continuous:

\[\|\nabla f(\mathbf{x}) - \nabla f(\mathbf{y})\| \leq L \|\mathbf{x} - \mathbf{y}\| \quad \forall \mathbf{x}, \mathbf{y}\]

Equivalently, \(f\) can be upper-bounded by a quadratic with curvature \(L\) at every point.

Definition — Strong Convexity

\(f\) is \(\mu\)-strongly convex (\(\mu > 0\)) if for all \(\mathbf{x}, \mathbf{y}\):

\[f(\mathbf{y}) \geq f(\mathbf{x}) + \nabla f(\mathbf{x})^\top (\mathbf{y} - \mathbf{x}) + \frac{\mu}{2}\|\mathbf{y} - \mathbf{x}\|^2\]

Equivalently, \(\mathbf{H}_f(\mathbf{x}) \succeq \mu \mathbf{I}\) everywhere — the surface curves at least as much as a paraboloid of curvature \(\mu\).

Theorem — GD Convergence Rates

For gradient descent with step size \(\eta = 1/L\) on an \(L\)-smooth function:

Convex: \(\displaystyle f(\mathbf{x}_k) - f(\mathbf{x}^*) \leq \frac{L\|\mathbf{x}_0 - \mathbf{x}^*\|^2}{2k}\) — sublinear \(O(1/k)\) rate.

\(\mu\)-strongly convex: \(\displaystyle \|\mathbf{x}_k - \mathbf{x}^*\|^2 \leq \left(1 - \frac{\mu}{L}\right)^k \|\mathbf{x}_0 - \mathbf{x}^*\|^2\) — linear (exponential) rate.

The ratio \(\kappa = L/\mu\) is the condition number. When \(\kappa\) is large, the error contracts by a factor of only \((1 - 1/\kappa)\) per step — convergence is slow. This exactly mirrors the intuition of the Rosenbrock valley: the condition number measures how elongated the loss surface is.

Pythondef convergence_comparison():
    """Compare GD convergence on well- vs ill-conditioned quadratics."""
    # Well-conditioned: κ = 2
    A_good = np.array([[2, 0], [0, 1]])
    b_good = np.array([2, 1])
    # Ill-conditioned: κ = 100
    A_bad = np.array([[100, 0], [0, 1]])
    b_bad = np.array([100, 1])

    for name, A, b in [("κ=2", A_good, b_good), ("κ=100", A_bad, b_bad)]:
        L = np.max(np.linalg.eigvalsh(A))
        lr = 1.0 / L
        x_star = np.linalg.solve(A, b)

        x = np.zeros(2)
        errors = []
        for k in range(200):
            errors.append(np.linalg.norm(x - x_star))
            grad = A @ x - b
            x = x - lr * grad

        half_idx = next(i for i, e in enumerate(errors) if e < errors[0]/2)
        print(f"{name}: halving time = {half_idx} steps, "
              f"error at step 200 = {errors[-1]:.2e}")

convergence_comparison()

Section 17.7

Advanced Optimizers

Plain gradient descent converges slowly on ill-conditioned surfaces. The ML community has developed a family of adaptive optimizers that automatically adjust per-parameter learning rates.

Momentum

Momentum accumulates past gradients into a velocity vector, helping the optimizer build speed along consistently downhill directions while dampening oscillations across the valley:

\[\mathbf{v}_{k+1} = \beta \mathbf{v}_k + \nabla f(\mathbf{x}_k), \qquad \mathbf{x}_{k+1} = \mathbf{x}_k - \eta \mathbf{v}_{k+1}\]

With \(\beta = 0.9\), the effective step in a consistent direction is amplified by a factor of \(1/(1 - \beta) = 10\).

Adam: Adaptive Moment Estimation

Adam combines momentum with per-parameter learning rate scaling using the second moment of the gradient. It is the default optimizer in most deep learning projects.

Definition — Adam Update

Maintain first and second moment estimates \(\mathbf{m}_k, \mathbf{v}_k\):

\[\mathbf{m}_{k+1} = \beta_1 \mathbf{m}_k + (1 - \beta_1)\,\mathbf{g}_k, \qquad \mathbf{v}_{k+1} = \beta_2 \mathbf{v}_k + (1 - \beta_2)\,\mathbf{g}_k^2\]

Bias-correct: \(\hat{\mathbf{m}} = \mathbf{m}_{k+1}/(1 - \beta_1^{k+1})\), \(\hat{\mathbf{v}} = \mathbf{v}_{k+1}/(1 - \beta_2^{k+1})\). Update:

\[\mathbf{x}_{k+1} = \mathbf{x}_k - \frac{\eta}{\sqrt{\hat{\mathbf{v}}} + \epsilon}\,\hat{\mathbf{m}}\]

Default hyperparameters: \(\beta_1 = 0.9\), \(\beta_2 = 0.999\), \(\epsilon = 10^{-8}\).

def adam(grad_fn, x0, lr=0.001, beta1=0.9, beta2=0.999,
         eps=1e-8, n_steps=5000, loss_fn=None):
    """Adam optimizer with trajectory logging."""
    x = np.array(x0, dtype=float)
    m = np.zeros_like(x)
    v = np.zeros_like(x)
    trajectory = [x.copy()]
    losses = [loss_fn(x)] if loss_fn else []

    for k in range(1, n_steps + 1):
        g = grad_fn(x)
        m = beta1 * m + (1 - beta1) * g
        v = beta2 * v + (1 - beta2) * g**2
        m_hat = m / (1 - beta1**k)
        v_hat = v / (1 - beta2**k)
        x = x - lr * m_hat / (np.sqrt(v_hat) + eps)
        trajectory.append(x.copy())
        if loss_fn:
            losses.append(loss_fn(x))

    return np.array(trajectory), losses

def momentum_gd(grad_fn, x0, lr=0.001, beta=0.9,
                n_steps=5000, loss_fn=None):
    """GD with momentum."""
    x = np.array(x0, dtype=float)
    vel = np.zeros_like(x)
    trajectory = [x.copy()]
    losses = [loss_fn(x)] if loss_fn else []

    for k in range(n_steps):
        g = grad_fn(x)
        vel = beta * vel + g
        x = x - lr * vel
        trajectory.append(x.copy())
        if loss_fn:
            losses.append(loss_fn(x))

    return np.array(trajectory), losses

x0 = [-1.0, 1.0]
n = 5000

_, loss_gd = gradient_descent(rosenbrock_grad, x0, 0.001, n, rosenbrock)
_, loss_mom = momentum_gd(rosenbrock_grad, x0, 0.0005, 0.9, n, rosenbrock)
_, loss_adam = adam(rosenbrock_grad, x0, 0.01, n_steps=n, loss_fn=rosenbrock)

for name, losses in [("GD", loss_gd), ("Momentum", loss_mom), ("Adam", loss_adam)]:
    print(f"{name:10s}: final loss = {losses[-1]:.6f}")

Why Adam works

The denominator \(\sqrt{\hat{\mathbf{v}}}\) contains the root-mean-square of recent gradients. Parameters with large gradients get smaller effective learning rates, while parameters with small gradients get larger ones. This implicit preconditioning adapts to the local curvature without computing the full Hessian — a massive computational saving when the parameter count is in the millions.

Section 17.8

Optimization in Machine Learning

Why This Matters for ML

The full picture: from loss surfaces to trained models

  • Stochastic gradients: In practice, we approximate \(\nabla f\) with a mini-batch gradient — the average over a random subset of training examples. This gives SGD: noisier but much cheaper per step, and the noise actually helps escape sharp local minima.
  • Non-convex landscapes: Neural network loss surfaces are highly non-convex with many saddle points. Remarkably, SGD with momentum or Adam navigates these effectively because most saddle points are "escapable" — random perturbations push the optimizer away.
  • Learning rate schedules: Start with a large learning rate for fast initial progress, then decay (cosine annealing, step decay, warmup) for fine convergence. Modern transformers use linear warmup followed by cosine decay.
  • Regularisation as constraints: L2 regularisation (\(\lambda\|\mathbf{w}\|^2\)) is equivalent to constrained optimization over a ball. L1 regularisation leads to sparsity via soft-thresholding in the gradient update.

Mini-batch SGD

Instead of computing the gradient over all \(N\) training examples, we sample a mini-batch \(\mathcal{B} \subset \{1, \ldots, N\}\) of size \(B\) and use the estimate:

\[\mathbf{g}_k = \frac{1}{B}\sum_{i \in \mathcal{B}} \nabla f_i(\mathbf{x}_k)\]

This estimate is unbiased: \(\mathbb{E}[\mathbf{g}_k] = \nabla f(\mathbf{x}_k)\). The variance decreases as \(O(1/B)\), but even \(B = 32\) is enough for effective training in most deep learning settings.

Pythondef sgd_linear_regression(X, y, lr=0.01, batch_size=32,
                           n_epochs=50, seed=42):
    """Mini-batch SGD for least-squares linear regression."""
    rng = np.random.default_rng(seed)
    n, d = X.shape
    w = rng.normal(0, 0.01, size=d)
    losses = []

    for epoch in range(n_epochs):
        indices = rng.permutation(n)
        epoch_loss = 0.0
        n_batches = 0

        for start in range(0, n, batch_size):
            batch = indices[start:start + batch_size]
            X_b, y_b = X[batch], y[batch]

            # Mini-batch gradient: (1/B) * X_b^T (X_b w - y_b)
            residual = X_b @ w - y_b
            grad = X_b.T @ residual / len(batch)

            w = w - lr * grad
            epoch_loss += np.mean(residual**2)
            n_batches += 1

        losses.append(epoch_loss / n_batches)

    return w, losses

# Synthetic example
rng = np.random.default_rng(0)
X = rng.normal(0, 1, (1000, 5))
w_true = np.array([1.0, -2.0, 0.5, 3.0, -1.5])
y = X @ w_true + rng.normal(0, 0.1, 1000)

w_sgd, losses = sgd_linear_regression(X, y, lr=0.05, batch_size=64)
print(f"True w:  {w_true}")
print(f"SGD  w:  {np.round(w_sgd, 3)}")
print(f"Final MSE: {losses[-1]:.6f}")

Putting it all together: training a neural network

Every neural network training loop is an instance of the following pattern: forward pass (compute loss), backward pass (compute gradient via chain rule), and update (apply optimizer step). The mathematics of this chapter — gradients, Hessians, condition numbers, adaptive learning rates — govern every aspect of this process.

Pythondef train_two_layer_net(X, y, hidden=32, lr=0.01,
                        n_epochs=100, batch_size=32, seed=42):
    """Train a 2-layer ReLU network with Adam from scratch."""
    rng = np.random.default_rng(seed)
    n, d = X.shape
    out_dim = 1

    # He initialisation
    W1 = rng.normal(0, np.sqrt(2/d), (d, hidden))
    b1 = np.zeros(hidden)
    W2 = rng.normal(0, np.sqrt(2/hidden), (hidden, out_dim))
    b2 = np.zeros(out_dim)

    # Adam state
    params = [W1, b1, W2, b2]
    m = [np.zeros_like(p) for p in params]
    v = [np.zeros_like(p) for p in params]
    t = 0

    losses = []
    for epoch in range(n_epochs):
        idx = rng.permutation(n)
        epoch_loss = 0.0

        for start in range(0, n, batch_size):
            batch = idx[start:start+batch_size]
            X_b, y_b = X[batch], y[batch].reshape(-1, 1)
            B = len(batch)

            # Forward pass
            z1 = X_b @ W1 + b1
            a1 = np.maximum(0, z1)        # ReLU
            y_hat = a1 @ W2 + b2
            loss = np.mean((y_hat - y_b)**2)
            epoch_loss += loss

            # Backward pass
            dloss = 2 * (y_hat - y_b) / B
            dW2 = a1.T @ dloss
            db2 = dloss.sum(axis=0)
            da1 = dloss @ W2.T
            dz1 = da1 * (z1 > 0)          # ReLU derivative
            dW1 = X_b.T @ dz1
            db1 = dz1.sum(axis=0)

            # Adam update
            grads = [dW1, db1, dW2, db2]
            t += 1
            for i in range(4):
                m[i] = 0.9 * m[i] + 0.1 * grads[i]
                v[i] = 0.999 * v[i] + 0.001 * grads[i]**2
                m_hat = m[i] / (1 - 0.9**t)
                v_hat = v[i] / (1 - 0.999**t)
                params[i] -= lr * m_hat / (np.sqrt(v_hat) + 1e-8)

            W1, b1, W2, b2 = params

        losses.append(epoch_loss * batch_size / n)

    print(f"Epoch  1 loss: {losses[0]:.4f}")
    print(f"Epoch 50 loss: {losses[49]:.4f}")
    print(f"Epoch {n_epochs} loss: {losses[-1]:.4f}")
    return params, losses

# Train on synthetic data
X_train = rng.normal(0, 1, (500, 5))
y_train = np.sin(X_train[:, 0]) + 0.5 * X_train[:, 1]**2
train_two_layer_net(X_train, y_train, hidden=64, lr=0.005)

The training loss oscillates wildly after an initial decrease. What is the most likely cause, and how would you fix it?
The learning rate is too large. Oscillation means the steps overshoot the minimum and bounce back and forth across the valley. The fix is to reduce the learning rate, or better, use a learning rate schedule (e.g., cosine annealing) that starts large for fast progress and decays over time. Alternatively, switching to Adam can help because its adaptive per-parameter rates dampen these oscillations automatically.
3
Why does Adam often outperform plain GD on neural network loss surfaces? Explain in terms of condition numbers.
Neural network loss surfaces are typically ill-conditioned: different parameters (or directions in parameter space) have very different curvatures. Plain GD uses a single learning rate for all parameters, so it must use the smallest safe rate (set by the largest curvature), making progress in low-curvature directions painfully slow. Adam divides each parameter's update by the RMS of its recent gradients, effectively giving each parameter its own learning rate. This implicit diagonal preconditioning approximately reduces the condition number, accelerating convergence along all directions.
4
You have a constrained optimisation problem: minimise cross-entropy loss subject to \(\sum_i w_i = 1\) and \(w_i \geq 0\). How do Lagrange multipliers connect to the softmax function?
When you minimise the cross-entropy (or equivalently, maximise entropy subject to moment constraints) with the constraint that the outputs form a valid probability distribution (\(\sum_i w_i = 1\), \(w_i \geq 0\)), applying the KKT conditions yields the exponential form \(w_i \propto e^{z_i}\). Normalising gives exactly the softmax function \(w_i = e^{z_i}/\sum_j e^{z_j}\). The Lagrange multiplier for the sum-to-one constraint becomes the log of the partition function. So softmax is not an arbitrary choice — it is the solution to a constrained optimisation problem.
5
A convex function with condition number \(\kappa = 1000\). Roughly how many GD steps does it take to reduce the error by a factor of \(e \approx 2.718\)?
For a \(\mu\)-strongly convex, \(L\)-smooth function with \(\kappa = L/\mu = 1000\), the error contracts by \((1 - 1/\kappa)\) per step. To reduce by a factor of \(e\), we need \(k\) steps where \((1 - 1/1000)^k = 1/e\), i.e., \(k = -1/\ln(1 - 1/1000) \approx 1000\). So roughly \(\kappa\) steps to reduce the error by one \(e\)-fold. This is why reducing the condition number (via preconditioning or adaptive methods) is so valuable.
