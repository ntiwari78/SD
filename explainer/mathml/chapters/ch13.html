Optimization

Minima, maxima, gradient descent, convergence theory, and why the algorithm that trains every neural network works.

50 min read Part II · Calculus
Mental Model

Rolling Downhill

Imagine a ball placed on a hilly landscape. Release it and gravity pulls it downhill — it accelerates where the slope is steep, slows where it flattens, and eventually settles in a valley. Gradient descent does exactly this, but in the landscape of a loss function. The derivative tells us which way is "downhill" at our current position, and we take a step in that direction. The step size (learning rate) controls how far we roll per step. Too large and we overshoot the valley; too small and training takes forever. Understanding the theory of minima and maxima tells us what kinds of valleys exist and when we are guaranteed to find the deepest one.

Section 13.1

Minima and Maxima

Definition — Local and Global Extrema

Let \(f : D \to \mathbb{R}\). A point \(x^* \in D\) is a:

  • Local minimum if there exists \(\delta > 0\) such that \(f(x^*) \leq f(x)\) for all \(x \in D\) with \(|x - x^*| < \delta\).
  • Global minimum if \(f(x^*) \leq f(x)\) for all \(x \in D\).
  • Strict local minimum if the inequalities above are strict for \(x \neq x^*\).

Maxima are defined analogously with reversed inequalities.

Every global minimum is a local minimum, but the converse is false. A function like \(f(x) = x^4 - 2x^2\) has two local minima (at \(x = \pm 1\)) and one local maximum (at \(x = 0\)). In deep learning, loss surfaces have astronomically many local minima, yet recent theory suggests that most are "good enough" — close to the global minimum in value.

Theorem — Fermat's Theorem (Necessary Condition)

If \(f\) is differentiable at an interior point \(x^*\) and \(x^*\) is a local extremum, then \(f'(x^*) = 0\).

Suppose \(x^*\) is a local minimum (the maximum case is analogous). For small \(h > 0\), \(f(x^* + h) \geq f(x^*)\), so \(\frac{f(x^* + h) - f(x^*)}{h} \geq 0\). Taking \(h \to 0^+\), the derivative from the right is \(\geq 0\). For \(h < 0\), \(\frac{f(x^* + h) - f(x^*)}{h} \leq 0\) (numerator \(\geq 0\), denominator \(< 0\)). Taking \(h \to 0^-\), the derivative from the left is \(\leq 0\). Since \(f\) is differentiable, both one-sided limits equal \(f'(x^*)\), giving \(0 \leq f'(x^*) \leq 0\), so \(f'(x^*) = 0\). \(\blacksquare\)

Definition — Critical Point

A point \(x_0\) where \(f'(x_0) = 0\) is called a critical point (or stationary point). Critical points are candidates for extrema, but not all critical points are extrema — some are saddle points or inflection points.

Fermat's theorem gives a necessary but not sufficient condition. The point \(x = 0\) for \(f(x) = x^3\) satisfies \(f'(0) = 0\) but is neither a minimum nor a maximum — it is an inflection point. We need the derivative tests of the next section to classify critical points.

Section 13.2

First and Second Derivative Tests

Theorem — First Derivative Test

Suppose \(f\) is continuous on an interval containing \(x^*\) and differentiable near \(x^*\) (except possibly at \(x^*\) itself).

If \(f'(x) < 0\) for \(x < x^*\) and \(f'(x) > 0\) for \(x > x^*\) (derivative changes from negative to positive), then \(x^*\) is a local minimum.

If \(f'(x) > 0\) for \(x < x^*\) and \(f'(x) < 0\) for \(x > x^*\), then \(x^*\) is a local maximum.

Theorem — Second Derivative Test

Suppose \(f'(x^*) = 0\) and \(f''(x^*)\) exists.

If \(f''(x^*) > 0\), then \(x^*\) is a strict local minimum.

If \(f''(x^*) < 0\), then \(x^*\) is a strict local maximum.

If \(f''(x^*) = 0\), the test is inconclusive.

The second derivative test is intuitive: \(f''(x^*) > 0\) means the function is curving upward (bowl shape), so the critical point sits at the bottom — a minimum. In the multivariable case (Chapter 17), the second derivative test generalises to checking whether the Hessian matrix is positive definite.

Python
import numpy as np

def find_critical_points(f, f_prime, f_double_prime, xs):
    """Find and classify critical points in a search grid."""
    # Find approximate zeros of f'
    signs = np.sign(f_prime(xs))
    sign_changes = np.where(np.diff(signs) != 0)[0]

    results = []
    for idx in sign_changes:
        # Bisect to find precise critical point
        a, b = xs[idx], xs[idx + 1]
        for _ in range(64):
            mid = (a + b) / 2
            if f_prime(mid) * f_prime(a) < 0:
                b = mid
            else:
                a = mid
        x_crit = (a + b) / 2
        f2 = f_double_prime(x_crit)

        if f2 > 1e-10:
            kind = "local minimum"
        elif f2 < -1e-10:
            kind = "local maximum"
        else:
            kind = "inconclusive"
        results.append((x_crit, f(x_crit), kind))
    return results

# Example: f(x) = x⁴ - 2x²
f = lambda x: x**4 - 2*x**2
fp = lambda x: 4*x**3 - 4*x
fpp = lambda x: 12*x**2 - 4

xs = np.linspace(-2, 2, 10000)
for x, fx, kind in find_critical_points(f, fp, fpp, xs):
    print(f"x = {x:+.6f}, f(x) = {fx:+.6f}  →  {kind}")
Section 13.3

Convexity

Convexity is the single most important structural property for optimization. On convex problems, every local minimum is global — there are no traps.

Definition — Convex Function

A function \(f : I \to \mathbb{R}\) (where \(I\) is an interval) is convex if for all \(x, y \in I\) and \(\lambda \in [0, 1]\):

\[f(\lambda x + (1-\lambda)y) \leq \lambda f(x) + (1-\lambda)f(y).\]

Geometrically: the line segment between any two points on the graph lies above (or on) the graph.

Theorem — Second-Order Characterisation

If \(f\) is twice differentiable on an interval \(I\), then \(f\) is convex on \(I\) if and only if \(f''(x) \geq 0\) for all \(x \in I\).

Proposition — Convex Functions Have Global Minima

If \(f\) is convex and \(x^*\) is a local minimum, then \(x^*\) is a global minimum. Moreover, if \(f\) is strictly convex, the global minimum is unique (if it exists).

Suppose \(x^*\) is a local minimum of convex \(f\), and suppose for contradiction that there exists \(y\) with \(f(y) < f(x^*)\). By convexity, for \(\lambda \in (0, 1)\):

\[f(\lambda y + (1-\lambda)x^*) \leq \lambda f(y) + (1-\lambda) f(x^*) < \lambda f(x^*) + (1-\lambda) f(x^*) = f(x^*).\]

But the point \(\lambda y + (1-\lambda)x^*\) can be made arbitrarily close to \(x^*\) by taking \(\lambda\) small. This contradicts \(x^*\) being a local minimum. \(\blacksquare\)

Convexity in ML

Linear regression and logistic regression have convex loss functions, so gradient descent is guaranteed to find the global minimum. Deep neural networks have non-convex losses, which is why training involves careful initialisation, learning rate schedules, and multiple restarts. Despite non-convexity, modern overparameterised networks often behave "almost convexly" in practice.

Section 13.4

Gradient Descent

Gradient descent is the workhorse algorithm of machine learning. The idea is simple: to minimise \(f\), start at some point and repeatedly take steps in the direction of steepest descent.

Definition — Gradient Descent Update

Given a differentiable function \(f : \mathbb{R} \to \mathbb{R}\), the gradient descent iteration is:

\[x_{k+1} = x_k - \eta \, f'(x_k),\]

where \(\eta > 0\) is the learning rate (step size).

The negative sign ensures we move in the direction that decreases \(f\). When \(f'(x_k) > 0\), the function is increasing at \(x_k\), so we step left (decrease \(x\)). When \(f'(x_k) < 0\), we step right.

Python
import numpy as np

def gradient_descent(f, f_prime, x0, lr=0.1, tol=1e-8, max_iter=10000):
    """Vanilla gradient descent in 1D with convergence tracking."""
    x = x0
    history = [x]
    for i in range(max_iter):
        grad = f_prime(x)
        x = x - lr * grad
        history.append(x)
        if abs(grad) < tol:
            print(f"Converged in {i+1} iterations")
            break
    return x, np.array(history)

# Minimise f(x) = (x - 3)² + 1
f = lambda x: (x - 3)**2 + 1
fp = lambda x: 2 * (x - 3)

x_min, hist = gradient_descent(f, fp, x0=0.0, lr=0.1)
print(f"Minimum at x = {x_min:.6f}, f(x) = {f(x_min):.6f}")
print(f"True minimum: x = 3, f(x) = 1")

The Learning Rate

The learning rate \(\eta\) is the most important hyperparameter in gradient descent. Its effect can be understood through the lens of Taylor expansion. Near a critical point of a quadratic \(f(x) = \frac{1}{2}Lx^2\), the GD update gives \(x_{k+1} = (1 - \eta L)x_k\). This contracts to zero when \(|1 - \eta L| < 1\), i.e., \(\eta < 2/L\). For \(\eta = 1/L\), the convergence is fastest (one-step for a pure quadratic). For \(\eta > 2/L\), the iterates diverge.

The Goldilocks Zone

Too small: convergence is painfully slow — the ball barely moves. Too large: the ball overshoots the valley and oscillates or diverges. Just right: smooth, rapid convergence to the minimum. In practice, learning rate schedules (warmup + decay) or adaptive methods (Adam, AdaGrad) automate finding this sweet spot.

Section 13.5

Convergence Theory

When and how fast does gradient descent converge? The answer depends on the regularity of \(f\).

Definition — L-Smoothness

A differentiable function \(f\) is \(L\)-smooth if its derivative is Lipschitz continuous with constant \(L\):

\[|f'(x) - f'(y)| \leq L|x - y| \quad \text{for all } x, y.\]

Equivalently (for twice-differentiable functions), \(|f''(x)| \leq L\) everywhere.

Theorem — GD Convergence for Convex L-Smooth Functions

If \(f\) is convex and \(L\)-smooth, and we run gradient descent with \(\eta = 1/L\), then after \(k\) iterations:

\[f(x_k) - f(x^*) \leq \frac{L\|x_0 - x^*\|^2}{2k}.\]

This is an \(O(1/k)\) rate. To achieve \(\varepsilon\)-accuracy, we need \(O(1/\varepsilon)\) iterations.

Definition — Strong Convexity

A function \(f\) is \(\mu\)-strongly convex (with \(\mu > 0\)) if \(f(x) - \frac{\mu}{2}x^2\) is convex. Equivalently, \(f''(x) \geq \mu\) for all \(x\). Strong convexity guarantees a unique minimum and faster convergence.

Theorem — GD Convergence for Strongly Convex L-Smooth Functions

If \(f\) is \(\mu\)-strongly convex and \(L\)-smooth, gradient descent with \(\eta = 1/L\) gives:

\[f(x_k) - f(x^*) \leq \left(1 - \frac{\mu}{L}\right)^k \bigl(f(x_0) - f(x^*)\bigr).\]

This is linear (exponential) convergence. The ratio \(\kappa = L/\mu\) is the condition number — it controls how many iterations are needed.

A well-conditioned problem (\(\kappa\) close to 1) converges in few steps. An ill-conditioned problem (\(\kappa \gg 1\)) converges slowly because the loss landscape is a narrow, elongated valley — the gradient zigzags across the narrow dimension rather than making progress along the long dimension.

Performance Deep-Dive

Visualising Convergence Rates

Compare convergence on well-conditioned vs. ill-conditioned quadratics.

import numpy as np

def gd_quadratic(L, mu, x0, num_steps, lr=None):
    """GD on f(x) = 0.5*L*x² (shifted so minimum at 0)."""
    if lr is None:
        lr = 1.0 / L  # optimal for L-smooth
    x = x0
    losses = []
    for _ in range(num_steps):
        grad = L * x  # f'(x) = L*x for f = 0.5*L*x²
        x = x - lr * grad
        losses.append(0.5 * L * x**2)
    return losses

# Well-conditioned: κ = 2
losses_good = gd_quadratic(L=2, mu=1, x0=10.0, num_steps=50)

# Ill-conditioned: κ = 100
losses_bad = gd_quadratic(L=100, mu=1, x0=10.0, num_steps=50)

print("Step | Well-cond (κ=2) | Ill-cond (κ=100)")
for i in [0, 5, 10, 20, 49]:
    print(f"  {i:2d}  | {losses_good[i]:14.6e} | {losses_bad[i]:14.6e}")
# Well-conditioned converges in ~10 steps
# Ill-conditioned still has significant error after 50
Section 13.6

GD Variants and Momentum

Stochastic Gradient Descent (SGD)

In machine learning, the loss is typically a sum over \(n\) training examples: \(L(\theta) = \frac{1}{n}\sum_{i=1}^n \ell_i(\theta)\). Computing the full gradient costs \(O(n)\) — prohibitive when \(n\) is millions. Stochastic gradient descent approximates the gradient using a random mini-batch \(\mathcal{B} \subset \{1, \ldots, n\}\):

\[g_k = \frac{1}{|\mathcal{B}|}\sum_{i \in \mathcal{B}} \nabla \ell_i(\theta_k), \quad \theta_{k+1} = \theta_k - \eta \, g_k.\]

The stochastic gradient \(g_k\) is an unbiased estimator of the true gradient (\(\mathbb{E}[g_k] = \nabla L(\theta_k)\)), but introduces variance. Larger batch sizes reduce variance but cost more per step.

Momentum

Momentum addresses the zigzag problem on ill-conditioned landscapes by maintaining a "velocity" that accumulates past gradients:

\[v_{k+1} = \beta \, v_k + f'(x_k), \quad x_{k+1} = x_k - \eta \, v_{k+1},\]

where \(\beta \in [0, 1)\) is the momentum coefficient (typically 0.9). The velocity \(v\) acts like a rolling average of gradients. Consistent gradient directions get amplified; oscillating directions cancel out. This smooths the trajectory and accelerates convergence along the long axis of narrow valleys.

Theorem — Nesterov's Accelerated Gradient (Optimal Rate)

For \(L\)-smooth convex functions, Nesterov's accelerated gradient method achieves:

\[f(x_k) - f(x^*) \leq O\!\left(\frac{L\|x_0 - x^*\|^2}{k^2}\right),\]

which is an \(O(1/k^2)\) rate — provably optimal among first-order methods. This improves on the \(O(1/k)\) rate of vanilla GD.

Python
import numpy as np

def gd_momentum(f_prime, x0, lr=0.01, beta=0.9, max_iter=500):
    """Gradient descent with momentum."""
    x = x0
    v = 0.0
    history = [x]
    for _ in range(max_iter):
        grad = f_prime(x)
        v = beta * v + grad
        x = x - lr * v
        history.append(x)
    return x, np.array(history)

def nesterov_gd(f_prime, x0, lr=0.01, max_iter=500):
    """Nesterov accelerated gradient descent."""
    x = x0
    y = x0
    history = [x]
    for k in range(1, max_iter + 1):
        x_new = y - lr * f_prime(y)
        y = x_new + (k - 1) / (k + 2) * (x_new - x)
        x = x_new
        history.append(x)
    return x, np.array(history)

# Compare on f(x) = 0.5 * 50 * x² (condition number = 50)
fp = lambda x: 50 * x
x0 = 10.0

_, hist_vanilla = gradient_descent(lambda x: 25*x**2, fp, x0, lr=0.019, max_iter=200)
_, hist_momentum = gd_momentum(fp, x0, lr=0.019, beta=0.9, max_iter=200)
_, hist_nesterov = nesterov_gd(fp, x0, lr=0.019, max_iter=200)

for name, h in [("Vanilla", hist_vanilla),
                ("Momentum", hist_momentum),
                ("Nesterov", hist_nesterov)]:
    final_loss = 25 * h[-1]**2
    print(f"{name:10s}: x = {h[-1]:+.6e}, loss = {final_loss:.6e}")
Section 13.7

Gradient Ascent

Maximising \(f\) is equivalent to minimising \(-f\). Gradient ascent simply flips the sign:

\[x_{k+1} = x_k + \eta \, f'(x_k).\]

We step in the direction of the gradient (uphill). This appears in several ML contexts:

Generative adversarial networks (GANs): The discriminator maximises its ability to distinguish real from fake samples, while the generator minimises it. This min-max game alternates gradient descent (generator) and gradient ascent (discriminator).

Reinforcement learning: Policy gradient methods maximise expected reward \(J(\theta) = \mathbb{E}[R]\) by ascending the gradient \(\nabla_\theta J\).

Maximum likelihood: Finding the parameters that maximise the log-likelihood \(\ell(\theta) = \sum_i \log p(x_i|\theta)\) is gradient ascent on \(\ell\), or equivalently gradient descent on the negative log-likelihood.

Why This Matters for AI/ML

The Optimisation Perspective on Deep Learning

  • Training = optimization: Every neural network is trained by minimising a loss function. The tools of this chapter — critical points, derivative tests, convexity, gradient descent — are exactly the tools used to find good parameters.
  • Learning rate is king: The most impactful hyperparameter. Theory says \(\eta < 2/L\) for stability; in practice, learning rate warmup prevents early instability and cosine decay helps fine-tune.
  • Condition number determines difficulty: Ill-conditioned losses (large \(\kappa = L/\mu\)) cause slow convergence. Batch normalisation, layer normalisation, and careful initialisation improve conditioning.
  • Momentum is essential: Vanilla SGD is rarely used alone. Momentum (\(\beta = 0.9\)) or Adam (adaptive learning rates per parameter) are standard. Nesterov momentum is provably optimal for smooth convex functions.
  • Non-convexity is manageable: Despite theory requiring convexity for guarantees, deep networks train successfully because: (1) loss surfaces of overparameterised networks have few "bad" local minima, (2) saddle points dominate over local minima in high dimensions, and (3) SGD noise helps escape shallow local minima.

Socratic Knowledge Check

You are training a neural network and notice that the loss plateaus after a few epochs but the learning rate is reasonable. A colleague suggests adding momentum. Using the theory from this chapter, reason about what is happening and whether momentum will help.
1
Find all critical points of \(f(x) = x^3 - 3x + 2\) and classify each as a local min, local max, or neither.
\(f'(x) = 3x^2 - 3 = 3(x-1)(x+1) = 0\) gives \(x = 1\) and \(x = -1\). Second derivatives: \(f''(x) = 6x\). At \(x = 1\): \(f''(1) = 6 > 0\) → local minimum with \(f(1) = 0\). At \(x = -1\): \(f''(-1) = -6 < 0\) → local maximum with \(f(-1) = 4\).
2
Why does gradient descent with \(\eta = 0.5\) diverge on \(f(x) = 5x^2\)? What is the maximum safe learning rate?
Here \(f'(x) = 10x\), so \(L = 10\) (the Lipschitz constant of \(f'\)). The GD update is \(x_{k+1} = x_k - 10\eta x_k = (1 - 10\eta)x_k\). For convergence we need \(|1 - 10\eta| < 1\), giving \(\eta < 2/10 = 0.2\). With \(\eta = 0.5\), the multiplier is \(|1 - 5| = 4\), so \(|x_k|\) grows by a factor of 4 each step — divergence. The optimal rate \(\eta^* = 1/L = 0.1\) gives one-step convergence for this quadratic.
3
Is the cross-entropy loss \(\ell(p) = -[y\log p + (1-y)\log(1-p)]\) convex in \(p \in (0,1)\)? What does this imply for logistic regression?
Compute: \(\ell'(p) = -y/p + (1-y)/(1-p)\) and \(\ell''(p) = y/p^2 + (1-y)/(1-p)^2 > 0\) for \(p \in (0,1)\) and \(y \in \{0,1\}\). So yes, cross-entropy is strictly convex in \(p\). For logistic regression, \(p = \sigma(w^T x)\) and the composition with the linear model is also convex in \(w\) (since \(\sigma\) is log-concave). This means gradient descent on logistic regression will find the global minimum — no worries about local minima.
4
Your loss surface has condition number \(\kappa = 1000\). Without momentum, roughly how many GD iterations are needed to reduce the error by a factor of \(e^{-1}\)? With Nesterov acceleration?
For strongly convex \(L\)-smooth functions, vanilla GD converges as \((1 - \mu/L)^k = (1 - 1/\kappa)^k\). To reduce error by \(e^{-1}\), we need \((1 - 1/1000)^k \approx e^{-k/1000} = e^{-1}\), giving \(k \approx 1000\) iterations. Nesterov acceleration improves this to \((1 - 1/\sqrt{\kappa})^k\), needing \(k \approx \sqrt{1000} \approx 32\) iterations. That is a ~30× speedup — which is why momentum matters enormously for ill-conditioned problems.
5
Back to the scenario: the loss has plateaued with a reasonable learning rate. Is the issue likely a local minimum, a saddle point, or ill-conditioning? How does momentum help each case?
In high-dimensional loss surfaces, plateaus are more often caused by saddle points or ill-conditioning than true local minima. At a saddle point, the gradient is near zero in most directions but negative curvature exists along some direction. Ill-conditioning causes zigzag behaviour that appears as stagnation. Momentum helps both: (1) at saddle points, accumulated velocity from before the plateau carries the iterate through the flat region, (2) for ill-conditioning, momentum damps oscillations across narrow dimensions and amplifies progress along the long dimension. It is less helpful for true sharp local minima, where stochastic noise from mini-batching may be more effective at escaping.
