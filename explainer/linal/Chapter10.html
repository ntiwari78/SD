10.1

Markov Chains & Steady States

A Markov chain models a system that jumps between states randomly, where each transition depends only on the current state — not the history. The key matrix is the transition matrix $P$, and repeated multiplication by $P$ drives the state distribution toward a steady state.

Imagine a frog that hops between lily pads. Each day, the frog jumps to a new pad with probabilities that depend only on where it currently sits — not how it got there. Track the probability vector $\mathbf{x}_k$ describing where the frog is on day $k$. Multiplying by the transition matrix $P$ advances time by one day: $\mathbf{x}_{k+1}=P\mathbf{x}_k$. Eventually, the frog's distribution stabilizes — it reaches the steady state — no matter where it started.

STOCHASTIC MATRIX & MARKOV CHAIN

A matrix $P$ is a stochastic matrix (transition matrix) if:

  • All entries $p_{ij}\geq 0$ (probabilities)
  • Each column sums to 1: $\sum_i p_{ij}=1$ for all $j$

Entry $p_{ij}$ is the probability of transitioning from state $j$ to state $i$. A state vector $\mathbf{x}$ has non-negative entries summing to 1. Evolution: $\mathbf{x}_{k+1}=P\mathbf{x}_k$, so $\mathbf{x}_k=P^k\mathbf{x}_0$.

STEADY-STATE VECTOR

A state vector $\mathbf{q}$ is a steady state if $P\mathbf{q}=\mathbf{q}$, i.e., $\mathbf{q}$ is an eigenvector of $P$ with eigenvalue $\lambda=1$. Equivalently, $\mathbf{q}$ solves $(I-P)\mathbf{q}=\mathbf{0}$ with $\|\mathbf{q}\|_1=1$.

PERRON-FROBENIUS THEOREM (informal)

If $P$ is a regular stochastic matrix (some power $P^k$ has all positive entries), then:

  • $P$ has a unique steady-state vector $\mathbf{q}$ with all positive entries
  • For any initial state $\mathbf{x}_0$: $\lim_{k\to\infty}P^k\mathbf{x}_0=\mathbf{q}$
  • $\lambda=1$ is a simple eigenvalue; all other eigenvalues satisfy $|\lambda|<1$
Example — Brand Loyalty Model. Brands A and B: each month, 80% of A customers stay, 20% switch to B; 30% of B customers switch to A, 70% stay. $$P=\begin{pmatrix}0.8&0.3\\0.2&0.7\end{pmatrix}$$

Steady state: $(I-P)\mathbf{q}=\mathbf{0}$. The equations give $0.2q_A=0.3q_B$, with $q_A+q_B=1$. Solving: $q_A=0.6,\,q_B=0.4$. Regardless of initial market share, A will capture 60% and B will hold 40% in the long run.

A stochastic matrix always has eigenvalue $\lambda=1$. Why?
Show Answer

The vector $\mathbf{1}=(1,1,\ldots,1)^T$ satisfies $P^T\mathbf{1}=\mathbf{1}$ (since each column of $P$ sums to 1, each row of $P^T$ sums to 1). So $\lambda=1$ is an eigenvalue of $P^T$, and since $\det(\lambda I-P)=\det(\lambda I-P^T)$, it's also an eigenvalue of $P$.

10.2

Graph Theory & Network Analysis

Every network — social, neural, transportation, communication — can be encoded as a matrix. The adjacency matrix $A$ of a graph stores connection information, and its powers $A^k$ count walks of length $k$ between nodes. Eigenvalues of $A$ reveal deep structural properties.

Label every person in a social network as a row/column in a matrix. Put a 1 in position $(i,j)$ if person $i$ and $j$ are friends. Matrix multiplication then does something magical: the $(i,j)$ entry of $A^2$ counts the number of mutual friends between $i$ and $j$ — i.e., paths of length 2 connecting them. $A^k$ counts paths of length $k$. The entire connectivity structure of the network is encoded in the eigenvalues of $A$.

ADJACENCY MATRIX

For a graph $G$ with $n$ vertices, the adjacency matrix $A$ is an $n\times n$ matrix where:

$$a_{ij}=\begin{cases}1 & \text{if there is an edge from vertex }j\text{ to vertex }i\\0 & \text{otherwise}\end{cases}$$

For undirected graphs, $A$ is symmetric. The $(i,j)$ entry of $A^k$ equals the number of walks of length $k$ from vertex $j$ to vertex $i$.

SPECTRAL PROPERTIES OF GRAPHS

  • Perron root: The largest eigenvalue $\lambda_1$ (spectral radius) satisfies $\lambda_1\leq\max_i\deg(i)$
  • Bipartite: $G$ is bipartite $\iff$ eigenvalues are symmetric about 0 ($\lambda$ eigenvalue $\Rightarrow -\lambda$ eigenvalue)
  • Connectivity: $G$ is connected $\iff$ $\lambda_1$ has multiplicity 1
  • Cheeger inequality: The second-smallest Laplacian eigenvalue $\lambda_2$ (algebraic connectivity) quantifies how well-connected the graph is
Example — Triangle Graph. Three vertices $\{1,2,3\}$ with edges $1\text{–}2$, $2\text{–}3$, $1\text{–}3$: $$A=\begin{pmatrix}0&1&1\\1&0&1\\1&1&0\end{pmatrix}$$

$A^2=\begin{pmatrix}2&1&1\\1&2&1\\1&1&2\end{pmatrix}$: diagonal entries = 2 (each vertex has 2 paths of length 2 back to itself, via the other two vertices). The eigenvalues are $\lambda_1=2$ (dominant) and $\lambda_2=\lambda_3=-1$.

Total walks of length 2: $\text{tr}(A^2)=6=$ twice the number of edges (3 edges). This is a general identity: $\text{tr}(A^2)=2|E|$.

The Graph Laplacian

LAPLACIAN MATRIX

The Laplacian of an undirected graph is $L=D-A$, where $D$ is the diagonal degree matrix ($d_{ii}=\deg(i)$). Properties:

  • $L$ is symmetric positive semidefinite
  • $L\mathbf{1}=\mathbf{0}$ (always has eigenvalue 0 with eigenvector $\mathbf{1}$)
  • $\mathbf{x}^TL\mathbf{x}=\sum_{(i,j)\in E}(x_i-x_j)^2$ — measures "smoothness" across edges
  • Number of connected components = multiplicity of eigenvalue 0

10.3

Leontief Input-Output Models

Wassily Leontief won the 1973 Nobel Prize in Economics for applying linear algebra to model entire economies. His input-output model tracks how industries consume each other's outputs — a system of linear equations that determines the production levels needed to meet any final demand.

Imagine a simplified economy: Steel needs coal to be made; coal needs steel (for mining equipment). Set up a matrix tracking how much of each sector's output goes to every other sector. The remaining output goes to consumers (final demand). Given a consumer demand vector $\mathbf{d}$, how much must each sector produce? This is the fundamental question Leontief answered with $\mathbf{x}=(I-C)^{-1}\mathbf{d}$ — an inverse problem solvable only with linear algebra.

LEONTIEF MODEL

Let $n$ industries produce outputs $\mathbf{x}=(x_1,\ldots,x_n)^T$. The consumption matrix $C$ has entry $c_{ij}=$ units of good $i$ needed to produce 1 unit of good $j$. Final (consumer) demand is $\mathbf{d}$. The balance equation is:

$$\underbrace{\mathbf{x}}_{\text{total output}}=\underbrace{C\mathbf{x}}_{\text{inter-industry}}+\underbrace{\mathbf{d}}_{\text{final demand}}$$

Rearranging: $(I-C)\mathbf{x}=\mathbf{d}$. If $(I-C)$ is invertible, the Leontief solution is:

$$\mathbf{x}=(I-C)^{-1}\mathbf{d}$$

$(I-C)^{-1}$ is called the Leontief inverse or total requirements matrix.

EXISTENCE THEOREM (Productive Economy)

The Leontief system $(I-C)\mathbf{x}=\mathbf{d}$ has a non-negative solution $\mathbf{x}\geq\mathbf{0}$ for every non-negative demand $\mathbf{d}\geq\mathbf{0}$ if and only if the economy is productive: there exists $\mathbf{x}\geq\mathbf{0}$ such that $\mathbf{x}>C\mathbf{x}$ (i.e., each industry can produce a surplus beyond its own needs). Equivalently, all eigenvalues of $C$ have magnitude less than 1.

When productive: $(I-C)^{-1}=I+C+C^2+C^3+\cdots$ (Neumann series), where $C^k\mathbf{d}$ represents "indirect demand" through $k$ rounds of production.

Example — 3-Sector Economy. Agriculture (A), Manufacturing (M), Services (S): $$C=\begin{pmatrix}0.1&0.3&0.2\\0.2&0.1&0.3\\0.1&0.1&0.1\end{pmatrix}, \quad \mathbf{d}=\begin{pmatrix}50\\30\\20\end{pmatrix}$$

Solve $(I-C)\mathbf{x}=\mathbf{d}$ using Gaussian elimination on the augmented matrix. The solution gives the total production required from each sector to meet consumer demand, including all intermediate needs cascading through the economy.

Interpretation: If manufacturing demand suddenly spikes, the Leontief inverse immediately quantifies the needed increase in steel, agriculture, and energy to support it.

A consumption matrix $C$ has $c_{ij}=0.5$ for all $i,j$ in a 2-sector economy. Is this economy productive?
Show Answer

No! Column sums of $C$ are $0.5+0.5=1.0$. For productivity, each sector must produce more than it consumes, requiring column sums strictly less than 1. With column sum $=1$, the sector produces exactly what it consumes — nothing is left for final demand. $(I-C)$ is singular and $(I-C)^{-1}$ doesn't exist.

10.4

Least Squares: Fitting Models to Data

Real data is never perfectly consistent. When $A\mathbf{x}=\mathbf{b}$ has no exact solution (overdetermined system — more equations than unknowns), the least squares approach finds the $\hat{\mathbf{x}}$ that makes $\|A\hat{\mathbf{x}}-\mathbf{b}\|$ as small as possible — the best approximate solution.

You have 100 data points and want to fit a line $y=mx+b$. Writing each data point as one equation gives 100 equations in 2 unknowns — a wildly overdetermined system with no exact solution. But geometry saves us: the closest point to $\mathbf{b}$ that lies in the column space of $A$ is the orthogonal projection. The least-squares solution finds this projection automatically.

NORMAL EQUATIONS

The least-squares solution to $A\mathbf{x}=\mathbf{b}$ (where $A$ is $m\times n$ with $m>n$) satisfies the normal equations:

$$A^TA\hat{\mathbf{x}}=A^T\mathbf{b}$$

If $A$ has full column rank, then $A^TA$ is invertible and:

$$\hat{\mathbf{x}}=(A^TA)^{-1}A^T\mathbf{b}=A^+\mathbf{b}$$

The residual $\mathbf{r}=\mathbf{b}-A\hat{\mathbf{x}}$ is orthogonal to $\text{col}(A)$: $A^T\mathbf{r}=\mathbf{0}$.

Geometric Interpretation. The least squares solution projects $\mathbf{b}$ orthogonally onto $\text{col}(A)$: $$\hat{\mathbf{b}}=A\hat{\mathbf{x}}=A(A^TA)^{-1}A^T\mathbf{b}=P\mathbf{b}$$ where $P=A(A^TA)^{-1}A^T$ is the projection matrix onto $\text{col}(A)$. Note $P^2=P$ (idempotent) and $P^T=P$ (symmetric) — the defining properties of an orthogonal projection.
Example — Linear Regression. Data points $(1,1),(2,3),(3,2),(4,5)$. Fit $y=\beta_0+\beta_1 x$: $$A=\begin{pmatrix}1&1\\1&2\\1&3\\1&4\end{pmatrix}, \quad \mathbf{b}=\begin{pmatrix}1\\3\\2\\5\end{pmatrix}$$

$A^TA=\begin{pmatrix}4&10\\10&30\end{pmatrix}$, $A^T\mathbf{b}=\begin{pmatrix}11\\31\end{pmatrix}$. Normal equations give $\hat{\beta}_0=0.5$, $\hat{\beta}_1=1.0$. Best fit line: $y=0.5+x$. Residual sum of squares: $\|\mathbf{b}-A\hat{\mathbf{x}}\|^2=1$.

Polynomial and General Curve Fitting

GENERAL LEAST SQUARES FITTING

To fit any linear model $y=\beta_0\phi_0(x)+\beta_1\phi_1(x)+\cdots+\beta_k\phi_k(x)$ to data $(x_1,y_1),\ldots,(x_m,y_m)$:

Build the design matrix $A$ with $a_{ij}=\phi_{j-1}(x_i)$. Solve the normal equations $A^TA\hat{\boldsymbol{\beta}}=A^T\mathbf{y}$. The basis functions $\phi_j$ can be polynomials $\{1,x,x^2,\ldots\}$, Fourier functions $\{\sin,\cos,\ldots\}$, or any other set.

Note: For numerical stability (especially high-degree polynomials), prefer QR decomposition or SVD over directly solving the normal equations.

Linear Regression is Just Least Squares. Every scikit-learn LinearRegression, every PyTorch loss minimization, every neural network weight update is, at its core, a least-squares solve or its gradient. Ridge regression adds $\lambda\|\boldsymbol{\beta}\|^2$ to the loss, turning the normal equations into $(A^TA+\lambda I)\hat{\boldsymbol{\beta}}=A^T\mathbf{b}$ — a regularized system that is always invertible (even when $A^TA$ is singular), as $\lambda>0$ raises all eigenvalues above zero.
You have an overdetermined system $A\mathbf{x}=\mathbf{b}$ with $A$ a $10\times 3$ matrix of rank 3. Is the least squares solution unique?
Show Answer

Yes, unique! Since $A$ has rank 3 = full column rank, $A^TA$ is a $3\times 3$ invertible matrix. The normal equations $A^TA\hat{\mathbf{x}}=A^T\mathbf{b}$ have a unique solution $\hat{\mathbf{x}}=(A^TA)^{-1}A^T\mathbf{b}$. If $A$ had rank $r<3$ (rank-deficient), there would be infinitely many least-squares solutions; the one of minimum norm is $A^+\mathbf{b}$ via SVD pseudoinverse.

10.5

Computer Graphics & Homogeneous Coordinates

Matrices drive every pixel — from 2D game engines to GPU shaders

🎮
The Graphics Pipeline is Pure Linear Algebra. Every modern GPU takes a stream of vertex coordinates and multiplies them by a sequence of matrices: the model matrix places the object in the world, the view matrix moves to camera space, and the projection matrix maps 3D to 2D screen coordinates. At 60 fps rendering 10 million triangles, a GPU performs roughly $10^{12}$ floating-point operations per second — all matrix-vector multiplies.

Homogeneous Coordinates: Translation Becomes Linear

In standard 2D, a translation $\mathbf{x}\mapsto\mathbf{x}+\mathbf{t}$ is affine, not linear — it can't be written as matrix multiplication. The trick: embed $\mathbb{R}^2$ in $\mathbb{R}^3$ by appending a 1:

$$\begin{pmatrix}x\\y\end{pmatrix}\;\longrightarrow\;\begin{pmatrix}x\\y\\1\end{pmatrix}$$

Now every affine transformation (rotation, scaling, shear, translation) is a $3\times 3$ matrix multiply:

$$\underbrace{\begin{pmatrix}\cos\theta&-\sin\theta&t_x\\\sin\theta&\cos\theta&t_y\\0&0&1\end{pmatrix}}_{\text{rotate + translate}}\begin{pmatrix}x\\y\\1\end{pmatrix}=\begin{pmatrix}x\cos\theta-y\sin\theta+t_x\\x\sin\theta+y\cos\theta+t_y\\1\end{pmatrix}$$

The payoff: any chain of transformations composes into one matrix product — computed once on the GPU, applied to millions of vertices.

Standard 2D Homogeneous Matrices

Translation by $(t_x, t_y)$:

$$T=\begin{pmatrix}1&0&t_x\\0&1&t_y\\0&0&1\end{pmatrix}$$

Scaling by $(s_x, s_y)$:

$$S=\begin{pmatrix}s_x&0&0\\0&s_y&0\\0&0&1\end{pmatrix}$$

Rotation by $\theta$:

$$R=\begin{pmatrix}\cos\theta&-\sin\theta&0\\\sin\theta&\cos\theta&0\\0&0&1\end{pmatrix}$$

Horizontal shear by $k$:

$$H=\begin{pmatrix}1&k&0\\0&1&0\\0&0&1\end{pmatrix}$$

3D Graphics: One More Dimension, Same Idea. In 3D, vertices are $4\times 1$ homogeneous vectors $(x,y,z,1)^T$, and the full MVP (Model-View-Projection) transform is a single $4\times 4$ matrix product. Perspective projection maps $z$-depth to the $w$ component; after the GPU divides by $w$ (the "perspective divide"), far objects shrink and near objects enlarge — all from one matrix multiply per vertex.
Why is it important that all standard 2D graphics transforms can be represented as $3\times 3$ matrix multiplications?
Show Answer

Because matrix multiplication is associative: if you apply transforms $T_1, T_2, \ldots, T_k$ in sequence, the combined effect is $(T_k\cdots T_2 T_1)\mathbf{v}$. Pre-computing the product $M = T_k\cdots T_1$ once and then applying $M$ to millions of vertices is far faster than applying each $T_i$ separately. Hardware GPUs are optimized exactly for this SIMD (Single Instruction Multiple Data) matrix-vector pattern. Without homogeneous coordinates, translation would break associativity, forcing sequential application.
