🎁 The Gift Draft: When Taking Turns Creates Strategy

Core Analogy

Imagine you and a friend are running a fantasy football draft. You alternate picking players from a pool, and each player has different values to each of you. Here's the strategic tension:

  • Do you always draft the player YOU value most? This "greedy" approach seems logical, but it might not maximize your total value.
  • Should you "hate-draft" — taking someone less valuable to you but extremely valuable to your opponent? Sometimes yes!
  • Can you strategically leave items you know your opponent won't take? Absolutely — this forces them to waste picks.
Four Key Insights
  1. Greedy isn't always optimal: Taking your highest-value item every turn can backfire. Sometimes delaying lets you accumulate more total value by forcing your opponent into worse positions.
  2. Turn order matters enormously: Switching from ABABAB to ABBABA creates a completely different game with different optimal strategies and outcomes.
  3. Information is everything: Both players know all values — this is perfect information. You can calculate what your opponent will do.
  4. Bottom-up analysis is a clever shortcut: Determine choices from last to first. The item each player values least will be taken last. Remove it and repeat.

📐 Sequential Allocation and Backward Induction

Game Setup

We formalize the sequential item division game:

  • Items: $\{C, D, E, F, G\}$ (or similar set of $n$ items)
  • Values: $a(X)$ = Ann's value for item $X$, $b(X)$ = Beth's value for item $X$
  • Turn order: A string like ABABAB or ABBABA indicating who picks each round
  • Payoff: Sum of values of items a player obtains

Greedy Strategy & Its Limits

Greedy rule: Always pick the item YOU value most among remaining items.

Counter-example showing greedy fails:

Items: $\{C, D, E\}$

Ann's values: $a(C)=1, a(D)=2, a(E)=3$ (prefers E > D > C)

Beth's values: $b(C)=3, b(D)=1, b(E)=2$ (prefers C > E > D)

Turn order: ABABAB

Greedy play: Beth picks C (highest for her), Ann picks E (highest for her), Beth picks D.
Result: Beth gets C+D = 3+1 = 4, Ann gets E = 3.

Better for Beth: Beth picks E (her 2nd choice), Ann picks D, Beth picks C.
Result: Beth gets E+C = 2+3 = 5, Ann gets D = 2.

By strategically NOT being greedy, Beth gains an extra point!

Backward Induction & Gain Values

Backward Induction: Works on the game tree as in Chapter 8 — compute optimal payoffs from leaves upward.

Set Positions & Gain Digraph:

  • We only care about which items remain (not the order of past choices).
  • For each set of remaining items, the gain value tells how much additional value each player will accumulate from this point, assuming optimal play.
  • This is much more compact: 63 positions vs 1,237 for 6 items.
Bottom-Up Analysis (Ordinal Approach)

Key insight: No player voluntarily takes the item they rank lowest among remaining items.

Algorithm:

  1. Look at the last pick in the turn order.
  2. Find the item that last picker values least.
  3. Assign it to the last picker (since they're stuck with it).
  4. Remove that item and that pick.
  5. Repeat on the reduced set.

Remarkable property: This only needs ordinal rankings (your preference order), not cardinal values! As long as the preference ORDER doesn't change, the same items get selected. This makes the method robust to uncertainty about exact values.

Python Implementation Reference

Here's a complete Python class for computing game outcomes:

class ItemDivisionGame:
    def __init__(self, items, ann_values, beth_values, turn_order):
        self.items = items
        self.a = dict(zip(items, ann_values))
        self.b = dict(zip(items, beth_values))
        self.turns = turn_order  # string like "ABABAB"
        self.memo = {}

    def greedy_play(self):
        remaining = list(self.items)
        ann_items, beth_items = [], []
        for i, turn in enumerate(self.turns):
            vals = self.a if turn == 'A' else self.b
            best = max(remaining, key=lambda x: vals[x])
            remaining.remove(best)
            (ann_items if turn == 'A' else beth_items).append(best)
        return ann_items, beth_items

    def backward_induction(self, remaining=None, turn_idx=0):
        if remaining is None:
            remaining = frozenset(self.items)
        if turn_idx >= len(self.turns) or not remaining:
            return 0, 0, []

        key = (remaining, turn_idx)
        if key in self.memo:
            return self.memo[key]

        turn = self.turns[turn_idx]
        vals = self.a if turn == 'A' else self.b
        best_total, best_item = -float('inf'), None

        for item in remaining:
            new_remaining = remaining - {item}
            a_gain, b_gain, _ = self.backward_induction(new_remaining, turn_idx + 1)
            my_val = self.a[item] if turn == 'A' else self.b[item]
            my_total = my_val + (a_gain if turn == 'A' else b_gain)
            if my_total > best_total:
                best_total = my_total
                best_item = item

        new_remaining = remaining - {best_item}
        a_gain, b_gain, rest = self.backward_induction(new_remaining, turn_idx + 1)
        item_val_a = self.a[best_item] if turn == 'A' else 0
        item_val_b = 0 if turn == 'A' else self.b[best_item]
        result = (item_val_a + a_gain, item_val_b + b_gain,
                  [(turn, best_item)] + rest)
        self.memo[key] = result
        return result

    def bottom_up_analysis(self):
        remaining = set(self.items)
        assignments = {}
        for i in range(len(self.turns) - 1, -1, -1):
            turn = self.turns[i]
            vals = self.a if turn == 'A' else self.b
            worst = min(remaining, key=lambda x: vals[x])
            assignments[i] = (turn, worst)
            remaining.discard(worst)
        return assignments

    def compute_payoff(self, choices):
        ann_total = sum(self.a[item] for t, item in choices if t == 'A')
        beth_total = sum(self.b[item] for t, item in choices if t == 'B')
        return ann_total, beth_total

🤔 Fair Division Reflection

Engage with these questions to deepen your understanding:

Question 1: Why isn't greedy play always optimal? Give an intuitive explanation.
Answer: Taking your best item now might hand your opponent their second and third best items. By taking a strategically less valuable item (to you), you might force your opponent to waste a pick on something you'd have gotten anyway, letting you claim a better overall portfolio. This is a form of strategic denial: you give up a little value now to prevent your opponent from getting a lot of value.
Question 2: Bottom-up analysis only needs ordinal rankings (who's your 1st, 2nd, 3rd favorite), not exact values. Why is this remarkable?
Answer: It means the optimal strategy is robust to uncertainty about exact values. As long as the preference ORDER doesn't change, the same items get selected. This makes the method practical — you don't need precise valuations, just the ranking. In real applications like divorce settlements or estate division, exact values may be disputed, but relative preferences are often clear.
Question 3: In ABBABA, Beth gets two consecutive picks (positions 2, 3 and 5, 6). Does this always help Beth compared to ABABAB?
Answer: Not necessarily! Two consecutive picks let Beth take her top two remaining items, but it also gives Ann the last pick (more information). The net effect depends on the specific values. Sometimes ABABAB favors Beth more because she gets the final selection. Use the Turn Order Comparison tool to explore this empirically.
Question 4: The game digraph for 6-item ABABAB has 207 positions vs 1,237 in the tree. What causes this reduction, and when would it NOT help?
Answer: The reduction comes from merging positions with the same remaining items regardless of history. It helps when only the current state matters (which items are left). It wouldn't help if item values depended on which items you already own (synergies or substitutes). For example, a left shoe and right shoe are only valuable as a pair. In that case, you'd need a larger game state that tracks ownership history.
Question 5: Could this item division framework model a real-world situation? What assumptions might break?
Answer: Real examples: divorce settlements, estate division, draft picks, award ceremonies. Assumptions that might break: (a) players may not know each other's valuations (incomplete information), (b) items may have synergies (a pair of shoes is worth more than one shoe), (c) items may not be indivisible (you can split a bank account or split time), (d) there may be more than two players, (e) players may have budget constraints or can't carry everything.
