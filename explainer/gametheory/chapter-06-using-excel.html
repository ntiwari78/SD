Chapter 6: Using Excel

Theory — Computational Game Analysis

🖥️ The Analyst's Toolkit: When Games Get Too Big for Paper

Imagine you're a chess player who just discovered that your opponent isn't playing 3×3 tic-tac-toe anymore — they've upgraded to a 10×10 board with custom rules. Your napkin analysis won't cut it. You need a systematic tool that can crunch through hundreds of strategy pairs, highlight patterns, and find equilibria faster than you can say "dominated strategy."

Why Computers?

Game theory with bimatrices works beautifully on paper for 2×2 or 3×3 games. But real games (even the ones in this textbook) quickly scale to 7×7, 10×10, or 24×24 matrices. The operations (maximin, domination, best response, NE) are simple but tedious — perfect for automation.

The Historical Connection

Von Neumann & Morgenstern's foundational game theory book (1944) coincided with the birth of modern computers. Game theory and computation have been intertwined from the start — the RAND Corporation used both extensively during the Cold War to analyze deterrence, negotiation, and arms races.

What the Analysis Does

Four key spreadsheet "pages" (analysis views):

  • Maximin: Find each player's security strategy (maximize your minimum payoff)
  • Weak Domination: Identify strategies that are never optimal (weakly dominated)
  • Best Response: For each opponent strategy, find your optimal reply
  • Pure Nash: Find strategy pairs where BOTH players are playing best responses
Two-Player vs Three-Player

The book uses SIMnonzero.xlsx for general 2-player games (up to 21×21), SIMzero.xlsx for zero-sum games (where Beth's payoff = −Ann's), and SIM3.xlsx/SIM33.xlsx for 3-player games. Our interactive tool handles all of these directly in your browser.

📐 The Four Pillars of Game Analysis

Let's formalize each analysis operation. These are the mathematical foundations behind the spreadsheet calculations.

1. Maximin Analysis

For player $i$ with strategy set $S_i$, the maximin strategy is:

$$s_i^* = \arg\max_{s_i \in S_i} \min_{s_{-i} \in S_{-i}} u_i(s_i, s_{-i})$$

The security level (maximin value) that player $i$ can guarantee:

$$\underline{v}_i = \max_{s_i \in S_i} \min_{s_{-i} \in S_{-i}} u_i(s_i, s_{-i})$$

How to compute: For Ann, look at each row, find the minimum value in that row, then pick the row with the highest minimum. For Beth, look at each column, find the minimum value in that column, then pick the column with the highest minimum.

2. Weak Domination

Strategy $s_i$ weakly dominates strategy $s_i'$ if:

$$u_i(s_i, s_{-i}) \geq u_i(s_i', s_{-i}) \quad \forall s_{-i} \in S_{-i}$$

with strict inequality for at least one $s_{-i}$.

A weakly dominated strategy is shown in gray — it's never uniquely optimal, though it might tie for optimal against some opponent strategies.

3. Best Response

Strategy $s_i^*$ is a best response to $s_{-i}$ if:

$$u_i(s_i^*, s_{-i}) \geq u_i(s_i, s_{-i}) \quad \forall s_i \in S_i$$

Best responses are highlighted in green. A cell $(s_A, s_B)$ has Ann's payoff as a best response if no other row in column $s_B$ gives Ann a higher payoff. Similarly for Beth across rows.

4. Pure Nash Equilibrium

Strategy pair $(s_A^*, s_B^*)$ is a pure Nash equilibrium if:

$$s_A^* \in BR_A(s_B^*) \quad \text{and} \quad s_B^* \in BR_B(s_A^*)$$

Equivalently: a cell where BOTH players' payoffs are best responses simultaneously. Highlighted in gold.

Zero-Sum Shortcut

In zero-sum games, $u_B(s_A, s_B) = -u_A(s_A, s_B)$ for all strategy pairs, so we only need Ann's payoff matrix. Beth's payoff is computed automatically. This is why the book's SIMzero.xlsx only asks for Ann's values. By von Neumann's Minimax Theorem, Ann's maximin equals Beth's minimax.

Three-Player Extension

For 3 players with strategies $S_A, S_B, S_C$, the payoff space is 3-dimensional: $u_A, u_B, u_C : S_A \times S_B \times S_C \to \mathbb{R}^3$. A pure Nash equilibrium requires all three players to simultaneously play best responses:

$$s_A^* \in BR_A(s_B^*, s_C^*), \quad s_B^* \in BR_B(s_A^*, s_C^*), \quad s_C^* \in BR_C(s_A^*, s_B^*)$$

Finding these equilibria requires checking all $|S_A| \times |S_B| \times |S_C|$ outcomes for mutual best responses.

A = np.array(ann_payoffs) return cls(A, -A) def maximin(self): """Compute maximin strategies and security levels.""" ann_row_mins = self.A.min(axis=1) ann_maximin = ann_row_mins.max() ann_strategies = np.where(ann_row_mins == ann_maximin)[0] beth_col_mins = self.B.min(axis=0) beth_maximin = beth_col_mins.max() beth_strategies = np.where(beth_col_mins == beth_maximin)[0] return { 'ann': {'strategies': ann_strategies, 'security_level': ann_maximin}, 'beth': {'strategies': beth_strategies, 'security_level': beth_maximin} } def weakly_dominated(self): """Find weakly dominated strategies.""" ann_dom = set() for i in range(self.m): for j in range(self.m): if i != j: if np.all(self.A[j] >= self.A[i]) and np.any(self.A[j] > self.A[i]): ann_dom.add(i) beth_dom = set() for i in range(self.n): for j in range(self.n): if i != j: if np.all(self.B[:, j] >= self.B[:, i]) and np.any(self.B[:, j] > self.B[:, i]): beth_dom.add(i) return {'ann': ann_dom, 'beth': beth_dom} def best_responses(self): """Find all best response cells for each player.""" ann_br = set() for col in range(self.n): max_val = self.A[:, col].max() for row in range(self.m): if self.A[row, col] == max_val: ann_br.add((row, col)) beth_br = set() for row in range(self.m): max_val = self.B[row, :].max() for col in range(self.n): if self.B[row, col] == max_val: beth_br.add((row, col)) return {'ann': ann_br, 'beth': beth_br} def nash_equilibria(self): """Find all pure strategy Nash equilibria.""" br = self.best_responses() return br['ann'] & br['beth']

🤔 Computational Reflection

Test your understanding of game analysis, computation, and equilibrium with these challenging questions.

Why can't we just use brute force to find Nash equilibria for any game? ⌄

For 2 players with $m$ and $n$ strategies respectively, checking all pairs is $O(m \cdot n)$ — feasible for moderate sizes (even a 100×100 game needs only 10,000 checks). The problem becomes exponentially harder with more players.

For $k$ players each with $n$ strategies, the number of outcomes is $n^k$. A 3-player game with 10 strategies each has 1,000 outcomes to check. A 10-player game with just 3 strategies each has 59 million outcomes.

For mixed strategies, the problem becomes much harder — finding ALL mixed-strategy Nash equilibria is PPAD-complete (a complexity class suggesting it's computationally as hard as problems are likely to be without being NP-hard). Excel-style tools work well for moderate pure-strategy games, but break down quickly for mixed strategies or very large games.

In zero-sum games, why is Beth's payoff matrix redundant? ⌄

In a zero-sum game, $u_B(s_A, s_B) = -u_A(s_A, s_B)$ for all strategy pairs. Knowing Ann's payoff determines Beth's completely — if Ann gets 5, Beth gets -5. This is why the book's SIMzero.xlsx only asks you to enter Ann's payoff matrix.

This structure also creates a beautiful symmetry: von Neumann's Minimax Theorem says that Ann's maximin value equals Beth's minimax value (the minimum of the maximum payoffs Beth can force). In other words, Ann's security level equals Beth's security level in any zero-sum game — perfect balance.

Furthermore, if a strategy weakly dominates another for Ann, we can deduce the domination structure for Beth directly from Ann's matrix. This is why zero-sum games are mathematically elegant but also competitive and easier to analyze.

A strategy can be weakly dominated yet still appear in a Nash equilibrium. How is this possible? ⌄

Weak domination means strategy $s$ is dominated by $s'$ if $u_i(s', s_{-i}) \geq u_i(s, s_{-i})$ for ALL opponent strategies, with strict inequality for at least one. But the "strictly better" case might not occur at the opponent's equilibrium strategy.

Example: In a 2×2 game where Ann has strategies Up/Down and Beth has Left/Right, suppose:

  • Up vs Left: Ann gets 2, Beth gets 1
  • Up vs Right: Ann gets 3, Beth gets 3
  • Down vs Left: Ann gets 2, Beth gets 4
  • Down vs Right: Ann gets 2, Beth gets 2

Down weakly dominates Up because Down gives Ann 2 against both strategies, while Up gives Ann 2 or 3. The "strictly better" case (2 vs 3) happens only against Right. If the equilibrium happens to be (Up, Left) where Ann gets exactly 2, then Up is optimal in equilibrium despite being weakly dominated overall. This can happen if Beth is "avoiding" Right because she also prefers Left.

The book mentions the QUATRO-UNO game has a 24×24 payoff matrix — how could we analyze such large games? ⌄

A 24×24 game has 576 outcomes — still checkable by computer in milliseconds, but too large for hand calculation. For even larger games, several approaches work:

  • Iterated Elimination: First remove all weakly dominated strategies, shrinking the matrix. Repeat until no more strategies can be removed. This often reduces a huge game to a tiny one.
  • Support Enumeration: Instead of checking all outcomes, assume Nash equilibrium involves only a subset of strategies (the "support"). Check if this subset can form an equilibrium. Much faster when the support is small.
  • The Lemke-Howson Algorithm: For 2-player games, this algorithm directly computes all Nash equilibria by solving systems of inequalities. Much faster than brute force.
  • Linear Programming: For zero-sum games, finding the maximin strategy reduces to a linear program, solvable in polynomial time.
  • Modern Solvers: Software like Gambit (open-source) can handle very large games using sophisticated algorithms. These tools do what Excel can't.
For 3-player games, NE requires three simultaneous 'yes' votes. Why do more players make equilibria harder to find yet often more plentiful? ⌄

Hardness: With 2 players, we check if both are playing best responses to each other. With 3 players, each player's best response depends on what BOTH opponents are doing simultaneously. The strategy space grows exponentially: $|S_A| \times |S_B| \times |S_C|$ instead of $|S_A| \times |S_B|$. A 10×10×10 game has 1,000 outcomes versus 100 for a 10×10 game.

Plentitude: Counterintuitively, 3-player games often have MORE equilibria than 2-player games (though not always). With more players, there are more ways for mutual best responses to align. A payoff structure that is unbalanced for 2 players might become more "coordinated" with a third player. Additionally, mixed-strategy equilibria become more common with more players, and these are not our main focus here.

Example: The 2×2×2 matching game (each player tries to match the other two) has 4 pure Nash equilibria (one for each of the four ways the players can all choose the same action pair). A comparable 2-player game would typically have fewer.
