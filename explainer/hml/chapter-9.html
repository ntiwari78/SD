Introduction to Artificial Neural Networks

Neural networks are sophisticated function approximators inspired by biological neurons. Rather than rigid algorithms, they learn hierarchical representations of data through layers of interconnected units. Each layer transforms data—from pixels to edges to shapes to objects—building increasingly abstract representations that enable powerful pattern recognition and prediction.

From Biological to Artificial Neurons

The journey from biological inspiration to artificial neural networks spans several decades of research and innovation.

Historical Timeline

📚 Key Milestones

1957: Frank Rosenblatt invents the Perceptron, the first machine learning algorithm that could learn linearly separable patterns. It could adjust weights based on errors.

1969: Minsky and Papert show that perceptrons cannot learn XOR (the limitation that caused the "AI winter").

1986: Rumelhart, Hinton, and Williams revive neural networks with the backpropagation algorithm, enabling multi-layer networks.

2006: Geoffrey Hinton's deep learning breakthrough using unsupervised pretraining launches the modern deep learning era.

Biological vs. Artificial Neurons

Key Insight

Biological neurons are complex and slow (firing at ~100 Hz). Artificial neurons are simplified mathematical functions that we can train to learn patterns. Inspiration ends at the concept level—implementation is pure mathematics.

The Perceptron: Foundation of Neural Networks

The perceptron is the simplest neural network unit. It takes weighted inputs, sums them, adds a bias, and applies an activation function.

Mathematical Definition

The output of a perceptron is:

$$h_{\mathbf{w}}(\mathbf{x}) = \text{step}(\mathbf{w}^T\mathbf{x} + b)$$

where:

  • w are the weights
  • x are the inputs
  • b is the bias
  • step() is the activation function (outputs 0 or 1)

Learning Rule

The perceptron learns by adjusting weights based on prediction errors:

$$w_{i,j}^{(\text{next})} = w_{i,j} + \eta(y_j - \hat{y}_j)x_i$$

where η is the learning rate, y is the true label, and ŷ is the prediction.

The XOR Problem

Fundamental Limitation

The perceptron cannot learn XOR because XOR is not linearly separable. The decision boundary must be curved or non-convex, which a single perceptron cannot achieve. This led to the "AI winter." The solution? Add hidden layers!

Multi-Layer Perceptron (MLP) Architecture

By stacking multiple layers of neurons, we gain the power to learn any continuous function (universal approximation theorem). Hidden layers learn intermediate representations that make the final classification easy.

Network Architecture

Typical MLP structure:

  • Input layer: One neuron per feature
  • Hidden layers: Typically 1-5 layers, with 10-1000 neurons per layer
  • Output layer: 1 neuron (regression/binary), K neurons (K-class classification)
  • Activation: ReLU (hidden), Linear (regression) or Softmax (classification)

Forward Propagation

Information Flow

In forward propagation, data flows from input → hidden layer 1 → hidden layer 2 → ... → output. Each layer computes:

$$h^{(l)} = \text{activation}(W^{(l)} h^{(l-1)} + b^{(l)})$$

The activation function introduces nonlinearity, enabling the network to learn complex patterns.

Activation Functions: The Heart of Neural Networks

Activation functions introduce nonlinearity. Without them, stacking layers would be useless—the result would still be just a linear transformation.

Common Activation Functions

Sigmoid Tanh ReLU ELU GELU

Sigmoid

$$\sigma(z) = \frac{1}{1+e^{-z}}$$

Range: (0, 1). Historically popular but saturates easily.

Tanh

$$\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}$$

Range: (-1, 1). Better than sigmoid (centered at 0).

ReLU

$$\text{ReLU}(z) = \max(0, z)$$

Fast, no saturation for z>0. The modern default.

Leaky ReLU

$$\text{LeakyReLU}(z) = \max(\alpha z, z)$$

Allows small gradients when z<0. Prevents dead neurons.

ELU

$$\text{ELU}_\alpha(z) = \begin{cases} \alpha(e^z - 1) & \text{if } z < 0 \\ z & \text{if } z \geq 0 \end{cases}$$

Smooth, outputs can be negative. Reduces vanishing gradient.

GELU

$$\text{GELU}(z) = z\Phi(z)$$

Modern activation (used in transformers). Smooth and effective.

Why ReLU Dominates

Practical Advantages

  • No saturation: For positive inputs, gradient is always 1 (no vanishing gradients)
  • Computationally efficient: Just a max operation, very fast
  • Empirically works well: Deep networks with ReLU train faster than sigmoid/tanh
  • Hardware acceleration: Many GPUs have ReLU-specific optimizations

Backpropagation: How Neural Networks Learn

Backpropagation is the algorithm that enables neural networks to learn. It computes gradients by applying the chain rule to propagate error signals backward through the network.

The Three Steps

1. Forward Pass

Compute predictions for all samples. Store intermediate activations for later use.

2. Compute Loss

Compare predictions to true labels. Common losses: MSE (regression), Cross-entropy (classification).

3. Backward Pass

Apply chain rule to compute gradients for all weights. Update weights in direction of negative gradient.

Chain Rule in Backpropagation

Gradient Computation

For a weight connecting layer l-1 to layer l, the gradient is:

$$\frac{\partial L}{\partial w^{(l)}} = \frac{\partial L}{\partial z^{(l)}} \cdot \frac{\partial z^{(l)}}{\partial w^{(l)}}$$

where the chain rule propagates the gradient back layer-by-layer.

Regression and Classification MLPs

Regression (Continuous Output)

Output Layer: Single linear neuron

Loss Function: Mean Squared Error (MSE)

$$L = \frac{1}{m} \sum_{i=1}^{m} (y_i - \hat{y}_i)^2$$

Example: Predicting house prices, stock prices, temperature.

Binary Classification

Output Layer: Single sigmoid neuron

Loss Function: Binary Cross-Entropy

$$L = -[y \log(\hat{y}) + (1-y) \log(1-\hat{y})]$$

Example: Email spam detection (spam/not spam), disease detection (positive/negative).

Multiclass Classification

Output Layer: K softmax neurons (one per class)

Loss Function: Categorical Cross-Entropy

$$L = -\sum_{k=1}^{K} y_k \log(\hat{y}_k)$$

Example: MNIST digit recognition (10 classes), image classification (1000 classes).

Building Neural Networks: Scikit-Learn & PyTorch

Scikit-Learn (Simple, Fast)

from sklearn.neural_network import MLPClassifier, MLPRegressor # Classification mlp_clf = MLPClassifier( hidden_layer_sizes=(100, 50), # 2 hidden layers activation='relu', solver='adam', # Adaptive optimizer max_iter=1000, random_state=42 ) mlp_clf.fit(X_train, y_train) accuracy = mlp_clf.score(X_test, y_test) # Regression mlp_reg = MLPRegressor( hidden_layer_sizes=(100, 50), activation='relu', solver='adam', max_iter=1000 ) mlp_reg.fit(X_train, y_train) mse = ((mlp_reg.predict(X_test) - y_test) ** 2).mean()

PyTorch (Flexible, Production-Ready)

import torch import torch.nn as nn import torch.optim as optim # Define model model = nn.Sequential( nn.Flatten(), nn.Linear(784, 300), # 28*28 = 784 inputs nn.ReLU(), nn.Linear(300, 100), nn.ReLU(), nn.Linear(100, 10) # 10 output classes ) # Define loss and optimizer criterion = nn.CrossEntropyLoss() optimizer = optim.Adam(model.parameters(), lr=0.001) # Training loop for epoch in range(10): for X_batch, y_batch in train_loader: # Forward pass y_pred = model(X_batch) loss = criterion(y_pred, y_batch) # Backward pass optimizer.zero_grad() loss.backward() optimizer.step() # Evaluation model.eval() with torch.no_grad(): y_pred = model(X_test) accuracy = (y_pred.argmax(1) == y_test).float().mean().item()

⚔️ War Room: Debugging Deep Learning

You built a 10-layer neural network for image classification. After 5 epochs, the training loss is not decreasing, and validation accuracy is stuck at 15% (random for 10-class problem).

What could be wrong? ▼

❓ Question 1: Could it be weight initialization?

Deep networks are sensitive to initialization. If weights are too large, activations saturate. If too small, gradients vanish. Use Xavier/Glorot or He initialization for ReLU.

❓ Question 2: Could it be learning rate?

A learning rate that's too high causes instability. Too low causes training to stall. Start with Adam (lr=0.001) which handles this better than SGD.

❓ Question 3: Could it be vanishing gradients?

In deep networks with sigmoid/tanh, gradients shrink exponentially backward. Layers near the input barely learn. Use ReLU and batch normalization to fix.

❓ Question 4: Could it be data problems?

Check if data is normalized to zero mean and unit variance. Check for class imbalance. Try a simple logistic regression baseline first—if it fails, the problem is the data.

❓ Question 5: Could it be architecture?

10 layers might be overkill for simple problems. Start with 2-3 hidden layers. Verify output layer matches problem (1 neuron for binary, K for K-class).

🔍 Debugging Checklist

✓ Start simple: 2-layer network on 1% of data

✓ Check data: visualize samples, check shapes, verify normalization

✓ Monitor metrics: plot training/validation loss and accuracy

✓ Check gradients: ensure they're not exploding/vanishing

✓ Try baselines: logistic regression, decision tree

✓ Hyperparameter sweep: learning rate, batch size, hidden size

Key Takeaways

What You've Learned

  • Neural networks are universal function approximators composed of layers of interconnected neurons
  • Single neurons (perceptrons) can only learn linearly separable patterns
  • Multiple layers and nonlinear activations enable learning of arbitrary patterns
  • Backpropagation efficiently computes gradients for all weights using the chain rule
  • Different architectures suit different tasks: regression (linear out), binary (sigmoid), multiclass (softmax)
  • Activation functions like ReLU are critical for enabling deep networks
  • Both scikit-learn and PyTorch provide practical implementations for different use cases
