Chapter 5: Decision Trees Part I: Fundamentals

Decision Trees

Flowcharts that learn from data. Decision trees ask yes/no questions about features to split data into pure groups. They are simple, interpretable, and powerful—but prone to overfitting without constraints.

Feynman Mental Model: The Flowchart Learner

Imagine you're a doctor diagnosing a patient. You ask questions:

"Is the petal width less than 0.8 cm?" If yes → likely Setosa flower. Stop.

If no, you ask another: "Is the petal width less than 1.75 cm?" If yes → likely Versicolor. If no → likely Virginica.

That's a decision tree! It recursively partitions data by asking binary questions about features, creating a flowchart-like structure. Each node tests a feature; each leaf gives a prediction. The algorithm learns these questions from data by minimizing impurity.

CART Algorithm & Gini Impurity

The CART (Classification And Regression Tree) algorithm recursively selects splits that minimize impurity. The most common impurity measure is Gini impurity.

Gini Impurity Formula

$$G_i = 1 - \sum_{k=1}^{n} p_{i,k}^2$$

Where p_{i,k} is the proportion of class k samples in node i.

CART Cost Function (Classification)

$$J(k, t_k) = \frac{m_{\text{left}}}{m}G_{\text{left}} + \frac{m_{\text{right}}}{m}G_{\text{right}}$$

The algorithm searches for feature k and threshold t_k that minimize the weighted sum of Gini impurities in the resulting child nodes.

Entropy (Alternative Impurity Measure)

$$H_i = -\sum_{\substack{k=1 \\ p_{i,k} \neq 0}}^{n} p_{i,k}\log_2(p_{i,k})$$

Entropy is another impurity measure. Scikit-Learn supports both; Gini is the default and typically faster.

Regularization & Pruning

Without constraints, decision trees grow unbounded and overfit. Scikit-Learn provides hyperparameters to control tree growth:

Key Regularization Hyperparameters

  • max_depth: Maximum depth of tree. Lower = simpler model.
  • min_samples_split: Minimum samples to split a node. Higher = fewer splits.
  • min_samples_leaf: Minimum samples in a leaf node. Higher = smoother boundaries.
  • max_leaf_nodes: Maximum number of leaf nodes. Limits complexity globally.
  • max_features: Number of features to consider at each split. Adds randomness.
1
Effect: As min_samples_leaf increases, decision boundaries become smoother and less prone to overfitting. Trade-off: might underfit if too high.

Regression Trees (DecisionTreeRegressor)

Decision trees can also perform regression. Instead of predicting class labels, they predict continuous values. The split criterion is MSE (Mean Squared Error) rather than Gini.

Regression Cost Function

$$J(k, t_k) = \frac{m_{\text{left}}}{m} \text{MSE}_{\text{left}} + \frac{m_{\text{right}}}{m} \text{MSE}_{\text{right}}$$

$$\text{MSE}_{\text{node}} = \frac{\sum_{i \in \text{node}} (\hat{y}_{\text{node}} - y^{(i)})^2}{m_{\text{node}}}$$

2
Piecewise Constant Predictions: Regression trees predict constant values in each region, creating a step-like function. Deeper trees fit training data better but often overfit.

Sensitivity to Training Data: High Variance

Small changes to training data can produce completely different trees. This is the high variance problem of decision trees.

Production Code: Scikit-Learn Decision Trees

Classification Example (Iris Dataset)

from sklearn.datasets import load_iris from sklearn.tree import DecisionTreeClassifier # Load data iris = load_iris(as_frame=True) X_iris = iris.data[["petal length (cm)", "petal width (cm)"]].values y_iris = iris.target # Train tree with max_depth=2 tree_clf = DecisionTreeClassifier(max_depth=2, random_state=42) tree_clf.fit(X_iris, y_iris) # Make predictions prediction = tree_clf.predict([[5, 1.5]]) # → [1] (Versicolor) # Class probabilities probs = tree_clf.predict_proba([[5, 1.5]]) # → [[0., 0.907, 0.093]] (91% confidence in class 1)

Scenario: Overfitting Crisis

Your decision tree classifier achieves 100% training accuracy but only 62% test accuracy on a customer churn dataset with 50 features.

Questions to diagnose:

  • Why is there such a large gap between training and test accuracy?
  • Which hyperparameters would you adjust first? Why?
  • When should you prefer Gini over Entropy?
  • Why are decision trees inherently unstable?
  • How does this motivate the next chapter (Random Forests)?

Analysis & Solutions

1. Overfitting Gap: The model has learned training data quirks and noise. With 50 features, the tree likely creates very deep, complex splits specific to training samples.

2. Hyperparameter Priority:

  • Start with max_depth: Reduce from unlimited to 5-10. Deep trees overfit.
  • Then min_samples_split/leaf: Increase to 10-20. Forces broader, more generalizable splits.
  • max_leaf_nodes: Limit globally (e.g., 31 nodes → depth ~5).
  • max_features: Try "sqrt" or "log2" to reduce feature overfitting.

3. Gini vs. Entropy: Gini is faster (no logarithm) and equally effective. Choose Gini unless you have a specific reason for entropy. For this case, Gini is preferred.

4. High Variance: Decision trees are unstable—small data changes create large tree changes. This is inherent to the CART algorithm's greedy splitting strategy.

5. Ensemble Solution: Train 100 trees on random data subsets (bagging) or with feature subsampling → Random Forest. Average predictions → lower variance without losing predictive power.

Key Takeaways

  • Decision trees are interpretable: Easy to visualize and explain to non-technical stakeholders.
  • CART algorithm: Recursively minimizes impurity (Gini or Entropy) to find optimal splits.
  • Prone to overfitting: Without regularization (max_depth, min_samples_leaf, etc.), trees fit noise.
  • High variance: Small data changes → very different trees. This motivates ensembles.
  • Works for both classification and regression: Use criterion="gini" or "entropy" for classification; "squared_error" for regression.
  • No feature scaling needed: Trees are invariant to monotonic transformations.
  • Axis-aligned splits: Trees create orthogonal decision boundaries. Rotated data may require deeper trees.
