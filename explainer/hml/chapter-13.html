Chapter 13: Processing Sequences Using RNNs and CNNs

Welcome to sequential data processing! This chapter explores how neural networks understand and predict sequences—from time series forecasting to natural language processing.

Feynman Mental Model

Think of RNNs as reading a book one word at a time, remembering what came before as you read each new word. They maintain a "hidden state"—your current understanding—that updates with each word, letting them understand context and predict what comes next. Just as you'd predict the next word of a sentence, RNNs predict the next value in a sequence.

Recurrent Neurons and Layers

Unlike feedforward networks where data flows in one direction, RNNs have feedback connections. Each neuron receives not only current inputs but also its own previous output, creating a loop.

Why Recurrence?

Recurrence allows the network to "remember" previous information. This is essential for sequences where context matters. The order of words in a sentence matters; the output at time step 10 depends on all previous inputs.

Simple RNN: The Core Concept

A recurrent neuron at time step $t$ computes:

$$\mathbf{h}^{(t)} = \sigma(\mathbf{W}_{hh} \cdot \mathbf{h}^{(t-1)} + \mathbf{W}_{xh} \cdot \mathbf{x}^{(t)} + \mathbf{b})$$

Where:

  • $\mathbf{h}^{(t)}$ = hidden state at time step $t$
  • $\mathbf{h}^{(t-1)}$ = hidden state from previous step (memory)
  • $\mathbf{x}^{(t)}$ = input at current time step
  • $\mathbf{W}_{hh}$, $\mathbf{W}_{xh}$ = weight matrices
  • $\sigma$ = activation function (tanh or ReLU)

Unrolling an RNN Through Time

RNNs are typically visualized as "unrolled" through time, showing how the same weights process each time step sequentially.

Backpropagation Through Time (BPTT)

Training RNNs requires a modified backpropagation algorithm called Backpropagation Through Time (BPTT). Instead of computing gradients through layers, we compute them through time steps.

⚙️
BPTT Process
  1. Forward Pass: Compute hidden states for all time steps
  2. Compute Loss: Calculate error at each time step
  3. Backward Pass: Propagate gradients backward through time
  4. Update Weights: Accumulate gradients across all time steps

The Gradient Flow

The gradient at time step $t$ depends on all future time steps:

$$\frac{\partial L}{\partial \mathbf{h}^{(t)}} = \frac{\partial L}{\partial \mathbf{h}^{(t+1)}} \cdot \frac{\partial \mathbf{h}^{(t+1)}}{\partial \mathbf{h}^{(t)}} + \frac{\partial L}{\partial \mathbf{y}^{(t)}}$$

The chain rule is applied repeatedly, multiplying Jacobian matrices. This chain of multiplications can cause gradients to vanish (become very small) or explode (become very large).

Forecasting a Time Series

A practical application of RNNs is time series forecasting. Given historical data, predict future values.

📊
ARMA as Baseline

In forecasting, RNNs are often compared to AutoRegressive Moving Average (ARMA) models:

$$y_t = \phi_1 y_{t-1} + \phi_2 y_{t-2} + \cdots + \epsilon_t + \theta_1 \epsilon_{t-1} + \cdots$$

Where $\phi$ are AR coefficients and $\theta$ are MA coefficients. RNNs are more flexible but require more data to train effectively.

The Vanishing and Exploding Gradient Problem

The core difficulty with RNNs is that gradients either vanish (become too small) or explode (become too large) when propagated through many time steps.

Why This Happens

$$\frac{\partial L}{\partial \mathbf{W}_{hh}} = \sum_{t=1}^{T} \frac{\partial L}{\partial \mathbf{h}^{(t)}} \frac{\partial \mathbf{h}^{(t)}}{\partial \mathbf{W}_{hh}}$$

The gradient contains products of Jacobians where eigenvalues < 1 cause vanishing, > 1 cause exploding.

🔧
Solutions
  • Gradient Clipping: Cap gradients to prevent explosion
  • Weight Initialization: Start with careful weight setup (e.g., orthogonal)
  • Recurrent Dropout: Apply dropout to recurrent connections
  • Layer Normalization: Normalize hidden states across features
  • LSTM/GRU: Use special cell architectures

Long Short-Term Memory (LSTM) Cells

LSTMs solve the vanishing gradient problem by introducing a cell state that can preserve information over long sequences.

LSTM Intuition

An LSTM has a "memory cell" separate from its hidden state. Three gates control what information to forget, what new information to add, and what to output. Like an information filter system.

The Three LSTM Gates

🚪
LSTM Gates Mathematics

Forget Gate: Decides what info to discard from cell state

$$\mathbf{f}^{(t)} = \sigma(\mathbf{W}_{f} \cdot [\mathbf{h}^{(t-1)}, \mathbf{x}^{(t)}] + \mathbf{b}_{f})$$

Input Gate: Decides what new info to add to cell state

$$\mathbf{i}^{(t)} = \sigma(\mathbf{W}_{i} \cdot [\mathbf{h}^{(t-1)}, \mathbf{x}^{(t)}] + \mathbf{b}_{i})$$

Cell Candidate: New values to add to cell state

$$\tilde{\mathbf{C}}^{(t)} = \tanh(\mathbf{W}_{C} \cdot [\mathbf{h}^{(t-1)}, \mathbf{x}^{(t)}] + \mathbf{b}_{C})$$

Cell State Update: Combine forget and input

$$\mathbf{C}^{(t)} = \mathbf{f}^{(t)} \odot \mathbf{C}^{(t-1)} + \mathbf{i}^{(t)} \odot \tilde{\mathbf{C}}^{(t)}$$

Output Gate: Decides what to output based on cell state

$$\mathbf{o}^{(t)} = \sigma(\mathbf{W}_{o} \cdot [\mathbf{h}^{(t-1)}, \mathbf{x}^{(t)}] + \mathbf{b}_{o})$$

Hidden State (Output): Filtered cell state

$$\mathbf{h}^{(t)} = \mathbf{o}^{(t)} \odot \tanh(\mathbf{C}^{(t)})$$

Note: $\sigma$ is the sigmoid function, $\odot$ is element-wise multiplication

Why LSTM Works

The cell state provides a gradient highway. Gradients flow directly through the cell state with multiplication by gate values (0-1), preventing vanishing gradients.

Gated Recurrent Units (GRU)

GRUs are a simplified variant of LSTMs with fewer parameters but similar effectiveness. They combine the forget and input gates into a single "update gate."

GRU Mathematics

⚡
GRU Equations

Reset Gate: Controls how much past state to forget

$$\mathbf{r}^{(t)} = \sigma(\mathbf{W}_{r} \cdot [\mathbf{h}^{(t-1)}, \mathbf{x}^{(t)}] + \mathbf{b}_{r})$$

Update Gate: Controls how much to update vs. keep

$$\mathbf{z}^{(t)} = \sigma(\mathbf{W}_{z} \cdot [\mathbf{h}^{(t-1)}, \mathbf{x}^{(t)}] + \mathbf{b}_{z})$$

Candidate Hidden State: Proposed new state

$$\tilde{\mathbf{h}}^{(t)} = \tanh(\mathbf{W}_{h} \cdot [\mathbf{r}^{(t)} \odot \mathbf{h}^{(t-1)}, \mathbf{x}^{(t)}] + \mathbf{b}_{h})$$

Hidden State Update: Interpolate between old and new

$$\mathbf{h}^{(t)} = (1 - \mathbf{z}^{(t)}) \odot \tilde{\mathbf{h}}^{(t)} + \mathbf{z}^{(t)} \odot \mathbf{h}^{(t-1)}$$

GRU vs LSTM Comparison

  • Faster to train: Fewer parameters (~75% of LSTM)
  • Easier to tune: Simpler architecture
  • Similar performance: On many tasks, GRU ≈ LSTM
  • Better for limited data: Fewer parameters to fit
📋
When to Use Each

Choose LSTM if:

  • You have plenty of training data
  • You need maximum modeling capacity
  • Your sequences are very long (100+ steps)

Choose GRU if:

  • You have limited data
  • You need faster training
  • You want simpler code and fewer hyperparameters

1D Convolutional Layers for Sequences

Not all sequential problems require RNNs. For shorter sequences or when pattern matching is important, 1D convolutional layers can be surprisingly effective and are much faster.

1D Convolution for Sequences

$$y_t = \sum_{k=0}^{K-1} w_k \cdot x_{t+k} + b$$

Unlike RNNs which process sequentially, CNNs can process entire sequences in parallel, making them much faster.

Advantages of 1D Conv for Sequences

  • Parallelizable: Process entire batch in parallel
  • Fast: GPU-friendly, trains faster than RNNs
  • Memory efficient: No need to maintain hidden states
  • Good for long sequences: Can use dilated convolutions for large receptive fields

WaveNet Architecture

WaveNet is a state-of-the-art CNN architecture that can handle sequences of tens of thousands of steps. It uses dilated convolutions to maintain a large receptive field while remaining efficient.

WaveNet Key Idea

WaveNet stacks 1D convolutions with exponentially increasing dilation rates (1, 2, 4, 8, 16...). This creates an exponentially growing receptive field while using relatively few parameters.

Dilated Convolution

$$y_t = \sum_{k=0}^{K-1} w_k \cdot x_{t + d \cdot k} + b$$

Where $d$ is the dilation rate. With $d=2$, the kernel looks at every other time step, greatly increasing the receptive field without increasing kernel size.

🌊
WaveNet Features
  • Dilated Convolutions: Exponentially increasing dilation
  • Residual Connections: Skip connections allow gradients to flow
  • Gated Activations: Separate tanh and sigmoid branches
  • Causal Convolutions: Only look at past, not future
  • Global/Local Conditioning: Can condition on metadata

Why WaveNet Works Well

  • Receptive field grows exponentially with depth: $2^L$ where $L$ is number of blocks
  • Can see thousands of steps back with relatively few layers
  • Fully parallelizable during training (unlike RNNs)
  • State-of-the-art on audio generation, music synthesis

Sequence-to-Sequence (Seq2Seq) Models

Seq2Seq models map variable-length input sequences to variable-length output sequences. Instead of predicting a single output, they predict entire sequences.

Two Approaches

🔄
Vector-to-Vector at Each Step

At each time step, output a vector of predictions for the next 14 steps ahead:

$$\mathbf{y}^{(t)} = [\hat{y}_{t+1}, \hat{y}_{t+2}, ..., \hat{y}_{t+14}]$$

Shapes:

  • Input: $[\text{batch}, \text{time\_steps}, \text{features}]$
  • Output: $[\text{batch}, \text{time\_steps}, 14]$
📡
Encoder-Decoder Seq2Seq

A more sophisticated approach uses an encoder-decoder architecture:

  1. Encoder: Reads entire input sequence, produces fixed-size context vector
  2. Decoder: Uses context vector to generate output sequence, one step at a time

This is the classic architecture for machine translation and other NLP tasks.

Shapes and Data Flow

\begin{align} \text{Encoder: } & [batch, T_{in}, features_{in}] \to [batch, hidden\_size] \\ \text{Decoder: } & [batch, T_{out}, hidden\_size] \to [batch, T_{out}, features_{out}] \end{align}

import torch.nn as nn

class SimpleRNNLayer(nn.Module):
    def __init__(self, input_size, hidden_size, num_layers=1):
        super().__init__()
        self.rnn = nn.RNN(input_size, hidden_size, num_layers,
                         batch_first=True)

    def forward(self, x):
        output, hidden = self.rnn(x)
        return output, hidden

class LSTMForecaster(nn.Module):

class Seq2SeqModel(nn.Module):
    def __init__(self, input_size, hidden_size, output_size, num_layers):
        super().__init__()
        self.encoder = nn.GRU(input_size, hidden_size, num_layers,
                             batch_first=True)
        self.decoder_gru = nn.GRU(output_size, hidden_size, num_layers,
                                 batch_first=True)
        self.decoder_fc = nn.Linear(hidden_size, output_size)

    def forward(self, x, target_len=14):
        _, hidden = self.encoder(x)
        batch_size = x.size(0)
        decoder_input = x[:, -1:, :self.output_size]
        outputs = []

        for _ in range(target_len):
            out, hidden = self.decoder_gru(decoder_input, hidden)
            pred = self.decoder_fc(out)
            outputs.append(pred)
            decoder_input = pred

        return torch.cat(outputs, dim=1)

class Conv1DSequenceModel(nn.Module):
    def __init__(self, input_channels, output_channels, kernel_size=3):
        super().__init__()
        padding = (kernel_size - 1) // 2
        self.conv_stack = nn.Sequential(
            nn.Conv1d(input_channels, 32, kernel_size, padding=padding),
            nn.ReLU(),
            nn.Conv1d(32, 64, kernel_size, padding=padding),
            nn.ReLU(),
            nn.Conv1d(64, 128, kernel_size, padding=padding),
            nn.ReLU(),
        )
        self.fc = nn.Linear(128, output_channels)

    def forward(self, x):
        x = x.transpose(1, 2)  # (batch, features, seq_len)
        conv_out = self.conv_stack(x)
        conv_out = conv_out.transpose(1, 2)
        output = self.fc(conv_out)
        return output
Python - PyTorch 
class WaveNetLike(nn.Module):
    def __init__(self, input_channels, output_channels, num_blocks=6):
        super().__init__()
        self.input_conv = nn.Conv1d(input_channels, 64, 1)
        self.blocks = nn.ModuleList([
            nn.Conv1d(64, 64, 3, padding=2**i, dilation=2**i)
            for i in range(num_blocks)
        ])
        self.output_conv = nn.Conv1d(64, output_channels, 1)

    def forward(self, x):
        x = x.transpose(1, 2)
        x = self.input_conv(x)
        for block in self.blocks:
            x = x + block(x)
        x = self.output_conv(x)
        return x.transpose(1, 2)

Training Tips

💡
Best Practices
  • Gradient Clipping: Use clip_grad_norm_() to prevent exploding gradients
  • Layer Normalization: Use for RNNs, not batch norm
  • Learning Rate: Start with 0.001 for Adam optimizer
  • Sequence Length: Truncate very long sequences (100-300 steps) for BPTT stability
  • Data Preprocessing: Normalize inputs to mean 0, std 1
  • Validation: Monitor on held-out test set to detect overfitting

Scenario 1: Choosing the Right Architecture
You need to forecast daily stock prices for 30 days using 5 years of historical data. Which architecture: LSTM, GRU, or 1D CNN? Why?
Best choice: LSTM

Why LSTM wins:

  • Plenty of data available (5 years)
  • Long-term dependencies (seasonal patterns, trends over months)
  • LSTM's cell state perfect for maintaining long-term context
  • Task requires true sequence understanding

Note: GRU would also work well and train faster.

You encounter vanishing gradients on sequences longer than 50 time steps. What three techniques could you try?
Three solutions:
  1. Gradient Clipping: clip_grad_norm_(model.parameters(), max_norm=1.0)
  2. Layer Normalization: Normalize hidden states or recurrent connections
  3. Orthogonal Weight Initialization: Initialize RNN weights orthogonally
Why use "teacher forcing" during training but not during inference in Seq2Seq models?
Teacher Forcing Paradox:

Training: Feed ground truth outputs. Makes training stable, prevents error accumulation.

Inference: Feed own predictions. Tests actual generalization.

The Problem (Exposure Bias): Decoder never sees its own mistakes during training. At test time, errors accumulate. Solution: gradually reduce teacher forcing during training.

WaveNet uses dilations (1, 2, 4, 8, 16, 32). What's the receptive field size after 6 blocks with kernel size 3?
Receptive Field: ~64 time steps

Why better than large kernels:

  • Fewer parameters: kernel size 3 vs 64
  • Multiple scales: local patterns → global patterns
  • Efficient gradient flow
  • Parallel computation (unlike RNNs)
