Vision and Multimodal Transformers

Transformers have revolutionized computer vision by treating images like sequences of tokens. From Vision Transformers that outperform CNNs to multimodal models that understand images and text together, we explore the frontier of deep learning.

Images as Text
ViT breaks images into patches and processes them like words in a sentence, enabling transformer attention over visual regions.
Efficiency First
DeiT uses knowledge distillation to train ViTs from scratch without massive datasets, making vision transformers practical.
Vision + Language
CLIP and Flamingo align visual and textual representations, enabling zero-shot classification and multimodal reasoning.

Vision Transformers: From Images to Sequences

The Big Idea
Imagine cutting a photograph into 16×16 pixel puzzle pieces. Now arrange these pieces in a line and let a transformer read them like a sentence. This is the essence of Vision Transformers (ViT): "An Image Is Worth 16 × 16 Words".

Before the Vision Transformer (2020), computer vision was the domain of Convolutional Neural Networks (CNNs). But the transformer's success in NLP raised a question: could we apply transformers directly to images without convolutions?

The answer was a resounding yes—and it changed everything. ViT demonstrated that pure transformer architectures could achieve state-of-the-art performance on image classification when trained on large datasets.

Why Transformers for Vision?

  • Global receptive field: Self-attention computes relationships between all pixels/patches from the start
  • Scalability: Transformers scale better with model size than CNNs
  • Transfer learning: Pretrained ViTs transfer remarkably well to downstream tasks
  • Multimodality: The same architecture handles text, images, and mixed inputs

Patch Embedding: Converting Images to Tokens

The first step in ViT is to convert an image into a sequence of patch embeddings. Here's how:

The Process

Step 1: Image Patching

Take an image of size $(H, W, C)$ (height, width, channels) and divide it into non-overlapping patches of size $(P, P)$. For a standard 224×224 RGB image with 16×16 patches, you get $(224/16)^2 = 196$ patches.

Step 2: Flatten & Project

Flatten each patch from shape $(P, P, C)$ to $(P^2 \cdot C)$, then project through a linear layer to embedding dimension $D$. This is mathematically equivalent to a 2D convolution with kernel size $P \times P$, stride $P$, and output channels $D$.

Step 3: Sequence Format

The result is a sequence of $N = (H/P) \times (W/P)$ token embeddings, each of dimension $D$. Add a special [CLS] token at the start for classification.

Interactive: Patch Tokenization

Mathematical Formula

$$\mathbf{z}^{patch}_i = \mathbf{W}_p \text{Flatten}(\mathbf{x}_i) \quad \text{where} \quad \mathbf{x}_i \in \mathbb{R}^{P \times P \times C}$$ $$\mathbf{z}^{(0)} = [\mathbf{z}^{[CLS]}; \mathbf{z}^{patch}_1; \ldots; \mathbf{z}^{patch}_N] \quad + \quad \mathbf{E}^{pos}$$

Where $\mathbf{W}_p$ is the projection matrix, $\mathbf{E}^{pos}$ are learnable positional embeddings, and [CLS] is the classification token.

Vision Transformer Architecture

Once we have patch embeddings, the rest is pure transformer magic—identical to the architecture used in NLP.

ViT Data Flow Architecture

Pipeline Components

Input: Patch Tokens
196 tokens (for 224×224 image, 16×16 patches) + 1 [CLS] token = 197 total tokens
Positional Embeddings
Learnable absolute position embeddings added to maintain spatial information
Transformer Encoder
Multi-head self-attention + feedforward MLPs, repeated 12 times (ViT-Base)
Classification Head
Linear layer on [CLS] token output for final class prediction

PyTorch Implementation

import torch import torch.nn as nn from transformers import ViTForImageClassification, ViTImageProcessor # Load pretrained ViT processor = ViTImageProcessor.from_pretrained("google/vit-base-patch16-224") model = ViTForImageClassification.from_pretrained( "google/vit-base-patch16-224-in21k", num_labels=10 ) # Prepare image from PIL import Image image = Image.open("photo.jpg") inputs = processor(images=image, return_tensors="pt") # Forward pass with torch.no_grad(): outputs = model(**inputs) logits = outputs.logits predicted_class = logits.argmax(-1).item()

Key Equations

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$ $$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)W^O$$
$$\text{FFN}(x) = \text{GELU}(xW_1 + b_1)W_2 + b_2$$

DeiT: Data-Efficient Image Transformers

The Challenge
Original ViT required training on JFT-300M (300 million images!) to match CNN performance. DeiT solved this using knowledge distillation—transferring knowledge from a strong teacher model to a student ViT trained on ImageNet alone.

Knowledge Distillation for Vision

DeiT introduced a clever trick: use a CNN teacher (like RegNet) to guide a ViT student during training. The student learns to match the teacher's predictions and intermediate representations.

$$\mathcal{L}_{distill} = \alpha \mathcal{L}_{CE}(\hat{y}_{student}, y) + (1-\alpha) \text{KL}(P_{student}, P_{teacher})$$

Where $P_{student}$ and $P_{teacher}$ are softmax probability distributions, and $\alpha$ is the distillation weight.

Why It Works

  • Dark knowledge: The teacher captures patterns that raw labels don't encode
  • Regularization: Matching soft targets prevents overfitting on limited ImageNet data
  • Efficiency: The student ViT learns to focus on important visual patterns

Results

  • DeiT-Base: 84.5% ImageNet accuracy with only ImageNet training
  • DeiT-Base distilled: 86.5% ImageNet accuracy
  • Competitive with CNNs, but with better scaling properties
# Load DeiT from Hugging Face from transformers import DeiTForImageClassification model = DeiTForImageClassification.from_pretrained( "facebook/deit-base-distilled-patch16-224" ) # This model was trained with knowledge distillation # and achieves 86.5% ImageNet top-1 accuracy

DETR: Detection Transformer for Object Detection

While ViT excels at classification, object detection requires localizing multiple objects with bounding boxes. DETR (Detection Transformer) brought transformers to detection by reformulating it as a set prediction problem.

The Key Innovation

DETR eliminates hand-designed components like non-maximum suppression (NMS) and anchor boxes. Instead, it uses a CNN backbone + transformer encoder-decoder to directly predict a fixed set of object detections.

CNN Backbone
Extract low-level feature maps (e.g., ResNet-50) from the input image
Transformer Encoder
Spatial self-attention over feature map positions to model relationships
Transformer Decoder
Cross-attention with object queries to generate N predictions
Prediction Head
MLP outputs class label and bounding box coordinates

Set Prediction Loss

DETR predicts a fixed number of objects (e.g., 100) and uses bipartite matching to assign predictions to ground truth:

$$\mathcal{L}_{Hungarian} = \min_{\sigma \in S_N} \sum_{i=1}^{N} \left[-\log \hat{p}_{\sigma(i)}(c_i) + \mathbb{1}_{c_i \neq \emptyset} L_{box}(\hat{b}_{\sigma(i)}, b_i)\right]$$

This ensures each prediction is matched to at most one ground truth object, solving the set-matching problem optimally.

Perceiver: Cross-Attention for Arbitrary Inputs

While ViT is specifically designed for images, the Perceiver is a more general architecture that can handle any type of input: images, audio, video, point clouds, text—even arbitrary byte strings.

The Key Insight

Instead of creating patch tokens for each input type, the Perceiver uses learned latent tokens that cross-attend to the raw input data. This decouples model complexity from input size.

$$\text{CrossAttention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$ $$\text{where } Q \text{ = latent tokens, } K,V \text{ = input encodings}$$

Architecture Layers

1. Input Encoding
Encode raw input (e.g., pixels, audio samples) into vectors
2. Latent Tokens
Learned, fixed-size set of tokens independent of input size
3. Cross-Attention Blocks
Latents cross-attend to inputs, then self-attend to each other
4. Output
Process latents through task-specific decoder

Why It's Powerful

  • Flexibility: Same architecture works for images, audio, video, text, point clouds
  • Efficiency: Latent bottleneck reduces computation vs. ViT-style patch tokens
  • Scalability: Model size independent of input resolution/length

Segment Anything Model (SAM)

The Segment Anything Model represents a major shift toward foundation models for vision. Trained on 1 billion masks from 11 million images, SAM can segment any object with minimal prompts.

Key Components

Image Encoder
ViT-based encoder (with adapters) extracts rich image representations
Prompt Encoder
Encodes clicks, bounding boxes, or text into prompt embeddings
Mask Decoder
Transformer decoder combines image + prompt embeddings to predict masks
Real-time
Lightweight architecture enables real-time inference on CPU

Prompt Types

  • Point prompts: Click on object → SAM segments it
  • Box prompts: Draw bounding box → SAM refines segmentation
  • Mask prompts: Provide rough mask → SAM refines it
  • Text prompts: Describe object → SAM segments matching regions

Impact

SAM has become the de facto segmentation tool for computer vision applications, used in countless downstream tasks and enabling efficient annotation of new datasets.

Multimodal Transformers: Bridging Vision and Language

The Multimodal Challenge
Humans understand the world through multiple senses simultaneously. A picture of a dog + the word "dog" creates a richer understanding than either alone. Multimodal transformers learn shared representations that bridge vision and language.

Why Multimodal?

  • Richer representation: Vision + language captures complementary information
  • Zero-shot transfer: Models trained on image-text pairs can classify new categories without fine-tuning
  • New capabilities: Enable image captioning, visual QA, image search, and more
  • Robustness: Learning from redundant modalities improves adversarial robustness

Two Architectural Approaches

Dual-Encoder
Separate encoders for vision and text, shared embedding space (CLIP)
Encoder-Decoder
Vision encoder → Transformer decoder for free-form text generation (Flamingo, BLIP-2)

CLIP: Contrastive Language-Image Pretraining

The CLIP Revolution
CLIP learns to align images and text by training on 400 million web image-caption pairs. Matching pairs are pulled together; mismatched pairs are pushed apart. This creates a shared embedding space where "a photo of a dog" and an actual dog photo are close neighbors.

Architecture

CLIP is deceptively simple: two independent encoders (one for vision, one for text) that project into a shared embedding space.

The Training Process

1. Encode
Pass images through ViT encoder, text through GPT-2-style encoder
2. Project
Project embeddings to a common dimension (e.g., 512-d)
3. Normalize
L2-normalize to unit sphere for cosine similarity
4. Contrastive Loss
Pull matching pairs closer; push mismatched pairs apart

Contrastive Loss (NT-Xent)

$$\mathcal{L}_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j^+) / \tau)}{\sum_{k=1}^{2N} \exp(\text{sim}(z_i, z_k) / \tau)}$$

Where $\tau$ is a learnable temperature parameter, $z_i$ is an image embedding, $z_j^+$ is its matching text embedding, and the denominator sums over all images and texts in the batch (positive and negative examples).

Zero-Shot Classification

CLIP's killer feature: classify images into categories it's never seen before, just by providing text descriptions.

from transformers import CLIPProcessor, CLIPModel import torch from PIL import Image model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32") processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32") # Load image image = Image.open("photo.jpg") # Candidate labels labels = ["ladybug", "beetle", "cricket", "spider"] texts = [f"a photo of a {label}" for label in labels] # Encode inputs = processor(text=texts, images=image, return_tensors="pt", padding=True) with torch.no_grad(): outputs = model(**inputs) logits_per_image = outputs.logits_per_image # shape: (1, 4) # Get predictions probs = logits_per_image.softmax(dim=1) top_label = labels[probs.argmax().item()] print(f"Predicted: {top_label} ({probs.max():.1%})")

Key Results

  • Outperforms supervised ResNet-50 on 60% of image classification datasets without any fine-tuning
  • Particularly strong on datasets with few examples per class (e.g., Stanford Cars)
  • Excellent for image retrieval via vector databases
  • Robust to distribution shift and adversarial examples

Contrastive Learning: Pulling Close, Pushing Away

The success of CLIP relies on contrastive learning—a self-supervised technique where the model learns by comparing similar and dissimilar examples.

Core Intuition

How Contrastive Learning Works
Imagine a crowd of people. You want to recognize your friend. Contrastive learning works by: (1) Looking at your friend (anchor), (2) Moving closer to them (positive), (3) Moving away from strangers (negatives). Same idea in embedding space!

Formula

$$\mathcal{L}_{contrastive} = -\mathbb{E}[\log p_{pos}] - \mathbb{E}[\log(1 - p_{neg})]$$ $$\text{where } p_{pos} = \frac{\exp(s(a, p) / \tau)}{\exp(s(a, p) / \tau) + \sum_n \exp(s(a, n) / \tau)}$$

Applications

  • CLIP: Align images and text descriptions
  • SimCLR: Learn visual representations from unlabeled images
  • DINO: Self-supervised vision transformer training
  • Instance discrimination: Learn representations by comparing image augmentations

The key advantage: no labels required. The model learns structure purely from comparing examples.

Flamingo: Open-Ended Visual Dialogue

The Vision-Language Leap
Flamingo takes a frozen vision model (CLIP) and frozen language model (Chinchilla) and connects them with learned adapters. The result: a single model that can caption images, answer questions, describe actions, and do it all in-context without fine-tuning.

Architecture

Flamingo's elegance lies in reusing pretrained models. It doesn't train vision or language from scratch—instead, it learns how to connect them.

Frozen Vision Encoder
CLIP ViT-L/14 extracts image tokens (frozen weights)
Perceiver Resampler
Cross-attends vision tokens to learnable latent tokens
Frozen Language Model
Chinchilla (70B) generates text (frozen weights)
Gating Mechanism
Learned gates control how much visual information flows to LLM

Key Advantage: Few-Shot In-Context Learning

Flamingo doesn't require fine-tuning for new tasks. Just show it a few examples and it generalizes:

# Few-shot example: License plate detection # Flamingo sees: # 1. A car photo → "License plate: ABC123" # 2. Another car photo → "License plate: XYZ789" # 3. New car photo → ??? # # Flamingo outputs: "License plate: DEF456" # No fine-tuning needed!

Capabilities

  • Image captioning: Describe images in natural language
  • Visual QA: Answer questions about images
  • Object counting: "How many people are in this image?"
  • Video understanding: Understand actions from video frames
  • Few-shot learning: Adapt to new tasks with examples

Limitations

Flamingo hasn't been released publicly by DeepMind. However, open-source alternatives exist:

  • OpenFlamingo: LAION's open reproduction (9B model)
  • IDEFICS: HuggingFace's multimodal model (8B/80B)
  • BLIP-2: Salesforce's more efficient alternative

BLIP-2: Bootstrapping Language-Image Pretraining

BLIP-2 achieves Flamingo-level performance with far fewer learnable parameters, using a two-stage training procedure that's both efficient and effective.

The Bootstrap Strategy

BLIP-2's innovation is in how it leverages existing pretrained models:

Stage 1: Q-Former Training
Train Perceiver-style Q-Former to extract visual features from frozen vision encoder
Stage 2: LLM Adaptation
Train linear layer to map Q-Former outputs to frozen LLM embedding space
Both Models Frozen
Vision encoder and LLM weights never updated—only adapters trained
Efficiency Gains
Only ~250M trainable parameters vs. billions for full models

Stage 1: Q-Former with Three Losses

The Q-Former learns three objectives simultaneously:

$$\mathcal{L}_{total} = \alpha \mathcal{L}_{ITC} + \beta \mathcal{L}_{ITM} + \gamma \mathcal{L}_{LM}$$

Where:

  • $\mathcal{L}_{ITC}$: Image-Text Contrastive (align embeddings like CLIP)
  • $\mathcal{L}_{ITM}$: Image-Text Matching (binary classification)
  • $\mathcal{L}_{LM}$: Language Modeling (predict next caption token)

Why It Works

  • Reusability: Leverages pretrained vision + language models
  • Efficiency: Only trains small adapters, not billions of parameters
  • Flexibility: Can swap different vision encoders and LLMs
  • Scalability: Works with different sized LLMs (8B, 70B, etc.)

Code Example

from transformers import Blip2Processor, Blip2ForConditionalGeneration from PIL import Image import torch processor = Blip2Processor.from_pretrained("Salesforce/blip2-opt-2.7b") model = Blip2ForConditionalGeneration.from_pretrained( "Salesforce/blip2-opt-2.7b", torch_dtype=torch.float16, device_map="auto" ) image = Image.open("photo.jpg") # Image captioning inputs = processor(images=image, return_tensors="pt").to(device, torch.float16) generated_ids = model.generate(**inputs, max_length=50) caption = processor.batch_decode(generated_ids, skip_special_tokens=True)[0] print(f"Caption: {caption}") # Visual QA prompt = "What color is the dog?" inputs = processor(images=image, text=prompt, return_tensors="pt").to(device, torch.float16) generated_ids = model.generate(**inputs, max_length=50) answer = processor.batch_decode(generated_ids, skip_special_tokens=True)[0] print(f"Answer: {answer}")

Performance

  • BLIP-2 (8B) outperforms Flamingo (9B) on many benchmarks
  • BLIP-2 (80B) achieves state-of-the-art on visual QA and captioning
  • Significantly smaller and more efficient than Flamingo

Text-to-Image Generation: From DALL-E to Stable Diffusion

While CLIP and Flamingo understand images, generative models create them. Text-to-image generation has evolved dramatically, from DALL-E's discrete token approach to diffusion-based models.

DALL-E 2 Architecture (Simplified)

Step 1: Text to Embedding
CLIP text encoder converts text prompt to embedding
Step 2: Prior
Transformer prior maps text embedding to image embedding space
Step 3: Decoder
Diffusion decoder generates image from CLIP image embedding
Step 4: Super-Resolution
Diffusion upsampler produces high-res images

Stable Diffusion: The Open-Source Alternative

Stable Diffusion (2022) democratized text-to-image generation by releasing model weights publicly. Unlike DALL-E, it uses a more efficient diffusion architecture:

  • Latent diffusion: Operates in compressed latent space, not pixel space (faster!)
  • Open source: Weights available on Hugging Face
  • Efficient: Runs on consumer GPUs
  • Composable: Easy to add LoRA adapters for style transfer
from diffusers import StableDiffusionPipeline import torch model_id = "runwayml/stable-diffusion-v1-5" pipe = StableDiffusionPipeline.from_pretrained( model_id, torch_dtype=torch.float16 ).to("cuda") prompt = "A serene landscape with mountains at sunset, oil painting style" image = pipe(prompt, guidance_scale=7.5, height=768, width=768).images[0] image.save("output.png")

The Diffusion Process

$$q(x_t | x_0) = \mathcal{N}(x_t; \sqrt{\bar{\alpha}_t} x_0, (1 - \bar{\alpha}_t)\mathbf{I})$$ $$p_\theta(x_{t-1} | x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \Sigma_\theta(x_t, t))$$

Where $q$ is the forward diffusion process (adding noise) and $p_\theta$ is the learned reverse process (denoising). The model learns to remove noise step-by-step until a clear image emerges.

Conditioning on Text

The text embedding guides the denoising process via cross-attention:

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q(K_{text})^T}{\sqrt{d_k}}\right)(V_{text})$$

This allows the text prompt to control which features appear in the generated image.

War Room: Socratic Sandbox

Let's test your understanding with challenging scenarios that connect theory to practice.

Scenario 1: Choosing an Architecture

Your challenge: You're building a system to segment medical images (X-rays, CT scans) and provide text descriptions. Which architecture would you choose and why?

Answer: You'd likely use a combination:

  • SAM for segmentation (trained on medical data or fine-tuned)
  • BLIP-2 for text description (efficient, good caption quality)

Why not Flamingo? BLIP-2 is more efficient and achieves similar performance. Medical data requires fine-tuning, and BLIP-2's Q-Former is easier to adapt than Flamingo's architecture.

Scenario 2: Zero-Shot Classification Challenge

Your challenge: You're given 10,000 images of rare mushroom species with only 5 examples per species. How would you classify them without fine-tuning?

Answer: Use CLIP zero-shot classification:

  • Create text prompts: "A photo of a [mushroom_species]"
  • Encode all species names with CLIP text encoder
  • Encode each image with CLIP vision encoder
  • Find the most similar species name for each image

Why it works: CLIP was trained on diverse web images. Even though mushroom images are specialized, CLIP learned general visual concepts that transfer. With 5 examples, you could even fine-tune a linear classifier on CLIP features.

Scenario 3: Efficiency vs. Quality

Your challenge: You're deploying a vision-language model on edge devices (smartphones). Should you use Flamingo, BLIP-2, or a smaller custom model?

Answer: BLIP-2 is the clear winner:

  • BLIP-2 (2.7B LLM) ≈ 250M trainable params
  • Flamingo (9B) ≈ billions of params
  • BLIP-2 achieves better quality on mobile-appropriate sizes
  • Q-Former bottleneck makes inference faster

Trade-off: For on-device deployment, you might further optimize BLIP-2 with quantization (8-bit or 4-bit) and knowledge distillation.

Scenario 4: Understanding Attention Flow

Your challenge: In a ViT processing a 224×224 image with 16×16 patches, each of the 12 attention heads in a layer can see all 196 patch tokens. How does the model learn that patches within the same object are more important than patches in the background?

Answer: Through learned attention weights during training:

  • Early layers learn low-level patterns (edges, colors)
  • Middle layers combine patches into object-like regions
  • Later layers focus attention on object patches based on classification task
  • Backpropagation amplifies connections that improve classification loss

Key insight: ViT doesn't have built-in inductive bias for spatial locality (unlike CNNs with small kernels). Instead, it learns which patches matter through data-driven attention, making it more flexible but requiring more data.

Key Takeaways

Vision Transformers
ViT treats images as sequences of patches. DeiT makes it practical with distillation. Hierarchical variants (PVT, Swin) handle dense prediction tasks.
Dual-Encoders
CLIP aligns vision and language via contrastive learning, enabling zero-shot classification and foundation models for understanding.
Encoder-Decoders
Flamingo and BLIP-2 combine frozen pretrained models with learnable adapters for generation tasks while remaining efficient.
Generative Models
Diffusion models like Stable Diffusion reverse the corruption process to generate high-quality images from text descriptions.
