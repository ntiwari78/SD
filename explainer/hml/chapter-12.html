Part II: Deep Learning

Deep Computer Vision Using Convolutional Neural Networks

Convolutional Neural Networks (CNNs) emerged from studying the brain's visual cortex and have revolutionized computer vision. Learn how these networks detect patterns, extract features, and achieve superhuman performance on image recognition tasks.

🔍 Feynman Mental Model: The Magnifying Glass Approach

Imagine you're examining a high-resolution photograph with a magnifying glass. You don't look at the entire image at once; instead, you slide your magnifying glass across different regions. Each position reveals local patterns—edges, corners, textures.

A CNN works exactly like this:

  • Filters (Magnifying Glass): Small learnable patterns that detect specific features
  • Convolution Operation: Sliding these filters across the input image
  • Feature Maps: The output showing where each pattern appears
  • Stacking Layers: Early layers detect simple features (edges), deeper layers combine them into complex objects (faces, dogs)

This hierarchical feature extraction is why CNNs are so powerful—they learn to recognize increasingly abstract concepts by building on simpler building blocks.

🧠 The Architecture of the Visual Cortex

CNNs are inspired by neuroscience discoveries about how biological vision systems work. In the 1960s, Hubel and Wiesel discovered that:

  • Visual cortex neurons respond to specific patterns in small regions called receptive fields
  • Some neurons detect horizontal lines, others vertical lines, at different orientations
  • Neurons with larger receptive fields detect more complex patterns combining lower-level features
  • This creates a hierarchical organization: simple features → complex objects
Key Insight

CNNs replicate this biological hierarchy: early convolutional layers learn edge detection, middle layers learn shapes, and deeper layers recognize objects.

📐 The Convolution Operation: Mathematics & Intuition

Why Convolution?

Fully connected layers in traditional neural networks:

  • Treat spatial position as irrelevant (flatten the image)
  • Have millions of parameters even for small images
  • Don't exploit the spatial structure of images

Convolutional layers:

  • Preserve spatial information through local receptive fields
  • Share weights (the same filter scans the whole image)
  • Dramatically reduce parameters while increasing expressiveness

The Convolution Equation

For a neuron at position $(i, j)$ in feature map $k$ of layer $l$:

$$z_{i,j,k} = b_k + \sum_{u=0}^{f_h-1} \sum_{v=0}^{f_w-1} \sum_{k'=0}^{f_{n'}-1} w_{u,v,k',k} \cdot x_{i \cdot s_h + u, j \cdot s_w + v, k'}$$

Where:

  • $z_{i,j,k}$ = output at position $(i,j)$ in feature map $k$
  • $f_h, f_w$ = filter height and width (receptive field size)
  • $s_h, s_w$ = vertical and horizontal strides (step size)
  • $w_{u,v,k',k}$ = weight connecting input position $(u,v)$ to output
  • $b_k$ = bias for feature map $k$
  • $f_{n'}$ = number of input feature maps (or channels)

Output Dimensions

$$\text{Output Height} = \left\lfloor \frac{H_{in} + 2p - f_h}{s_h} + 1 \right\rfloor$$ $$\text{Output Width} = \left\lfloor \frac{W_{in} + 2p - f_w}{s_w} + 1 \right\rfloor$$

Where $p$ is the padding (often zero padding added around edges).

Parameter Count

$$\text{Params per filter} = f_h \times f_w \times f_{n'} + 1$$ $$\text{Total params} = \text{Params per filter} \times \text{Number of filters}$$

This is vastly fewer than fully connected layers (often 1000x fewer!).

🎯 Filters and Feature Maps

What are Filters?

A filter (or kernel) is a small matrix of weights that the network learns. Each filter detects a specific type of feature:

  • Early layers: edge detectors (horizontal, vertical, diagonal)
  • Middle layers: texture and shape detectors
  • Deep layers: object part detectors (eyes, wheels, noses)

Stride and Padding

Stride: How many pixels we move the filter at each step.

  • Stride = 1: Apply filter to every position (no skip)
  • Stride = 2: Skip every other position (reduce dimensions by 2)

Padding: Adding extra rows/columns around the image edges.

  • No padding: Edge pixels participate in fewer receptive fields (lose information)
  • Zero padding: Pad with 0s to preserve spatial dimensions
  • Same padding: Pad so output size = input size (with stride=1)

Feature Maps

The output of applying a filter to an input is a feature map—a 2D array where each element represents how much that filter's pattern appears at that location.

Practice Insight

A convolutional layer with 32 filters produces 32 feature maps. Each filter learns to detect different patterns, giving the network diverse feature representations.

🏊 Pooling Layers: Downsampling and Robustness

Why Pooling?

Pooling layers serve multiple purposes:

  • Reduce computational cost by downsampling feature maps
  • Reduce overfitting by summarizing information
  • Add translation invariance (small shifts in input don't change output)
  • Preserve the most important information

Types of Pooling

Max Pooling (Most Common): Takes the maximum value in a window.

$$\text{Output}_{i,j,k} = \max_{u,v} x_{i \cdot s_h + u, j \cdot s_w + v, k}$$

Interpretation: "Does this feature appear anywhere in this region?"

Average Pooling: Takes the average value in a window.

$$\text{Output}_{i,j,k} = \frac{1}{f_h \times f_w} \sum_{u,v} x_{i \cdot s_h + u, j \cdot s_w + v, k}$$

Interpretation: "What's the average activation in this region?"

Global Average Pooling: Reduces entire feature map to a single value.

  • Often used before final classification layers
  • Dramatically reduces parameters
  • Provides spatial regularization

Pooling Dimensions

$$\text{Output dimensions} = \left\lfloor \frac{\text{Input dim} - \text{Pool size}}{\text{Stride}} + 1 \right\rfloor$$

Typical configuration: 2×2 pooling with stride 2 halves both width and height.

🏗️ CNN Architectures: From LeNet to Modern Networks

LeNet-5 (1998)

Input (28×28) → Conv (6,5×5) → Pool (2×2) → Conv (16,5×5) → Pool (2×2) → FC → 10

Revolutionary at the time, used for digit recognition. Simple but inspired all modern CNNs.

AlexNet (2012)

Input (224×224) → Conv (96,11×11) → Pool → Conv (256,5×5) → ... → FC (4096)

Won 2012 ILSVRC by massive margin (17% vs 26% error). Introduced ReLU activation and dropout regularization.

VGGNet (2014)

Key Innovation: Stack many small 3×3 convolutions instead of few large ones.

  • 5 convolutional blocks + max pooling
  • Deeper networks = better features
  • 3×3 filters more efficient than larger kernels
  • Very regular and interpretable architecture

GoogLeNet/Inception (2014)

Key Innovation: Inception modules with parallel pathways.

  • Multiple filter sizes in parallel (1×1, 3×3, 5×5)
  • Learns which scales matter for different features
  • 10x fewer parameters than AlexNet
  • Inspired modern multi-scale approaches

ResNet (2015)

Key Innovation: Residual connections (skip connections).

$$y = F(x) + x$$

Where $F(x)$ is the convolutional block and $x$ is the input.

  • Solves vanishing gradient problem
  • Enables training of very deep networks (152+ layers)
  • Skip connections: "Learn residual (difference) not absolute mapping"

Other Important Architectures

  • Xception: Depthwise separable convolutions for efficiency
  • EfficientNet: Balanced scaling of depth, width, resolution
  • MobileNet: Lightweight networks for mobile/edge devices
  • DenseNet: Dense connections between layers

⚙️ Implementing CNNs with PyTorch

Basic Convolutional Layer

import torch import torch.nn as nn # Create a convolutional layer # in_channels: 3 (RGB images) # out_channels: 32 (32 filters) # kernel_size: 5 (5x5 filters) conv_layer = nn.Conv2d( in_channels=3, out_channels=32, kernel_size=5, stride=1, padding=2 # "Same" padding ) # Input shape: (batch_size, channels, height, width) x = torch.randn(16, 3, 224, 224) # Batch of 16 RGB images output = conv_layer(x) print(output.shape) # (16, 32, 224, 224)

Building a CNN Model

class SimpleCNN(nn.Module): def __init__(self, num_classes=10): super().__init__() # Feature extraction layers self.features = nn.Sequential( nn.Conv2d(3, 32, kernel_size=3, padding=1), nn.ReLU(inplace=True), nn.MaxPool2d(kernel_size=2, stride=2), nn.Conv2d(32, 64, kernel_size=3, padding=1), nn.ReLU(inplace=True), nn.MaxPool2d(kernel_size=2, stride=2), nn.Conv2d(64, 128, kernel_size=3, padding=1), nn.ReLU(inplace=True), nn.MaxPool2d(kernel_size=2, stride=2), ) # Classification head self.classifier = nn.Sequential( nn.Linear(128 * 28 * 28, 256), nn.ReLU(inplace=True), nn.Dropout(0.5), nn.Linear(256, num_classes) ) def forward(self, x): x = self.features(x) x = x.view(x.size(0), -1) # Flatten x = self.classifier(x) return x # Create model and check output model = SimpleCNN(num_classes=10) x = torch.randn(16, 3, 224, 224) output = model(x) print(output.shape) # (16, 10)

Transfer Learning with Pretrained Models

import torchvision.models as models # Load pretrained ResNet50 (trained on ImageNet) model = models.resnet50(pretrained=True) # Freeze all parameters for param in model.parameters(): param.requires_grad = False # Replace final classification layer for your task num_classes = 10 model.fc = nn.Linear(model.fc.in_features, num_classes) # Now only the new classification layer will be trained # This is much faster and uses less memory than training from scratch

Training Loop

import torch.optim as optim model = SimpleCNN(num_classes=10) criterion = nn.CrossEntropyLoss() optimizer = optim.Adam(model.parameters(), lr=0.001) # Training loop for epoch in range(num_epochs): for images, labels in train_loader: # Forward pass outputs = model(images) loss = criterion(outputs, labels) # Backward pass optimizer.zero_grad() loss.backward() optimizer.step() print(f'Epoch [{epoch+1}/{num_epochs}], Loss: {loss.item():.4f}')

⚔️ Socratic Sandbox: Test Your Understanding

Interactive scenarios to deepen your understanding of CNNs.

Scenario 1: Calculate Output Dimensions ▼

Your Task

Given an input image of 224×224 pixels, a 5×5 convolutional filter, 2 pixels of zero padding, and a stride of 1, calculate the output dimensions.

Formula
Output = ⌊(Input + 2*Padding - FilterSize) / Stride + 1⌋
Output = ⌊(224 + 2*2 - 5) / 1 + 1⌋ = ⌊(224 + 4 - 5 + 1)⌋ = 224
Scenario 2: Filter Learning Intuition ▼

Your Task

A convolutional layer has 64 filters. How many independent feature maps will this layer produce from an RGB input?

Explanation

Each filter produces one feature map. The 3 RGB channels are inputs to each filter, combined in the convolution operation. The layer outputs 64 feature maps (one per filter), regardless of input channels.

Scenario 3: Receptive Field Calculation ▼

Your Task

In a neural network, the receptive field of a neuron is the region of the input that influences its value. Calculate the receptive field after two 3×3 convolutional layers with stride 1.

Explanation

Receptive fields grow as you stack layers. After one 3×3 layer: RF = 3×3. After second 3×3 layer: RF = 5×5 (because the second layer's 3×3 neighborhood over the first layer's 3×3 neighborhood spans 5×5 of the input).

RF_new = RF_old + (filter_size - 1) × stride_product

✨ Key Takeaways

  • Convolution = Weight Sharing: Use same filters across entire image, reducing parameters
  • Hierarchical Features: Early layers learn edges → deeper layers learn objects
  • Local Connectivity: Neurons only connect to local regions, respecting image structure
  • Translation Equivariance: Shift input = shift output (+ pooling = translation invariance)
  • Transfer Learning: Pretrained models on ImageNet transfer excellently to new tasks
  • Receptive Field Matters: Larger receptive fields capture context; balance with computational cost
  • Skip Connections (ResNet): Enable training of very deep networks by improving gradient flow
  • Modern Trends: Efficient architectures (MobileNet, EfficientNet) trade accuracy for speed
