Autoencoders, GANs, and Diffusion Models

Learning Dense Representations and Generating New Data

Three powerful paradigms for unsupervised learning: Autoencoders compress data into efficient latent codes, GANs pit a forger against a detective, and Diffusion Models gradually remove noise. Each approach learns different representations and generates high-quality synthetic data.

Autoencoders: Learning Efficient Representations

The Feynman Mental Model: "Compress Then Reconstruct"

Imagine you're trying to memorize a long sequence of numbers. If you notice a pattern (e.g., "even numbers from 50 down to 14"), you only need to remember the pattern and two endpoints—compression through intelligence. Autoencoders do this: compress high-dimensional inputs into a low-dimensional code that captures the essence, then reconstruct the original.

The Core Idea

Autoencoders are unsupervised neural networks that learn to copy their inputs to their outputs, but with constraints that force them to discover efficient representations. They consist of:

  • Encoder: Maps input to a latent code (low-dimensional representation)
  • Decoder: Reconstructs the original input from the latent code
  • Bottleneck: The constraint that creates compression pressure

Key Insight

Unlike classification networks trained on labeled data, autoencoders work entirely unsupervised. The target output equals the input, creating a self-supervised training signal.

Undercomplete Autoencoders

When the bottleneck dimension is smaller than the input dimension, the autoencoder must compress. This forces it to learn the most important features while discarding noise.

PCA via Linear Autoencoder

A linear autoencoder with MSE loss performs Principal Component Analysis (PCA), finding the lower-dimensional subspace that captures maximum variance.

PyTorch Implementation

import torch
import torch.nn as nn

class LinearAutoencoder(nn.Module):
    def __init__(self, input_dim, latent_dim):
        super().__init__()
        self.encoder = nn.Linear(input_dim, latent_dim)
        self.decoder = nn.Linear(latent_dim, input_dim)

    def forward(self, x):
        z = self.encoder(x)
        return self.decoder(z), z

# Usage
ae = LinearAutoencoder(input_dim=784, latent_dim=32)
criterion = nn.MSELoss()
optimizer = torch.optim.Adam(ae.parameters(), lr=0.001)

# Training loop
for epoch in range(epochs):
    z_codes, _ = ae(x_batch)
    loss = criterion(z_codes, x_batch)
    loss.backward()
    optimizer.step()

Variational Autoencoders (VAE): Probabilistic Generation

The Feynman Mental Model: "Compress to Essence, Sample New Examples"

A regular autoencoder creates discrete codes. A VAE instead learns a probability distribution over the latent space. Instead of mapping to a single point, it learns a region that's likely to produce good reconstructions. By sampling from this region, we can generate new data.

From Autoencoders to VAEs

A VAE adds two key components:

  • Probabilistic Latent Space: Encoder outputs mean and variance, not a point
  • Generative Capability: Sample from the latent distribution to generate new data

The ELBO (Evidence Lower Bound)

VAEs maximize:

$$\mathcal{L} = \mathbb{E}_{q_\phi(z|x)}[\log p_\theta(x|z)] - \text{KL}(q_\phi(z|x) \| p(z))$$

Where:

  • Reconstruction Term: $\mathbb{E}_{q_\phi(z|x)}[\log p_\theta(x|z)]$ - How well can we reconstruct?
  • KL Divergence: $\text{KL}(q_\phi(z|x) \| p(z))$ - How close is learned distribution to prior?

KL Divergence: Measuring Distribution Distance

KL divergence measures how much one probability distribution differs from another:

$$\text{KL}(Q \| P) = \sum_i Q(i) \log \frac{Q(i)}{P(i)}$$

In VAEs, we regularize the learned distribution $q(z|x)$ to stay close to a standard normal prior $p(z) = \mathcal{N}(0, I)$.

The Reparameterization Trick

To backprop through sampling, VAEs use a clever trick: instead of sampling $z \sim q_\phi(z|x)$ directly, they reparameterize as $z = \mu + \sigma \odot \epsilon$ where $\epsilon \sim \mathcal{N}(0, I)$.

PyTorch VAE Implementation

class VAE(nn.Module):
    def __init__(self, input_dim, hidden_dim, latent_dim):
        super().__init__()
        # Encoder
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc_mu = nn.Linear(hidden_dim, latent_dim)
        self.fc_logvar = nn.Linear(hidden_dim, latent_dim)
        # Decoder
        self.fc3 = nn.Linear(latent_dim, hidden_dim)
        self.fc4 = nn.Linear(hidden_dim, input_dim)

    def encode(self, x):
        h = torch.relu(self.fc1(x))
        mu = self.fc_mu(h)
        logvar = self.fc_logvar(h)
        return mu, logvar

    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        eps = torch.randn_like(std)
        z = mu + eps * std
        return z

    def decode(self, z):
        h = torch.relu(self.fc3(z))
        return torch.sigmoid(self.fc4(h))

    def forward(self, x):
        mu, logvar = self.encode(x)
        z = self.reparameterize(mu, logvar)
        x_recon = self.decode(z)
        return x_recon, mu, logvar

def vae_loss(x_recon, x, mu, logvar):
    # Reconstruction loss (binary cross-entropy)
    BCE = nn.functional.binary_cross_entropy(x_recon, x, reduction='sum')
    # KL divergence: D_KL(N(mu, sigma) || N(0, I))
    KLD = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return BCE + KLD

VAE Sampling and Generation

Once trained, we can generate new samples by directly sampling from $\mathcal{N}(0, I)$ and passing through the decoder:

$$\text{Generate: } z \sim \mathcal{N}(0, I) \rightarrow x_{\text{new}} = p_\theta(z)$$

Generative Adversarial Networks: The Game Theory Approach

The Feynman Mental Model: "Forger vs. Detective"

A GAN is like a criminal counterfeiting money (generator) competing against a police investigator (discriminator). The forger gets better at making fake money indistinguishable from real, while the detective gets better at spotting fakes. This adversarial dynamic drives both toward perfection.

The GAN Architecture

A GAN consists of two networks in an adversarial game:

  • Generator ($G$): Takes random noise, produces fake data
  • Discriminator ($D$): Classifies data as real or fake

The Minimax Game

Training alternates between:

$$\min_G \max_D V(G, D) = \mathbb{E}_{x \sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]$$

Discriminator's Goal: Maximize $V$ (correctly identify real vs. fake)

Generator's Goal: Minimize $V$ (fool the discriminator)

Training Dynamics

Ideally at equilibrium: $D$ is 50% confident (always uncertain), real and fake distributions match, and we can sample from $G$ to generate new data.

Training Challenges

Challenge What Happens Solution
Mode Collapse Generator produces limited diversity, few distinct modes Minibatch discrimination, feature matching, unrolled GANs
Vanishing Gradients When discriminator is too good, generator gradient becomes zero Wasserstein GAN, spectral normalization
Unstable Training Loss oscillates wildly, networks don't converge Gradient clipping, learning rate scheduling, spectral norm
Non-Convergence No equilibrium reached, continuous oscillation Wasserstein distance, improved loss functions

Simple PyTorch GAN

class Generator(nn.Module):
    def __init__(self, latent_dim, image_dim):
        super().__init__()
        self.fc = nn.Sequential(
            nn.Linear(latent_dim, 128),
            nn.ReLU(),
            nn.Linear(128, 256),
            nn.ReLU(),
            nn.Linear(256, 512),
            nn.ReLU(),
            nn.Linear(512, image_dim),
            nn.Sigmoid()
        )

    def forward(self, z):
        return self.fc(z)

class Discriminator(nn.Module):
    def __init__(self, image_dim):
        super().__init__()
        self.fc = nn.Sequential(
            nn.Linear(image_dim, 512),
            nn.LeakyReLU(0.2),
            nn.Linear(512, 256),
            nn.LeakyReLU(0.2),
            nn.Linear(256, 128),
            nn.LeakyReLU(0.2),
            nn.Linear(128, 1),
            nn.Sigmoid()
        )

    def forward(self, x):
        return self.fc(x)

# Training loop alternates:
# 1. Update discriminator: maximize log D(x) + log(1 - D(G(z)))
# 2. Update generator: minimize log(1 - D(G(z)))

Diffusion Models: Denoising to Generate

The Feynman Mental Model: "Gradual Denoising"

Imagine a photograph gradually covered by noise until it becomes pure static. A diffusion model learns to reverse this: start with random noise and gradually denoise it into a coherent image. By repeatedly removing noise, high-quality images emerge.

The Two Processes

Diffusion models involve two complementary processes:

1. Forward Process (Diffusion): Adding Noise

Gradually add Gaussian noise to data over $T$ timesteps:

$$x_t = \sqrt{\bar{\alpha}_t} x_0 + \sqrt{1 - \bar{\alpha}_t} \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

At $t=0$: clean data. At $t=T$: pure noise.

2. Reverse Process (Denoising): Removing Noise

Learn to reverse: predict the noise added at each step and remove it:

$$p_\theta(x_{t-1}|x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \Sigma_\theta(x_t, t))$$

DDPM: Denoising Diffusion Probabilistic Models

DDPM trains a neural network to predict the noise $\epsilon_\theta(x_t, t)$ at each timestep:

$$\mathcal{L} = \mathbb{E}_{x_0, t, \epsilon}[\| \epsilon - \epsilon_\theta(x_t, t) \|^2]$$

The network learns: "Given noisy image at timestep $t$, predict what noise was added."

Generation Process

  1. Start with $x_T \sim \mathcal{N}(0, I)$ (pure noise)
  2. For $t = T$ down to 1:
    • Predict noise: $\hat{\epsilon} = \epsilon_\theta(x_t, t)$
    • Denoise: $x_{t-1} = \frac{1}{\sqrt{\alpha_t}}(x_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}}\hat{\epsilon}) + \sigma_t z$
  3. Result: $x_0$, a high-quality generated image

Score-Based Generative Modeling

An alternative formulation: learn the score (gradient of log probability) at each noise level:

$$s_\theta(x, t) \approx \nabla_x \log p_t(x)$$

Generation uses Langevin dynamics with these scores to iteratively denoise.

Latent Diffusion Models (Stable Diffusion)

Computing diffusion in pixel space is slow (1000+ steps needed). Latent Diffusion Models apply diffusion in a compressed latent space:

  1. Encode image to latent space with VAE encoder
  2. Run diffusion on compact latent representation
  3. Decode back to pixel space with VAE decoder

This dramatically speeds up generation while maintaining quality.

DDPM Training Pseudocode

# Training
for step in range(num_steps):
    x_0 = sample_real_images()  # Clean image
    t = sample_random_timestep()
    epsilon = torch.randn_like(x_0)  # Random noise

    # Add noise: x_t = sqrt(alpha_bar[t])*x_0 + sqrt(1-alpha_bar[t])*epsilon
    x_t = sqrt_alpha_bar[t] * x_0 + sqrt_1_minus_alpha_bar[t] * epsilon

    # Predict noise
    epsilon_pred = model(x_t, t)

    # Loss: MSE between predicted and actual noise
    loss = F.mse_loss(epsilon_pred, epsilon)
    loss.backward()
    optimizer.step()

# Sampling (Generation)
x_T = torch.randn(batch_size, channels, H, W)  # Start with noise
for t in range(T-1, 0, -1):
    z = torch.randn_like(x_T) if t > 1 else 0
    epsilon_pred = model(x_T, t)
    x_T = (x_T - (1-alpha[t])/sqrt(1-alpha_bar[t]) * epsilon_pred) / sqrt(alpha[t]) + sigma[t] * z

return x_T  # Generated high-quality image

War Room: Generative Model Decision Scenarios

Scenario 1: E-Commerce Image Generation

Problem: You run an e-commerce platform with 10,000 product categories. You want to generate lifestyle images showing products in use to increase conversion rates. You need to generate images in real-time (< 100ms latency) for product pages.

Questions:

  • Which generative model would you choose and why?
  • How would you handle the speed constraint with diffusion models?
  • What training data would you need?
Show Discussion

Answer: Use a latent diffusion model (like Stable Diffusion) fine-tuned on your product category images. While slower than GANs, modern optimizations (distillation, fewer steps) reduce inference to ~500ms. Alternatively, use a DCGAN for < 100ms generation, trading quality for speed. The key is caching frequently generated images and using conditional generation (product ID → image).

Scenario 2: Anomaly Detection in Medical Imaging

Problem: You're building an anomaly detection system for chest X-rays. Normal X-rays are common, but abnormal ones (tumors, pneumonia) are rare. You need to identify images that deviate from normal patterns.

Questions:

  • Which model is best suited for unsupervised anomaly detection?
  • How would you measure what's "anomalous"?
  • Why wouldn't you use a classifier trained on labeled abnormalities?
Show Discussion

Answer: Use a denoising autoencoder or VAE. Train on normal X-rays only. For a test image: if reconstruction error (original vs. reconstructed) is high, it's likely anomalous. Autoencoders are better than classifiers here because: (1) You can train on unlabeled normal data, (2) You catch unknown types of abnormalities not in training labels, (3) You learn what "normal" looks like rather than memorizing specific diseases.

Scenario 3: Conditional Image Synthesis

Problem: You want to generate faces given attributes: age (18-80), gender, expression (smile, neutral, frown). Users specify attributes and receive a unique generated face.

Questions:

  • How would you incorporate conditional information into your generative model?
  • Which model architecture (AE/VAE/GAN/Diffusion) best supports this?
  • What challenges arise with conditional generation?
Show Discussion

Answer: Use Conditional GAN (cGAN) or conditional diffusion model. Concatenate attribute vectors to both encoder and noise input. cGANs are efficient (fast generation), while conditional diffusion models produce higher quality but slower. The challenge is disentanglement: changing one attribute shouldn't affect others. Solutions include: (1) Attribute-specific latent factors, (2) Contrastive learning for disentanglement, (3) Progressive training (train age, then gender, then expression independently).

Scenario 4: Data Augmentation for Small Datasets

Problem: You're training a classifier on a rare disease with only 500 labeled images. You need more training data to prevent overfitting but collecting more real images is expensive.

Questions:

  • How would you use generative models for data augmentation?
  • Which model would be safest to use (avoid mode collapse, training issues)?
  • How do you ensure synthetic data actually helps rather than hurts?
Show Discussion

Answer: Use a VAE or diffusion model (more stable than GANs). Train on the 500 real images. Generate synthetic images but: (1) Mix synthetic 30-50% with real data, not 100% synthetic, (2) Validate: check if synthetic + real outperforms real-only, (3) Monitor classifier performance on held-out test set, (4) Use diversity metrics (FID, IS) to ensure generated variety. Diffusion models are safer due to stable training but slower. VAEs are a good balance.
