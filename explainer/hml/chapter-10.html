PyTorch: Building Neural Networks Like LEGO

Tensors, Autograd & GPU Acceleration

PyTorch provides the essential building blocks to construct, train, and deploy neural networks at scale. Think of it as NumPy with automatic differentiation and GPU acceleration baked in. Master tensors, backpropagation, and the canonical training loop that powers modern deep learning.

PyTorch Tensors: Multi-dimensional Arrays on Steroids

Tensors are the fundamental data structure in PyTorch. They are like NumPy arrays, but with two critical superpowers: GPU acceleration and automatic differentiation (autograd).

Creating Tensors

PyTorch provides multiple ways to create tensors:

torch.tensor()

Create tensor from Python list or NumPy array. Directly specify the data.

torch.zeros() / torch.ones()

Create tensors filled with zeros or ones. Useful for initialization.

torch.randn()

Create tensors from normal distribution. Perfect for weight initialization.

torch.eye()

Create identity matrices. Essential for diagonal operations.

Python import torch # Create from list x = torch.tensor([1.0, 2.0, 3.0]) # Create zeros, ones zeros = torch.zeros(3, 4) ones = torch.ones(2, 3) # Create from normal distribution weights = torch.randn(784, 128) # Create on GPU (if available) gpu_tensor = torch.randn(100, 100).to('cuda') # Check device print(gpu_tensor.device) # Output: cuda:0

Tensor Operations & Broadcasting

PyTorch tensors support element-wise operations, matrix multiplication, and automatic broadcasting (like NumPy). The real magic happens when operations integrate seamlessly with autograd.

Autograd: Automatic Differentiation Magic

Autograd is the mechanism that makes neural networks trainable. It automatically computes gradients via backpropagation by tracking all operations in a computation graph.

The Computation Graph

When you set `requires_grad=True` on a tensor, PyTorch begins recording every operation on that tensor. This builds a computation graph—a DAG (directed acyclic graph) of operations. When you call `loss.backward()`, it traverses this graph in reverse, computing gradients using the chain rule.

$$\frac{\partial L}{\partial w} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial w}$$

This is the chain rule of calculus—exactly what backpropagation computes. PyTorch handles all the bookkeeping automatically.

requires_grad & Tracking

Python import torch # Create a tensor with gradient tracking x = torch.tensor(2.0, requires_grad=True) y = torch.tensor(3.0, requires_grad=True) # Build computation graph z = x**2 + 2*x*y + y**2 # Backward pass (compute gradients) z.backward() # Access gradients print(x.grad) # dz/dx = 4x + 2y = 8 + 6 = 14 print(y.grad) # dz/dy = 2x + 2y = 4 + 6 = 10 # Gradients accumulate! z.backward() # This adds to existing gradients print(x.grad) # Now 28 (accumulated) # Clear gradients x.grad.zero_() y.grad = None

Inference Mode: torch.no_grad()

During inference or when computing validation loss, you don't need gradients. Use `torch.no_grad()` to disable autograd, which saves memory and computation:

Python import torch model = ... # some trained model # Disable gradient computation (critical for inference!) with torch.no_grad(): predictions = model(test_data) val_loss = criterion(predictions, test_targets) # Or use @torch.no_grad() decorator @torch.no_grad() def evaluate(model, data): return model(data) # Alternatively, set model to evaluation mode model.eval() # Disables dropout, batch norm changes behavior with torch.no_grad(): predictions = model(test_data)

Debugging Gradients

When training doesn't work, gradient problems are often the culprit. Check for NaN gradients, vanishing gradients, or exploding gradients:

Python import torch import torch.nn as nn # Detect NaN gradients for name, param in model.named_parameters(): if param.grad is not None: if torch.isnan(param.grad).any(): print(f"NaN gradient in {name}") if torch.isinf(param.grad).any(): print(f"Inf gradient in {name}") # Monitor gradient norms (useful for debugging) total_norm = 0.0 for param in model.parameters(): if param.grad is not None: total_norm += param.grad.data.norm(2).item() ** 2 total_norm = total_norm ** 0.5 print(f"Total gradient norm: {total_norm}") # Clip gradients if they explode (gradient clipping) torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

Building Models with nn.Module

`nn.Module` is the foundation class for all neural networks in PyTorch. It provides parameter management, device handling, and training/evaluation modes. There are two main approaches: sequential and custom.

nn.Sequential for Simple Models

For straightforward stacks of layers, `nn.Sequential` provides a concise API:

Python import torch import torch.nn as nn # Simple sequential MLP model = nn.Sequential( nn.Linear(784, 300), nn.ReLU(), nn.Linear(300, 100), nn.ReLU(), nn.Linear(100, 10) # 10 classes for MNIST ) # Forward pass x = torch.randn(32, 784) # Batch of 32 images, flattened y = model(x) # Shape: (32, 10) # Inspect model print(model) print(f"Total parameters: {sum(p.numel() for p in model.parameters())}")

Custom nn.Module Subclass

For complex models with skip connections, multiple branches, or custom logic, subclass `nn.Module`:

Python import torch import torch.nn as nn class ResNetBlock(nn.Module): def __init__(self, in_channels, out_channels): super().__init__() self.conv1 = nn.Conv2d(in_channels, out_channels, 3, padding=1) self.bn1 = nn.BatchNorm2d(out_channels) self.conv2 = nn.Conv2d(out_channels, out_channels, 3, padding=1) self.bn2 = nn.BatchNorm2d(out_channels) self.relu = nn.ReLU() # Skip connection self.skip = nn.Identity() if in_channels == out_channels else \ nn.Conv2d(in_channels, out_channels, 1) def forward(self, x): residual = self.skip(x) x = self.relu(self.bn1(self.conv1(x))) x = self.bn2(self.conv2(x)) return self.relu(x + residual) # Skip connection! # Usage block = ResNetBlock(64, 128) x = torch.randn(4, 64, 32, 32) # Batch of 4 images y = block(x)

The Canonical PyTorch Training Loop

Every PyTorch training loop follows the same 5-step pattern. Mastering this loop is essential to all deep learning:

1. Forward Pass

Feed data through the model to get predictions: `predictions = model(X)`

2. Compute Loss

Measure prediction error: `loss = criterion(predictions, y)`

3. Backward Pass

Compute gradients via backprop: `loss.backward()`

4. Update Weights

Apply gradient descent: `optimizer.step()`

5. Zero Gradients

Clear old gradients: `optimizer.zero_grad()`

Python import torch import torch.nn as nn from torch.optim import Adam from torch.utils.data import DataLoader # Setup model = nn.Sequential(nn.Linear(784, 128), nn.ReLU(), nn.Linear(128, 10)) criterion = nn.CrossEntropyLoss() optimizer = Adam(model.parameters(), lr=0.001) train_loader = DataLoader(train_dataset, batch_size=64, shuffle=True) # Training loop model.train() # Set to training mode for epoch in range(10): total_loss = 0.0 for X_batch, y_batch in train_loader: # 1. Forward pass predictions = model(X_batch) # 2. Compute loss loss = criterion(predictions, y_batch) # 3. Backward pass loss.backward() # 4. Update weights optimizer.step() # 5. Zero gradients optimizer.zero_grad() total_loss += loss.item() print(f"Epoch {epoch+1}/10, Loss: {total_loss / len(train_loader):.4f}")

Training vs. Evaluation Mode

Some layers (Dropout, BatchNorm) behave differently during training and evaluation. Always set the model mode explicitly:

Python import torch # During training model.train() for X_batch, y_batch in train_loader: predictions = model(X_batch) # Dropout active, BatchNorm uses batch stats loss = criterion(predictions, y_batch) loss.backward() optimizer.step() optimizer.zero_grad() # During validation/testing model.eval() with torch.no_grad(): for X_batch, y_batch in val_loader: predictions = model(X_batch) # Dropout off, BatchNorm uses running stats val_loss = criterion(predictions, y_batch)

Data Loading: Dataset & DataLoader

PyTorch provides `Dataset` and `DataLoader` abstractions to efficiently load, shuffle, and batch data. For simple cases, you can subclass `Dataset`; for common datasets, use `torchvision.datasets`.

Loss Functions & Optimizers

The loss function quantifies prediction error. The optimizer updates weights to minimize loss. PyTorch provides many standard choices, each suited for different problems.

Loss Functions

Learning Rate Scheduling

Adjust learning rate during training to improve convergence. Common strategies include step decay and cosine annealing:

Python import torch.optim.lr_scheduler as lr_scheduler optimizer = optim.Adam(model.parameters(), lr=1e-3) # Step decay: reduce LR by factor every N epochs scheduler = lr_scheduler.StepLR(optimizer, step_size=10, gamma=0.1) # Cosine annealing: decay LR following cosine curve scheduler = lr_scheduler.CosineAnnealingLR(optimizer, T_max=100) # Exponential decay scheduler = lr_scheduler.ExponentialLR(optimizer, gamma=0.95) # Training loop with scheduler for epoch in range(100): for X_batch, y_batch in train_loader: predictions = model(X_batch) loss = criterion(predictions, y_batch) loss.backward() optimizer.step() optimizer.zero_grad() scheduler.step() # Update learning rate after each epoch print(f"Epoch {epoch}, LR: {optimizer.param_groups[0]['lr']}")

Complete MNIST Example

Let's build a complete deep learning pipeline on MNIST: load data, build model, train, and evaluate. This is the template you'll use for most projects.

Python import torch import torch.nn as nn from torch.optim import Adam from torch.utils.data import DataLoader from torchvision import datasets, transforms import matplotlib.pyplot as plt # 1. Data Loading transform = transforms.Compose([ transforms.ToTensor(), transforms.Normalize((0.1307,), (0.3081,)) ]) train_data = datasets.MNIST(root='./data', train=True, transform=transform, download=True) test_data = datasets.MNIST(root='./data', train=False, transform=transform, download=True) train_loader = DataLoader(train_data, batch_size=128, shuffle=True, num_workers=4) test_loader = DataLoader(test_data, batch_size=128, shuffle=False, num_workers=4) # 2. Model Definition class MNISTClassifier(nn.Module): def __init__(self): super().__init__() self.flatten = nn.Flatten() self.layers = nn.Sequential( nn.Linear(784, 256), nn.ReLU(), nn.Dropout(0.2), nn.Linear(256, 128), nn.ReLU(), nn.Dropout(0.2), nn.Linear(128, 64), nn.ReLU(), nn.Linear(64, 10) ) def forward(self, x): x = self.flatten(x) return self.layers(x) # 3. Setup model = MNISTClassifier() optimizer = Adam(model.parameters(), lr=1e-3) criterion = nn.CrossEntropyLoss() device = 'cuda' if torch.cuda.is_available() else 'cpu' model.to(device) # 4. Training Loop train_losses, val_accs = [], [] for epoch in range(10): # Train model.train() total_loss = 0.0 for X_batch, y_batch in train_loader: X_batch, y_batch = X_batch.to(device), y_batch.to(device) predictions = model(X_batch) loss = criterion(predictions, y_batch) loss.backward() optimizer.step() optimizer.zero_grad() total_loss += loss.item() avg_loss = total_loss / len(train_loader) train_losses.append(avg_loss) # Evaluate model.eval() correct, total = 0, 0 with torch.no_grad(): for X_batch, y_batch in test_loader: X_batch, y_batch = X_batch.to(device), y_batch.to(device) predictions = model(X_batch) _, predicted = torch.max(predictions.data, 1) correct += (predicted == y_batch).sum().item() total += y_batch.size(0) accuracy = correct / total val_accs.append(accuracy) print(f"Epoch {epoch+1}/10 | Loss: {avg_loss:.4f} | Accuracy: {accuracy:.4f}") print("Training complete!")

With this architecture trained for 10 epochs on MNIST, you should achieve approximately 97-98% accuracy on the test set. MNIST is a relatively easy dataset; harder datasets (CIFAR-10, ImageNet) require deeper architectures and more training.

Saving & Loading Models

Once trained, save your model for later inference or continued training. PyTorch provides two main approaches: save weights only (state_dict) or save entire model.

State Dict (Recommended)

Save only the model weights. This is the standard approach—it's portable and doesn't depend on the exact code structure:

Python import torch import torch.nn as nn # Save model = ... # Your trained model torch.save(model.state_dict(), 'model_weights.pth') # Load model = MNISTClassifier() # Recreate architecture model.load_state_dict(torch.load('model_weights.pth')) model.eval() # Set to evaluation mode # If model was trained on GPU but you're loading on CPU model.load_state_dict(torch.load('model_weights.pth', map_location='cpu'))

Save Entire Model (Simpler but Less Portable)

Python import torch # Save entire model (architecture + weights) torch.save(model, 'model_full.pth') # Load entire model model = torch.load('model_full.pth') model.eval()

Save Checkpoint (For Resuming Training)

Python import torch # Save checkpoint with epoch, optimizer state, etc. checkpoint = { 'epoch': epoch, 'model_state_dict': model.state_dict(), 'optimizer_state_dict': optimizer.state_dict(), 'loss': loss, } torch.save(checkpoint, f'checkpoint_epoch_{epoch}.pth') # Resume training checkpoint = torch.load('checkpoint_epoch_5.pth') model.load_state_dict(checkpoint['model_state_dict']) optimizer.load_state_dict(checkpoint['optimizer_state_dict']) start_epoch = checkpoint['epoch']

War Room: Debugging Neural Networks

Your PyTorch model trains but loss becomes NaN after 3 epochs. The architecture has 5 layers with ReLU activations, using SGD with learning rate 0.1. What went wrong?

Scenario

Problem: Loss = NaN after epoch 3
Model: 5-layer deep MLP, ReLU activation
Optimizer: SGD, lr=0.1
Question: How do you debug this?

Socratic Questions to Guide Debugging

Is the learning rate too high?

NaN often indicates gradient explosion. With lr=0.1 on a 5-layer network, you might have unstable updates. Try reducing to 0.01 or 0.001.

Did you normalize your input data?

Raw unnormalized data can cause gradient explosion. Always normalize inputs: subtract mean, divide by std. Without this, first-layer weights explode.

Is your network too deep without normalization?

5 layers with vanilla ReLU can suffer vanishing/exploding gradients. Add BatchNorm after each Linear layer: `Linear -> BatchNorm -> ReLU`. This stabilizes training dramatically.

Did you check for NaN in gradients?

Monitor gradient norms during training. Insert: `for p in model.parameters(): assert not torch.isnan(p.grad).any()` in your loop. This catches NaN before loss blows up.

Are weight initializations appropriate?

PyTorch uses reasonable defaults, but with deep networks, He initialization is better for ReLU. Most layers already use this, but verify.

What happens if you use gradient clipping?

Add `torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)` after `loss.backward()`. This caps gradient magnitude and prevents explosion.

Should you try a different optimizer?

SGD with momentum or Adam handles gradient issues better. Try: `optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)`. Adaptive optimizers are more forgiving.

Is your loss function appropriate?

For classification, always use `CrossEntropyLoss` (not softmax + NLLLoss manually). For regression, check for unbounded targets or outliers causing large gradients.

Quick Debugging Checklist

Python # Add this to your training loop to catch problems early import torch for epoch in range(num_epochs): for X, y in train_loader: logits = model(X) loss = criterion(logits, y) # Check loss before backward assert not torch.isnan(loss), f"NaN loss at epoch {epoch}" assert loss > 0, f"Non-positive loss: {loss}" loss.backward() # Check gradients before step for name, param in model.named_parameters(): if param.grad is not None: assert not torch.isnan(param.grad).any(), f"NaN grad in {name}" assert not torch.isinf(param.grad).any(), f"Inf grad in {name}" # Clip gradients (optional but helpful) torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0) optimizer.step() optimizer.zero_grad() print("Training completed successfully!")

Key Takeaways

Tensors are NumPy++

Tensors support GPU acceleration and automatic differentiation. Learn their operations deeply.

Autograd is Magic

Set `requires_grad=True`, build computation graphs, call `backward()`. Gradients compute automatically via chain rule.

5-Step Training Loop

Forward → Loss → Backward → Step → Zero. Master this pattern; everything else builds from it.

nn.Module is the Foundation

Subclass for complex models. Use Sequential for simple stacks. Manage parameters and modes systematically.

Data Loading Matters

DataLoader handles batching, shuffling, and parallel loading. Custom Datasets give you flexibility.

Normalize Everything

Normalize inputs, use BatchNorm in deep networks, apply weight initialization. These prevent training instabilities.
