PART II: NEURAL NETWORKS & DEEP LEARNING

Natural Language Processing with RNNs and Attention

Mastering Sequence Learning: From Character Generation to Machine Translation

This chapter explores how RNNs process language sequentially and how attention mechanisms focus on relevant information. We'll build character-level language models, sentiment classifiers, and end-to-end machine translation systems—the foundational techniques that led to modern transformers.

Mental Model: Attention as a Spotlight

The Simplest Explanation

Imagine you're reading a long essay and your brain naturally focuses on the most relevant words. When translating "The bank vault" from English to Spanish, you pay attention to both "bank" and "vault" but focus more heavily on their relationship. Attention in neural networks works the same way:

  • Query: "What am I trying to understand right now?" (the target word being generated)
  • Keys: "What could be relevant?" (all source words)
  • Values: "What is the actual information?" (the encoded source words)

The attention mechanism computes how much each source word is relevant (via a softmax over query-key similarities) and mixes their values proportionally.

Why This Matters

Before attention, RNNs struggled with long sequences because information had to flow through a single "bottleneck" vector. Attention allows the model to directly access any part of the input, making translation, summarization, and other tasks dramatically more accurate.

Generating Shakespearean Text Using a Character-RNN

A character-level RNN learns to predict the next character in a sequence by training on character-by-character data. Despite its simplicity, it captures grammar, vocabulary, and even writing style.

How Character RNNs Work

At each time step, the RNN:

  1. Takes a single character as input
  2. Updates its hidden state based on previous state and input
  3. Outputs probabilities for the next character
  4. Samples from this distribution to generate text

Sample Output from Shakespeare Model

Trained purely on character sequences, the model learns words, grammar, and punctuation automatically.

Building a Character RNN in PyTorch

import torch
import torch.nn as nn

class CharRNN(nn.Module):
    def __init__(self, input_size, hidden_size, output_size):
        super().__init__()
        self.hidden_size = hidden_size
        self.embedding = nn.Embedding(input_size, hidden_size)
        self.rnn = nn.LSTM(hidden_size, hidden_size, batch_first=True)
        self.fc = nn.Linear(hidden_size, output_size)

    def forward(self, x, hidden=None):
        x = self.embedding(x)
        out, hidden = self.rnn(x, hidden)
        out = self.fc(out)
        return out, hidden

Encoder-Decoder Architecture for Neural Machine Translation

Machine translation maps sequences to sequences. The encoder-decoder (seq2seq) architecture:

  • Encoder: Processes the source sentence and outputs a fixed context vector
  • Decoder: Uses the context vector to generate the target sentence

The Information Flow

Encoder-decoder data flow

Mathematical Formulation

$$c = h_T^{(e)} = \text{final hidden state}$$
$$h_t^{(d)} = \text{LSTM}(h_{t-1}^{(d)}, y_{t-1}, c)$$
$$P(y_t | y_1, \ldots, y_{t-1}, c) = \text{softmax}(W \cdot h_t^{(d)} + b)$$

Code: Basic Seq2Seq Model

class Seq2SeqModel(nn.Module):
    def __init__(self, encoder, decoder):
        super().__init__()
        self.encoder = encoder
        self.decoder = decoder

    def encode(self, src):
        _, (hidden, cell) = self.encoder(src)
        return hidden, cell

    def forward(self, src, tgt, teacher_forcing_ratio=0.5):
        batch_size = src.size(0)
        tgt_len = tgt.size(1)
        vocab_size = self.decoder.output_size
        outputs = torch.zeros(batch_size, tgt_len, vocab_size)

        hidden, cell = self.encode(src)
        decoder_input = tgt[:, 0].unsqueeze(1)

        for t in range(1, tgt_len):
            output, hidden, cell = self.decoder(decoder_input, hidden, cell)
            outputs[:, t] = output

            use_teacher = random.random() < teacher_forcing_ratio
            decoder_input = tgt[:, t].unsqueeze(1) if use_teacher else output.argmax(1).unsqueeze(1)

        return outputs

Teacher Forcing

During training, we feed the true previous word to the decoder. This speeds up training but can cause mismatch with inference. A solution is curriculum learning: gradually reduce teacher forcing probability.

        self.lstm = nn.LSTM(input_size, hidden_size, bidirectional=True, batch_first=True)

        out, (hidden, cell) = self.lstm(x)

        return out

BiLSTMs are powerful for encoding but cannot be used for generation, since we don't have future tokens at inference time.

Beam Search: Finding Better Translations

Greedy decoding picks the highest-probability token at each step. This is fast but often suboptimal. Beam search explores multiple hypotheses in parallel.

Greedy vs. Beam Search

Beam search (K=3) maintains multiple hypotheses

Algorithm Overview

  1. Initialize: Start with K copies of decoder state (K = beam width)
  2. Expand: Generate all possible next tokens from K hypotheses
  3. Prune: Keep K hypotheses with highest cumulative probability
  4. Repeat: Continue until hypotheses reach end-of-sentence
  5. Score: Normalize by length to avoid bias toward short sequences

Attention Mechanisms: Bahdanau & Luong Attention

Basic seq2seq compresses the entire source into a single context vector, losing information for long sentences. Attention solves this by letting the decoder focus on different source parts at each step.

The Attention Formula

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

The softmax produces weights indicating relevance. Multiplying by V produces a weighted context vector.

Bahdanau vs. Luong Attention

Aspect Bahdanau Luong
Score Function $\mathbf{v}^T \tanh(W_1 q + W_2 k_i)$ $q^T k_i$
Computation More parameters Fewer parameters
Year 2014 (Original) 2015 (General)

Computing Attention Weights

Step-by-Step Breakdown

  1. Score: $s_i = \text{score}(q, k_i)$
  2. Weights: $\alpha_i = \frac{\exp(s_i)}{\sum_j \exp(s_j)}$
  3. Context: $c = \sum_i \alpha_i v_i$
  4. Use: Concatenate context with decoder state

Code: Luong Attention Layer

class LuongAttention(nn.Module):
    def __init__(self, hidden_dim):
        super().__init__()
        self.attention = nn.Linear(hidden_dim, hidden_dim)

    def forward(self, decoder_hidden, encoder_outputs):
        query = self.attention(decoder_hidden).unsqueeze(2)
        scores = torch.matmul(encoder_outputs, query).squeeze(2)
        weights = F.softmax(scores, dim=1)
        context = torch.sum(encoder_outputs * weights.unsqueeze(2), dim=1)
        return context, weights

End-to-End Seq2Seq with Attention

class AttentionalSeq2Seq(nn.Module):
    def __init__(self, encoder, decoder, attention):
        super().__init__()
        self.encoder = encoder
        self.decoder = decoder
        self.attention = attention

    def forward(self, src, tgt):
        encoder_outputs, hidden = self.encoder(src)
        outputs = []
        decoder_input = tgt[:, 0].unsqueeze(1)

        for t in range(1, tgt.size(1)):
            context, _ = self.attention(hidden[-1], encoder_outputs)
            output, hidden = self.decoder(decoder_input, hidden, context)
            outputs.append(output)
            decoder_input = tgt[:, t].unsqueeze(1)

        return torch.cat(outputs, dim=1)

Visual Attention: Beyond Text

Attention mechanisms extend to images. For image captioning, a CNN encodes the image into feature maps, and an RNN decoder generates captions word-by-word while an attention layer focuses on relevant regions.

Image-to-Caption Generation

Attention over image regions when predicting words

Architecture: CNN + Attention + RNN

class ImageCaptioner(nn.Module):
    def __init__(self, cnn_backbone, hidden_dim, vocab_size):
        super().__init__()
        self.cnn = cnn_backbone  # e.g., ResNet-50
        self.attention = LuongAttention(hidden_dim)
        self.decoder = nn.LSTMCell(hidden_dim + hidden_dim, hidden_dim)
        self.fc = nn.Linear(hidden_dim, vocab_size)

    def forward(self, image, max_len=20):
        features = self.cnn(image)  # (batch, hidden_dim, H, W)
        features_flat = features.permute(0, 2, 3, 1)
        features_flat = features_flat.view(features_flat.size(0), -1, features_flat.size(-1))

        batch_size = image.size(0)
        h = torch.zeros(batch_size, self.hidden_dim)
        c = torch.zeros(batch_size, self.hidden_dim)
        captions = []

        for t in range(max_len):
            context, _ = self.attention(h, features_flat)
            h, c = self.decoder(torch.cat([h, context], dim=1), (h, c))
            logits = self.fc(h)
            captions.append(logits)

        return torch.stack(captions, dim=1)

Applications of Visual Attention

  • Image Captioning: Generate descriptions focusing on regions
  • Visual Question Answering (VQA): Answer questions about images
  • Object Tracking: Maintain focus on moving objects
  • Interpretability: Visualize decision influences

From RNNs to Transformers: The Bridge to Chapter 15

RNNs process sequences sequentially, limiting parallelization. Transformers replace recurrence with stacked attention layers, enabling:

  • Full Parallelization: Process all positions simultaneously
  • Longer Context: Better long-range dependency modeling
  • Stable Gradients: No vanishing/exploding gradients
  • Transfer Learning: Pre-trained on massive datasets

The Transformer Architecture (Overview)

Input Embeddings
↓
Positional Encodings
↓
[Encoder: Multi-Head Attention + FFN] × N
↓
[Decoder: Masked Attn + Cross-Attn + FFN] × N
↓
Output Linear + Softmax

Transformers replace RNNs with stacked attention. Chapter 15 covers details.

Key Insight

The attention mechanism from this chapter is the core of Transformers. Multi-head attention applies multiple attention operations in parallel. Positional encodings handle token order (attention is permutation-invariant). The Transformer is a highly parallel attention-only architecture.
