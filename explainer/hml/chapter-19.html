Part II: Deep Learning

Reinforcement Learning

Training agents to learn by interacting with environments, optimizing rewards through exploration and exploitation—from AlphaGo to language model alignment.

What Is Reinforcement Learning?

Reinforcement Learning (RL) is a paradigm where an agent learns to make optimal decisions by interacting with an environment. Unlike supervised learning (which needs labeled data) or unsupervised learning (which finds patterns), RL learns through trial-and-error—receiving rewards for good actions and penalties for bad ones.

Note: RL powers some of the most remarkable AI achievements: DeepMind's AlphaGo (2016) beat the world Go champion Lee Sedol, and AlphaFold (2020) revolutionized protein structure prediction—earning the 2024 Nobel Prize in Chemistry.

The RL Loop

The core interaction between agent and environment:

  1. State (s): Current observation of the environment
  2. Action (a): The agent chooses an action to take
  3. Reward (r): Environment provides immediate reward/penalty
  4. Next State (s'): Environment transitions to new state
  5. Repeat: Agent uses experience to improve future action selection

Key RL Components

  • Agent: The learner making decisions
  • Environment: The world the agent interacts with
  • State: Complete description of the environment
  • Action: Choices available to the agent
  • Reward: Signal indicating action quality
  • Policy: Mapping from states to actions (what the agent learns)

Feynman Mental Model: Training a Dog

RL is like training a dog to fetch a ball:

  • State: Dog sees ball, sees owner, sees obstacles
  • Action: Run forward, jump, turn left, grab
  • Reward: Get a treat if it fetches correctly
  • Learning: Dog tries different behaviors, remembers which led to treats
  • Policy: Over time, dog learns "when I see ball + owner pointing = run and grab = treat"

Just like a dog learns from treats and praise, RL agents learn from rewards and penalties.

From Dog Training to AlphaGo

The same principle scales to complex games:

  • State: Board position (19×19 stones in Go)
  • Action: Place stone at any valid location
  • Reward: +1 for winning, -1 for losing
  • Learning: Neural network learns which moves lead to victory across billions of games

Markov Decision Processes (MDPs)

An MDP formally defines an RL problem. It consists of:

  • S: Set of states
  • A: Set of actions
  • P(s'|s,a): Transition probability (Markov property)
  • R(s,a): Reward function
  • γ (gamma): Discount factor (0 to 1)

The Markov Property

An MDP satisfies the Markov property: the next state depends only on the current state and action, not on history. In other words:

$$P(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \ldots) = P(s_{t+1} \mid s_t, a_t)$$
Example: In chess, what matters is the current board position, not how you got there. This makes MDPs computationally tractable.

Returns and Discounting

The return (or cumulative reward) is:

$$G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$$

The discount factor γ balances immediate vs. long-term rewards:

  • γ ≈ 0: Agent is myopic (greedy), only cares about immediate reward
  • γ ≈ 1: Agent plans far ahead, values all future rewards equally
  • Typical: γ = 0.99 or 0.999 (weight future rewards but discount them)

The Bellman Equation & Optimal Policy

Value Function

The state value function $V(s)$ is the expected return from state $s$ under policy $\pi$:

$$V^\pi(s) = \mathbb{E}[G_t \mid S_t = s] = \mathbb{E}\left[\sum_{k=0}^{\infty} \gamma^k r_{t+k+1} \mid S_t = s\right]$$

The Bellman Equation

This recursive equation relates a state's value to its successor states:

$$V^\pi(s) = \sum_a \pi(a|s) \sum_{s',r} P(s',r|s,a)[r + \gamma V^\pi(s')]$$

Bellman Intuition

The value of being in a state is the immediate reward plus the discounted value of the next state. By solving this equation, we find optimal values.

Optimal Policy & Value Function

The optimal value function is the maximum achievable return:

$$V^*(s) = \max_\pi V^\pi(s)$$

The optimal policy greedily selects actions that lead to the highest-value next states:

$$\pi^*(s) = \arg\max_a \sum_{s',r} P(s',r|s,a)[r + \gamma V^*(s')]$$

Q-Learning: Learning Action Values

Q-Values & the Q-Function

Instead of learning state values, we learn action values (Q-values): the expected return from taking action $a$ in state $s$:

$$Q(s,a) = \mathbb{E}[G_t \mid S_t = s, A_t = a]$$

Once we have Q-values, the optimal policy is simply:

$$\pi^*(s) = \arg\max_a Q^*(s,a)$$

The Q-Learning Update Rule

Q-Learning is an off-policy algorithm that learns the optimal policy while exploring. The update rule is:

$$Q(s,a) \leftarrow Q(s,a) + \alpha \left[r + \gamma \max_{a'} Q(s',a') - Q(s,a)\right]$$

Breaking Down the Update

  • $\alpha$: Learning rate (controls step size)
  • $r + \gamma \max_{a'} Q(s',a')$: Target (observed return + best next action)
  • $Q(s,a)$: Current estimate
  • $r + \gamma \max_{a'} Q(s',a') - Q(s,a)$: Temporal Difference (TD) error

Q-Learning Algorithm

Initialize Q(s,a) = 0 for all states and actions
for each episode:
    s = initial state
    while s is not terminal:
        a = select action (ε-greedy: mostly argmax Q(s,·), randomly explore with ε)
        take action a, observe r and s'
        Q(s,a) ← Q(s,a) + α[r + γ max_a' Q(s',a') - Q(s,a)]
        s ← s'

Exploration vs. Exploitation

Q-Learning uses ε-greedy exploration:

  • With probability 1-ε: Exploit (choose best known action)
  • With probability ε: Explore (choose random action)

Small ε (e.g., 0.1) balances learning with exploitation. Too high: agent wastes time on random actions. Too low: agent gets stuck in local optima.

Deep Q-Networks (DQN)

Q-Learning works great for small problems, but fails with high-dimensional state spaces (e.g., raw pixels from Atari). Deep Q-Networks replace the Q-table with a neural network:

$$Q(s,a) \approx Q(s,a;\theta)$$

where $\theta$ are network parameters.

DQN Architecture

Input: Game frames (84×84 pixels) → CNN layers → Fully connected layers → Output: Q-values for each action

Key Innovation: Experience Replay

DQN samples mini-batches of past experiences $(s,a,r,s')$ from a replay buffer instead of learning on consecutive transitions. This:

  • Breaks correlation between consecutive samples
  • Allows reuse of old experiences
  • Makes learning more stable

Target Network

DQN also uses a separate target network $Q(s,a;\theta^-)$ that's updated slowly. The loss function is:

$$L(\theta) = \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}}\left[\left(r + \gamma \max_{a'} Q(s',a';\theta^-) - Q(s,a;\theta)\right)^2\right]$$

The target network is updated every N steps: $\theta^- \leftarrow \theta$

DQN Training Loop

Initialize Q-network and target network
replay_buffer = empty
for episode = 1 to num_episodes:
    s = reset environment
    while not done:
        a = ε-greedy(Q-network, s)
        r, s' = take action a
        store (s, a, r, s') in replay_buffer

        if len(replay_buffer) > batch_size:
            sample batch from replay_buffer
            compute loss using target network
            update Q-network with gradient descent

        s = s'

    if episode % target_update_freq == 0:
        target_network ← Q-network

Double DQN Improvement

Standard DQN overestimates Q-values because it uses $\max_{a'} Q(s',a')$. Double DQN decouples action selection from evaluation:

$$y = r + \gamma Q(s', \arg\max_{a'} Q(s',a';\theta);\theta^-)$$

This reduces overestimation and improves stability.

PyTorch DQN Implementation Skeleton

import torch
import torch.nn as nn

class DQN(nn.Module):
    def __init__(self, state_size, action_size):
        super().__init__()
        self.fc1 = nn.Linear(state_size, 128)
        self.fc2 = nn.Linear(128, 128)
        self.fc3 = nn.Linear(128, action_size)

    def forward(self, state):
        x = torch.relu(self.fc1(state))
        x = torch.relu(self.fc2(x))
        return self.fc3(x)

# Training loop
model = DQN(state_size, action_size)
target = DQN(state_size, action_size)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for batch_s, batch_a, batch_r, batch_s_prime in replay_buffer:
    # Current Q-values
    q_vals = model(batch_s).gather(1, batch_a.unsqueeze(1))

    # Target Q-values
    with torch.no_grad():
        target_q = batch_r + gamma * target(batch_s_prime).max(1)[0]

    # Loss
    loss = nn.MSELoss()(q_vals.squeeze(), target_q)

    # Update
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
DQN Limitations: Works only for discrete action spaces. For continuous actions, policy gradients are better suited.

Policy Gradient Methods

From Value Functions to Policies

Instead of learning value functions and deriving a policy, policy gradient methods directly learn the policy by maximizing expected return.

Policy Parameterization

Represent the policy as a neural network:

$$\pi(a|s;\theta) = \text{probability of action } a \text{ given state } s$$

For discrete actions: softmax output. For continuous actions: Gaussian mean and variance.

The Policy Gradient Theorem

The key insight: the gradient of expected return with respect to policy parameters is:

$$\nabla_\theta J(\theta) = \mathbb{E}[G_t \nabla_\theta \log \pi(a|s;\theta)]$$

This means: increase log probability of actions proportional to their returns.

REINFORCE Algorithm

The simplest policy gradient method:

$$\theta \leftarrow \theta + \alpha G_t \nabla_\theta \log \pi(a_t|s_t;\theta)$$ 
Initialize policy π(a|s;θ)
for episode = 1 to num_episodes:
    s = reset
    trajectory = []
    while not done:
        a ~ π(a|s;θ)  # Sample action
        r, s' = take action
        trajectory.append((s, a, r))
        s = s'

    # Compute returns
    for t in range(len(trajectory)):
        G_t = sum(γ^k * trajectory[t+k].r for all future rewards)
        θ ← θ + α * G_t * ∇_θ log π(a_t|s_t;θ)

Policy Gradient Visualization

Landscape shows how return changes with policy parameters. Arrows point toward higher returns.

Issues with REINFORCE

  • High variance: Each return uses only one sample trajectory
  • Slow learning: Takes many episodes to estimate good gradients
  • Solution: Use baseline functions (value functions) to reduce variance

Actor-Critic Methods

Combining Value and Policy

Actor-Critic methods use two networks:

  • Actor: Policy network $\pi(a|s;\theta_\pi)$ — learns what actions to take
  • Critic: Value network $V(s;\theta_v)$ — estimates action quality (reduces variance)

Advantage Function

The advantage measures how much better an action is than the baseline:

$$A(s,a) = Q(s,a) - V(s)$$

Or using TD error:

$$A(s,a) = r + \gamma V(s') - V(s)$$

Actor-Critic Update Rules

Actor: Policy gradient with advantage baseline:

$$\theta_\pi \leftarrow \theta_\pi + \alpha_\pi A(s,a) \nabla_\theta \log \pi(a|s;\theta_\pi)$$

Critic: Temporal difference learning:

$$\theta_v \leftarrow \theta_v + \alpha_v (r + \gamma V(s';\theta_v) - V(s;\theta_v)) \nabla_\theta V(s;\theta_v)$$

Benefits of Actor-Critic

  • Lower variance: Critic provides baseline, reducing gradient noise
  • Faster convergence: Learns from individual transitions (not full episodes)
  • Scalable: Works for both discrete and continuous action spaces

A2C (Advantage Actor-Critic)

Synchronous version using mini-batch gradient updates.

A3C (Asynchronous Advantage Actor-Critic)

Multiple parallel workers collect experience asynchronously, sharing gradients. Faster and more efficient.

PyTorch Actor-Critic Skeleton

class Actor(nn.Module):
    def __init__(self, state_size, action_size):
        super().__init__()
        self.fc = nn.Sequential(
            nn.Linear(state_size, 128), nn.ReLU(),
            nn.Linear(128, 128), nn.ReLU()
        )
        self.policy = nn.Linear(128, action_size)

    def forward(self, state):
        x = self.fc(state)
        return torch.softmax(self.policy(x), dim=-1)

class Critic(nn.Module):
    def __init__(self, state_size):
        super().__init__()
        self.fc = nn.Sequential(
            nn.Linear(state_size, 128), nn.ReLU(),
            nn.Linear(128, 128), nn.ReLU(),
            nn.Linear(128, 1)
        )

    def forward(self, state):
        return self.fc(state)

# Training
actor = Actor(state_size, action_size)
critic = Critic(state_size)
actor_opt = torch.optim.Adam(actor.parameters(), lr=1e-3)
critic_opt = torch.optim.Adam(critic.parameters(), lr=1e-3)

for state, action, reward, next_state in batch:
    # Critic loss (TD error)
    value = critic(state)
    next_value = critic(next_state)
    td_error = reward + gamma * next_value - value
    critic_loss = td_error.pow(2).mean()

    # Actor loss (policy gradient with advantage)
    log_prob = torch.log(actor(state)[action])
    actor_loss = -(log_prob * td_error.detach()).mean()

    # Updates
    critic_opt.zero_grad()
    critic_loss.backward()
    critic_opt.step()

    actor_opt.zero_grad()
    actor_loss.backward()
    actor_opt.step()

Trust Region Methods: PPO & TRPO

The Problem: Large Policy Updates

Naive policy gradients can fail if the policy changes too much in one step:

  • New policy might be far from old policy (different distributions)
  • Off-policy corrections become invalid
  • Learning becomes unstable

Trust Region Policy Optimization (TRPO)

TRPO constrains policy updates to stay within a trust region, measured by KL divergence:

$$\max_\theta \mathbb{E}[A_t] \quad \text{subject to} \quad \mathbb{E}[\text{KL}(\pi_{old} \| \pi_\theta)] \leq \delta$$

This ensures the new policy stays close to the old policy, preserving learning stability.

Challenge: TRPO requires computing Hessian-vector products, which is computationally expensive.

Proximal Policy Optimization (PPO)

PPO is a simpler, more practical alternative to TRPO. It uses a clipped objective:

$$L^{\text{CLIP}}(\theta) = \mathbb{E}[L^t(\theta)] \quad \text{where}$$
$$L^t(\theta) = \min\left(r_t(\theta) A_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) A_t\right)$$

The probability ratio is:

$$r_t(\theta) = \frac{\pi(a_t|s_t;\theta)}{\pi(a_t|s_t;\theta_{old})}$$

PPO Clipping Intuition

If advantage is positive (good action): increase log probability, but stop at $(1+\epsilon)$ to prevent overconfidence

If advantage is negative (bad action): decrease log probability, but stop at $(1-\epsilon)$ to prevent overly large updates

This is simpler than TRPO's constraint and works remarkably well in practice.

PPO Algorithm

Initialize policy π(a|s;θ) and value V(s;φ)
for episode = 1 to num_episodes:
    # Collect trajectory
    trajectory = []
    s = reset
    for t = 1 to horizon:
        a ~ π(a|s;θ)
        r, s' = take action
        trajectory.append((s, a, r, log_prob))
        s = s'

    # Compute advantages
    advantages = compute_advantages(trajectory, V, γ)

    # PPO update (K epochs)
    for k = 1 to K_epochs:
        sample mini-batch from trajectory

        for s, a, advantage, old_log_prob in mini_batch:
            new_log_prob = log π(a|s;θ_new)
            ratio = exp(new_log_prob - old_log_prob)

            # Clipped objective
            clipped_ratio = clip(ratio, 1-ε, 1+ε)
            loss = -min(ratio * advantage, clipped_ratio * advantage)

            # Value loss
            value_loss = (V(s) - target_value)^2

            total_loss = loss + c_value * value_loss

            # Gradient update
            θ ← θ - α ∇_θ total_loss

Why PPO Works So Well

  • Simple to implement: No Hessian computations
  • Sample efficient: Reuses data from multiple epochs
  • Stable: Clipping prevents catastrophic policy changes
  • Versatile: Works for discrete, continuous, and hybrid action spaces
  • State-of-the-art: Used in ChatGPT, GPT-4, and other major LLMs
Tip: PPO is often the first choice for new RL problems due to its simplicity and reliability.

RLHF: Reinforcement Learning from Human Feedback

From Supervised Learning to Alignment

Pre-trained language models (like GPT) are trained on next-token prediction. But this doesn't guarantee they'll be helpful, harmless, and honest. RLHF fine-tunes models using human preferences.

The RLHF Pipeline

  1. Pre-training: Train language model on large text corpus
  2. Supervised Fine-tuning: Fine-tune on high-quality examples of desired behavior
  3. Reward Modeling: Train reward model on human comparisons
  4. RL Fine-tuning: Use PPO to optimize policy against reward model

Reward Model

Human raters compare pairs of model outputs and label which is better. This creates a dataset of preferences. Train a reward model:

$$r_\phi(s, a) = \mathbb{R} \quad \text{(single reward score)}$$

Trained via supervised learning on the preference labels.

RL with Reward Model

Once we have a reward model, we can treat language generation as an RL problem:

  • State: Prompt tokens
  • Action: Generate next token
  • Reward: Reward model score + KL penalty (keep model close to original)
  • Policy: Language model to optimize

The objective is:

$$\max_\theta \mathbb{E}[r(s,a)] - \beta \text{KL}(\pi_\theta || \pi_{ref})$$

The KL term prevents the model from drifting too far from the original language model (language collapse).

Why RLHF Works

  • Scalable feedback: Reward model generalizes human preferences to new examples
  • Flexible objectives: Can optimize for different criteria (truthfulness, harmlessness, helpfulness)
  • Alignment: Directly optimizes what humans care about, not just predicting next token

Practical Implementation

import torch
from transformers import AutoModelForCausalLM

# Load pre-trained model
model = AutoModelForCausalLM.from_pretrained("gpt2")
ref_model = AutoModelForCausalLM.from_pretrained("gpt2")

# Reward model (trained on human preferences)
reward_model = load_reward_model()

# PPO training loop for language model
optimizer = torch.optim.Adam(model.parameters(), lr=5e-6)

for prompt_batch in prompts:
    # Generate responses
    with torch.no_grad():
        responses = model.generate(prompt_batch, max_length=50, do_sample=True)

    # Get rewards
    rewards = reward_model(prompt_batch, responses)

    # KL penalty with reference model
    log_probs = model(prompt_batch + responses).logits
    ref_log_probs = ref_model(prompt_batch + responses).logits
    kl_penalty = torch.nn.functional.kl_div(log_probs, ref_log_probs)

    # Objective
    objective = rewards.mean() - 0.01 * kl_penalty

    # Optimize
    loss = -objective
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Advanced Topics: Reward Shaping & Sparse Rewards

The Sparse Reward Problem

In many real-world problems, rewards are sparse:

  • Game: Only reward at end (+1 for win, -1 for loss)
  • Robotics: Only reward when task succeeds
  • Navigation: No reward until reaching goal

With sparse rewards, exploration becomes extremely difficult—the agent rarely receives feedback about good vs. bad actions.

Reward Shaping

Add intermediate rewards to guide learning without changing the optimal policy:

$$r'(s,a,s') = r(s,a,s') + \gamma \Phi(s') - \Phi(s)$$

where $\Phi(s)$ is a potential function.

Example: Robot Reaching a Target

  • Sparse reward: +1 if target reached, 0 otherwise
  • Shaped reward: Subtract distance to target (potential function = negative distance)
  • Result: Agent gets reward every step proportional to progress

Intrinsic Motivation

Alternative to manual reward shaping: use curiosity as an intrinsic reward.

$$r_{total} = r_{environment} + \beta r_{intrinsic}$$

Common intrinsic rewards:

  • Curiosity: Reward prediction error (states the model hasn't seen)
  • Empowerment: Ability to reach diverse states
  • Random Network Distillation: Surprise of neural network features

Summary: From Theory to Impact

Q-Learning

Simple, model-free, learns action values. Great for discrete action spaces, but doesn't scale to high-dimensional observations without function approximation.

Deep Q-Networks

Neural network function approximator for Q-Learning. Enables Atari mastery. Key innovations: experience replay, target networks, double DQN.

Policy Gradients

Directly optimize policy by following gradient of expected return. Simple but high variance. Foundation for modern RL.

Actor-Critic

Combines policy and value networks. Lower variance than policy gradients, works for continuous actions. A2C/A3C are practical variants.

PPO & TRPO

Trust region methods for stable policy updates. PPO is simpler and more practical—state-of-the-art for most applications including LLM alignment.

RLHF

Human feedback → Reward model → RL fine-tuning. Powers ChatGPT, GPT-4, and modern aligned language models.

Key Takeaways

  • Markov Property: Future states depend only on present state/action, enabling dynamic programming
  • Bellman Equation: Recursive relationship between state values—foundation of all RL algorithms
  • Exploration-Exploitation: Critical tradeoff that must be managed carefully
  • Function Approximation: Neural networks enable RL in high-dimensional spaces
  • Stability: Large policy updates cause instability; trust regions, clipping, and target networks help
  • Alignment: RLHF shows how to optimize AI systems for human values
