Linear Regression

The foundation of supervised learning: predicting a continuous value through a linear combination of features.

The Linear Regression Model

Linear regression assumes a linear relationship between input features and the target variable:

\[\hat{y} = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \ldots + \theta_n x_n = h_\theta(x)\]

Where:

  • θ are the parameters (weights)
  • x are the feature values
  • ŷ is the prediction

Mean Squared Error (MSE) Loss

We measure model performance using MSE, the average squared difference between predictions and actual values:

\[MSE = \frac{1}{m} \sum_{i=1}^{m} (h_\theta(x^{(i)}) - y^{(i)})^2\]

Minimizing this loss function finds the optimal parameters. Note that MSE is convex, meaning it has a single global minimum—perfect for optimization.

The Normal Equation

A closed-form, non-iterative solution to linear regression.

Vector Form

We can write the prediction as:

\[\hat{\mathbf{y}} = \mathbf{X} \boldsymbol{\theta}\]

Where X is the design matrix (m × n+1) containing all training examples and features, and θ is the parameter vector.

The Solution

By taking the derivative of MSE with respect to θ and setting it to zero:

\[\boldsymbol{\theta} = (\mathbf{X}^T \mathbf{X})^{-1} \mathbf{X}^T \mathbf{y}\]

This is the Normal Equation—a direct formula for optimal parameters.

Computational Complexity Analysis

Aspect Normal Equation Gradient Descent
Time Complexity O(n³) for matrix inversion O(k·m·n) where k=iterations
Space Complexity O(n²) O(n)
Scaling Slow for large n Scales to millions of features
Iterations None (direct solution) Many (convergence dependent)

SVD Decomposition Alternative

For numerical stability, the Singular Value Decomposition (SVD) provides an alternative:

\[\mathbf{X} = \mathbf{U} \boldsymbol{\Sigma} \mathbf{V}^T\]

Then:

\[\boldsymbol{\theta} = \mathbf{V} \boldsymbol{\Sigma}^{-1} \mathbf{U}^T \mathbf{y}\]

SVD is more numerically stable than directly computing XTX, especially when X is ill-conditioned. NumPy and scikit-learn use SVD internally for this reason.

# Python implementation using Normal Equation import numpy as np def normal_equation(X, y): """Solve using Normal Equation: θ = (X^T X)^-1 X^T y""" X_with_bias = np.column_stack([np.ones(len(X)), X]) theta = np.linalg.inv(X_with_bias.T @ X_with_bias) @ X_with_bias.T @ y return theta # Better: Use SVD via lstsq theta_lstsq, _, _, _ = np.linalg.lstsq(X_with_bias, y, rcond=None)

Gradient Descent

An iterative algorithm that repeatedly adjusts parameters to reduce the loss function.

Core Algorithm

Gradient Descent works by computing the gradient (derivative) of the loss function and moving parameters in the opposite direction:

\[\boldsymbol{\theta}^{\text{next}} = \boldsymbol{\theta} - \eta \nabla J(\boldsymbol{\theta})\]

Where:

  • η (eta) is the learning rate—controls step size
  • ∇J(θ) is the gradient of the loss function
  • We move against the gradient (steepest descent)

Gradient for MSE Loss

The gradient with respect to parameter θj is:

\[\frac{\partial J}{\partial \theta_j} = \frac{2}{m} \sum_{i=1}^{m} (h_\theta(x^{(i)}) - y^{(i)}) x_j^{(i)}\]

This tells us how much to adjust each parameter to reduce error.

Batch Gradient Descent

Use the entire training set to compute gradients before each update.

Pros: Stable, efficient computations

Cons: Slow on very large datasets, may get stuck in local minima

Stochastic Gradient Descent (SGD)

Update parameters using one random example at a time.

Pros: Fast, escapes local minima, online learning

Cons: Noisy updates, harder to converge

Mini-Batch Gradient Descent

Update using a small batch (32-256 examples) at a time.

Pros: Best of both worlds—stable and fast

Cons: Requires tuning batch size

Feature Scaling

Features with different scales can make gradient descent converge slowly or diverge.

\[x_{\text{scaled}} = \frac{x - \mu}{\sigma}\]

Standardization (zero mean, unit variance) helps tremendously.

# Gradient Descent from scratch def gradient_descent(X, y, learning_rate=0.01, iterations=1000): m = len(y) theta = np.zeros(X.shape[1]) costs = [] for i in range(iterations): predictions = X @ theta errors = predictions - y gradient = (2/m) * (X.T @ errors) theta -= learning_rate * gradient cost = np.mean(errors**2) costs.append(cost) return theta, costs # With scikit-learn from sklearn.linear_model import SGDRegressor model = SGDRegressor(loss='squared_loss', learning_rate='optimal')

Polynomial Regression

When linear relationships aren't enough: add polynomial features to capture curves.

The Idea

Instead of:

\[\hat{y} = \theta_0 + \theta_1 x\]

We use:

\[\hat{y} = \theta_0 + \theta_1 x + \theta_2 x^2 + \theta_3 x^3 + \ldots\]

Polynomial regression is still linear in the parameters—it's just a special case of linear regression with engineered features.

Learning Curves

Diagnose whether your model is underfitting or overfitting by analyzing performance vs training set size.

What Are Learning Curves?

Learning curves plot training and validation errors as a function of training set size. They reveal:

  • If adding more data will help: convergence patterns
  • Model bias: train and validation error at large m
  • Model variance: gap between train and validation error
  • Bias-variance tradeoff: the fundamental learning tradeoff

Interactive Learning Curves

Observe how different model complexities behave as we increase training data.

Blue line: Training error. Red line: Validation error. Good models have small gap between train/val error.

Interpreting Learning Curves

High Bias (Underfitting): Both train and validation errors are high and remain high as m increases. Adding more data won't help—the model is too simple.

High Variance (Overfitting): Train error is low, validation error is high. Large gap indicates overfitting. More data may help bridge this gap.

Just Right: Train error and validation error are both low and close together, with both gradually decreasing as m increases.

from sklearn.model_selection import learning_curve # Generate learning curves train_sizes, train_scores, val_scores = learning_curve( estimator=LinearRegression(), X=X, y=y, train_sizes=np.linspace(0.1, 1.0, 10), cv=5, scoring='neg_mean_squared_error' ) train_mean = -train_scores.mean(axis=1) val_mean = -val_scores.mean(axis=1) plt.plot(train_sizes, train_mean, label='Train Error') plt.plot(train_sizes, val_mean, label='Validation Error') plt.legend()

Regularization

Control model complexity and combat overfitting by penalizing large parameter values.

The Problem: Overfitting

Complex models fit the training data perfectly but generalize poorly. Regularization adds a penalty term to the loss function:

\[J(\boldsymbol{\theta}) = MSE(\boldsymbol{\theta}) + \lambda \cdot R(\boldsymbol{\theta})\]

Where λ (lambda) controls regularization strength: higher λ means stronger regularization.

Ridge Regression (L2)

Penalize the sum of squared parameters:

\[R(\boldsymbol{\theta}) = \sum_{j=1}^{n} \theta_j^2\]

Effect: Shrinks all parameters proportionally.

Best for: When all features are informative.

Lasso Regression (L1)

Penalize the sum of absolute parameters:

\[R(\boldsymbol{\theta}) = \sum_{j=1}^{n} |\theta_j|\]

Effect: Shrinks unimportant features to exactly zero.

Best for: Feature selection, sparse solutions.

Elastic Net

Combines L1 and L2 penalties:

\[R(\boldsymbol{\theta}) = r \sum |\theta_j| + (1-r) \sum \theta_j^2\]

Effect: Balances ridge and lasso benefits.

Best for: When you want both effects.

Early Stopping

Stop training when validation error starts increasing, even if training error is still decreasing.

Effect: Implicit regularization without penalty term.

Best for: Iterative algorithms (SGD, neural networks).

Regularization Parameter Tuning

The regularization strength λ must be tuned via cross-validation:

  • λ too small: Little regularization, overfitting risk
  • λ too large: Too much regularization, underfitting risk
  • λ just right: Best generalization performance
from sklearn.linear_model import Ridge, Lasso, ElasticNet from sklearn.model_selection import GridSearchCV # Ridge regression with cross-validated parameter tuning param_grid = {'alpha': [0.001, 0.01, 0.1, 1, 10, 100]} ridge = GridSearchCV(Ridge(), param_grid, cv=5) ridge.fit(X, y) # Lasso (automatic feature selection) lasso = Lasso(alpha=0.1) lasso.fit(X, y) # Elastic Net (mix of L1 and L2) elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5) elastic_net.fit(X, y)

Logistic Regression

Transform linear regression for binary classification using the sigmoid function.

The Sigmoid Function

Logistic regression outputs probabilities instead of continuous values using the sigmoid:

\[\sigma(z) = \frac{1}{1 + e^{-z}} = \frac{1}{1 + e^{-\theta^T x}}\]

Properties:

  • Output range: (0, 1) — perfect for probabilities
  • Smooth S-curve: no sharp transitions
  • Differentiable: works with gradient descent

Decision Boundary

We predict class 1 when σ(z) > 0.5, which happens when:

\[\theta^T x > 0\]

This defines a linear decision boundary in feature space. The sigmoid maps distances to probabilities.

Sigmoid Function Visualizer

See how the sigmoid smoothly transforms linear output (z) into probabilities:

Left: Linear decision boundary in 2D space. Right: Sigmoid probability curve. Notice how z=0 maps to P(y=1)=0.5.

Cross-Entropy Loss

Logistic regression uses cross-entropy instead of MSE:

\[J(\boldsymbol{\theta}) = -\frac{1}{m} \sum_{i=1}^{m} [y^{(i)} \log(\hat{y}^{(i)}) + (1-y^{(i)}) \log(1-\hat{y}^{(i)})]\]

This loss heavily penalizes confident wrong predictions, which is desirable for classification.

from sklearn.linear_model import LogisticRegression from sklearn.metrics import classification_report, roc_auc_score # Train logistic regression classifier model = LogisticRegression(solver='lbfgs', max_iter=1000) model.fit(X_train, y_train) # Get probability predictions y_pred_proba = model.predict_proba(X_test)[:, 1] y_pred = model.predict(X_test) # Evaluate print(classification_report(y_test, y_pred)) print(f"ROC-AUC: {roc_auc_score(y_test, y_pred_proba):.4f}")

Softmax Regression

Extend binary logistic regression to multi-class classification problems.

Generalizing to Multiple Classes

For K classes, softmax computes a probability for each class:

\[P(y = k | x) = \frac{e^{\theta_k^T x}}{\sum_{j=1}^{K} e^{\theta_j^T x}}\]

Where:

  • We have K separate parameter vectors θ1, ..., θK
  • Probabilities sum to 1
  • Highest probability determines the predicted class

Cross-Entropy Loss for Multi-Class

We use categorical cross-entropy (one-hot encoded labels):

\[J(\boldsymbol{\theta}) = -\frac{1}{m} \sum_{i=1}^{m} \sum_{k=1}^{K} y_k^{(i)} \log(\hat{y}_k^{(i)})\]

Where y(i) is a one-hot vector (1 for true class, 0 elsewhere) and ŷ(i) are predicted probabilities.

One-vs-All (OvA)

Train K separate binary classifiers, each distinguishing one class from the rest.

Simple to implement, but doesn't model class relationships.

Softmax Regression

Train a single K-output classifier that models all classes jointly.

Preferred when classes are mutually exclusive.

from sklearn.linear_model import LogisticRegression from sklearn.metrics import confusion_matrix, accuracy_score # Softmax regression (multi-class logistic regression) # sklearn uses One-vs-Rest by default, but max_iter controls convergence model = LogisticRegression(multi_class='multinomial', solver='lbfgs', max_iter=1000, random_state=42) model.fit(X_train, y_train) # Predictions and probabilities y_pred = model.predict(X_test) y_pred_proba = model.predict_proba(X_test) # Evaluation accuracy = accuracy_score(y_test, y_pred) conf_matrix = confusion_matrix(y_test, y_pred) print(f"Accuracy: {accuracy:.4f}")

Concept Checker

Normal Equation:

  • Closed-form solution: θ = (XTX)-1XTy
  • Solves in one step
  • O(n³) complexity—slow for many features
  • No hyperparameters to tune
  • Works best when n < 10,000

Gradient Descent:

  • Iterative algorithm: update θ repeatedly
  • O(k·m·n) where k = iterations
  • Scales to millions of features
  • Requires tuning learning rate
  • Works for large datasets and streaming

When features have different scales (e.g., age in years vs income in thousands), the contours of the cost function become elongated. This causes:

  • Slow convergence: Gradient descent must take tiny steps to avoid overshooting
  • Poor numerical stability: Matrix operations become ill-conditioned
  • Divergence risk: Learning rate suitable for one feature may be too large for another

Solution: Standardize features to zero mean and unit variance:

\[x_{\text{scaled}} = \frac{x - \mu}{\sigma}\]

This makes the cost function more circular, enabling faster convergence with larger learning rates.

Test error can be decomposed into three components:

\[E[\text{Test Error}] = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}\]
  • Bias: Error from wrong assumptions (underfitting)
  • Variance: Sensitivity to training set fluctuations (overfitting)
  • Irreducible Error: Noise in the data itself

The Tradeoff: Simpler models have high bias but low variance. Complex models have low bias but high variance. The goal is finding the sweet spot that minimizes total error.

Ridge (L2): Use when you believe all features are useful.

  • Shrinks all weights proportionally
  • Never eliminates features (weights → 0 but never = 0)
  • More stable numerically

Lasso (L1): Use when you suspect many features are irrelevant.

  • Can shrink weights exactly to zero
  • Automatic feature selection
  • Produces sparse solutions

Elastic Net: Use when you're unsure or want a balanced approach.

  • Combines both L1 and L2 penalties
  • Often more robust than pure L1 or L2
  • Requires tuning an additional parameter (mixing ratio)
